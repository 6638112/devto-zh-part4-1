泛型从 C# 2.0 开始就已经存在，并且已经成为我们很自然地利用的工具，现在在 C#中，我们甚至几乎不去想它。所以让我们改变这一点...

### 定义

*泛型引入到。NET Framework 类型参数的概念，它使得设计类和方法成为可能，这些类和方法推迟一个或多个类型的规范，直到类或方法被客户端代码*声明和实例化。

本质上，泛型允许参数化类型和方法。正如普通方法有*参数*来告诉它们使用什么*值*，泛型类型和方法有*类型参数*来告诉它们使用什么*类型*。

### 澄清

类型参数是类型的占位符。

有*构造类型*和*泛型类型*。
构造类型是指指定了类型参数的类型。例如，

```
Dictionary<int, string>() 
```

泛型类型是指没有指定类型参数的类型。例如，

```
Dictionary<TKey, TValue>() 
```
