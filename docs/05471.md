如果 web api 可以被认为等同于 OOP 类，那么微服务可以被认为是一个维护坚实原则的类。

微服务不是魔术，而是一个设计良好的服务组件与其他组件一起服务于特定的职责。

就像类维护单一责任原则(SRP)有助于可维护性和可扩展性一样，微服务的模块化本质在不同规模的项目中产生了巨大的差异。
