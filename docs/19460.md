# 我对 REFACTR 的体验(棒极了)。TECH 2019

> 原文:[https://dev . to/eripdev/my-awesome-experience-with-ref actr-tech-2019-27id](https://dev.to/eripdev/my-awesome-experience-with-refactr-tech-2019-27id)

这个周末，我有幸参加了 REFACTR。TECH 2019 在亚特兰大举行。这是一个为期三天的会议，重点是在多元化和包容性的环境中提供技术内容。
从他们的网站，

> REFACTR。技术是关于发展和展示边缘人群和技术盟友的强大声音。三天，重复。技术专注于技术，同时围绕技术的多样性、包容性和交叉性创造一个安全的空间或深思熟虑和细致入微的对话。

根据我的经验，我认为他们肯定达到了目标！与会者来自各种生活方式、身份、技能水平和职业。作为一名崭露头角的初级开发人员，我总是乐于提问(和回答！)的问题，没有任何害怕嘲笑或感觉我不属于。这个周末我学到了很多，让我们来回顾一下，好吗？

## 第一天

第 1 天由各种全天研讨会组成，包括免费的 Vue Vixens 研讨会和 AMP 路演。Vue Vixens 是一项倡议，旨在创建研讨会，以一种酷而有趣的方式向那些认为自己是女性的狡猾的人教授 Vue.js。AMP 是 Accelerated Mobile Pages 的缩写，AMP 项目是一个开源项目，旨在通过在各种设备和平台上提供一致的快速、美观和高性能的网站，让每个人都喜欢 web。我没有参加这些会议，但我知道有人参加了，他们离开时变得更加明智，并带着新的信息去执行。

我参加了家得宝的布兰登·卡恩斯举办的盖茨比研讨会。起初，我很紧张，因为我只知道 React 的基本知识，我对 Gatsby 的唯一了解是它是一个“超快的静态站点生成器”。我的担心很快被抛在一边，因为 Brandon 确保没有人掉队，我们学会了如何启动和运行环境，在我们知道之前，我们正在创建我们的第一个 Gatsby 网站。通过这次研讨会，我了解到 Gatsby v =可以有更多的用途，当它与 GraphQL 结合使用时，可以成为一个非常强大的工具。我期待着今年晚些时候将我学到的知识融入我的个人网站。

你可以在这里找到工作坊和主持人[的完整描述。](http://refactr.tech/schedule/)

## [](#day-2)第二天

第 2 天由各种 45 分钟的讲座组成，分为前端工程、软件工程、产品/UX/用户界面、社会影响和社区、职业和领导力以及酷狗屎。最酷的部分是，你不必只致力于一个轨道，所以像我这样有各种兴趣的人可以尽可能多地参加你的日程安排。会谈的次数多得你都数不过来，要想全面了解，请点击这里。

我主要关注前端工程，并从 Mike Hartington 的关于渐进式网络应用(PWAs)的演讲开始。我总是听说这个短语，但从来没有真正的*知道*他们是什么，所以这很酷。接下来，我参加了 code daily . io . io .的创始人 [Tae'lur Alexis](https://twitter.com/TaelurAlexis) 的 React Accessibility 演讲

数十亿人使用互联网，每天都有新人有他们的第一次体验，作为开发者，我们必须确保每个人在访问我们的产品时都有相同的体验。可访问性不是基础知识的一部分，今后我会保证制作可访问的产品，并向其他人教授/介绍其重要性。

第三天结束时，我的脑子里充满了信息，但我准备好了更多的信息(和小睡)。

## [](#day-3)第三天

第三天包括更多的主题演讲和分组会议。

考虑到现在有数十亿人可以访问网络，而且每天都有新的人访问网络，您是否知道互联网用户对移动设备/平板电脑的使用已经超过了台式机？亨利·Helvetica 的《API 星球》解决了这个问题，并专注于我们可以用来测量和提高应用程序资源加载和页面渲染效率的 API，这非常重要，因为我们希望人们在不同的网络和设备之间拥有快速的移动用户体验。

我知道 VSCode 很棒，但在第 3 天，我发现我在 Visual Studio Code Sparks Joy 的研讨会上肯定没有充分利用它。我确实获得了一些扩展，比如更漂亮和有用的提示和技巧，我将把它们合并到我的工作流程中。更聪明地工作，而不是更努力地工作？

我还没有完全准备好开始面试，但是多亏了 Salesloft UI 工程师 Adrianna Valdivia 的 React 开发者技术面试指南。如果你等到开始申请工作时才开始面试准备，那就太晚了。尽早开始，将常见的面试项目纳入你的学习路径，这样你就可以真正熟悉了。

## 摘要

总之，我在 REFACTR 上玩得很开心。TECH 2019。我学到了很多东西，遇到了很多了不起的人。如果你参加了今年的会议，我很想听听你的经历，如果你没有参加但感兴趣，我强烈建议你参加明年 4 月 22 日至 24 日的会议。
