# 试图把果冻钉在树上

> 原文:[https://dev . to/naked powershell/try-to-naked-naked-jelly-to-the-tree-227 c](https://dev.to/nakedpowershell/trying-to-nail-jelly-to-the-tree-227c)

一个**移动的**目标会让你发疯吗？

当一个开发者工具每个月换**的时候，你会不会迷路？**

什么时候太多的进步才算太多？

让我告诉你我在过去六个月中使用 Power BI 的用户体验。

我称之为**搅动**。

每个月微软都会发布一个新版本的 Power BI。

我觉得很好。

但是等一下...

我不能依靠我的旧知识来了解这个工具是如何工作的，我必须每个月重新学习一次。

他们制作了非常好的视频，并发布了关于所有变化的说明。

但是功能、菜单和用户可点击区域的改变，感觉就像我每个月都要从头开始学习一样。有点像“土拨鼠日”重演。

让**更仔细地检查一下**这种情况。

[![Alt Text](img/aeeffa61e3f618cf83b8f0130bd44c77.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--JHLHok73--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/hzgev0ec325qx6fj072h.jpg)

我同意通过为一个工具创造新的特性或增强功能、修正错误来取得进展。

但是代价是什么呢？

所有这些**在我周六去 Redmond 参加 SQL 时达到了顶点**,我挑选的一个曲目是关于 Power BI 中的高级特性。

主讲人是一位著名的专家、作家、博客作者和关于 Power BI 的讲师。

他用 PowerPoint 做了一个简短的介绍，然后就没有幻灯片了。

**演示**时间！

因此，演示者继续**尝试**现场演示如何制作一些非常漂亮的图表和仪表盘。

我说试试。

每次他试图找到一个特定的控制或按钮或诸如此类的东西，他都失败了。

他变得非常沮丧。

您客户办公室的超级用户应该做什么？他们不可能每个月都花时间重新学习这个工具，他们有真正的工作要做。

**自助 BI** 到此为止。

演示者最后一次练习演示是在 SQL 周六会议前的几周前。

在此期间，微软发布了 Power BI 的新版本。

如果专家努力，我们这些凡人还有什么机会？

观众中有几个人刚刚使用了最新版本，每当主持人遇到困难时，**就大声喊**帮忙。

搅乳器抓住了他。

他好像戴着护目镜。

我听到的反驳

我向另一个开发者提到了我对客户流失的挫败感，他的反应是，哦，好吧。

这不是答案。

你对搅乳器有什么看法？

微软应该怎样做**来帮助**开发人员每月跟上进度？

作为开发人员，我们如何将每月的新知识传递给我们的客户呢？

我留给你这个:

他们说，“早起的鸟儿有虫吃”。我说看看虫子早起后发生了什么:}
