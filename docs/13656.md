# PWAs 的下一步是什么？

> 原文:[https://dev.to/stefandorresteijn/what-s-next-for-pwas-942](https://dev.to/stefandorresteijn/what-s-next-for-pwas-942)

渐进式网络应用已经风靡一时。Instagram、Twitter、Forbes 和阿里巴巴等公司投入大量精力，让它们的移动网站看起来像本地应用。然而，有很多理由不把你的下一个移动应用变成一个进步的网络应用。主屏幕的*添加功能*用户体验感觉不够流畅，我们不能忽视苹果在 Safari 中实现 PWA 功能的蜗牛速度。那我们去哪里？渐进式网络应用的下一步是什么？请允许我推测。

## [](#all-native-functionality)所有原生功能

pwa 目前最大的问题是缺乏它们在 iOS 上支持的原生功能。推送通知和完整的传感器信息只是 iOS 版 Safari 仍然缺少的两项主要功能，但它们的清单仍然很长。我们已经看到谷歌大幅改善了他们的支持，Android 上的 Chrome 现在允许几乎所有你可能想要的原生功能。因此，苹果在 iOS 上对 Safari 的原生功能提供全面支持似乎是显而易见的下一步，但由于他们在 WWDC 期间根本没有提到 PWA，人们开始怀疑他们是否真的有兴趣进入 PWA 市场。

苹果公司*不*改善他们的 PWA 支持有很多原因。网络应用没有向苹果支付从本地应用获得的 30%的“应用商店税”，它们没有向苹果提供安装或使用数据，也没有增加应用商店中的应用数量。然而，随着开发者兴趣的增加，苹果实际上已经在 PWA 支持方面取得了很大的进步，它似乎正在走上 Web 开发者的道路。我相信他们会在今年年底添加推送通知，适当的传感器信息可用性可能会在 2020 年添加。就原生支持而言，Google 没有太多需要改进的地方，所以可以肯定他们会继续改善 PWA 开发者的生活。

## [](#store-discovery)店铺发现

渐进式网络应用程序的第二个问题是，你无法在 App Store 或 Play Store 中找到它们作为可本机安装的应用程序。

在 Android 上，pwa 实际上是作为 [WebAPKs](https://developers.google.com/web/fundamentals/integration/webapks) 安装的，因此作为本机应用程序存在于您的手机上。随着他们为托管的 Google Play 引入了[“Google Play iFrame”](https://www.blog.google/products/android-enterprise/managed-google-play-iframe/)(我知道，多好的一个名字)，谷歌已经有效地迈出了第一步，允许 pwa 直接在 Play Store 上发布，这意味着你不再需要在 Cordova 中作为 WebView/运行它们，或者作为 [TWA](https://developers.google.com/web/updates/2019/02/using-twa) 部署它们。

在 iOS 上，恐怕我们没有这样的运气，但这种类型的支持在不久的将来也不是完全不可能。看，如果你需要从 App Store 安装一个 PWA，你最终会再次向苹果支付发行费用，并向他们提供安装和使用数据。这可能是在 iOS 上同时添加应用商店发现和原生功能支持的途径。

商店优先并不是渐进式网络应用的固有特性。PWA 的整个想法是，它可以从你的浏览器安装，而不需要使用商店。然而，由于苹果需要控制其产品上的应用程序，几乎不可能看到 pwa 拥有所有原生功能但不在 App Store 中列出的未来。很难说苹果和谷歌什么时候会允许 PWAs 出现在 app store 中，但我相信这是迟早的事。

## [](#add-to-homescreen)添加到主屏幕

正如我前面提到的，“添加到主屏幕”的体验严重缺乏。在桌面上，很难找到，在移动上，如果你不定制它，它看起来很可怕。“添加到主屏幕”这一行对公众来说也没有任何意义。他们习惯于在手机上安装一些东西，而不是在主屏幕上添加一些东西。我估计，一旦 pwa 被添加到各自的应用商店，这种措辞就会改变，并希望看到它变成“将*应用名称*安装到您的设备上？”

覆盖“添加到主屏幕”弹出窗口的过程实际上一点也不困难，建立一个自定义提醒是一个很好的方式，可以让你的用户更清楚当你点击 *yes* 时会发生什么。这就是为什么我不认为这个问题是一个特别优先的问题，所以我希望当谷歌或苹果决定给 PWAs 一点重新命名时，这种命名会改变，可能是当苹果决定将其作为他们发明的全新产品推出时；也许是*闪电应用*。

## [](#in-closing)在关闭

在过去的几年里，渐进式 Web 应用程序走过了一段令人难以置信的漫长道路。他们从这个小众功能发展到现在被世界上最大的公司所使用。一次又一次地，pwa 被证明对于构建无需学习 Swift 或 Java 就能感觉很自然的应用程序非常有用。我相信 PWAs 的未来是光明的，苹果和谷歌都全力支持他们，并开始将他们视为一等公民。我真心希望苹果不要用“PWA”这个术语，并把它重新包装成他们自己的东西，但只要这意味着我们可以在网络上获得原生功能，我就不在乎了。
