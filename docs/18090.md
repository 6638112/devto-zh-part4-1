# 让你成为初级开发人员的 5 个技巧(来自资深开发人员)

> 原文:[https://dev . to/Jame esy/5-tips-from-a-senior-developer-to-get-you-hired-as-junior-420 p](https://dev.to/jameesy/5-tips-from-a-senior-developer-to-get-you-hired-as-a-junior-420p)

这是我作为首席开发人员的建议。我认为这五件事是求职者在求职过程中脱颖而出的原因。

## [](#online-presence)💻在线状态

花时间通过 Instagram、Dev.to、LinkedIn、Twitter 或博客来写博客、记录和写下你的编码经历，这显示了你的奉献精神和真正的热情。更不用说，处于招聘职位的人可能会试图寻找申请人。我们都做到了！

公司通常希望所有员工都承担一定程度的倡导责任，无论是发布公司的最新消息还是撰写博客文章。在这种情况下，证明你有围绕开发创建内容的技能可能是一大优势。

拥有一个好的社交账户可能看起来微不足道，但会帮助你脱颖而出，让你有机会间接分享更多关于你自己的细节。

## [](#%E2%9A%92-open-source-experience)⚒开源体验

开源体验不仅展示了您编写代码的能力，还展示了您使用标准 Git 实践(如问题、分支、拉请求和合并)处理大型代码库的能力。

这些都是日常工作环境中至关重要的东西。更不用说它在你的 GitHub 账户上看起来真的很棒，如果你能展示你做过的大型项目的话！

对于希望开始接触开源的人来说，最好的项目之一就是 Gatsby。它拥有一个真正友好的环境，更不用说贡献的免费赠品了！！！

## [](#honesty)🌱诚实

如果你撒谎或试图夸大自己，人们会一眼看穿。对你的经历诚实，对你的智力诚实，对你想去的地方诚实。如果你试图证明自己比实际情况更有经验，那么你就有可能在面试中犯错，这将会毁掉你的机会。

你必须记住，拥抱你的弱点并从错误中学习是一种力量。我承认，有很多事情我不明白，但是，我不会试图欺骗自己。如果我们在 scrum 中讨论一个特性，而我不确定如何实现它，我会说“我不知道，但是，我会找到的”。这种诚实得到了各方的赞赏，因为它避免了你什么都不说，一个星期下来，这就成了一个问题。

## [](#a-wellwritten-resume-portfolio)📝写得很好的简历/作品集

保持你的简历和作品集简洁明了，包含所有相关信息。

我使用这个场景:一天中，我工作的公司可能有 10-20 个以上的各种角色的申请。设身处地为人才经理着想。当他们浏览候选人的申请时，他们想要一些可以得到相关信息的东西。

## [](#past-experiences)👨‍🍳过去的经历

人们太容易忽视他们以前的非编码工作，认为它们对申请过程有帮助。这些过去的工作通常非常有用，因为它们给了你一些经验和技能，而这些是那些刚从大学毕业的人还没有学到的。为你的过去感到骄傲，并利用它。

如果你在当地商店工作过，那么你有一定程度的面对客户的能力，也就是说你没有和客户打过交道。如果你从事过体力劳动工作(在转到 dev 之前，我是一名建筑工人)，那么你就有勤奋工作的精神。

我希望这些建议能对你们有所帮助，我完全理解申请工作的过程有多艰难！<3
