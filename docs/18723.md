# SDK 和 API 有什么区别？

> 原文:[https://dev . to/alexsklv/what-s-the-difference-SDK-and-API-22b 6](https://dev.to/alexsklv/what-s-the-difference-between-sdk-and-api-22b6)

你好。我叫亚历克斯·索科洛夫。我是 iTechArt 集团的高级工程经理。在我的工作经历中，我曾无数次面临在 SDK 和 API 之间做出选择的挑战。我决定写这篇文章来澄清 SDK 和 API 之间的区别，因为人们经常对定义这两个概念感到困惑。

**那么，什么是 SDK 呢？**

SDK(软件开发工具包)是一组开发工具，如库、各种实用程序(CLI 和 GUI)、文档、代码示例等。帮助开发人员为特定平台构建应用程序。SDK 的主要组成部分是可以集成一个或多个平台 API 的库，通常包含可配置和可重用的组件。不同的 SDK 实用程序(如果 SDK 中有)可以通过自动化开发过程来进一步简化开发人员的工作。

什么是 API？

API(应用程序编程接口)是为不同应用程序、程序或平台之间的交互而设计的接口。与其说它是一套创建新解决方案的工具，不如说它是一套应用间交流的方法。应用程序之间的通信由一组指定数据结构、例程、协议等的规则来执行。作为一套应用程序开发工具，SDK 通常包括一个或多个 API，以及代码示例、库和其他组件。SDK 和 API 的区别可以总结为:“所有的 SDK 都包含 API，但不是所有的 API 都是 SDK 的一部分。”

**什么时候使用 SDK 和 API？**

开发人员在构建应用程序时总是使用 SDK。而且还有专门为某个平台设计的 SDK，比如 Android SDK 或者 Azure SDK。

API 用于通过各种方法和协议与外部应用和/或平台进行通信。

例如，要将. NET 应用程序与特定的 Azure 服务(例如，文件存储)集成，可以使用他们的 Web API(通过使用对某个 URL 的 HTTP 请求)或。NET 库(这更容易，因为不需要处理直接的 HTTP 请求。)

总之，在开发任何类型的软件应用程序时，工程师都会使用 SDK。当需要与其他第三方服务提供商(应用程序/平台)通信时，将使用 API 集成，这有时可以是 SDK 的一部分。
