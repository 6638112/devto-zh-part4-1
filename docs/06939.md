多年来，Solace 一直在努力帮助使用 Spring 框架的开发者用 Solace 事件代理创建基于 Spring 的应用和事件驱动的微服务。我们发布了一个全面的[集成指南](https://docs.solace.com/Developer-Tools/Integration-Guides/Spring-Framework.htm),展示了如何将 JMS 与 Spring Java 框架集成，以生产和使用 JMS 消息。我们还发布了开源项目，如 [Spring Boot 自动配置](https://github.com/SolaceProducts/solace-jms-spring-boot)，完成了用于 Solace JMS API 的 Spring Boot 启动器和用于与 Solace PubSub+集成的 Spring 云流绑定器的[实现。](https://github.com/SolaceProducts/spring-cloud-stream-binder-solace)

我很自豪地宣布我们正在提供的一些新资源，以及一个新的登录页面，其中将包含所有这些资产，并帮助 Spring 开发人员了解:

*   **PubSub+可以使用哪些** Spring 项目
*   **为什么** Spring 生态系统可以受益于 Solace PubSub+
*   如何将 PubSub+与受支持的 Spring 项目结合使用

至于我上面提到的新资源…

## [](#spring-cloud-streams-with-solace-tutorial)春云流伴慰藉教程

除了向您介绍使用 Spring Cloud Streams 和 Solace PubSub+ Binder 的基础知识之外，本实用教程[还概述了一种在 Solace 中创建消息服务以快速访问 Solace 消息的简单方法。它向您展示了如何创建 3 个应用程序:一个 Spring Cloud Streams 源(发送应用程序)，一个 Spring Cloud Streams 接收器(接收应用程序)，以及一个 Spring Cloud Streams 处理器(发送&接收应用程序)。](https://solace.com/samples/solace-samples-spring/spring-cloud-streams/)

查看 GitHub 中的[教程](https://solace.com/samples/solace-samples-spring/spring-cloud-streams/)和[示例代码](https://github.com/SolaceSamples/solace-samples-spring)。

## [](#spring-boot-autoconfig-jms-tutorial)Spring Boot 自动配置(JMS)教程

本教程是关于使用带有自动配置的 Spring Boot 将 JMS 客户机连接到 Solace 消息传递，以交换 JMS 事件。它将引导您逐步创建一个用于接收 JMS 消息的应用程序和一个用于发送 JMS 消息的应用程序。

查看 GitHub 中的[教程](https://solace.com/samples/solace-samples-spring/spring-boot-autoconfigure/)和[示例代码](https://github.com/SolaceSamples/solace-samples-spring)。
