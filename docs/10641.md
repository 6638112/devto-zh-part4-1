### [](#why-cosmos-db)为什么是 Cosmos DB？

数据的分发可能很困难，拥有一个针对全局 NOSQL 数据的统包解决方案是巨大的，但是随后添加使用已经熟悉的工具查询这些数据的能力？这就是 Cosmos DB 庞大的原因。与其花时间考虑自己的内部数据库解决方案(SQL、Oracle，无论是什么)的长期可行性，不如转移到像 Cosmos DB 这样的托管服务。

以下是 Azure Cosmos DB 的一些属性，这些属性使其非常适合具有全球抱负的高性能应用程序。(点击此处查看全文...)

*   Azure Cosmos DB 对您的数据进行本地分区，以实现高可用性和可伸缩性。Azure Cosmos DB 为所有单区域帐户和所有具有宽松一致性的多区域帐户提供 99.99%的可用性、吞吐量、低延迟和一致性保证，并为所有多区域数据库帐户提供 99.999%的读取可用性。
*   Azure Cosmos DB 拥有 SSD 支持的存储，具有低延迟毫秒级响应时间。
*   Azure Cosmos DB 对一致性级别的支持，如最终、一致前缀、会话和有限过时，允许完全的灵活性和低性价比。没有任何数据库服务能像 Azure Cosmos DB 一样在一致性级别上提供如此大的灵活性。
*   Azure Cosmos DB 有一个灵活的数据友好的定价模型，可以独立地计量存储和吞吐量。
*   Azure Cosmos DB 的预留吞吐量模型允许您从读取/写入数量的角度来考虑，而不是底层硬件的 CPU/内存/IOPs。
*   Azure Cosmos DB 的设计让您可以扩展到每天数万亿次的巨大请求量。

这些属性在 web、移动、游戏和物联网应用中非常有用，这些应用需要较短的响应时间，并且需要处理大量的读写操作。

您将学习如何启动您的数据库，通过 Node.js 应用程序访问它，并了解一些使 Cosmos 对开发人员如此重要的简单特性。
