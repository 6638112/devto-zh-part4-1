# 日志文件的重要性

你什么都试过了。有各种各样的 console.log 消息设置。Try-catch 语句无处不在。您已经浏览了文档，搜索了 2009 年的论坛，但仍然一无所获。这不是一个普通的错误。这是一个高级错误，来自一个隐藏在框架层后面的匿名异步函数。

您得到的唯一错误是调用有问题。没有，谷歌搜索已经强大到足以回答你的问题，而你团队中的其他开发人员也从未见过这一点。你在这里只能靠自己了。直到有人提到查日志。

## 什么是日志

系统会自动创建日志文件来存储应用程序中所有事件的记录。您使用的几乎所有东西都会创建或添加到日志文件中。从你电脑上运行的操作系统到你手机上的应用程序，它们都会生成日志文件。它们记录了您通常不会在自己的错误消息中跟踪的事情，比如导致错误的特定数据库列。

它跟踪从开始运行应用程序到停止运行应用程序的每一个事件。您对第三方 API 的任何调用或在后台运行的任何脚本都将在此处有记录。这是您找到应用程序幕后发生的一切的来源。

## 你为什么需要它们

我们需要日志的原因是因为它们保存了在其他地方找不到的信息。例如，假设您向一个域类添加了一些东西，但是您忘记了运行迁移。这可能会引起一些问题。您不希望告诉用户，由于很多原因，您缺少表中的某一列。

像这样的错误将被记录在日志中，这样只有有权访问服务器的人才能看到这些错误。大多数情况下，当您在几个小时的调试后找不到代码的问题时，这是您应该查看的地方。答案可能不总是在这里，但它会给你另一个地方去检查。

一旦当你有奇怪的错误时开始查看日志文件，找到修复它们的方法就变得容易了。至少，你会排除另一个地方看。很多新开发人员不了解日志，所以我们花时间教他们很重要，这样他们就可以学习如何更好地研究 bug。

## 关于它们的提示

当你打开一个日志文件时，它看起来很吓人。它有成千上万的行，看起来非常技术性和机械化。日志中的大部分内容对你来说都无关紧要。你只需要知道要找什么。大多数情况下，您会检查日志中的错误，偶尔会出现警告。

通常，当你打开日志时，你会想要搜索“错误”这个词。当您这样做时，请注意错误旁边的时间戳。您希望确保您看到的是记录的最新错误。此外，确保您在项目开始时配置了日志。有些东西会被自动记录，但是如果你想找出具体的错误，你必须做一些自定义的配置。

日志是这些神奇的文件，可以跟踪我们甚至没有想到的错误。不要担心它们看起来有多脏。只需搜索错误，然后你可能会发现一些新的谷歌。还有人对日志有什么建议吗？
