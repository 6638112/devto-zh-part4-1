并非所有的公司都是恶意的，正如并非所有的程序员都是跳槽者，并非所有 40 岁以上的工程师都被困在 COBOL 中，也并非所有的博士学位持有者都会在无人监督的情况下设计 Rube Goldberg 机器。对于软件工程中的子行业，如医疗设备、银行、保险等，也有一些要说的，在这些子行业中，许多负面的刻板印象被*颠倒了*，但是在最初的文章中提到的那些是你今天会听到的最普遍的一些。

#### 关于不在同一家公司任职超过 2 年

在同一个地方工作很长时间没问题，但我认为这条特别的坏建议的原因是，如果你在一个地方工作很长时间*和*那个地方已经形成了一种特定的做事方式，那么它就限制了你接触的方法的多样性。

在谈到这一点时，我不能不提到，跳槽更容易获得加薪...但我也不认为金钱是任何一家公司的全部。

你也可以考虑一下你是否已经在公司竭尽所能了。我的一个朋友用了这个汽车修理工的比喻:

> 你是商店里的前灯工人，你在一个特定的车库里为一辆特定的汽车工作。你给商店买了所有的设备，你做了很多工作。那辆车的前灯可能已经优化到你在商店里能买到的程度了。可能是时候去一家从未有过车灯工人的商店了，让一个对散热器感兴趣的人从你身后进来，推动项目/团队前进。

这些都是几年后离开的正当理由，但如果那家店每天都有新车进店，钱不是你的主要动力(这些人确实存在)，你会获得各种各样的体验，你会觉得你在以一种有价值的方式扩展你的技能，那么就没有理由把它作为一项要求。

我认为对一个年轻工程师来说，一个好的模式是每隔几年换一次工作。当你对自己的专业感到非常满意后，要么开始寻找新的东西来学习和专攻，要么开始寻找一个长期的职位。你可能永远也找不到它，但总有一天，新工作的旋转门开始变得越来越薄。

如果读这篇文章的人没有这样做，这肯定不是职业杀手，以此为基础取消某人的资格没有任何真正的商业意义。新员工*很贵*。有人长期与你在一起对公司来说是一个巨大的好处，但请确保你推动你的工作场所不断寻找更好的解决方案，不要陷入“我们一直以来的做法”的陷阱。

#### 论激情...两害相权取其轻

当谈到“激情”(Github，当你学会编程时，你在业余时间编程了多少)这个总话题时，我发现当面对一家真正相信这一点的公司或招聘经理时，有两种可能性，这两种可能性都不会让我想为他们工作。

要么是因为他们“对这个问题如此感兴趣”，所以他们正在寻找一个愿意在下班后工作而没有额外报酬的人，要么是他们没有真正考虑过这种询问的真正含义。这只是鹦鹉学舌，因为这是一个“正确”的问题。两者都不是很吸引人的样子，作为受访者，这种方法对我来说是一个标志，我需要开始寻找其他危险信号(尽管，公平地说，对各地的面试官来说，它本身并不是不合格的)。

具体到 Github，我的第一反应是“在 Github 存在之前，我正和妻子、儿子一起写代码，还要割草，我的工作中没有一项要求我提交开源回购”，我在采访中从未遇到过这个问题。对于一个年轻人来说，这可以很容易地修改为“我从未真正从事过开源工作，所以我没有在 Github 上构建很多东西。”如果你真的厚脸皮，试着加上一句“我们会在这个团队中开发开源软件吗？”最后，如果答案不是太明显的话！

#### 论年龄歧视...两者中的较大者...等待...他们都是邪恶的，好吗？

年龄歧视是受害者面临的另一个现实问题，甚至不是犯罪者想让你相信的实际问题。更多的是公司能促使你以比你的年龄更少的钱给他们更多的工作。

虽然我遇到过一些年长的工程师，他们在很多年前就学会了一种编程方式，不想学习更多(这是年龄歧视者用来证明他们为什么需要年轻工程师的理由，如果他们甚至承认这一点，而不是出于法律原因躲在“资历过高”这样的短语后面)，但我遇到过更多的人，他们仍然保持着最新的技能，拥有几十年的经验，这对任何团队来说都是一笔财富。

不幸的是(根据从事这项工作的公司的说法)，那些“资历过高”的工程师也在工作之外建立了生活和兴趣(前面提到的“伴侣/孩子/草坪/在此插入爱好”问题)，他们更有可能在获得报酬的工作时间工作，在空闲时间过自己的生活。使用这种逻辑的雇主将这比作一个刚从大学毕业的新毕业生，他有更少的家庭责任，更多的自由时间，渴望“证明”自己，并选择了更(看似)有利可图的选择，忽视了经验带来的好处，而倾向于更直接的感知成本/收益。

#### 两者的结合(两种不好的味道加在一起味道很可怕)

这位雇主认为，如果一个人在工作之外的责任更少，并且员工认为他们“欠”了“给他们机会”的公司额外的时间，那么几周的紧张时间就会遇到更少的抱怨。这对年轻的工程师非常不公平。这让他们明白，这只是这个行业的“现状”，他们的空闲时间并不重要。你必须有休息时间，你必须有时间去建立一种生活，一旦你到了“某个年龄”，一个更邪恶的公司会讨厌你拥有这种生活。

时刻处于紧要关头是组织的失败。而不是你或你周围人的技能。

这也是我不得不提到的一个个人的小毛病。带回家的编码测试。我把这个放在两者的组合下，因为虽然我确实感觉它不成比例地影响了年长的员工(又是“伴侣/孩子/家务”问题)，但它既是对你能力的测试，也是对你将花多少空闲时间在工作任务上的测试。

我吃得不好，我拒绝再吃了。

那跟我作为工程师的能力关系不大，跟[这个](https://heeris.id.au/2013/this-is-why-you-shouldnt-interrupt-a-programmer/)关系更大(“你刚才黑了我”在我们家是个笑话)。

我的“空闲时间”并不真正属于我，而且十多年来都不是。这是我妻子的。这是我儿子的。如果你没有对等的关系，你的空闲时间属于你的漫画书，或者你的爬山，或者你的志愿活动，或者你的电子游戏。你的空闲时间不属于一家公司，这家公司甚至不会为你的“两小时”测试提供补偿，这实际上占用了你两天的空闲时间。

根据一些消息来源，我是最老的千禧一代之一。或者最年轻的 X 世代。或者俄勒冈小道一代...没关系，有了这胡子，我以前被问过是否有过孙子，我感觉自己就像很多商店里的“老人”(这甚至出现在我的个人资料里)。“激情”——偏见和年龄歧视还没有开始影响我们大多数人...但是很快我们中的许多人将开始在工作场所面对它，Z 世代(他们是比任何其他世代的[更大的群体，并且开始从大学毕业)将称我们为老一代。](https://www.statista.com/statistics/797321/us-population-by-generation/)

#### 充分披露...我从未修完学士学位的最后 12 个学分

最后，反学术运动。这个有很多根源。同样，这里面的真理种子根植于一些负面的经验，并利用了一些现存的偏见。原因是，虽然我与一些拥有研究生学位的伟大程序员一起工作过，但我也遇到过(非常)少数更关心非常简单的问题的极其复杂(有人可能会说复杂，甚至[偶然复杂](https://en.wikipedia.org/wiki/No_Silver_Bullet))的解决方案的人。在这种情况下，这个小群体中的许多人实际上没有花写论文那么多时间写代码，这显示了他们的能力。这不是全部，甚至不是大多数的学术界人士(我目前和一些受过高等教育的工程师一起工作，他们都很棒，这篇文章是受一个博士的启发写的！)，但这种偏见就是从这里来的。

在商业方面，更高的资历意味着更高的报酬。绝大多数的编程工作并不涉及优化编译器、处理器中的编程门，也不需要详细了解数据包是如何通过网络发送的。一般都是代码管道。管道工程是一个非常艰难的领域，它需要大量的技能和知识，比如建筑规范、地方法规、关于系统中流体动力学如何工作的技术知识，以及无数其他我不会假装知道的东西，但这是一个需要技能的行业，并且这种技能会得到很好的补偿。这种技能来自多年实践的实际经验，而不是通过学习高级流体动力学来理解真空中的超新星如何传播或大气如何移动以创造各种天气模式。作为一家企业，我不会雇佣一个专攻流体力学的物理学博士来建造我的大楼，同样的情况也可以说是实现一个代码库。

如果那个博士学位持有人写了关于管道的论文，花了职业生涯设计和实施管道，并在该领域有明显的技能，这也绝对不是一个取消资格的标准。如果我的问题的内在复杂性需要有博士学位的人来解决，那么我需要有博士学位的人。美国宇航局不能雇佣水管工来做他们的工作。在这种情况下，这是你团队中一个有吸引力和有益的特质，值得为高级学位支付的每一分钱。但是很少有公司是 NASA。

#### 这对现实世界(不是谷歌)有什么影响？

大多数偏见都是为了不惜一切代价为公司获取最大利益。不幸的是，这些想法是从与大公司的面谈和经历中播种到社区中的，而货物崇拜的追随者想要效仿它们。这个社区包括招聘经理，他们希望将他们的工程团队建设成一个类似谷歌的发电站(但只有五个人)。

这些大公司有比他们能雇佣的更多的申请人，并且能够挑选雇佣和不雇佣谁。它们也是一个非常大的就业来源，但在寻求招聘工程师的公司总数中所占比例非常小。试图在初创公司或小公司模仿他们会不自然地限制你的招聘范围，而这些正是我想去工作的地方(所以如果这些地方不那么歧视的话，对我来说很好)。

全球范围内，工程师比工作岗位少，某些市场急需人才，而其他市场则被淹没。如果你有良好的面试和社交技巧，加上良好的技术能力，那么现在没有人是不可解雇的。也许你不会去谷歌...但是你也许可以创造一些伟大的东西。
