# 用户界面，逻辑，重构

> 原文：<https://dev.to/oieeaaaa/ui-logic-refactor-50lh>

在我开始创建新的 react 组件之前。我首先用 mongol 3 铅笔在我的黄色平板上画出 UI，然后用 HTML & CSS 把它翻译成代码。因此，我可以看到我试图构建的概述...也许这是我的组件的蓝图🤔。然后，我使用 React 中的状态和内容将逻辑添加到 UI 中。然后我将删除这个新的组件文件。是的删除它，嗯嗯。你知道为什么吗？我发现已经有一个来自 npm 现有包等待安装在我的代码库上。

节省了我的时间。是啊！！

我从中得到的教训是，我永远不应该重新发明轮子，因为你的代码库中最好的代码不是你的代码，而是其他人的代码，我会珍惜这一点，不要重新发明他们的轮子，而是对它进行逆向工程，从中学习，也可能从中构建一些新的东西。因此，在我开始自己做事情之前。我必须首先知道从哪里开始。

萨拉马特。
