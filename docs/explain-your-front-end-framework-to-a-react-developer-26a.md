# 向 React 开发人员解释您的前端框架

> 原文：<https://dev.to/stereobooster/explain-your-front-end-framework-to-a-react-developer-26a>

嗨，我是 React 开发者。我不是 React 的粉丝。我不认为我们需要在浏览器中内置反应。我不认为你需要为登陆页面使用 React(几乎没有任何交互)。第一次看到 JSX 时，我想“哦，又是 XML”。我每天都和 React 一起工作(最近；之前是 Ruby)。我希望我多少有点开明。

考虑到这一点，**试着解释为什么我想尝试另一个前端框架**。有哪些甜蜜点？你喜欢你的框架的什么？

PS。很久以前，我尝试过 Angular 和 Ember 等不同的框架，所以我的理解很可能已经过时了。
