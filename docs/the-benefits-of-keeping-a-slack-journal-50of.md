# 写呆滞日志的好处

> 原文：<https://dev.to/x-team/the-benefits-of-keeping-a-slack-journal-50of>

“你这些肌肉是从哪里来的？”你的朋友会问，就好像你从街角的卖肌肉的人那里花了 0.42 比特币买了这些二头肌一样。仿佛这不是在健身房里几个月的奉献，数百个 burpees，数千个哑铃弯举💪。

或者“你是如何变得如此擅长编码的？”，就好像你的头撞到了门上，它激活了你大脑的 C++部分。就好像它不是花费在堆栈溢出上的时间，寻找丢失的逗号的夜晚，以及令人沮丧的错误挤压🐛。

有意义的改变是一个过程。它从来不会突然发生。

但是如果你不记录你每天投入的小工作，你将会失去你进步的轨迹。你会变得消极，因为你不能回头看你已经走了多远。你会不知不觉地停止做那些让你成功、让你快乐的事情。

这就是为什么 X-Team 杂志的每个人📕。这已经改变了游戏规则。你看，我们不按老方法写日志。我们想看看是否有日志 2.0。

事实证明是有的。标题暴露了这一点，但是 Slack 已经证明是一种很好的记录方式。这是如何工作的...

除了定期写日志给健康带来的令人惊讶的好处之外，在空闲时间写日志是创建一个相互信任的社区的关键因素。

它是这样工作的。首先，你有“今天我学会了”(TIL)笔记。它们是分散在一天中的小学习时刻，如果你不在 Slack 上快速记录它们，那些金块就会消失。这些笔记快速而原始，是来自战场的报告。

其次，你有“反思”笔记。你胜利地坐在你征服的山顶上🗻喝着瓷杯里的茶，看着战场。你想想那些顺利的事情和那些不顺利的事情。但最重要的是，你用语言表达出为什么你的一天会这样。

例如，有时你会本能地觉得一段代码不太对劲。不要让那些时刻溜走！写下你为什么会有这种感觉。这是一种强有力的学习方式。

我们大部分的 Slack 日志都是公开的，可以让 X-Teamers 加入。而且所有的 Slack 日志都至少加了两个人。它大大减少了对耗时会议的需求，因为相关人员总是与时俱进。他们知道你在做什么。

这也是一条双行道。如果你被卡住了，鼓励你把它写在你的日记里。会有人跳出来帮忙的。工作完成得更快，因为你的同事可以分享他们的相关经验，并为你指出正确的方向。

在 Slack 上写日志是远程公司的 2.0 生存方式。它增加信任，减少压力，帮助更快地完成工作。把所有这些日志结合在一起，你就有了一个名副其实的知识宝库。
