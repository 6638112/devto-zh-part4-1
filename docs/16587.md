# 什么是 A11y，为什么我们要关注它

> 原文：<https://dev.to/eripdev/intro-to-accessibility-31e5>

你有没有想过，你对一个主题已经有了很好的理解，开始四处挖掘，几乎没有触及表面，然后意识到当谈到手头的主题时，你不知道自己在干什么？这不是“我也不是”的情况。这就是我。高举我的手。我很惭愧地承认，作为一名有抱负的前端开发人员，我真的不太了解可访问性。平心而论，知道的人不够多。但是没关系，因为从今天开始，我们将一起克服它，变得更好。我的目标是从可访问性的高层次概述开始，浏览一些构建我的作品集的代码挑战和教程，并放下一些(希望)有用的资源，您可以开始使用它们来使网络对所有人都棒极了。听起来不错吧？很好。

## 好吧，那么什么是可达性呢？

根据 work 的说法，简而言之，网络是为每个人设计的，不管硬件、软件、语言、地点或能力如何。当网络达到这个目标时，它可以被各种各样的个人访问，一切都很好。当网络达不到这个目标时，特定人群中的人就会被排除在外，缺乏平等参与的能力，而我的朋友就是我们遇到问题的时候。

> 网络的力量在于它的普遍性。每个人，无论是否残疾，都能获得服务是一个重要方面。
> 蒂姆·伯纳斯·李，W3C 主管和万维网发明者

基本上，A11y 确保每个人都能平等地获得我们提供的信息。

## 谁应该练习无障碍？

每个人都应该。对此你可能会(理所当然地)问我为什么我自己没有实现它？好吧，我不认为我有一个伟大的答案，但我诚实的答案很简单，无知。大多数教程从来没有提到可访问性的实现，当他们提到时，可能会使用语义 HTML 或添加模糊的图像 alt 标签。可访问性(或#A11y)是那些被描绘成利基开发人员技能的主题之一，而事实上，它是我们今天都可以开始实现的东西，以使网络成为一个更好的地方。有大量有用的信息，所以不要再找借口了。

## 谁从无障碍中受益？

所有人。就是这样。这就是答案。当大多数人想到可访问性时，他们可能会想到诸如屏幕阅读器或文本到语音助手之类的技术，这只是需要可访问产品的人群中的一部分。我们还希望纳入可能有暂时性残疾的人，这可能是从最近的疾病或伤害中恢复过来，这可能会影响身体功能。

可访问性不仅仅包括残疾，我们还想考虑环境因素，并想办法确保最终用户仍然可以使用我们的产品。语言障碍和识字假设可能会无意中排除某些人。我目前在法国巴黎，虽然我不会说法语，但当有直观的图标来帮助用户理解内容时，使用系统和解释标志会容易得多。我们还希望开发时考虑到互联网连接速度较慢的用户，不要让大量不必要的图片拖累应用程序。
