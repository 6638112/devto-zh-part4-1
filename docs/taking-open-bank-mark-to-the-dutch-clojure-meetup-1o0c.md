# 带着开放银行马克去荷兰 Clojure 聚会

> 原文：<https://dev.to/gklijs/taking-open-bank-mark-to-the-dutch-clojure-meetup-1o0c>

我刚刚在[荷兰 Clojure Meetup](https://www.meetup.com/The-Dutch-Clojure-Meetup/) 上展示了 [open-bank-mark](https://github.com/openweb-nl/open-bank-mark) 。比我在 Kafka meetup 上更关注 Clojure 库的使用。

在准备阶段，我将`linger.ms`设置为 0 和 100 毫秒进行了测试。女巫是 Kafka 中真正影响延迟和吞吐量的变量之一。

很高兴 [Open Web](https://www.openweb.nl/) 能够赞助。因为这是一个只有 10 个人的小型聚会，所以有很多提问的空间，有时也有其他人回答我不知道的问题。原始数据可以在[这里](https://github.com/gklijs/open-bank-mark/tree/100-vs-0-linger-ms)找到。
