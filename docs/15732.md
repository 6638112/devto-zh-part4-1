如果你还没有跟上:1。你真可耻。2.为了学习 Node，我决定建立一个 API 来自动为我追逐自由职业者的工作。

“aaarrrggghhhhhhhhhhhhhhhh ffffaaaaaarrrrrkkkk！！(以尽可能好的方式)”这是愤怒(但不是真正的*愤怒*)澳大利亚语的意思是:“哦，拜托谷歌，T2 真的需要这么复杂吗。你让我看起来很糟糕。”

好吧，好吧。首先，我一直忙于我的日常工作。我在州际间有一份要求很高的工作，昨天刚回到镇上。我很累，脾气暴躁，注意力不集中。这是免责声明。不是借口...去吧。好吗？

一条小巷...今天的目标是理解 Google APIs 的授权如何分别在开发和生产环境中工作。结果如何？我完全不知道。没有。什么都没有。说真的，谷歌的说明就像汗衫上的臀部口袋一样有用。我甚至完全不能理解它是如何工作的。“Arrrrrggghhhh ffaaaarrrrkkkkkk！(以尽可能好的方式)。”

我看了[这个视频](https://www.youtube.com/watch?v=DYAwYxVs2TI&list=PLOU2XLYxmsILOIxBRPPhgYbuSslr50KVq&index=2)，用户界面已经改变了，所以它让这个几乎没用了。但并非完全无用，我设法将 gmail 和日历添加到我的开发者控制台，在我的应用程序下。所以这是一个胜利。我会抓住我能得到的任何胜利，现在。老实说，一旦我正确地阅读了 OAuth 文档，我认为验证过程将会非常顺利，并且有了正确的授权密钥，twc2k 将能够继续开发。

今天努力的好处是:我需要进一步把这个问题分解成更小的问题。我想。

1.  我知道我需要访问敏感数据。所以我需要 OAuth，让我们验证一个域并使用它。
2.  然后，我需要弄清楚给我的用于日历应用程序认证的 google 样板文件中有什么需要更改。我想这将会给我一个如何从那里开始的想法。祝我好运。

有人想保释我吗？？？？我给你买杯啤酒。
