# 什么时候不用继承？

> 原文:[https://dev.to/jckuhl/when-isn-t-inheritance-used-273c](https://dev.to/jckuhl/when-isn-t-inheritance-used-273c)

我在一次采访中被问到“什么时候你会使用继承而不是继承”我停顿了一下，因为我唯一能说清楚的是“当它没有意义的时候？”

顺便说一下，这是为了一个. NET 职位。

这个问题让我困惑的是，我真的不知道该怎么说。当继承不适用于软件架构时，你就不会使用它。我差不多就是这么说的，但我对自己的回答完全不满意。

采访结束后，我正在做事后报告，我暗暗怀疑他想让我提到作曲。在我写这篇文章的时候，我也觉得我应该增加不可变性，你不希望继承那些应该是不可变的对象(比如 Java 的`String`类)。

但现在它困扰着我。这个问题的好答案是什么？
