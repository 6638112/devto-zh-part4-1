# 工程师总是房间里最聪明的人

> 原文：<https://dev.to/remotejavadev/engineers-are-always-the-smartest-people-in-the-room-29l6>

几周前我在一次电话会议上，我们的项目经理提到“工程师总是房间里最聪明的人。”他开玩笑地说另一个团队的一个无所不知的工程师。我相信你知道这种类型。他认为，业务人员毫无头绪，其他工程师都是新手，管理层中到处都是技术上不会的工程师。

值得庆幸的是，开发人员的世界并不完全都是像我刚刚描述的那种傲慢的工程师。最终，我认为这种观点来自于没有意识到成功交付有用软件所必需的广泛的技能和任务。

毫无疑问，作为工程师和开发人员，我们的工作会非常复杂。然而，这本身并不能带来真正的商业价值。我们近视的工程师不明白。

在企业中，如果没有产品开发、销售、客户服务、物流、管理和其他许多方面的专业知识，构建技术上完美的软件是没有意义的。不管任务有多复杂，如果没有其他的考虑，设计软件基本上是没有用的。

### 有关系吗？

按照像智商这样的任意标准，工程师甚至可能是房间里“最聪明”的人。有关系吗？不会。在商业世界里，我们需要协作来提供真正的商业价值。如果你认为这很重要，你最好去找一个黑客马拉松或者其他人为的环境，在那里你可以和其他人比较你的智慧。

然而，良好的工程技能是非常有用和有价值的。傲慢的工程师和谦逊的工程师在商业价值上有很大的不同。

没人想和一个混蛋一起工作。当他们的同事自然而然地避免与他们打交道时，傲慢的工程师会让自己立即处于不利地位。他们降低了团队的士气。同样，团队焦点不可避免地偏离交付商业价值，并尽可能地转向避免自大狂。

### 时间接收器

傲慢的工程师也是团队巨大的时间浪费。你见过多少次像这样的工程师为无意义的技术细节大打出手？如果你已经在科技世界呆了一段时间，这种情况就太多了。

代码审查是他们最喜欢的战场之一。房间里最聪明的人喜欢通过从其他开发人员的代码中挑出毛病来显示自己的价值。“您使用了普通的旧 foreach 循环，而不是遗留代码库中的流？你真是个傻瓜！”

所以我们浪费了大量的时间去安抚或回避那个专横的人。他中断会议，专注于没有价值的无关技术细节。最终，我们花更少的时间来交付有价值的商业软件，而花更多的时间来平息不满。

### 真正的商业价值

在企业 Java 世界中，真正的商业价值创造需要的远远不止工程本身。在任何合理的大型项目中，我们开发人员和工程师都会从我们的同事那里受益匪浅。

我们需要产品所有者提供关于构建什么的指导。一个好的项目经理可以消除我们前进道路上的障碍，并确保交付依赖项。管理层可以确保我们可以专注于手头的项目，而不必处理大量其他竞争优先级的请求。销售团队向企业交付付费客户，没有他们，一切都会崩溃。客户服务弥合了软件所做的和最终用户认为它应该做的之间的差距。

这个清单可以一直列下去。很容易看出一个目光短浅的工程师在全球背景下看待事物时是如何只见树木不见森林的。

### 摘要

我期待有一天项目经理不再正确地认为“工程师总是房间里最聪明的人。”这种工程师的态度无疑对企业和他的团队是一个重大的伤害。

即使这是真的，从任何有意义的意义上来说都无关紧要。像这样的工程师对他的同事来说是一个很大的时间陷阱，因为他们被迫处理他错误的优先事项。真正的商业价值来自许多不同关注点的协作，其中只有一个是工程。
