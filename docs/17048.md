##### [t1**的固体原理是什么？**](#solid-prensipleri-nelerdir)

SOLID 是物件导向程式设计(OOP)专案的标准。由五种物质组成。
**1-单责任原则(单责任原则):**对象必须只有一项责任。例如，每个方法都不应该一次更新多个模型，而只能执行指定的工作。

**2-开/关原则(开/关原则):**对象的开发是开放的，但改变是封闭的。

**3-李斯科夫替换原则(李斯科夫替换原则):**类继承

**接口分离(隔离)原理**:接口不应添加超过所需的技能。尽可能多地从界面中分析对象不需要的功能。

**5-依赖反转原则(绑定反转原则):**顶级类、模块、方法等依赖于底层类子类中的更改不应影响父类。高级类不应链接到低级类。
