计算机科学似乎每周都给我们带来突破。当我们探索这些新技术时，重要的是保持对这些技术普及时什么是重要的观点。体验推动采用。

我们有这么多围绕计算的伟大的东西，它们中的任何一个被普通大众采用都有很好的用户体验。公众知道他们被期望做什么，使用计算机科学带来的技术几乎不需要认知负荷。但是，当我们在为下一个版本的网络——Web 3.0——做准备时，技术还没有变得易于使用，就已经普及了。[许多](https://hackernoon.com/internet-vs-blockchain-revolution-challenges-in-the-early-days-part-3-e8e26dd2ab57) [将 Web 3.0 的现状](https://www.ico.li/is-blockchain-like-the-early-internet/)与 20 世纪 90 年代初至中期的 Web相提并论。但是，早期的 web 还是很好用的。看看有什么可用的。雅虎的目录格式！在有围墙的花园世界里，AOL 是用户体验之王。

新的网络技术出现了，人们理解和使用起来非常简单。它可能做得不多，但正如我们所知，这种情况很快就会改变。当然，可能会很慢。但它是有用的。区块链技术和整个 Web 3.0 社区即将改变我们的计算方式，但随着该技术普及到普通大众，它甚至远不如早期 Web 普及时那样可用。

那我们该怎么办？

在基础层构建的内容非常复杂，需要特别关注。它需要迭代和打破东西的意愿。在这一层工作的人现在根本无法专注于用户体验。他们也不应该。

相反，我建议围绕 Web 3.0 兴起一个便利层。就像比特币基地为加密货币交易增加了一个便利层一样，必须增加一个只关注 Web 3.0 技术用户体验的层。这一层可以随着时间的推移而改变。随着技术的变化，随着人们适应正在建立的新网络，它可以变得不那么受监管。但在那之前，像 SimpleID 这样的公司应该存在。

SimpleID 并不适合所有人。对于那些在技术上已经熟悉区块链技术、加密和对等网络的人来说，便利层没有任何意义。但是如果我们想改变现状，让人们少用谷歌和脸书，我们需要给他们一条通往 Web 3.0 的捷径。

简单 ID 就是这么做的。[第一个计算机密码产生于 1961 年](https://mashable.com/2013/12/30/history-of-the-password/)。从那以后，计算世界围绕着用户名和密码的概念发展起来。它是如此根深蒂固，以至于人们有时会纠结于社交登录。因此，创建一个允许公众开始使用 Web 3.0 技术的用户体验的第一步是让他们感觉就像使用他们习惯的 Web 一样。要求他们写下一个 12 或 24 个单词的种子短语是不可能的。仅仅为了登录而要求他们安装本地 web 服务器或浏览器扩展是行不通的。相反，我们可以给他们一个用户名/密码体验，他们仍然保留他们身份的所有权，可以在任何地方使用它，而不需要保管解决方案层，并开始探索和使用 Web 3.0 技术。

这就是我们如何将体验提升到当前网络用户的期望。这就是我们如何从谷歌、脸书、亚马逊等的寡头垄断中转移力量。

如果你是去中心化 web 的开发者，如果你希望吸引人们使用 Web 3.0，如果你希望给用户带来他们期望的体验，同时让他们控制自己的数据，那么 [SimpleID 可以帮助](https://www.simpleid.xyz/?t=med_ux_end)。
