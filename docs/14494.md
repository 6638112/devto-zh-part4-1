# “成人”问题

> 原文:[https://dev.to/justzeros/grown-up-issues-10j0](https://dev.to/justzeros/grown-up-issues-10j0)

我相信 99%让人被解雇的问题在某一点上是“没什么大不了的”。

但是后来，问题“长大了”

趁杂草还小的时候拔掉，你就不那么需要除草剂了。

## TL；速度三角形定位法(dead reckoning)

当经理看到与他们有关的事情时，他们可能会视而不见，希望问题会自行解决。

我明白了——我自己也做过太多次了。你不想显得像一个微观管理的混蛋。

例如，也许杰克没有像其他人一样使用分支。

或者他只是偶尔批准一些请求。

或者他的 UI 和设计不太匹配，小方面。

在每种情况下，都是“足够接近”，所以你什么都不用说。

毕竟我们都很忙。我们都在尽力。

你也不想打击杰克的士气。

或者，更糟的是，让他生气然后辞职。

所以，你说“已经够近了”，希望他假以时日能想明白。

或暗示，或怒目而视，或讽刺的评论。

## [](#but-then)但是后来……

很快你就不信任杰克了，但又不知道为什么。

然后，有一天，他在公关上盖了太多次橡皮图章，你决定和他谈谈。

对杰克来说，这是件小事，因为这是他第一次听说。

对你来说，这是件大事，因为你有一千个伤口。

杰克很困惑，因为事实上这对你来说很重要。

即使很接近，也要做出艰难的决定。

尤其是和杰克一起工作的头几个月。

从第一天开始，对最微小的事情给予诚实的反馈。

杰克会知道这很正常，没什么大不了的。

我相信 99%让人被解雇的问题在某一点上是“没什么大不了的”。

但是后来，问题“长大了”

趁杂草还小的时候拔掉，你就不那么需要除草剂了。

在你的团队中，谁有一些“没什么大不了”的问题可能会出现？

你今天愿意在它们变成大问题之前进行一次快速讨论吗？

极品，马库斯
