# 个人项目如何让你成为更好的开发人员

> 原文：<https://dev.to/flippedcoding/how-personal-projects-make-you-a-better-developer-1n46>

对于许多开发人员来说，在工作了一整天之后，他们最不想做的事情就是编写更多的代码。整天盯着屏幕确实会对你造成损害。一旦你有机会去散步和吃东西，你可能会考虑开始一个个人项目。没必要太复杂。

可能有你想学习的框架或者你感兴趣的库。甚至不一定要和你的工作有关系。这是你学习你真正关心的东西和做你觉得有趣的事情的时间。做个人项目会让你成为更好的开发人员，这里有一些方法。

## 你可以不加判断地进行实验

没有人必须批准你的拉取请求，甚至看你的代码。漂亮不漂亮不重要因为只是你在玩。你可能想开始用虚拟现实工作，你只是想让你的小应用程序运行。代码可以想多难看就有多难看，没有人会知道。

这就像你让孩子们在线外涂颜色一样。你开始以不同的方式思考和看待事物。另外，在这个过程中你可以尝试不同的东西。你可以开始学习一个新的框架，并意识到你不喜欢它，然后立即停止。你的个人项目不会有任何后果。

## 您将学习如何处理更多的应用程序生命周期

因为这是一个只有你会参与的项目，所以你要做所有的决定。这意味着你将学习软件架构，因为你必须建立项目的结构。您将学习如何选择符合您需求的技术组合，即使您并不了解所有相关语言。项目的所有细节，比如谁会使用它或者它解决了什么问题，都取决于你。

个人项目让你对发展过程的每个阶段都有所了解。你甚至需要决定使用哪种托管服务，需要什么样的资源。这是获得广泛技能的最佳途径之一。当你进入一个专业的环境时，你会对你专业范围内的一切有所了解。

## 你可以让你的思绪游走

编程很像一门艺术。有许多方法可以完成同样的任务。现在你想试多少就试多少。对于个人项目，你不必遵循你在工作中使用的相同模式。你可以把它混合起来，尝试你想要的不同的东西。如果你没有机会，试试函数式编程。

这也是你实现百万美元想法的时候。如果你想让别人做一件东西，那就自己做。你可能不会真的赚到一百万美元，但是你已经解决了一个问题。你的项目是你发挥创造力和做你希望在工作中能做的事情的时间。尽情享受吧！

## 如何挑选项目

有时候很难确定一个项目的想法。如果你有这样的问题，从用不同的技术栈或其他东西复制一个不同的应用程序开始。这会让你的大脑兴奋起来，最终你会想出一些你更想做的事情。关键不是想出一个好主意。关键是开始做一些事情。

当你在你的复制品上工作了一段时间后，你可能会注意到你可以修复的应用程序中的一些缺点。或者你可能意识到你不想再做这个复制品了，于是你开始做别的东西。复制现有应用程序的目的并不是真正制作复制品。目的是让你开始做一些事情，这样你就会发现你真正想做的事情。

个人项目帮助你学习你不知道你需要的技能，它们给你一个学习新框架或编程语言的机会。通过玩代码并从中获得乐趣，你会使自己成为更好的开发人员，因为你还在学习。做点什么总比什么都不做好。

因此，即使你每天只花 30 分钟在你的项目上，它仍然有价值，因为你学到了一些东西。不要失去你曾经对代码的奇思妙想。你必须让它活着！你做过什么很酷的个人项目？我训练了一个神经网络来预测一个人下一步会点击什么，我发现你可以用 JavaScript 编写微控制器。🤪…我需要停止寻找可以玩的东西。
