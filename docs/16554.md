# 随着冲刺停止#2

> 原文:[https://dev.to/simme/as-the-sprints-stop-2-h32](https://dev.to/simme/as-the-sprints-stop-2-h32)

又到了一年的时间。包括我在内的所有书呆子都被迫离开办公室去享受受之无愧的 RNR——就像我们小时候父母告诉我们必须出去玩“呼吸点新鲜空气”一样。

玩笑归玩笑，我真的很喜欢我的暑假。它给了我足够的时间去旅行、读书和反思过去的一年。去年，这最终导致我辞去了之前的工作，创办了自己的独立咨询公司。到目前为止，我还没有后悔过哪怕是一丁点儿。这是一个新朋友、像 Github 卫星(T1)这样的酷事件、像 T2 了望塔(T3)这样的新开源项目以及一些公开演讲的美妙混乱。

我很想听听你在过去的一年里都做了些什么，以及你认为应该列入清单的任何文学作品。欢迎在下面的评论中或在 [twitter](https://www.twitter.com/0x12b) 上联系我。

现在，让我们来看看这些书:

### 《devo PS 手册》(2014)，作者吉恩·金等人。

去年，我的一个建议是凤凰计划。一部关于虚拟公司 Parts Unlimited 的 IT 经理比尔的小说。Bill 被指派扭转公司最重要(也是最失败)的项目，他开始意识到 IT 工作与制造业有很多共同点。精益和丰田生产系统中常见的技术似乎也有类似的效果。

《凤凰计划》是一部商业小说，因此，它并没有详细讨论为什么和如何做。另一方面,《DevOps 手册》的唯一目的是教你如何在你的组织中实施 DevOps，以及为什么这对你有好处。

[在 Amazon.com 购买](https://www.amazon.com/DevOps-Handbook-World-Class-Reliability-Organizations/dp/1942788002)

### [](#ghost-in-the-wires-2012-by-kevin-mitnick)《电线中的幽灵》(2012)，作者凯文·米特尼克

《电线中的幽灵》讲述了凯文·米特尼克的故事，他一度是世界上最受通缉的黑客之一。这本书是一次伟大的回忆之旅，回到了黑客攻击主要是关于挑战而不是关于经济激励的时代。虽然不是虚构的，但它真的和我遇到的任何虚构的犯罪或惊悚片一样紧张和充满悬念。

[在 Amazon.com 购买](https://www.amazon.com/Ghost-Wires-Adventures-Worlds-Wanted/dp/0316037729)

### [](#rolling-rocks-downhill-2014-by-clarke-ching)《滚石下山》(2014)，作者 Clarke Ching

滚石下山对于软件开发就像凤凰计划对于 IT 运营一样。

许多，我甚至敢说大多数，今天的软件开发团队都在努力满足预算、期限和需求。每个曾经在软件开发项目中工作过的人都记得并且害怕当项目接近最后期限时所需要的关键时刻。

《滚石下山》试图用另一种方式来教授。使用敏捷方法和 Goldratt 的约束理论，关键时刻很可能成为过去。

[在 Amazon.com 购买](https://www.amazon.com/Rolling-Rocks-Downhill-Business-mentions-ebook/dp/B00PJ8HBW8)

### [](#the-goal-1984-by-eliyahu-m-goldratt)《进球》(1984)，由艾利·M·高德拉特拍摄

《凤凰计划》和《滚石下山》的前身，据我所知，也是第一本广受欢迎的商业小说。

到目前为止，它已经快 40 岁了，但仍然保持着相关性。在这段时间里，世界发生了很大的变化。比如说；制造工厂在西方不再那么普遍，大多数成功的工厂似乎已经在实践精益制造。

尽管如此，阅读这本书感觉就像阅读大量知识、方法和技术的来源，这些知识、方法和技术仍然和以前一样相关。

[在 Amazon.com 购买](https://www.amazon.com/Goal-Process-Ongoing-Improvement/dp/0884271951)
