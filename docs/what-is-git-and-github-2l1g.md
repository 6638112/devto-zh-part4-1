# 什么是 Git 和 GitHub？

> 原文：<https://dev.to/decipherzonesoft/what-is-git-and-github-2l1g>

因为我们对软件开发行业有着大量的了解，所以你很可能听说过代码版本控制和代码库。即使你对这些术语一无所知，读完这篇博客后你也会很熟悉。

**什么是 Git 和 GitHub？**

代码库是一种租用的代码仓库或带有版本控制软件的代码托管服务。最受欢迎的是 GitHub、Bitbucket、CloudForge、Source Forge 和 GitLab，它们的存储库是公开和私有的。每个开发人员作为一个项目的团队成员开发小的部分，这些小的部分随后与来自其他团队成员的代码集成，以部署一个特性。

**Git 和 GitHub 一言以蔽之**

如果一个开发人员为进一步的软件开发做了一些必要的变更，那么这些变更不会反映在其他团队成员的系统中，这会导致代码集成上的冲突。很难知道为了解决冲突应该保留哪些更改，应该放弃哪些更改。因此，简而言之，代码库帮助开发人员协作、同步团队并避免冲突。

**Git 是什么？**

Git 是由 Linus Torvalds 开发的代码版本控制系统或代码版本控制工具，他也开发了 Linux。如果开发人员注意到上一次更新的代码有问题，那么他/她可以检查什么被改变了，并迅速找出并修复问题。Git 在本地系统上进行版本控制；因此，需要一个云版本来防止系统崩溃和软件开发团队之间的高效协作。

**什么是 GitHub？**
GitHub 是一个用于代码版本控制系统的 web 服务，具有许多其他特性。GitHub 也是一个面向开发者的专业网络，人们可以在这里看到开发者对其他项目的贡献。每个开发人员在 GitHub 上注册时都会创建一个概要文件，在这个概要文件下，他们可以创建公共或私人访问的代码库。

软件公司设计了一种复杂的树状结构，用于将代码上传到代码库，这种复杂的结构及其用法是在 Git 分支模型和 Git 流程下定义的。每个开发人员只负责在他们各自的分支上传他们的代码，其他分支用于代码集成、测试、快速修复和部署。

**以下是 GitHub 上版本控制系统的主要操作:**

*Git 初始化*
将现有目录初始化为 Git 仓库

开发者可以通过命令行将代码库中的代码克隆到他们的系统中，甚至可以从 GitHub 下载成一个包。

*提交*
Git commits 是指你想在现有文件中写入数据或者想覆盖现有文件。

Git Add
Git adds 命令仅仅意味着你想在下一次提交时发送提到的文件。

*Push*
Git push 将代码发送到代码存储库，代码存储库将反映云上先前保存的代码的变化。

*Pull*
Git pull 命令用于从云上的代码库中获取变更(如果有的话)。

分叉是一个过程，开发人员可以克隆一个代码库，将其用作新项目的参考点，或者他们可以通过向管理员发送请求来编码和建议对当前库的改进。
