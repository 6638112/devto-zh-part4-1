<u>Linux 中的运行级别</u>
Linux 中的运行级别决定了什么程序将在系统启动时运行
或者将被执行
这决定了您的系统在启动时是加载 GUI 还是
CLI 模式

从 0-6
开始有 7 个运行级别，每个运行级别都有一些启动或停止的服务，这通常决定了机器的默认行为
我不会详细讨论这些级别，如果你想要更详细的解释，请留下评论。
运行级别是
运行级别 0 -系统停止
运行级别 1 -单一模式
运行级别 2 -没有网络文件系统的多用户模式
运行级别 CLI 模式下的多用户模式
运行级别 4 -此处留空，因此可以由用户定义
运行级别 5 -图形用户界面下的多用户模式(大多数桌面的默认设置
运行级别 6 -重新启动

默认情况下使用的两种模式是运行级 3 和运行级 5
。如果你想知道你在哪个运行级上，在终端类型
$runlevel
上，驱动这些级别的脚本位于/etc 目录中

**<u>本文的核心</u>**
我们主要关注的是运行级别 1(单用户模式)
这可以与 windows 中的人们所说的安全模式相比较
在这种模式下，网络服务将不会启动，但它允许用户对系统进行更改

**注意:这只有在系统的 grub 菜单没有密码保护的情况下才有效**

让我们开始吧:
第一步:重启你的机器，在 grub 菜单上点击“e”进入编辑模式。
第二步:找到以 Linux 开始的行并滚动到该行的末尾
第三步:删除一直到‘ro’的字符
-ro 代表只读，将其更改为‘rw’以允许写入
-在用 rw 命中空格替换 ro 并添加 init=/bin/bash
后，您的行看起来像这样
rw init=/bin/bash
第四步:按 Ctrl-x 或 f10 以当前设置启动

引导程序现在为您提供了一个具有 root 权限的 shell，您可以继续按照通常的方式更改 root 密码。

$passwd 根

确认密码后，重新启动，我们有了一个新的根密码。
