# 作为一名开发人员，你可以“升级”的 4 种方法

> 原文：<https://dev.to/skill_pathway/4-ways-you-can-level-up-as-a-developer-17ol>

不同“级别”的开发者之间的界限是模糊的。问任何一个高级开发人员、开发经理或技术总监，初级、初级、中级和高级之间的区别是什么，你很可能会得到截然不同的意见。

既然有如此广泛的观点，那么花时间弄清楚如何成为“高级”或“中级”开发人员真的没有用——在很大程度上，级别是观点。

相反，让我们看看一些你可能没有考虑到的事情，如果你实现了其中的一些，将会提高你作为一个开发者的水平。

# 1)建设实质性工程

React JS 计算器？很酷的演示，但不是实质性的。待办事项 app？再次，有趣的小工具，但不是实质性的。大多数人可以跟随教程，我们在谈论别的东西...

具有许多不同视图、端点和功能块的东西，即*独特的*和*展示了一些开发人员的创造力*。

有哪些我们认为是“重大项目”的例子？

*   狗的火种。
*   口袋妖怪爱好者的微型社交网络。
*   一个框架式的 web 后端或前端框架，其他人可以使用——不是说你期望一定要竞争，而是让你了解如何构建一个框架！
*   用您使用的主要编程语言构建一个包——一个您将在多个项目中使用的包。

这里有一些有趣的例子，但重点是——要么*围绕你超级热爱的东西*建立一个更大的项目，要么*建立一些彻底突破你目前理解的界限的东西*。根据定义，这两件事都会“提升你的水平”,因为你会遇到一些概念

# 2)促成开源

这意味着*构建自己的开源包/应用*或者*贡献给现有的库*。

## 自己的包

如果你要构建自己的包，只要确保它不是来自教程，而是你实际上想发布给其他人使用的东西。

### 记录你的过程

如果你围绕你如何/为什么创建它和你的思考过程创建了一个博客帖子或视频系列，加分。这不仅让你从讲故事的角度来思考它，获得新的见解，而且这是你能力的有力证明，成千上万试图学习你所学的东西的人将从听你如何进行这个项目中受益。

## 别人的储存库

你最喜欢的节点、PHP、python、java 包和插件都在 GitHub 上。很有可能这些包中至少有一打未解决的问题。

检查存储库贡献者指南，通常在代码根中的. MD 文件中，很好地理解它，然后离开并修复问题。更好的是，给它添加一些功能，这些功能对*实际上是有用的*。

这有多种用途

*   证明你可以遵循编码指南(对潜在雇主有好处)
*   证明你是开源社区的一员，并且是一个知识库的命名贡献者
*   您将更深入地了解分叉、拉请求、提交
*   您将看到社区维护的包是什么样子，并有机会采用一些更好的编程实践

# 3)看一个资深开发者的代码(或者你认识的一个比你更有经验的程序员)

这是提升开发人员的最快方法之一。因为(大多数时候)你将会看到比你高一级的代码。

尝试理解它——将其与您自己的模式进行比较，找出您可以实现哪些模式来改进您的模式——记下他们所做的、您可以在自己的工作流中实现的所有小的技术事情。

如果你目前没有在一家公司工作，这并不总是可能的，如果你没有，你可以从你最喜欢的社区开发者那里寻找开源项目，他们经验丰富，阅读代码并试图理解。

如果你正在看的代码的开发者有时间提问并乐意回答他们，那也很好——只要让他们知道你正在努力更好地理解一个前辈的代码是什么样的，并获得他们的许可，就你正在努力理解的事情问几个问题。很多人会很乐意帮忙的。

# 4)了解更多编程设计模式和原理

深入研究 SOLID、KISS、YAGNI、MVC，以及大量其他听起来奇怪的缩写设计模式和原则。这里有一个关于他们的巨大的维基百科页面和[另一个就在这里。](https://en.wikipedia.org/wiki/Category:Programming_principles)

通读每一个的定义，谷歌每一个的例子——例子是*超级重要的*,如果你有关于一个模式是如何*实际上*实现的上下文，你将能够更好地理解它。

很有可能，有很多你没听说过的。这里的大部分招聘经理都记不起所有的人，快速浏览其中的一些可以刷新我们的记忆，让我们看看它将如何改进我们的编码过程。

将来我们可能会发布一篇博文，提供一些上下文相关的伪代码示例——编程模式是许多开发人员从未试图完全理解的东西，我们知道为什么:因为它们作为解释毫无意义。我们需要例子！！！
