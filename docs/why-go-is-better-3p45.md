# 为什么围棋更好？

> 原文：<https://dev.to/ajayatgit/why-go-is-better-3p45>

GoLang 是由谷歌的开发人员构思和开发的。据报道，这是他们在等待项目完成时构思的。

它被称为 Go 或 GoLang，是一种健壮的系统级语言，用于跨大规模网络服务器和大型分布式系统编程。Golang 是在 Google 需要网络服务器和分布式系统的背景下，作为 C++和 Java 的替代品出现的

让我们逐一讨论使 go 不同于大多数现有编程语言的因素。

# 1)多线程和并发

绝大多数编程语言在处理多线程时都缺乏并发执行。它们通常会降低编程、编译和执行的速度。这就是 Go 成为支持多线程环境和并发性的最可行选择的原因。随着时间的推移，硬件制造商不断向系统添加更多内核，以确保更好的性能。
Go 是在多核处理器广泛应用于复杂硬件的时候构想出来的。Go 的创始人特别关注并发性。Go 与 goroutine 一起工作，这允许它同时处理大量任务。

# 2)围棋的简单性

## 无泛型:

泛型或模板仍然是各种编程语言的中流砥柱，通常会增加理解的模糊性和难度。Go 设计师决定不用它，让事情变得简单。

## 单次可执行:

GoLang 没有任何单独的运行时库。它可以生成单个可执行代码，只需复制即可部署。这有助于消除在依赖关系或版本不匹配上犯错误的所有顾虑。

## 无动态库:

Go 只是决定放弃任何动态库，以保持语言的简单明了。尽管在最新的 Go 1.10 版本中，开发者可以选择通过插件包上传动态库。这只是作为一项扩展功能包含在内。
