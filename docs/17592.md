# 不得不匆忙学习的挫折

> 原文:[https://dev . to/dechamp/the-fuckness-have-to-learn-on-the-fly-the-fly-553h](https://dev.to/dechamp/the-frustrations-of-having-to-learn-on-the-fly-553h)

我最近拿起打字稿。来自面向对象的背景，它感觉很自然，很容易学习(大部分情况下)。

我一天就学会了大部分。然而，有一些非常重要的部分我仍然不知道。这是因为需要完成一个项目，而不是花时间学习 TypeScript 的另外 10%，这是重要的花絮所在。

我花了两个小时试图找出为什么我的新模块声明文件没有找到。IDE 点击通过，可以很好地找到它们。

我检查了它们是否都在那里，package.json“类型”是否指向该文件，它们似乎都配置正确。

显然，出于某种原因，你必须加上“baseUrl”:添加到 tsconfig.js 文件.....我以为它会默认为根目录。

为什么这能修复我的代码？多么令人沮丧！

这让我感到沮丧，魔鬼就在细节中。

你们中有多少人被难住了，只是意识到由于没有完全学会一门语言而错过了一些简单的东西？

那是什么，你从中学到了什么吗？
