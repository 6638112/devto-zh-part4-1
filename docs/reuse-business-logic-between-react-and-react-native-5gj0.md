# 在 React 和 React Native 之间重用业务逻辑

> 原文：<https://dev.to/spoman007/reuse-business-logic-between-react-and-react-native-5gj0>

React 生态系统鼓励一次学习到处写的风格。一旦我们知道了 react 的核心概念，我们就可以轻松地编写 web 和移动应用程序。即使存在组件类型等微小差异，应用程序的核心仍然是相同的。

作为一个库，我们通常需要一些状态管理包的帮助来开发大型应用程序。Redux 是比较首选的。使用 redux 有助于我们定义流程、业务逻辑和应用程序状态。

当我们为客户开发项目时，我们意识到即使 web 和移动应用程序的视图层是不同的，我们也在编写大量重复的代码。例如，我们的网络和移动应用都有一个登录界面。现在，它的 UI 层在 react 和 react native 中是不同的。但是一旦我们克服了这一点，我们需要分派相同的动作，进行相同的服务调用，使用相同的缩减器等等。
所以我们决定将这个业务逻辑放在一个单独的 npm 包中。我们称这个 npm 包为核心。移动和 web 应用程序都可以通过 symlink 将此作为依赖项或链接加载到此包。

这帮助我们减少了代码的重复。我们能够在 React 和 React 本地应用程序之间重用近 60%的代码。这也有助于我们更好地预测和一致的错误再现。如果某件事成功或失败了，它会发生在 web 和移动应用程序中。

所以最终我们能够以更快的周转和更少的预算交付产品。
