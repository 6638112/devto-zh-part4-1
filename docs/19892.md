# 7 年编程后的经验教训

> 原文:[https://dev . to/k88 manish/7 年编程经验教训-3oc](https://dev.to/k88manish/lessons-learned-after-7-years-of-programming-3coc)

去年 12 月，我完成了 7 年的网页开发，但我仍然觉得有很多东西要向其他人学习，不管他们是否更有经验。在当今快节奏的环境中，开始使用技术或学习一门新的编程语言只需要几天或几周的时间。

但是我相信有些东西你需要时间才能学会，所以我将分享我到现在为止学到的经验。

# [](#fail-fast-but-try-not-to-fail-for-the-same-reason-again)快速失败但尽量不要因为同样的原因再次失败

当我开始每隔一个小时学习编程的旅程时，我会在某件事情上坚持几个小时甚至几天(我仍然会坚持一些时间:P)。

我总觉得这些问题势不可挡，没完没了。我总是向资深团队成员寻求帮助，这种情况持续了我第一份工作的前 3 个月。然后有一天学长给了我一个金咒！他说`Pay more attention to error details`。我拿着这个手到手，挖掘错误到更多的细节，并能够在几天内理解模式。从那时起，学习过程变得比以前简单多了，我能够通过观察错误和模式来避免常见的问题。这也帮助我不去打扰正在解决更复杂问题的学长们

# [](#dont-hesitate-to-ask-for-help)不要犹豫，寻求帮助

但是我的小脑袋花了很长时间才认识和消化这个事实。

我花了数不清的时间来发现一些非常愚蠢的错误或我的任务中的一些遗漏的步骤。我大脑的一部分在寻求帮助，但另一部分告诉我不要犯傻，自己解决。相信我，这是一个非常糟糕的状态。我自己也在这种状态下浪费了很多时间。因此，如果你刚刚开始你的旅程，不要犹豫，只是问。

我想在这里再补充一点，你也需要尊重他人的时间。你不应该不断地 ping 其他人，只是耐心地等待，并使用其他资源，以及像 StackOverflow 一样，看看是否有人已经解决了相同的问题。

# [](#learn-to-say-no)学会说‘不’

经过两年的学习和不断的练习，我在编程方面比刚开始时进步了很多。在创业公司工作的好处是你永远不会没有任务，所以我的盘子里也装满了东西。事情进展得越来越快，所以我也必须加快步伐来完成任务。对吗？我也是这么想的，也犯过错误。

质量因素开始下降，我在代码中有更多的错误，设计模式没有被正确遵循，并得到了很多' WTF！!'在代码评审会议期间。我知道时间是一个重要的因素，在合理的短时间内完成任务也很重要，但是你不应该在那些会适得其反的事情上妥协。

如何处理这样的情况？花时间分析任务考虑所有的参数(编写测试用例，测试，修复 bug，如果有的话)，计算一些粗略的时间，看看你是否能在时限内完成。如果你不知道如何分析，从你团队中的资深/同辈成员那里获得一些帮助。如果你发现时间限制离你很远，那么你应该对时间限制说“不”,要求更多的时间来完成它。

# [](#seek-a-mentor)求导师

当你在学习编程的时候，有一个导师是你能得到的最好的东西，相信我，同样很难找到一个导师。我很幸运，在我找到第一份工作时找到了它。我们大多数人都处于某件事情的起点，正在寻找一位导师，如果你找到了能帮助你做出正确决定的导师，你的生活会变得简单很多。我将告诉你我的故事，我当时打算加入一个支付给我比我的工资高得多的组织，我和我的导师谈了这件事，他给了我一些关于这件事的见解，并拒绝了这个提议，几个月后我听说同一家公司失去了一个大客户，现在为了节约成本解雇了员工。我得救了，唷...这只是导师可以帮助你的众多方式之一。怎么找到你？有不止一种方法可以加入一些休闲频道，在 Twitter 上关注你欣赏的人，还有更多。

# [](#seeing-things-from-a-bigger-perspective)从更大的角度看问题

有时我们可能会觉得需求不是很有用，或者会有不好的用户体验，由于这种感觉，我们可能会降低需求的优先级

例如，在我们的应用程序中，我们已经有了记录编辑功能，在后期阶段，我们收到了使用 CSV 上传编辑记录的新功能请求。最初，我们认为这不是很有必要，因为我们已经有编辑功能，但当我们了解到用户已经非常熟悉电子表格，他们可以更容易地一次编辑 CSV 中的多个记录时，我们也在我们的应用程序中实现了它。

所以，当你在开发一个新的功能时，试着从一个更大的画面来想象它，想想它能如何帮助你的应用程序的用户，用户能与它交互的可能方式是什么。

# [](#be-a-helping-hand)助一臂之力

开发人员总是被困在学习循环中，幸运的是，其他开发人员可以帮助我们。他们告诉我们什么是我们的错误或者我们错过了什么。这是我们社区成长的方式之一，也是我们社区变得对需要帮助的人如此有帮助的方式之一。所以每当你认为你可以的时候，帮助别人。

# [](#do-something-for-yourself)为自己做点什么

很难让自己长时间保持动力和专注。对于程序员来说，单调的生活方式让他们筋疲力尽是很常见的。你不应该忽视电脑之外的生活，所以你应该为你的爱好或任何让你开心的事情留些时间。做一些除了编码之外的事情，比如去健身房，学一些乐器，读一本书，见见家人或朋友。相信我，你会成为更好的自己。

所以这是我到目前为止的建议。希望能帮到你:)
