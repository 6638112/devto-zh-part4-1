# 让 Magento 技术支持团队变得优秀并让客户满意的 4 条规则

> 原文：<https://dev.to/berk_scandiweb/4-rules-to-make-magento-technical-support-team-great-and-customers-pleased-4la8>

指导如何让 Magento 支持团队和客户。在 Magento 领域工作了十多年，在开发和支持 Magento 项目方面，我们学到了很多东西。我们想分享我们的经验，更具体地说，有一个高效和满意的支持团队需要遵循的 4 条规则。

有一个问题:许多 IT 专家认为技术支持开发人员的工作不是一份好工作。它通常被认为处于职业阶梯的最底层——好像这是一个为那些没有足够的知识和经验成为专业人士并从事真正重要的事情的人准备的职位。

这不是真的，差远了。当然，做好这份工作并享受其中的能力在一定程度上取决于我们的个性、技能和态度。Scandiweb 是一家致力于在 Magento 平台上为纽约时报、乐高、欧莱雅和许多其他公司开发高级电子商务解决方案的公司，通过这个例子，我们找到了在技术支持团队中工作的原因。如果你愿意，你也可以用它来指导如何让 Magento 支持对团队和客户都有好处。

技术支持开发人员 DMITRIJ

“你需要有不同的心态。当你在一个标准项目中工作时，所有的事情都已经写在你的技术任务中，你不需要考虑该做什么:你只需要拿起并处理任务，就是这样。在技术支持中，没有人向你解释任何事情:你遇到了问题，你必须找到解决问题的方法。怎么做？这全靠你了。”

Magento 支持规则#1:高效善变，不厌倦技术支持对于那些在快节奏、动态的环境中舒适工作的人来说是一个很好的地方——这里的一切都对厌倦、停滞和低生产率说“不”。如果你是一名技术支持开发人员，你不会得到一个项目——你有一个项目和不同类型任务的列表，你必须根据优先级和可用资源在它们之间进行选择和切换。大多数情况下，完成一项任务不需要超过 16 个小时的实际工作时间，所以你几乎每天都要做一些新的事情。

有一种流行的观点认为，技术支持是关于故障排除、修复和错误的，而不是现在的样子——或者至少不是 Scandiweb 通常的样子。技术支持团队收到关于添加新功能、特性和改进的请求。向技术支持部门提交请求可以让客户在任务制定、预算和截止日期方面更加灵活。对于技术支持开发人员来说，这意味着从头开始思考解决方案的机会。

Magento 支持规则#2:对你所做的事情负责
你完全控制着你的工作过程。在 Scandiweb，技术支持部门是由这样的人组成的，他们只对自己负责完成任务和目标。日常职责由团队成员决定。

在 Scandiweb 技术支持部门，开发人员没有一个项目经理站在背后告诉他们该做什么——每个人都做每件事，因此团队内部的人可以自主地计划、协调、指导和控制他们所有的工作活动。他们最引以为豪的是，通过任务的整个生命周期来管理任务，现在甚至不会吓到那些曾经习惯躲在键盘后面的专家。发展一个自我管理的组织让他们的技术支持部门变得快速、灵活和可扩展。

LIVIJA，一名技术支持开发人员

“就我个人而言，我喜欢与客户面对面交流:倾听他们表达自己的需求，并提出我自己的想法。责任全在你，有些人可能还没准备好承担——当你可以推卸责任时，事情总是变得更容易，但我一直认为这是一个提高自己专业技能和工作能力的机会。”

Magento 支持规则#3:互相帮助
在 Scandiweb 技术支持部门工作的人互相交流很多:分享专业知识，讨论我们遇到的问题，从错误中学习。他们的技术支持部门建立在每个人都有平等的职业价值观、技能和经验的原则之上，这样你就可以一直依靠其他团队成员的帮助。每天早上，团队都会召开 10-15 分钟的会议，这在 web 开发中很常见，但不同的是，在 Scandiweb 上，他们不会通过列举昨天做了什么来提供状态更新，而是专注于他们是如何做的。技术支持部门每两周召开一次全体会议，讨论日常工作中的改进和不足。讨论决策过程会使项目协作变得更容易。

技术支持开发人员 DMITRIJ

“从某种意义上来说，没有不可替代的人。我们都有相同的职业价值观，团队中的每个人都互相支持。如果发生了什么事情，比如你突然生病了或者休假了，工作过程中不会有任何的拖泥带水。其他人会立即接手你的任务，不会浪费时间去弄清楚到底需要做什么。”

Magento 支持规则#4:学习更多关于技术支持的知识你永远不会停止学习。即使你已经是一个有经验的开发人员，你经常会被分配到你以前从未做过的任务，你必须对一个项目，特性和功能，它的工作原理进行研究。你可以从“A”到“Z”了解你正在做的项目，包括所有方面:前端、客户端、后端和服务器端。

在技术支持部门工作也是初级开发人员提高知识、技能和能力的绝佳机会，在 Scandiweb 已经在实践中得到证明。他们的超级英雄团队之一——钢铁侠团队——在不到半年前成立。当时的想法是在它的顶部放一个愿意成为导师并传递知识的高级开发人员，让他从初级开发人员中招募团队的其他成员，并看着他们从种子成长为大树。

技术支持开发人员 KASPARS

“我喜欢把在技术支持部门工作比作消防员或护理人员，因为我们也在紧急情况下为人们提供帮助。我们做不可或缺的重要事情。”
