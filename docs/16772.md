# 任何人都可以编码

> 原文:[https://dev.to/nfairbairn/anyone-can-code-2ln3](https://dev.to/nfairbairn/anyone-can-code-2ln3)

在过去的一年里，我一直在学习编码，更重要的是，学习社区。成为一名程序员需要什么？程序员应该具备哪些素质？为了成为一名优秀的程序员，我需要关注什么？

带着所有这些问题，我做了一些研究，总结如下…

没有什么秘方。

编程就像任何事情一样，对一些人来说比其他人来说更自然，但最终它就像任何技能一样，你必须去做，并学习如何学得最好。

我将其归结为 3 个主要概念，我认为这将帮助任何新程序员以更加积极的态度开始接触软件世界。

**解决问题**

编程就是解决问题。

工程学就是要有一个想法，并在现实生活中用你所拥有的工具去实现它。

当你着手一个项目或一个想法时，通常创造它的第一步是计划。

将一个想法分解成更小的部分，开始研究如何实际创建和实现这些部分。

程序员就像工匠一样。一个木匠有锯子、锤子、尺子、磨砂机或任何他们可以用来制作任何东西的工具，从精致的桌子或摇椅，到在海上航行的大船。

程序员使用不同的语言、框架、算法以及他们能利用的任何解决问题的方法来创建一个能让观众眼花缭乱的软件。

就像一个工匠的桌子会因为他们在桌子的细节上投入的时间、精力和注意力而卖出高价一样。从挑选将成为餐桌的那棵树，到进行切割，再到修整外观，使其成为完美的图片，展示在某人的家中。

程序员(至少是好的程序员)会像其他工匠一样关注代码的细节。通过广泛的计划，绘制设计，选择最适合项目的语言和框架。就像一个工匠在头脑中有一个他们的作品对客户来说是什么样子的图像，一个程序员有一个他们心目中的成品，并开发一个客户会喜欢的用户体验。

考虑到用户/消费者，对细节的关注，加上对细节的极度关注和对项目卓越的追求，都标志着工匠的特质。

**好奇心**

掌握一门交易需要一定的决心和好奇心，才能最大限度地从事一门交易。渴望学习的不仅仅是什么，而是为什么和如何学习。

好奇心是点燃发现、创新和决心之火的东西。

***微观与宏观相结合***

*微观*是语义、语法和创建更大画面或程序的实际构件。

*宏观*是所有更抽象的概念，彼此相关。它们共同创造了项目本身，它包含功能、交互性、统一的主体或概念，包含所有较小的部分，所有这些部分一起工作来创造一个更大的东西来执行任务。

把它想象成一个时钟。在里面，所有的小齿轮都在转动，互相利用，共同完成一项更大的任务。没有适当的计划，没有每个部分同时工作，整体的完整性就会受到损害。

编程也是如此。程序员花费他们所有的时间和精力来确保这些齿轮中的每一个都到位并正常工作，就像一个主时钟工人专注于齿轮以确保时钟正常有效地工作一样。

**创意**

创造力是激励我们去追求或创造比现有事物更伟大的东西。创造力使人们在处理这些技能的方式上与众不同。艺术家、工匠、作家、画家、摄影师等等。

创造力是我们跳出框框思考的方式，是我们创造新想法的方式，是我们找到解决问题的新方法的方式。

程序员也是有创造力的，就像画家选择在画布上用什么颜色来传达他们的信息一样，程序员选择他们的工具、优势和劣势来将他们的信息传达给软件。

在电影《料理鼠王》中，有一句著名的台词是“任何人都可以做饭”。认为烹饪是专门为有“天赋”或“神童”的人准备的误解，或者认为任何技能或行业都是某个特定群体专有的想法是完全错误的。

在编程中，存在一个巨大的误解，即所有的程序员都必须是天才，整天只知道喷涌算法和数学，而且不知何故超级聪明，鹤立鸡群。

事实是程序员也是人。有长处和短处，我们都利用这些长处和短处来帮助我们学习如何做一件事。我们只是碰巧选了编程。

我认为我们需要开始相信“任何人都可以编码”这句话，就像我们可以相信一只老鼠可以相信“任何人都可以做饭”一样。

对编程的误解就是这样，误解。

伟大的厨师不仅在烹饪方面出类拔萃，而且将食物发挥到极致，创造出震撼人心的美味佳肴。伟大的画家不只是在画布上画画，他们努力完善自己的技艺，并试图追求超越画布的愿景。

话虽如此，程序员也可以如此。不是每个程序员都会开发下一个大型社交媒体应用或在线流媒体服务，就像不是每个画家都会画下一个蒙娜丽莎一样。

这并不能阻止人们绘画、烹饪或编程。

认为程序员必须是天才的误解，就像每个画家必须是有远见的人一样正确，或者每个厨师都必须是五星级厨师，做着从未见过的疯狂菜肴。

我们都只是探索交易、技能和学习更多我们作为人类所拥有的工具的人。

就像任何其他行业或技能一样，编程需要解决问题、好奇心和创造力。这需要练习，肯定会有很多人比你“更好”。但是有没有一个五星级的厨师吓得你不敢自己做饭？或者博物馆的一个展览突然让你再也不会拿起画笔？

你不需要成为编程天才，你只需要开始。
