# 成为伟大程序员的 101 个技巧

> 原文：<https://dev.to/emmabostian/101-tips-for-being-a-great-programmer-human-36nl>

# 1。擅长谷歌搜索

做一名程序员就是要学习如何寻找问题的答案。通过学习有效地用谷歌搜索东西，你会节省很多开发时间。

# 2。承诺不足，兑现过度

最好让你的团队知道一项任务需要三周时间，并在两周内完成，而不是相反。通过承诺不足和兑现过多，你会建立信任。

[![Designer](img/3f8ca664201efcc5857aa89ae6cfdea2.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--XuAuxqWV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/umrbnosn8g68nep19y21.png)

# 3。善待你的设计师；他们是你的朋友

设计师针对用户痛点提供解决方案。向他们学习，团结一致地工作，打造有效的产品。

# 4。找一个导师

找一个你可以向其学习和交流想法的人。如果你需要一个技术指导，编码蔻驰是一个很好的起点！

# 5。做导师

做一个别人可以学习和借鉴的人。我们希望你能在[编码蔻驰](https://codingcoach.io/)担任导师

# 6。写有用的评论

写下解释“为什么”而不是“什么”的评论。

# 7。恰当地命名变量和函数

函数和变量应该准确地表示它们的用途，这样`myCoolFunction`就不会飞了。

# 8。休假

我们都需要时间来解压。进行你一直想要的旅行。你的大脑和你的同事会感谢你的。

# 9。删除未使用的代码

没有理由累积更多的技术债务。

# 10。学习阅读代码

阅读代码是一项被低估的技能，但却是非常宝贵的。

# 11。建立健康的工作/生活平衡

漫长的一天之后，你需要时间来解压。关闭工作通知，删除手机上的应用程序。

[![Meeting](img/d655bddf776e887a71b219b1fc070970.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--VqSUBCSo--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fv2vh91wlhe9dh046mub.png)

# 12。只安排必要的会议

可以在一封邮件或者一条 Slack 消息中解决吗？如果是这样，避免见面。如果没有，要注意持续时间。少点目标。

# 13。结对程序

结对编程允许你同时扮演老师和学生的角色。

# 14。写好电子邮件

学会通过简洁明了的邮件抓住你的读者。没人想看你的四页邮件，杰瑞。

# 15。参与社区活动

和志同道合的人在一起会激励你度过低谷。

[![Tree](img/b796b9e831c4048cbc14c38d9ecbe729.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--T7zjbX2p--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/gpvnfky8mmwakwy8eix1.png)

# 16。清理你的树枝

清理您的版本控制分支，就像在您的姻亲来访之前清理您的房子一样。如果不需要，就丢弃；不要只是把它扔在壁橱里。

# 17。不要看门

要包容。不要告诉别人他们不够好，不能进入这个行业。每个人都有价值。

# 18。不断学习

你选择了一个需要不断学习的职业。学会爱它。

# 19。不要放弃

这不会总是容易的。但我们都从同一个地方开始。你能做到的。

# 20。接受让你害怕的任务

如果它没有吓到你，它就不会帮助你成长。

# 21。开始前澄清要求

在深入编写代码之前，您应该理解验收标准。这将节省你的时间和以后的痛苦。

[![React](img/0a5128a44fb8315dc5befe1522f87b40.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--LXs3CSyq--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/iiagpq5fbypu8h9sggqn.png)

# 22。有一个工具箱

有一套你非常熟悉的工具。了解哪些工具服务于哪些目的，以及项目何时可以从使用一个工具中获益。

# 23。学会喜欢建设性的批评

向信任的同事和朋友寻求建设性的批评。它将帮助你成长为一名程序员和一个人。

# 24。思想开放

技术在变化，而且变化很快。不要反对新技术；学习它，然后形成一个观点。

# 25。保持相关性

通过关注出版物、博客、播客和技术新闻，了解最新的技术新闻。

# 26。专注于解决问题

强大的解决问题的能力可以征服任何问题。专注于解决问题的方法。

# 27。保持谦逊

无论你拥有什么头衔，在什么公司工作，保持谦逊。

[![Presentation](img/c191f6e526ca4bb3f7be190fab476731.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--nhgfS-7z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3skzkor6amr46291uvtp.png)

# 28。学会做一场精彩的演讲

学习如何吸引你的听众并进行有效的演讲。

# 29。在开始之前检查所有的解决方案

不要直接进入第一个可能的解决方案。在深入研究代码之前，检查所有路径。

# 30。找到你的位置

科技行业内部有许多部门。找到你最感兴趣的领域，成为专家。

# 31。养成良好的习惯

试着建立一致、健康的习惯，比如排除干扰、限时任务、出席会议、从最重要的任务开始。这可能需要一些时间来适应，但从长远来看是值得的。

[![Debug](img/bd90f6860a5c16d4a9fbb6c97bdc7ed4.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--zD_K7d71--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/scn44lf4b9moiyp1teei.png)

# 32。学会调试

探索浏览器调试器工具。了解使用 IDE 进行调试的详细情况。通过学习调试问题和跟踪错误的最有效方法，您将能够解决甚至是最困难的错误。

# 33。锻炼你目前的技能

仅仅因为你目前知道一项技能并不意味着你不应该练习它。除非有意识地提高，否则技能会随着时间的推移而消失，而这个行业发展如此之快，坚持练习非常重要。走出“我总是这样做”的思维定势，进入“有更好的方法吗？”

仅仅因为你现在有六块腹肌，并不意味着你可以吃一块🍩一天，并保持这种状态。

# 34。了解原因

有些时候你不得不表达你的观点，所以理解背后的原因是很重要的。为什么方案 A 比方案 B 好？提供一个有效的论据，你的观点会更加合理。

[![Money](img/45bf55d91750ad24d52f92bd0e054d6c.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--3rau99gs--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bwftxcf9uigytg4bgfxm.png)

# 35。了解你的价值

你是商品，应该得到适当的报酬。注意你所在地区的行业平均水平。如果你挣的钱少了，是时候和你的经理聊聊了。追求你应得的。

# 36。不要害怕寻求帮助

如果你被困在一个问题上，花了太多时间寻找解决方案，是时候寻求帮助了。我们都是人。我们都需要帮助。向同事寻求支持并不可耻。

# 37。学会学习

人们以不同的方式学习。有些人通过视频教程学得最好，有些人通过看书学得最好。弄清楚自己的学习风格，勤勤恳恳的练习。

# 38。善良一点

有时，您会被要求对同事提供反馈。要善良。你可以表达你对黛博拉缺乏主动性的看法，而不必把她撕成碎片。

# 39。休息一下

连续花 8 个小时编码几乎是不可能的。你会很快筋疲力尽，犯很多错误。所以设置一个计时器，提醒自己停下来休息一下。去散步。和同事喝杯咖啡。离开屏幕会对你的工作效率和质量产生积极的影响。

# 40。追踪你的进度

学习编码需要时间，当你看不到进步的时候会非常沮丧。所以追踪你的成就和朝着目标的进展是很重要的。在你的电脑旁边放一个小清单，每当你完成一件事，就把它写下来，不管它有多小。原子成就会带来更大的回报。

[![React](img/3ae34629927ea0ace2eea1d621ab995a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--97NnU31z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/iu5gfm3zns37g763quxt.png)

# 41。不要依赖框架或库

学习一门语言的细微差别要比学习一个框架或库的细节更好。你不一定需要一个接一个地学习，但是理解为什么一个框架或者库会以这样的方式工作将会帮助你写更干净和更有性能的代码。

# 42。学会热爱代码评审

让别人阅读和分析你的代码可能很可怕，但是可以给你提供宝贵的反馈，让你成为更好的程序员。您还应该努力提高自己进行良好代码评审的能力。

# 43。了解相切空间

学习一些切线空间的基础知识，比如设计，营销，前端开发或者后端开发。它会帮助你成为一个更全面的程序员。

# 44。不要选择舒适的技术；选择正确的

每个项目都有不同的需求，因此我们必须为工作选择合适的工具。尽管选择以前使用过的技术很舒服，但是如果它们不适合项目的需求，就应该探索替代方案。

# 45。为你的错误负责

所有人都会犯错，在你的职业生涯中，你会犯很多很多错误。因此，当你犯了错误时，坦白并承担责任是很重要的。这将与你的团队成员和管理层建立信任。

# 46。审查您自己的代码

在打开拉请求之前，请检查您自己的代码。如果这是同事的作品，你会有什么评价？在请求代码评审之前，首先尝试诊断问题或错误是很重要的。

# 47。从失败中吸取教训

失败只是没有达到预期的结果，并不一定是一件坏事。在我们的职业生涯中，我们都经历过许多失败。从失败中吸取教训。下次你能做些什么不同的事情？

# 48。认识到你的弱点

了解你自己。你的弱点是什么？也许你总是忘记在推送前更新测试。或者你真的不擅长回复邮件。了解你的弱点，这样你就可以积极地解决它们。

# 49。保持好奇

这个行业在不断发展，所以好奇心很重要。如果你有不明白的地方，不管是项目需求还是一行代码，说出来。没有人会因为你要求澄清而批评你，结果你会创造出更好的代码。

[![Book](img/65724ca33588caa4b0f24c675947528c.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ypw1fzcI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qzhic8yqs0pu28bo78lf.png)

# 50。不要试图什么都学

世界上有无穷无尽的知识，要征服它是不可能的。挑选几个主题来掌握，剩下的就不管了。你可以获得其他领域的工作知识或相关知识，但你不可能掌握所有的知识。

# 51。杀了你的宝贝

仅仅因为你写了一些代码，并不意味着你不需要对它产生感情。没有人喜欢他们的工作被扔掉，但是代码有生命周期，所以没有必要对它有所保留。

# 52。支持你的团队

好的团队互相支持。这为尝试新事物创造了一个安全的空间，而不用担心受到惩罚。

# 53。在社区中寻找灵感

找几个你佩服的行业内的人。它会激励你继续做你的项目或者尝试新事物。

# 54。珍惜你的工作

不管你有多少经验，也不管你的职位是什么，你的工作都有价值。给它应有的价值。

[![Phone](img/77a08149434d34078cc0638521e06c2e.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--QyhOZjHL--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9gg3o6rl7retjl9n90ku.png)

# 55。禁用干扰

关闭空闲通知、短信、电子邮件和社交媒体将有助于你集中精力，最大限度地利用你的工作日。如果你花 30 分钟回复他的信息，杰里不会崩溃。

# 56。支持我

试着支持你的团队成员，无论是参加一个重要的演示还是在他们遇到困难时帮助他们。

# 57。该表扬就表扬

如果有人做得很好，告诉他们。积极的强化是与团队成员建立信任的好方法，有助于他们的职业发展。他们也更有可能帮助你。

# 58。测试您的代码

测试很重要。单元测试、回归测试、集成测试、端到端测试。测试你的代码，你的产品会更加稳定。

# 59。计划好你的方法

当您收到新的功能请求或新的 bug 单时，首先要计划好攻击。解决这个问题或者开发这个功能需要什么？即使只花几分钟来计划你的攻击，也能让你避免几个小时的挫败感。

# 60。学习伪代码

伪代码是一项很好的技能，因为它允许你思考复杂的问题，而不用浪费时间写代码。在纸上写下一种方法，运行不同的测试用例，看看缺陷在哪里。

[![Win](img/76c8f9ab855a7bb5fb168ab09c6cf112.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--uQ2UJkJY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6l0lkgxbxz60sbsa3o59.png)

# 61。记录你的成就

如果你在工作中获得奖励，把它写下来。如果你开发了一个关键的特性，把它写下来。你会积压一大堆有助于升职或在艰难的一天鼓舞士气的事情。

# 62。学习编程基础

学习一些基本的排序和搜索算法以及数据结构。这些是与语言无关的，可以帮助你解决跨语言的问题。

# 63。为长寿选择技术&可维护性

尽管测试最新的技术很有趣，但是选择那些在企业应用程序中易于维护的技术。你的团队在未来的岁月里会感谢你的。

# 64。学习设计模式

设计模式是架构代码的有用工具。您可能不需要在每个项目中都使用它们，但是对它们有一个基本的了解将有助于构建更大的应用程序。

# 65。减少歧义

不要写令人费解的代码来炫耀你时髦的编程技巧，而是以可读性和简单性为目标。这将使你的团队成员更容易做出贡献。

[![Debt](img/0c12d9dd30e61002eb1c931e3ea32e9d.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--fX3cKq9j--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6rg8pvj8ezhreshjjx6o.png)

# 66。还清技术债务

技术债务会对性能产生巨大的影响，所以如果你能够重构，你就应该重构。

# 67。经常运送

与其每月发布一次大规模升级，不如更频繁地发布较小的变更日志。你不太可能引入错误和突破性的改变。

# 68。尽早并经常承诺

尽早提交并经常提交是确保您的工作保持整洁的最佳方式，也减少了意外恢复重要变更的压力。

# 69。学会何时寻求帮助

你不仅不应该害怕寻求帮助，而且应该学会什么时候寻求帮助。在寻求帮助之前，你应该试着解决问题，并记录你所做的努力。但是，当你被一个简单的问题难倒超过一个小时时，成本超过了收益，你应该向同事求助。

# 70。提出有效的问题

问问题的时候，尽量具体。

# 71。获得关于未完成工作的反馈

你不需要完成工作就能得到反馈。如果你不确定方向，请一位信任的同事检查你的解决方案的有效性。

[![Read](img/ad7865bf142efd49eca556bc1f87fc4a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ajqLQ2p0--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/pizzzb8twdan6231fdjq.png)

# 72。阅读文档

文档是了解技术真相的最纯粹的来源，所以学会阅读它可以帮助你迅速成为专家。

# 73。尝试所有的事情

没有什么能阻止你尝试解决问题。你会失去什么？

# 74。在会议上大胆发言

您的想法和意见很有价值，因此参加会议将有助于您与团队和管理层建立融洽的关系。

# 75。跨团队协作

如果你有机会和公司的另一个团队合作，那就去争取吧。

# 76。有激情的项目

当你一周工作 40 个小时时，花时间做一些有激情的项目是很重要的。他们帮助你重新燃起对编程的热爱，尝试你在工作中可能接触不到的新技术。

# 77。定义你的职业目标

对自己的职业理想轨迹有个概念很重要。如果你不这样做，你就是在无目标地射箭。

[![Talk](img/3d0f0fbe5e4267f76a39d8939d982f3b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--SPitWbzG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fjqq1ghzpz4qqcjhoew6.png)

# 78。参与对话

评论博客，参与 Twitter 帖子。参与社区活动。作为一个积极的旁观者，你会比一个局外人学到更多。

# 79。区分任务的优先级

学会分清任务的轻重缓急将有助于你提高工作效率。保留一份日常任务和长期任务的清单，并按最重要的来排序。

# 80。不要忽略细节

细节会对项目产生很大的影响。

# 81。相信你的队友

你的队友被雇佣是因为他们的技能。使用他们并信任他们来完成工作。

# 82。学会授权

如果你处于领导位置，学习如何有效地授权。这会节省你的时间和挫折。你不能什么都做。

# 83。不要拿自己和别人比较

你唯一应该比较的是昨天的你。

# 84。让你的周围都是盟友

学习编程将是一个漫长且不容易的旅程。让你周围的人帮助你，鼓励你继续前进。

# 85。不要为了规模而开始

开始规模化肯定会让你不知所措。构建时要考虑可伸缩性，但是不要等到需要时才开始伸缩。这样你就不会因为不必要的膨胀而淹没你的团队，但是你保持了成长的能力。

# 86。权衡性能影响

如果你想使用一个很酷的新技术，你应该权衡这样做的性能影响。你能在不影响性能的情况下实现类似的东西吗？如果是这样，你可能要重新考虑你的方法。

# 87。不要歧视

不要歧视新技术或新想法。对学习新技能的可能性持开放态度。也不要歧视人。我们都值得尊重。

# 88。申请你不胜任的工作

你永远不会满足一份工作的所有要求。所以抓住机会申请吧！你会失去什么？

# 89。模块化你的代码

您可以将所有代码写在一个长文件中，但这是不可维护的。通过模块化，我们确保我们的代码易于理解和测试。

# 90。不要只是复制和粘贴

如果你打算从堆栈溢出中复制并粘贴一个解决方案，你应该确切地理解*它做什么。对你选择引入的代码要有意识。*

# 91。创造一个鼓舞人心的环境/环境

如果你喜欢你的工作空间和技术设置，你会更有动力去工作。让它成为你。

# 92。记住你从哪里来

我们都是从同一个地方开始的。随着你的技能和职位的发展，不要忘记你来自哪里。

# 93。试着保持乐观

如果出了问题，试着乐观一点。明天是新的一天。乐观将有助于你的团队活力和你的心理健康。

# 94。不断重新评估您的工作流程

仅仅因为某样东西现在有效并不意味着它会一直有效。重新评估您的工作流程，并在必要时进行调整。

# 95。学习如何在家工作

如果你有能力在家工作，那就学会有效地在家工作。找一个独立的办公室，远离干扰。Boneskull 写了一篇关于在家工作的文章，你应该看看。

# 96。可访问性代码

可访问性不是事后才想到的，也不一定很难。每个人都应该能够使用你的产品。

# 97。兑现你的承诺

如果你告诉某人你会在某个日期前交付某样东西，兑现这个承诺。如果你不能在截止日期前完成，早点说出来。

# 98。积极主动

如果你有一些额外的带宽，找一个任务来帮助你的团队！他们会感谢你的积极主动。

# 99。建立一个惊人的投资组合

一个好的投资组合会让你与众不同。利用这个机会展示你的编程和设计技能吧！

# 100。记住你为什么喜欢编程

你进入这个行业是因为它激发了你的兴趣。如果你感到沮丧和怨恨，休息一下。给自己空间，重新点燃编程的激情。

# 101 分享你的知识

如果你学到了很酷的东西，分享一下吧！出席当地集会或会议。午餐时教你的同事或学员。分享你的知识巩固你的知识，同时传播财富。
