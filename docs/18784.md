# ELI5:有用的单元测试

> 原文:[https://dev.to/deciduously/eli5-useful-unit-testing-1gon](https://dev.to/deciduously/eli5-useful-unit-testing-1gon)

我一直在努力写更完整的测试，但是经常觉得自己有点多余。有时候我写的测试感觉就像是在验证语言是否如预期的那样工作(如果我传入一个参数，它在本地范围内是可访问的吗？这个接受整数类型的数学函数拒绝非整数输入吗？)，或者框架工作正常(如果我传入一个 string prop 来呈现为一个文本节点，React 真的会呈现这个文本节点吗？).

在动态语言中，对正确类型的额外检查是有意义的，但是在类型安全的语言中，我非常乐意相信我的编译器。结果，我的单元测试感觉很稀疏。以端到端或往返的方式编写包含应用程序多个部分的集成测试感觉更自然。
