# 在我的机器上工作

> 原文：<https://dev.to/dowenb/works-on-my-machine-17eb>

最初发布在我的博客上:
[https://www.dowen.me.uk/it-works-on-my-machine/](https://www.dowen.me.uk/it-works-on-my-machine/)

所以，你刚刚发现了这个伟大的错误，它返回“无法重现”，或者你已经被告知“它在我的机器上工作”，经典的权利？

作为一名测试人员，在这种情况下，很容易产生防御性。有人告诉我这是我编的吗？我是不是不被尊重，如果 bug 没有重要到需要修复，我为什么要提出来？

我的建议是，反过来考虑一下。是时候了解技术了！可以添加哪些细节来帮助确定问题的根本原因？您能做更多的调查并帮助理解您测试的环境和开发人员试图重现的环境之间有什么不同吗？

您可以通过以下方式增加“它在我的机器上工作”的价值:

1.  确定您观察到问题的确切构建或部署，并检查这是否与开发人员正在检查的构建相同？你能检查一下他们使用的版本吗？也许问题已经解决了，但是你还没有得到最新的代码。
2.  确定你们都在使用什么平台。如果是 web，确保不仅检查浏览器，还要检查浏览器的版本。如果是移动的，安装的是什么版本的 Android 或者 iOS？
3.  有没有额外的日志记录来帮助添加上下文？如果您可以在日志中发现抛出的异常，这将有助于识别问题。
4.  如果您能继续重现该问题，您能让开发人员访问您的机器或环境吗？如果你是远程的，你能设置一个屏幕共享会话吗？至少，应该很容易捕捉到这些步骤的视频。为此，我使用开放广播软件工作室[https://obsproject.com/download](https://obsproject.com/download)，但有许多替代方案。
5.  如果这些有助于找到问题的根源并得到错误修复，那么尝试在您通常的错误报告中更多地使用它们，看看是否可以减少“在我的机器上工作”的次数。
