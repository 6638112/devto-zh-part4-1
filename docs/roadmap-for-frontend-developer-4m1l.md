#### 基础知识:

**👉HTML:** 永远不要低估 HTML。HTML 中有很多东西没有被人们使用，但是如果你知道它也是可用的，你会感到惊讶。

**👉CSS:** 这很棘手也很有趣，因为它设计了页面的样式，有时它的工作方式有点令人惊讶。你用它来美化你的页面，你自己定义每一个属性。就像建造自己的家一样。

**👉JS:** JS 是唯一将逻辑和编程带入前端开发者生活的东西。它非常灵活，令人惊讶。它使自己的网站功能，就像我们看到幻灯片，我们从验证之前发送到服务器。动态地向服务器发送请求。

HTML 只是一种标记语言，它永远不会给你错误。CSS 也不会抱怨你写错了代码，但是 JS 会这样做。一旦你学会了基础知识，你就可以进入下一部分了。

学习 CSS 框架是非常重要的，因为它会让你的生活更轻松。你不需要一个人做所有的造型。您可以使用框架中的标准预定义样式。如果您对 CSS 有很好的理解，那么您可以很容易地覆盖 CSS 属性并自定义页面的样式。

#### 包管理器

包管理器是一种编程语言的工具，可以自动完成安装、升级、配置。它还创建项目环境，并轻松导入外部依赖项。

在学习 CSS 框架之后，你可能会想到学习 JS 框架，但是当你开始学习的时候，你最终会来到包管理器。

> *NPM 是 node 官方包经理。*

有些人不使用 npm，他们使用的是 **Yarn** (非官方的包管理器)

> Yarn 是由脸书、Exponent、Google 和 Tilde 等公司开发的一个常见项目。

所以你可能会认为我告诉了你不同的事情。你可以用 npm 生活，这没什么不好，但 yarn 是 npm 的增强版。

几年或几个月后，你会知道什么是纱线的优势。

**❌** 不要直接跳到 yarn，因为师父一定也知道 npm。

#### CSS 预处理器

一个 **CSS 预处理器**是一个让你从预处理器自己独特的语法中生成 [CSS](https://developer.mozilla.org/en-US/docs/Glossary/CSS) 的程序。有很多 CSS 预处理器可供选择，然而，大多数 CSS 预处理器都会增加一些纯 CSS 中没有的特性，比如 mixin、嵌套选择器、继承选择器等等。

这些特性使得 CSS 结构可读性更强，更易于维护。

4.  **POSTCSS**

#### 测试

在测试中，我们测试我们的 UI 是否良好，因为我们的代码将在不同配置的不同设备上运行，不可能每次都手动测试所有地方和每个页面。所以我们写了一个测试来自动化这些事情。有 3 种类型的测试

👉单元测试:单元测试是一种检查一小段代码是否做了它应该做的事情的测试。单元测试检查应用程序的单个组件。

👉集成测试:集成测试是一种检查模块的不同部分是否一起工作的测试。在集成测试中考虑集成模块的行为。

👉功能测试是一种软件测试，通过这种测试，系统根据功能要求/规格进行测试。
