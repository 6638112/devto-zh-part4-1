# Scrum 对你意味着什么？

> 原文:[https://dev.to/dealeron/what-does-scrum-mean-to-you-3gn5](https://dev.to/dealeron/what-does-scrum-mean-to-you-3gn5)

Scrum:一个框架，在这个框架中，*的人可以解决复杂的适应性问题，同时富有成效和创造性地交付最高价值的产品。*

 *在我的前一篇文章中，我表达了 Scrum 团队成员之间仪式的美妙之处，但这仅仅是 Scrum 所能提供的最好品质的表面。

> *“Scrum 很简单，* ***做着*** *Scrum 很难”*

这段引用自 [Scrum Guide](https://www.scrumguides.org/docs/scrumguide/v1/scrum-guide-us.pdf) 的话让我们知道，虽然 Scrum 的框架可能很容易理解，但确实很难掌握。每当有人对学习 Scrum 感兴趣时，我都会记住这一点。

当我向人们介绍自己时，不可避免的问题是“你做什么工作？”出现，在回复*“我是 Scrum Master”*的时候，经常会收到一个奇怪而不解的眼神回敬。

除了可能的最高价值是 Scrum 的主要焦点之外， ***人*** 也将永远是一个焦点。我们必须永远记住，没有我们的同胞，我们就不会尽可能地有效。这就是为什么我着迷于和对 Scrum 感兴趣的人讨论，以及 Scrum 能提供什么。

最近在我们公司的[，两位同事找到我，他们是我曾经经常共事的一个部门的项目协调员，但在我的新职位上，他们之间的关系并没有那么紧密。聪明热情的年轻人对 Scrum 有一些疑问，以及如何将它整合到他们不断变化的过程中。](https://www.dealeron.com/join-our-team/)

“Scrum 适合我们目前正在做的事情吗？”简短的介绍后，问题提出来了。他们的部门最近与另一个部门进行了整合，不仅在工作流程上，而且在座位上。他们都需要更多地了解各自部门的流程，如何将它们整合在一起，并简单地了解彼此。

当然，这是一个完美的例子，在这个例子中，他们可以将 Scrum 框架应用于沟通和共同目标，我让他们了解了一些仪式，这些仪式可以让他们在这个过渡时期受益于高质量的参与和透明度。这两位先生都受到了启发，我很高兴地说，他们不仅开始在他们的部门内实施 Scrum 实践，而且他们还很快成为了 Scrum 认证，成为了组织内 Scrum 大师的音乐！

在另一个与工作无关的场合，因为那是一个 5 岁孩子的月球弹跳生日聚会，我也不知何故无意中发现了一段关于 Scrum 的对话。在那个炎热的夏日，当其他爸爸围在冷藏箱旁喝着冰镇啤酒时，工作和旅行的话题就来了。其中一位父亲告知了他最近的海外之行，以及他的公司如何希望他成为 Scrum 认证，以帮助将框架安装到他管理的开发团队中，他不太愿意，因为他觉得这是强加给他的。我让他知道当我报名参加 Scrum 认证时，我也曾经不确定我将进入什么领域，但我也告诉他在完成了我在 [Fox Hedge Ltd.](https://www.foxhedgeltd.com/) 的课程后，我变得多么开明

虽然这位不情愿的经理在我们继续讨论时明确表示他更喜欢[瀑布模型](https://en.wikipedia.org/wiki/Waterfall_model)，但他后来承认他欣赏 Scrum 的一点，*“快速失败，我想知道我们没有快速浪费我们的时间！”*。 [**Fail-fast**](https://en.wikipedia.org/wiki/Fail-fast) 是一种系统设计，允许在错误检测期间进行通知，以允许相应地解决它。虽然这可能更符合硬件或软件设计，但在我们团队的高质量互动中，Scrum 和交互之间也是如此。当一个团队进行最有效的沟通时，他们会很快知道某个流程或系统是否无法实现某个目标，并通知其他人，以便做出适当的调整… ***透明度！*T11】**

瀑布模型可能不允许同样快速重新调整以进行适当的更改，因为官僚主义或先前决定的决定可能是现有合同的一部分。这可能会导致错过可交付成果或截止日期，而没有人能够在相关各方之间进行沟通，以通知预先安排的工作将无法满足目标。我很高兴地告诉他，在我们公司，当我们必须在一个项目中进行调整时，我们很快就认识到了这一点，即使这可能很困难，我们的团队成员也能很快适应高质量的开发… [***敏捷！***](https://www.agilealliance.org/agile101/)

我自己、项目协调员同事以及孩子生日派对上的父亲之间的交流让我认识到，不仅我们组织内部存在提高透明度、协作和愿景的机会，许多其他组织也存在这种机会。很多次，当我参加网络研讨会时，我看到这些相同的短语，其他与会者也有类似的问题。我对自己说:“我们不是唯一的！”很高兴知道其他公司，甚至那些在团队中成功实施 Scrum 的公司，也有这些需要继续改进的地方。这就是为什么我很高兴通过启动 [*元 Scrum* 的概念和实践来开始我们不同团队和部门之间的合作努力。](https://app.box.com/s/vpvhdpxjdf)

在不久的将来，我们的大量团队领导将开会讨论产品协作，这将是我们的人统一愿景、路线图、发布计划和架构，以及解决团队在其范围内未能解决的任何障碍的关键时刻。在此期间，我会回忆起我在同事中的讨论，在生日聚会上持怀疑态度的偏爱瀑布的经理，或者我以前与人的任何其他谈话，以及我们如何才能一致改进。

***人*** 是我们为之工作并与之共事的人，无论是你团队中的人，你为之开发产品的人，还是你甚至不知道你影响了的人。倾听这些人的心声，在工作时将他们牢记在心，并解决问题以改善人口状况是愿景的根本目标。这就是 Scrum 对我的意义……与人交往的机会，做出更明智的决定，并且不要忘记互相学习， ***人民！***

人员超过流程。
