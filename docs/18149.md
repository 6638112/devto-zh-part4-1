# 通过对开源做出贡献来学习

> 原文：<https://dev.to/subbramanil/learn-by-contributing-to-open-source-4kf>

在最近的谷歌 I/O 中，谷歌宣布 **Kotlin** 成为编写 Android 应用程序的首选语言。自从几年前谷歌宣布， **Kotlin** 是首选语言，并为 android studio 添加了 Kotlin 支持，这个消息对我来说并不意外。

一年前宣布的时候，我注册了很多课程，完成了其中的几门。我对完成练习和样本项目感觉很好。然而，这仅仅持续了一段时间。由于我个人(和一个很棒的女孩结婚了)和工作的承诺增加了一倍(他们解雇了一些同事)，我暂时停止了学习科特林。

当我再次开始时，我挣扎了很久，不知道从哪里开始，因为我正在学习几门科特林课程。虽然我注册学习科特林的课程相当不错，但我意识到我更喜欢阅读。我不介意阅读文档或几个博客来理解一个概念，并自己尝试。观看视频课程和跟随教程对我来说太慢了。

我在学习的时候做一些小项目。虽然个人项目有助于理解基本原理，但挑战很少。

1.  关注范围:由我来决定项目的范围。有时我倾向于扩展我对个人项目的想法，我花更多的时间在与科特林无关的事情上。
2.  **关注技术栈:**还是由我来决定应用的架构和设计，以及要使用的库。在寻找这些技术问题的答案时，我忘记了我的目标(学习科特林)。为了前任。，当我开始寻找与数据类相关的博客时，我浪费了大量的时间，不知何故，我在使用 MVVM + RxKotlin + Dagger2 构建的示例应用程序上关注博客。虽然了解这个设置很好，但它并没有直接帮助我实现最初的目标。

就在那时，我决定从头开始学习科特林，但这次有了更清晰的计划。

所以我之前在学习科特林时犯的错误是，

1.  试图从太多的来源了解科特林
2.  没有从最适合自己的开始(阅读和视频课程)
3.  没有明确时间表的计划
4.  个人项目没有明确的范围

我从自己的经历中吸取教训，制定了一个计划。

**目标:**学习&在 Kotlin 开发应用程序，并于 2019 年 9 月发布到商店

做这件事的计划，

1.  买一本学习 Kotlin 的书: [Kotlin 编程:大书呆子牧场指南](https://www.amazon.com/Kotlin-Programming-Nerd-Ranch-Guide/dp/0135161630)
2.  在有限的范围内做个人项目:一个 Android 应用程序
    1.  概述了应用程序的基本思想
    2.  确定应用程序的功能范围
    3.  确定应用程序的技术范围
    4.  使用草图应用程序设计简单的用户界面/UX 流程

## 通过贡献开源来学习

我参加了去年的 Hactoberfest，并为 [Beautus S .古梅德](https://dev.to/sduduzog)的 [Slim-Launcher](https://github.com/sduduzog/slim-launcher) app 投稿。我[完成了我的啤酒节挑战](https://dev.to/subbramanil/hactoberfest-challenge-completed-e49)，也拿到了我的 t 恤。回馈社区的感觉真好。

当回首往事时，

1.  我在做这个项目的时候学到了很多东西。这个应用程序使用 Kotlin，对我来说，这是一个很好的锻炼技能的机会。
2.  该应用程序发布到商店，并有一个明确的技术范围。它帮助我专注于一件事，并在界限内工作。
3.  我非常喜欢与项目开发人员 Beautus S .古梅德互动，并在这个项目上工作。虽然我们都是陌生人，通过他在 dev.to 上的博客联系在一起，但我喜欢参与他的项目，喜欢 Beautus S .古梅德的坦率和欣赏的性格。

这个练习让我意识到，通过从事具有明确设计和功能的项目，我可以学到更多东西。我可以查看代码，修复一个 bug，或者添加一个特性，并从项目中学习。

这让我在学习计划中又增加了一步，

1.  买一本学习科特林的书
2.  从事个人项目:Android 应用程序
3.  **参与开源项目**

到目前为止，我已经为，

*   [超薄发射器](https://github.com/sduduzog/slim-launcher)
*   [Android-Lock9View 的分支](https://github.com/TakWolf/Android-Lock9View)
*   [开发机器人](https://github.com/subbramanil/DEV-Android)

目前，我正在处理项目中的一个增强请求，

*   [简单的待办事项](https://github.com/subbramanil/SimpleToDo)
*   [简单-安卓](https://github.com/subbramanil/simple-android)

我通常会在做项目时记下我的心得，我计划在提交和合并 PRs 后写一系列博客。
