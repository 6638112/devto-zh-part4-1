# 与你的开发者建立关系

> 原文:[https://dev . to/jessekphillips/making-a-relationship-with-your-developer-4enp](https://dev.to/jessekphillips/making-a-relationship-with-your-developer-4enp)

我发现我通常与我测试的开发人员有着良好的工作关系。我也发现对其他人来说并不总是这样。这一直困扰着我，为什么会发生这种情况，特别是因为我和任何一方都没有问题。

我可能已经确定了原因，但没有任何证据支持它。我的测试方法更多的是对实现的好奇。“您使用 SVG 实现了一个日期选择器，SVG 是如何帮助实现的？IE 支持日期吗？SVG 如何伸缩并保持事件标记在正确的位置。当用户选择黑暗模式时，你是如何处理交换 SVG 的？”

这种方法为正在讨论的替代方案和可能有助于更好测试的修改打开了大门。由于 SVG 需要每个日期都有一个关联的动作处理程序，所以我可以验证每个日期都有一个处理程序。开发人员和我可以一起工作，在一个解决方案，我们都感到舒服，验证功能。如果我的测试失败是因为他的工作还是我的工作，他们也欢迎提问，我们会通过假设和进一步的测试找到答案。

所有这些都占用了开发人员宝贵的时间。除了用户所看到的，还需要对系统有一定程度的了解。开发人员没有所有的答案，也没有时间去找到它们，这是我自己进一步挖掘的地方，甚至可以报告我的发现。

我将我的关系从一个固有的对立的分级系统转变为一个我们合作以达到质量特性的最终状态并使之进入生产的系统。
