# Mapbox 是谷歌地图价格上涨的一个有价值的替代品

> 原文：<https://dev.to/yalantis/mapbox-as-a-worthy-alternative-to-google-maps-price-hike-3572>

谷歌地图是在选择合适的地图整合时想到的第一个全球地图服务。它赢得了全球 web、iOS 和 Android 开发者的信任。谷歌地图很容易与 API 集成，可以服务于如此多的用例，为您的网站提供交互式地图，为出租车和旅游服务提供基于位置的服务，为巨大的物流平台提供导航。

然而，从 2018 年 7 月开始，谷歌大幅减少了免费动态地图 JavaScript API 调用的数量。之前的限额是每天 25000。现在，限额是每月 28000。谷歌还开始要求所有 API 调用使用有效的 API 密钥，该密钥必须与谷歌云平台账户相关联。不支持无钥匙访问。因此，所有对地图 JavaScript API 和街景 API 的无键调用都会返回带有“仅供开发使用”水印的低分辨率地图。

不会生成很多地图视图的较小网站和应用程序不太可能达到每月 28，000 次的限制。但是，如果你的网络或移动解决方案积极使用更复杂的谷歌地图服务呢？例如，如果你使用允许用户交互的动态地图会怎么样？在这种情况下，你的网站或应用程序可能会面临成本增加。

有没有谷歌地图的替代品？考虑到谷歌地图的高成本，我们想弄清楚 Mapbox 是否是一个值得的谷歌地图替代品。在我们深入研究我们的发现之前，让我们先来定义一下什么是 Mapbox。
