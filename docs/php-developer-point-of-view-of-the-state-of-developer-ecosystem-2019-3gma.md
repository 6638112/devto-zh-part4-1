# 《2019 年开发者生态系统状态》PHP 开发者观点

> 原文：<https://dev.to/inspector/php-developer-point-of-view-of-the-state-of-developer-ecosystem-2019-3gma>

[**开发者生态系统状态 2019**](https://www.jetbrains.com/lp/devecosystem-2019/php/) 是 Jetbrains 发布的一份报告，旨在调查大多数已知技术的软件开发者的习惯。

作为 PHP 开发人员，我从个人角度仔细阅读了这份报告。这份报告和每年的许多其他报告一样，只是向我们展示了一些统计数据，并不是事实的来源。

我坚信 PHP 7 为 PHP 打开了大门，推动了企业环境中更多的采用。只差一点:“生态系统”。

这就是为什么 Laravel 正在试验在 PHP 世界中大量采用。它的生态系统为构建业务关键型应用程序提供了合适的基础，因为您知道您将拥有合适的工具来发展，从而确保您的业务安全。

## 收养

30%的开发者经常使用 PHP。我觉得真的真的很好。围绕 PHP 建立职业生涯可能是一个非常好的想法，而且它可能非常有利可图，因为 PHP 是如此广泛传播。

## 开发者工具

一些索引告诉我，与其他技术领域的开发人员相比，PHP 开发人员并没有过多地自动化他们的工作。

34%的人使用纯 PHP，而不是 Twig、Blade 等模板引擎。不太好。这意味着在复杂的技术问题上浪费了大量时间。

大多数 PHP 开发人员不使用任何东西来分析或测量性能。报告说。

我们中的许多人几乎每天都交付新的代码变更。如果我们等待用户报告错误或瓶颈，我们实际上是将 bug 发现外包给了用户。用户不会报告错误，他们最终只会停止使用你的应用程序。

这就是为什么对您的业务健康至关重要的应用程序无法进行手动监控的原因。如今，大多数开发人员使用的手动监控过程成本高、耗时，并且存在巨大的误差。

最好的 IDE 是 PHPStorm。我是 happy PHPstorm 的数百万客户之一，我真的相信 Jetbrains 应该得到这个结果。

## PHP 7 的良好用法

许多开发人员已经从 PHP 5.x 版本迁移到更新的 7.x 版本。在一年的时间里，PHP 5 的份额从 29%骤降至只有 14%。

正如我上面所说的，PHP-7 为在更多的企业环境中支持 PHP 提供了正确的基础。

## 结论

感谢 PHP-7、Laravel 和 Jetbrain 本身，PHP 开发人员可以重新思考他们的业务，将其视为“企业就绪”，但他们需要明白这不是一个技术问题。更多的是心态问题。

我的产品适合这个场景，帮助 PHP 开发人员用已知的最佳策略发展他们的业务:为他们的客户创造有意义的体验！
