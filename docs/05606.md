# 发布我的第一个游戏

> 原文:[https://dev.to/humrochagf/publishing-my-first-game-3dm4](https://dev.to/humrochagf/publishing-my-first-game-3dm4)

原文:[https://humberto.io/blog/publishing-my-first-game/](https://humberto.io/blog/publishing-my-first-game/)

游戏从一开始就总是把我和科技联系在一起。

我的父亲和我，我们制造了我们的第一台计算机(奔腾 286 ),我记得做的第一件事是玩一些 DOS 游戏，如波斯王子和月球登陆者。我学了一堆 CLI 命令，就是为了玩我喜欢的游戏。

玩游戏和制作游戏的热情作为一种爱好跟随着我。我在我的博客上有一个 [pygame 系列的帖子](https://humberto.io/tags/pygame/)，在那里我浏览了游戏开发的基本概念，试图向开始学习它的人解释它们。

时间过去了，一天一天的沉默开始占用我大部分的精力，我的 pygame 系列已经有一段时间没有看到新的帖子了，我的爱好也是如此。

然后，去年 12 月的假期，我发现了 [Pyxel](https://github.com/kitao/pyxel) 。这是一个 Python 复古游戏引擎，遵循了 [pico-8](https://en.wikipedia.org/wiki/Pico-8) 的限制，并带有一个精灵、tilemap 和音乐编辑器。

我喜欢像素艺术游戏，所以我决定利用假期从头开始写我的第一个完整的游戏，直到它出版。这个游戏是对我小时候玩过的旧的月球登陆者 DOS 游戏的致敬。

[![pyxel lander landing](../Images/6ed01eb0cbcef2ffa979e99894be1910.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uJhXTbwE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kumixuprpvae4pa9ym7s.gif)

这个主题选择对我来说很有意义，不仅因为我小时候玩过，还因为我和一个朋友在我的计算机科学课程的一个[项目](https://github.com/ravishi/lunar-lander-ex/commits/master)中编写过代码。

构建游戏的过程相当有趣，在一天结束时，我让登月舱在屏幕上飞行。在我一头扎进程序性的月球表面建筑的第二天，当我正要解决碰撞检测问题时，我的假期结束了，日复一日的忙碌又回来了，我的项目变得陈旧了。

更多的时间过去了，在一次午休中，我回复了一条推特，提出了以下问题:

我年轻的时候有过数不清的 RPG Maker 未完成的游戏，还有其他许多不同工具的游戏。与我的工作或我的开源项目有什么不同，让我完成了它，但却没有完成我的游戏项目？

我决定完成这个游戏，去了解是什么阻碍了我...我已经完成了游戏，并且发布了。你可以在[https://humrochagf.itch.io/pyxel-lander](https://humrochagf.itch.io/pyxel-lander)看看并玩玩

另外，我把它作为开源发布了，任何有兴趣看代码的人都可以去[https://github.com/humrochagf/pyxel-lander](https://github.com/humrochagf/pyxel-lander)

## [](#wrapping-up)包装完毕

经历这整个过程让我意识到一些事情:

*   游戏开发就像九头蛇一样。当你在解决一件事情的时候，你的大脑正在创造新的功能。因此，就像软件开发的任何其他领域一样，关注于 [MVP](https://en.wikipedia.org/wiki/Minimum_viable_product) ，并将新特性保留到下一个版本或项目中。
*   如果这是你的第一个游戏，试着不要把事情搞得太复杂，更重要的是经历从想法到出版的整个过程，而不是做出完美的游戏。
*   像 Pyxel 这样范围明确且有限的工具非常适合制作你的第一个游戏。保持简单对你很有帮助。
*   当你需要帮助时，打电话给你的朋友。感谢海罗·杰尔对 MacOS 构建的帮助。
*   为乐趣而编码是很棒的事情，可以学习新的东西，并在过程中获得乐趣，让我们做更多的事情。
