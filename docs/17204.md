# 如何在共享主机上上传 Laravel 项目？

> 原文：<https://dev.to/msamgan/how-to-upload-a-laravel-project-on-shared-hosting-2o1h>

我可以理解，如果你正在寻找这篇文章，你是在绝望的需要和 VPS 不是一个选项提供给你。我不会告诉你 VPS 是一个好的选择，但是如果你有，那就去做吧。

上传文件很简单，你所要做的就是。压缩和上传，棘手的部分如下。

1.  配置缓存问题
2.  公共目录问题。

我告诉你如何逐一解决这两个问题。

## 配置缓存发布

有两种可能的方法来解决这个问题。

1.  第一个是通过 URL 使用 artisan 命令，是的，你没听错。我们可以通过 URL 使用所有 artisan 命令。这里是更多细节的链接。[点击此处了解更多详情](https://laravel.com/docs/5.0/artisan)

2.  您可以手动执行该命令将执行的工作。也就是说，您必须删除旧的缓存。删除旧缓存后，下次运行时将自动生成新的配置缓存。要做到这一点，你所要做的就是
    *根目录- >引导目录- >缓存*
    并删除所有文件，我们就可以开始了。

## 公共目录发布

要解决这个问题，你所要做的就是用一些规则将一个. htaccess 文件添加到你的根目录。你可以在这里找到[的完整细节](https://codebysamgan.com/htaccess-to-remove-public-from-laravel-app/)
