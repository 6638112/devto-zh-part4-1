# 专攻前端框架有什么缺点吗？

> 原文:[https://dev . to/nprimak/any-cons-to-specialized-in-a-front-end-framework-30 E3](https://dev.to/nprimak/any-cons-to-specializing-in-a-front-end-framework-30e3)

* *这篇文章是我在[www.multimedia-minds.com](http://www.multimedia-minds.com)**的博客的[转贴](http://www.multimedia-minds.com/2019/07/02/why-specializing-in-tech-can-be-double-edged-sword/)

有时候每天做同样的事情会让人觉得有点老四年前，我在一家初创公司找到了第一份工作。这是非常值得骄傲的一天。我在星巴克给父亲打电话，颤抖的手里拿着聘书，想象着自己在接下来的一周乘坐地铁，走进市中心崭新闪亮的办公楼，感觉自己像一个成熟的城市女性，终于进入了我奋斗了两年才被认真对待的领域，一点一点地建立起我的投资组合。

那是一个特殊的时刻，我至今仍记忆犹新，但我没有预见到，也没有人真正警告过我的是，一旦你真的被雇佣了，就很难站在新技术的前沿。当时很酷的新前端框架是 AngularJS，我抓住了掌握这个框架的机会，因为 a)工作需要它，b)它看起来会持续一段时间。

[![picture of pigeon with text "I will not be defined"](../Images/de7d589c999b5bb66b7482060f53470e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--96-oA3Fv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.multimedia-minds.com/wp-content/uploads/2019/07/pidegon.png)

有很多 Javascript 前端框架
现在，我并没有完全错误地坚持使用它，但是我没有想到四年后我仍然在使用它，不仅仅是同一个框架，而是同一个版本的框架。对于那些不知道的人来说，Angular 今天的最新版本是 Angular 8，它与 AngularJS 有很大的不同。事实上如此不同，学习它实际上就像学习一个全新的框架。此外还有 React，这是一个迅速流行起来的框架，实际上它比 Angular 被更广泛地采用。除此之外，还有 Vue，它不太受欢迎，但也很快获得了关注。

我并没有打算被归类到 AngularJS，但它发生了，因为在过去的三年里，我在一个城市工作，那里有许多政府相关的工作和许多企业公司，它们行动缓慢，使用像 AngularJS 这样过时的技术。我通常还会发现，在一天工作结束时，我已经没有多少精力来自学其他框架了。作为一个不仅仅生活和呼吸在日常工作中的人，如果我要在工作之外编码，我希望它是编码游戏或用处理或其他生成艺术技术进行创造性编码项目。我想还有其他人在读这篇博客，他们渴望创造性地表达自己，也有同样的感受。

[![picture of React, Vue, Angular logos](../Images/0812d058b65f4d39dcce1bb4c5db0333.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--KZjFTlKE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.multimedia-minds.com/wp-content/uploads/2019/07/js-frameworks.jpg.png)

Angular 有很多版本，React 也有不少版本。它一直在变化。许多开发人员会告诉你，在这个行业工作并保持一种爱好是很难的。但是，如果不锻炼我的创造性肌肉，我会觉得我的灵魂缺失了一部分，当一切都结束时，我仍然有表达自己的冲动。我并不总是有精力去学习最新最棒的框架，因为整天编码真的会让你筋疲力尽。所以现在我被困在 AngularJS 的这个专业，这个专业在一个不断变化的领域中不再特别受欢迎，但也是一个很难摆脱的专业，因为我已经有了四年的经验。

[![illustration depicting evolution of man but subtitled with angular versions 1 (JS),2,4, and 7](../Images/9d5ce57b1860a7db9a9845220a217ff5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--kJvX-5NB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.multimedia-minds.com/wp-content/uploads/2019/07/angular-evolution-heros-720x404.png)

我不想以这种悲观的方式结束，因为谢天谢地，我的爱好项目也有助于向未来的雇主展示，我有能力自学其他语言和框架，即使它们不在前端工具集之内。然而，我认为这个问题没有得到足够的讨论。有很多文章是关于进入技术行业并获得第一份工作的。下一份工作总是比第一份容易找到，这也是事实。不过，一旦你习惯了，也很容易变得自满，因为推动你不断学习和开拓新领域的外部动力少了很多。这种自满很难防范，但你必须防范，因为否则你可能会发现自己在职业上停滞不前，没有动力，不快乐。
