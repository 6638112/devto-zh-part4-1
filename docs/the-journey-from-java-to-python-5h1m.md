# 从 Java 到 Python 的旅程

> 原文：<https://dev.to/claudiodavi/the-journey-from-java-to-python-5h1m>

### 简介

我绝对不是那种 13 岁就开始编程，20 岁就成为天才的巫师小孩。我 18 岁开始编程，从 Java 开始。我在这方面很糟糕，第一次看到时，我认为这是魔术。
我现在 25 岁，从那以后一直从事编程工作，职业生涯 3 年，其中两年是用 Java 编程。

### 后台和 Java

我在大学获得了软件工程学士学位，我以为我所有的同事都处于同一水平-- 0 级--但是我错了。几乎每个人都会编程，但我不会。这让我成了失败者，因此我必须比大多数人更加努力地学习和研究。从 2011 年到 2016 年，我一直在学习 Java，它的模式、代码风格、最佳实践和特质。我确信我仅仅触及了这门语言的表面，但过了一段时间后，我开始感到有所收获。这是我在 2017 年 1 月获得第一份工作，即软件工程师 Jr。
快进到同年 9 月。那时我迷上了机器学习，已经用 Java 手工开发了一些 ML 算法，然后我尝试用 Python 开发了一些。

### 巨蟒与爱情

此时，我已经知道如何编码和协作。学习一门新语言不会像第一次那么难，但是 Python 非常友好，社区非常活跃，我几乎没有遇到任何问题。我又花了一年时间才找到一份全职 Python 开发人员的工作，但是今天我在 Python 中的工作效率比在 Java 中要高得多。感觉清新，干净，有力。
过去，我用 Java 从工厂到完全 REST 端点需要一天时间，现在我用 Python 和 Flask 建模一个基本的 CRUD 应用程序需要几个小时。

### 我最怀念 Java 的什么

Python 也有它的弱点，在我转换之后，我最怀念的是静态检查类型系统。这省去了我很多麻烦，并确保我的代码可以在我需要使用它的任何地方工作。在 python 中，为了解决这个问题，我使用了类型提示和文档字符串。它仍然没有被检查，仍然会给我运行时错误，但是当你的软件在增长的时候，它可以为你节省很多时间，而这些时间你会花在阅读和重构你的旧代码上。

### 如何过渡到另一种语言？

我开始阅读和尝试，用葡萄牙语(我的母语)发表了几篇关于 Medium 的文章，开始在推特上谈论它，并获得了一些联系。我建议你走同样的路，学习，写点东西，告诉别人你的经历，然后教书。考虑到我的同龄人，我开始编程相当晚，如果我做到了，你也可以做到。
