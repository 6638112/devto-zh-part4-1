# 我是如何在短短几年内成为 CTO 的

> 原文：<https://dev.to/hahnmatthieu/how-i-became-cto-in-just-a-few-years-464a>

我不知道如何开始，所以让我们从头开始。

我进入 webdev 行业才几年，是一家小型法国电子健康公司的首席技术官，该公司为受体重、失眠或压力问题困扰的人提供项目。相信我，那感觉棒极了！

所以你可能会问自己，我要带着这个去哪里...

# 一开始

问题是，当我开始上大学，然后毕业的时候，这并不是我想要的职业。我的确是从一门为期两年的计算机科学课程开始的，在这门课程中，我学习了算法的基础，以及如何用 C、C++和其他类似的很酷的东西进行编码，但当时的老师一直告诉我们，作为一名开发人员就像是处于食物链的最底层。

显然，根据他们的说法，趋势是在技术的功能方面。所以我继续攻读硕士学位，成为了项目经理。所以，我的职业生涯是从写功能规范(Gasp)开始的。

# 转折点

我在一家大型电力公司工作，编写不同应用程序之间的数据交换规范，并注意到我们无法知道数据的真实位置。所以我开发了一个小的 Java JSX 应用程序，使我们能够跟踪数据，它工作得非常好，他们把它投入生产，我很激动！

我想，我到底在做什么，写功能规范，想成为项目经理，我讨厌这一切！在这一点上，我对我之前所做的一切感到后悔，认为我永远不会对我的事业感到完全快乐，永远不会。

这家公司随后开始用 iPads 取代他们糟糕的 palm 设备，并寻找开发人员来开发新的应用程序。所以他们雇佣了这个人开始研究这些东西，我大胆地问了他我是否可以试着和他一起工作，他是否可以指导我。这是我整个职业生涯中做过的最好的决定！

他们接受了！他们给了我需要的机会。由我来决定是否值得。

我在那里工作了两年，有很多东西需要学习，但大公司的问题是，你永远不会真正看到全貌。你只是在做项目的一部分。所以我离开了，在一家初创公司做了一年的首席 Fullstack 开发人员，然后有机会成为另一家小公司的 CTO。

我决定试一试，尽管我并不真的觉得自己适合这份工作。我觉得我错过了太多的知识，我永远无法足够快地做出正确的决定。

# 到了现在这个地步

我希望科技招聘人员不要忘记以下几点:

1.  学习是一件好事，但它不会让你成为一名优秀的开发人员，激情和实践会。
2.  如果你给他们一个机会，初级开发人员真的可以做得很好，我们都是初级的
3.  真的有可能在几周内学会如何编码(这是给我的 CEO 的，他认为最好不要雇佣没有计算机背景的人)

这是我写的第一篇博文，感觉很对。我不想从一些技术性的东西开始。我真的想感谢一些我在 twitter 上关注的人，他们让我觉得在这里写作是合理的。

所以谢谢你
