有许多工具可以用来构建“无服务器”应用程序，即服务器的配置是其他人的问题的应用程序。作为开发人员，您只需编写业务逻辑，AWS、Netlify 或 Google 的工程师可以担心所有那些无聊的基础设施。

至少，这是个想法。

今天，构建无服务器应用程序最流行的工具可能是无服务器框架。如果您想用 JavaScript 构建应用程序，这是一个很好的工具。

然而，最近用其他语言编写无服务器应用程序也变得更容易了！

我对用 Ruby 构建软件特别感兴趣，所以我在看 [Jets](https://rubyonjets.com/) 框架。大约 98%的喷气机是我的一个叫[东阮](https://www.boltops.com/about)的朋友的作品(他很棒)。

Tung 是 AWS 专家，热爱 Ruby。他选择构建 Jets 作为一个接口来构建由 AWS 和 Ruby 支持的无服务器应用程序。它能为你做这么多工作，真是太酷了。

> 如果你想更多地了解董，[他在我的播客](https://www.devpath.fm/episodes/aws-consultant-and-open-source-engineer-tung-nguyen) 🎙上讲述了他的职业生涯

在本教程中，我将构建一个最基本的应用程序。嗯，不完全是“你好，世界！”但是很简单。

本教程假设你已经安装了 [Ruby](https://www.ruby-lang.org/en/) 和 [Bundler](https://bundler.io/) 。

这是对 Jets 中自动化的一次肤浅的探索，最终产品不会有什么值得大书特书的，但它会让您了解 Jets 如何轻松地帮助您使用其类似 Rails 的代码生成原型。

第一步很容易。安装喷射器:

```
gem install jets 
```

希望你能在终端上看到`Successfully installed jets-2.0.4`或类似的东西。

现在，我们要求 Jets 为我们生成一个应用程序！

```
jets new example_app 
```

这应该会产生一个包含一些样板代码的目录。

```
cd example_app
ls -A 
```

```
.babelrc         .git             .rspec           README.md        config           package.json
.env             .gitignore       Gemfile          Rakefile         config.ru        public
.env.development .jetskeep        Gemfile.lock     app              db               spec
.env.test        .postcssrc.yml   Procfile         bin              node_modules     yarn.lock 
```

我们现在可以让 Jets 为我们的应用程序生成一些代码:

```
jets generate scaffold post title:string 
```

您应该会看到类似这样的内容:

```
INFO: You're missing AWS credentials. Only local services are currently available
      invoke  active_record
      create    db/migrate/20190814012442_create_posts.rb
      create    app/models/post.rb
      invoke  resource_route
       route    resources :posts
      invoke  scaffold_controller
      create    app/controllers/posts_controller.rb
      invoke    erb
      create      app/views/posts
      create      app/views/posts/index.html.erb
      create      app/views/posts/edit.html.erb
      create      app/views/posts/show.html.erb
      create      app/views/posts/new.html.erb
      create      app/views/posts/_form.html.erb
      invoke    helper
      create      app/helpers/posts_helper.rb 
```

不要太担心 AWS 凭证，我们不会将这个玩具应用程序部署到 AWS(尽管如果您愿意进行 AWS 设置，部署这个应用程序是完全可能的)。

现在，我们需要让 Jets 为我们建立一个数据库:

```
jets db:create db:migrate 
```

这将为您创建一个数据库，并运行由我们的 scaffold 生成的迁移。

此时，如果我们启动服务器，我们应该能够与`posts`资源进行交互。

```
jets server 
```

导航到`localhost:8888/posts`，您应该会看到以下界面:

此时，您可以对该资源执行基本的 CRUD 操作:
