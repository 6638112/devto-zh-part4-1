# 斯德哥尔摩室内发展之旅

> 原文:[https://dev.to/stacy_cash/insiderdev-tour-stockholm-509i](https://dev.to/stacy_cash/insiderdev-tour-stockholm-509i)

几周前，我参加了在斯德哥尔摩举行的 2019 InsiderDevTour。有几个理由让我非常兴奋:又一次去瑞典看朋友，又一次演讲的机会——这次是关于 Microsoft Graph，我的第一次现场编码会议和...参加我的第一次主题演讲！🤩

当然，最后三个意味着我需要花相当多的时间来确保我对演示和演示感到满意！

哦，没有压力，但不知何故，我设法成为巡回演出的[特色发言人](https://developer.microsoft.com/en-us/collective/events/insider-dev-tour)！大口喝！🤪

由于它是 Microsoft InsiderDevTour，所以提供会话，以便每个会议具有相同的内容。因此，与其花大量的时间来准备演讲，不如进行大量的练习，包括内容和演示。

我立即注意到，在演示时，我所有的个人信息都会显示出来。

不是这样的！为了解决这个问题，我创建了一个 [Office 365 开发者账户](https://developer.microsoft.com/en-us/office/dev-program)，我可以用演示数据设置它，而不用显示我自己的所有信息。

自从我开始演讲以来，真正让我感到震惊的是，我们在会谈中做了多少工作。我想了很多，但我还没有完全意识到有多重要！即使有了提供的会话信息，我还是花了几个周末做笔记、练习、运行演示、在犯错误时骂人，然后重新开始！但是，还有一周的时间，我已经准备好出发了！

嗯，我说就是这样，但是生活当然不是那么容易的！显然，作为内部开发人员，我的机器必须运行内部版本的 Windows 进行演示。在飞往斯德哥尔摩的那天早上，它自己更新了(我试图推迟更新，但是太迟了😶).更新进行得很顺利，然后我去了瑞典！

仅仅...更新进行得很顺利，但是在晚上我做最后一遍的时候，我遇到了问题。我的演示都不起作用了😫。付出了很多努力，摆脱了几年的压力，我又重操旧业了。和...放松点。

第二天早上，闹钟在一个周末应该是犯罪的时间响了，我走到外面灿烂的阳光下！我开始帮忙搭建舞台，反复检查我的演示是否还能用，我是否可以连接到 HDMI 开关。

在此期间，人们开始到达，其他发言者和与会者，神经开始上升！

时间流逝，我们开始了！许多有趣的事情发生了...轮到我拿起麦克风走上舞台了！

说到说话，我还在学习我的手艺，但迄今为止我的经验都是一样的。当我走向舞台的时候，我完全害怕，当我对观众说第一个词的时候，这种恐惧就被嗡嗡声取代了(我真希望这种情况一直保持下去！).我确实感到额外的压力，因为它是我的第一个主题演讲和我的第一个现场编码演示的两倍！但是，我正玩得开心，这时微软的 Graph 登场了。

在我们一起玩了一会儿之后(是的，微软图形的吉祥物是微软 G-Raph😀)G-graph 去做 G-graph 做的任何事情，我回到我的机器前。

忘记了英语。完完全全。可能是几秒钟，但似乎是几个小时！没错。没错。好的。没错。没错。没错。

但我设法恢复，做我的演示，不要抖得太厉害以至于我不能打字，我真的很享受。

然后我可以再次坐下来，听听基调中的其他精彩内容。

主题演讲结束后，我又站了起来——这次是为了更深入地研究图表。再次紧张起来，再次，一旦我开始说话，能量变成了好东西。

我真的很喜欢做这个演讲，这可能不是我的内容，但我真的很喜欢做这个演讲，充满了热情！

我的现场演示呢？几乎都好。我记得几乎所有的都是按正确的顺序排列的😃。有一次我的代码出了问题，杰西卡(我的朋友和会议的组织者)大喊我忘记了一个 html 标签中的`t`。哎呀-这至少解释了为什么它不工作。这是我一直担心的，但我很高兴地说，这并没有影响到我。我修复了第一个问题，并开了一个关于修复结束标记的玩笑。代码成功了，我继续前进。

在会议结束时，我甚至得到了我的第一个会议问题(我之前的会议，我只是邀请人们在会后来找我)。我设法回答了这个问题，然后走下了讲台。唷。我这一天结束了，可以放松一下了！

花一天时间看其他会议。一些很棒的演讲者展示了一些非常有趣的新东西。

一天结束时，我又有了那种熟悉的感觉，那就是‘太好了，我不知道该从什么开始！’我总是认为这是大量优秀内容的标志！

如果你在微软生态系统工作，我真的建议你去[内部开发人员参观实验室](https://aka.ms/insiderdevtour-labs)进行探索！如果你有，给自己弄一个 [Office 365 开发者账户](https://developer.microsoft.com/en-us/office/dev-program)。它让你的生活更轻松！

当然，会议只是组织者和与会者创造的。感谢[杰西卡](https://twitter.com/EngstromJess)、[吉米](https://twitter.com/EngstromJimmy)、[克里斯汀](https://twitter.com/CristianCson)以及所有参与组织此次活动的人，感谢所有其他演讲者的演讲，当然，还有所有放弃周六美妙天气前来热情聆听我们谈论微软生态系统新功能的人！
