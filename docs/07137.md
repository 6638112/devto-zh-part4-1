## 经常被忽视的一个主要 GitHub 特性是 GitHub 页面。

如果你是一名正在找工作的网络开发人员，这可能是你申请材料中的一个杀手。这个特性让你可以直接在 GitHub 上免费将你的代码转换成工作演示。有什么比在 GitHub 上向雇主展示你的能力更好的方法呢？

### GitHub pages 的额外好处是设置和部署非常快。

虽然您不能演示任何全栈或后端项目(除非您使用 Firebase 之类的东西)，但您的前端项目可以成为焦点。

例如，假设您开发了一个字数统计可视化应用程序。这是 GitHub 页面的一个很好的候选，因为它不需要访问数据库。

向下滚动到 GitHub 页面块。

[![GitHub pages step 3](img/dc97a8fe228193867ae10645fe69a628.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--_BNo419x--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3es1xato2v32yvehwo62.png)

在*源*下，默认为*无*，这样你就可以选择你想要的分支(本例中为*主*)。单击保存。

现在，您将看到为您的项目新创建的 URL:

[![GitHub pages step 4](img/9970a4b5be5f71807579376b2f05e52a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--j398iWyr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/vzsp0gnhbyjyjc31gi9k.png)

几分钟后(有时更长)，该页面将被激活。几乎完成:我们仍然需要雇主和其他重要的人来看到这个项目实际上有一个工作演示。

[![GitHub pages step 5](img/e9137bd7e8c27b5b0aef6695181f7293.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--u5LHLWQt--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/hnhyfc3kedn7t02iuwcs.png)

复制项目 GitHub 页面的 URL。导航到代码选项卡。在右边你会看到编辑按钮。粘贴到网站标签。

[![GitHub pages step 6](img/e13e7aae2bfda31612a71fd4f87744ba.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--mUW4te31--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/aib4o9tm95c1xinay61p.png)

访问者现在可以在回购中点击您的演示。
