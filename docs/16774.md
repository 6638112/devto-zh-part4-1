# 在线社区的松弛。我们不能做得更好吗？呃呢..论坛？

> 原文：<https://dev.to/brpaz/slack-for-online-communities-cant-we-do-better-what-about-err-forums-c02>

Slack 被大量用作在线社区互动的平台。从聚会和会议小组到专门的编程语言或工具。

现在我想我可能在从 PHP 到 Kubernetes 的大约 10 个不同的 Slack 组中。

Slack 可能是公司内部交流的一个很好的工具(甚至这也是有争议的)，但是对于社区来说，我看到了一些问题:

*   它没有打开。你需要找到一个邀请链接才能加入。搜索引擎无法对内容进行索引。
*   内容是短暂的。我知道的大多数社区都有免费的 slack 计划，限制你可以拥有的消息数量。当您达到该限制时，旧邮件将开始消失。
*   除非你积极参与对话，否则事后很难找到有用的信息。没有人会试图在一个有 1000+条未读消息的频道里找到有用的东西。
*   你需要在每一个你想加入的 slack 群组里创建一个账户，并且找到你所属的所有群组是一件痛苦的事情。
*   它缺乏调节工具

不和谐越来越受欢迎，成为松弛的选择。至少你只需要一个账号，但这是一个更适合游戏玩家的平台，而不是技术社区或其他类型。

所以我在想，在线社区并不是什么新鲜事物。数量和受欢迎程度是的，但我记得在 21 世纪初属于一对夫妇，大多与游戏有关。那时候我们是怎么交流的？论坛！！

让我们比较一下 Slack 和那个叫做 Forums 的非常古老的东西:

*   Slack 是封闭的，论坛可以是公共的或私人的。

*   Slack 有渠道，论坛有分论坛和话题。

*   Slack 有线程和消息，Forums 有线程和消息。

*   Slack 允许你直接发送消息给某人。论坛有私信。

*   Slack 在调节工具方面非常有限。论坛有先进的审核工具。

*   Slack 有表情符号。论坛有表情符号和自定义签名！！

对于一个社区来说，论坛似乎是 Slack 的一个不错的替代品，但是似乎没有人再使用论坛了？你最后一次参观是什么时候？对于技术相关的东西，我真的不记得了。Reddit 可能是我能想到的最接近的了。

并不是说没有像 [Discourse](https://www.discourse.org/) 、 [Flarum](https://flarum.org/) 或[nodeb](https://nodebb.org/)这样的现代论坛平台。

也许我们需要一个免费的“论坛即服务”平台？

我不知道论坛是不是所有问题的答案，Slack 肯定不是。在在线社区工具领域肯定有很多工作要做。我猜完美的工具是论坛、消息平台和社交网络的结合。
