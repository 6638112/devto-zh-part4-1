# 是时候打破任务管理中“多个受托人”的神话了

> 原文：<https://dev.to/olivier32621338/time-to-break-the-myth-of-multiple-assignees-in-task-management-44g2>

我们经常被问到，“我能把一项任务分配给多个人吗？”也许你也有这个疑问。现在答案是，“是的，你可以！”。

虽然其他[任务管理工具](https://quire.io/compare/best-task-management-software-for-creative-teams)或[待办事项应用](https://quire.io/compare/best-to-do-list-apps)说你可能会搞不清楚谁负责什么，或者任务可能会被多个受托人忽略，但我们可以说，有时候你不想麻烦决定团队中的哪个成员应该处理一项任务，或者将一项不应该被分解成子任务的任务分解，这样你就可以单独分配它们。

## 案例 1 努力分配任务

假设您和您的销售团队有潜在的客户咨询，有三个人负责回复他们。

如果你能把这样的任务分配给这三个人，当有新的询问进来时，他们每个人都会收到一个通知，他们中的任何一个人都可以简单地把它捡起来。

老实说，只要任务是由团队完成的，你就不会想去检查你的资源，决定团队中谁应该做这件事，或者更糟的是，改变你分配任务的习惯。

## 案例 2 难以划分的任务

你和你的同事需要制作一个动画视频，有两个图形设计师负责角色的概念。

这个任务缺少具体的细节，比如有多少个角色，什么样的角色以及角色之间的关系，以使它变得清晰。

所以，想象一下，当它完全没有问题的时候，你会去打破它。

如果你能把这个任务分配给两个设计师，他们就能自己解决来完成它。这就是团队合作的意义。

## 案例 3 甲板上人手不够

这里还有一个你可能已经亲身经历过的场景。

您和您的工程朋友已经完成了一个产品的构建，并且因为它已经可以进行评估了，所以您创建了一个名为“Review”的任务

当然，你可以指派一个人去做，但是如果有更多的人去做会更好，这样可以给你尽可能多的反馈。换句话说，越多越好。

[![Multiple assignees](img/ce805810602048b9d91cfe7920f40ead.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--s8avHZDK--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/oz6i4ql23os8vjp68ox7.png)

现在你知道了，有些任务并不总是一个人的工作。把你限制在一个人的范围内，让你决定谁是这项工作的最佳人选，或者把一项实际上不需要分解的任务分解成更小的任务，这些都令人沮丧。这也违背了我们的思维方式。

这就是我们灵活分配人员的原因，这样您就可以两全其美，而不会影响您的日常行为，也不会影响我们简单的 UI！
