# 实际的前端实践

> 原文：<https://dev.to/rfornal/practical-front-end-practices-32in>

前端开发需要的不仅仅是编写干净的代码。虽然编写简洁易读的代码不是强制性的，但它将在将来省去许多令人头痛的问题。编写永远不会被修改或永远不会被再次查看的代码的可能性微乎其微，在旧代码中迷失的时间是浪费时间。以下是 HTML、CSS 和 JavaScript 的一些关键 web 开发最佳实践。

## 代码一致性

### 布局

一致地缩进、嵌套、包含大括号、引号和换行符，以便代码清晰易读。添加的新代码不应该偏离现有的格式约定或改变缩进级别。

### 可读性

使用空格、注释和描述性变量名来编写易读的代码。

没有必要为了节省文件大小而以混淆或压缩的方式编写代码。

使用自动化的服务器端或其他构建过程来优化文件。这包括连接文件、代码缩减和 This 压缩。

其他开发人员阅读代码的能力比其他问题更重要，尤其是如果优化可以用其他方式处理的话。

### 第三方库

> 为正确的工作使用正确的工具。

如果可能的话，应该在本地开发环境中利用未精简的库和第三方脚本来简化调试。代码应该以未修改的状态提交给源代码管理，或者只是在依赖项管理组件中引用，这些组件在构建中解析文件。最终产品将与源的其余部分一起压缩交付。

同样，第三方代码和库永远不应该被修改，它们的原始源代码和许可证必须被记录并适合于项目。对第三方代码的任何更改都必须得到同意，并且必须是出于特定的原因。如果变更是由 bug 修复命令的，那么适当的上游项目应该提交变更(假设代码是开源存储库的一部分)。

库代码应该被视为一种外部依赖，应该被认为是在以后可能需要大规模更新或替换的东西。

包含任何第三方代码都应该被仔细考虑，并被验证为给定问题的适当解决方案。添加另一个插件并不总是最好的解决方案。

## HTML

> 超文本标记语言，通常称为 HTML，是用于创建网页的标准标记语言。Web 浏览器可以读取 HTML 文件，并将其呈现为可视或可听的网页。HTML 在语义上描述了一个网站的结构以及表示的提示，使它成为一种标记语言，而不是一种编程语言。—维基百科

### 语义元素

语义元素是带有标签的元素，这些标签清楚地定义了开发者和 web 浏览器所包含的内容。基本的 HTML 包括一些标签，比如代表粗体的**和代表斜体的*。在 HTML 4.01 中，**代表强烈(一般为粗体)和*强调(代表斜体)被引入。HTML5，HTML 的最新版本，引入了对**语义元素**的支持；其中的一个例子是<标题>和<导航>，其含义立即变得清楚。******

> 感谢[娜塔莉·德·维尔德](https://dev.to/nataliedeweerd)纠正上面的历史。

这些元素现在被开发人员更普遍地使用，并且改进了文档的自动化处理；谷歌 Chrome、Mozilla Firefox、苹果 Safari 等所有主流浏览器都支持 HTML5 和语义元素。

### ID 和类别

使用 ID 和类名来定义元素是实用且相关的 HTML 实践。在 HTML 中，class 属性用于定义元素的特定样式。类属性可以在多个元素之间重用，允许元素携带相同的修饰符。

ID 标签用于定义一个特定的变量，可以和类属性一起使用。id 只分配给一个特定的元素，永远不应该被重用。这允许使用 ID 标记对元素进行特定的定制，而不会影响同一个类中的其他元素。

## CSS

当访问一个网站时，层叠样式表(CSS)是几乎所有事物的背后。HTML 负责网站所拥有的内容，而 CSS 则是视觉上使内容和网站风格化的东西。

> 级联样式表(CSS)是一种样式表语言，用于描述以标记语言编写的文档的外观和格式。尽管这种语言最常用于改变用 HTML 和 XHTML 编写的网页和用户界面的样式，但它可以应用于任何类型的 XML 文档，包括普通 XML、SVG 和 XUL。与 HTML 和 JavaScript 一样，CSS 是大多数网站用来创建视觉上引人入胜的网页、web 应用程序的用户界面和许多移动应用程序的用户界面的基础技术。—维基百科

### CSS 复位

默认情况下，所有 web 浏览器都添加了自己的样式，并且每个 web 浏览器都以自己独特的方式应用这些样式。例如，许多浏览器默认将链接设为蓝色，将访问过的链接设为紫色。其他人可能会给表格添加一定量的边框和填充。利用 CSS 重置是重置所有样式以从头开始样式化的一个很好的实践。

CSS 重置，或重置 CSS，是一个简短的 CSS 规则集，通常被压缩(缩小)，将所有 HTML 元素的样式重置为一致的基线。使用 CSS 重置将允许每个浏览器将其所有样式重置为空。这有助于避免跨浏览器的差异。

例子...

*   [迈耶复位](https://meyerweb.com/eric/tools/css/reset/)
*   [娜塔莉·德·维尔德的重置](https://github.com/nataliedeweerd/blank_website/blob/master/css/reset.css)

### 一致的 CSS 结构

CSS 文件的结构应该与 HTML 文件的结构保持一致。这对开发人员和那些将来可能需要对代码进行调整的人都有好处。如果 CSS 代码到处都是，并且不遵循与 HTML 文档相同的模式和流程，那么在做出更改之前，可能需要花费大量的时间和精力来理解这两者。

### 内嵌样式

通常，CSS 用 ***写在一个单独的 CSS 文件中。css*** 文件扩展名。内嵌样式是包含在 HTML 文档中的 CSS 表单。在编写 HTML 和 CSS 代码时，应该尽量避免这种做法。

#### 何时使用内嵌样式

虽然大多数 web 开发人员都避免使用内联样式，但是也有一些例外情况，应该在什么时候使用它们。这方面的一些例子如下:

*   HTML 电子邮件
*   生成性能基线
*   内容管理系统内容(WordPress、Drupal 等。)
*   动态内容(由 JavaScript 创建或修改的 HTML)
*   旧网站

## [房子](#dom)

> 文档对象模型(DOM)是一种跨平台和独立于语言的约定，用于表示 HTML、XHTML 和 XML 文档中的对象并与之交互。每个文档的节点都被组织成一个树形结构，称为 DOM 树。DOM 树中的对象可通过使用对象上的方法来寻址和操纵。DOM 的公共接口是在其应用程序编程接口(API)中指定的。—维基百科

## JavaScript

JavaScript 是 HTML 和 Web 的编程语言。90%以上的网站都使用 JavaScript 来创建动态的交互式网页。

作为一名 web 开发人员，有几个流行的 JavaScript 框架需要了解。框架是大型主体或许多类预写代码，可用于创建代码库。利用这些框架可以快速创建 JavaScript 应用程序。

> JavaScript 是一种高级的、动态的、无类型的、解释的编程语言。它已经在 ECMAScript 语言规范中实现了标准化。与 HTML 和 CSS 一起，它是万维网内容生产的三个基本技术之一；大多数网站都使用它，并且所有现代网络浏览器都支持它，无需插件。JavaScript 是基于原型的，具有一流的功能，这使它成为一种多范式语言，支持面向对象、命令式和函数式编程风格。它有一个用于处理文本、数组、日期和正则表达式的 API，但不包括任何 I/O，如网络、存储或图形设施，这些都依赖于它所嵌入的主机环境。—维基百科

*   机器人
*   铬
*   ios
*   OS X
*   Ubuntu(或者某种 Linux 风格)
*   windows 手机
*   Windows 操作系统

这些操作系统通常运行在以下一个或多个设备上:

*   台式电脑
*   笔记本电脑/上网本
*   手机
*   药片
*   电视
*   看
*   东西(即你能想象到的任何东西，汽车、冰箱、灯、恒温器等。)

一般来说，前端技术可以使用以下运行时 web 平台场景在上述操作系统和设备上运行:

*   网络浏览器(例如:Chrome、IE、Safari、Firefox)。
*   无头浏览器。
*   嵌入在本机应用程序中的 WebView/browser 选项卡(想想 iframe ),作为运行时与本机 API 之间的桥梁。WebView 应用程序通常包含一个由 web 技术构建的 UI。(即 HTML、CSS 和 JS)。(例如:Apache Cordova、NW.js、electronic)
*   从 web 技术构建的本地应用程序，在运行时通过到本地 API 的桥梁进行解释。UI 将利用原生 UI 部件(例如，iOS 原生控件)，而不是 web 技术。(示例:NativeScript、React Native)

严格为桌面屏幕设计的日子已经过去了。移动网络浏览已经超越了传统电脑上的浏览。处理这种变化的最有效方法是设计一个可以适应视区大小的站点。这就是响应式网页设计的用武之地。

### 响应式网页设计

响应最好的网站利用流动网格、灵活的图像和特定的 CSS 样式技术来改变网站的设计，并根据浏览器的宽度来呈现。开发人员必须确保在所有浏览器上测试网站，而不仅仅是最喜欢或最常用的浏览器。一些浏览器以不同的方式解释 CSS，并需要它们自己的 CSS 属性来实现特定的样式。

## 大图

### 库和框架(依赖关系)

成为一个特定库或框架的大师没有错。重要的是不要完全依赖它们。所有的库和框架都是基于本地语言本身构建的，并且不断更新和淘汰。有了对本地语言的扎实理解，开发人员可以不断学习，这将确保工作保持最新。

### 页面速度

网站加载的时间很重要。虽然页面速度对于较小的网站来说不是问题，但是更强大、更密集的网站可能需要更长的加载时间。的确，在某些情况下，有些因素会不可避免地影响页面速度。然而，有一些更好的实践应该被考虑在内，以确保网站尽可能的大小高效。

#### 图像文件

实际需要的图像大小肯定是需要注意的。一个大的图像将需要很长时间来加载，并将使用大量的带宽。如果一个网站上有很多大图片，那么每张图片都需要额外的时间。

文件格式也很重要。浏览器可以又好又快地加载 JPG、PNG 和 GIF 图像。但是，像 TIFF 或 BMP 这样的大格式会增加加载时间。

尽可能避免使用图像来显示文本。同样，图像比文本占用更多的空间，即使文本需要大量的样式。

#### 代码密度

尽可能优化代码。大而密集的代码会降低网站的速度，浏览器需要更多的时间来处理代码。重用相似元素的样式将有助于缩短 CSS 文件。保持 JavaScript 的干爽(不要重复)。

## 超越技术

### 团队

前端开发人员通常只是设计和开发网站、web 应用程序或基于 web 技术运行的本地应用程序的团队中的一个成员。

为 web 平台构建专业网站或软件应用程序的基本开发团队通常至少包含以下角色。

*   UX /用户界面设计师(用户体验/用户界面)
*   前端开发人员(客户端)
*   后端开发人员(服务器端)

假设上面提到的团队由一个项目领导和/或某种产品所有者(例如，涉众、项目经理、项目领导等)领导。)

一个更大的 web 团队可能包括以下上面没有显示的角色:

*   DevOps 工程师
*   数据库管理员
*   质量保证工程师/测试人员

## 结论

前端开发需要的不仅仅是编写干净的代码:它是几个元素的微妙平衡。有时候，这似乎更像是艺术而不是科学；虽然，两者都有自己的位置。编写永远不会被修改或永远不会被再次查看的代码的可能性微乎其微，在旧代码中迷失的时间是浪费时间。这些是 HTML、CSS 和 JavaScript 的一些关键的 web 开发最佳实践，在前进的过程中要记住。
