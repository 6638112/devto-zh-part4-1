# 每日挑战#12 -下一个更大的数字

> 原文：<https://dev.to/thepracticaldev/daily-challenge-12-next-larger-number-3f3o>

抱歉错过了昨天的挑战。今天我们有一个有趣的话题。如果您还没有了解这个系列，可以尝试一下这个系列。如果你是我们的高级用户之一，就当这是你的休息日吧。

这个挑战来自 CodeWars 上的用户 GiacomoSorbi

> 您的目标是创建一个函数，该函数接受一个正整数，并返回使用相同数字形成的下一个更大的数字。
> 
> 像 2019 这样的数字不会变成 9210，因为这是使用这些数字可以创建的最大可能的数字。答案是 2091 年，因为这是下一个更大的数字。
> 
> 比如:
> 12 == > 21
> 513 == > 531
> 
> 如果不能用这些数字组成一个更大的数，返回-1 或`null`。
