# 如何在初创企业中进行代码评审？

> 原文:[https://dev.to/ruvans/how-to-code-review-in-a-start-up-57pb](https://dev.to/ruvans/how-to-code-review-in-a-start-up-57pb)

我在一个由 5 名开发人员组成的小团队中工作，我们已经决定需要开始做代码审查，这是我们肯定已经应该做的。我以前在一家大公司工作时有过做代码审查的经验，我们有能力在这样的任务上花费时间和人力，但在这里不是这样。

*   有人对小团队中的代码审查有什么至理名言吗？
*   你如何腾出时间来审查代码？
*   如果没有能够理解代码或编程语言的评审人员，你会怎么做？
*   当审稿人和作者是同一技术水平时，你们是如何解决纠纷的？
