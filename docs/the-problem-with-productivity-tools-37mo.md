# 生产力工具的问题

> 原文：<https://dev.to/sergix/the-problem-with-productivity-tools-37mo>

*大家好！你可能已经看到了我最近的一些评论，但这是我在这个网站上的第一篇文章。任何批评都接受！*

## 你说的这个问题是什么？

最近，我一直在研究静态网站前端框架，试图改造我目前的网站。我已经遇到了这么多伟大的，但我决定使用 Gridsome 与 Vue。然而，本质上，所有这些框架都承诺了同样的事情:节省我的努力。然而，在研究这些框架的过程中，我似乎做了更多的工作来寻找工作的最佳方式，而不是实际上*做*的工作； *metawork* ，如果你愿意的话。

既然我意识到了这一点，我现在注意到了其他的例子。无论是寻找完美的`npm`包，试图组织我的电子邮件文件夹，还是创建和管理完美的待办事项列表，我似乎更痴迷于元工作本身，而不是实际完成所有这些元工作应该帮助我做的事情。希望其他人也能有这种感觉。

## 担心工作效率不够高

我在研究框架时遇到的主要问题是，很多人对哪个最好有不同的看法，尤其是关于 React 和 Vue。这让我觉得无论我选择什么，我都会错过一些东西；这也适用于其他工具。使用哪个编辑器、哪个 JavaScript 样式指南、哪个代码高亮器、哪个代码片段管理器、哪个按键绑定...关于为什么有些比其他的更好，存在着太多的争论，以至于在实际尝试做出决定时，很快就会变得令人不知所措。

## 解决方法:做你的研究，但是选择一个点停止研究

我们都可以通过点击那些描述使用最闪亮的新工具来节省时间和精力的方法的文章。但是在某些时候，你的脑子里充满了比有用的信息更多的信息。写下你认为会有帮助的主要解决方案或工具，单独研究它们，然后比较它们的利弊。这很简单，我相信你以前也做过。然而，一旦你做出了决定，就要坚持下去。熟悉你选择的工具。与其跳到下一件大事上，不如试着充分利用你的工具。除非有什么明显的东西成为你工作流程中的主要障碍，否则你会对这种策略的效果感到惊讶。

## 努力变得富有成效并没有错

让我澄清一下:元工作是*而不是*一件坏事。在当今的软件开发世界中，我们比以往有更多的选择来节省完成任务的时间！然而，有时候我们不知道在哪里划定界限，不知道坚持我们已经拥有的。如果你只是试着充分利用我们现有的资源，而*没有*安装你在媒体上读到的每一个新的库和工具(甚至是在 DEV 上！)，你可能会发现你更少分心，可以更好地集中注意力，因为你可以更好地利用你已经拥有的工具。

## 生产力工具我*做*使用

既然我们谈到了这个话题，我想我可能会列出一些我日常使用的工具来帮助我保持条理。

### [托比(Chrome 扩展)](http://www.gettoby.com/)

这个扩展旨在成为一个更好的书签工具，当你需要关闭浏览器，但又不想把所有东西都加入书签，或者在你的历史记录中寻找你打开过的内容时，它会有所帮助。它可以取代你的新标签页，并有一个非常直观的界面，以任何你喜欢的方式组织你的标签。一开始我有点怀疑，但是一旦我安装了它，它变得比我想象的更有用。

### [Boostnote](https://boostnote.io/)

Boostnote 是每个现代开发人员记笔记的梦想，它已经成为我记下代码或想法的常用工具。目前仍有一些 bug，但也是[开源](https://github.com/BoostIO/Boostnote)！
