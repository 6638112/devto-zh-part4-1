我真的很喜欢在会议上，在 Twitter 上，在 Instagram 上，通过电子邮件，以及最近在我全新的 [MindsersIT 社区论坛](https://community.nathanaelcherrier.com)上与人们谈论编程。我也很感激人们(通常是开发者)分享他们对我在这里写的帖子的想法和反思。我注意到人们经常认为我无所不知，因为他们忘记了没有人知道一切。

***⚠️在我的个人博客上读到了我更多关于科技和商业的博文[！⚠️](https://mindsers.blog/)***

这篇文章的灵感来自于[丹·阿布拉莫夫在他的](https://overreacted.io/things-i-dont-know-as-of-2018/)[个人博客【反应过度](https://overreacted.io/)上的文章。如果你不知道这个马尔是谁，他是脸书[反应库](https://reactjs.org/)的主要开发者之一。他得出了与我相同的结论，并决定[写一篇文章](https://overreacted.io/things-i-dont-know-as-of-2018/)，鼓励人们也这样做。这是我的。

> 尽管如此，即使是经验丰富的工程师也有许多知识缺口。这篇文章是关于我的，我鼓励那些能够承受类似脆弱性的人分享他们自己的脆弱性。

因为我想要诚实和透明，所以我打算发布我去年的常用(也是无聊的)最佳帖子列表。当然，我变懒了，我们已经是 2019 年年中了。所以我会做一个“我知道什么和不知道什么”，我的意思是，这没什么大不了的，不是吗？

* * *

## 什么我不知道

让我们从我不幸不知道的事情开始。“不幸”是因为这些事情我真的希望我能知道更多。

*   电子:当这个项目出来的时候，我真的很感兴趣，我甚至测试了一些概念证明，但我从来没有真正走得更远。我有一个全球的想法，我可以用它来做什么。
*   CSS 方法论:我了解 CSS，我每天都在使用它，我对它非常满意。然而，在我的学习道路上，我主动地把诸如边界元法这样的方法论放在一边。我知道它们有多有用，但是我不能正确地使用它们。
*   **WebGL:** 我从来没有和 WebGL 合作过。我在 CodePen 上看到了美丽的片段，这就是我所知道的一切。
*   我关注这项技术已经有一段时间了，但我一直缺少时间，所以我学习了 RESTfull，因为它更常用。我只用 GraphQL 开发了一个网站。今年我会全力以赴。
*   函数式语言:这个很难写，因为我真的希望我能对它了解更多。这个话题真的很有趣，我做了一些研究，我玩过 haskell，我甚至在博客上写了 JavaScript 中的函数，但仅此而已。我不掌握任何纯函数式语言。
*   **Android 开发:**我还在上学的时候就学会了 Android 开发。很久以前在开发世界里。当我在一家软件公司工作时，我也写过一些快速的错误修复。我忘记了我所知道的一切，我今天仍然拥有的知识现在可能已经过时了。
*   **Shell 脚本:**我从编码开始就写了很多脚本！不幸的是，我不能说我掌握了这种强大的语言。我注意到我总是使用相同的语言结构，我不清楚我们可以用 shell 脚本做什么(甚至不谈论解释器之间的差异)。
*   在这里列出的所有技术中，这可能是我所知较少的一项。我知道它的目标是为开发者清除服务器管理，但对我来说，这听起来像魔术。
*   **Python:** 这种语言看起来挺有意思的。我编写了几个小的 Python 程序(例如:用来自 RaspberryPi 的 Arduino 串行链接的 2 个 UNIX deamons)。然而，我的知识也就到此为止了。我一点也不精通，也许有一天会懂。
*   系统和网络管理:你可能认为这很正常，因为我不是系统和网络管理员。你可能是对的，但是它今天对我很有帮助。我管理我自己的服务器和我的客户的服务器，但我坚持基本的任务。
*   **Streams:** 那可是个大话题。流永远存在，几乎存在于每一种语言中。我没有借口，我只是从来没有花时间这样做。我使用了流，我知道它的用途，在这种情况下，你应该使用它，但我不知道它是如何工作的。我也得花时间去做。

## 我所知道的

既然你都知道了，我想以一个积极的方式结束这篇文章:我所知道的。在我想知道的一长串事情之后，我的自我需要它。

*   JavaScript: 我会一直学习它，我在考虑 AST 的创建，streams...等等。但是我认为我获得了大量的 Javascript 知识和经验。
*   前端框架:我已经学习 Angular 和 React 很多年了。我读了 Vue doc。这些框架在方法上非常相似。我并不精通所有的前端框架，但我认为由于我每天使用的框架，我有很好的全球知识。
*   **NodeJS & ExpressJS:** 实际上我写的 NodeJS 代码(个人框架、命令行界面等)比我用 ExpressJS (web app)写的还多。但这是我使用 NodeJS 以来学到的唯一一个后端 JS 框架。我想说我非常了解它。我自己做了[编码 NodeJS 框架的测试](https://github.com/mindsers/yabf)，它的灵感来自我所知道的前端、ExpressJS 和后端。我认为更好地理解事情是如何运作的是有用的*在引擎盖下*。
*   **可及性:**一个很重要的话题。当我不得不重新设计 SNCF 的残疾人网站时，我才真正意识到这一点。当开发人员不努力去适应这些人的时候，上网对他们来说就是活受罪！很少有开发者对这个话题敏感，这是一个遗憾。知识问题之外，是一个[内含物](https://blog.nathanaelcherrier.com/tag/diversity/)。
*   和许多人一样，我的第一种打字方式是通过 Angular 2。我发现它非常有趣和强大，足以让我自己的大部分项目都基于它！
*   我是一名 iOS 开发者，我仍然记得苹果宣布 Swift 的那一天。从那以后，尽管有些客户犹豫不决，我还是有机会开始我所有的 Swift 项目。我认为我在使用 Swift 进行服务器开发方面还有一些弱点。
*   我对这个话题充满热情。我确信开发人员的工作是一种手艺，对于每一种手艺，你都可以变得更好，直到掌握你的手艺，成为一名专家。我读了，写了，分享了很多。
*   干净的代码&架构:对我来说，一个好的开发人员不仅要写代码让它工作，还要向他的客户保证代码会尽可能长久，并且是可维护的...等等。干净的代码和干净的架构是软件工艺运动的一部分，所以我完全投入其中。
*   **OAuth 2.0:** 这个话题困扰我有一段时间了。从开发者的角度来看，这是一个标准/协议，但它大大简化了用户的生活！我从零开始学习实现 OAuth 2.0 服务器，并研究了专门的解决方案，如`oryd/hydra`。很糟糕，但我熬过来了。我想现在我知道了。在“安全”方面，尽管我对这个话题有所了解，但还是很紧张。
*   国际化是我开发几乎每一个网站都会重新出现的话题。但是每当我开始一个新项目时，我都会问自己是否有更好的工具可以使用。尽管知识越来越丰富，但这可能是唯一一个我没有意见的话题。我在整个项目中使用、试验、见识了许多不同的工具，我了解它们是如何工作的。

* * *

这些列表并不详尽，我还有很多要弄清楚。计算机科学和编程是如此广泛的话题！

这篇文章的目的是向所有人展示，尤其是初学者，没有人知道所有的事情。我们都有需要学习和提高的地方。

不要错误地认为，因为人们在社交媒体、网站或博客上谈论和分享他们的知识，他们就比别人知道得更多。更重要的是:不要贬低自己。不管我们的职位或专业水平如何，你和我们一样都是一个进步中的人！
