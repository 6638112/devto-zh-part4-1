# 框架和库来来去去。自学技能与你同在。

> 原文：<https://dev.to/tiagodcosta/frameworks-and-libraries-come-and-go-self-learning-skills-stay-with-you-3ocp>

[![Alt Text](img/10e6f973f1456d81060dbd3c01d72224.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--z7xKCnEr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mgp47paoauvtqe6l7zxa.jpg)

情况是这样的:五年前你所学的一半知识现在已经不相关了。

对于我们这些身处 IT 第一线的人来说，这不是新闻。我们需要不断地提高技能，再提高技能，再提高技能。然而，这些技能的寿命越来越短，这就提出了一个新的要求:自学的能力。

抱歉，没有训练营、硕士学位或代码实验室能让你不断更新技能。你必须成为一个自学者。

自学需要我们有一种传统学校不教的承诺。渴望学习，培养专注，激励自己是我们在学校里不会经常看到的话题。

说到底，自学就是既当学生又当老师。如果你能接受[同时成为学徒和导师](https://www.facultyfocus.com/articles/effective-teaching-strategies/why-being-a-student-made-me-a-better-teacher/)(初级和高级开发人员)的想法，你可能会在当前的时代茁壮成长，并多次重新获得技能。

顺便说一下，学习是开发人员职业生涯的核心。我们当中有多少人在职业生涯中的某个时刻因为没有学到新东西或者学得不够快而辞掉工作？我已经这样做过至少三次了——辞职，因为我没有在我正在做的事情上学习或者变得更好。学习在 web 开发的职业生涯中至关重要。

我最近读了[彼得·霍林斯](http://petehollins.com/)写的[自学的科学](https://www.amazon.com/Science-Self-Learning-Yourself-Anything-Education-ebook/dp/B07KKLGYWF)。这本书帮助我充分利用我的自学技能。这是我的一些笔记。做一个自学者，要注意三个话题。

**1)培养自我管理和动力**

如果你怀疑自己有自学的能力，不要想太多。我们习惯于不自学。你习惯于在恐惧中学习，等待别人告诉你你必须学习什么，什么时候学习。也许这就是传统学习和自学的主要区别——知道[如何激励自己](https://time.com/2933971/how-to-motivate-yourself-3-steps-backed-by-science/)。

发展自我管理的最好方法是像科学家一样思考。在开始实验之前，科学家会制定一个计划，写下将用于得出结论的假设、材料和方法。

同样的，在你开始学习之前，问一些问题，在深入学习内容之前先了解全局，试着思考你将要学习的东西，记录你的成就和期望，并且总是回顾你刚刚学到的东西。

**2)与信息互动**

玩你想吸收的信息。使用类比。寻找模式。将你正在学习的东西与你已经知道的东西和技能进行比较。教一个朋友。综上所述，带着你想掌握的信息和技能，像孩子一样玩耍。

**3)有记笔记的方法**

自学的时候，脑子里的东西都整理不出来。你应该学会如何记笔记。这个我不知道，但是有很多[记笔记的方法](https://www.oxfordlearning.com/5-effective-note-taking-methods/)。我最喜欢的是[康乃尔系统](https://www.educationcorner.com/cornell-note-taking-system.html)。

对了，记笔记是发现自己知识盲点的好方法。做笔记是一种自我解释。如果你无法解释某事，可能是你头脑中的概念不清楚。

从我的经验来看，自学是培养开发人员自信的最好工具之一。我将告诉你真相——它有时会令人沮丧和艰难，但好处将持续一生。它们的用途比你可能意识到的还要广泛；你在学习期间培养的自我管理技能也可以应用到你生活的其他领域。

这就是为什么当我参加招聘委员会时，我对候选人的自学能力最感兴趣。我希望我的团队中有人能跟上变化的步伐。毕竟，框架和库来来去去。自学技能与你同在。
