# 为什么 Scratch 是理想的开发环境

> 原文:[https://dev . to/pull Josh/why-scratch-is-the-ideal-development-environment-4d dk](https://dev.to/pulljosh/why-scratch-is-the-ideal-development-environment-4ddk)

不用我说你也知道 [Scratch](https://scratch.mit.edu/) 不是专业工具。当然，也有一些[给人留下深刻印象的人](https://scratch.mit.edu/users/griffpatch/)创造[给人留下深刻印象的项目](https://scratch.mit.edu/projects/113321949/)，但他们是例外，不是规律。

Scratch 是为儿童设计的。它不是为了强大而构建的。它不是为快速而设计的。不能将数组或对象作为数据传递，也不能创建返回值的函数。作为真正开发的工具，Scratch 毫无价值。**但是 Scratch 做对了一件事:它让编程变得直观。**

### [](#you-know-your-options)你知道你的选择

在 Scratch 中，每个“块”——您可以使用的每个功能和方法——都可以在屏幕上的单个滚动面板中看到:

[![](../Images/57b793ff4dc96b319aac1098ba45a20e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--HsmjQkzl--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/e7yd5ck9x6gukjj9smui.png)

你不需要文档，因为你能做的一切就在你面前。如果你不知道某样东西是做什么的，你可以*试一下*。

### [](#experimentation-is-everything)实验决定一切

**想运行一些代码？只要点击它。**就这么简单。如果你有一两个区块，你可以点击它们，它们就会运行。一直都是。

[![](../Images/2ac10dcf25d183dcee16e3cfab9eaad9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--EzOrqo-e--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/c15p6qgqptehbs6qhsra.gif)

这意味着除了愿意尝试，你不需要文档、教程或任何东西。如果您发现一个看起来像您想要做的块，您可以点按它(即使您的项目正在运行),并立即看到结果。

### [](#militarygrade-hot-reloading)军用级热重装

在 Scratch 中，你的项目是一个舞台，你是导演。代码仅仅是一种与你的演员交流的方式(技术上来说，是“精灵”)。当然，你可以提前给你的演员一个剧本，但是没有什么可以阻止你随着场景的展开给出额外的指示。

Scratch 解释器直接嵌入到编辑器中，这意味着当你与代码交互时，你是在直接操纵世界。你可以在脚本*运行*的时候修改它，它会正常工作。这就是魔法。

## [](#the-result-insight-and-intuition)结果呢？洞察力和直觉。

当你把这些元素结合在一起，你得到的是一个非常直观的编程体验。当然，它动力不足，但它也赋予了力量。通过给你尽可能多的工具来观察和操作你的项目，Scratch 让你很容易掌握代码是如何工作的。当你想做什么的时候，你就开始点击砖块。你可以看到它们是如何以一种摸得着的方式结合在一起的。代码不再是黑匣子。

## [](#i-want-scratch-back)我要抓回来

早年我花了[很多时间](https://scratch.mit.edu/users/PullJosh/projects/)在 Scratch 上，但最终走向了更大更好(？)的事情。更广阔的编程前景有很多值得喜爱的地方，但老实说，我并不为离开曾经如此贴近我的心灵的编辑体验而感到激动。

但是为什么直观的编程环境是留给孩子的呢？当然，即使是大孩子也需要一些额外的帮助。编程的挑战在于理解我们的小黑盒里发生了什么。但是我们越能[在盒子里戳洞](https://techcrunch.com/2019/06/03/goodbye-trash-can-hello-cheese-grater-apples-reinvented-mac-pro-will-shred-your-workflow/)——我们越能从内部洞察我们的代码看起来和感觉起来如何——我们就能创造出更好的软件。
