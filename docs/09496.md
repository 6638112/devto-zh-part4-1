# 采用开源项目

> 原文:[https://dev.to/jcabot/adopt-an-open-source-project-1lc4](https://dev.to/jcabot/adopt-an-open-source-project-1lc4)

软件是支撑我们数字社会的基础设施。而且这种软件大部分不是开源软件(OSS)就是严重依赖它。虽然从很多方面来说这是一个好消息，但这也是一个大问题，因为大多数开源项目面临严重的可持续性问题，这使得它们被放弃。像许多其他的“公共领域产品”一样，开源软件遭受了所谓的“T2”，“公共资源的悲剧”:“T3”每个人都想从软件中获益，但他们都希望其他人也能参与进来。也就是说，在消费软件的人和参与构建软件的人之间存在着严重的不平衡。

有几种方法可以降低放弃开源项目的风险:

*   遵循策略[让整个社区(即不仅是维护者，还有用户)更多地参与到项目中](https://livablesoftware.com/large-community-software-development-erc-proposal/)。
*   使用一群软件机器人来协助项目监控和维护。简单的机器人将负责看门的任务(例如给新问题分配标签)。智能机器人将从项目数据中学习，成为高质量的助手(例如，能够与用户聊天，以确保错误报告足够详细，并自动对它们进行分类)。
*   获得更多的资金来支付维护者(例如，见 Tidelift 发布的[100 万美元支付开源维护者](https://blog.tidelift.com/1m-to-pay-open-source-maintainers-on-tidelift))。

但是今天机器人能够做的事情实在太多了(参见我们更新的改善你的软件开发过程的最佳机器人列表)，金钱并不总是人们留在/离开开源的原因，甚至那些遵循某种程度上民主的[治理模型](https://modeling-languages.com/add-a-governance-md-file-to-all-your-oss-projects/)([不多](https://livablesoftware.com/transparency-democracy-open-source-not-thought/)顺便说一下)的项目仍然需要一群领导者或“冠军”来带来许多真正保持项目前进的无形价值。对于某些项目，上述策略只是非常局部的解决方案，他们可能仍然倾向于放弃。

那么在其他都不管用的时候怎么办呢？最好的选择是**让开源项目被采用**。明确这一点有两大好处:

1.  它阐明了项目的所有者正在寻找某人来接管，并乐于在过渡中提供帮助(与项目的硬分叉相反，从技术上讲可能会有相同的结果，但会混淆和破坏项目周围的社区)。所有者可以使用一个特殊的标签，在某种徽章的名称或项目描述中使用一个关键词。
2.  它有助于搜索孤儿项目，因此潜在的“寄养维护者”可以很容易地找到潜在的项目来采用。

这不是一个新的想法。在 [WordPress](https://livablesoftware.com/wordpress-content-management-democratize-publishing/) 中，标签 *[adopt-me](https://wordpress.org/plugins/tags/adopt-me/)* 用来给寻找新主人的插件发信号。类似地，Jenkins 使用了[*adopt-this-plugin*](https://wiki.jenkins.io/display/JENKINS/Adopt+a+Plugin)标签。npm 墓地是无人维护的 npm 包的休息地，等待被收养。

不幸的是，这是例外而不是规律。我们需要其他社区效仿这一举措。更重要的是了解这对双方是否都有效。我的意思是，我相信这个想法，但需要进行研究，以得出一些科学结论，即实际上成功转移并设法与新“父母”一起生存的项目的百分比。

我们可能还需要一种更主动的方法来检测处于风险中的项目，为此，我们需要一个程序来与所有者开始对话，以说服他们采纳这些项目，而不是仅仅依靠项目所有者自己得出这个结论。这个过程还应该包括一些关于如何进行所有权转移的指导方针(包括任何版权、知识产权或其他法律问题，如果必要的话)，这些移交项目的 [10 条规则](http://third-bit.com/10rules/#handing-over-and-moving-on)将是一个良好的开端。

对于那些还没有准备好完全放弃一个项目，但仍然拼命寻找帮助的人来说， [Code Shelter](https://www.codeshelter.co/) 可能是一个暂时的和中间的解决方案。Code Shelter 是一个志愿者软件开发人员的集体，旨在帮助维护流行的开源项目，这些项目的作者需要帮助或者没有时间维护它们。

你对此有什么想法？你会考虑领养一个项目吗？
