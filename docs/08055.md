## 大约四年前，当我在德克萨斯州的休斯顿无家可归、苦苦挣扎时，我发现了 web 开发。

Joe 从 YouTube 上的 CodingPhase 第一次把我介绍给 Udemy。他总是提到 Udemy 上的 5 门具体课程，他用这些课程来学习帮助他成为一名开发人员的技能。我购买并完成了其中的一些课程，并开始从那里扩展到其他课程。Udemy 在我学习技能的过程中发挥了巨大的作用，这些技能最终让我在 web 开发领域找到了第一份工作。

[![Shane](../Images/dda30df56539820217dec370e7c5e24a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--tLARXXbU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://miro.medium.com/max/928/1%2ApCyaoHHetf6kfQMb6vo1lA.png)

### [](#i-think-as-a-selftaught-dev-there-are-many-advantages)我觉得作为一个自学成才的 dev，有很多好处。

最大的一个就是在学习不同技术的过程中，可以慢慢来，按照自己的时间表学习。

相比之下，试图跟上一套课程，这可能会导致你必须跳过某些概念，然后才能完全理解它们。另一个好处是，你可以在闲暇时跳过某些课程和主题，专注于你最需要学习的东西，而不用浪费时间在填充科目上。在我看来，自学途径效率更高，而且总的来说教育质量更高。

#### 和任何自学平台一样，在使用 Udemy 时，我唯一会考虑的一个缺点是，当你陷入困境时，没有人可以“亲自”帮助你。

这个问题的美妙之处在于，它自然而然地教会了你 web 开发中最强大的技能。这项技能是学习如何使用谷歌快速找到你遇到的问题的解决方案。任何人都不可能完美地记住每一个 CSS 属性，甚至多种语言的基本语法。

即使是最有经验的开发人员也在每天查找基本的东西。关键是在这个过程中要非常高效，知道如何检索你需要的信息并尽快回到项目中来。你编码的越多，默认情况下你就能做得越好。

除了我上面提到的最重要的技能(在我看来),我想说的是坚持每天写代码，即使只有 30 分钟。

[![code](../Images/610ae32fb1721a870a33bf80476a7354.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AsUj9a2Q--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://icon-library.net/images/code-icon/code-icon-5.jpg) 
*与编码的一致性是关键，表示谢恩。*

显然，你投入的越多，收获的就越多，但是坚持是关键。每天花 30 分钟编码比一周只花一天时间编码更有价值。这些信息会在你的脑海中保持新鲜，你会有清晰的时刻，概念似乎更频繁地点击。

对于那些对自学持怀疑态度的人，我想说至少试一试，给自己一个机会。从一些免费资源开始，看看会发生什么。在你尝试了一段时间后，如果你仍然觉得你需要更多的结构和指导，那么我会说去训练营或学校只是最后的手段。

我想说，作为一个自学成才的开发人员，他进入了这个行业，**我完全靠自己做到了这一点，这是我做过的最伟大的事情**。事实上，我从一个无家可归的人到白手起家建立起自己的 web 开发事业，这给了我一种无法解释的自豪感。

有一件事我可以 100%自信地说，如果我能进入这个行业，那么任何人都可以做到。只需要奉献、承诺和坚持。

### [](#it-took-me-about-3-years-from-the-time-i-started-learning-web-development-until-i-started-applying-for-jobs)我从开始学习 web 开发到开始求职，大概用了 3 年时间。

我得说，我等得太久了。我知道我本可以在一年左右后找到一份工作。

我给每个人的建议是，一旦你有一个投资组合，里面有一些好看的项目，就开始申请。不管你是否符合要求，也要申请这些工作。如果你的投资组合中有高质量的项目，表明你拥有他们需要的工作技能，那么你就可以得到一份工作。

## [](#portfolio-advice-for-newbies)给新手的投资组合建议:

*   我认为拥有有效投资组合的关键是拥有看起来专业的真实项目。当我说真正的项目时，我指的是实际业务的实时网站，或者一个完整的功能性应用程序。

*   虽然不是每个人都擅长设计，包括我自己，但要确保你的项目看起来现代、干净。你需要向未来的雇主展示你知道如何把一个项目组合在一起，并且你拥有他们想要的员工的技能。

### 这是你的选择…反正我们都要变老…为什么不珍惜呢。

[![Alt Text](../Images/88fa945768a8fcd23b76bf016e9f3a5a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2yZhm83T--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/t99y62w78jeuc66s3x8f.png)

Ian 认为 Coding Phase、Udemy 和 YouTube 帮助他学习了现代 web 开发。

我第一次学习编程是在 2003 年，第一门语言是 ActionScript 2。

以前，你可以在 YouTube 上搜索“JavaScript ”,看到数百万个视频出现。与今天相比，学习如何编程简直是一种苦差事。

我确实在 2003 年到 2005 年去了 ITT 理工大学，在那里我学习了 HTML、CSS、Javascript、Actionscript 和 PHP…但是为了取得好成绩，我不得不自学。我寻找任何书籍，购买 CD/DVD 课程，去任何有编程的网站。对我帮助很大的几个网站是 tinkernut.com 和 developphp.com。

我也没有一个可以用来帮助编程的社区…在 2008 年我们都从我们的第一份开发工作中被解雇后，我和我一起上学的朋友最终离开了编程世界。

**这真的就像推一块巨石上山一样**…在尝试了多年的自由职业，为沃尔玛、政府、银行、一家小科技公司(不按此顺序)工作之后，我终于在 2011 年有幸得到了我的第二份开发工作，在一家讨债公司工作，年薪 3 万美元。

有时候为了达到顶峰，你必须努力。

现在在我的第四份开发工作中，我仍然在努力达到那个高度…但是现在我有了一个实际的计划。

## [](#we-are-all-selftaught-and-we-are-all-learning-from-someone)我们都是自学的，都是在向某个人学习。

即使去学校学习代码，你也不会被填鸭式地灌输，你必须自己学习……如果你打开 Udemy 教程，你也像在学校一样向某人学习。

那些看不起那些宁愿自学而不上学的人的人必须认识到，无论如何你都必须自学。不像几年前的世界，你可以去上学，找工作，工作 40 年，永远不会提升；今天，如果你不“自学”,你就会落后。

### 事物变化如此之快，以至于几年后科技就被认为是陈旧的了。

你最终会把自己固定在一个位置上，最后当你抬头说**‘嘿，我想我需要一份新工作’**时，你会发现你不得不从头开始。这就是发生在我身上的事。在我的第二份开发工作中，我在一家公司的专有软件上工作了 3 年，然后我离开了那里，开始了我的第三份开发工作，使用 ActionScript。

当我试图寻找一份新工作时，我意识到我必须从头开始，因为没有人雇用 ActionsScript 3 / VXML 开发人员。

## [](#insider-tips-for-newbies)新手内幕提示:

*   每天至少 3 到 4 小时(如果你没有工作，8 小时)编码。你不能把这当成一种爱好。如果你错过了没关系的一天，第二天就回去工作……你的工作/兼职是找一份开发工作。照此处理。

*   在社交媒体上发布你的进展，尤其是 LinkedIn。这会给你一个更有经验的开发者的印象。此外，当招聘人员在 LinkedIn 上搜索特定技能时，你发布的代码和正在从事的项目会出现，招聘人员会开始联系你。你也可以创建一个博客或 YouTube 频道…这样做的目的是当公司搜索你的名字时，他们会看到你的进步…从 HTML 新手到 JS 高手。

## [](#portfolio-advice)投资组合建议:

只要确保你放在那里的项目是你自己的。使用你从课程中创建或开发的项目是可以的…只是要确保你改变了颜色、函数和变量的名称，甚至使用了不同的技术——比如如果你构建的应用程序使用了 MySQL 的 MERN 堆栈 change out MongoDB，或者 ASP.NET 的 change out Node/Express，等等。

## [](#tips-for-newbies)新手小贴士:

*   花时间休息。

问:你是如何克服学习中的绊脚石的？

一开始最重要的一点是动力。你没有同学或者面对面的老师来督促你。由你来安排工作，为自己想出任务，并实际去做。我不认为我已经完全克服了这一点，但我肯定可以看到一个项目结束，这是我在真正开始学习 web dev 之前遇到的困难。

我认为，在一个项目中，让别人和你一起参与总是更好；他们给你建议，新想法，新技能，给你一个为项目工作的理由。有一个伙伴在身边，你不仅仅是为了自己，所以你更有可能打开你的 IDE 开始工作。这是我的经验。

## 大约两年前，我第一次听说 freeCodeCamp。

[![Jr](../Images/d6cd0dd738ce41e8197a40f5612aa7a4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Zh6WU9fI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://miro.medium.com/max/1012/1%2A26yOc07T7L1f0SLkk9Xxzw.png)

我有时会看马特·陈(Matt Tran)的 YouTube 频道《工程真相》(Engineered Truth)，我发现这很有趣，也很有违常理，他给那些打算读完传统的 4 年大学的工程师们提供了一些反驳性的建议。

我的意思是，他会采访那些花了 4 年或更长时间获得各种工程学位的人，却发现他们最终做的工作根本不是他们想象的那样，实际上相当没有回报。

### 我当时看到的几个视频或多或少暗示了相当多的工科学生被教育范式欺骗了。

[![freecodecamp](../Images/b8ab1543eff7eb4f7a70b696196176d6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_FCMloRj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://miro.medium.com/max/1102/1%2ASO9xrFTrPRWkIX_xWNW-ng.png)

然而，当他谈到编写 bootcamps 代码的途径(STEM 工作的传统途径的替代途径)时，他似乎没有像他在谈到例如如果一个人的目标是做 web 开发，花四年时间获得计算机科学学位时那样严厉地揭开虚幻的面纱。

他提到的备选路线之一是免费代码营，我记下了这个名字。

考虑到工程真相频道对许多其他 STEM 课程的批评，他似乎尊重 freeCodeCamp 项目，并且 freeCodeCamp 在我眼中获得了可信度。直到一年多以后，我才真正去看看 freeCodeCamp，但这主要是因为 YouTube 频道的支持。

(还有，freeCodeCamp 是免费的——所以……)

当我到达课程的 JavaScript 算法和数据结构部分时，我发现自己被一个问题的一个方面难住了，我不能完全理解，需要一点提示。我不想要一个直截了当的答案，而是要适量的长辈指导，帮助我看到我没有看到的东西。

### [](#i-got-a-response-two-responses-within-an-hour-by-two-freecodecamp-forum-sages)我得到了回应——两个回应！—在一个小时内由两位 freeCodeCamp 论坛的智者完成。

[![freecodecamp badge](../Images/aaab0936be62f87ea513133e7f64e39e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--qcI88LLf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://miro.medium.com/max/1260/1%2AFFYAUdq35LpZLlHoNO-JnA.png)

人们总是在那个论坛上，我很欣赏这个论坛，因为它是一个远离自我学习的孤岛。自学的旅程很长，感觉反社会，但 FCC 论坛提供了一些社区意识，我认为这对保持心理健康和动力是必要的(例如，你可以寻求帮助，而不是因为挫折而放弃)。

这听起来像是一个悖论，但我认为“自学”路线相对于其他路线的一个优势是，在自学时，你很容易偏离轨道，停止进步。

我的意思是，如果你能坚持自学，那么你要么是高度自我激励和自律的，要么你会因为走自学路线而发展出自我激励和自律。

不得不服从不是你的人设定的外部截止日期，尽管这显然是有效的，但从某种意义上说，这似乎更像是一种被动的动力，它让你被别人的意愿所驱使。另一方面，根据自己的意愿和计划，在 6 个月到 1 年内完成学习全栈开发的严格目标似乎更积极主动，可能会提高你的自我启动能力。

问:你在学习 freeCodeCamp 时遇到了哪些明显的障碍，你是如何克服的？

正如我前面提到的，当我到达课程的 JavaScript 算法和数据结构部分时，我开始面临前进的阻力。

这个绊脚石很明显，因为我必须开发一个系统的方法来解决这些问题。我不能只是坐在扶手椅上思考一个算法挑战，并期望找到解决方案。这种系统化的方法包括在干净的桌子上放一张干净的纸，并用工整的笔迹写出我的代码通过算法挑战需要遵循的确切步骤。

### 对我来说，干净的纸、干净的桌子和整洁的笔迹是通过算法挑战的必备条件。

这个仪式给了我一个清晰的临床视角，允许我关注每个编码挑战中什么是相关的，什么是不相关的。

当我开始 freeCodeCamp 课程时，我强调每天都要编码——即使只有 20 分钟。

我有一个书面日志，记录了我花了多少时间做与编码相关的活动。此外，我甚至写下了我每天在 freeCodeCamp 课程中工作的部分。所以，如果你问我在 12 月 31 日晚上编码了什么，我可以告诉你(是的，我在新年前夜编码！).

## 这种奉献精神正是我走“自学”路线的原因。

我希望最终能在一家鼓励创造力和创新的公司工作。

我理想的最终就业地点将充分利用我广泛的技能。我是一名音乐家、语言爱好者(我喜欢研究“死”语言)、电子修理工和初露头角的程序员。

基本上，最理想的情况是，我能在某个既需要“右脑”又需要“左脑”的地方结束。

问:你能给那些对自学持怀疑态度的人什么建议或见解？

我相信任何对自学成才的开发人员的怀疑都是可以理解的。

如果有人参加了一个严格组织的训练营或大学，他们会得到这个机构以完成证书的形式为他们担保。

然而，任何通过自学成功获得知识的人都有明显的主动性。在没有外界支持的情况下完成一件事需要很大的决心，我甚至可以说这反映了对材料的真正兴趣。你可以保证在某一学科中自学的人是勤奋的，并且对所学的东西真正感兴趣。

### 大约六年前，当我第一次对从计算机支持到 Web 开发的职业转变感兴趣时，我在互联网上寻找学习如何编程的好资源。

[![donovan](../Images/9429b2988626b875901857d41b2bc076.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--g3wYIwj6--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://miro.medium.com/max/800/1%2AP0IHXPsRObs2i7sYsz5XUA.jpeg)

很久以前，你的选择非常有限。Udemy.com 对我来说很突出，因为你可以选择各种各样的课程，一次性购买一门课程，它就是你的终身课程！你可以随时按照自己的节奏学习课程，最后，那些每月销售额超过 100 美元的 Udemy.com 课程会降到 10 美元到 35 美元不等。你真的打不过它。

问:你认为自定进度、自学的开发路线比更结构化的方式(如编码训练营或大学)有什么优势？

自学/自定进度你可以慢慢来，按照自己的进度学习——这比去编码训练营或大学要便宜得多。这对那些有工作、家庭或其他重要责任的人来说非常好。

编码训练营可能很贵，你很可能不得不放弃几乎所有的事情来参加编码训练营，而且不是每个人都有能力投资这些类型的承诺。大学也很昂贵，然而当你毕业时，你可能会学到过时的技能。另外，成为一名开发人员不需要学位。

问:你在 Udemy 上学习时遇到过哪些明显的障碍，你是如何克服的？

参加一次课程，并宣称自己是“大师”，因为你参加了一次课程:为了避免这种情况，请做笔记，与讲师一起编写代码，如果你不理解某个概念，请重播或回放视频，在问答板中提问或谷歌一下。

## 如果可以的话，真的要努力去理解至少尽可能多的东西。

我喜欢从头到尾上两到三次 Udemy.com 的课，这样核心概念会一直跟着我。

一次上各种课程:**不要这样！特别是如果你是初学者，坚持一次只学一门我的课程**。弄清楚你想完成什么或专攻什么，坚持学习这门课程，然后继续下一门课程。****

多诺万强调一次只专注于一门必修课的重要性，甚至多次选修以确保知识的巩固。Unsplash 上 Chase Clark 的照片

参加质量差的 Udemy 课程:如果你对 Udemy 课程感兴趣，我强烈建议你在投资一门课程之前，查看一下学生评论，看看其他学生是怎么说的。如果你碰巧买了一门你不满意的课程，或者你理解老师有困难，Udemy 提供 30 天退款保证，所以利用这一点来学习不符合你学习标准的课程。

我通常投资不低于 4 星的课程。

## [](#advice-to-code-newbies)给代码新手的建议:

*   腾出时间来学习:因为你是自学的，所以你要对你的成功负责。

*   我喜欢在任何地方学习和参加课程，每天至少 30 分钟到大约 3 到 4 个小时，并在必要时使用番茄工作法(如果你不知道这种技术，请查阅)休息一下，进行平衡休息，这样你的大脑可以有机会充电。我个人没有孩子或家庭要照顾，所以你个人对学习的投入时间可能会有所不同。

*   学会自律:有点重复前面的说法。作为一名自学成才的开发者，你可能需要牺牲一些东西。我花更少的时间玩电子游戏(这里的大玩家！)，看动漫，参加漫画书和游戏大会，和朋友出去玩/花更少的时间和有毒的人在一起，关闭我的脸书账户。在学习编码时，我并没有完全放弃这些(除了脸书)。

*   如果可能的话，我至少会每天安排一些时间学习。

多诺万抛弃了脸书，这样他就可以专注于更重要的事情，比如学习编码。

## 找到抵制这些诱惑的方法，我保证结果会显现出来。

我喜欢做的一个学习技巧可以追溯到番茄工作法。通常在周末，我会在当地的咖啡馆学习几个小时的代码，然后我会奖励自己，因为我花了很多时间学习代码，然后回家，玩视频游戏或和朋友出去玩。我发现进入正确的氛围，没有分心或拖延的诱惑，真的有助于我长时间保持专注。

如果可以的话，去参加聚会。如果你不知道，在 Meetups.com 上有各种各样基于编码的聚会，几乎在世界的任何地方都有。

从普通的 JavaScript 到网页设计，几乎每个城市都有 Meetups，它们迎合了众多网页开发的需求。

加入编码聚会，与其他技术专家交流。你可能会发现有人也在学习，可以互相提供提示，帮助，成为学习伙伴，或者最终可以通过网络找到一份开发工作。

### 如果你想在学会如何编码后找一份开发工作，拥有一个好的稳定的工作关系网是关键。

问:你花了多长时间学会足够自信去申请工作？

做了五年自学成才的 dev！我会帮你节省 5 年的时间。:)

搞清楚自己想专攻什么，是前端开发，后端开发，全栈开发，移动等。并且坚持下去！

在之前的 RTC 博客文章中，我提到我浪费了宝贵的时间从一个堆栈跳到另一个堆栈，从一种技术跳到另一种技术，从一种编程语言跳到另一种编程语言，认为这是成为一名成功的开发人员的正确方法，而当时我没有导师或指导告诉我这是绝对错误的，我在浪费时间。

长话短说，我得到了这个教训。新闻快讯，你不必学习编程和开发的所有知识就能获得成功。
