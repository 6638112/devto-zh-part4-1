# 改革内容管理系统

> 原文:[https://dev.to/shannonreca/revolutionizing-the-cms-377f](https://dev.to/shannonreca/revolutionizing-the-cms-377f)

当你看今天的 CMS 时，大部分(如果不是全部的话)遵循传统的主题和插件开发概念。完美的 CMS 是不存在的，这也是插件存在的原因，它提供了一定程度的定制能力来满足人们的需求。

那么是什么将一个 CMS 与另一个 CMS 区分开来呢？嗯，有一些特性通常会被强调，还有 CMS 的类型，比如数据库驱动、平面文件、静态等等。然而，我发现自己又回到了同一个问题。概念上，有什么区别？

我得出结论，没有区别。当然，功能可能不同。存储和交付数据/内容的过程可能不同。无论是哪种情况，都要回到主题和插件上来。

作为一名开发人员，我认为时间花在了“再造轮子”上当你自己有一个代码库，你可以不断地复制代码，并稍加修改来完成一个项目时，这就很容易了。然而，你总是会跨越一个你根本没有它的点，并开始在网上搜索免费的开源库，在那里你可以重新适应你的项目。这基本上是每个开发者。

从我创建 SumCMS 和 BKDR 的那几天起，我就一直在思考，从开发和用户的角度来看，怎样才能成为一个优秀的 CMS？今天，我相信我有了，**不是答案**，而是**一个答案**，可以解决大多数 web 开发人员的一系列共同兴趣。

### [](#the-wish-list)愿望清单

*   它不像今天所知道的那样使用“主题”。
*   它不使用“插件”。
*   它应该是模块化的，可以与其他网站互换。
*   它应该允许您从本质上创建所需的特性，而不是等待特性被实现。

对它感兴趣吗？良好的...我的 MVP 公开发布已经完成了大约 60%,但我真的很兴奋能分享到目前为止我所拥有的。所以我没有发布一个半成品项目，而是决定创建一个订阅列表，这样你就可以在正式发布之前注册并获得最新的特性信息。

介绍基于 JSON 的开发管理系统 [**JayDMS**](http://jaydms.com/) 。我期待在这个项目背后创建一个社区，因为我真的相信这是 web 开发的一个全新面貌。感谢你的兴趣和花时间阅读这篇文章，并希望注册。
