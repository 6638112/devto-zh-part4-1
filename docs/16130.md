# 同情另一个开发人员的代码

> 原文:[https://dev . to/luci unknown/empathy-with-other-dev-s-code-4 abm](https://dev.to/luciunknown/empathy-with-another-dev-s-code-4abm)

我作为一名软件开发人员已经工作了几年，我必须处理的最常见的事情就是弄清楚另一个开发人员写的一些代码是如何工作的。

在我开始的时候，经常听到我批评别人写的代码，即使这些代码运行良好，而且在大多数情况下是基于个人的偏好和选择。

随着时间的推移，我学会了与另一个开发人员的代码产生共鸣。每个人都有自己的背景、信念、政策和思维模式，只要他们的代码遵循团队/公司惯例，并且不存在一些实际问题，就没什么好担心的。
