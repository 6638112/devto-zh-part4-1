# 结构化编程范例

> 原文:[https://dev.to/vorakl/structured-programming-paradigm-4gi9](https://dev.to/vorakl/structured-programming-paradigm-4gi9)

曾经有一段时间，计算机程序是如此的冗长和无结构，以至于在大型软件项目中，有时只有少数人能够逻辑地浏览源代码。使用低级编程语言，程序员使用“goto 语句”的各种等价物进行条件分支，这通常导致可读性降低，并且难以保持逻辑上下文，尤其是跳到另一个子例程时。

在最终以[结构化编程范例](https://en.wikipedia.org/wiki/Structured_programming)的形式出现的解决方案的过程中，发生了一些事情。1966 年，*科拉多·博姆*和*朱塞佩·雅科皮尼*证明了一个[定理](https://en.wikipedia.org/wiki/Structured_program_theorem)即任何可以用流程图表示的计算机程序都可以只用 3 种控制结构(顺序、选择、迭代)重写。

1968 年， *Edsger W. Dijkstra* 发表了一篇很有影响力的文章“ [Go To 语句被认为是有害的](https://vorakl.com/files/goto/Dijkstra68.pdf)”，他在文章中指出，使用过多的 **goto** 语句对计算机程序的可读性和理解有负面影响。然而，不幸的是，他的意图被误解和误用了，因为他几乎完全放弃了在高级编程语言中使用“goto ”,甚至以可读性更差和模糊的代码为代价。

作为改进 ALGOL 语言的成果，*尼古拉斯·沃斯*设计了一种新的命令式编程语言，[帕斯卡](https://en.wikipedia.org/wiki/Pascal_(programming_language))，并于 1970 年发布。从那以后的几十年里，它被广泛用于教授学生结构化程序设计。
