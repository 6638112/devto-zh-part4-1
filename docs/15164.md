# 享受不舒适

> 原文:[https://dev.to/llotz/enjoy-uncomfortability-4k9d](https://dev.to/llotz/enjoy-uncomfortability-4k9d)

### [](#no-pain-no-gain)没有付出就没有收获

这不仅仅对在健身房训练有效。不时地离开你的舒适区去接触最新的技术是很重要的，尤其是当你受雇于 it 行业的时候。有时，当一项新技术发布时，我听说了它，我想“啊，没有那个框架，我也会来的。”只是因为现场没看懂，怕学那个新东西。

> 哦，我可以写普通的 PHP。过去 10 年都是这样，不需要拉弗尔。
> 
> 不，香草 JS 做的，我根本不需要反应。
> 
> 。网芯只是一个趋势，还是停留在。新项目的. NET 框架。

我打赌你以前说过一次。当然，并不是每一个流行的框架、数据库或服务器软件都需要学习，对于某些主题来说，只需触及皮毛就足够了。但是如果你一直这样会怎么样呢？

基本上，会发生两件事:

1.  你忘记了如何学习新的东西，必须跨越的界限变得越来越高。
2.  你会排在每一行的最后。

我打赌你也能在那个故事中找到自己:

> 有一次，我被要求为我的同事们举办一个关于 git 的研讨会。甚至在我考虑之前，我听到自己回答道:“是的，当然。”我做了什么？我离 git 的尖端还差得很远，我使用了一些基本的东西，但仅此而已。我曾经为一小群人举办过一次研讨会，但我甚至忘记了该怎么做，也不知道那东西是如何工作的。
> 将所有这些差异抛在脑后，我开始创建一个降价文件，该文件本应在我的研讨会中起主导作用。假设我是我刚刚创建的工作室的一员，我开始研究并写下有用的东西。我把我要教授的知识按照顺序排列，随着参加研讨会的时间增加。研讨会召开了，交付了我所准备的东西，这太棒了。我得到了积极的反馈，感觉棒极了！

并不是说我只是增加了我同事的知识。在研究过程中，我开始更好地理解自己的事情。

有时一个草率的答案可能是学习新事物或改进某事的解决方案。反思这段经历后，我开始更频繁地走出自己的舒适区。
