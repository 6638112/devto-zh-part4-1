# 从建立和部署投资组合网站中学到的经验教训

> 原文:[https://dev . to/lewiskori/lessons-learn-from-building-and-deployment-a-portfolio-website-563g](https://dev.to/lewiskori/lessons-learnt-from-building-and-deploying-a-portfolio-website-563g)

> “经验是最好的老师。”

我开始制作我的作品集网站，作为摆脱困扰我很久的教程炼狱的一种手段。忠于上面的陈述，正如它被广泛宣扬的那样，通过构建一个真实世界的项目来弄脏你的手是一个在软件开发中获得你的脚的可靠方法。我选择我的初始项目作为投资组合网站，这一经历给我留下了巨大的影响。这里只是获得的一些见解。

### [](#discovering-my-passion)发现我的激情。

当我开始制作网站时，最初的目标是展示我做过的一些数据科学项目。当时，我真的很喜欢数据科学和分析，正如人们所称赞的那样

> “21 世纪最性感的职业”

然而，当我制作 web 应用程序时(我选择了 Django 作为开始)，我开始质疑自己是否像我想象的那样喜欢数据科学，或者是围绕该领域的大肆宣传将我推向了那个方向。我是否设想过自己在余生都这样做？

我越来越多地为我的网络应用做后台，我越来越注意到我实际上很兴奋，很享受软件工程，而不是我以前的激情。不用说，我仍然是数据分析的爱好者，我相信从所有实践中获得的知识不会浪费。然而，我们都有不同的成功之路。

如果没有尝试不同的领域，我不会找到我的家。将来事情可能会改变，这就是知识的美妙之处。你永远不会停止学习。

### [](#focusing-on-my-strengths)专注于自己的长处。

> “我将如何应对所有这些技术？”

这是我在开始攻读计算机科学学位时经常问自己的问题。随着信息的不断流入，选择拿起什么和放下什么变得非常混乱。再加上担心离开一个技术团队会让我在就业市场上不够好，这让我变得一团糟。然而，很快，有人告诉我拿起一件事，然后顺其自然。所以我选择了 Python 编程语言，用 Django 做了网站。我在 HTML 方面完美吗？没有。所以我选择请一个朋友帮助我处理前端，因为我专注于后端。不用说，随着时间的推移，我学会了那些我不具备的技能，我意识到所有美好的事情都需要时间。一次专注于一件事。

### [](#opportunity-for-growth)成长的机会。

所以我在我的电脑上完成了这个项目的文件。为我的小胜利感到骄傲。然后我开始思考。“接下来呢？”显然，下一阶段是部署。所以这带来了一个你不知道存在的全新世界。在此基础上，我研究了网络服务器，如何选择主机提供商，区分 Nginx 和 apache😂。起初一切都很忙乱，但是我又想起来了，所有的好事都需要时间。
我今年年初做的这个教程，应该对你到这个阶段有帮助。
