# 减少首次成功的时间

> 原文:[https://dev . to/cameronblandford/decrease-your-time-to-first-success-2o7d](https://dev.to/cameronblandford/decrease-your-time-to-first-success-2o7d)

学习新事物时，从最小的目标开始。

当你学习音乐理论时，优先写一首只有旋律的小叮当。做很多这样的！当旋律结束时，就认为它们完成了。

当你在学习游戏设计的时候，尝试使用一个有关卡设计功能的游戏，或者为你喜欢的游戏设计修改过的关卡/ steam workshop 之类的东西。

当你学习编码时，在做其他事情之前，花些时间在 LAMP 上创建静态网页。

你参与技能发展的能力很大程度上取决于首次成功的时间。时间越短，你就越快陷入正反馈循环。我的意思不是把建造这样的东西作为把它变成更大的东西的一部分。我的意思是，你试图建立的小东西应该发挥 MVP 的功能。当你完成了它，你开始了一个新的项目，并能从完成对你有意义的事情中感受到成就感，不管这些事情有多小。
