# 在 Docker 变得更好的好项目/挑战？

> 原文:[https://dev . to/jchetters/good-projects-challenges-to-get-better-at-docker-2ha 3](https://dev.to/jchetters/good-projects-challenges-to-get-better-at-docker-2ha3)

我是前端开发人员。我喜欢 UX、CSS、JavaScript / React、GraphQL、Node，但我从未喜欢过 devops 和基础设施。

我知道 Docker 的基本知识，但我讨厌使用它。我们工作的管道非常复杂，所以当有东西坏了的时候，我从来不知道如何修理它。

有哪些好的项目或挑战可以让你对 Docker 更加得心应手？我发现韦斯·博斯的课程非常适合学习 JavaScript，因为它是由许多小任务逐渐变难的。我发现很难激励自己，因为我有任何很酷的项目想法都不需要 Docker。
