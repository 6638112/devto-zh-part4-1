# 可访问性:隐藏 CSS 中的内容

> 原文：<https://dev.to/adrianbdesigns/accessibility-hiding-content-30of>

在 web 开发中，可以安全地假设隐藏某些元素或部分内容是不可避免的。只使用一种隐藏内容的方式会损害可访问性，因为使用辅助技术的用户不会像普通屏幕用户那样获得页面的完整上下文。在本文中，我们将根据可访问性介绍几种隐藏元素的方法。

## 对所有设备隐藏内容

当隐藏元素或部分内容时，首先想到的是使用 CSS 规则，如`display: none;`或`visibility: hidden;`，或 HTML `hidden`属性。使用这两个 CSS 属性将使显示设备和辅助技术(如屏幕阅读器)完全忽略该元素，不向用户显示它。

当然，在某些情况下，我们希望对所有设备隐藏内容。例如，我们可以在折叠、标签或其他可切换元素中使用这些 CSS 规则。

## 对屏幕设备隐藏内容

在某些情况下，我们希望在显示器上隐藏一些元素，但我们希望让辅助技术向用户呈现视觉上隐藏的内容。我想到的一个例子是无标签输入。标签通常用`display: none;`隐藏，但是我们更希望标签可以被辅助技术访问，以便为用户提供更多的上下文。

如果我们只想隐藏屏幕设备上的内容，这个解决方案有点粗糙，但是非常有效。我们可以使用下面的 CSS 规则:

```
.hidden--visually {
    border: 0;
    clip: rect(1px 1px 1px 1px);
    clip: rect(1px, 1px, 1px, 1px);
    height: 1px;
    margin: -1px;
    overflow: hidden;
    padding: 0;
    position: absolute;
    width: 1px;
} 
```

## 对辅助设备隐藏内容

有些情况下，我们希望对屏幕阅读器和其他辅助技术隐藏内容。例如，如果我们有一个图像轮播，开头是“上一页”按钮，中间是轮播图像，结尾是“下一页”按钮，我们不希望屏幕阅读器按照这个顺序读出它。我们可以提供一个单独的、更易访问的控件，这些控件在视觉上是隐藏的，但是对辅助设备隐藏“上一页”和“下一页”按钮。

如果我们想对辅助设备隐藏内容，我们可以使用`aria-hidden="true"` HTML 属性。

## 超越怪癖

需要注意的是，像`display: none;`这样的 CSS 规则会覆盖`aria-hidden="false"`并且内容会被辅助技术忽略。
