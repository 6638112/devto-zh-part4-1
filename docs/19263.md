# 有效的团队:当你是一个情绪受到挑战的开发人员时

> 原文:[https://dev . to/mattkocaj/effective-teams-when-you-re-a-emotional-challenged-developer-308](https://dev.to/mattkocaj/effective-teams-when-you-re-an-emotionally-challenged-developer-308)

我是一个情感上受到挑战的开发者！

我不是世界上社交意识最强的人。我有时会变得粗鲁、极权、“直接”、敌对，甚至是一个十足的混蛋。我是坏人吗？不要！人们喜欢我吗？显然，他们会邀请我吃午饭，下班后一起喝酒，问我好不好，一般都希望在我身边。我只是说我和每个人一样有我的优点和缺点。坦诚面对这些很重要，如果你有足够的自信承认这一点，那么试着与同事分享——你会惊讶于它能给你的工作环境带来的力量。

因此，以下是我的三个提高团队效率的建议。这些对那些发现很难以非技术能力融入团队的人特别有用

## [](#be-kind)为人和善

*   与你的同龄人交谈。
*   和他们一起吃午饭。
*   给他们买咖啡。
*   让他们给你买咖啡(对我们这些难以接受礼物的人来说尤其重要)！
*   在讨论中要有耐心，即使事情让你感到沮丧。
*   记住你不需要总是赢。团队合作就是妥协，就像所有的人类关系一样。为了清楚起见:在软件团队中工作是人际关系的集合。是的，你做"[做船](https://www.youtube.com/watch?v=F3yZYkE32Ec&feature=youtu.be&t=59)"！
*   尽你所能去参与。如果你很内向，那么偶尔分享你的想法和观点是很重要的。别人需要知道你在乎，哪怕只是一点点。请尝试加入进来。
*   并且永远记住:不要打断别人！太没礼貌了。尊重是人类交往的基石。如果你经常打断别人或者谈论他们，那么请开始倾听你自己。如果你自己还听不到，先听听别人是谁做的。重要的是要在这一点上有所突破。如果你设法控制住了这种情况，你就不会让别人模仿你，因为你突然变得有礼貌了。但是没关系。

## [](#be-vulnerable)不堪一击

*   与你的同龄人分享一些关于你自己的事情。不一定要在一个团体中，可能只是一个你似乎有点信任的人。不一定是“我的婚姻正在失败”或“我的爸爸刚刚去世”，但它应该是透露一点关于你的事情，你没有“拥有一切”。
*   向别人展示你并不完美。“接得好”——在拉动请求时；或者“我的傻瓜！我会修复损坏的构建”。换句话说，表明自己犯错是可以的。从纯知识传递的角度来看，这些都是不必要的，但是我们在这里不是在和机器打交道。人们想知道他们能认同你，你也是人，像他们一样不完美，也会犯错。这有助于你变得更讨人喜欢，而你确实想变得讨人喜欢。
*   求教(哪怕需要装一点)。想办法让初级开发人员帮助你，即使你认为自己更有经验。这不仅会增强他们的信心，还会帮助他们更加信任你。当你提出大规模重构肯定会减少技术债务，而你需要一个额外的盟友时，这就特别方便了。

## [](#give-something-back)回馈

*   你和某人的时间配对；提议评审更多的代码；花更多的时间来解释你的激情、观点和你如此热衷的偏好的原因。
*   购买咖啡或甜甜圈(或不含麸质和香料的纯素食“强力球”)。如果你没有钱，写一张便条纸给你的同事，感谢他们昨天的帮助。只是不要向别人炫耀。
*   喊乡亲们搭车去餐厅吃午饭；接一个从公交车站走过来的同事，因为你今天有车；当你看到他们忙得不可开交时，主动提出帮他们带装备去开会。记住，你是一个善良的人——你可以通过给予来表现出来。
*   一个方便的生产力提示，一个#random 频道的有趣链接(每个人都可以多笑笑)，一个笔记本电脑的库尔贴纸，一个新的鼠标垫(哈哈哈哈哈！抓住你了。没人再用鼠标垫了！)，或者只是在成功部署的最后击掌。布雅！是的，如果合适的话，在办公室接触其他人是可以的。当你已经解决了那个 bug，部署到 PROD 中，或者从客户那里收到了非常棒的反馈时，击掌肯定是合适的。

这些大多数都很容易，你可以学习。他们是可以被教会的——如果你不确定，向同伴寻求帮助，他们可能会因此更加尊重你。此外，我也不是这方面的专家。它们是每天的挑战，但我愿意为之努力。我希望你也能这样想。

这就是我今天的全部内容。我很快就要去参加一个团队午餐，并试着将这些付诸实践。

祝你好运！
