# 高效的每日 scrum 会议

> 原文:[https://dev . to/mikulskibartosz/an-efficient-daily-scrum-meeting-1p J2](https://dev.to/mikulskibartosz/an-efficient-daily-scrum-meeting-1pj2)

在一个典型的、功能失调的日常 scrum 会议中会发生什么？最有可能的是，一群人站在一台电脑前看着吉拉。他们还会更新任务状态，因为他们太忙了，没有时间在会议前更新。

他们每个人都讲述了他们前一天所做的事情。他们说他们今天想做什么。很多时候，他们说的话是“我将在吉拉完成下一个任务”大概没人敢提什么问题吧。

他们彼此不说话。房间里还有别人。当地的半神，有权力奖励服从和惩罚不正当行为。通常是项目经理或者产品负责人。团队看着她/他，把“每日单口相声会”变成状态报告。

至少他们还站着。显然，站起来是每天 scrum 会议中最重要的部分。没有人会抱怨旁敲侧击、打断别人、不听同事的话，但是千万不要在会议期间试图坐下来！他们会把你生吞活剥！

会议结束后会发生什么？团队再次开会，因为他们必须计划即将到来的一天。在过去的 15 分钟里他们在做什么？！

# [](#daily-planning-meeting)每日计划会议

我们能做得更好吗？我有一个建议。首先，更改会议名称。说真的。我知道这只是一个名字，但单词是有意义的，这个名字会影响你在会议中的态度。

这不是一个站立会议。没人在乎你是站着、坐着、走着还是跳着。如果你称之为“站着说话不腰疼”，那么站着说话不腰疼将是你唯一关心并正确做的事情。

取而代之的是，我建议将其重命名为“每日计划会议”我认为它清楚地传达了目标。毕竟，我们应该计划一天。

会议本身也必须改变。我们不需要著名的三个问题。先说第一个。

## [](#what-i-did-yesterday)我昨天做了什么？

不要告诉我们你昨天做了什么！

我愿意相信我和值得信任的成年人一起工作。这意味着如果人们告诉我他们将要做某事，我会认为他们已经做了。这也意味着，如果他们在完成任务方面有任何问题，他们会通知每个人，这样我们就可以一起处理它。

## [](#do-i-have-any-problemsblockers)我有什么问题/阻碍吗？

我不明白为什么人们在会议结束时回答那个问题(或者根本忘记做)。这是最重要的一条！

如果出现问题，整个团队的计划可能会改变。首先制定一个计划，然后问是否有更重要的事情或事情阻止我们实现目标，这是没有意义的。你需要重新计划一整天！

这就是为什么讨论问题应该是我们在每日计划会议上做的第一件事。如果有什么阻碍了团队实现 sprint 目标，应该尽快公布出来。

## [](#what-am-i-going-to-do-today)我今天要做什么？

讨论完问题后，我们终于可以计划即将到来的一天了。

在这一部分我也想提出一个改变。我们只谈有助于实现 sprint 目标的任务。

### [](#one-goal)一个目标

我假设团队只有一个冲刺目标。(不准作弊！如果冲刺目标里有连词，你就不止一个！)我还希望团队中的每个人都致力于与这个共同目标相关的任务。

老实说，我认为如果你有多个目标，或者一些团队成员在兼职目标上工作，你就不再是一个团队了。

没有共同目标的团队只是一群人，他们碰巧在同一个房间，使用相同的吉拉，将代码推送到同一个存储库，在同一个电子表格中跟踪他们的工作时间，但不需要相互合作。也许他们应该参加单独的每日计划会议？

# [](#nothing-more)仅此而已

仅此而已。看起来很简单，不是吗？换个名字。忘了一个问题。颠倒剩余问题的顺序。仅此而已。

看起来是一个小变化，但这个小变化可能会帮助你专注于工作的关键方面:实现 sprint 目标。

你是来完成工作的，不是来浪费时间在无意义的会议上。
