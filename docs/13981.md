# 在哪里发布内容？

> 原文:[https://dev.to/maxkatz/where-to-publish-content-53ao](https://dev.to/maxkatz/where-to-publish-content-53ao)

有很多网站可以让你发布内容。但是，什么地方最好呢？嗯，我不喜欢“最好”的东西。你的“最好”取决于你，什么适合你，基于你的要求，每个人都不同。在这篇博文中，我将分享我对在哪里发布内容的建议，你可以用它们来决定(或不)在哪里发布。

我想发表这篇文章有一段时间了，也因为最近与媒体的争议。你可以在这里阅读[，在这里](https://dev.to/devteam/medium-was-never-meant-to-be-a-part-of-the-developer-ecosystem-25a0)阅读[。](https://dev.to/ben/i-m-concerned-with-the-move-that-freecodecamp-just-pulled-by-leaving-medium-io8)

有许多网站可以发布内容。这只是一个简短的网站列表:

*   媒体(加上各种媒体出版物)
*   德瓦达(以前的 Dzone)
*   Dev.to
*   LinkedIn(发布平台)
*   InfoQ
*   个人博客
*   工作博客
*   和许多其他人…

我的建议几乎总是首先在你自己的个人博客上发表。这有许多原因:

1.  它允许你建立你的投资组合，并把你所有的工作放在一个地方
2.  它允许建立你的个人品牌
3.  它可以作为你的“第二份”简历，因为它表明你确实发表过论文。在简历上写任何东西都很容易。博客展示了实际的工作
4.  当你得到一份新工作时，你发布的内容将会出现在你的个人博客上，而不是公司的博客上

为了接触到更多的人，有很棒的社区，你可能想在那里重新发布内容或进行联合。例如， [Dev.to](http://dev.to) 对于开发者来说是一个很棒的社区。Dev.to 允许你整合或重新发布你的博客内容。它甚至可以自动从你的博客中导入内容，并将内容保持在草稿模式。您仍然需要单击发布(如果愿意，您也可以在发布前编辑内容)。你也可以写一篇新的污点文章。

当一篇博客文章被导入时，Dev.to 向导入的文章添加一个[规范 URL](https://en.wikipedia.org/wiki/Canonical_link_element) 。当一个圆锥 URL 出现时，它告诉搜索引擎原始内容在哪里发布，避免重复内容。

当一篇文章在没有经济 URL 第三方网站上重新发布时，会导致搜索引擎将其视为重复内容，并对其 SEO(搜索引擎优化)产生负面影响

Medium 还支持导入现有的博客文章，它还添加了一个经济 URL。

LinkedIn Publishing 是一个很好的发布内容的平台，但是不允许导入现有内容，也不允许你手动设置经济网址。如果你想重用现有内容，但不想简单地复制和重复它，一个选择是在 LinkedIn 上写一个新的介绍段落，然后链接回原始文章。

我建议在工作博客上采用类似的策略。如果你的公司允许这样做，首先在你的个人博客上发布内容，然后在公司的博客上整合内容。

正如我在开始提到的，这些只是一般的指导方针。如果你认为在公司的博客上发表一篇新文章是正确的做法(然后在你的个人博客上重新发表)，那就去做吧。只有你自己才能决定什么对你“最好”。

总的来说，内容是扩展你的宣传计划的绝佳方式。你可以向世界上任何地方的人提供内容。了解[内容如何创造内容](https://dev.to/ibmdeveloper/how-content-creates-content-3edj)来帮助你扩展。
