# 在空旷的地方建设未来。

> 原文：<https://dev.to/koredefashokun/building-the-future-in-the-open-21jp>

> 我相信，在未来，技术的重大进步将由普通人来实现，将代码推向公共存储库。

## 开源软件的力量

随着 OSS 世界的不断扩张，开发人员团结起来构建解决日常问题的软件势在必行。许多公司已经跳上了这趟列车。像 [Comma](https://comma.ai) 、 [Rainbow](https://rainbow.me) 和 [Keybase](https://keybase.io) 这样的初创公司正在公开开发他们面向用户的软件，结果令人震惊。例如，由于社区支持，Comma 比任何其他自动驾驶软件都支持更多的汽车。甚至[黑洞](https://github.com/achael/eht-imaging)成像用的软件也在 GitHub 上。不可思议吧。

## OSS 和开发者工具

纵观当今存在的大多数开源软件，只有很小一部分是面向消费者的。大部分都是开发者工具。尽管开发人员工具很重要，但它们只是触及了开源社区潜力的皮毛。想象一下，如果[上的许多贡献者做出反应](https://facebook.github.io/react)，致力于在自然灾害发生前发现它们的解决方案。<sup>【1】</sup>

## 障碍和解决办法

那么，是什么阻碍了这场能够催化技术进化的运动呢？害怕竞争对手会窃取你的源代码。

好吧，这就是许可证的用处。例如，在 GNU GPL V3 许可证下,“竞争对手”有权克隆和使用你的软件，条件是他们的版本也是开源的。(意思是你可以从他们做出的改进中受益；)).这可能不是一个完美的解决方案，但大多数情况下，获得的贡献超过了复制的代码(读为窃取)。

#### 不要脸的塞:[公然的](https://overt.dev)

为了在建设这一未来的过程中尽绵薄之力，我已经开始着手 [Overt](https://overt.dev) 项目，该项目旨在开发软件，解决人类日常面临的许多问题。首先，我正在开发 [Auxilium](https://github.com/overthq/Auxilium) ，这是一款帮助人们报告身边的事故和紧急情况，并通知附近其他人的应用程序。
