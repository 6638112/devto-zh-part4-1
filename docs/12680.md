# AWS 术语

> 原文:[https://dev.to/thewhitewulfy/aws-terms-388p](https://dev.to/thewhitewulfy/aws-terms-388p)

AWS 中有许多术语对简单的事物使用容易混淆的名称。即使在 AWS 工作了一年多，我还是错过了一些术语。这就是我写这篇文章的原因。输入我记得的任何东西。我正在为我的 AWS 考试学习这些，我将在几周内参加考试！

这是一些条款:

## [](#terms)术语

1.  **访问控制列表(ACL)** :子网级的防火墙/安全层
2.  **自动扩展**:根据应用程序的流量需求，自动添加或删除 EC2 实例
3.  **桶**:根级“文件夹”
4.  **CloudFront** :内容交付网络(CDN)，允许您将内容存储在位于世界各地的“边缘位置”,允许客户更快地访问您的内容
5.  **CloudTrail** :允许您监控 IAM 用户的所有操作
6.  **CloudWatch** :允许你监控你的 AWS 账户的各种元素的服务
7.  **合并账单**:允许您在一个用户界面中查看、管理和支付多个 AWS 账户的账单
8.  **DNS 服务器**:网站域名及其对应 IP 地址的数据库
9.  **DynamoDB** :不提供其他 NoSQL 软件选项的 NoSQL 数据库服务
10.  **弹性计算云(EC2)** :虚拟计算机，非常类似于台式机/笔记本电脑
11.  **弹性负载平衡(ELB)** :在与之相关的 EC2 实例之间平均分配流量
12.  **elastic cache**:数据缓存服务，用于帮助提高运行在 AWS 上的 web 应用程序的速度/性能
13.  **弹性**:一个系统增加和减少大小的能力
14.  **容错**:在一个或多个组件出现故障的情况下，使系统能够继续正常运行的特性
15.  **防火墙**:一种允许或阻止特定种类的互联网流量通过的软件
16.  **文件夹**:在桶中创建的任何“子文件夹”
17.  **高可用性**:指持久的、可能长时间连续无故障运行的系统
18.  **IAM 用户**:被授权访问 AWS 帐户的个人
19.  **身份和访问管理(IAM)** :管理 AWS 用户帐户及其对各种 AWS 服务的访问的服务
20.  无服务器计算将在很大程度上取代 EC2 实例
21.  **对象可用性**:存储在 S3 的文件在一年时间内可被访问的百分比
22.  **对象持久性**:存储在 S3 的文件在一年时间内不会丢失的百分比
23.  **对象生命周期**:设置规则，以定义的时间间隔在存储类之间自动转移对象
24.  **对象共享**:通过 URL 链接公开任何对象的能力
25.  **对象版本**:自动保留一个对象的多个版本(启用时)
26.  **组织**:允许您或您的公司在一个用户界面中管理账单和访问多个 AWS 账户
27.  **最低特权原则**:仅授予用户完成工作所需的 AWS 服务和资源的权限/访问权，仅此而已
28.  **发布者**:给 SNS 需要发送的消息的人/警报/事件
29.  **关系数据库服务(RDS)** : SQL 数据库服务，提供多种可供选择的 SQL 数据库选项
30.  **红移**:数据仓库数据库服务，旨在处理数 Pb 的数据进行分析
31.  **角色**:不同的 AWS 服务如何被授予通信和共享数据的许可
32.  **Route 53** :为 AWS 上的网站或应用程序配置和管理域名
33.  **可伸缩性**:系统以经济高效的方式轻松增加规模和容量的能力
34.  **安全组(SG)** :服务器/实例级的防火墙/安全层
35.  **共同责任模式**:定义您和 AWS 在安全和合规性方面的责任
36.  **简单通知服务(SNS)** : AWS 服务，允许您根据 AWS 帐户中发生的事件自动发送电子邮件或短信通知
37.  **简单存储服务(S3)** :几乎可以从任何设备访问的在线批量存储服务
38.  **存储类别**:代表分配给 S3 每个对象的“分类”(标准、RRS、S3-IA、冰川)
39.  **子网**:网络的一部分，通常包括特定位置的所有计算机
40.  **订阅**:主题向其发送消息的端点
41.  **主题**:如何标记和分组你发送消息的不同端点
42.  **值得信赖的顾问**:向您提供“建议”并帮助您优化 AWS 帐户的服务
43.  **用户凭证** : IAM 用户登录 AWS 的用户名和密码
44.  **虚拟私有云(VPC)**:AWS 的一个私有子部分，由您控制，您可以在其中放置 AWS 资源
