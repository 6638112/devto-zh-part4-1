## 什么是 API？

API 缩写的字面意义是“应用编程接口”。然而，API 远不止这些。为了更深入的理解，我们需要关注接口部分。考虑到这一点，我们可以将 API 定义为通过互联网向世界展示其他程序、人类(开发者)和 Web APIs 的接口程序。

开发 API 是为了与其他程序一起运行和工作。因此，考虑到它们被其他程序和人类使用和读取，需要进行开发，毕竟，我们是开发人员，是编写程序的人类，至少在天网征服世界之前，事情会是这样的=】。

如今，API 使得其他不可计数的软件工作/发挥作用成为可能。例如，从那些你用来订购比萨饼的人，甚至到那些管理你的信用卡并使之成为可能的人。随着时间的推移，越来越多的 API 已经成为许多大公司成功的关键，如亚马逊、Stripe 和谷歌。因此，了解你的行为和基本原则是非常必要的，尤其是如果你在日常工作中与此相关的话。

## 为什么我们需要 API？

API 诞生于解决特定问题的数据提供者的需要，他们将这些解决方案分享给其他开发者，这样他们就不需要解决同样的问题。谷歌地图是一个非常好的例子，如果你需要在你的网站中使用地图，你不需要从头开始开发地图。你需要使用谷歌地图的 API，并将其嵌入到你的网站。这个地图怎么开发就不用操心了。你唯一关心的是你的生意，把地图的问题留给谷歌地图的 API。

实际上，这里实际发生的是使用由专业平台提供的数据和交互来开发或创建产品或功能，因此，API 提供的最大优势是它通过节省开发人员的大量工作和时间，以快速和独特的方式实现产品和业务的开发。这是因为他们不需要一直从基数一开始。

## 好吧，我明白了，但是怎样才算是一个好的 API 呢？

当 API 以一种简单、直接和快速的方式帮助你解决问题时，它对你是有益的。这是否意味着这是一个相对的问题？yes and no =)，一般来说，一个好的 API 需要一些重要的方面。

一个好的 API 应该提供清晰性(目的、设计和上下文)、灵活性(适应不同使用方式的能力)、功能强大(需要能够整体解决问题)、可攻击性(能够快速提供交互和实验)以及最后但同样重要的文档。

简而言之，一个好的 API 应该是有用的，可伸缩的，性能好的。一个非常好的文档和好的资源是开发所必需的让你解决你所需要的开发你的应用程序。毫无疑问，API 是现代 web 应用程序开发的基础部分，有很多方法可以使用 API 来开发这种应用程序。因此，有必要了解一些关于这些 API 的知识，以便您能够做出正确的选择，成功地开发您的应用程序。
