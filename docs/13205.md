# [](#over-the-hump)越过了驼峰！

我已经完成了超过一半的[制造商](//makers.tech)课程。作为一名软件开发人员的职业生涯即将到来，从这门课程中学到的东西越来越清晰。

## [](#testing-with-ash-winter)用灰烬检验冬天

[Ash](https://twitter.com/northern_tester) 来创客做软件测试的讲座。他给出了将测试人员融入技术团队的有用建议，以及优秀测试人员所看重的行为。

Ash 提到了测试系统的成本——或者更确切地说，测试系统的价值——这在很大程度上取决于一个产品或公司的风险规避程度。有道理——我希望任何用于我身上的医疗软件都经过严格测试。混合了手动、自动、随机和确定性测试的软件，让开发人员对他们软件的性能更有信心，即使是在大规模测试中——然而，总是有盲点

> 💡软件测试将风险降至最低，但它并不能保证软件不会过时，不会出错。

### 测试人员应该问的三个问题

目标:测试一支笔。

*   你说的钢笔是什么意思？
*   为什么你认为你需要一支笔？
*   你还做过什么测试？

> Ash 提到这也适用于砖块，这导致了我的 Twitter feed 中出现砖块广告。

### [](#great-behaviours)伟大的行为

*   开放性！提供建议并随时可用，以免错过人类或现场反馈的机会
*   对你所做的和不理解的要诚实，要有风度，要成为团队的一员
*   熟悉领域或者熟悉领域的人，甚至(特别是？)技术团队以外的人

## [](#meeting-the-careers-team)会见职业团队

本周，我们与职业团队进行了第一次会面。很难想象 12 周的编码工作会让你变得有工作能力，但是招聘人员的公司是在投资我们的态度、能力、技能和行为——而不仅仅是我们的技术经验！这次会面非常及时，为我们接下来两周的敏捷项目工作、实践过程、主动性和提问做好了准备。

在学生进入职业池并开始寻找初级开发人员的工作之前，Makers 有一个审查过程。评审者寻找解释你的代码的能力，作为一种展示技能的手段，清晰地表达你的过程，并展示你解决超出你的技能水平的问题的方法。

> 下一级开发者也通过参加聚会和黑客马拉松来展示他们在编程社区中的主动性。

## [](#projects)项目

这个星期的项目是一个纯用普通 JavaScript 编写的单页 web 应用程序，测试和所有-一个相当不寻常和有趣的项目！对于周末挑战赛，我们遵循类似的规范创建了一个新闻应用程序，它与 API 一起工作来截断新闻标题。我参加了一个解决 JavaScript 中的中间字符的过程研讨会，我这样做是为了挑战自己，我惊讶于我写 JS 的流利程度——我想我学到了一些东西！
