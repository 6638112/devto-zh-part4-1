# 一个伟大的副业项目的特点

> 原文：<https://dev.to/brightdevs/traits-of-a-great-side-project-l63>

你有没有想过，为什么总有人有一种不变的“造东西”的动力？虽然你可能看起来厌倦了漫长的一天工作，但他们就是无法忍受带着好心情等着离开办公室去做大量的其他事情。

他们是怎么做到的？为什么他们决定花自己宝贵的空闲时间只是为了多工作？众所周知，我就是这些怪胎中的一员，我认为我们这样做只有一个原因…那就是我们选择了我们真正关心的项目。如果你认为那些开源贡献或辅助项目不适合你，让我来改变你的想法。这只是因为你选择了错误的工作，而不是因为整个想法不适合你。我想通过描述“梦想中的那个”来激励你去做另一个副业。也许它会推动你迈出最重要的第一步。

**它解决了一个“真实”的问题**
这听起来很琐碎，不是吗？事实上，我发现这是清单上最重要的事情。做某事(不仅仅是一个项目)最重要的事实是做有用的事情的感觉。它可能很小，甚至可能别人察觉不到，但重要的是它对你有帮助。观察到你让自己的生活变得更容易总是一个巨大的动力。请记住，即使是最小的，但真实的事情也会产生很大的影响。

**你终于可以自私了**
这不是工作，你不需要关心别人。您可以忽略测试、文档和可访问性。并非所有东西都需要漂亮、通用和可重用。一些硬编码的常量？没问题。它只在你的机器上运行——有人在乎吗？成吨的虫子？更重要的是，你的三条“快乐之路”能够发挥作用——这才是它对你有用的地方。可能看起来我喜欢制作低质量的软件，嗯？我只是认为副业需要尽快开始运作。如果这个想法让你工作得很好，你可以以后再润色它们。

你现在是老板
这是你的项目，只有你自己独立完成。你不需要在意别人怎么想。没人能告诉你该做什么，不该做什么。也许你一直想尝试一些“出血的边缘技术”？也许你只是想做些傻事？即使一切都完了，也不会有什么坏事发生。这就是为什么兼职项目是一个很好的实验机会。

你可以慢慢来，这对我来说是一个障碍——我讨厌截止日期。事实上，我必须在特定的期限内交付某些东西，这剥夺了我编程的全部乐趣。这就是为什么我喜欢做一些仅为我个人使用的事情。我不需要关心客户、截止日期、条款和会议。我可以坐下来，打开音乐，打开我的 IDE，写代码。可能不是每个人都有这种“开关”，但我觉得很寒心。

没有失败者
事实证明，我最近的副业对其他人一点用都没有。还有其他更方便的解决方案，这使得说服人们使用我的工具变得非常困难。它会浪费我在这上面花的时间吗？对我来说-绝对不是。我在学习新东西的过程中度过了一段美好的时光，这些东西是我平时都不会接触的。

总结一下，“更重要的是旅程，而不是目的地本身”。
