# 网络开发会让你心碎

## [](#impostor-syndrome-and-not-letting-your-fire-go-out)冒名顶替综合症，欲火未熄

我的第一份网页开发工作是一场灾难。这是我没有负责的部分:早上的单口相声是 35 个人。我生活在恐惧中，害怕不小心点击了 Twitter 视频上的 play。这是我应该负责的部分:我从来没有得到我需要的帮助。

我花了两个星期试图做一个 docker 文件。或者可能是 Docker Hub 上的图片？或者是…好吧，我还是不知道 Docker 是如何工作的。几年后，我在 Reddit 上看到一个帖子，来自一个无论阅读多少文档都被 Docker 难倒的人。我对开发人员深表同情，也感谢那些给了我如何提高技能的有用建议的人。在*那次*几年后，我发现 Docker 上的共识是 Docker 需要一个专家来成功使用，在生产中更是如此。

在没完没了的站立会议上，我曾几次提到，我正在努力完成项目的下一步。然后，我坐在办公桌前，在 Docker 文档上戳了几分钟，从 CLI 得到一个错误，然后花了几个小时紧张地从文档点击到 Twitter 和工作列表网站。

这种情况持续了五个月。

我想在这里提一下，我不是一个坏人。

我是怎么陷入这种境地的？我怎么能让糟糕的情况持续这么久？我认为这需要一些关于冒名顶替综合症的讨论。关于冒名顶替综合症有两件事是完全不真实的:

*   感觉自己没有资格做一件事意味着你停止工作，退出，也许永远离开技术行业。
*   一旦你治愈了冒名顶替综合症，你可以做任何事情！既然你相信自己，谁还需要飞行员课程。

## [](#impostor-syndrome-means-not-asking-for-help)冒名顶替综合症的意思是不寻求帮助

当我试图保持乐观时，我总是觉得，但实际上我并没有对自己的状态做出准确的估计。更糟糕的是，我刚刚回答了错误的问题，不是“我需要做这个项目”而是“这个项目什么时候能完成”。对我来说，我最大的恐惧是承认“我自己做不到”。“从一开始，我就说服自己，其他人都有他们需要的技能，没有要求我接受培训。

这可能以几种方式结束，但肯定不会以成功的项目结束。要强调的是，在这个领域，缺乏信心是一个自我实现的预言。

## 冒名顶替综合症的意思是不做你力所能及的事情

我写了很多关于黑客马拉松的文章，但很少写那些我们苦苦挣扎的。在随机分配的团队中，当团队成员拒绝参与时，我常常会摇头。“我是一名作家而不是摄影师”，“我更像是一名后端开发人员”，这些都是团队成员给出的不做贡献的借口。虽然我总是发现黑客马拉松是一种解放(不用再担心编写糟糕的代码！在黑客马拉松上写的所有代码都是糟糕的)我在其他人身上看到了我在工作中遇到的问题:我正在建立一个我不能做什么的列表，而不是问我能做什么。

当一个团队进展顺利时，你应该尽你所能帮助他们:如果代码改进看起来很明显，问问为什么不能完成。如果我不能解决问题，我会为更有经验的开发人员写笔记。如果我不能写代码或者解释它，我会写测试。如果我做不到这一点，我会写文档。做这一切经常感觉像是一次失败，但它也让我得到了团队内部的很多感谢和赞赏。后来，当我是编写大部分代码的人时，我非常感谢编写测试覆盖和记录应用程序的团队成员。

## 英雄如何拯救了我

在我多年的开发生涯中，我仍然觉得自己不是一个“真正的”程序员。我没有一个真正的程序员的确切定义(焦虑意味着不断地改变目标)，但有一点我可以指出，我在构建和维护 web 服务时遇到了真正的麻烦。我做了这么多艰苦的工作！学会了*nix 命令，弄清楚了端口配置，所有的一切。当我想构建一个 Twitter bot 时，我会在 Amazon Web Services 上仔细配置一个虚拟机(EC2 实例)，设置一个 cron 作业来定期让 bot 检查 tweets 并采取适当的行动，并制定出必要的存储配置。一切都很好，但在几周内，系统的某个部分会崩溃，我会开始收到笔记，上面写着“嘿，你的 Twitter 机器人坏了”或“它似乎工作不正常。”

几十张复印纸和一个作为服务工具的市场领先平台在很大程度上挽救了我的自尊。

Amy Wibow 的“计算器如何计算”教授了电子学的基础知识，将半加法器构建成能够执行数学运算的电路。她的“文字推特机器人”将微控制器焊接成一个微型机器人，为推特提供物理接口。解释很清楚，不要期望任何先验知识。她的教程使用 Heroku 定期安排 Twitter 客户端发布新内容，检查回复，并跟踪其他帐户。

在 Heroku 容器(dyno)上托管我的 web 服务比我以前使用的虚拟机要容易得多。我觉得是冒名顶替综合症让我无法使用一个感觉比手动配置服务器“更容易”的工具。我最终得到的是一个比虚拟机更容易启动的东西，而且当我的一个服务突然出现流量高峰时，处理更多的流量也没有问题。在我的职业生涯中，这是我无数次了解到最简单的途径也是最实际的途径之一。

当我想轻松地从业余爱好者规模转向高流量生产时，Heroku 仍然是我的首选工具之一，承认我需要帮助来获得信心仍然是我最重要的技能。
