# 使用 Raspberry Pi 3 进行移动流媒体播放

> 原文:[https://dev . to/godsgood 33/mobile-streaming-with-a-raspberry-pi-3-3 bcj](https://dev.to/godsgood33/mobile-streaming-with-a-raspberry-pi-3-3bcj)

我有 8 个孩子(快 9 岁了)。是的，我知道是什么原因造成的(而且我很擅长)！我以前有一辆福特 E-350，上面有一个 DVD 播放器。所以当我们进行家庭旅行时，我们会带上各种各样的 DVD 在上面播放。那辆货车已经不在我们身边了，所以我想找一个不用带着一堆 DVD 的解决方案。

然后我有了一个好主意！我有一个 PlexPass 会员和几个额外的树莓派(模型 3)。后来，我在谷歌上快速搜索了一下，发现了 RaspAP-WebGUI，然后我就去参加比赛了！

经过一番惊愕之后，我确定 Plex 和 RaspAP 之间存在兼容性问题。当我想到我还能做些什么的时候，我最终把它放了起来。

几个月后，我意识到我只需要一台 DLNA 服务器。后来我又在谷歌上搜索了一下，发现了这些关于迷你 DNA 的说明

带着这些新知识，我再次出发了。

1.  安装 Raspbian
2.  根据自述文件安装 [RaspAP-WebGUI](https://github.com/billz/raspap-webgui)
3.  每个链接安装 [MiniDLNA](https://www.instructables.com/id/Raspberry-Pi-Media-Server-MiniDLNA/)
4.  优化和复制媒体
5.  安装 VLC 和享受！！

前三个是相当不言自明的，尽管有一些注释。当 Pi 启动时，MiniDLNA 服务在我的外部介质安装之前启动。这导致它找不到介质。我修改了`/etc/init.d/minidlna`,并在脚本顶部附近添加了一个`sleep 10`,以便在服务启动之前给 Pi 一些时间来装载驱动器，这样一旦它启动，就会在驱动器上找到介质。如果您有问题，您可以重启 minidlna 服务(因此，请确保您在 Pi 上设置了 SSH 服务器，并且在您的设备 ie 上安装了 SSH 客户端。JuiceSSH)。#2 由于需要权限，我在查找媒体时也遇到了问题。您可能需要调整 MiniDLNA 服务运行的权限(我将用户更改为 root，我知道这不安全，但它工作了)。

因此，在第四步中，我使用我的 Plex 服务器为我的孩子优化了一系列合法获得的电影(针对移动设备优化)，然后将它们复制到外置 USB-C 外壳中的 NVMe SSD 中。这在几个不同的方面有所帮助。1.它使 Pi 变得更容易，所以它不需要代码转换。2.它可以缩小文件，这样你就可以在上面放更多的东西。

最后一步，将您的移动设备连接到您设置的新 RaspAP 网络。然后使用 VLC(搜索你的应用商店)，你可以进入“流”并输入`upnp://http://10.3.141.1:8200`(前提是你保留了 RaspAP 的默认值。当它读取外部媒体时，你会看到几个文件经过，但你可以从屏幕右上角选择“播放列表”,然后选择外部驱动器上的任何文件。在 iPad 上，你可以选择“网络流”,它会找到 UPNP MiniDLNA 服务器，然后你就可以浏览你的列表。

在我的测试中，我能够让 6 个 Kindles、2 部手机和 1 部 iPad 同时播放流媒体！没有水滴，没有吻痕，完美的流！我确实有另一个 iPad，我试着连接，但它不可靠。所以看起来 9 是 WiFi 的上限。如果我买了一个 AC 路由器并直接将 Pi 连接到它，它可能会做得更多。
