# 有硬件组件的规格说明说它们与 Linux 兼容吗？

> 原文:[https://dev . to/francos carpa/are-there-hardware-components-of-specs-say-they-re-compatible-with-Linux-2690](https://dev.to/francoscarpa/are-there-hardware-components-whose-specs-say-they-re-compatible-with-linux-2690)

我想从头开始构建我的个人 Linux 计算机。我喜欢定制配置，并钦佩那些花时间仔细选择每个组件的人:CPU、SSD、机箱、RAM、GPU。我喜欢 Linux，因为我在大学用 Ubuntu 写 C 源代码，Ubuntu 提供了开箱即用的 GCC 编译器。编写代码非常容易，这让我想到 Linux OSs 家族首先是为开发者设计的。让我慢下来的是计算机硬件没有提供完全的 Linux 兼容性。到目前为止，我还没有找到一个组件的说明书上明确写着:“它与 Linux 兼容”。他们只说:“不保证与 Linux 兼容”。我不喜欢它。当然，网上到处都是写关于他们的硬件组件与 Linux 兼容性的评论的人，我经常读到这样的话:“我没有问题，Linux 似乎得到了完全的支持”。“貌似”。不幸的是，我是一个完美主义者，我不喜欢“似乎”这个动词，因为我们不能确定这是真的，我关心的是最佳性能。所以我在这里问你:你有没有发现一个组件的说明书上说它与 Linux 兼容？如果有，你能在评论上写下这些规格的链接吗？总的来说，你怎么看待这个问题？
