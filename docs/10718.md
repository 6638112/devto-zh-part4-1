# [](#introduction)简介

[![](../Images/b079f53e550444a082d4d096239813fe.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--K7VQaHKf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/0bs2d9anf1x7p2j4sgmt.png)

首先，你可以阅读这篇[博文](https://blog.boltops.com/2017/08/03/the-10x-1x-0x-and-x-engineer)来了解一下 0x 和-x 开发者的概念。

是的，这是一个真实的故事，从 2017 年开始，我仍然遇到这种情况。

我们回到了 2017 年，我是一名初级 web 后端开发人员。

在这个团队中，我负责基于网络的电力数据管理服务、电力数据分析服务和电力标准数据交换系统的开发。

同时，我还必须维护另一个电力投标数据系统。

那个系统太糟糕了，无法阅读这些代码，因为它是由 0x 和-x 开发人员开发的。他们不熟悉 Laravel 框架。

当我工作了大约六个月的时候，0x 开发人员离开了我们的团队并找到了另一份工作，然后-x 开发人员将在那个时候休怀孕假。

当他们离队时，对我来说是黑暗的一天。

他们为我编写了一个控制器内的 3000 行代码，其中包括数据库操作，我在这个项目中没有找到任何模型，甚至文档也不存在。

我也无法想象我要独自维护这个系统。

当我读到这些代码时，我很头疼，想叫救护车来帮我去医院。

这非常非常艰难，我想离开球队。

不幸的是，我要为这个研究所服兵役。

也就是说，我有一个三年的合同，为这个研究所工作，我不能在这个时候离开这个团队。

幸运的是，我向我们的部门经理寻求帮助，他同意了我的请求，然后我可以找到两个实习机会来帮助我完成我的项目。

# [](#problems)问题

*   很明显，当前的问题来自于 0x 和-x 开发人员。
*   我也不知道为什么研究所雇佣他们来造成我们团队的问题。
*   他们也没有好的项目经理来领导团队，所以他们开发了一个糟糕的基于网络的应用程序。(这个故事是关于项目经理的我会再写一个帖子。)
*   幸运的是，我通过两次实习重组了我的团队，并处理即将到来的项目工作。
*   今年七月，x 已经结束了怀孕假期，回到了我们的团队。我不会和她一起工作，因为这会浪费我的时间。

# [](#summary-of-problems)问题总结

*   如果可能的话，我认为你/我们应该帮助他们开发应用程序/服务，让他们至少成为 1x 开发者。
*   我在前面提到的这项工作非常困难，因为我没有太多的时间去做。我是一名全周期开发人员，不包括我的实习。

*   希望每个好的开发者不要遇到 0x 和-x 的开发者。如果你遇到他们，我也希望你有足够的时间帮助他们成为 1x 开发者:)。
