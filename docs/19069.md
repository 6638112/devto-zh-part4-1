# 为一个项目收集想法

> 原文：<https://dev.to/davidemily/gathering-ideas-for-a-project-1bn6>

# 入门

好像当我在学校的时候，我有很多项目的想法，但是当我保持(或试图保持)我的成绩时，我从来没有找到时间去做它们。我不好意思承认我想到了多少好主意，搁置了，然后完全忘记了。现在我是一名全职开发人员，回家后似乎再也不想编码了。不是我不喜欢编码，恰恰相反，但生活似乎总是碍事。我已经决定了一件事，可能有助于这只是让一个项目进行下去。目前我真的没有什么突破性的东西，但我知道我希望它涵盖一些事情。

#### 1。计划

有时，我只是在网上浏览，听到我真正感兴趣的新技术或服务，但该服务需要某种现有的基础设施。例如，我非常喜欢阅读关于 GraphQl 的文章，并想方设法了解更多。不幸的是，你不能构建一个没有任何查询的 GraphQl 服务——或者你可以，但我不打算这么做。像 gRPC 和 Redis 这样的技术听起来非常有用，但是需要一个目的。了解了所有这些之后，我决定构建一个项目，从 CRUD(创建、读取、更新、删除)web 应用程序开始，看看它会发展到什么地步。

#### 2。数据

我知道我的项目需要大量的数据。当有预先存在的信息可以提取时，我想在工作中发挥作用的许多事情会变得更好。我没有大量的数据可以提取，所以我想我可以从别人那里得到。幸运的是，我的家乡密苏里州在 data.mo.gov 收集了大量免费使用的数据。虽然我对这个项目使用的数据并不特别挑剔，但我想我会使用一些至少是个人感兴趣的东西，酒精。啊，是的，没有什么比跟踪我最喜欢的酒吧更好的了。我决定这个项目将使用“密苏里啤酒，葡萄酒和酒类批发商名单与邮政编码”数据集，正好每天更新。既然我已经决定了托管什么数据，那么是时候决定如何托管它了。

#### 3。主办；主持

这确实是托管选项的黄金时代。仅仅通过网页搜索主机，我就能找到诸如:谷歌、微软、亚马逊、数字海洋、Linode 等。对于许多学校项目，我最终使用了数字海洋，主要是因为他们有很酷的贴纸。我想既然他们已经有了我的账单信息，而且他们为我所寻找的东西提供了非常合理的价格，我会继续使用他们。那我要找什么？我认为学习经历的很大一部分是从零开始构建整个项目，因为我正在寻找学习经历(和博客项目)，我只需要一个基本的盒子。

#### 结论

既然我已经想好了这个主意，它应该只是通过这些步骤慢慢地工作。有时候，坚持下去似乎是一项艰巨的任务，但我发现最困难的部分就是开始。我很高兴生活在这样一个世界里，我可以分享我的想法和目标，你们都可以让我负责任。我计划在一些帖子中记录我的步骤，希望其他人可以从我的成长和错误中学习，这样你就可以让我负责任:)
