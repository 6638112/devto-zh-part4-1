# 为什么你花了更长的时间来学习新技能！

> 原文：<https://dev.to/stuartmuyambi/why-its-taking-you-longer-to-learn-that-new-skill-l91>

新手、初级开发人员和经验丰富的开发人员之间有一个普遍的误解，即如果他们思考足够长的时间，你最终会做出正确的决定，而这个决定是绝对错误的。问题是我想开始学习技能 **X** ，但是我不想浪费时间去学那些不会让我找到工作的东西。这就是业内人士俗称的**分析麻痹**。这些是大多数开发人员不时要处理的问题。关于这一点，我想告诉你的是*你宁愿做出错误的决定并开始行动，也不愿做出正确的决定，但却等待很长时间*。

大多数人陷入教程炼狱而没有提高的原因是因为他们试图找到正确的资源，完美的教程，完美的指南，完美的项目，这就是为什么他们从来没有提高，他们从来没有得到一份工作，他们从来没有得到一个客户，因为他们甚至没有采取这些措施。尝试了一次然后错过了它和从不尝试然后你有 100%的机会错过它是一回事。

也就是说，这同样适用于编程。如果你不在编码和编程上搞得一团糟，并且做得非常快，你将会在你现在所处的位置上停滞不前，这就是分析瘫痪的概念。

生活不是由一个决定组成的，而是你所有决定的总和。提高你的决策能力和你选择的道路的唯一方法，就是通过决策。无论你在做什么，你都想通过做得更多来做得更好。

达到**质量**的唯一途径是通过**数量**。这意味着，做很多你正在做的事情，然后开始改进它。当我面临一个问题时，比如说在编码方面，我变得更好的方法是每天投入精力和时间，大量编码并向我的 Github 提交大量数据。

生活是你所有决定的总和，而不仅仅是几个决定，提高决定质量的唯一方法是做很多决定。我通过编程提高的唯一方法就是犯大量的错误并快速失败。

生活中做决定也是如此。你宁愿很快做出错误的决定，做出很多错误的决定，然后你的决策能力会提高，你的决策质量会提高，通过你做出大量的决定，你会将你的决策能力从十年减少到一年，这样你就能像你等了 10 年一样熟练地做出决策。

忘记做出最佳决策，或者有一个最佳编码项目，或者有一个不浪费时间编码的最佳方法。你还会做什么呢，你会浪费时间去做一些愚蠢的事情。你可能会浪费时间在一个“你认为不正确”的项目上编码，而不是在几天或几周后才发现为什么不正确，但是你必须非常非常快地做出错误的决定。

你需要成为一个行动者，不再游手好闲，想着“哦，有一个做事情的最好方法”。没有最好的方法，只是尽可能先用错误的方法，这才是真正令人兴奋的，因为如果你知道结局，那就不好玩了。你想走一条路，也许是一条走得不够远的路，去发现它，弄清楚它，看看发生了什么，然后回来，从那里改进。所以，不要害怕浪费时间，只要进去开始编码，这样你就可以对自己负责，完成事情，成为所有行动者的一部分，因为这就是我想要你做的。
