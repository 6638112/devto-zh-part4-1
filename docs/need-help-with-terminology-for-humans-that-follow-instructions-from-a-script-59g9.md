# 需要关于遵循脚本指令的人类术语的帮助

> 原文：<https://dev.to/idanarye/need-help-with-terminology-for-humans-that-follow-instructions-from-a-script-59g9>

我正在尝试给我的项目命名，我为两个角色的名字挣扎了太久。这些名称很重要，因为它们将作为前缀/后缀出现在整个代码库中。

其中一个角色是连接到服务器并传输指令的脚本。另一个角色是通过客户端连接到服务器的人。客户端向人们展示脚本中的指令，他们需要遵循这些指令，并在完成后点击一个按钮进行报告。

我最初的想法是称它们为*主机*和*客户端*——但是主机也是连接到服务器的客户端，所以感觉不太对。

我也想过称他们为*剧本*和*人类*，但是“剧本”已经包含了太多的含义。与*计划*和*工人*相同。

有人有想法吗？也许这些角色的通用术语来自这种模式常见的领域？
