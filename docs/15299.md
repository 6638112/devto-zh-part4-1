# 编程是部落的，找到你的人

> 原文:[https://dev . to/Tim clicks/programming-is-tribal-find-your-people-4igf](https://dev.to/timclicks/programming-is-tribal-find-your-people-4igf)

我曾经认为决定一种编程语言是一个技术决定。不是的。这是一个社会问题。

一些社区对新思想怀有敌意。其他人如此欢迎新思想，以至于吓跑了传统主义者。有些社区是由公司赞助的，有些则是完全独立的。随着时间的推移，志同道合的人组成了社区。

一旦你开始为乐趣而编程，你应该在太投入之前花些时间考虑你想和什么样的社区交往。

我什么意思？好吧，这里有一些(可能有偏见和不公平的)例子:

**生锈**

Rust 是一项不可思议的技术。但这也是创造一个支持和包容的生态系统的社会实验。这最突出地体现在它的口号中，该口号以“赋予每个人权力”为开头。

**C++**

虽然它早在 20 世纪 90 年代初就已经存在，但 C++ events 在 2017 年采用了[行为准则](https://github.com/brycelelbach/cpp_code_of_conduct)。根据我的经验，C++程序员可能倾向于用更严肃、更工业化的语气思考行为。我不期望在 C++大会上看到独角兽表情符号。

**你从未听说过的语言**

如果你决定加入一个更小的、小众的社区，期待找到充满激情，但也相当非正统的人。根据定义，他们认为主流不是。世界上有成千上万的语言社区。

我特别喜欢一种叫[迪伦](https://opendylan.org)的语言。好像是被命运诅咒了。那里的人们才华横溢，热情好客。只是没有那么多。

## [](#what-language-to-choose)选择什么语言？

了解你的人是谁。你应该找一种语法和语义都不讨厌的语言，然后潜伏起来，直到找到适合你的社区。例如，如果您在 R 和 Python 之间选择数据科学工作，您是希望周围都是其他统计学家(选择 R)还是希望与接触到许多想法的人一起工作(选择 Python)？

在社区中要寻找的一些东西包括:

*   欢迎的大门——如果你是初级程序员，网站应该邀请新人
*   与你的价值观保持一致——一些技术社区，比如 Rust，是明确的价值驱动的。其他许多人则更加中立。
*   友好的交流方式——浏览邮件列表档案，潜伏在聊天频道中，感受人们是如何交流的。如果你不确定，创建一个假账户，问一个愚蠢的新手问题。让回答的语气引导你的决定。
