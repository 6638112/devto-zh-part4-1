## [](#babylonjs)巴比伦. js

[![babylon](../Images/96875e70db679181ef0849ed91865ac6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--xTSyMZPE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://storage.googleapis.com/replit/images/1563204593377_d164803549b1d0a3f37feb0ecf1c38fe.gif)

**@katyadee:** 我非常喜欢你的[两个巴比伦教程](https://repl.it/talk/learn/Making-3D-Babylon-Scenes-more-Dynamic-with-Actions/16132)——实际上我已经在时事通讯中介绍过它们了。

能不能给可能不熟悉的人解释一下 Babylon.js 是什么？

**@eankeen:** Babylon.js 是一个 3D JS 库。如果你在考虑一个 3D 的东西，比如任何物体，你想让它…或者让它有活力…或者让它有活力，你会用巴比伦。

**@katyadee:** 你用它做过什么项目吗？

**@eankeen:** 对，所以其中一个游戏卡壳，我用它做了[一个小游戏](https://Repl.it/talk/share/Silly-Game/13366/33047)。那是我第一次学习巴比伦的时候。

**@katyadee:** 你认为什么样的项目最适合 Babylon vs 另一个库，比如 three.js？

我想如果你想在一个项目上工作更长时间，你可能会选择 Babylon，因为 Babylon 有一个语义版本系统…但是使用 three.js，你可以在不同版本之间切换，会有很多变化，所以 Babylon 在这方面更友好，因为更新之间会有更少的东西被破坏。但是另一件要考虑的事情是…如果你想要更精益的东西，three.js 感觉更精益。
