# CSS 概念问题

> 原文：<https://dev.to/alohci/so-you-think-you-know-css-3bl2>

在我开始创建网页的时候，CSS 已经发明出来了，但是浏览器支持还处于起步阶段。因此，我在学习 HTML 的同时开始了 HTML 的样式化，主要是以一种尝试和观察的态度来对待这两者。

想加粗我的文字？嗯...`<B>...</B>`。嗯，这似乎很容易。斜体？`<I>...</I>`。检查。把我的文字变成红色？啊，那稍微有点棘手。我需要使用带有颜色属性的`<font>`,不过还好，还不错，哦，我还可以用这个标签做一些其他有趣的事情。

然后浏览器支持开始改善，我被告知应该使用 CSS。最初的反应是怨恨。为了一些新的、奇怪的语法而放弃这些简单的标签？但显然我必须这么做。

好的。让我们来看看。我们有一个样式属性，我们可以把它放在任何元素上。好的。因此..style="color:red "。也许这没那么糟。浅灰色的背景色怎么样？让我们试试 style="color:红色背景-color:浅灰色"...嗯，那没用，文字都不是红色的了。啊，它需要一个分号。try style = " color:red；背景色:浅灰色"...是的，找到了。但是背景和正文有点紧。需要扩大一点背景。啊填充！是的，那看起来更好。

就这样继续下去。几乎每个人都或多或少地通过这种方式学习 CSS。我们学习了越来越多的属性以及它们是如何相互作用和冲突的，我们学习了 CSS 规则集、选择器、级联、继承，并且更全面地掌握了盒子模型。这些都是从烹饪书上学来的，现在，如何做的问题和答案来自 Stack Overflow 这样的网站。

然而最终，我开始感到沮丧。有些事情我就是不能按照我期望的方式去做，烹饪书也帮不上忙。许多 CSS 开发人员在进行垂直对齐时会遇到这种情况，我也是，但还有其他一些奇怪的地方。于是我开始阅读 CSS 规范。几次之后，这些单词开始变得有意义，我意识到我已经错过了学习几乎所有支撑属性的 CSS 概念。

为了解释我的意思，这里有十个关于 CSS 概念的问题，而不是如何做 x。答案都可以在 CSS 2.x 规范中找到，该规范现在已经有二十多年的历史了，并拥有出色的浏览器支持。考虑到它的年龄，任何经常使用 CSS 的人都没有理由不知道答案。那么，你能回答几个？

*   画布有多大？
*   选择器的主题是什么？
*   什么是！重要的在级联中的位置？
*   初始包含块的位置和尺寸是什么？
*   支柱的垂直对齐方式是什么？
*   什么时候 clear:left 导致间隙出现，它如何影响边距？
*   当正常流程中的非替换块级元素的宽度被过度约束，并且总和不完全是其包含块的宽度的 100%时，哪个属性的已用值不同于其计算值？
*   什么时候会生成匿名的表行对象？
*   什么时候单个元素同时建立块格式上下文和行内格式上下文？
*   什么是内联元素的对齐子树？
