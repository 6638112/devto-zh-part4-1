# 在智能手机上为乐趣而编码

> 原文:[https://dev . to/gtanyware/coding-for-fun-on-a-smart phone-3j 80](https://dev.to/gtanyware/coding-for-fun-on-a-smartphone-3j80)

智能手机除了能打电话，也是一种吸收少量业余时间的设备。无论何时何地，当人们无事可做时，智能手机就会出现，然后我们会一起啄掉它们，低下头，通常会忘记我们周围发生的任何事情。

智能手机拥有它的哥哥个人电脑的所有功能，甚至更多。毕竟，很少有电脑能打电话、充当火炬或显示你在地球上几米以内的位置。智能手机的功能相当于 5 年前的笔记本电脑，所以它在处理方面毫不逊色，但它最显著的特点也是它与电脑相比的主要弱点。虽然屏幕分辨率通常比笔记本电脑好，但在没有放大镜的情况下，用它来阅读这种分辨率的文本是不可能的。

因此，在智能手机上最难做的事情之一就是编写和测试代码。这是一个遗憾，因为程序员也有时间填充；在公交车上，在等牙医的时候，或者在其他人都盯着脸书、WhatsApp 或 Instagram 的日常社交环境中。

我不是说无论我们去哪里都要带着我们的工作，或者为即将到来的面试死记硬背基本技能。尽管这很有价值，但我们都需要时不时地休息一下，对于我们中的一些人来说，仍然可以编码，只是要求不那么高的编码活动；甚至更有趣。

在这种时候，我们需要的是一个*编码场*。 [JSFiddle](https://jsfiddle.net/) 和 [CodePen](https://codepen.io/) 可能是其中最著名的，尽管前者在手机上相当笨重。然而，问题并不在于工具；这是语言的问题。要让任何有意义的东西在你的浏览器中运行，你需要修改 3 个脚本；HTML、CSS 和 JavaScript，其中最后一个可能会变得很长，特别是如果您必须保持源代码行很短，以防止它们在您的小屏幕上换行。(我排除了使用横向，因为键盘最终会占据大部分空间。)你的结果出现在第四个窗口中，所以你一直在来回切换，你必须下定决心(或者被困在一个荒岛上)才能继续实验很长时间。

在这一点上，想想世界上一些不太富裕的地区的年轻程序员，那里的智能手机数量比个人电脑多很多倍。(事实上，在所谓的‘发达西部’有许多弱势群体，情况也是如此；这不是一个局限于非洲和其他后殖民国家的问题。)有很多潜在的才能，但我们却很少注意去鼓励它们。

正是这种“搔自己痒处”的尝试，让我开始编写一种完全基于浏览器的高级语言。我想要比 JavaScript 更高级的东西，它能让我在一个脚本中编写整个网页，它的词汇和语法足够丰富，不仅能处理代码，还能处理所有的标记和样式，对我来说，“更高级”意味着“更接近英语”。我自己的母语是这方面的黄金标准；我把它视为人类交流的巅峰，有一个使命，就是训练计算机至少听懂一些人和机器能相遇的方言。

当然，当你写你自己的编程语言时，没有 IDE 能理解它，所以你必须放弃颜色编码编辑器、自动完成等等的细节，除非你自己能做到。这开始看起来像一个交易破坏者，所以我去打猎，很快发现了 [CodeMirror](https://codemirror.net/) 。经过一番挠头之后，我想出了如何定制它来对我的脚本进行颜色编码，我又回到了正轨。

因此，我现在开始用一种表达性语言为浏览器编写代码，这种语言不需要外部编译器或构建工具，可以做从基本算术到图形动画的一切事情。不言而喻，我也希望能够在业余时间玩这个，编写简单的游戏和模拟，或者只是尝试在其他地方有用的想法。

围绕这一点，产生了构建一个结合了教程和参考手册的脚本平台的想法，自然它也必须在移动浏览器上运行。同样自然的是，它必须全部用相同的脚本语言编写；如果它不能处理 ***那*** 那么它会是一种多么可悲的语言借口呢？

真的没花多长时间。也许这只是我的大脑的方式，但我发现当我可以把我脑海中的想法(我想用英语，而不是 JavaScript)用最少的翻译输入时，我的效率最高。对我来说，这是完成事情最快的方法。脚本语言允许在多个脚本中分担工作，每个脚本做自己的工作。通常每个都处理不同的屏幕面板，所以主脚本所要做的就是在需要时调用它们。事实上，编写描述该语言每个特性的定制[摊牌](https://github.com/showdownjs/showdown)参考页面比编写驱动 IDE 的代码花费的时间还要长。

该项目仍在进行中，但现在我有足够的信心将其提供给其他人，希望反馈将为未来的增强指明方向。也许有些人会像我一样，喜欢使用一个移动友好的脚本平台，在脸书开始变得索然无味之后，好好享受那些乏味的时光。如果你有，让我知道你的想法；你喜欢什么(如果有的话)和不喜欢什么，以及接下来应该做什么。
