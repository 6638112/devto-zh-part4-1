## [](#functions-are-like-plays-in-football)功能就像足球比赛

我很少用体育类比来解释 web 开发的概念，但是*这个*才是真正的 MVP。😜

足球比赛是每次都以基本相同的方式进行的一系列动作。最著名的戏剧之一是“万福玛利亚”我不太喜欢足球，但是，就我对比赛的理解，所有的接球手都尽可能快地跑向终点区，四分卫尽可能远地向其中一个接球手投球。

《万福玛利亚》是一部简单的戏剧，但戏剧可以复杂得多。如果你有一场复杂的比赛，在比赛过程中有多个步骤和决定要做，如果教练和球员有一个简单的方法来引用它，就会容易得多。这使得一遍又一遍地调用它变得容易，而不必每次都列出所有的 15 个步骤。

Javascript 中的函数允许您命名 Javascript 代码中的几个步骤。您可以使用这个名称来告诉人们这些步骤要做什么。它还能让你**轻松地一遍又一遍地执行同样的步骤**。(是的，Javascript 甚至使用了同一个词——“调用”——来谈论在一个函数中运行代码，这个函数是 sports 在谈论重复一场比赛时使用的。)

## [](#updating-plays)更新播放

想象一下，一名足球教练发现球队的一种打法在面对即将到来的对手时不太奏效，并希望修改这种打法。现在，他们可以在每次想要运行该游戏并解释所有新步骤时，要求暂停，而不是调用旧游戏。这可能行得通，但是会很麻烦。

相反，他们可以告诉所有玩家关于游戏的修改，并在游戏开始前的练习中更新剧本。这将允许他们**用相同的名字调用相同的游戏，但是得到新的游戏执行**。

Javascript 函数也有类似的优势。如果您需要更改现有函数中的步骤，您可以在一个地方更改它们**——在函数内部——并且函数的每个其他调用都将使用新的步骤。**

## [](#sharing-plays)分享玩法

教练患了流感，将错过下一场比赛。我们如何让助理教练使用我们现有的战术？我们可以将剧本传递下去，也可以将单个剧本拿出来分享。因为它们被写下来并命名，**它们很容易在**之间传递。

Javascript 允许你以同样的方式传递函数。有时在 Javascript 中，你会触发一些异步代码——这些代码需要一段时间才能完成，并且会在后台继续运行，而你的应用程序的其余部分会继续运行。通常需要**对异步操作的结果**做一些事情。

一种方法是将函数传递到异步函数调用中。当异步操作完成后，**它会调用你传递给它的函数**。这有点像分享一个剧本，让其他人以后尝试。在其他情况下，您会使用这样的函数(通常称为“回调函数”)，但这是一个您会经常使用的示例。
