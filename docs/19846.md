# 为什么我学围棋，为什么我从不后悔

> 原文:[https://dev . to/wlun 001/why-I-learn-go-and-why-I-never-retail-5a1a](https://dev.to/wlun001/why-i-learn-go-and-why-i-never-regret-5a1a)

# [](#background)背景

几个月前，当我开始设计学士学位的最后一年项目时，我需要一种能够很好地处理并发性，同时具有现代语法的编程语言。我研究了多种编程语言，而 [Go](https://golang.org/) 从我的意见列表中脱颖而出。

# [](#learning-process)学习过程

### [](#pick-up-go)收拾走了

当我决定学习围棋时，我做的第一件事就是寻找示例项目并阅读源代码。事实证明，这是一个糟糕的举动。由于 Go 的语法和结构。所以我后退了一步，浏览了一下[围棋之旅](https://tour.golang.org)的每张幻灯片。令人惊讶的是，我花了一个晚上来完成它，之后我就能够阅读 Go 代码了。我花了大部分时间的部分是关于并发性和通道的主题。我用了老式的方法，在纸上画出来，这对我很有帮助。

### [](#think-like-go)思如围棋

我面临的挑战之一是设计范式。来自面向对象和函数式编程的背景，Go 一开始对我来说没有任何意义。想象一下，如果没有许多语言都有的像`map`、`reduce`这样强大的函数。基本上，在围棋中我们必须自己编写循环。想象一下没有`try`和`catch`处理错误。我花了一段时间来适应它。

### [](#read-more-code)阅读更多代码

由于我已经掌握了基础知识，我通过阅读更多的 Go 代码继续我的学习之旅。

> 我读得越多，就越意识到自己对围棋的不了解。

开源肯定有帮助。我看了很多开源代码。

# [](#why-i-never-regret)为什么我从不后悔

在 Go 工作几个月后，确实给我带来了好处

### [](#open-up-new-opportunity)开辟新的机会

当我们学习或了解更多的东西时，我们的价值就会增加，尤其是在事物变化如此之快的技术行业。Go 可能不是最流行的语言，但它正在增长。

### [](#think-from-another-perspective)换个角度思考

正如我前面提到的，在 Go 中我们没有大多数编程语言中的强大功能，它允许我们从最基本的编程结构来考虑解决方案。对一些人来说可能不舒服，但是它帮助我从另一个角度思考问题。

### [](#performance-is-awesome)表现令人敬畏

围棋的表现太牛逼了。我没有运行任何基准来做比较，对我来说，我认为是相当不错的。编译时间真的很快，我的应用程序的 CPU 使用率也很低。

# [](#conclusion)结论

Go 是一种很棒的编程语言，我鼓励你们都去尝试一下。我将使用不同的应用程序继续我的 Go 之旅。
