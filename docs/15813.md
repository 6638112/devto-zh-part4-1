# 编写课程的新老兵！

> 原文:[https://dev.to/vetswhocode/new-vets-who-code-curriculum-nd8](https://dev.to/vetswhocode/new-vets-who-code-curriculum-nd8)

终于兴奋地完成了#VetsWhoCode 更新课程！我围绕五件事设计了这个课程:

技术的可访问性
易于采用
市场趋势
我们的生产堆栈
无服务器网络实践

我花了大约 18 个月的时间学习、犯错和采访，把这些和对 edge 的关注放在一起，仍然使用稳定的技术，并重点关注 API。整个过程对我来说不仅是新的，而且是不同的。我关注的不仅是我们教授的内容，还有如何教授。最重要的是，这将是仅由# VetsWhoCode 校友教授的第一堂课。这意味着老师已经去过你去过的地方，并且在你想去的地方。也就是说，公司付钱让他们每天写代码。

这和我们早年的情况完全 180 度大转弯。在我们早期的几年里，我们玩防御，最初像许多营利性代码学校一样思考，但在五年帮助退伍军人获得 jobs plus 后，作为行业中唯一的创始人，在最大的媒体公司之一工作，我很荣幸一直在工程师身边，并能够采访他们对行业发展方向的看法。

我们还注重与其他教育工作者/开发人员建立关系，如 Zed A Shaw、Kyle Shevlin 和 Brian Holt，听取他们的意见，然后将他们加入我们的董事会。我们还增加了来自谷歌的云技术专家，并从脸书招聘人员来帮助我们更好地构建流程。这一过程包括与前端大师建立关系，将他们作为一种资源加以利用，因为我们的目标不是成为退伍军人的代码学校，而是成为技术教育的首要场所，从高级前端架构的角度深入关注和专业化建设，恰好专门面向退伍军人。

除此之外，随着 Node/ Express 的兴起，我们能够消除大多数代码学校 IMHO 的头号弱点，即上下文切换，他们现在必须推动两种语言和另一个堆栈，但因为所有的填鸭式教学本质上都是高水平的，利用框架来完成繁重的工作，而不是能够深入基础学习，如架构、工作流、函数式编程与面向对象编程以及可访问性。

不过，对我们来说，决定性的技术时刻来自于一年多前我们对无服务器 web 和 JAMstack a 的早期采用。作为我们用来建设 VWC 的工具，看到其他公司在电子商务、博客、政府和教育领域推动它的水平，我们很快意识到，我们可以教我们的部队这个框架，让他们做一些其他项目没有做的事情:允许他们的学生推出他们的生产代码，从事具有商业意义的事情，所以你没有投资组合，但你也有人们正在使用的应用程序的生产门票，可以在商业方面发言。这使得我们可以超越其他地方，让他们解决问题，并向在工作环境中工作的开发人员学习。

然而，这其中的秘方将永远是我们的退伍军人。即将到来的这一批将有# VetsWhoCode 校友导师，他们从编码的 Vets 那里获得了支付账单的技能。所以他们知道在军队里是什么感觉，过渡出来，被告知“谢谢你的服务”，但没有得到工作，如何训练，如何准备和如何得到工作。总之，他们知道如何完成任务。

这样做意味着要做出一些牺牲。我不得不增加一周的教育，但我宁愿你学习这些东西，并准备做真正的工作。这真的是一项爱的工作。点击和`git commit`查看课程[，否则就不会发生。](http://bit.ly/vwc-github-curriculum)
