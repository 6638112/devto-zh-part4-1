随着我成为一名成熟的开发人员，我自己也符合其中的一些模式，我意识到真正的 10 倍开发人员并不是那里描述的那些人。是的，漫画家或开发人员可能比团队中的其他人产生更多的产出，但是，最多，他们通常对团队有轻微的影响，而且，更多的时候，我看到他们拖垮了团队。通过将自己与团队的其他成员隔离开来，他们负面地影响了团队动态，并且经常产生除了他们自己之外任何人都无法维护的代码——但是他们不想维护它，因为那没有“真正的编码”有趣

如我所说，我是那个开发者。幸运的是，我长大了。我已经意识到，唯一真正的 10 倍开发人员是使整个团队生产率提高 10 倍的人——指导他人，分享好的想法，接受反馈，在他们维护代码时清理代码，编写好的文档，并使组织成为人们想要工作的那种社会环境。大多数时候，软件开发不是单人游戏，而是一项团队运动。

*(编辑——在我还没打字完就意外贴上来——这就是我单手在手机上打字的下场。额外的部分从这里开始。)*

我发现有一个军事术语适用于这类开发者- [力量倍增器](https://en.wikipedia.org/wiki/Force_multiplication)。简而言之，力量倍增器是使一支部队比其他情况下更有效的一些因素——例如，拥有卫星导航可以使一支小部队像没有卫星导航的 5 倍大的部队一样有效。

伟大的开发人员是团队的力量倍增器。他们不会通过自己做更多的事情来提高生产率，而是帮助其他人提高生产率。

[![theringleman profile image](img/04c3ed64fa267114c021e844e1c585d3.png) ](/theringleman) [ Sam Ringleman ](/theringleman) • [<time datetime="2019-07-15T04:30:42Z" class="date-short-year"> Jul 15 '19 </time>](https://dev.to/theringleman/comment/d2o4) 

我真的不喜欢这个视角。在我看来，蓝领工作在“白领”中有负面的含义。

我一生都在做蓝领工人、铁路工人、制造业工人和建筑工人，我很幸运地过着两种生活。我会告诉你，不，编码不是蓝领工作。然而，编码以另一种方式令人疲惫不堪。

仅仅因为代码的世界对“普通人”开放，并不意味着它就是蓝领。我没有以体力的方式拼命工作。我的工资很高，还有在办公室工作的好处。我为非常大的公司解决复杂的问题和建立惊人的系统。你猜怎么着，我没有为此上过大学。然而，我正在努力工作。

以我过去的经验，我会说我在蓝领工作中遇到过一些最聪明的人。一些会让我们 10 倍的工程师自惭形秽。请注意 10 倍工程师这个术语中的严重讽刺。

我认为，作为白领工作者，我们需要认识到，如果没有蓝领兄弟姐妹，我们就不会有今天，反之亦然。是的，我们可能有不同的工作条件，但我要说，我作为一个蓝领的生活比作为一个白领要艰难得多。不，编码不是也永远不会被认为是蓝领工作，因为它对“普通男人或女人”开放。就我而言。

> JavaScript 从右向左进行操作。

在某些情况下是这样的，但不是在这种情况下。您之所以会看到看似从右向左求值的行为，是因为[运算符优先级](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Operator_Precedence)，其中一元运算符`!`的优先级高于二元运算符`+`。

`+`实际上是从左到右计算的，所以用括号括起来，看起来像这样:

```
(((!(+[])) + []) + (![])).length 
```

逐步评估这个带括号的表格，我们得到这样的结果:

```
/*  1\. */ (((!(+[])) + []) + (![])).length  // parethesized form.
/*  2\. */ (((!0) + []) + (![])).length      // `+[]` is evaluated to `0`.
/*  3\. */ (((!false) + []) + (![])).length  // `0` is coerced to the boolean `false`.
/*  4\. */ ((true + []) + (![])).length      // `!false` is evaluated to `true`.
/*  5\. */ (("true" + "") + (![])).length    // `[]` is coerced as a primitive to the string `""`, which means `true` is also coerced to a string.
/*  6\. */ ("true" + (![])).length           // `"true" + ""` is string concatenation, which evaluates to `"true"`.
/*  7\. */ ("true" + (!true)).length         // `[]` is coerced to the boolean `true`.
/*  8\. */ ("true" + false).length           // `!true` evaluates to `false`.
/*  9\. */ ("true" + "false").length         // `false` is coerced to a string because `"true"` is a string.
/* 10\. */ "truefalse".length                // `"true" + "false"` is evaluated as string concatenation to `"truefalse".
/* 11\. */ 9                                 // the length of `"truefalse"` is `9`. 
```
