## [](#supercharge-your-editing-experience)增强您的编辑体验🎉

Visual Studio Code 是微软的一款免费、跨平台、开源的代码编辑器。你可能也听说过它叫做 VS 代码，或者有时候，只是代码。

这是我每天使用和喜欢的编辑器。您可能已经知道这是 Python 开发人员的一个很好的选择——它支持林挺、调试、人工智能辅助的代码自动完成等功能。

只需一点点配置代码也可以做得更多！

在下面的代码中了解三个令人兴奋的特性。

## [](#1-extensions)1:扩展🔗

代码最初是一个轻量级编辑器，可以通过扩展进行扩展。这允许你只安装对你有用的工具和插件。

当您第一次下载和安装代码时，您将拥有一个非常简单的编辑器。它附带了对 JavaScript、TypeScript、CSS 和 HTML 等语言的简单支持，但对其他语言的支持可以作为扩展下载。

扩展生态系统不仅仅针对语言。您还可以找到有用的插件、轻量级工具、主题、定制、使用其他编辑器的键盘快捷键的键映射等等。

我最喜欢的一些扩展是:

*   [Python 扩展](https://marketplace.visualstudio.com/items?itemName=ms-python.python&WT.mc_id=pybay-blog-ninaz)
    *   Python 语言的丰富特性支持。令人敬畏的功能，如[人工智能智能感知](https://marketplace.visualstudio.com/items?itemName=VisualStudioExptTeam.vscodeintellicode&WT.mc_id=pybay-blog-ninaz)，Jupyter 笔记本支持，测试浏览器，等等！
*   [autoDocstring](https://marketplace.visualstudio.com/items?itemName=njpwerner.autodocstring&WT.mc_id=pybay-blog-ninaz)
    *   为 Python 函数快速生成文档字符串。
*   [GitLens](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens&WT.mc_id=pybay-blog-ninaz)
    *   使用一目了然的 git 责备注释、git 存储库导航、强大的比较命令等等来增强 git by。
*   [夜猫子主题](https://marketplace.visualstudio.com/items?itemName=sdras.night-owl&WT.mc_id=pybay-blog-ninaz)
    *   一个为夜猫子设计的 Visual Studio 代码主题。针对光线较暗的情况进行了微调，适合那些喜欢深夜编码的人。
*   [孔雀](https://marketplace.visualstudio.com/items?itemName=johnpapa.vscode-peacock&WT.mc_id=pybay-blog-ninaz)
    *   允许您更改每个新窗口的颜色，在处理多个项目时可以快速直观地识别。
*   [待办事项高亮显示](https://marketplace.visualstudio.com/items?itemName=wayou.vscode-todo-highlight&WT.mc_id=pybay-blog-ninaz)
    *   真正让你的`TODO` s 和`FIXME` s 流行起来！

## [](#3-live-share)3:直播分享👥

我在波特兰或者。作为远程开发人员，能够与其他人协作是重中之重。

长期以来，远程软件合作一直是一个难题。令人欣慰的是，Visual Studio 代码有一个插件，使这个过程变得简单而轻松。

在此之前，您可能会被诸如 tmux 之类的工具和基于文本的编辑器所困扰，或者处理不稳定的网络连接、模糊的屏幕和视频通话中的音频衰减。

[Live Share 扩展](https://marketplace.visualstudio.com/items?itemName=MS-vsliveshare.vsliveshare-pack&WT.mc_id=pybay-blog-ninaz)极大地改善了远程协作的过程。

首先，你和你想合作的开发者都需要安装代码，然后[下载 Live Share 扩展包](https://marketplace.visualstudio.com/items?itemName=MS-vsliveshare.vsliveshare-pack&WT.mc_id=pybay-blog-ninaz)。现在，你们中的一个人需要 [*开始*一个共享会话](https://marketplace.visualstudio.com/items?itemName=ms-vsliveshare.vsliveshare&WT.mc_id=pybay-blog-ninaz#quickstart-sharing)，另一个人需要 [*加入*会话](https://marketplace.visualstudio.com/items?itemName=ms-vsliveshare.vsliveshare&WT.mc_id=pybay-blog-ninaz#quickstart-joining)。

[![](../Images/92a267c5670e50ab22552440ec04107b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--GLSGN0TV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://d33wubrfki0l68.cloudfront.net/81ae9eee56ac3a60826ce02890b13ef5437b3e21/1642a/2019/vs-code-ls-session.png)

一旦你这样做了，你就可以很容易地合作。您的团队成员将能够在他们的编辑器中看到您的代码，而不必克隆一个 repo，甚至从您的代码库中安装依赖项。

您将看到其他人的光标，但您也可以在自己的代码实例中独立导航。这意味着您可以使用您的主题、键盘快捷键、扩展和自定义设置。作为奖励，你甚至可以和你的编码伙伴通过音频聊天。

这个特性对于代码审查特别有用。比方说，你想谈谈你对某人的拉动请求所做的一些评论。只需启动一个实时共享会话，检查 PR 分支(使用 [GitHub Pull Requests](https://marketplace.visualstudio.com/items?itemName=GitHub.vscode-pull-request-github&WT.mc_id=pybay-blog-ninaz) 扩展使其更加简单)，就这样。有时候，沟通不畅比无休止的来回评论更容易解决。

这确实是最好的合作方案。
