# 学习 Webdev:研究要学习的技术

> 原文：<https://dev.to/bbarbour/learn-webdev-researching-techs-to-learn-5ggn>

想象一下，在你的家乡有一个巨大的洞，如此之深，看起来就像一个无底的深渊。你的任务是探索这个洞，看看它通向哪里，里面有什么。现在你必须想出如何实现它。

除了漆黑一片，你什么也看不见。

捡起一块石头并把它扔进洞里可能是一个好的开始。你可以听听，看看是否有飞溅声，或者需要多长时间才能当啷落地。

你在学习代码时也做了类似的事情。对你即将开始的探险有一个初步的概念。如果你心中没有目的地，你就无法到达某个地方。没有一个坚定的计划，你无法探索未知的黑暗，也无法在知识的洞穴中遨游。计划让你脚踏实地，最重要的是让你专注。

设定切实可行的目标并执行它们。这是一项自我激励的任务，需要投入大量的时间。当然，我可以给你一长串你应该掌握的语言和框架。但是，我不认为那会帮助你。最重要的部分是跟随你自己的兴趣并理解你的动机。我的个人列表可能不适合你。此外，随着行业的变化，这份名单很有可能很快变得无关紧要。

我们在 2019 年建立网站的方式与 2014 年不一样。它们在 2014 年的建造方式与 2010 年的建造方式不同。

从现在开始的好处是每隔几年事情就会发生变化。新的框架，新的做事方式，一直在向我们施加压力。你正在步入的洞穴不断变化，并且总是朝着不可预见的方向开辟新的道路。技术不稳定，可塑性强，容易受到周围行业的影响。

在你最喜欢的搜索引擎中输入“2019 年学习 Web 开发”(或者无论是哪一年)，你会看到大量关于这个主题的视频和文章。我鼓励这种类型的探索。事实上，如果你正在读这篇文章，你可能在某种程度上已经做到了。你甚至可能对学习什么和使用什么资源有一个模糊的计划。

那个计划，如果没有写下来，就马上做。列一个清单，一张便条，或者一个电子表格。无论什么对你的大脑有效，你都可以用来追踪你的进步。这将是一次由结果驱动的探险，而不是由异想天开的好奇心驱动的。你需要目标，它们是前进道路上的垫脚石。

手里拿着你的清单，打开一个求职网站——LinkedIn，Careerbuilder，事实上。基本上，任何资源，你可以找到你所在地区或偏远地区的帖子。你要花很长时间研究这些招聘信息。

将你的个人列表与网站开发人员的工作要求进行比较。请记住，它们是以关键词为中心的(毕竟它们是广告)。还有，它们可能不是开发者写的。没关系。你要找的东西仍然会在那里，在某个地方，稍微筛选一下。

使用不同的职称，如:Web 开发人员、前端开发人员、全栈开发人员——只是为了了解一系列的角色和职责。在这些广告中，如果你看到一项你的清单上没有的技术，请从你自己的清单中单独记下。

在你这样做了几个小时后，你可能会被一些你从未听说过的随机术语和软件弄得不知所措。如果你刚刚开始，这听起来会像胡言乱语。这很正常。我自己一开始也有这种感觉。我记得第一次看到这个名字时，我不知道它是什么。

我鼓励你接下来做研究。查找这些技术，找出它们的位置和作用。确定他们的先决条件。接下来要解决细节问题，这样你就有了一套基本的技术，你必须了解这些技术，以便将来成为一名新员工时，这些技术是相关的和受欢迎的。

注意:不要删除你感兴趣或感兴趣的技术，即使你没有在招聘信息中看到它们。给它们加粗，给它们加下划线，只要对你有用。这些将是你的基石，基于奖励的自我学习的命脉。通过你的研磨把它们撒进去。最后，你想让编码变得有趣。要获得乐趣，好奇心必须无拘无束。否则，你会感到厌倦，停滞不前，并最终退出。

学习新技术时，灵活性和分析能力非常重要。早期确定方向对站在稳定的基础上是很重要的。

知道自己的道路会给你信心，让你勇往直前，即使你心存疑虑。

### 行动事项

*   寻找推荐的网络开发学习途径。
*   把你想学的东西写出来。
*   研究相关的招聘信息，将你没有的通用技术添加到列表中。弄清楚先决条件。
*   让你兴奋的马克科技。把这些作为奖励自己和减轻压力的一种方式。
