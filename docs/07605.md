## [](#the-beginning-of-everything)万物之始

我叫达米尔，土生土长的克罗地亚人，目前住在萨格勒布。我一直与政治保持距离，从未对国家象征、政党或任何爱国主义的东西产生任何形式的依恋。总的来说，我一直忠于计算机和技术。最开始，也是我第一次面对当时被认为是工程学巅峰的东西是这样的:

[![zx](../Images/631a498689e6931db366b0d9757f25f4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--zdUjpXwg--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://upload.wikimedia.org/wikipedia/commons/thumb/3/33/ZXSpectrum48k.jpg/800px-ZXSpectrum48k.jpg)

我对这个设备很着迷，但当时我太小了，除了玩游戏和敬畏地看着那些磁带在我的小电视机上神奇地转换成移动图像之外，我什么都不会做。对于年轻一代，你可以在[维基百科](https://en.wikipedia.org/wiki/ZX_Spectrum)上找到更多关于 **ZX 光谱**的信息。

随着时间的推移，我的好奇心越来越强，我慢慢地开始学习编程语言，这样我就可以最终解决驻留在我的模拟磁带中的想象中的动画角色的日益增长的神秘，当在我的随身听中播放这些磁带时，这些磁带发出的令人难以忍受的刺耳噪音不知何故被赋予了生命。从那一刻起，技术和软件完全占据了我的大脑，并一直困扰着它。

我有史以来第二棒的设备是 Atari ST T1，当我打开它，不得不一直更换 3.5 英寸软盘时，我的感觉无法用语言来形容。哦，是的，没有更多的磁带，我跳过了 5.25“时代，直接到 3.5“魔术。这仍然主要是关于游戏，但与我这一代玩**射击游戏**类型游戏的大多数人不同，我总是不知何故地粘在屏幕上玩冒险游戏，像**猴岛的秘密**，这可能是(现在仍然是)我最喜欢的游戏之一。

## [](#the-progression-of-curiosity)好奇心的增长

在雅达利 ST 时代之后，一切都开始加速，我在那里，从 **286** 到**奔腾**和 **Windows 95** 。当时我已经忙于编程，我选择的语言是 [Pascal](https://en.wikipedia.org/wiki/Pascal_(programming_language)) 。我花了相当多的时间做各种有用和无用的编码来积累经验，当我找到第三份工作时，我部门一半的人都在运行 Linux，大部分是 Fedora。因为这项工作是关于运行在 Linux 服务器上的 PHP 的，所以试着把它作为一个客户端操作系统似乎是合乎逻辑的。这是我做过的最好的决定，也有点幸运的是，我有一个非常好的、知识渊博的导师，他从不缺少建议，并在路上的每一步都指引我正确的方向。说实话，我一直使用 Linux 直到今天，我只选择了 Gentoo 作为我的发行版。在那些年里，我尝试过许多不同的语言、脚本和框架，就我所能记得的，下面是清单:

*   帕
*   德尔斐
*   服务器端编程语言（Professional Hypertext Preprocessor 的缩写）
*   动态服务器页面
*   斯卡拉
*   Java 语言(一种计算机语言，尤用于创建网站)
*   C++
*   C
*   Nvidia CUDA
*   人寿保险商协会(Life Underwriters Association)
*   eBPF(有限制的 C)
*   Java Script 语言
*   动作脚本
*   HTML/CSS(无脚本)
*   React.js
*   节点. js
*   引导程序
*   SCSS
*   XML/XSLT

## [](#the-dystopian-present)反乌托邦的呈现

从所有这些不同的技术中，我发现常规的 C 是我最喜欢的。它定义良好，简单快速，因为我用它进行系统和套接字编程，所以它非常适合。我也发现了自己的定位；似乎创建解析器( **Bison/Flex** )和实现协议解析器( **Wireshark** )是我隐藏的天赋。当我沉浸在这类项目中时，我感到一种奇怪的吸引力和满足感。

如今的趋势是不断增加更多更高的层，以使编程语言更简单，并使学习曲线不那么陡峭。我更喜欢低级语言，我并不想贬低这种新趋势，一点也不想。我只会打自己耳光，因为我的大部分工作也是在与 **React.js** 、 **Node.js** 和 **Android (Java)** 一起做项目。

## [](#the-not-so-social-network)不那么社交的网络

我一直存在的一个问题是维持私人关系和自尊问题。在过去的 8 年左右的时间里，我决定尝试建立一家自己的公司，如果你愿意，可以创业，看看会发生什么。我预计这将是大量的工作，长时间和与市场上其他竞争对手的持久战斗(电信相关业务)。长话短说，由于一些错误的决定和我信任的人之间不稳定的关系，一切都以一场灾难告终。

我在这里想说的是，激发最初想法并通常维持相当长时间的热情只能到此为止。一旦你的热情开始下降，感觉就像你有戒断症状，因为你服用了一些想象中的高科技药物，这些药物让你坚持了这么多年。只有那时，你的视野才变得明朗，面具才会摘下。不幸的是，这个时刻也预示着即将到来的失控螺旋，这可能已经过去了，只是潜伏着，等待着让人们知道它的存在。

在我看来，创业公司的唯一支柱是人，而不是花哨的技术、产品、编程语言、网站等。我现在知道了，我也开始理解一些术语，如**非发明综合症、**独角兽**开发者、**团队合作**以及一些新的术语，如**10x 工程师**。回想起来，我似乎做了一切与我应该做的相反的事情，基本上是在与我自己和我的幸福作对，并且完全不顾后果。**

伤害已经造成，从那以后我的生活一直处于混乱之中。也许我只是一个性格软弱、反应过度敏感的人，这很难说。我慢慢接受了把我的状态归类为**精疲力竭**可能是相当准确的。

## [](#the-doubleedged-sword-aka-wfh)双刃剑，又名 WFH

> WFH 是在家工作的首字母缩略词(完全是你真正发明的)

软件开发人员和 IT 行业的人似乎对 WFH 很着迷，这是一种因通信技术的进步而成为可能的现代现象。前几天我读过一篇文章，文章指出，一个人即使只经历了很短时间的远程工作，也会带着轻蔑的眼光回顾常规的办公室工作。

我不同意这种说法，不是说它在统计上不准确，而是试图指出它不健康。人类是群居动物，并不以二元 **0 || 1** 方式工作；我们都应该遵守我们的生活如此渴望的内在平衡，0 和 1 之间的灰色区域。

## [](#the-physical-part)物理部分

从某种意义上说，我已经不再年轻了，时间确实是相对的，因为它似乎在加速🤔。我快 38 岁了，可能永远不会下 ***编码列车*** 。

在编码界有一个明显的趋势，那就是努力争取管理职位；一些人认为这是一个加薪的机会，而另一些人只是意识到编码不适合他们，并且看不到自己在未来几年从事这种工作。

年龄根本不是问题，但有些不可避免的变化是每个人都必须面对的。

*   你需要更多的休息
*   锻炼在你的待办事项中应该排在更靠前的位置
*   当年轻的开发人员不同意你的观点，只是因为他们沉迷于一些最新的框架时，不要太认真，让他们玩他们的玩具

锻炼是至关重要的，无论是慢跑、散步、跑步、去健身房还是任何能让你离开椅子的运动。就我个人而言，我选择了瑜伽(身体出汗的部分)。这是我和我的小助手，努力保持健康:

[![img](../Images/58913f712771ec7367539222b43cb6f6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--5UgISnJ7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://pbs.twimg.com/media/EBelOtXXkAA84ou%3Fformat%3Djpg%26name%3Dmedium)

每个人都试图尽可能地跟上 IT 领域的最新发展，但由于这些变化的频率呈指数增长，你需要一个超人的能力才能跟上。

如今，一切都与 Web 和 Javascript 有关，这在很大程度上是人们对编程的看法。年轻的开发人员学习与 web 相关的技术，作为他们在编程世界中的起点，这与大多数老开发人员的道路有很大的不同，老开发人员通常包括 Pascal、C、COBOL 甚至 ASM。

我想通过推荐老开发者坚持他们所知道的，甚至尝试去完善它来结束这一部分。我也在做同样的事情，尽管公司只雇佣精通 web 的开发人员这种先入为主的想法有些道理，但对于使用 C 和 VHDL 等低级语言的开发人员来说，还有很多工作要做。

也许我是幸运的，但是每当我透露我对 C 和低级编程的热爱时，不知何故我得到了一份工作。哦，还有一件事。我之前说要坚持你所知道的，并不是说要做到专一。花些时间学习新的网络技术是个好主意，只是为了保持在圈子里，而不是成为专家或什么的。虽然，也许有 40 岁的开发人员成为前沿技术专家的例子；也许他们需要一个挑战，一个改变，谁知道呢，这不是闻所未闻的。

经验永远是有价值的，每个公司都需要年轻的经验不足的开发人员，他们不会因为长时间工作而感到疲劳，也需要年长的经验丰富的开发人员，不管喜欢与否，他们都有一些年龄限制。

# [](#the-conclusion)结论

在过去的 8 年里，我一直在家工作，起初我认为这很棒，但我最终产生了社交焦虑和无数其他问题。在我看来，有一个工作的地方，周围都是真实的人，而不是只使用 Slack 或其他聊天工具是至关重要的。我目前失业，仍然在家不穿裤子工作，证实了这种刻板印象。

经过仔细考虑和专业人士的建议，我的结论是:对于软件开发人员或任何大部分时间都呆在屏幕前的人来说，最好的设置是平衡办公室和 WFH，并争取达到 **50/50** 的比例。这可能不是对每个人都有效，但这是值得考虑的事情。

我的故事并不独特。我相信有很多类似的，引发这篇文章的原因之一是我想分享我的经历，并向处于类似位置的人传达我的想法。作为一个内向的人，写作对我来说比口头交流更自然。我目前正在进行的探索是自我反省和自我批评；我正试图走出自己的舒适区，接受一份迫使我与真实的人交往的工作，并在一间真实的办公室里呆上一段时间。我也可以选择在家工作，或者按照自己的喜好安排工作场所。尽管已经很难了，但我不会走上 100% WFH 的道路，所以我会试着从 3 天在办公室，2 天在家开始，但老实说，我不知道会发生什么。

感谢您阅读这篇文章。写作也是一种缓解压力的活动；我现在 30 多岁才发现这一点，我非常感激地意识到，还有一种活动可以有效地刺激我的大脑，用多巴胺奖励我，从而产生成就感。
