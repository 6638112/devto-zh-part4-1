# 你会报告你在其他网站上遇到的错误吗？

> 原文:[https://dev . to/desi/do-you-report-bugs-you-run-into-on-other-websites-5d gf](https://dev.to/desi/do-you-report-bugs-you-run-into-on-other-websites-5dgf)

也许这只是我和我的 Bug Queen 声誉，但如果我在网站上看到一些不能正常工作的东西——无论是某人投资组合的设计问题还是某项服务的功能错误，我都很难忽视。

我开始怀疑这是我的肛门天性，还是其他人也这样。

一般来说，如果我在某人的个人网站上看到一个问题，我会试着在 Twitter 上提醒他们。但是如果他们的 DMs 关闭了，我倾向于就此打住。有很多网站也很难报道——当我为这篇文章寻找图片时，我遇到了一个不合适的问题！😂他们有一个方便的表格，当他们遇到已知的中断时会显示出来，而且还有一篇帮助文章，详细说明如何报告问题或错误。

Basecamp 有很好的 Twitter 支持，而且他们通过 DM 有很强的响应能力。

当然，还有像 DEV.to 这样的开源项目！让在 Github 中创建问题变得非常容易。

对于没那么容易的网站，你甚至还会去费心吗？视觉或功能缺陷不会困扰你吗？你如何伸出手？你提供什么样的信息？
