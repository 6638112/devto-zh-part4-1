加利福尼亚州 PALO 阿尔托，2019 年 6 月 6 日-[SQL 和 NoSQL 领域的数据库即服务(DBaaS)领导者 ScaleGrid](https://scalegrid.io/) 宣布扩展其完全托管的 [MySQL 托管](https://scalegrid.io/mysql.html)服务，以支持亚马逊网络服务(AWS)云。该平台允许 [MySQL AWS](https://scalegrid.io/mysql/aws.html) 管理员在云中自动执行耗时的数据库操作，并通过高可用性、灾难恢复、多语言持久性以及高级监控和分析提高他们的性能。[读取释放](https://scalegrid.io/blog/press-release-scalegrid-dbaas-expands-mysql-hosting-services-through-aws-cloud/)。

[![MySQL Slow Query Analyzer - ScaleGrid DBaaS](../Images/8d94884b37993d3d7e1e432b38d1a0cd.png)T2】](https://scalegrid.io/blog/press-release-scalegrid-dbaas-expands-mysql-hosting-services-through-aws-cloud/)

多年来，将数据迁移到云已经成为希望实现基础设施现代化以提高安全性、性能和灵活性的组织的首要任务，紧随其后的是从商业数据库管理系统到开源数据库的趋势转变。毫不奇怪， [#1 开源数据库](https://scalegrid.io/blog/2019-database-trends-sql-vs-nosql-top-databases-single-vs-multiple-database-use/)和世界上最受欢迎的云提供商非常适合这种转变。

ScaleGrid 的解决方案通过他们的[自带云](https://scalegrid.io/pricing.html?db=mysql&cloud=cloud_aws_standard&replica=deployment_standalone&instance=micro#section_pricing_byoc) (BYOC)计划带来了独特的变化，该计划允许您通过自己的 AWS 帐户托管 MySQL。虽然所有其他 DBaaS 平台都要求您通过他们的服务进行托管，但 ScaleGrid 鼓励用户在自己的帐户中进行安全托管，以便他们可以利用 AWS 虚拟专用云(VPC)等高级安全功能来保护他们的数据免受互联网攻击，并利用安全组来锁定对其服务器的访问。此外，组织能够通过 ScaleGrid 的 BYOC 计划利用保留的实例，从而节省高达 60%的长期数据库托管成本。

ScaleGrid 创始人兼首席执行官 Dharshan Rangegowda 表示:“不得不放弃管理控制一直是企业采用 DBaaS 的最大障碍之一。“在 ScaleGrid，我们将权力掌握在拥有完全 MySQL 超级用户管理权限和对您机器的 SSH 访问权限的用户手中，因此您不必牺牲控制来利用托管数据库服务。”

ScaleGrid 早在 2018 年 11 月就首次在 Azure 上添加了对 [MySQL 的支持，加入了他们完全托管的](https://scalegrid.io/mysql/azure.html) [MongoDB 托管](https://scalegrid.io/mongodb.html)和 [Redis 托管](https://scalegrid.io/redis.html)开源数据库服务家族， [PostgreSQL 托管](https://scalegrid.io/postgresql.html)预计将于 2019 年 6 月推出。通过在 AWS 上添加 MySQL，ScaleGrid 客户现在可以在五个北美地区、一个南美地区、三个欧洲地区和六个亚太 AWS 地区部署该 SQL 数据库。
