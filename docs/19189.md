朋友们，我非常激动地宣布我们今年的工作坊！在过去的三年里，我们为恐龙举办了研讨会，我们学到了很多。其中之一是参与者和组织者都有信息。就像，所有的 fomo。我们制作这首单曲的原因之一是为了让每个人都能体验到我们的演讲者如此努力的演讲，而你不需要选择。现在，车间在一天内完成这项工作有点困难；然而，今年我们对 fomo 做了一些改变。

## 😻处理我的情绪，一个 CSS 在 JS 工作坊😻

曼尼·卡雷拉四世和威尔·克莱因，工作日

在这个工作坊中，你将学习用 JS 编写 CSS 的细节！我们将采用一个基于 React 的应用程序，带有经典的样式表，并使用 Emotion([https://t.co/5IFDWRYjjF](https://t.co/5IFDWRYjjF))一步一步地迁移到 JS 中的 CSS。我们将讨论在 JavaScript 中管理样式的好处和缺点。你不仅会学到所有利用情感的方法，你还会成为一名时尚大师！

## 🔥基于 Firebase 的 Web 性能🔥

大卫·伊斯特，消防基地

"我的网站应该多快加载？"这是一个简单的问题，但没有简单的答案。网络性能有很多方面；不管你是用 React，Angular，还是 slicing PSDs 对表(好吧，不要那么做)。为了理解你的站点的性能，你需要正确的工具，更重要的是正确的心态。这个工作坊将帮助你在建立网站的三个阶段中培养这种心态。第一阶段:学习关于 web 性能的重要概念，比如延迟和带宽。第二阶段:测量 web 性能(使用 Firebase 性能监控),并了解常见的性能指标，如 FCP。第三阶段:诊断问题并部署修复。在本次研讨会结束时，您将能够了解 Firebase 或 Chrome DevTools，并了解导致性能低下的原因。我们还将谈到像 Docker 这样的工具来在 Cloud Run 上部署无服务器后端。所以带上你的笔记本电脑，准备好回答“我的网站应该多快加载？

## 🛡 TwilioQuest 🛡

特维利奥·张秀坤·昆德尔

这不是典型的技术研讨会。你不会出现，坐下来，坐着看几个小时的幻灯片。你是自己学习日程的主人。选择你想参与的活动，按照你喜欢的节奏进行。TwilioQuest 是一款互动、自定进度的游戏，您可以在其中学习如何 Twilio。TwilioQuest 介绍 Twilio 产品、功能和概念，同时奖励您经验值和战利品。没有比掌握语音、短信、视频或我们的其他产品更快的方法了。无论你是已经写了几十年的代码，还是刚刚开始一个训练营，你都可以来提升你的开发技能。您将学习如何在专家的指导和支持下大规模启动和运行。

## 🚗面向非 Go 开发人员🚗

布伦纳·马滕森，想象分析

在“你应该知道的相关语言”中，Go 正成为越来越必要的工具。尽管这种语言的美妙之处在于它的简单性，但是从 JavaScript 或 Ruby 的过渡仍然会有些颠簸。在本次研讨会中，我们将学习 Go 的基础知识，特别注意从 JavaScript 等语言转换时可能遇到的陷阱。这个工作坊是为很少或没有 Go 知识的开发人员设计的，虽然协作是没有糟糕的工作坊经验的关键部分，所以所有技能水平都是受欢迎的！
