## [](#vs-color-output)VS 颜色输出

## [](#visual-studio-intellicode)Visual Studio IntelliCode

[Visual Studio IntelliCode](https://marketplace.visualstudio.com/items?itemName=VisualStudioExptTeam.VSIntelliCode) 是一组人工智能辅助技能，通过上下文智能感知、参数完成、代码格式化和样式规则推断等特性来提高开发人员的工作效率。IntelliCode 通过机器学习工具增加了当前构建者的工作流，这些工具提供了对代码及其上下文的理解。它适用于 C #、C++(在预览版中)、JavaScript/TypeScript(在预览版中)和 XAML 代码，预计将来会更新以支持更多语言。默认情况下，启用 C#和 XAML 基本模型。您必须启用预览中的语言。【T2![Visual Studio IntelliCode - Visual Studio Extension](../Images/86b224aa4240e6105a8f7345907e2170.png)

## [](#trailing-whitespace-visualizer)尾随空白可视化器

## [](#resharper)重磨

<figure>

ReSharper 是一个流行的微软 Visual Studio 开发人员生产力扩展。它自动化了您在编码例程中可以自动化的大部分工作。它会在您键入时发现并纠正编译器错误、运行时错误和冗余，并提出智能调整建议。ReSharper 使您能够通过可视化文件组成、类别和设计层次、调用和价值链以及项目依赖性来研究软件。它可以让您立即遍历整个解决方案，直接跳到您正在搜索的精确文件夹和行。如果需要，它还可以反编译库代码。【T2![Options in ReSharper](../Images/1f053c99e282d5c46c27acdaf20bf3b0.png)

<figcaption>Options in ReSharper</figcaption>

</figure>

## [](#glyphfriend)雕刻朋友

Glyphfriend 是一个 Visual Studio 扩展，它增强了现有的 Intellisense，可以显示许多常见的基于字形的字体库的预览字形。Glyphfriend 支持的一些库有 Bootstrap Glyphicons、Entypo、Font Awesome、Foundation、Ionicons、Material Design 和 Octicons。【T2![Glyphfriend - Visual Studio Extension](../Images/ff3eee342a2c62a6732ddef53c10b96a.png)

## [](#slowcheetah)慢行

当你在 Visual Studio 中按 F5 键时， [SlowCheetah](https://marketplace.visualstudio.com/items?itemName=vscps.SlowCheetah-XMLTransforms) 包允许你自动转换你的 app.config(或任何文件)。基于构建配置，您可以有不同的转换。这将很容易使你有不同的应用程序设置，连接字符串等。，用于调试和发布。如果你想转换其他文件，你也可以这样做。当你有多个配置的项目时，dev- > staging- > production 任务会变得非常不愉快。我记得在 app.config 文件中为每个环境添加了不同的注释配置，并努力注释和取消注释正确的配置。嗯，我很高兴在使用 SlowCheetah 之后，那些日子已经过去了。【T2![slow cheetah - Visual Studio Extension](../Images/ab8e543ea9f0e066088c0cdc1bb9458d.png)

## [](#image-optimizer)图像优化器

<figure>

为了给客户提供最佳的体验，我们都希望我们的应用程序能够尽可能快地加载。实现这一目标的措施之一是确保优化我们使用的图像。Visual Studio 的 [Image Optimizer](https://marketplace.visualstudio.com/items?itemName=MadsKristensen.ImageOptimizer) 扩展是最流行的扩展之一，因为它使用简单，优化算法强大。它使用行业标准工具来优化任何 JPEG、PNG 和 GIF，包括动画 GIF。它可以进行有损和无损优化。【T2![Optimizing image using Image Optimizer - Visual Studio Extension](../Images/5c947028e952d946622c957b42eabe04.png)

## [](#file-nesting)文件嵌套

Visual Studio 已经在解决方案资源管理器中嵌套相关文件很长时间了，但是并不是每个人都同意它使用的规则。这不再是问题，因为 Visual Studio 现在让您可以完全控制解决方案资源管理器中的文件嵌套。[文件嵌套](https://marketplace.visualstudio.com/items?itemName=MadsKristensen.FileNesting)可以根据文件名自动嵌套文件，也可以手动嵌套和取消嵌套任何文件。文件嵌套还为您提供了根据文件命名规则自动嵌套的选项。然后，您可以将这些规则应用于任何文件、文件夹或整个项目。 [![File nesting - Visual Studio Extension](../Images/2f653a2501913185f375f15a213d1260.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--C1xMVD96--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.syncfusion.com/blogs/wp-content/uploads/2019/08/File-nesting.png)
