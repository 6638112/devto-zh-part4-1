**迈克尔·杰克逊讨论 React-路由器 v4、React Native 和 React Fiber**

[https://www.youtube.com/embed/79Jc-F1b7x4](https://www.youtube.com/embed/79Jc-F1b7x4)

React-路由器 v4 发布了，它对 React 社区中的每个人都有好处。不像 v2 和 v3，最新的版本把所有东西都分开了，使它更加模块化。这意味着你可以在任何你想的地方绘制一条路线，从而简化嵌套、代码分割和热重载。

React-Router 的这种基于组件的变体将理想地解决 React Native 中路由定制代码的需求，并有望弥合 React 和 React Native 社区中的巨大差距。Michael Jackson 希望 v4 能够通过增强代码使团队之间的讨论更容易转移，从而向加强 React 社区迈出一步。

除了 React-Router 之外，React Fiber 还在 React 社区掀起了一股浪潮，它突破了优先考虑用户交互的寻址方式的界限，同时允许异步事件触发。

**Devon Lindsey 谈 React 社区的前景**

[https://www.youtube.com/embed/TgOwnhc6JwE](https://www.youtube.com/embed/TgOwnhc6JwE)

**Preethi Kasireddy 在 Redux、MobX 和 React 社区上**

[https://www.youtube.com/embed/MPsW_pjIyjY](https://www.youtube.com/embed/MPsW_pjIyjY)

Preethi Kasireddy 的工程师职业生涯只有两年，但她已经在许多会议的演讲中崭露头角，并在自己的公司担任 UX/UI 工程师。Preethi 发现与 React 一起工作很愉快，因为它的即时反馈和与欢迎社区的互动。

当她开始学习 React 时，她在 Redux 的 companion 中使用 React，但后来发现了 Redux 和 MobX 的最佳用例。虽然两者都有助于管理组件外部的状态，但两者存在于相反的范例中。

Redux 源自 flux 架构，提供一种可预测的单向数据流，因此随着时间的推移，它有助于提高可维护性和可伸缩性。在 Preethi 看来，这使得 Redux 非常适合更复杂的代码库和大型团队。另一方面，MobX 有很多隐藏在幕后的抽象。它使用可观察的日期来存储状态，这使得它非常适合于简单的应用程序和小型团队，它们可能不需要依赖 Redux 提供的可预测性。

要了解 Preethi 对 React 工具的更多想法，您可以在 Twitter @iam_preethi 上关注她
