# 招聘和面试开发人员

> 原文:[https://dev . to/Kyle cordes/hiring-and-interview-developers-3gb](https://dev.to/kylecordes/hiring-and-interviewing-developers-3agb)

我以一系列推文的形式写了这篇文章，但也许这里有一个更好的地方来长期捕捉它。

我一直在研究和尝试招聘+面试的技巧。还不如赶上现在的“10 倍”潮流呢，嗯？以下是我学到的一些东西。(语境:我大多是雇软件开发人员，也有其他角色。)

无论招聘启事中有或没有什么词语，大量明显与工作不匹配的人都会申请。每个公司都只雇佣最优秀的人。不可思议的是，尽管如此，美国经济的失业率仅为百分之几。

许多专家主张花数小时挖掘前任雇主不恰当的内部细节。不可思议的是，所有人都希望新员工能保守秘密。

许多专家认为他们独特的面试方式显然是最好的。但是任何人都可以在 YouTube 上学习如何应对任何流行的采访风格。

许多专家的基本观点是“过去的表现等于未来的结果”。你可以为这个建议付很多钱。关于一个假想的世界，在这个世界里，人们不会成长或改变。

许多公司面试软件开发人员时都会问一些二年级计算机科学的愚蠢问题。显然，他们这样做是出于习惯，因为这似乎很难揭示工作表现。

时间是宝贵的。有时间预算，并且毫不留情地坚持下去。中断可能是必要的。

对于技术类的工作，是要雇擅长说 X 的人还是做 X 的人？如果是后者，想办法“试镜”。

许多申请人想“跳上一个电话”。时间是宝贵的。先交换一些信息，用这些信息来决定是否面试。然后通过视频聊天与议程会面，进行更全面的交流。

只有一个正确答案的问题没那么有用。揭示技能和经验水平的问题非常有用。
