# JavaScript 开发人员的可视化和设计

> 原文：<https://dev.to/rkristelijn/javascript-developer-s-visualization-and-design-39hp>

几周前，我为 github 回购做了一点贡献。当我想尝试另一个开发者的东西时，这是我经常做的事情；我克隆了回购，通常安装说明中有一些过时的依赖项或错误，分支它并制作 PR 有多难？通常这是高度赞赏的。

在这种情况下，这是一个我经常使用的项目，它是 Visual Studio 代码中的一个插件，名为 [yUML](https://marketplace.visualstudio.com/items?itemName=JaimeOlivares.yuml) 。在我输入代码或想法之前，我用它来形象化我的代码或想法。这很简单；

1.  你创建了一个新的*。yuml 文件，然后输入下面的一个单词:[class | activity | use case | state | deployment | package | sequence]，这样就会为您预先生成一个示例，可视化您想要的图，这样您就可以根据自己的需要开始修改了。完成后，您将`// {generate:true}`添加到顶部，您可以将您的设计导入到您的。md 文件使用`![](img/f5d702f30cf806fef3bb9551a787494b.png)。

我发现例子有点难找。所以我为维基做了贡献，所以无论何时你在预览中按下‘维基’，你都会看到我的贡献。

除此之外，我还发现了另外两个类似的项目:

1.  [Chart.js Markdown 预览](https://marketplace.visualstudio.com/items?itemName=FlomoN.chartjs-markdown-preview)但是这在 github 上不起作用
2.  [美人鱼编辑器](https://marketplace.visualstudio.com/items?itemName=tomoyukim.vscode-mermaid-editor)可以手动生成 svg 文件，没有 [yUML](https://marketplace.visualstudio.com/items?itemName=JaimeOlivares.yuml) 那么严格

我很想听听你的(免费)可视化技术，它可以跨操作系统工作，而无需安装复杂的程序！
