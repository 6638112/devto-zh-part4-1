# 为什么使用 Smalltalk？

> 原文：<https://dev.to/smalltalkrenais/why-use-smalltalk-hfl>

当然，不是为了工作。Smalltalk 的工作机会很少。然而，出于所有其他原因，这是一个伟大的选择。

首先，Smalltalk 极其简单易学。甚至比 Python 和 Go 还要多。

Smalltalk 拥有全部六个保留字。完整的语法可以放在明信片上！

施乐 PARC 公司的 Alan Kay、Dan Ingalls 和 Adele Goldberg 专门创建了 Smalltalk 来教授年轻人编程。没有比这更简单的了！

看看你如何在 15 分钟内学会完整的语法:[用 ProfStef](https://amber-lang.net/learn.html) 学习 Smalltalk。(确保你遵循第 5 行的指示:“**选择**下面的文本，并点击‘DoIt’按钮”。)

第二，尽管 Smalltalk 体积小且简单，但用途非常广泛。它几乎可以用于任何事情！Web、桌面、移动、数据科学、机器学习、物联网、嵌入式、机器人、虚拟现实、企业商业计算，应有尽有。

Smalltalk 的生态系统可能比较小，但是它内置的类库非常丰富，确实如此。Smalltalk 实际上是程序员的一站式购物。

我强烈推荐 Pharo，它是 Smalltalk 的现代变体。对于 web 开发，你有[海边](http://www.seaside.st)、[茶壶](https://github.com/zeroflag/Teapot)和[法老](https://pharojs.github.io)。

对于数据科学，你有[博学](https://github.com/PolyMathOrg/PolyMath)和[罗萨](http://agilevisualization.com)。

对于物联网，你有[pharhothings](https://github.com/pharo-iot/PharoThings)。

第三，根据 Namcook Analytics 进行的[研究，Smalltalk 的*现场编码*能力使其成为世界上最高效的通用编程语言之一。平均而言，它的工作效率是 Javascript、C++、Go、Java、PHP、Python 和 C#的两倍。这是平均值。在许多情况下，生产率的提高可以归功于 5X。](https://www.ifpug.org/wp-content/uploads/2017/04/IYSM.-Thirty-years-of-IFPUG.-Software-Economics-and-Function-Point-Metrics-Capers-Jones.pdf)

第四，Smalltalk 对于*元编程*来说很棒。这方面很像 Lisp。参见 [Lisp，Smalltalk，和对称的力量](https://medium.com/smalltalk-talk/lisp-smalltalk-and-the-power-of-symmetry-8bd96aaa0c0c)。

最后，Alan Kay 的 OOP 概念是纯金的，Smalltalk 是它的缩影。观看艾伦·凯对泰德·纳尔逊的致敬。

艾伦·凯的《[small talk](http://worrydream.com/EarlyHistoryOfSmalltalk/)的早期历史》(1993 ACM)提供了更多的见解:

> Smalltalk 是对计算机本身概念的递归。每个 Smalltalk 对象都是对计算机的全部可能性的递归，而不是将“计算机的东西”分成比整体更弱的东西，如数据结构、过程和函数，这些都是编程语言的常用工具。因此，它的语义有点像成千上万的计算机通过一个非常快速的网络连接在一起。

和

> Smalltalk 的贡献是一种新的设计范式——我称之为*面向对象*——用于解决专业程序员的大问题，并使新手用户的小问题成为可能。面向对象的设计是一次成功的尝试，它从质的方面提高了硅爆炸所带来的越来越复杂的动态系统和用户关系的建模效率。

[艾伦·凯的名言](https://youtu.be/oKg1hTOQXoY?t=600):

实际上，我创造了“面向对象”这个术语，而且我可以告诉你我脑子里没有 C++。

艾伦·凯(Alan Kay)说:“互联网做得太好了，以至于大多数人认为它是像太平洋一样的自然资源，而不是人造的东西。上一次如此大规模的技术如此准确无误是什么时候？”

艾伦·凯将他的面向对象哲学比作互联网是如何建立的，或者由数十亿个细胞组成的生物有机体是如何建立的。Smalltalk 非常适合真正可扩展的解决方案。
