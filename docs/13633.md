大家好！

今天，我想向大家展示我最近在做的一些东西，并征求关于下一步的任何反馈和想法:)

我建立了一个 web 应用程序，它分析 GitHub 上的存储库，并在仪表板上显示它们，比如 Rails 的或 T2 的【dev . to】T3。

最初的想法是对团队的表现给出一个舒适的看法，提出可能的问题并提出见解。
由于 GitHub 在该领域收购了一家提供类似功能的竞争对手，我将稍微转而展示不同软件库之间的[比较](https://rails.devlytics.io/rails?compare_to=laravel/framework)，并考虑我的下一步行动。

展望未来，我认为以下内容可能有助于充分利用我迄今为止构建的内容并提供价值:

*   徽章储存库可以添加到他们的自述文件页面，用于合并率，平均响应时间等。

*   贡献者的个人排名:平均审查周期，他们的推送失败测试的频率，提高覆盖率，公关规模等等..应该鼓励更好的工作方式的事情。

*   更聪明的洞察力——基于该区域的 PRs 被合并的速度，需要多少审查周期等等，代码库中的问题区域在哪里..

*   开源项目的排名:我对此持观望态度。我不想根据 PRs 被处理或问题被解决的速度来对开源项目进行排名，但是我试图找出一种方法来使维护者的生活更容易并为他们的工作感到自豪。
