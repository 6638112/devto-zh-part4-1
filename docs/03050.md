# 冒名顶替综合症及如何摆脱它

> 原文：<https://dev.to/jude_johnbosco/imposter-syndrome-and-how-to-get-off-it-1pb3>

利用技术人员，尤其是开发人员作为话题

冒名顶替综合征是指当某人试图做某事时，感到自己“无能”。

实用插图:

招聘人员:你有什么技能我:我用 javascript 和它的框架如 React 编写 web 应用程序。我过去做过一系列的项目。招聘人员:好的，很好。你被录用了，明天可以开始工作

我的头:哇！！太快了，没想到会这么容易和快速。反正我已经被录用了。

在办公室老板:添加功能 ABC 在我们现有的应用程序中，这样做…..那个…..我:哦！！！天啊，我没有这些人想象的那么好，我想我一直在欺骗自己。我的头:这些家伙下个月就会在你意识到之前把你踢出去。你不能这么做。

我:现在感觉像个骗子，我应该告诉他们我什么都不知道吗？

好吧，这就是我现在能以我的水平解释冒名顶替综合症的实际程度，哈哈。

所以我决定写一写这种骗子综合症，特别是对开发人员来说，因为我经常陷入这种状态，也看到过朋友陷入其中。冒名顶替综合症让你觉得自己是个骗子，很快就会被抓住，它让你觉得自己非常不胜任自己的工作或分配给你的工作。耶！！但好消息是，你不是唯一一个受这件事影响的人，即使是高级开发人员大多数时候也会发现自己处于这种破坏的卡车中，坏消息是，如果你让它深深影响到你而没有采取适当的行动，它可能会毁了你的职业生涯。最近，在技术行业，特别是在开发人员的生态系统中，这种冒名顶替综合症已经成为一个话题，已经摆脱它的有经验的人试图宣扬高级开发人员、初级开发人员和代码新手如何摆脱冒名顶替综合症。在此之前，你应该

不要拿自己和任何人比较

没有人知道这一切(绝对没有人)

开始设定目标，培养成长，而不是坐下来比较技能

学习如何按照你的节奏做事，以及哪种学习模式适合你

当你遇到困难时，向朋友求助(作为一名开发人员，这是加入开发人员社区的一个重要原因和重要性)

学会如何学习(自己开车)

不要放弃任何一步，如果你让它休息一会儿，然后再回来就好了

当你有了冒名顶替综合症的感觉时，试着做一些你喜欢做的不同的事情，这样你就可以在那一刻清除你的记忆。

不要强迫自己做任何事情

所以在最后，我鼓励你不要觉得你是唯一一个有这种冒名顶替综合症经历的人。你不是一个人，你不是不合格。
