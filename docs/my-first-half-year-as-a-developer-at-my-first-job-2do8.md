# 我在第一份工作中作为开发人员的头半年

> 原文：<https://dev.to/lukasderksen/my-first-half-year-as-a-developer-at-my-first-job-2do8>

# 简介

因为这是我在这里的第一篇文章，我想有一个介绍是合适的。我是一名来自荷兰的 23 岁的开发人员。我从高中开始涉足编程和脚本。我想我开始写一些静态 HTML 网站的时候是 12 岁左右，开始研究脚本语言的时候是 14 岁。当我 16 岁的时候，我通过我们的 IT 课开始接触 Java 小程序。我 18 岁时进入大学。我的学校提供了很多选择。我们有 4 条主要路线；软件工程、技术/嵌入式系统、媒体设计和 IT 业务。此外，我们还有 12 个子路线，包括网络安全、智能手机、数据科学、教育等。我参加了软件工程和网络安全的课程。因为我的实习都是在网络安全领域，所以我获得了软件工程和网络安全的学士学位。

现在，为了让它变得更加个人化，我在业余时间会演奏很多音乐。我学会了弹吉他、小号、钢琴、蓝调竖琴，最近还开始唱歌。我是披头士的超级粉丝，通常喜欢 60、70 和 80 年代的音乐。我在业余时间不怎么写代码。我已经每周花 40 个小时做这件事了，所以我喜欢先把其他事情按优先顺序排好，然后看看如果我想的话，是否能把一些代码放进去。

# 获得工作机会

我读过很多关于人们试图进入这个领域却难以获得工作机会、面试等的故事。就我个人而言，我不能认同这一点，因为我必须为这份工作做的唯一面试是获得实习机会，这显然比全职工作面试容易得多。我得到了实习机会，得到了用 python 写一个开源漏洞扫描工具的任务。大约进行到一半的时候，董事们和另一个开发商让我去会议室。他们对我的工作和帮助非常满意(我曾为他们做过一些小项目，以推广他们的产品)，并问我是否愿意在实习结束后留下来。想了一下，我说好。我喜欢我的同事，项目很有趣，我想我可以从小公司开始学到很多东西。

# 我的前六个月

所以。回顾我的第一个半年，我认为我过得很好。我将首先讨论我对公司的影响，然后讨论我学到了什么。

我们的开发团队由两名开发人员组成。另一个开发者在过去的两年里开发了一个应用程序来做匿名的在线研究。该应用程序工作得非常好，我们的客户都很热情，但代码库是谋杀。他实际上是一名网络架构师，碰巧懂一点 PHP，所以他开始努力让它工作起来。由于公司里没有其他人知道如何编码，没有人能告诉他如何更好、更高效地做事。直到我苏醒过来。我必须承认，有时我对我的代码审查相当无情，但他能处理好，并感谢反馈。他甚至说他在去年学到的编程知识比之前的几年都多。让我觉得自己有点像资深开发者。

反正从接受 offer 开始，我就一直主张重写申请。我知道在大多数情况下这是多余的，但是请相信我，重构代码会花费更长的时间，而且效率可能会更低。我也喜欢花时间不去用锤子砸自己的脑袋，所以重写似乎是最好的选择。另一个 dev 同意了，所以我们只需要说服董事们。最终，我们成功地实现了这一点，并制定了一个计划，说明新版本的开发轨迹。我们已经研究了选项，并选择用 Laravel 和 Vue.JS 编写应用程序。我们最初选择 Laravel 是因为它是一个 PHP 框架，而其他开发人员已经知道 PHP。后来我们知道这确实是正确的选择，因为这是一个非常完整的框架。我们选择 vue，因为 Laravel 支持它开箱即用，它越来越受欢迎，还因为它极简，但非常可扩展。

在我开始在这里工作之前，我不知道 PHP。由于它的坏名声，我总是避开它。我事先也不知道 Vue.js。我确实知道一些 Javascript，但是我不知道更高级的概念，比如 promises、async/await 等等。你可以说我对我现在使用的工具几乎一无所知。这一切都是在我尝试概念验证、阅读关于最佳实践的文章、阅读关于堆栈溢出的解决方案等过程中成长起来的。我确实有在学校学过 C#、Java (SE 和 EE)和 python 的优势，所以我对编程概念有广泛的掌握，并且用新的语言很快就学会了。但是，你仍然需要一些时间来适应这些新的语言，学习它们的来龙去脉，以及如何用它们解决某些问题。

有些人可能会说你需要一个高级开发人员来指导你完成这个阶段，教你最佳实践等等。也许是这样，但在我看来，我自己到目前为止做得还不错，我认为这种在职业生涯早期独自建立项目、使用新工具并将其应用于实际项目的经历对我未来的职业生涯来说是无价的，可能会给我一个飞跃。

我还想提一下我在组织层面上所做的改变。从我开始在那里工作开始，我就一直提倡使用 git。由于他的背景，开发人员不知道这项技术，所以起初他有点不屑一顾。我想大概过了一个月，我们为这个应用程序计划了一个发布版本(包括怪物代码库)。我开发了一个特性，但是因为我们没有真正的协议来将代码从开发推向生产(我们基本上是直接在生产应用程序上工作，但是在一个只有我们可以访问的区域。我知道，这是次优的，但我们现在做得更好)，我的代码被覆盖，并设置回我一周前的代码。我们能够通过备份恢复我的工作，但这确实让 git 走上了优先通道，所以不到两周之后，我们有了一个自托管的 Gitlab 服务器，我们现在在其中存储我们所有的代码。我们也采纳了我的建议，用 SCRUM 的方式工作，以简化迁移到新版本和新技术的过程。

# 结论

总的来说，我认为到目前为止我做得很好。我的意思是只有半年，但是到目前为止，我对我的轨迹和我学到的所有东西都感觉很好，公司对我也很满意，因为我上周刚刚签署了我的延长合同。我意识到这是一个关于初级开发人员如何发展的高度个人化的描述，如果有人能完全理解这个故事，我会感到很惊讶，但我认为这可能会让你们感兴趣。我也只是想为自己在相对较短的时间内所取得的成就表示自豪。

好了，各位，干杯！
