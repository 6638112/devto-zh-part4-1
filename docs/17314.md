# 经过一年半的开发，我学到了什么

> 原文:[https://dev . to/dt roode/what-I-learn-after-a-years-developing-1362](https://dev.to/dtroode/what-i-learned-after-a-year-and-half-of-developing-1362)

2018 年 1 月开始做代码。一年多后，我在开发过程中改变了很多东西。

首先，我是前端开发者。这意味着我使用 HTML，CSS，JS。

让我们从一个列表开始:

*   原子→崇高文本→ VS 代码。我开始在 codecademy 上编码(它是 [Google](https://www.google.com/search?client=firefox-b-d&q=learn+to+code) 里的第一个)。我的第一个 HTML 代码复制到 Atom 并保存在电脑上。我选择 Atom 是因为很久以前我就在寻找一个好的文本编辑器，并看到了很多推荐。接下来，我转到了崇高的文字，并爱上了它。我写了很多未实现的网站，有些实现了。接下来，我听说了 VS 代码。对我来说这很复杂，但是几个星期后，我完全改变了。
*   美化→更漂亮。手工格式化代码非常困难，所以我需要自动化这个过程。我已经使用美化，但最近切换到更漂亮的配置文件。
*   默认等宽字体→ Fira 代码。字体是开发过程中非常重要的东西。以前觉得不怎么样。有一次，我在 Twitter 上看到一些有趣的字体，对我来说这是一个新事物——连字支持。这是 Fira 代码。现在它是一种非常流行的字体。
*   jQuery →清理 JS。我从 YouTube 上的 jQuery 教程开始用 JS 编码。教程说 jQuery 更干净、更小、更容易，使用 jQuery，你可以用几个包含 JS 200 字符串的字符串来编写任务的解决方案。但是过了一段时间后，我明白了 clean JS 有更多的支持，更干净，更容易，占用浏览器默认支持的最小空间。
*   CSS → SASS。喜欢这个`parent { child }`语法。
*   哲基尔→盖茨比。Gatsby 是一个更先进的静态站点生成器，有很多默认优化。
*   灯塔→ PageSpeed 洞察。所以我爱《灯塔行》里的绿谱，爱死了。最近谷歌将 Lighthouse 转移到网络上，更加稳定。

如果我记得更多，我会扩展一篇文章。
