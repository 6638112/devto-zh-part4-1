# 亲子关系和编码:混合，不要混杂

> 原文：<https://dev.to/eekayonline/parents-and-coding-mix-don-t-mingle-12db>

被一个最新的开发者社区拒绝感觉很心痛，但是把父母身份和编码混在一起有时会导致这种情况。我花了两个多小时在三个 C#开发任务上，但不是我花了它们，而是我什么时候花了它们。

你看，我是一个有超过 15 年开发经验的资深开发人员。我已经从一个初级的 web 开发人员变成了一个资深的 web 开发人员，开始开发移动应用程序(首先是本地的，然后是跨平台的)，最后迷上了开发和定位数字产品，成为一家初创公司的技术领导者。

现在，38 岁的我想去建立自己的梦想:创建自己的公司，通过产品和咨询提供价值。在那里，咨询仅仅是维持我财务状况的一种方式，并为我赢得从事自己工作的时间。所以最近，我开始做那个梦。为此，我开始在另一家公司做兼职团队首席开发员(3 天时间很少)，剩下 2 天时间在我自己的公司工作，同时照顾孩子。

但是，回到我的故事的开头，这个令人敬畏的在线社区的审查过程包含了其他人的任务，这些任务完全符合我的远程咨询心态，包括在线编写 3 个任务，以查看我是否是一个优秀的开发人员。我太天真、太放肆了，以为当我的孩子们在后花园玩耍，我最小的孩子在床上睡觉时，我也能做这些。

天哪，我犯了一个错误。除了孩子们带来的小干扰和互动之外，我的思想不能足够集中在作业上。我的孩子们真的很可爱，但直到现在我才真正意识到为人父母的重要性:当你的孩子在身边时，你总会有一个内部线程在运行，检查你的孩子是否在身边。

正是这种我称之为“养育线程”的东西，让我非常感激，它阻止了我正确地完成这三项任务，并让我自己加入社区项目。

所以我学到了什么？我爱我的妻子和孩子，我喜欢为我的梦想而努力。但是有一个地方和时间让两者都得到我的奉献和关注。混合我的注意力，甚至完全不在他们身边，会让我的两种激情都以我不想再遇到的方式出错。

社区呢？

我给他们发了一封电子邮件，声明我理解并尊重他们的审查过程，他们的社区值得并需要关注和奉献。

对我来说，我知道只要我在适当的时候为我的公司提供焦点和关注，我就会摧毁它——即使它没有来自我所瞄准的社区的工作。

保重，我希望你能投入你的发展事业和家庭应得的全部精力和奉献。
