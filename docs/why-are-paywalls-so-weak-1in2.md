# 为什么付费墙如此脆弱？

> 原文：<https://dev.to/ohryan/why-are-paywalls-so-weak-1in2>

作为我日常工作的一部分，我一直在调查(软)付费墙的实施情况。

当我浏览各种有影响力的网站时，我发现它们都很软。当用户注销时，他们似乎都在使用带有简单整数计数器的 localStorage，或者有时是带有文章阅读记录的 json。

显然，追踪注销用户存在严重的技术、隐私和(可能)法律挑战。我永远不会希望网站实现这样的东西。

但是为什么更多的站点不强迫用户立即登录呢？

由于跟踪仅限于本地存储，用户可以通过切换到匿名、切换浏览器或清除缓存来绕过防火墙；而且这些步骤通常没有实际注册账户那么麻烦。
