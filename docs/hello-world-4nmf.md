# 另一个 Hello World！

> 原文：<https://dev.to/phantomhaze/hello-world-4nmf>

嘿戴夫托！

我刚刚整理完我的作品集(你可以在这里找到)，我想现在是时候开始和其他开发者交流了！

简单地说，从高中开始，我就把编程作为一种爱好。在我上大学之前，我从未遇到过任何其他科学家或技术人员(我家第一个这样做的人)，所以我对如何在软件开发的职业生涯中继续前进缺乏方向。通过参加科学实习和在科学家周围从事短期的政府合同工作，我获得了很多信心，现在我觉得我有信心进入专业编程的世界，并利用我的编码知识成就一番事业。

大学毕业后(只完成了计算机工程学士学位的前半部分)，我开始在业余时间认真对待编程。不幸的是，我在我的方法中分散了注意力，从事使用新的实验框架的副业项目，即在加密货币领域。首先是去年夏天的 Cosmos Sdk，虽然我很欣赏我在 Golang 中获得的经验，但我意识到该框架在简单地创建一个易于采用的 dapp 方面的缺点。

后来，我转到 Hyperledger Fabric，使用 Hyperledger 的沉着，因为它声称可以轻松建立一个生产就绪的区块链。不久之后，开发团队宣布他们将停止进一步的开发，专注于 Fabric，我也失去了用 Composer 完成项目的热情。

最后，我登陆了 Truffle.js，开发以太坊生态系统。在那里我完成了这个项目[你可以在我的文件夹](https://ahernandez.glitch.me/)上看到，这是一张以太坊礼品卡，是给我母亲的生日礼物。

虽然有相当多的项目没有完成，框架仍在开发中，但我很欣赏这种体验，因为它让我习惯于使用实验性的 API，这些 API 每周都会发生变化，或者有时甚至不会像官方文档所声称的那样运行！

我也很高兴学习了 Clojure 和 Clojurescript(并成为了 emacs 的粉丝)，谢天谢地，我可以说我学习这些语言的时间更有收获，因为我计划在下周将这些项目发布到我的作品集上。

然而，为了采取一种更实际、更正常的方式成为一名专业开发人员，我一直在重新熟悉 node 和 react，并在 glitch.com 上发布了一个完整的网站[，你可以在这里找到](https://chow-chow-shop.glitch.me/)。

我有相当多的项目打算很快发布，例如我的。zshrc 文件增加了我的 linux 工作流的趣味。

所以，看在我还是个新手的份上，我请求你，dev.to 的观众们，给我任何可能的帮助或批评！也许[检查我的故障项目](https://glitch.com/@BleedingEffigy/production-ready)。请随时给我个人发展的建议，或者任何你知道的工作机会的线索。与此同时，我会浏览自由职业者，事实上。

谢谢你从头到尾的阅读。
