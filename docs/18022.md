## [](#the-challenge)挑战

所以，我决定通过 Udemy 上的 Angela Yu 的 Web 开发课程，走上我的 Web 开发之路。这是她在第一个 JavaScript 部分提出的所谓的“老板挑战”之一。在这个挑战中，你必须这样做:

### [](#simon-says-game)[西门说游戏](https://londonappbrewery.github.io/Simon-Game/)

当然，我可以马上得到解决方案。但我想看看我能否自己找到解决办法。因此，我在旅途中写下这篇文章，因为我正在解决这个问题。

## [](#second-stage-lets-code)第二阶段:让我们编码

好了，我顺利地完成了以下任务:

1.  创建数组来存储数字/颜色序列
2.  将 6 种声音存储在“音频”数组中，创建新的音频以便于访问它们
3.  让游戏从“回车键”开始
4.  由于我的随机函数创建的数字必须与以颜色命名的类相关联，所以我想出了一个函数，每当你按下一个按钮时，它就会播放相应的声音和一些效果
5.  捕捉用户的点击并将颜色“翻译”成数字。还带有开关功能。一旦它检查了颜色(类),它就把数字放入数组。

## [](#the-problem)问题

我大概明白了顺序。但是我无法在数组之间进行比较。我知道首先我必须检查数组中相同位置的值。此外，在此之后，我应该检查用户数组的长度是否等于系统数组。然后，启动一个新的序列。

我尝试了很多方法和变量。在陷入同一个问题一天后，我决定看看安吉拉是如何解决的。老实说，感觉有点失败。当然，她的代码比我的更简洁明了。但令我惊讶的是，我的方法是正确的。逻辑、功能和步骤实现了相同的目标。

所以问题在于我如何获取用户输入的值。她用下面这行代码解决了这个问题:

```
 checkSequence(sequenceUser.length-1); 
```

## [](#how-i-felt-during-the-process)我在这个过程中的感受

我觉得有必要分享这一点，因为任何人都可能经历和我一样的事情。开始的时候，我感觉很好。我可以很容易地想出逻辑，甚至是我解决如何将数字转换成颜色、声音等的方法。

然而，当我陷入 *checkSequence* 函数时，我感到沮丧、恼怒，有时甚至有点愤怒...当然也对自己失望。

看着这些解决方案对我来说是一个巨大的失败。但是后来我试着不要对自己那么苛刻。看看我自己实际上解决了什么，尽管它并没有尽可能的聪明和干燥。

重构代码不是我现在的首要任务。将来会的。但到目前为止，我专注于让事情运转起来。

最后，当我遇到这些挑战并陷入困境时。我认为忍受挣扎是件好事。试图自己想出一个解决办法。但是你真的需要看到解决方案，对吗？否则，你怎么知道事情可以用不同的资源或其他方法来完成呢？
