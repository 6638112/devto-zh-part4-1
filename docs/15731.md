**包装管理和包装便利**

Ubuntu 使用 APT 包管理器来提供和管理包(应用程序、库和不同的所需代码),而 hat 使用 DNF 包管理器。

一些第三方软件包开发者联合提供点击安装。类似于 UNIX 操作系统的包。在 Ubuntu 中，这些包以。deb 格式，而 hat 支持。rpm 包。

**支持和用户群**

Ubuntu 和 Fedora 都通过社区论坛提供支持。Ubuntu 有两个主要论坛:Ubuntu 论坛和提升 Ubuntu。Fedora 有一个主要论坛。

Ubuntu 的流行促使各种各样的网站和博客完全以 Ubuntu 为目标。这样，你在 Ubuntu 上比在 hat 上获得了很多故障排除技巧和学习资料。

**发布周期**

每六个月发布一个新的 hat 版本，并且每个 hat 的发布都只支持 13 个月。这意味着在 6 到 13 个月之间，您需要进行升级。升级 hat 版本很简单，但是需要一个诚实的网络关联。不是每个人都对每 9 个月 one.5 GB 的版本升级感到满意。

Ubuntu 有两个版本:常规释放和未来支持(LTS)释放。定期释放类似于仇恨。间隔 6 个月出院，支持 9 个月。

LTS 发布间隔 2 年，支持 5 年。常规版本带来了新的选择，新的软件包版本，而 LTS 释放坚持旧版本。对于那些不喜欢频繁变化而喜欢稳定的人来说，这是一个不错的选择。

**固体碱分布**

Ubuntu 是基于 Debian 的。Debian 是最大的社区项目之一，也是自由软件包世界中最受尊敬的项目之一。

Fedora 可能是红帽的一个社区项目。Red Hat 是一个面向企业的 UNIX 操作系统发行版。在 Red hat Enterprise UNIX 操作系统中包含的选项区域单元之前，Hat 充当全新选项的“试验场”(技术术语中的上游)。

Ubuntu 和 hat area 都有母公司支持。Ubuntu 来自 Canonical，而 hat 来自 Red Hat(现在是 IBM 的一部分)。企业支持是非常重要的，因为它确保 UNIX 操作系统发行版得到良好的维护。

由一群人创建的业余爱好者发行版通常在就业不足的情况下崩溃。你可能会看到适度的普通分布因为这个唯一的原因而终止。Antergos，Korora area unit 只是众多此类例子中的几个，在这些例子中，由于开发人员没有足够的空闲时间参与项目，发行版被中断了。

事实上，每一个 Ubuntu 和 hat area 单元都由基于 2 Linux 的企业支持，这使得它们成为不同自由软件发行版的可行选择。

**Ubuntu vs hat 作为服务器**

迄今为止，Ubuntu 和 hat 之间的比较主要是针对桌面用户的。然而，关于 UNIX 操作系统的讨论直到您接受了服务器才算完整。

Ubuntu 不仅在桌面上很常见，它还结合了服务器方面的智能存在。如果你在家使用 Ubuntu 作为桌面，你可能不会对 Ubuntu 服务器版感到不舒服。我从 Ubuntu 桌面开始，目前我的网站区托管在运行 Ubuntu 的 UNIX 操作系统服务器上。

Fedora 也有服务器版，而且有一些人使用它。然而，大多数系统管理员不喜欢每 9 个月就必须升级和重启的服务器。

了解 hat 有助于您开发红帽企业 UNIX 操作系统(RHEL)。RHEL 可能是一个付费产品，你应该购买订阅。如果你想要一个可以运行服务器的操作系统，我建议使用 CentOS。CentOS 是另外一个与 Red Hat 相关的社区项目，但是这个项目集中在服务器上。

**结论**

正如你将看到的，每个 Ubuntu 和 hat area 单元在很多方面都很相似。一旦涉及到软件包便利性、驱动安装和在线支持，Ubuntu 将会领先。这些区域是构建 Ubuntu 的要点，是一个改进的选择，特别是对于没有经验的 UNIX 操作系统用户。

最后，你完全可以决定是使用 hat 还是 Ubuntu。我建议让每个发行版都有 USB 接口，或者把它们转移到虚拟机中。
