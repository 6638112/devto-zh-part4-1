# 建筑砌块

> 原文：<https://dev.to/hhheath/builder-s-block-39m9>

我是一个初学者，和大多数初学者一样，我还在学习。

自然，当学习一件事时，奋斗是必不可少的。不过，我喜欢斗争。我喜欢被困在某件事情上，然后寻找答案。通常情况下，我会尝试四五种不同的解决方案，然后才选择一种有效的方案。

我喜欢这种感觉:最终让那段代码工作，最终得到我想要的输出。

* * *

这里有一点关于我的:

我的本科学位是石油工程，但我的全职工作是合同机械设计工程师。从小到大，我总是围着电脑转(我想大多数同龄人都是这样)。我父亲经营着自己的计算机服务公司，并获得了网络 A+认证，所以我总是在计算机周围，但几乎从不参与。

我的大学学位并没有真正教授计算机科学方面的任何东西。我的大部分课程都是在纸上进行的，如果我们真的使用笔记本电脑，那也是为了 excel。

作为合同工程师，我没有太大的活动空间。无论是新技术，更多的责任，甚至是晋升的机会。我每天做的事情没有挑战性，甚至没有那么费时间。

我想进入科技行业。我完全被创业公司和小公司迷住了。一个小团队努力推出产品的想法让我很兴奋。因此，我开始学习如何成为一名开发人员。

在过去的 6 个月里(以及 2018 年中期的一点时间)，我开始学习 web 开发，特别是 MERN 堆栈。我已经构建了 2 个 API，2 个 react 项目(其中一个我正在集成 API)。我通过创建一个 Pomodoro 定时器(我实际上在工作中使用它)涉足了 ElectronJS 的开发。

* * *

那么，我该如何挣扎？

我没有创造力。我不是零对一类型的人。我非常喜欢 1 到 1.5 或 2.0。我绞尽脑汁想出一些东西放在我的文件夹里，并展示我已经准备好成为团队的一员。当我有明确的目标或可交付成果时，我工作得最好。我也不擅长设计。

当我有一个想法时，我通常可以在几天内完成 MVP(下班后编码)，但设计和让东西看起来漂亮总是我开始失去兴趣的地方。

我最想进入科技行业，但这比我想象的要难。我想我有点期待，你不知道你不知道的事。我想我会从导师或老师那里受益匪浅，但如果不能补偿他们的时间，我会感到很难过。

我得到的关于“如何变得更好”的每一点建议都是去做事情。“你必须投入进去，建造一些东西。那是你学到最多东西的时候！”我完全同意，我从实践中学习，我想大多数人都是这样。但是如果我没有任何想法，我该怎么办呢？

* * *

我很有兴趣听听你要说什么。我该如何克服这种“建筑工的阻碍”？
