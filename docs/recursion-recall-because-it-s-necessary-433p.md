# 递归调用，因为它是必要的

> 原文：<https://dev.to/jonathanmkpt/recursion-recall-because-it-s-necessary-433p>

继我上一篇关于 Big O(涉及程序的时间和空间复杂性)的文章之后，我添加了一个经常使用的，如果没有完全理解的概念的快速回忆。递归！

它能做什么？

用最基本的术语来说，递归是一个有能力调用自身的函数。它在编程中一直被使用，被视为迭代的一种更干净的替代方法。

使用递归，每次使用不同的输入，直到到达基本情况。

例如，您有一个数字数组，您想找出哪些元素是奇数。所以知道偶数可以被 2 整除，没有余数，我可以说我的基本情况是
N%2，其中 N 是数组中的一个元素被 2 整除，没有余数。

我可以把它放在一个助手函数中，不管我输入什么，它都以此为基础。我们需要一个基本案例和不同的输入，以确保程序在满足条件时返回基本案例。
