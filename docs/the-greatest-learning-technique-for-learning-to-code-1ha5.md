# 学习编码的最好的学习技巧

> 原文：<https://dev.to/stevenanthony/the-greatest-learning-technique-for-learning-to-code-1ha5>

我 18 岁开始编程，但我做编程只是因为这是学校要求的，我只是做了要求的事情，仅此而已。我没有保留任何信息，什么都没有。

当我开始上大学时，我开始更认真地对待事情。我的很多课程都集中在面向对象编程上，这很酷，但不是我想学的。

我喜欢构建 web 应用程序，所以我试图找到一些在线学习的方法。我买了一些全栈 javascript 的 Udemy 类，看了 Youtube 教程，关注了互联网指南。结果是...

可怕的是，我一直忙于遵循指南，只是复制面食编码。当开始尝试从零开始构建东西时，我不知道自己在做什么，因为我实际上并没有在学习，我得到了一切。

实际上，编码是关于阅读文档、应用其他开发人员给出的技术和解决问题(堆栈溢出是一个好朋友)。这就是你学习编码的方式。按照指南来构建项目的最大问题是你已经得到了一切。软件开发的很大一部分是解决问题，通过遵循指南，这个方面被消除了。

我能给的最好的建议就是开始做一些事情。无论你想构建什么，无论是网络应用还是移动应用，等等。打开文档，阅读“入门”部分，瞧。你正在创建和完成你的第一个真正的项目。

还有一个很好的帮助我学习的小技巧:每天至少花 1 个小时尝试编码。这真的有助于记住你前几天学过的东西，让你保持头脑清醒。没有什么比回到一个 3 周前的糟糕的代码库更糟糕的了:P
