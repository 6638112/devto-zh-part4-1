# 在上下文中析构数组和对象

在为一个项目做发现时，我使用了一些开发资源来重新定位。这里有很多非常有才华的人做的很棒的东西。

通过拼凑下面的代码，我希望能更好地理解析构的微妙之处，并自己决定在数组和对象之间实现哪个最好。代码可能被过度注释了，但是它确实包含了一些到 DEV 上的源代码的链接。

[https://codesandbox.io/embed/03eh9](https://codesandbox.io/embed/03eh9)

对于任何在代码中使用析构的人来说，对于复杂和分散的项目，对象提供了最好和最内在的清晰性，这并不奇怪；键名变成了变量名，没有比这更好的了。

另一方面，数组必须单独注释。这带来了更大的分离风险。如果您丢失了为数组的每个值拼写名称的注释，数组将不会通过拼写出一个揭示性的名称来帮助您，这些值可能正在使用中，您必须在接收端找到它们才能理解您正在查看的内容。

如果你是初学者，仔细阅读这个文件可能会帮助你学习一些关于析构、生成器模式和函数 currying 的知识。它还可以帮助您在传递设置参数的数组和对象之间进行选择，就像我在创建它时所做的那样。
