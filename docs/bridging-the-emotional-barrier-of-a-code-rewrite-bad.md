# 跨越代码重写的情感障碍。

> 原文：<https://dev.to/andreicalazans/bridging-the-emotional-barrier-of-a-code-rewrite-bad>

作为开发人员，我们就像艺术家一样，我们为自己编写的代码感到自豪。你曾经花了大量的时间，在一个特性上工作，小心翼翼地测试和验证它的功能吗？如果是的话，那么有人可能已经告诉你改变它，如果不是完全改变，至少是部分改变。我也经历过这种情况。尽管经历了几年，但这种变化产生的压力和情感障碍总是巨大的。

这个怎么处理？毫无疑问，处理这种情绪是多么困难，这种失望、没有动力、不和谐的感觉。我们不能否认这样一个事实，我们是容易被篡改的情感生物。第一步，承认自己的情绪。想想它们，试着找出为什么你会有这样的感觉。第二步是发泄出来，试着用语言表达你的感受，讲述你是如何产生这种感受的。发泄时，人们通常会意识到他/她的情绪状态背后是什么，因为他/她听到了自己的声音。

## 有人了

找到一个理想的交谈对象可能很难。有一个朋友是个好主意，你可以对他敞开心扉，他不会对你有偏见，也不会被和你一样的感觉所陶醉。因为这个原因，你可能想避开你的同事，因为他们有相同的背景，他们可能会感到同样的痛苦。如果你发泄的对象激发了你的负面情绪，那也没用。

不要忽视你的感受。当一种感觉由潜在的创伤引发时，我们通常对此毫无头绪。发泄完就很容易不了了之。当我们试图理性地评价某样东西，但它毫无意义时，我们的大脑就会抛弃它。然而，不要忽视这一点，如果这种感觉不断回到类似的触发点，你应该寻求专业治疗师的帮助。

自我意识是处理我们情绪的一个很好的工具。爱自己，照顾好自己。感谢您阅读这篇文章，我是在面临一个由代码重写引发的情绪问题后写的这篇文章。我感到如此强烈，令我震惊。理智处理了自己的情绪状态后。我试着去想所有让我产生这种感觉的事情。在这个过程中，我发现了一些关于自己的有趣的事情，我相信这些事情让我在情感上成长了。希望这篇文章能激励你意识到自己的情绪并照顾好自己。
