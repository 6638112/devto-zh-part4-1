### [](#part-3-migrations)**第三部分:迁徙**

当您使用您选择的框架时，您不希望担心框架更新会影响您的代码。虽然在大多数情况下，你不会遇到从一个版本到另一个版本的很多问题，但保持你的手指在脉搏上是重要的，因为一些更新可能更重要，需要调整来保持事情的兼容性。

Angular [计划每半年进行一次重大更新](https://angular.io/guide/releases)。在任何主要的 API 被弃用之前，还有另外六个月的时间，这给了你两个发布周期(一年)的时间来进行必要的修改。

当谈到 Angular vs React 时，脸书表示[稳定性对他们来说是最重要的](https://reactjs.org/docs/design-principles.html#stability)，因为像 Twitter 和 Airbnb 这样的大公司都使用 React。在 React 中，通过版本升级通常是最容易的，像 [react-codemod](https://github.com/reactjs/react-codemod) 这样的脚本可以帮助你进行迁移。

在关于迁移的 FAQ 部分，Vue 提到如果你从 1.x 迁移到 2 ，90%的 API 是相同的。有一个[迁移助手工具](https://github.com/vuejs/vue-migration-helper)在控制台上工作，评估你的应用程序的状态。

### [](#components)**组件🏗️**

组件是所有三个框架不可分割的一部分，不管我们是在谈论 [Vue](https://www.codeinwp.com/blog/vue-ui-component-libraries/) 、 [React](https://www.codeinwp.com/blog/react-ui-component-libraries-frameworks/) 还是 Angular。组件通常获得一个输入，并基于它改变行为。这种行为变化通常表现为页面某些部分的用户界面的变化。组件的使用使得重用代码变得容易。一个组件可能是一个[电子商务网站](https://www.codeinwp.com/blog/best-ecommerce-platform/)上的购物车，或者是一个社交网络上的登录框。

在 Angular 中，组件被称为[指令](https://angular.io/api/core/Directive)。指令只是 DOM 元素上的标记，Angular 也可以跟踪和附加特定的行为。因此，Angular 将组件的 UI 部分分离为 HTML 标签的属性，并将它们的行为以 JavaScript 代码的形式显示出来，这是 Angular 与 React 的区别。

### [](#learning-curve)**学习曲线🎓**

那么学习这些框架有多难呢？

[![Alt text of image](../Images/1cce8d5975bb76244010ad7ff0ce7cae.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--umBVCKMx--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://iotvnaw69daj.i.optimole.com/w:30/h:52/q:auto/https://mk0codeinwp10tp0961a.kinstacdn.com/wp-content/uploads/2019/01/angular-logo-copy.png)

[Angular](https://angular.io/docs) 有一个陡峭的学习曲线，考虑到它是一个完整的解决方案，掌握 Angular 需要你学习像 TypeScript 和 MVC 这样的相关概念。尽管学习 Angular 需要时间，但在了解前端如何工作方面，投资是有回报的。

[![Alt text of image](../Images/73481db6486f5a50661c6251abf76925.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Pe1iyDEh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://iotvnaw69daj.i.optimole.com/w:30/h:47/q:auto/https://mk0codeinwp10tp0961a.kinstacdn.com/wp-content/uploads/2019/01/react-logo-copy.png)

React 提供了一个[入门](https://reactjs.org/docs/getting-started.html)指南，应该可以帮助一个人在大约一个小时内设置好 React。文档是全面和完整的，StackOverflow 上已经提供了常见问题的解决方案。React 不是一个完整的框架，高级功能需要使用第三方库。这使得核心框架的学习曲线不那么陡峭，而是取决于您使用附加功能的途径。然而，学习使用 React 并不一定意味着你在使用[最佳实践](https://americanexpress.io/clean-code-dirty-code/)。

[![Alt text of image](../Images/b491e5a0559020f63c19ff4fec10dc9b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--CmFXKuA_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://iotvnaw69daj.i.optimole.com/w:30/h:43/q:auto/https://mk0codeinwp10tp0961a.kinstacdn.com/wp-content/uploads/2019/01/vue-logo-copy.png)

[Vue](https://vuejs.org/v2/guide/#Getting-Started) 提供更高的可定制性，因此比 Angular 或 React 更容易学习。此外，Vue 与 Angular 和 React 在功能上有重叠，就像组件的使用一样。因此，从两者中的任何一个过渡到 Vue 都是一个简单的选择。然而，Vue 的简单性和灵活性是一把双刃剑——它允许糟糕的代码，使得调试和测试变得困难。

虽然 Angular，React 和 Vue 有一个重要的学习曲线，但它们在掌握时的用途是无限的。例如，你可以将 Angular 和 React [与 WordPress 和 WooCommerce 整合起来，创建渐进式网络应用](https://www.codeinwp.com/blog/progressive-web-apps-on-wordpress-and-woocommerce/)。

然而，它在过去的几年里做得非常好，成为 Angular 和 React 的有力竞争对手。这可能是阿里巴巴和百度等许多中国巨头选择 Vue 作为其主要前端 JavaScript 框架的原因之一。

然而，它在未来的表现如何还有待观察，人们有理由对此保持谨慎。如果你喜欢简单，但也喜欢灵活，Vue 应该是你的选择。

每个库都有自己的优点和缺点。基于你正在做的项目和你个人的需求，其中一个会比其他的更合适。在决定之前做自己的研究总是很关键的，尤其是如果你要从事商业冒险而不是个人项目。
