# 这些 HTTP 请求之间的区别

> 原文:[https://dev . to/flipped coding/difference-between-that-http-requests-2m 13](https://dev.to/flippedcoding/difference-between-those-http-requests-2m13)

大多数时候，你在网站上提交表格，你是在提出请求。每当你进入一个新的页面，你都是在提出一个请求。这些请求被放入一个头中，并发送给另一段代码，然后你就可以得到你想要的页面了。您可能在代码中的某个地方看到过 HTTP 请求，尤其是在使用 API 的情况下。

为了让您快速了解这些 HTTP 请求是什么以及何时使用它们，下面是一些最常见的请求。

## [](#post)岗位

这是最常用的 HTTP 请求方法之一，因为这是获取用户输入并将数据发送到服务器的方式。当一个用户创建了一个新帐户，这意味着你要执行一个 POST 请求。如果有人在你的网站上买东西，你使用的是发布方法。

从前端到后端的信息传输通常与 POST 方法有关。它主要用于 HTML 表单，但是你会经常以属性的形式用于 API。你必须小心发布请求。多次调用它们可能会对您的服务器进行多次您不想要的更新。

## [](#get)得到

GET 方法是另一种常用的 HTTP 请求方法。您可以使用它从特定资源请求数据。该资源可以是数据库、blob 存储或其他类型的数据资源。这就是如何根据用户拥有的权限或其他条件为用户加载页面。

此方法仅用于请求数据，而不是更改数据或添加新数据。如果你需要向用户显示他们的一些信息，你可以使用这种方法，但是你必须小心。不要对社会安全号码或银行帐户信息等敏感信息使用这种方法。那些信息可能会被一个坏人截获，然后你就有大麻烦了。

## [](#put)放

PUT 请求与 POST 请求非常相似。它生成新资源或更新现有资源。主要区别在于调用方法时会发生什么。与 POST 方法不同，多次调用 PUT 每次都会对您的数据资源产生相同的影响。如果多次调用 POST 请求，就有可能出现重复数据或不正确的数据。

如果一个资源，比如 blob 存储，已经有了您试图创建的数据的实例，那么您只需更新它。对于 POST 请求，您可能会得到一个副本，而不是更新的版本。当您对资源如何设置以及应该如何管理有更多了解时，PUT 请求会更好。

## [](#head)头

HEAD 请求类似于 GET 请求，只是它不返回响应体。这意味着您将无法看到请求的详细结果。您只能访问返回的标题。如果您试图决定是否要发出 GET 请求，这可能很有用。

HEAD 请求最常见的用途是当您试图决定是否下载一个大文件时。你可以做一个 HEAD 请求来告诉用户什么文件将被下载，然后基于他们的响应你可以运行 GET 方法或者什么都不做。这种技术可以为您节省一些带宽，尤其是当用户决定不下载文件时。

## [](#options)选项

如果您有一些现有的代码，并且不确定它支持哪些请求方法，您可以使用 OPTIONS 方法进行检查。当您发出这个请求时，您将获得 web 服务器将支持的其他 HTTP 请求方法的列表。如果您试图判断一个方法是否正在发出 PUT 或 POST 请求，这可能会很有帮助。

如果您正在使用 API，并且需要了解可以使用哪些方法，这也很有帮助。一旦您从服务器获得了批准的 HTTP 请求方法的列表，您就可以开始进行任何需要的更改。

HTTP 请求是我们一直在使用的东西，但它几乎是下意识的。我们知道如何使用这些东西，但了解一点它们背后的概念会让我们更容易欣赏它们或更好地使用它们。我认为对我来说最难理解的区别是 POST 和 PUT 之间的区别。
