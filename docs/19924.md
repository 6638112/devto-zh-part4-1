# 您可以从混合云实施中的这 5 个错误中学到什么

> 原文：<https://dev.to/chenshemtov/what-you-can-learn-from-these-5-mistakes-in-hybrid-cloud-implementations-486g>

## 概述

对于希望利用云的企业来说，迁移到混合设置是一个受欢迎的选择。这篇文章强调了混合云实施中的五个常见错误，有助于确保您的公司顺利进行混合云迁移。

然而，首先准确定义什么是混合云并理解选择这种形式的云计算实施的动机是很重要的。

## 混合云定义

混合云是一个集成的计算环境，结合了内部(私有)和外部(公共)云服务的使用。私有云只能由单个客户端通过安全连接进行访问和控制，并且位于企业防火墙内，但是，实际的物理基础架构可能位于内部数据中心或辅助位置。公共云是由第三方提供商通过互联网提供的计算服务和资源。

云服务的一些主要供应商包括:

*   AWS(公共)
*   Azure(公共)
*   IBM(私有)
*   VMWare(私有)

还有与这些供应商提供的服务相关联的第三方解决方案，例如 NetApp 的 Azure 存储解决方案，它为 Azure 云数据存储添加了更多数据管理功能，或者 Cloudhealth，它使企业能够通过自动化策略集中管理其 AWS 环境。

选择混合设置的动机包括:仅在需要时使用公共云资源可以节省成本，在两种类型的云之间使用基于策略的自动化可以轻松实现可扩展性，以及用低成本公共云存储补充内部数据存储的能力。

然而，由于这是一种相对较新的思考企业 IT 的方式，当您决定迁移到混合云时很容易出错。

## 常见的混合云错误

### 1。忽视安全性和合规性

由于确保公共云和私有云环境之间的无缝集成涉及到所有复杂的体系结构问题，安全性和合规性可能会成为事后考虑的问题。虽然云服务提供商通常负责环境的整体安全性，但是您(客户端)需要管理混合设置中的安全性。

未加密的数据和懒散的安全策略会在这种类型的设置中造成严重破坏，导致可能毁掉企业的数据泄露类型。此外，如果您不优先考虑对 PCI DSS 或 [GDPR](https://www.cloudindustryforum.org/content/cloud-and-eu-gdpr-six-steps-compliance) 等法规的合规性，您很容易失去对混合云中敏感数据的可见性，从而可能因不合规而被罚款。

确保安全性是您在混合云中的重中之重。考虑雇佣一个云安全专家来确保你从一开始就获得安全。始终对潜在的公共云供应商进行尽职调查，确保他们遵守保护您所在行业中敏感数据的任何法规。

### 2。同时做这件事

对于任何混合云实施来说，试图从最深处切入，将所有期望的工作负载和数据一次性迁移到公共云都不是一个好策略。谨慎的做法是通过识别最容易“爆发”到公共云的数据、应用程序或流程来分阶段迁移。您可以先做这些较小的试点项目，它们会让您有信心和知识在更大范围内推进混合云迁移。

每个主要的云提供商都有既定的迁移最佳实践。例如，这里有学习 Azure 迁移和 AWS 迁移的好资源。

### 3。选择错误的公共存储类型

有几种类型的公共云存储可供企业使用，选择错误的公共云存储可能会对您的混合云实施造成严重破坏。例如，您将需要一种高性能存储类型，如块存储，用于您决定在需求高峰期扩展到公共云的任何应用程序。选择缺乏所需性能的低成本存储类型将导致不理想的性能。

同样，在较便宜的冷归档对象存储已经足够的情况下，选择昂贵的存储类型可能会导致您浪费大量资金。为了避免这种错误，您必须了解可用的公共云存储类型，并将数据的特征与正确的存储类型相匹配。

### 4。未能正确监控云的使用

企业迁移到混合云的主要驱动因素之一是这种选择所涉及的成本节约。然而，许多组织错误地认为这些节省是理所当然的，并且没有充分监控云的使用。缺乏监控的最终结果往往是每月账单远高于预期。

无论是为工作负载选择不正确的实例类型、选择不正确的存储类型，还是让实例在不使用时运行，这些问题都会迅速累积，如果不监控您的云使用情况，很难发现这些问题。

为了避免这种错误，除了在您即将超出云预算时向您发出警报之外，还需要一个带有仪表板的集中式工具来监控公共云和私有云之间的云使用情况。

### 5。不考虑网络瓶颈

两种计算环境的融合会对您的网络产生巨大影响，而企业往往没有考虑到这一点。如果您正在从本地实施迁移到混合云，您可能已经习惯了局域网上的低延迟、高吞吐量。

然而，当在私有和公共云服务之间通过无线网络传输流量时，可能会出现网络瓶颈，尤其是当您开始更多地利用公共云时。必须通过评估您的无线网络、您的互联网服务提供商来为这种潜在的瓶颈做好准备，并通过使用多个 ISP 或升级您的现有合同来确保网络能够处理大量流量。

## 总结起来

要为混合云做准备，请确保您:

*   从一开始就优先考虑安全性和合规性
*   分阶段迁移
*   评估您现有的网络基础设施
*   充分研究可用的云提供商，阅读他们的服务水平协议
*   实施可监控云使用情况的集中式工具

避免这些常见的混合云错误，并为迁移做好适当的准备。通过这样做，您的企业可以最大限度地提高一切顺利进行的可能性，并且您可以充分利用混合云的优势。
