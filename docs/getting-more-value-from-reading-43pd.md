# 从阅读中获得更多价值

> 原文：<https://dev.to/lightalloy/getting-more-value-from-reading-43pd>

在“技术阅读”系列的最后一部分，我想分享一下帮助我更好地理解和记忆信息的方法。对我来说，从阅读中学习成为一名更好的开发人员与为考试而学习是不同的。读书的时候，我想把知识尽快用起来，而不仅仅是为了通过一次考试而复制它。
但是，我发现类似的技巧对这两种类型的学习都很有用。编程时，我们在练习的同时自然地记住了许多概念，但也有一些理论概念需要记住。

虽然我相信阅读本身是有用的，但遵循[主动回忆原则](https://www.speaktoyourmind.com/blog/active-recall)有助于我们学得更好。这个原则是反对被动复习的。当只是阅读和重读的时候，我们可以得到一种“能力错觉”。这些材料看起来很熟悉，但我们可能无法在必要时回忆起来。

以下是我使用的一些技巧:

## 记笔记

这是一个受欢迎的，但应该明智地使用。只是突出线条不会有太大帮助。回忆更有用，所以我尽量在读完之后用自己的话写出一章(或一部分)的主要思想。

## 回答问题

有些书在每章的结尾都有问题和练习。显然，不跳过它们是有帮助的。但是大部分书上都没有这样的问题，所以我一般都是自己做。有时我会把这个技巧和上一个技巧结合起来，写笔记作为问题的答案。制作抽认卡也是一种类似的技术，但它似乎对考试学习更有用，对自学来说则有些过头了。然而，它们在学习基础知识时会很有帮助，你可以随时将它们作为参考。

## 与他人讨论你读过的内容

即使是与你的伙伴、朋友或网上社区的简短讨论也会有助于知识的巩固。在讨论中，你也可能获得新的想法和见解。我在上一篇文章中提到过一个读书俱乐部，几个月前我组织了一个，虽然只有我们两个人阅读和讨论，但这仍然是一次非常有趣的经历，我从中受益匪浅。

## 研究更多的话题

对这个话题有多种观点是很有用的。虽然我发现打断阅读其他资料不方便，但我在读完一章后会做更多的研究。另一种方法是列出你想了解更多的最有趣的话题或概念，并在读完一本书后搜索更多的信息。

## 写下你学到的东西

这对我来说太难了！但是写一篇文章会迫使你思考和研究得更多。除了其他好处，比如帮助别人，它还能帮助你构建自己的知识。当然，我不建议你写下所有你读过的东西，但是如果你有一些新鲜的想法想要分享，不要犹豫。

## 不要忘记其他的学习方式(和练习)

人们普遍认为使用自己喜欢的学习方式会学得更好，但实际上并不是这样
你可以猜到我最喜欢的学习方式是阅读，但当我发现结合不同的学习方式会更有效时，我开始在学习中加入更多的听力甚至一些视频内容。我很难专注于视频，但即使我开始喜欢它。
我从芭芭拉·奥克利的 MOOC [“思维转变”](https://ru.coursera.org/learn/mindshift)和她的书[“数字思维”](https://amzn.com/B00G3L19ZU)中学到了学习风格

可能所有这些动作都显得过分和费时。我不会对我读过的每一本书和每一章都做这些。但是即使在最重要的作品中使用这些技术也会有所帮助。
