# [](#what-sparked-the-question)是什么引发了这个问题

我最近翻阅了《TED* 的力量》，这是一本关于人际关系中人类心理破坏性循环的寓言书。

起初，我对这些抽象概念感到恼火。由受害者、迫害者和拯救者角色组成的“可怕的戏剧三角”看起来过于整洁，以至于*实际上*无法描述现实。作者似乎产生了错觉，以为他们推开了窗帘，瞥见了真实的一面。

但是后来我意识到抽象只是抽象——帮助人们问和回答正确问题的有用启发。“我是否认为自己是受害者，这会对我的人际关系产生怎样的影响？我这辈子认不认这种循环？”

(旁白:承认受害者身份可能是有用的。据我所知，这本书深思熟虑地避免了对承认“迫害”的人的评判，而是专注于如何打破有害的行为循环。如果人们选择批判性地运用这本书的观点，那显然是不好的。)

# [](#the-question)提问

最近有人对我说，“一个开发人员对其他人的代码的第一反应是**‘这是垃圾，让我们全部重做吧。’**

尽管言过其实，但这句话击中了要害。默认情况下我会重构，我没有足够的工具来识别什么时候应该放弃不太完美的代码。

所以我想知道:**你如何决定什么时候*而不是*重构？你问自己或别人什么问题？你遇到什么样的「直觉」？**

我也对硬性规则感兴趣，但我怀疑许多读者手边会有更多近似的工具。
