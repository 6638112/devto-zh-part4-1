# 每日挑战#11 -立方数

> 原文：<https://dev.to/thepracticaldev/daily-challenge-11-cubic-numbers-21am>

欢迎来到我们每日挑战系列的第 11 天，希望你有一个美好的星期一。

在这个挑战中，我们将会看到 g964 的“隐藏的‘立方’数字”的修改版本。

#### 首要目标

> 这个挑战的第一部分是创建一个可以确定一个数是否是立方数的方法。如果数字被确定为不是立方，那么输出应该是`null`或者字符串“Unlucky”
> 
> **立方数**是*最多三位数*的数，这样他们位数的立方之和就是数本身。
> 
> 比如:153 是一个立方数，因为`1^3 + 5^3 + 3^3 = 153`。

#### 次要目标

> 如果您正在寻找更多的挑战，您可以尝试创建一个函数，可以找到一个字符串中的立方数。在输出中，数字应该按照它们在输入字符串中出现的顺序返回。
> 
> 例:
> `s = “aqdf& 0 1 xyz 153 777.777"`应回`“0 1 153 154”`
> `s = “QK29 45[&erui”`应回“不吉利”或`null.`

**注意:**在一个三位数或三位数以上的字符串中，该函数应该从左边开始以三位数为一组检查这些数字。在字符串“001234”中，该函数应该看到 001，然后是 234，对每个进行求值，然后返回找到的任何立方数(例如，“24172410”变成“241”、“724”和“10”)。
