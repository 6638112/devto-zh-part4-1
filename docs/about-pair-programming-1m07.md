# 关于结对编程

> 原文：<https://dev.to/flippedcoding/about-pair-programming-1m07>

结对编程是一种有趣的实践，越来越受欢迎。对于一些商店来说，这是他们唯一的工作方式。两个开发人员，一个屏幕，一次一大块代码。通常一个开发人员编写代码，而另一个开发人员观察正在编写的代码。它的目标是产生更好的代码和更易维护的代码，因为你必须让熟练的开发人员和特定的角色一起工作。

它是 XP(极限编程)方法的一部分，试图以更高的质量更快地驱动开发。你会看到它一直被用在敏捷环境中，很多开发人员真的很喜欢它。你不会独自努力解决问题，而是总会有另一双眼睛以稍微不同的方式看待同一件事情。这里有一些关于结对编程工作方式的独特之处。

## 它关注正在编写的代码

在很多情况下，你会看到有人在观察，有人在打字。当你们两个都在完成任务的过程中前进时，你们中的一个会发现另一个错过的东西。例如，您可能在键入时忘记初始化数组，而另一个开发人员发现了它。或者其他开发人员可能会问你为什么以某种方式编写代码。

你在写代码的时候要小心，因为确实有人在看着你。没有更多的空间给懒惰的代码，最好的部分是它让你的代码在未来可维护。你不需要一个特定的开发人员来解释一切是如何工作的，因为你有可读的代码。这也有助于将最佳实践放在你所写的任何东西的最前面。当你思考问题时，多一双眼睛可以帮助你写出更好的代码。

## 这与师徒关系或教学无关

这需要一些开发人员一段时间来掌握。两个开发人员处于同等水平，他们都有需要证明的有效观点。当你进行结对编程时，你并不是在一个指导的情况下。你们是两个开发人员，互相帮助为这项任务编写最好的代码。

在一起工作的过程中，你一定会从你的伴侣那里学到新的东西。请记住，这不是首要目标。结对编程的要点是让对方专注于任务，并在遇到问题时集思广益。更重要的是编写正确的代码，以符合团队实践的方式解决问题。由此产生的任何指导或教学都是一个很好的副作用，但这不是主要的焦点。

## 为结对编程培养正确的技能和心态需要时间

与另一个开发人员合作开发代码并不是大多数商店的工作方式。大多数地方会让你独自完成任务，如果你遇到困难，你可以寻求帮助。转向结对编程是一个巨大的变化，开发人员需要一段时间来获得结对工作所需的新技能和心态。没有什么是一夜之间发生的，会有一些艰难的时刻，但是一旦你度过了难关，你就会收获所有的好处。

一些组织花时间为开发人员如何结对工作创建规则。这就是结对编程中不同角色的来源，比如导航员-驾驶员角色。制定规则需要一些时间，开发人员提供一些信息也很重要。这些规则到位后，你会发现你的配对过程变得更加顺利。

## 当你的团队成功时，收益是巨大的

当你完成结对编程过程时，它就开始形成势头。越多的开发人员成对工作，代码就会变得越好，特别是当你使用测试驱动开发时。一起编写测试可以确保两个开发人员都理解手头的任务，这意味着知识是共享的。这一点在很多团队中被高度忽视。交叉训练是你从结对编程中得到的一个微妙的好处，它能带来巨大的不同。

真的需要一些时间来习惯有人看着你写代码，并在最好的实现上来来回回。大多数开发人员习惯于独自编写代码。我知道我是。从长远来看，您最终确实会学到一些实现代码的新方法，因为您已经实时地接触到了另一个开发人员的观点，并且有了合理的解释。

说实话，我一开始并不喜欢结对编程。我认为这很奇怪，很慢，代码审查和拉请求已经足够好了。我对它有了一些了解，受益也很大，但我仍然喜欢独自编写代码，以后再看。你怎么想呢?你们都喜欢结对编程还是更喜欢单独编程？
