# 不再有“新开源”的借口！

> 原文:[https://dev . to/arit developer/no-more-new-to-open source-excuses-1e E0](https://dev.to/aritdeveloper/no-more-new-to-opensource-excuses-1ee0)

作为一名新毕业的编码训练营的毕业生，我害怕开源。在 bootcamp 期间，我曾试图为几个著名的开源项目配置开发环境，但那是一次不知道如何处理这些来自深渊的错误的练习。所以我从开源社区后退了几步...

在 bootcamp 之后，我意识到我需要一种方法来继续与其他开发人员一起编码。我周围的社交圈里没有人对编码感兴趣，我多方面的责任也使我无法参与我所在城市的编码聚会。我决定冒险回到开源领域，碰巧遇到了 Dev.To。

(提示天使唱诗班...)

配置我的 DevTo dev 环境简直轻而易举。他们超级支持的社区在我遇到一些错误的时候帮助了我。我的 DevTo 开源经历在我的工作面试中成了热门话题，并帮助我展示了我与开发团队融合并发挥作用的能力。

我最近发现 DevTo 在 GitPod 中增加了第三个安装选项。我非常兴奋，以至于我决定创建一个关于使用 GitPod 的简短教程来开始 DevTo 项目。我真心希望能激励像我一样的代码新手最终进入开源领域！
