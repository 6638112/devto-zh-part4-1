# [](#scrum)Scrum

几周前我们尝试了看板，是时候尝试 Scrum 了！这两种敏捷方法的一个很大区别是在冲刺阶段工作。这意味着您要估计积压工作中作业单的复杂性，并选择冲刺阶段的工作单(考虑团队的速度和任务的复杂性)。scrum 中像站立和回顾这样的典型会议被称为仪式。

### 我们的 Scrum 方法

我们使用 [Trello](https://trello.com) 来组织我们的门票，每个功能都在 git 和 [github](https://github.com) 中有自己的分支。
我们一天举行两次站立表演(这在以前的项目中效果很好)。在每个 sprint 结束时，我们会举行一次回顾，展示团队的成就，并思考如何在下一次 sprint 中改进。我们与产品负责人(其中一名教练)一起审查我们完成的功能——这很可能发生在现实世界中，但对我们来说，这发生在生产中(这意味着 [Travis CI](https://travis-ci.org/) 可以提示 [Heroku](https://www.heroku.com) 从主分支自动部署代码！).

# [](#projects)项目

本周，我们在一个团队中用两周时间，在 [Rails](https://guides.rubyonrails.org/) 中构建了一个脸书克隆体。这是我们研究 Ruby on Rails 框架的第一周，令人大开眼界——Rails 基本上可以从终端用几个命令完成我们前 4 周学到的所有事情。先学了 Ruby，看了应用程序设计，巨大的文件树比它可能有的更熟悉，但是 5 天了，有很多仍然是一个谜(这是魔术)。周末的挑战是在 rails 中构建一个 Instagram 克隆，虽然我花了太多的时间进行前期规划，但没有足够的时间进行测试，我有一个工作应用程序，它的构建速度相对较快！
