# 倦怠如何危及你的 iOS 职业生涯和福祉

> 原文:[https://dev . to/essential developer/how-burnout-hasked-your-IOs-career-and-福祉-22l3](https://dev.to/essentialdeveloper/how-burnout-endangers-your-ios-career-and-well-being-22l3)

职业 iOS 开发人员普遍存在倦怠现象。然而，我们没有看到足够的谈话、讨论和内容来帮助开发者理解和预防它。

许多 iOS 开发者都没有意识到自己已经精疲力竭了。有些人甚至开始错误地认为“我一直处于的这种消极状态就是事物的本来面目。”

结果，他们最终会危及自己的幸福、职业和人生计划。

在本文中，您将了解到:

*   iOS 开发者倦怠的常见原因有哪些
*   倦怠如何对你的职业和生活计划产生负面影响
*   防止倦怠和体验充实职业生涯的行动

## [](#common-reasons-for-burnout)倦怠的常见原因

倦怠最近被世界卫生组织(世卫组织)列为一种职业现象。它被描述为一种慢性工作压力导致的综合症，没有得到成功的控制。倦怠的特征是精疲力尽、与工作相关的负面情绪和低生产率。

软件开发是一项创造性和耗费脑力的活动，因此工作超负荷是一个常见的主题。很难跟上不断的挑战、需求、短的截止日期和不断到来/变化的需求。

iOS 开发人员一直在与我们分享他们每天面临的各种专业挑战。我们反复听到这样的话，“我一直在工作”，“我希望我有更好的同事”，“我希望我有更多的时间和空间来交付更好的结果”，“我对我们交付的工作不感到自豪”，“我不觉得被欣赏”，“我没有时间学习”，“我没有时间做任何有趣的事情……”

如果处理不当，这些挑战可能会导致精疲力竭，并对他们的工作表现、整体职业发展和人生计划产生负面影响。然而，*很多 iOS 开发者似乎不明白的是他们倦怠的根本原因*。

导致 iOS 开发人员压力和倦怠的一些主要原因如下:

*   没有明确预期的不断增长的任务积压(即使团队生产了很多，他们也不觉得他们完成了任何目标/里程碑；缺少庆祝活动是一个明显的迹象)
*   同时进行太多的工作
*   错过最后期限的累积
*   由经理或非开发人员强制执行的不可能满足的截止日期
*   难以(或不可能)估计的肤浅和模糊的需求
*   非常漫长和紧张的迭代；通常每 3 到 12 个月发布一次新的应用程序版本
*   意大利面条式的代码和[糟糕的架构拖了团队的后腿](https://www.essentialdeveloper.com/articles/clean-ios-architecture-part-2-good-architecture-traits)
*   总体上缺乏自动化(测试、持续集成、持续部署...)
*   需要几天、几周甚至几个月才能合并的 git 分支(当开发人员试图合并它们时，工作已经不同步，导致代码和团队中的冲突)
*   缺乏适当和[持续的培训和教育](https://www.essentialdeveloper.com/articles/engineering-the-journey-of-continuous-learning)
*   在将应用推向生产或发布到应用商店时感到非常焦虑，因为可能会出现 bug、错误或只是做错了什么(例如，发布时使用了错误的开发端点配置)
*   [担心错过行业内的新发展](https://www.essentialdeveloper.com/articles/3-steps-to-prepare-your-apps-for-swiftui-combine-ipados-project-catalyst-and-any-other-leaps-in-the-ios-industry)或[被新技术和平台抛在后面](https://www.essentialdeveloper.com/articles/should-ios-devs-learn-kotlin-or-other-programming-languages)
*   [无法看到清晰的职业发展](https://www.essentialdeveloper.com/articles/why-ios-developers-feel-stuck-in-their-careers-and-what-to-do)；"现在情况似乎不错，但五年后会怎么样呢？"
*   没有被欣赏的感觉
*   在不提供职业规划的公司工作
*   欺负老板/经理/同事和充满敌意的工作环境
*   [频繁跳槽](https://www.essentialdeveloper.com/articles/why-switching-ios-dev-jobs-is-not-the-best-way-to-increase-your-salary)
*   糟糕的饮食选择
*   缺乏体育锻炼

可悲的是，这种高度紧张的情况可能会剥夺 iOS 开发人员享受和欣赏他们的贡献，并大大降低他们辛勤工作的回报。

## [](#how-burnout-affects-your-career)倦怠如何影响你的职业生涯

如果你能理解上述问题，或者你觉得自己也在为类似的问题而挣扎，你并不孤单。

多年来，我们一直在帮助 iOS 开发者实现丰富和充实的职业生涯。倦怠是我们看到的破坏职业计划的首要因素之一。

根据我们的经验， *[一个职业规划只有在你能长期](https://www.essentialdeveloper.com/articles/number-1-reason-why-you-dont-improve-as-a-software-developer)* 地遵循它时才是好的。

如果你制定了一个职业计划，六个月后就精疲力竭并放弃了，那是一个糟糕的——不可持续的——计划。

可能需要几年时间才能从疲惫中恢复过来，重新获得对工作的热情。没有适当的帮助，它实际上永远不会消失。因此，每当开发人员精疲力竭时，他们的职业生涯和生活计划就很有可能会倒退数年。

因此，长期收益的一个基本要素是可持续的计划。换句话说，你必须掌握能在短期和长期产生积极回报的技能和实践。

## [](#supercharging-your-skill-set)增强你的技能

我们面临过导致精疲力尽的挑战，根据我们的经验，这些挑战是可以控制的，也是可以预防的。事实上，解决这些挑战并帮助你的开发伙伴茁壮成长是你作为专业人员所能经历的最有价值的经历之一。

例如，当开发人员面对所谓的“意大利面条”代码时，他们通常会感到紧张，甚至恐慌，这些代码有大量的依赖项，却没有自动化测试。

如此脆弱的系统很难充满信心地改变。因此，确保您的贡献能够发挥作用，并且没有引入任何缺陷，当然是非常有压力的。此外，短的截止日期只会增加额外的焦虑。

解决这些挑战并不容易，但这是值得的(比仅仅应付要好得多！).

在这个例子中，解决方案是设计耦合度低的组件，并编写一套快速、可靠、全面的测试。

当然，如果你的同行不像你一样遵循或关心好的实践，你不能独自完成。软件开发是一项社会活动，所以健康和可持续的运营是团队的努力。

工作场所中冲突和挫折的另一个来源是，当开发人员付出如此多的努力来创建好的解决方案，而他们的同事却用糟糕的更改完全抵消了这些努力。

这就是为什么优秀的教育、培训、指导、高标准、团队建设练习、领导力和健康的团队文化如此重要。

这并不容易，但如果你和你的团队[致力于日常纪律](https://www.essentialdeveloper.com/articles/the-importance-of-discipline-for-ios-programmers)，这很简单。反过来，它可以大大减轻你的压力，帮助别人，为你的长期职业生涯扫清道路。

也许你在现在的工作环境中找不到快乐。如果你在和恶霸一起工作，也许是时候去找一个更好的地方了。但是如果你喜欢你的同事，确保开始*带领*他们走向更健康、更可持续、平衡和有益的工作生活会是一次有益的经历。

如果你没有这样做的技能，是时候开始建立它们了。你可以要求你的雇主/人力资源部门提供更好的培训，也可以利用你的资源这样做。无论如何，对自己进行投资以实现你的目标并体验短期和长期的职业成就感是至关重要的。

## [](#summary)总结

如果你感到疲惫、压力或焦虑，你不应该感到孤独。这可能发生在每个人身上。

然而，你不必接受这种消极状态为“常态”如果你觉得自己的压力和焦虑水平正在失控，你应该寻求专业帮助。

你可以通过建立有价值的开发和领导技能，将可持续的实践和健康的习惯融入日常生活来防止精疲力竭。

记住:交付你引以为豪的工作——同时帮助你的 iOS 开发伙伴——是非常值得的。
