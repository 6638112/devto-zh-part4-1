# 如果你能给你的袜子分类，你就能编程

> 原文：<https://dev.to/jenshaw/if-you-can-sort-your-socks-you-can-program-5cai>

### 编程慌！以及如何克服它

大多数人都会在某个时候被编程吓到并感到沮丧。

有一天下午，我发现我的一个同学深深地跌坐在座位上，他哀叹自己最近越来越不像一个程序员了。有一个他自己无法解决的编码问题，因为这不是唯一的一次，他开始经历典型的冒名顶替综合症。他觉得解决问题对他来说并不像对其他人来说那样自然，并且随着那些已经失败的尝试的记忆在他脑海中回响，他可能会怀疑这种职业转型到 web 开发是否是一个好主意。

当我听他说话的时候，我想到了我自己在编码方面的挣扎。在我投身编程之前，我担心自己也不是“天生的”。我记得在我 9 岁或 10 岁的时候，我妈妈失望地对我摇摇头，大声问我，一个不能像她预期的那样快速学会基本分数和小数的孩子是否还有希望或未来。整个高中时期，我都是一名数学不错的学生，在大学里，我几乎完全回避这个话题。那时，我已经接受了我根本没有数学头脑，而且永远也不会有，我决定任何涉及科学或计算机的职业道路都不会出现在我的生活日程上。

然后，惊喜！我去年开始自学编程，发现自己真的很享受。但是，当我环顾编码问题时，我很容易感到沮丧和失落。要么我不知道如何着手解决一个问题，要么即使我知道，在看了其他解决方案后，我觉得自己很愚蠢，因为总有人用更少、更简洁的行或“更聪明”、更有效的方式编写他们的算法。我用前 100 名的记分牌来衡量我的成功，给自己设定了不合理的高标准，这让我既没有更高的效率，也没有动力。这非常令人不快，与我同学的感受类似，我几乎认为我在追求这一点上犯了一个巨大的错误。

* * *

但是后来，我遇到了两条改变人生的建议，帮助我克服了这个障碍。

第一个问题来自互联网上一些很棒的人，他们提醒我们这些编程新手，解决问题应该和听起来一样。解决问题。只要解决方案*有效*，解决方案是什么并不重要。解决问题的方法总是不止一种，任何可行的解决方案都是可以的，因为它能完成工作。看到另一个程序员写这个让我觉得非常有说服力，它让解决问题的目标压力更小，更容易实现。

第二条建议极大地影响了我想象和学习概念的方式，这条建议来自于阅读芭芭拉·奥克利的 [Mindshift](https://barbaraoakley.com/books/mindshift/) ，她在书中阐述了在学习中使用隐喻的奇妙好处。她强调说，不管你在学什么，使用隐喻可以“让一个关键的想法变得生动起来”。有了隐喻和类比，概念变得更有意义，它们更容易记忆，更相关，更有形，也更容易使用。

受到这些优秀建议的启发，我认真努力改变我处理编码和编码问题的方式。因此，我向我的同学建议，也许他只需要在不同的背景下或从不同的角度思考编码问题，一个对他来说更相关和熟悉的角度。

* * *

很多人认为这是理所当然的，但是很难*去思考和解释每一个细节步骤，例如，对数组中的数字进行排序。另一方面，整理袜子是一个物理的、视觉的过程，也是我们很多人都非常熟悉的例行公事。如果你已经在生活中走到这一步，我敢打赌，整理袜子是你足够了解的事情，你可以教一个孩子。如果这是真的，你肯定可以处理为你的计算机键入一个算法来排序数字。*

我已经可以听到人们通过他们的监视器对我尖叫，“没那么简单！编程和袜子分类不是一回事！”我并不是说编程就像整理袜子一样简单，而是说使用袜子整理这样的隐喻可以帮助你思考一个抽象的问题，并赋予它一些有形的形式和清晰性。

隐喻和类比对我来说经常是最有效的学习和教学工具。当我看到一个只涉及数字的编码问题时，我觉得我必须更加努力地掌握所有的信息，并在努力工作的同时跟踪它。但是如果我能以对我来说有意义的方式与它联系起来，试图解决它就不会觉得是一项如此困难的任务。它真的为建立对手边概念的理解做出了惊人的事情。(另外，它也给了我一些很好的视角，让我了解如何处理现实生活中的其他问题。)

### 我们一起来整理袜子吧

[![](img/f570e6586b19c33b0b11547cb6a130b0.png)](https://i.giphy.com/media/nf0vI3nx69SwM/giphy.gif)

这里有一个来自 Leetcode 的[问题，几周前我的同学被要求考虑这个问题。](https://leetcode.com/problems/single-number/) 

```
Given a non-empty array of integers, every element appears twice except for one. 
Find that single one.

Input: [4,1,2,1,2]
Output: 4 
```

这个问题看起来超级简单，怎么解决也够明显了。但是如果你不能直接用手指指着答案，而是必须写说明才能得到答案，你会怎么做呢？

我会想象非空的整数数组是一个装满袜子的篮子。每只袜子都属于一双，除了一只孤独的袜子。一个再熟悉不过的情况。

所以，我们把这个问题重新写一遍来配合一下。元素名称会改变，但问题是完全一样的。

```
Given a basket full of socks, every sock appears twice, except for one. 
Find that single lonely sock.

Input: ['blue', 'striped', 'yellow', 'striped', 'yellow']
Output: 'blue' 
```

所以，现在我们正在讨论一个我们都知道和理解的现实生活中的情况，让我们考虑一下如何处理它。

1.  首先，我会确保我的篮子周围有足够的空间来放置每只袜子。
2.  我会从篮子里拿出一只袜子，记下颜色或图案，然后看看是否有一只相配的袜子已经放在一边。
3.  如果有匹配的，我会把它和那只匹配的袜子放在一起。
4.  如果没有，我会把它放在别的地方，等待它的对手。
5.  我会继续这个过程，直到袜子用完。
6.  而当没有更多的袜子时，我就看哪一堆袜子是成对的，哪一堆是孤独的。

完美！现在我有了一个游戏计划，我准备把它翻译成代码。

```
function findSingleSock(socks) {           // Mission: find single sock
  let lonelySock = "";                     // Target: the lonely sock
  let sortSpace = {};                      // Make space

  socks.forEach(sock => {                  // Go through each sock
    if (sortSpace.hasOwnProperty(sock)) {  // See if space has a sock with same property
      sortSpace[sock] += 1;                // If so, add to sock count
    } else {
      sortSpace[sock] = 1;                 // If not, create new pile 
    }
  });

  for(let pile in sortSpace){              // Go through each sorted pile
    if (sortSpace[pile] === 1) {           // If there's a pile with only 1 sockj
      lonelySock = pile;                   // Declare it the lonely sock
    }
  }

  return lonelySock;                       // Done!
} 
```

正如我之前说过的，解决一个问题有很多方法。也许我没有选择最快或最简单的方式，但这是我认为最现实的方式。我们现在不担心速度或效率。重要的是我们找到了那只袜子(或号码)，而且我们做得很好！

### 结论

人们一直在解决问题，不仅仅是程序员。无论是在学校、工作还是在家里，我们都会遇到一些需要解决的问题，或者至少是需要改进的例行程序或过程。如果你能解决或批判性地思考你生活中的日常问题，你当然能解决编码问题。仅仅因为一个问题涉及到代码并不意味着它必须是复杂的或者解决方案必须是奇特的。最终，所有的问题都有一个理想的结果，那就是*有效*的解决方案，不管它是什么，只要能完成工作。效率、速度，有时还有独创性都是加分项，这些都是随着实践而发展的技能。

关于效率和速度，还有最后一条建议，我在这里没有提到很多，但仍然值得一提。和其他人一起解决问题真的是你能得到的最好的练习。如果你不知道有什么选择和工具，就很难想出不同的解决方案。如果你能让自己接触不同的思维方式，你就能更快地达到解决问题的专家水平！
