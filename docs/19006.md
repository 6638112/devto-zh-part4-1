# 你有双重引导吗？

> 原文:[https://dev.to/ferricoxide/do-you-dual-boot-3cmk](https://dev.to/ferricoxide/do-you-dual-boot-3cmk)

另一个“讨论”主题让我想到了这个问题。对我来说，虚拟化是如此简单，我很惊讶地看到(尤其是)Linux 用户声称使用双启动而不是仅仅使用一些虚拟化框架。

我的意思是，游戏操作系统和非游戏操作系统之间的双引导仍然是有意义的。然而，如果您没有需要“裸机”访问的使用案例，虚拟化对我来说似乎是十多年来的发展方向。

那么，我想我真正的问题可能是，“如果您正在进行双启动，是什么驱使您这样做，而不是使用虚拟化解决方案？”
