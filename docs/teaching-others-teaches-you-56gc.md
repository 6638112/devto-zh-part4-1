# 教别人也会教你

> 原文：<https://dev.to/saulsilver/teaching-others-teaches-you-56gc>

我很幸运有一个我们经常招实习生的工作场所。今天是实习生的最后一天，所以我想写一写教学以及它如何对双方都有帮助。

一些人认为有实习生会浪费时间、降低代码质量、浪费资源等等。根据我的经验，我可以保证不是。

当然，我的陈述取决于许多因素；实习生的水平，导师的水平，工作环境，时机等。然而，我个人并没有看到这种情况发生。不久前，我偶然发现了 MPJ 的一个周一视频(如果你是网络开发人员，他的频道很棒)，视频[的主题是为什么教学是好的。另外，我读了一些支持他在视频中提到的内容的文章。](https://www.youtube.com/watch?v=RleN-6uMF04&t=94s)

### 将文章付诸实践

在我读到关于教学的内容之前，我对给实习生的建议非常谨慎。因为我想:

*   他们没有足够的知识/经验来判断我说的话(可能是错的),很可能会认为这是理所当然的
*   我觉得我对自己知道的东西(例如框架或算法)没有 100%的把握，所以最好不要提它

读完之后，我意识到我的想法有一部分是不正确的。

*   诚然，实习生还没有足够的工作知识/经验，这很自然。然而，这并不意味着他们不会评判我的建议。即使他们做了我建议的不正确的事情，它也教会了我和他们下次如何克服这个错误。
*   我对某件事的了解总是不完整，但这并不意味着我不应该提及它。大多数时候，这种情况都会发生。由于我第一次不正确的陈述，我们都获得了知识，这太令人惊讶了！即使没有人知道我说的是错的，这也会让我在说一些我不确定的事情之前三思，并且先做更多的研究。

### 如何传递思想

大声解释一个想法并不像独自思考那样容易，尤其是如果这种语言不是你/他们的第一语言。此外，思想可以在你的头脑中形成完美的意义，但却非常混乱。所以当你把它们说出来的时候，它们会按照你想的顺序出现，最终，这些单词对任何人都没有意义，可能对你自己也没有意义。

教学告诉我，当我得到一个问题，做一个快速处理的问题和答案，通过这样做

1.  确保理解问题，也许通过反过来问问题，这有时已经解决了问题
2.  找到答案，然后完成缺失的部分
3.  将答案分成要点，并按照逻辑/时间顺序重新排序(取决于答案)
4.  以一种可以理解的方式说话，适时的停顿，这给了听者消化你所说的话的时间。这取决于你在和谁说话，有多少人
5.  检查听众是否听懂了你说的话。这也很棘手，因为“清楚了吗？”提问可能毫无用处。我想到的一个窍门是用我的解决方案准备一个“问题”(我已经知道了答案)，大声讨论，然后看看听众是否会对此做出反应。如果没有反应，用不同的方法从步骤 1 开始重复。

当然，步骤 1-4 对我来说很棒，我设法找到了一个答案，增加了我的知识。但是第 5 点是最重要的部分，我在第 1-4 点所做的对听者来说是没有用的，除非第 5 点是好的。

这个过程只是一个大纲，它适合我，但不一定适合每个人。找到自己的方式，如果你已经有了，请在评论中分享！
