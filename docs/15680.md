# 在公共场合学习 DevOps

> 原文:[https://dev.to/quinncuatro/learning-devops-in-public-c26](https://dev.to/quinncuatro/learning-devops-in-public-c26)

嗨，我叫亨利·奎因，七月份我将“在公共场合学习”

上个月，我的好友亚历克斯·特罗斯特在阅读了王自如的帖子后，开始了一个公众月的学习。在短短的 30 天里，他真的用 SVG 动画提升了他的游戏(足够让他创建一个关于它的电子学习课程)，所以我认为每天学习和教授新东西肯定会有一些好处。

“亨利，在公共场合学习实际上意味着什么？”

问得好，读者。公开学习很大程度上意味着它听起来像什么。很多人会自己学习，和同事一起，或者在孤立的社区里学习。他们在提升自己的技能，这很好，但他们不一定会与更大的开发社区分享他们的发现。公共学习“运动”(lol)就是分享你所学到的东西。写博客，创建教程，或者制作视频来传递你所学到的东西。

冒名顶替综合症在我们的行业中是一个非常真实的事情，但重要的是要记住，不管你认为你实际上知道的有多少，总会有人比你知道的更少。他们也许能从你身上收集到一些激发兴趣并加速他们职业生涯的东西。天知道，如果没有纽黑文 IO 的一些人把我推向正确的方向，我现在就不会在 DevOps 领域。

所以说，多年来我一直告诉那些寻求技术帮助的人“如果你找不到博客帖子，那是你写博客帖子的时候了”…现在是我开始真正写博客帖子的时候了。

“这很酷，但我为什么要读你写的任何东西？”

再次，很好的问题。我应该告诉你一点关于我自己的事情。(我还会在本文底部放一个我的简历和 LinkedIn 的链接，以防有人好奇。)

我于 2015 年从尚普兰学院毕业，获得了计算机网络与信息安全专业的学位。我很快找到了一份工作，成为美国康涅狄格州地区法院的 DBA，在那里我自学构建 web 应用程序，并最终成为该地区的一名开发人员。

在构建了一些东西并被邀请在几次司法技术会议上发言后，我最终被华盛顿特区的美国法院行政办公室雇佣了一半时间来扩展我们为 ct 构建的产品，供全国法院使用。

我离开了那份工作，原因我将在本月晚些时候谈到(筋疲力尽是真实的事情)，并找到了一份很好的工作，作为 Clarity Software Solutions 的 DevOps 工程师。我担任这个新角色才五周，但目前为止我很喜欢。它是发布管理和过程自动化的结合。

我在这里学到了很多，我想和你分享我学到的东西。如果没有当地技术团队的人把他们的知识传授给我，我就不会有今天，所以这在一定程度上是我对这种善意的回报。我想把我学到的一切都记录下来，这样当我长大后就可以深情地回顾这些了。这也是我想做得更好的一部分。在公共场合学习的原则之一是教学是最好的学习方式之一。如果你一定要说我自私。:)

因此，对我来说，7 月将是 Bash 和 GoLang 深入研究流程自动化的一个月。然而，既然这是现实生活，事情有时需要改变。这个月，我发现自己需要转而学习更多关于 Kubernetes 的知识。我们正在开发一些非常酷的容器化工具，所以拥有一个可以托管所有工具并远程启动工作的平台将是一个很大的帮助。到这个月底，我希望有一个集群集来存放我们的工具，我对这些工具了如指掌。尽管如此，为了更好地衡量，肯定会有一些自动化投入其中。

综上所述，我认为这将是一个非常有趣的月份。我的计划是每天与你分享一些新的东西，我们可以作为一个社区一起交流和学习。

我明天会带着一个关于 DevOps-y 的新帖子回来，但在那之前，先按下“喜欢”按钮，点击“订阅”，保持冷静。
