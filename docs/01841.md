# 如何不学习 JavaScript

> 原文:[https://dev.to/maciekchmura/how-not-to-learn-javascript-i79](https://dev.to/maciekchmura/how-not-to-learn-javascript-i79)

几年前，当我开始我的程序员之旅时，我问了我的朋友(一个无名英雄，一个了不起的程序员)一个问题:

*   怎样才能学习 JavaScript？

他回答说:

*   你不想学 JavaScript，你想学编程。

我当时不明白。我想学 JS。我想学习 React、Node 和 WebGL。

然后我犯了很多错误。当我努力理解一些基本概念时，我遇到了一个又一个障碍。

我记得有一个特定的时间点，我的匿名朋友试图教我数据结构。当我们进入哈希映射时，我非常沮丧，以至于我把这个练习的文件保存为“有些愚蠢”💩。为什么有人会需要这个！！？？

我陷入绝望的深渊...

我是按照网上找到的一些不适合我的建议来到这里的。我不会说这是浪费时间，因为所有的经历都是宝贵的。但我可以更有效地利用这段时间。

以下是我对不要做什么的看法:

*   不做[降临的代码](https://adventofcode.com/)。你会不惜一切代价去完成任务，而不是学习一些好的实践。这会产生包含大量 if 语句的非常难看的代码。
*   不要读[雄辩的 JavaScript](https://eloquentjavascript.net/) 不要误解我的意思，这是一本好书，不适合没有 CS 学位的初学者。它试图通过解决具有大量抽象的复杂问题来教你 JS。
*   不要读[你不知道 JS](https://github.com/getify/You-Dont-Know-JS/tree/1st-ed) 。同样，伟大的书籍，但太复杂的开始。一年后再回来。
*   不要学框架。反应，Vue，角度。你需要知道这些框架首先解决什么问题。
*   不要在没有实践你所学的东西的情况下从一个教程跳到另一个教程。你很容易陷入观看源源不断的 YouTube 视频。也许第二天你会忘记大部分内容。
