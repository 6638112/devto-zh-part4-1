### [](#what-is-http)什么是 HTTP？

HTTP 代表超文本传输协议。这是万维网使用的协议，它构成了数据如何在互联网上传输以及服务器和浏览器如何响应各种请求的基础。从本质上说，它是“程序”,有助于定义互联网上的不同方应该如何交换数据或消息。

### HTTP 的问题

HTTP 的问题在于它没有提供完整性、认证和隐私。这意味着仅通过 HTTP 传输的数据很容易被拦截，并使您容易受到一系列攻击，这些攻击会危及上述 3 个因素(这是您在网上冲浪时最理想的情况)。

### HTTPS 怎么样？

HTTPS 本质上是在安全套接字层(SSL)上运行的 HTTP。HTTPS 的工作原理是在交换之前加密 HTTP 消息，当另一端收到消息时，它解密消息。

相比之下，如果使用 HTTP，敏感信息可能是信用卡信息、您的身份号码、登录凭证(如密码)等。将作为明文在服务器上传输。这使得交换易受“窃听”的攻击，这意味着攻击者可以截获消息并将其用于有害目的。

### SSL 证书是如何工作的？

*   当浏览器/服务器尝试连接到受 SSL 保护的网页时，它首先请求 web 服务器识别自己。
*   然后，web 服务器向客户端或服务器发送其 SSL 证书。
*   浏览器检查 SSL 证书，以了解它是否可以信任它。如果它能够信任 SSL 证书，它会向 web 服务器发送一条消息来确认这一点。
*   web 服务器现在准备好与客户机/服务器建立 SSL 加密会话，并发回数字确认。
*   现在，当消息或数据在这两个实体之间传输时，它们将基于双方同意的密码套件进行加密。

浏览器和 web 服务器发起安全会话的上述过程称为 SSL 握手。

*注意:*任何人都可以创建 SSL 证书，但它必须由可信的[认证机构(CA)](https://en.wikipedia.org/wiki/Certificate_authority) 进行数字签名。大多数计算机都预先安装了可以信任的 ca 列表，当浏览器检查 SSL 证书时，它会根据该列表验证数字签名。

### [](#public-and-private-keys)公钥和私钥

你可能听说过[非对称加密](https://en.wikipedia.org/wiki/Public-key_cryptography)，也称为公钥加密。在这种类型的非对称加密中，涉及两种类型的密钥:公钥和私钥。名字本身就说明了一切。公钥是公开的，即任何人都可以知道它们。私钥是私有的，这意味着除了您自己，没有人能够访问这个私钥。

现在，当有人想给我发送一些文档时，他们只需用我的公钥(任何人都可以访问)对文档进行加密。然后，他们寄给我这份文件。现在，我只需要用我的私钥(只有我自己知道)来解密这个。这确保了只有消息的预期接收者能够解密并访问该消息。

类似地，如果我向某人发送我的文档，我将使用我的私钥加密该文档。文档的接收者现在将使用我的公钥来解密它并访问该文档。这有助于识别文档*确实来自我。*

### [](#ssl-transaction)SSL 交易

为了建立 SSL 会话，我们需要 3 个密钥:公钥、私钥和会话密钥。如上所述，公钥和私钥可以有效地确保数据的可靠性和真实性。然而，这需要相当大的处理能力。因此，我们使用会话密钥来加密 SSL 握手之后的任何传输数据。
