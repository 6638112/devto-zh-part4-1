# 你好世界

> 原文:[https://dev.to/max_tulian/hello-world-e6p](https://dev.to/max_tulian/hello-world-e6p)

首先，我对开始写作和分享我的想法以及了解你的故事和经历感到非常兴奋。

我叫马克西米利亚诺·图连。我是来自阿根廷 UTN 的系统工程师。我二十六岁了。如果我必须用几句话来描述我自己，我会说:“我对学习和教学充满热情。”

四年前，我在大学读书，需要做 300 小时的督导工作。这个小时数将开启大学的最后工作。与此同时，在 Globant 有一个开放的房子。

我和一个非常亲密的同事去了开放日。我记得那个时刻，就好像是昨天一样。我们就像有新玩具的婴儿。活动结束后，我们讨论了成为 Globant 的一员，但没想到这对我们来说是可能的。我们就像战场上的其他人一样。

几个月后，我的一个朋友告诉我关于梦想公司的新兵训练营。你可以想象，我的第一反应是肯定的！我记得好像是，这是我的机会。我真的很想去。

但是，然后，回到思考。我懂编程吗？什么是 Web UI 开发者？我该怎么办？这是适合我的工作吗？…

我决定不加思索就开始学习这门课程。这很难。一天八小时左右。先阅读，然后做练习。开始的时候，和很多事情做斗争…但是后来，和自己做斗争。有时候，我确实有一种完美感，这可能是个麻烦。

完成新兵训练营后，我被认为是课程的“关键/明星球员”。(让我想想是那样)，啊哈。

我没有完成课程，但公司希望我尽快开始。这个项目是为美国一家重要的航空公司重新设计网页。有很多人参与其中。如果我没记错的话，该帐户有 500 多名员工。

技术堆栈非常棒。Reactjs、nodejs、gulp、sass、bem、mocha、chai、karma 等。这是第一个项目，第一家公司，许多事情中的第一件。那一刻我没有意识到，但这是一个非常非常非常好的开始。谢谢你的球。我还要感谢我的同事和朋友，让我认识了这个世界的 Andres Troiano。

责任如下:

1.  创建可访问的可重用组件(双 A)。
2.  为组件中的每个功能创建一个单元测试。
3.  遵循客户制定的几个标准。
4.  应用队友的反馈，并向他们学习。

这些组件被多于或少于五个单页面应用程序使用。此外，还有一个巨大的设备列表(平板电脑、手机、显示器，每一个都有不同的尺寸)。

开发过程被很好的定义了。到目前为止这听起来像一场梦，不是吗？

问题是我感觉一点也不好。我压力很大。我意识到这个机会既不是技术上的，也不是我同事的知识。这有点像自动驾驶仪，实现目标并要求自己做得越来越多。

最终，我将输入组件的性能提高了十倍。我认为我一点也不喜欢它。这是一个非常巨大的进步。

最后，这个项目完成了，我开始了另一个更加杂乱无章的项目。没有定义标准或流程的技术堆栈。完全混乱。我不希望你有那样的事情…

无论是第一个还是第二个项目都没有。是在第三部里。我意识到到目前为止我并不喜欢我的职业生涯。

那一刻，我正面临着一个充满压力和自我需求的时期。仍然不知道为什么，但正在努力。

我想爆发点是因为我感觉很糟糕，不知所措。然后我开始重新思考实现的目标。我不是超级英雄，但我认为我做得很好。最重要的一点是，现在我很享受我的工作和生活。

这个断点不仅是因为我想改变，也是因为一个更亲密的朋友告诉我有什么不对劲。

最后，我想和你分享我学到的一些东西:

> 别忘了深呼吸...抬头，社交，玩乐！分享的时候感觉更好。不要过分逼迫自己。当你感到放松时，最好的想法就会出现。
