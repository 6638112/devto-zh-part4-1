# 牛逼 Blazor 有哪些计划？

> 原文:[https://dev . to/adrientorris/which-plans-for-awesome-blazor-4f5k](https://dev.to/adrientorris/which-plans-for-awesome-blazor-4f5k)

两年来，我一直在管理一个 GitHub 库，名为 [Awesome Blazor](https://github.com/AdrienTorris/awesome-blazor) ，这是一个关于 Blazor 的伟大资源的集合，它是微软的一个开源产品，使我们能够运行。(是的，听起来很棒，如果你现在发现了这个术语，在读完这篇文章后，你真的必须检查一下)。

像 [Blazor](https://dotnet.microsoft.com/apps/aspnet/web-apps/client) 一样，两年前还只是一个个人降价文件的存储库已经成长了很多，作为使用和期待来自 Awesome Blazor 的大量资源和信息的社区，它的责任也与日俱增。此外，Awesome Blazor 是由微软从 Blazor 官方网站链接的，所以这个资源库肯定达到了一定的质量水平。

令人敬畏的 Blazor 已经达到了一个点，这使我想知道如何做好这一点，从长远来看。在过去的几个星期里，我想了很多，我正在考虑向资源库添加新的合作者，主要有两个原因。

首先，我真的很喜欢布拉索的精神，和所有的精神。NET 核心产品，它是开源的，非常受社区欢迎。这就是为什么我觉得作为这个资源库的唯一所有者有点令人不安，而且我觉得它将更多地拥抱。一旦它有了几个主人。

第二个原因是我并不总是有空，牛逼的 Blazor 应该有一些备份。例如，我在今年五月的最后两个星期休假，在一个没有互联网连接的地方(老实说，这是一个有点目的)，当我回来时，有 10 个拉请求和几个问题没有答案，因为 10 天了。我认为这是对那些想帮助我们建立一个更好的库的人和维护一些库或样本的人的不尊重，除此之外，这没有给 Blazor，甚至微软一个好的形象，他们不值得。相反，我不打算在这个项目上休息或变得不那么活跃，所以我觉得是时候打开这个回购协议，让社区完全参与进来了。

我仍然需要解决一些问题，比如“有多少合作者？”，“哪种类型的配置文件适合成为合作者？”这些新合作者的责任是什么？“，”在这一切中，我的位置在哪里？以及“将这个库附加到我的 GitHub 个人资料中有意义吗？或者将这个存储库转移到一个专门的组织更有意义吗？“，反正过程已经开始了。

两天前，我创建了一个问题并发布了一条推文，但我没有想到会有这么多兴趣，有几个人想参与进来，这是我没有预料到的。这些问题现在比以往任何时候都更紧迫，我会很快回答它们，所以这里有一点要记住:向社区开放 Awesome Blazor 的过程已经开始了！我希望这将有助于建立一个更好的知识库，帮助 Blazor 尽我们所能发展壮大，并关心它真正令人敬畏的社区。

干杯。
