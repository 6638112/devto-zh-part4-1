# 在主机上持续部署

> 原文:[https://dev . to/lautarolobo/continuous-deployment-on-hosting er-103n](https://dev.to/lautarolobo/continuous-deployment-on-hostinger-103n)

我找不到任何关于这个话题的好的指南，所以我自己写的！

首先，什么是持续部署？真正广泛的:你改变文件，测试它，构建它，然后自动部署(激活)。

听起来不错，不是吗？

让我们看看如何实现这一点！

> *本教程假设你在 GitHub 的知识库中有你的站点，如果不是这样，[提交给 GitHub](https://github.com/new) ，你就可以开始了。*

## [](#on-hostinger)在主机上

*   进入你的 cPanel >高级> Git。

*   添加您的存储库地址，那里有您网站的所有文件(图像，样式表，一切都必须在这个 repo 中)。复制并粘贴当你点击 GitHub 库的“克隆或下载”按钮时出现的地址。让其他事情保持原样，除非你知道你在做什么。

*   你可能在哪里读到过你的 public_html 文件夹必须是空的。绝对正确，在你的左边栏，进入文件>文件管理器。

*   在那里，点击绿色按钮。

*   现在把所有的都删掉(一定要有备份，以防万一)。

*   现在回到 cPanel 上的 Git，单击刚才添加的 repo 旁边的 Manage 按钮，然后单击 Auto Deployment。

*   Webhook 是一个 HTTP 回调(HTTP POST)，在特定事件发生时发生。在这种情况下，当您的回购发生提交时。还有[GitHub 指南](https://developer.github.com/webhooks/creating/)关于设置 Webhooks，但是我发现我的解释更加直截了当。在你复制了 Webhook URL 之后，让我们进入第二个简单的部分。

## GitHub 上的[](#on-github)

*   进入你网站的 GitHub 库，进入设置> Webhooks >添加 webhook。

*   在那里，将您从 Hostinger 复制的 URL 粘贴到*有效负载 URL* 字段中。在*内容类型*选择*应用程序/json* 选项，其他一切保持原样。改东西风险自担！(其实没有什么不好的事情会发生，只是，可能不会像预期的那样起作用)。之后，点击绿色按钮。

## [](#thats-it)就是这样！

现在，您可以通过 Git 在 Hostinger 上进行持续部署。
