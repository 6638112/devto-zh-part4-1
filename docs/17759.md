# 为什么我叉我找到的每一个项目

> 原文：<https://dev.to/triptych/why-i-fork-every-project-i-find-ijp>

当我遇到一个很酷的网站，或者一些有 [GitHub](https://github.com) repo 的应用程序时，我总是会做以下事情:

*   开始回购(显然)
*   找到作者->关注他们！
*   叉回购

最后一项可能会让你觉得奇怪。如果我不打算为代码库做贡献，我为什么要分叉它呢？答案很简单——以后找东西。假设你在网上找到了一些很棒的图形库。你在回购协议上打了星号，并在链接上加了书签。但是...它在哪里？你搜索网络，它迷失在时间和空间中。但是等等！你叉了回购！你去看看你的回购清单，嘣！你可能有一个稍微过时的代码版本，有到原始版本的链接。还有——不止一次——原始代码“不见了”。回购可能已被删除，或者用户不再维护它。你有一个副本，你可以在你自己的回购清单中永远使用。

因此，如果你偶然发现一个 GitHub 回购，你甚至有一点兴趣，总是叉它。和...您永远不知道您可能会受到启发，为代码库做出贡献！
