# 为什么以及如何使用 Twitter 进行社交

> 原文:[https://dev . to/ashleemboyer/why-and-how-you-use-Twitter-for-networking-357m](https://dev.to/ashleemboyer/why-and-how-you-should-use-twitter-for-networking-357m)

*免责声明:这篇文章不是为了吹牛。有人问我推特是否值得。*

我加入 Twitter 还不到 60 天。在我写这篇文章的时候，我有 358 个粉丝，关注着 460 个其他了不起的人，写了将近 1300 条推文。我的推文也获得了超过 60 万的浏览量。(我特别自豪的是，我的前两条推文占据了大约 20%的印象，它们都与网络可访问性有关。😊)

[![Screenshot of my current Twitter profile stats](../Images/09d3cc8130063b4cc425b25c1cbe2203.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--aaG2pWhu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/05kj0j110n03m6raxnhn.png)

加入 Twitter 也是我在 4 月份采取的最大行动，让我找到了即将开始的新工作。

# [](#why-should-i-even-bother)我为什么还要自寻烦恼？

Twitter 上有很多鼓舞人心的人在做大事。对我来说，看到科技领域的成功女性让我充满力量。如果你想找更多的人关注，这里有几个列表。

1.  [fempire](https://twitter.com/ladyleet/lists/fempire) ，作者[李美玲](https://twitter.com/ladyleet)
2.  [dev.to](https://twitter.com/dev_avocado/lists/dev-to) ，由[开发商 Avocado](https://twitter.com/dev_avocado)
3.  [科技女性大师榜](https://twitter.com/TheCodePixi/lists/tech-ladies-master-list)，作者[艾米莉·a·皮西](https://twitter.com/TheCodePixi)

Twitter 也是了解你的圈子之外的人正在使用什么技术或框架的好地方。我们很容易对我们使用的堆栈有狭隘的看法，因此看到其他人正在学习的东西*和*他们对此的看法是非常有启发性的。如果你不确定从哪里开始，这些是值得注意的标签。

1.  [#CodeNewbie](https://twitter.com/hashtag/CodeNewbie?src=hash)
2.  [#100DaysOfCode](https://twitter.com/hashtag/100DaysOfCode?src=hash)
3.  [#免费代码营](https://twitter.com/hashtag/freeCodeCamp?src=hash)

最后，Twitter 非常适合工作关系网和/或与你所在地区的人见面。正如我前面所说，Twitter 是我进入即将开始工作的公司的原因。在创建我的个人资料大约一个月后，我决定给另一位技术领域的女性发消息，告诉她关于雇佣关注可访问性的前端开发人员的任何想法。她给我介绍了我所在城市的一个开发商，然后他给我介绍了我未来的老板。说真的，社交媒体就像魔术一样。我不仅最终从这次交流中找到了一份工作，还找到了一些真正伟大的朋友，他们支持我，充满智慧。

# [](#ok-so-how-do-i-get-started)好了，那么我该怎么入门呢？

我最大的建议是:把自己放在那里。毫无歉意地真实。如果人们喜欢你，很好。如果没有，那就只有互联网了。你(希望)不只是为了给陌生人留下深刻印象。如果你没有伤害任何人，继续前进，做你自己。

我的第二条建议是:走出你的舒适区。与“著名的”人交往。提问。回答问题！我们都在这里学习。如果人们因为你是这个领域的新手或尝试新事物而让你感觉不好，那不是你的错。有很多支持你的人会支持你，不管你在职业或技能方面处于什么样的位置。

我的最后一条建议是:大声学习。起初似乎没有人在听，但这就是方便的标签的作用！人们*做*跟随那些。如果你用盖茨比创建了一个静态站点，@ them。他们很可能会转发！这只是获得一些曝光甚至反馈你正在做的事情的一个好方法。
