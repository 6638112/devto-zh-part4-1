# 你也为设计而挣扎吗？

> 原文:[https://dev . to/nanythery/do-you-dight-with-design-too-e32](https://dev.to/nanythery/do-you-struggle-with-design-too-e32)

自从我开始编码以来，我发现设计决策是工作中最难的部分。我的意思是，当然，你与你的代码斗争。你可能最终会想出一个很棒的应用或网络应用...但是当你访问你的 localhost:3000 时，一个悲伤的、糟糕的网站回头看着你。我觉得当你不精通设计的时候，那么辛苦的工作就不会以同样的方式被欣赏。

我喜欢纠结如何让某样东西看起来是我想要的样子。但是我还没有开发出任何做好设计的天赋。当我谈论好的设计时，我谈论的是什么？你知道:

*   不像 90 年代的东西
*   漂亮、干净、整洁的设计。
*   在他们应该在的地方抓住注意力
*   和谐而又新颖的色彩组合。
*   那种感觉就像每个元素都在它该在的地方！

我发现自己在代码行中更舒服，但我确信这是因为我没有足够的信心用自己的代码做出好的设计。但是，嘿，我也认为专业设计师真的是一笔宝贵的财富，也许做出令人惊叹的设计真的不是我的职责？这个不确定。你怎么想呢?
