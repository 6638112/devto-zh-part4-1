## **Wordpress 的移动应用开发 API&内容管理**

与 Roy Sivan 一起使用 WordPress API 的好处

罗伊·西万是华特·迪士尼的高级 WordPress 开发者。他是 WordPress 和在 SPA 和移动开发中使用 WordPress 的技术专家。

WordPress 仍然是 CMS 解决方案的有力竞争者。将应用程序从 WordPress 中分离出来，并允许那些在网站内容上工作的人构建材料，然后这些材料可以用来创建任何框架选择的应用程序，包括 Angular。

WordPress API 最近已经成为寻求开发移动应用的公司的热门选择。因为 WordPress API 与 WordPress 是分离的，所以您可以使用 AppPresser 从主站点提取数据，并将其显示在本地移动应用程序中。

如果你以前研究过 WordPress 并且担心安全问题，开源社区一直在努力解决这个问题。由于平台的巨大规模，安全性一直是 WordPress 的一个重要考虑因素，但由于最近对这一问题的贡献增加，现在有多种身份验证选项。

Roy 鼓励那些对 WordPress 感兴趣的人阅读新的开发者手册，以更好地理解定制的可能性。它是一个很好的资源，提供了深入的教程，指导您完成从高级定制到构建自己的 API 端点的所有工作。很有可能，无论你想用 WordPress 完成什么，都已经想好了，并且在文档中。

## n 升级

*整合 ngUpgrade 和 Angular* 的最新发展*与 George Kalpakas*

Angular 团队的不断创新最近为我们带来了 ngUpgrade。NgUpgrade 通过在组件之间启用升级(使 AngularJS 组件在 AngularJS 中可用)和降级(使 Angular js 中的 Angular 组件可用)来工作。ngUpgrade 项目的目标和愿景是让使用大型 AngularJS 应用程序的开发人员更容易升级到 Angular 的最新版本。要使用 ngUpgrade，开发人员只需将升级添加到他们的项目中，并使用一系列操作引导两个框架。

George Kalpakas 是 Angular 核心团队的一员，也是 ngUpgrade 项目的核心贡献者，他向我们讲述了这个项目以及 Angular 团队为开发人员制定的计划。

到目前为止，ngUpgrade 项目已经有了很大的改进。最近，重新编写了 ngUpgrade，纳入了 AOT 汇编。最近的另一项改进是改进 ngModel 和 control access monitor 之类的东西的互操作性，以便更容易使用组件和在 AngularJS 和 Angular js 之间进行插值。

创建 ngUpgrade 的道路并不平坦。团队在这个过程中面临的一些挑战是让两个框架互相了解。该团队目前面临的挑战是 ngUpgrade 的调试，以及解决用户没有正确配置 ngUpgrade 和遇到错误的挑战。

展望未来，ngUpgrade 团队正在开发文档和其他资源，以帮助人们开始使用 ngUpgrade。其他计划包括提高 ngUpgrade 应用程序的性能，并为各种用例提供更多工具和更多模板。

如果用户在升级过程中有问题或反馈，您可以通过 ngUpgrade GitHub 项目联系 George，Angular 团队一直希望听到更多来自用户的信息。

## RxJS

*RxJS 6 和 7，测试，以及 RxJS 5+的作者和项目负责人 Ben Lesh 的更多内容*

Ben Lesh 是谷歌的软件工程师，是 RxJS 5+的作者和项目负责人。他向我们提供了 RxJS 当前状态的更新。目前 RxJS (minified，bundled，gzipped)的大小是 33k。然而，Ben 创建了一个库，将 RxJS 的大小缩小到只有 3k！

为了做到这一点，他将所有的错误处理转移到一个中心点，这样 v8 就可以内联所有其他函数，而不必处理在函数中有 tryCatch 块时得到的非优化函数。

为了让它变得更小，Ben 不得不做出一些妥协，但是对这个库的新改进应该是大约相同的速度和小 10 倍。

虽然 RxJS 目前没有利用 Angular 的编译器，但理想的情况是获得 Angular 的模板编译器，并在构建时使用它来产生能够基于编译后的模板处理 DOM 中的更新的函数。

许多使用 RxJS 的人想知道 RxJS 的测试故事。RxJS 有一个很棒的测试套件，可以在一秒钟内运行超过 2500 个测试。不幸的是，它还没有为普通用户提供太多的实用性。测试套件目前在通常异步运行的事物上安排同步测试，但是以确定的方式运行。它基于这样一个事实做了一些假设，即它实际上只在 RxJS 的核心库中使用，但是正在做一些工作使它在外部更有用。在它成功之前，还有许多具有挑战性的问题有待解决。

测试故事的新更新包括 T-Rx 中测试套件的原型。Ben 告诉我们，他能够重写调度程序，这意味着他可以访问一个全新的虚拟调度程序，他将该程序封装在一个高阶函数中，使用户可以访问测试所需的一切。这将自动设置其他调度程序，以便它们以预期的方式与测试调度程序一起工作。
