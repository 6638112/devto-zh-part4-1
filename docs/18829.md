# 交叉发布到开发

> 原文:[https://dev.to/devdrake0/cross-posting-to-dev-33m2](https://dev.to/devdrake0/cross-posting-to-dev-33m2)

我专门为我自己的平台写我所有的内容。

这仍然是一个相对较新的网站，所以，为了让更多的人看到内容，我交叉张贴到开发。

到目前为止，我已经将全部内容交叉发布到 dev，如果我引用了以前的帖子，还会链接到我的网站。

我现在正在考虑改变这一点，相反，如果你想看的话，我会发布一篇关于这篇新文章的简短摘要，并链接回[代码提示](https://www.codetips.co.uk)。

最终，这使得原创内容获得了流量，希望我能让 CodeTips 获得成功。但我不想这样做，最后把所有人都惹火了。

因此，我有两个问题:

1.  我已经设法在 Dev 上获得了一个不错的追随者，所以我想假设我的内容是受欢迎的。这种变化会被我所接触的观众视为负面的吗？

2.  有人做过类似的改变吗？如果有，你看到了什么影响？
