# 我应该选择哪种语言

> 原文:[https://dev.to/xtabbas/which-language-should-i-choose-21kc](https://dev.to/xtabbas/which-language-should-i-choose-21kc)

嗨伙计们！因此，我开始开发一个新的应用程序，并想讨论我应该选择哪种编程语言来构建我的服务器端基础设施。

该架构将基于容器和 Kubernetes，以实现未来的可伸缩性。这个应用程序的想法非常简单，只有几个 CRUD 端点。它涉及视频存储、语音转录、视频处理和用于情绪和观点挖掘的自然语言处理，以及大量的服务器端文本操作。

现在我知道如果涉及到 ML，我可能会选择 Python，然后就到此为止，但这正是我想要讨论的。这些服务中有许多是由云解决方案提供的，我们希望在最初使用它们，然后在未来几年我们拥有大量数据集时构建我们自己的解决方案。

所以，我的问题是 Golang 在字符串操作和人工智能方面的可行性如何？我倾向于 Golang，因为它无缝集成在可扩展的云基础架构中。我听说了很多关于它超快和节省计算时间收费的云基础设施部署成本的好消息。

对于更多的上下文，我过去已经用 Node 和 Python 组合构建了后端应用程序，并取得了良好的效果，但它不是按规模构建的，而是一个概念，而不是一个有投资的成熟的初创公司。

期待您的宝贵意见。
