# 软件工程的禁忌

> 原文:[https://dev . to/bensassialadin/no-no-s-of-software-engineering-2pbf](https://dev.to/bensassialadin/no-no-s-of-software-engineering-2pbf)

在像软件工程这样压力大、期限紧的工作中，说“不”比说“是”要多得多。

尽管客户或经理对你的工作缺乏了解，但他们觉得有义务向你提供他们对技术细节和最佳实践的意见。这就像我在手术过程中走进医生的办公室，暗示说也许他的病人不需要新的心脏，他真正需要的是爱。

伪“摇滚明星”员工。典型的高材生在成长过程中受到了太多的表扬，现在他们觉得自己太优秀了，不需要遵循指示，因为他们编写了一个应用程序，可以根据你当天的穿着找到最适合你个性的冰淇淋口味，或者他们在大学时没有用的东西。然而，他们不明白的是，不管你有多聪明，你永远不会聪明到听取有经验的开发人员的建议并遵循它，错误的宏伟感会让你失望一辈子。

那些自以为是的人，因为他们是开发人员。这个社区最近变得越来越毒，因为有些人认为他们更聪明/更好/更明智，只是因为他们学会了如何组合代码行。你可以在 Quora，或黑客新闻，或 Reddit，或这些妄想狂经常光顾的任何其他地方看到这一点。带着键盘，喝着红牛和浓咖啡，他们在互联网上漫游，让人们知道他们的生活方式有多优越，他们的想法将如何改变世界，以及除了开发者之外的任何事情如何让你成为平民。我停止在 Quora 上回答问题有一年多了，因为它太多了，无法处理；大多数问题是这样的:“为什么没有更多的人成为软件开发人员？”或者“我怎样才能学会 web 开发，加入高手竞赛？”你的工作不会让你比其他人更聪明或更特别。

我不禁想，如果我心情好一点的话，这会不会更轻松一些。精神食粮。
