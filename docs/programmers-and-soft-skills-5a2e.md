# 程序员和软技能

> 原文：<https://dev.to/rlxdprogrammer/programmers-and-soft-skills-5a2e>

介绍

有哪些著名的程序员？大家都知道比尔·盖茨和 Linux Torvald。很多程序员也知道鲍勃大叔(罗伯特·c·马丁)。他们为什么会出名？他们技术上有这么深吗？当然他们在技术上很好，但是世界上有很多程序员在技术上很棒，但是几乎没有人认识他们。那他们为什么特别呢？他们知道一些别人缺乏的东西。这些就是所谓的软技能:良好的沟通、营销技巧、自我管理、时间管理、出色的表达技巧等等。这些技能让他们与众不同，这些技能让他们出类拔萃。而这些技能可以帮助每一个程序员获得更好的工作，做更好的事业，更成功。有一点很棒:你可以随时提高这些技能。提高这些技能是完全有意义的。这是成功之路。

# 如何提高软技能

正如我刚才提到的，你可以随时提高这些技能。一方面，大多数公司意识到这些技能的重要性，所以他们提供这些领域的培训。加入这些培训吧！

另一方面，你可以在网上读到很多关于他们的东西，或者你可以买一些好书。

但更重要的是:这些都是实用技能。光学理论是不够的。你必须在日常生活中使用它们。你可以一直注意尽可能多地练习它们。不仅仅是在工作的时候。你可以在家里和你的家人或朋友一起练习。

# 程序员最重要的软技能

## 时间管理

作为一名程序员，你必须回复邮件，参加会议，帮助其他同事，在截止日期前完成你的任务。这真的很有挑战性。要么你在加班，要么你开始使用一些时间管理技术，比如根据任务的紧急程度对其进行优先排序等等。没有良好的时间管理技巧，作为一名程序员是很难生存的。

## 自我管理

如果你想干一番事业，你需要能够为自己找到合适的职位，你需要看看你什么时候有可能证明你的技能，你需要知道什么时候是改变的合适时机。你需要向别人展示你工作得很好。大多数没有得到提升的程序员仅仅是因为他们杰出的编程技能，他们也能够把他们的工作卖给管理层。

## 沟通

编程不是孤独的狼的职业，即使它是几个人的事情。

编程是团队合作，为了能够在团队中工作，你需要非常好的沟通技巧。你需要理解其他人的问题，你需要向他们保证你的解决方案，你需要能够以非常清晰的方式传达你的状态。你还需要与你的经理沟通，在某些情况下还需要与客户沟通。用一种友好而清晰的方式与他们交流真的很重要。作为第一步，我建议学习自信沟通。

## 演示

作为一名程序员，特别是如果你是一个较高的职位，你必须做几个演示。这些要么是技术演示(一种培训)，要么是关于项目内容和状态的演示。这些演示应该以专业的方式进行，你需要学习它，它到底是什么意思。

## 压力处理

作为一名职业程序员工作压力很大。压力很大，因为严格的截止日期，因为开发团队内部或者团队和管理层之间的冲突。你需要能够以适当的方式处理这种压力，否则你要么变得好斗，要么变得抑郁，最坏的情况是两者兼而有之。这不仅会对你和你的工作产生负面影响，也会对你的家人和朋友产生负面影响。

## 基本营销技巧

很明显，如果你决定自己做生意，你需要一些营销技巧。但是作为一名开发者，如果你正在寻找一份新工作，你需要做一种自我营销。你基本上是开发商市场上的一个产品，你必须以尽可能高的价格出售自己。

# 总结

正如你所看到的，要成为一名成功的开发者，你需要几项软技能，但幸运的是，你可以很容易地学会它们，只需要一些时间。
