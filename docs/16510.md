# 万金油还是独一无二的大师？

> 原文：<https://dev.to/katieadamsdev/jack-of-all-trades-or-master-of-one-56b2>

总有一天，我会写一篇 Dev.to 文章，而不仅仅是将想法倾倒在页面上。然而，今天不是那一天！

我为自己是知识的海绵而自豪。然而，最近，我从四处阅读中抽出一点时间，尝试新的语言，给我的大脑一个受之无愧的休息站。想象我的大脑在某个温暖阳光的地方，穿着凉鞋的脚支撑在躺椅上，一杯冰冷的柠檬水让它的手因为冷凝而变得光滑。(不要质疑我的大脑是如何发展出四肢和明显有效的消化系统的。)

在给自己几天时间的时候，我脑海中出现了一个我一直无法摆脱的问题:所有这些额外的学习对我有好处吗？

我对面试实践和技巧了解得越多，我就意识到，当那些自称比我了解得多的人问我一些技术性问题时，我是多么地不知所措。一想到要去面试和被测试我的能力，我就非常紧张。

当然，这其中有些可以归因于冒名顶替综合症，但我也必须承担一定程度的责任。与两年前相比，我对自己的编程能力更有信心，但我仍然知道，在时间敏感的情况下，我可能不会表现得最熟练。

我也从来没有真正知道我想做什么。

例如，我喜欢视频游戏设计和开发——因为我喜欢故事、人物、艺术和音乐。我想成为把我最喜欢的游戏带到我面前的过程的一部分。

我也喜欢网络开发。当我设计了一个漂亮的、有反应的、易访问的、功能性强的网站时，我的感觉是什么？我想这是一只孔雀刚在当地的孔雀沙龙做了羽毛后的感觉。

软件工程也给了我极大的快乐。移动应用程序、网络应用程序、程序——凡是你能想到的，我都想知道怎么做！

我对创造、对制造东西有一种孩童般的好奇，并且能够说‘看，这是我做的！’

但是它在哪里结束？我在哪里停下来，意识到我必须满足于一个？

离我的“知识广度”被描述为“缺乏高级专业知识”还有多久？

有人遇到过同样的困境吗？不管是关于你还是你的同事、朋友或家人:我很想听听你的想法。不管你在面试台上的哪一边，你是如何处理这种类型的简历的？
