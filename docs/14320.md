# 雷达枪问题

> 原文:[https://dev.to/wuz/the-radar-gun-problem-3jk0](https://dev.to/wuz/the-radar-gun-problem-3jk0)

有一天，我参加完一个活动开车回家，在高速公路上出现了严重的减速。我以为前面发生了车祸或一些建筑事故，但当我靠近瓶颈时，我意识到唯一的区别是有一个警察站在公路边上，手里拿着一把雷达枪。仅仅是监视器的存在就降低了系统的性能。

那周晚些时候，我正在调试一些 Javascript 代码，遇到了一个我遇到过几次的问题。我有一个相当密集的过程，在很多值上循环。我采用标准方法来调试 JS 问题，并加入了一个`console.log`。我刷新页面，打开控制台...差点让我的浏览器崩溃。

事实证明，用密集进程调用一个`console.log`会占用大量内存。我以前遇到过这种情况，但这次交通减速真的让我想到了这一点。让我们来讨论一下我称之为 RadarGun 问题的东西。

在许多情况下，解决问题的方法是改变观察发生的位置。将`console.log`移出循环——无论是在循环之前还是之后——都是一个好的选择。你也可以改变观察的方法——放入一个`debugger;`并逐步通过它，引入一个有目的的瓶颈来防止一个意外的瓶颈。
