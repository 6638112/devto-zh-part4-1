# 我不知道我是什么水平的开发人员\_(ツ)_/

> 原文:[https://dev . to/Abdul isbomb/I-have-no-idea-what-level-developer-I-am-3n 16](https://dev.to/abdulisbomb/i-have-no-idea-what-level-developer-i-am---3n16)

这是我第一次在这个网站上发帖，但我真的很喜欢阅读其他人分享的内容，所以我想我也应该分享一下。

自从我作为一名计算机科学学生开始学习以来，我就一直在编写代码，说实话，我确实是这样......很糟糕。我有很多朋友很快就接受了它，这让我觉得我不适合它。我考虑退出这个项目，因为我已经失败了。出于某种奇怪的原因，我去找我的教授，恳求她让我留在这个项目中。令我惊讶的是，她又给了我一次机会，我努力和其他同学一起步入正轨。最终，由于许多不眠之夜，大量的茶，以及 YouTube 上一些伟大的人，我能够拾起这些概念，并以一些该死的好成绩毕业。

一旦我真的毕业了，我很快意识到我在学校学到的一切并不是我作为初级开发人员所需要知道的全部。我一直认为初级开发人员意味着你有一些学校项目，也许是一些你感兴趣的项目，以及学习的意愿。我没想到会看到要求 2-4 年经验、精通框架和水上行走能力的招聘信息。

[![Confused person](../Images/5f2c5972bd2ea3ceb2203e2a3f7c5b1e.png)T2】](https://i.giphy.com/media/lkdH8FmImcGoylv3t3/giphy.gif)

我已经离开学校快两年了，现在在 SaaS 的一家公司做软件支持(写作)💩大量的 SQL、CSS、调试 JS ),我觉得我作为一个开发人员成长了很多，但是，我仍然觉得我不知道我是什么级别的开发人员。我在网上阅读/观看了很多教程，我觉得他们中的许多人都是为了教授编程的基础知识，我觉得我很难理解知道基础知识的人和最初级的开发人员之间的真正区别。

在过去的两年里，我建立了一些项目，我甚至卖掉了我的第一个网站(在[https://montebellohistoricalsociety.org/](https://montebellohistoricalsociety.org/)有售)，这让我非常兴奋。我获得了一些很酷的新工具和框架。我最喜欢的是 Netlify 和 VueJS。尽管经历了这一切，我仍然觉得自己还没有达到初级开发人员应该达到的水平。在我职业生涯的这一点上，我知道我不是一个初学者，但我仍然不知道什么是真正的“初级开发人员”。
