液体错误:内部

Mary 是一个真正的 devrel 专家，她是比业内更多人更早看到 DEV 魔力的第一批人之一。

正因为如此，她是第一个在 DEV 上请求大量与 org 帐户相关的功能的人。为了满足玛丽的需求，我们基本上只是为她编写代码，并把它藏在条件句后面。今天，@peter 和她共进午餐，并告诉她我们是如何为她运送这个的故事。

我记得在她询问的同一天，我发布了这个功能，所以我加上了“你好，玛丽👋“作为硬编码的繁荣。

从那以后，我们发展了很多 org 特性。我们最活跃的组织是 Azure T1，我们一直都有新人加入，比如前几天的 T2 Angular T3。我们一直认为组织(本质上是社区内托管的工程博客)是生态系统的自然组成部分，但这需要像 Mary 这样的人来帮助实现。
