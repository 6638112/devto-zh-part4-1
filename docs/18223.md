# 让我们来谈谈微前端

> 原文：<https://dev.to/gsto/let-s-talk-about-microfrontends-5hao>

这似乎是 JavaScript Twitter 上的所有热门话题:

[https://twitter.com/rickhanlonii/status/1139323696953352192](https://twitter.com/rickhanlonii/status/1139323696953352192)

有些人一直在谈论全栈微服务，其中前端的独立部分通过缝合层连接在一起。例如，一个团队可以在 React 中开发一个组件，在 Angular 中开发另一个组件，每个团队都有自己独立的构建管道。

尽管它今天很热门，但这个想法已经存在至少一年了，正如 dev.to 帖子所看到的:

[https://dev.to/onerzafer/understanding-micro-frontends-1ied](https://dev.to/onerzafer/understanding-micro-frontends-1ied)

你觉得怎么样？我认为 dev.to 会在 Twitterverse 上引发一些更微妙、更有趣的讨论。
