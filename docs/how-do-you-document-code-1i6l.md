# 你如何记录代码？

> 原文：<https://dev.to/iamschulz/how-do-you-document-code-1i6l>

今天我和一位同事讨论了如何在一个大型 web 项目中记录代码。我更喜欢的方法是相信我的代码能够自我解释*它做了什么*，并依靠 JSDocs 或 block 注释来解释*为什么*它会做它该做的事情。粗糙或奇怪的行偶尔会得到行注释。
我不喜欢在其他地方记录我的代码(比如 wikis 或 tickets)，因为那会很快过时并被遗忘。

你是怎么处理的？你相信你的代码是自我解释的吗？你用 JSDocs/PHPDocs/etc 吗？你广泛评论吗？还是维护一个庞大的维基？
什么适合你，为什么？
