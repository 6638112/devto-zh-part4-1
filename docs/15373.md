## **棱角分明的社区**

**Angular 和 Rob Wormald 社区的现状和未来**

Angular 核心团队成员 Rob Wormald 讨论 Angular 的发展。

*从其他社区领养*

这些年来，使用 Java 和 ASP.NET 等技术的后端开发人员对 Angular 的采用越来越多。这在很大程度上可以归功于 TypeScript、Angular 的依赖注入和丰富的 IDE。由于 Angular 的 IDE，熟悉类型和依赖注入的后端开发人员在使用 Angular 时感觉如鱼得水；完整的语法突出显示、自动完成和其他功能。

*改进的语义版本化*

在 Angular 4 中，团队采用了更严格的语义版本，为平台提供了更多的稳定性。开发人员在升级他们的应用程序时不再担心代码被破坏。

新的 RFC 流程？

创建 RFC 过程允许开发人员提出一个想法，并请求对新特性的反馈。目前的价值是社区和核心团队的合作，他们将一起评估想法的可行性。

## **未来的棱角**

**Angular 的未来——沃德·贝尔访谈**

沃德·贝尔是 Angular 的谷歌开发专家，也是微软的 MVP。他向我们承诺 Angular 将会有一个激动人心的未来。沃德相信 Angular 正在(再一次)改变我们编写和设计应用程序的方式。

Angular 提供的新的编程方式和特性，包括 RxJS，正在开辟另一个可能性的维度。例如，使用基于推送的验证的反应式表单将会受益，并改变在线投标等应用程序的工作方式。

沃德对新技术的热情不止于 RxJS。他评论了 TypeScript 在提高代码质量和简化开发人员体验方面是多么有效。

Angular 的另一个重要里程碑是，开发人员已经开始将 Angular 视为一个平台，不仅用于 web 应用程序，还用于其他项目，如 WebVR 和 WebGL。

虽然像 RxJS 这样的新概念对开发人员来说可能很难理解，但随着时间的推移，他们将开始理解可观打开的大门，并重新思考他们认为可能的事情。

## **角形配合无功编程**

**Angular 核心团队成员 Kara Erickson 谈论用反应式编程改进表单**

Kara Erickson 是谷歌 Angular 团队的软件工程师。在这次采访中，她谈到了使用 RxJS 改进角形验证。这些计划中的改进将使反应式表单更加基于流，并为开发人员提供在 RxJS 中使用操作符的能力。

这种新的表单方法将支持基于推的验证，而不是基于拉的验证。实时验证和实时数据库管理都将得到显著的改进。

## **棱角分明的物质**

**Angular Material 核心团队成员 Paul Gschwendtner 对图书馆的改进**

Paul Gschwendtner 是 Angular Material 核心团队的成员，他向我们介绍了 Angular Material 即将发布的产品、新功能和改进。

Angular Material 为开发者提供了一个专门为 Angular 构建的 UI 组件框架。图书馆未来有许多改进计划。就在最近，发布了数据表和日期选择器模块。材料模块的弃用也是出于性能原因。

还计划对库的包结构进行更改。这些变化将更好地使库与 Angular 核心包保持一致，并允许 webpack 和其他捆绑器更加有效，消除深度文件搜索，并为树抖动提供性能改进。

## **组件架构**

**Justin schwartzen Berger 的角形组件架构**

AngularAir 的演讲者和主持人 Justin 讲述了组件架构和组件分析在 Angular 应用中的重要性。

当使用 Angular 构建组件时，在实现过程中经常会出现问题。许多开发人员和设计人员在试图理解整个组件的架构之前就开始构建了。

如果您将 Angular 应用程序想象成一棵大的组件树，那么您构建的每个新组件都会添加到这棵树上。了解每个组成部分继承的税收是至关重要的。每个组件都有有效负载和执行税。因此，开发人员应该意识到这样一个事实，即税收随着每个新组件的增加而增加。Justin 通过几种不同的方式来思考一个人的组件架构。
