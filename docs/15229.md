# 你应该总是参与其中吗？

> 原文:[https://dev . to/yanns 1/should-you-always-engage-you-you-elf-1 gab](https://dev.to/yanns1/should-you-always-engage-yourself-1gab)

我猜“深度工作”适用于我们生活中的许多领域，但我发现编程更重要。

我目前刚开始进入这个领域，就像每个“自学成才”的人一样，我必须找到一些东西来学习。好吧。我做了一些研究，输入了那句陈词滥调:“如何学习代码？”并按回车键...

我兴奋极了，伙计！这么多东西！我以为我会很容易成为天才。但现实并非如此。

我很快意识到，我不可能在那里学到所有的东西。于是我开始搜索最好的“re sources”(但总是免费的 re sources，这似乎有点违背之前的目标但我是一个少年，我没有钱可以随心所欲地投资)。

几天后，我不再关注任何资源，因为我仍在寻找最好的“资源”。

这对我来说是一个真正的问题。我倾向于总是在以下两者之间犹豫不决:“我应该深入这个问题吗？”或者“不，老兄，这还不算太糟”。

我读过《注意力饮食》( Attention Diet )( T1 ),我喜欢这种消费长期内容的想法。大多数时候，我们需要在一定的时间内几乎不专注于某件事。这可能是创造价值和技能的关键。

但是我纠结于这样一个事实，在某些时候，你只是需要特定的信息来达到一个目标。你不想从头到尾理解一个主题，只为一个信息花费时间和精力。

这就是为什么我最终会有这样的想法:
→要么你正在寻找的信息是一个精确问题的答案(例如“创建分支的 git 命令是什么？”:git branch
→或者你的问题需要广泛的信息来回答(例如“如何做一个网站？”:erghhhhh...)

事实上你会说:“当然。但是你可以把这个问题分解成更小的问题。”我会回答:“好的。但是你怎么能找到所有足够具体的问题，重新组合，允许回答最初的问题？”

当你不知道某事时，你不知道它。仅此而已。

所以，我想，我们需要有所作为。

## [](#conclusion)结论:

*   当你想回答的问题足够具体的时候，输入谷歌，得到你的答案，以便继续追求你的目标。

*   当问题很大时，选择最多 5 分钟(如果需要，设置一个时钟)来选择你的主题中看起来最适合你的“长篇内容”，然后不受干扰地深入其中。
