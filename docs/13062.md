# 参照系

> 原文:[https://dev.to/jondouglas/a-frame-of-reference-nnd](https://dev.to/jondouglas/a-frame-of-reference-nnd)

以软件专业人员为生有点像晚上开车。你只能通过你的前灯看到前方。无论这是你职业生涯的下一步，下一个语言/框架趋势，还是行业方向的转变，相关性就像卡车一样撞向我们。

我喜欢放在书桌上的一件东西是我女儿和妻子的照片，它们帮助我提醒自己我想要的生活。每当我思考下一件我应该做的事情时，不管是大是小，我都会看一眼这个相框，我会想起我为什么要这样做，并想出我下一步要做的事情。

当我着手下一步时，我会一步一步来。我喜欢提醒自己，有一小部分人**知道他们实际上在做什么。**那些真正知道自己在做什么的人，以前已经做过，并且知道什么对他们有效。对我来说，迈出第一步是最可怕的。它为我设定了一个方向，让我可以在那里茁壮成长。

我知道下一件事的第一次迭代通常是我最糟糕的第一次迭代。在这里，我让每个想法都在纸上倾泻出来，不管这个想法是好是坏。我只是想耗尽自己的想法，这样更容易专注于要做的实际工作。

我花了很多年才适应这一点。我终于知道没有人会看到这个作品，我可以在以后塑造它。任何好事都始于糟糕的第一次努力。你需要从某个地方开始，你可以从今天开始做一些事情...任何事...尽管它并不完美，但它的存在是可以接受的。

你和你糟糕的第一次迭代之间的主要障碍是完美主义。完美主义会让你完全止步不前。它会毁掉你的代码，阻碍你的创造力，扼杀你的创造力，最终毁掉你的生活。完美主义是我们防止自己受伤或失望的机制。

完美主义是很难克服的。在将某样东西公之于众之前，你会想确保你已经处理好了每一个细节。只要知道一旦你习惯了不完美的礼物，那么你就真正征服了它。
