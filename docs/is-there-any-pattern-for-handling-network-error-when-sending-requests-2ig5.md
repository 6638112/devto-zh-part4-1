大家好，很高兴你们回到 Dev。

今天，我想谈谈一些我认为从未尝试在真实应用程序中测试的东西。

假设我正在回复一封电子邮件。例如，我打开我的网络应用程序，登录 Gmail。

我的网络很好，我键入我的答案，随着时间的推移，我有一个网络故障。应用程序没有警告我，我按了“发送”。

# 你会如何解决这个问题？

一方面，经典的方法是捕捉 500 并通知用户暂时没有什么可以继续。

另一方面，Gmail 正在将请求排队，并将尽快发送电子邮件。

你觉得这件事怎么样？

*   在向服务器发送请求时，有没有一种好的方法来处理网络波动，让用户获得最佳体验？

*   如何在浏览器缓存中对请求进行排队，并在网络正常时发送它们？

*   你认为重试策略，比如 n 次重试然后失败，适合客户端到服务器架构吗？
