## 对话中不断出现的是什么？

出现最多的语言是普通 JavaScript 和 T2 类型脚本。在框架方面，已经有了一个长得多的列表: **Angular** ， **Ember** ， **Express** ， **Gatsby** ， **Next** ， **Node** ， **React** ， **React Native** ，以及 **Vue** 。相关的工具和应用程序包括 **Jest** 来测试 Vue 框架；Firebase 用于将他们的 JavaScript 应用程序连接到云托管的实时数据库；和 **Redux** 管理状态。

这让我想到了这些开发人员是如何在日常工作和兼职项目中使用 JavaScript 的。

## JavaScript 是如何被使用的？

以下是我遇到的一些用例，JavaScript 开发人员对它们都很熟悉:

*   用 Node 构建后端，这很重要，因为 JavaScript 本身是一种经典的前端语言，但 Node 在此基础上进行扩展
*   *创建 web3 区块链应用*使用 React 作为现有 web2 架构和以太坊等 web3 主干网之间的接口
*   *使用像 **p5** 这样的库可视化大量数据*
*   *用 Gatsby、GraphQL + **Gridsome** 或 React + Next 创建响应式或静态网站*
*   使用 React Native，*构建针对高流量&高 API 调用量优化的移动业务应用*
*   *JavaScript 中的函数式编程*与 **Ramda** 库
*   *构建实时应用*和无服务器应用

还有很多其他的。

## JavaScript 开发者为什么爱 JavaScript？:)

出于许多不同的原因，开发人员倾向于使用不同的语言。如果你正在为关键的汽车应用程序构建后端，为大型社交网络构建编译器，或者为金融科技初创公司构建移动前端，你学习和使用的语言会有很大不同，从 C++和 C#到 JS。如果你沉浸在 Kubernetes，你可能知道去。如果你的梦想是开发下一代移动应用，那么 Kotlin 和 Swift 就在你的菜单上。除了行业和软件类型之外，其他因素，比如个人偏好和一门语言的技术文档有多好也很重要。这可能是它自己的跨语言文章，但是这里是 JavaScript 开发者提到的他们如此喜欢它的原因:

*   JavaScript 是*易掌握*，*易搭建用*，*漂亮*
*   JavaScript 有许多框架，这让开发者在构建什么和如何构建方面有了灵活性
*   JavaScript 语言和框架生态系统*变化迅速*。例如， **React Hooks** [作为 React 16.8](https://reactjs.org/docs/hooks-intro.html) 的一个特性在五个月前问世，然而它已经为一些开发者提供了 Redux 的替代方案
*   如前所述，JavaScript 可以和像 p5.js 这样的图形库一起使用来表示大量的信息，这对于喜欢使用 JavaScript 而不是 Python 的数据科学家来说是很有吸引力的
*   由于世界上最流行的网络浏览器(Chrome)中使用的 **V8 JavaScript 引擎**被“优化得一塌糊涂”，JavaScript *在数十亿人使用的设备*上运行非常快。我还被告知，JavaScript *很好地处理了事件循环队列*之类的事情，这为 web 应用程序创造了更少的瓶颈，并吸引了注重性能的工程师和开发人员

因此，JavaScript 开发人员喜欢 JavaScript 的原因超出了语言本身:它不仅仅是易用性、灵活性和让用户耳目一新。这也是因为现代世界的许多技术都运行在 JavaScript 上，并且运行良好，在当今的网络互联时代，这种语言是构建的绝佳选择。这似乎是许多人对 JavaScript 看法的概要。

## JavaScript 开发者在说什么...

以下是我在以下方面遇到的一些观点和观察:

*   打字稿语言
*   JavaScript 友好的函数式语言 **Elm**
*   静态站点和渐进式 web 应用框架 Gatsby
*   UI 和单页应用程序框架 Vue
*   **测试的话题**

### [](#typescript)打字稿

我采访的一位开发人员提到，因为 JavaScript 最初构建得非常快——“一周就完成了”——所以它总是有点粗糙。所以“TypeScript 试图抹平 JavaScript 的粗糙边缘”。此外，随着越来越多的开发人员认识到 TypeScript 对他们的代码的重要性，TypeScript 也越来越受到类型安全倡导者的欢迎。尽管 TypeScript 是相当新的，仅仅在六年前发布，但它已经成熟到一些开发人员正在使用它和 C#等语言来构建复杂的系统，组装命令行界面等离线工具，以及用 Angular 创建高级用户界面和 web 应用程序，根据我遇到的一些例子。

### [](#elm)榆树

一些 JavaScript 开发人员认为，Elm 可能有一天会成为 TypeScript，就像今天的 TypeScript 对于 JavaScript 一样。从这个角度来看，像 Elm 这样的函数式编程语言给前端程序员带来了巨大的工作优势，因为它可以与 JavaScript 进行互操作，并可以用于为 web 浏览器构建高性能的图形用户界面(GUI ),也许是为了比使用 TypeScript 和 vanilla JS 更轻松的用例场景。因此，如果您希望构建明天的前端，那么今天就开始学习 Elm 可能是值得的。关于这一点，想要接受这个挑战的 JavaScript 开发人员可能会喜欢这个技巧:因为 Elm 启发了 Redux，所以先学习 Redux *再学习*可能会让新手更快适应 Elm。

Elm 开发者目前确实面临着一个挑战，那就是当其他语言和框架的社区正在经历快速增长的时候，Elm 社区仍然非常小。

### 视图

说到社区，Vue 框架的采用在全球范围内不断增长，在中国、东欧、法国和尼日利亚等地拥有大型用户中心。Vue 发展如此迅速的一个原因是，与其他单页网站和用户界面框架相比，它非常容易学习。Vue 社区还散发出一种包容、欢迎的氛围，在社交上围绕并吸引其在线和离线成员。这很有意义，对任何空间都是如此:当更容易得到答案并且开发者愿意帮助你时，开始创造和继续解决问题就变得更容易。

一些 Vue 开发人员还补充说，Angular 比 React 更难学，React 比 Vue 更难使用，这使得 Vue 成为一些人的默认最佳选择。有这种感觉的开发人员仍然称赞 Vue 支持高级用例，并拥有一些框架没有的功能(例如列表)。与我交谈过的 Vue 开发人员似乎非常看好 Vue 为他们的工作带来的好处。

### [](#gatsby)盖茨比

Vue 用于构建单页应用程序和网络界面，而 Gatsby 用于创建静态和响应性网站。Gatsby 的采用者说，他们正在用低计算开销、可靠的缓存和高性能来构建静态站点，度过了一段美好的时光。盖茨比还让他们更容易创建跨浏览器的渐进式网络应用程序。作为优秀的基于 Ruby 的静态站点生成器 **Jekyll** 的一个受人喜爱的 JavaScript 替代品(或类似物),它似乎正在赢得声誉。

### [](#the-importance-of-testing)测试的重要性

最后，还有一点与框架无关，许多开发人员会理直气壮地说，每个程序员都应该能够编写*可测试的代码*，并且能够证明他们能够这样做。这对于渴望领导开发团队的开发人员来说尤其重要，因为可测试的代码会导致更少的错误、退化和其他妨碍一致、可靠、可伸缩的用户体验的问题。换句话说，如果你想要代码运行良好，你也必须测试好代码。一般来说，花时间理解像*手动和自动化测试*这样的主题，用于两者的工具，以及如何用这样的工具编写好的测试可能是明智的。

相应地，对特定于 JavaScript 的单元测试和自动化软件测试知识的需求也在不断增长，因为随着 JavaScript 的发展，越来越多的 JS 开发人员在他们的职业中取得了进步。随着他们变得越来越资深，衡量他们的部分标准是他们是否能够编写和发布可持续测试和改进的代码。

那么 JavaScript 开发人员对测试有什么看法呢？能够使用 Jest 之类的工具来测试他们的 Vue 应用程序，并使用 Redux 之类的状态管理库来使他们的代码从一开始就更易于测试，这对他们的开发工作非常重要。他们的同行需要了解一般的*和* JavaScript 特定的单元测试、性能测试、自动化测试和 QA。不幸的是，我还听说，目前还没有与技术领域相同数量的高质量学习材料可供测试，如 **Docker** 、 **Kubernetes** 、云基础设施和 web 应用架构，但希望这种情况即将改变。例如，在过去的几周里，一些开发人员(在教育学习平台上我们称之为[作者](https://www.educative.io/authors))告诉我，他们计划创建完全致力于测试的课程。

想了解更多关于 JavaScript 测试的知识吗？看看[用 Jest](https://www.educative.io/collection/10370001/6751199023857664) 测试 Vue.js 组件还是[从头开始学习 Vue.js:构建&测试一个电影应用](https://www.educative.io/collection/5024793980043264/5629499534213120)能有所帮助。

## JavaScript 开发者说 JavaScript 开发者需要什么才能成功

我把我的分享中最好的部分留到了最后:世界各地的 JavaScript 开发人员认为他们的同行应该学习的东西，以便保持领先地位，促进艺术发展。

JavaScript 开发人员分享的第一个观点是，如果你从像 Java、Kotlin 或 Objective-C 这样的强类型语言切换到像 JavaScript 这样的弱(松散)类型语言，可能很难进行上下文切换。因此，JavaScript 的学习曲线可能会很陡，即使对于经验丰富的开发人员来说也是如此，所以投资时间来学习*如何*进行这种转换，并开发充分利用 JavaScript 所需的心智模型。对于 JS 新手和寻找新挑战的有经验的开发人员来说，这都是很重要的一点。

一些 JavaScript 开发人员也认为，他们那些来自非传统背景、没有接受过计算机科学正式培训就学习 JavaScript 的同龄人将会从花时间学习正式语言理论中受益。这里的观点是，自学 JavaScript 的开发人员在某个时候需要与正式语言概念进行斗争，之后他们可以将 JavaScript 推向其真正的极限，并在他们的编程生涯中更快更进一步。我喜欢这种观点，它尊重软件开发的内在实践和边做边学的本质，同时也承认概念理论可以帮助开发人员转变他们能够用自己的技能做什么。这似乎是一个平衡的策略。

第三点，也是更具战术性的一点，许多开发人员提到了理解和实现[承诺](https://medium.com/javascript-in-plain-english/promises-promises-promises-5ae43c26bb8d)的重要性，这些承诺是你今天可以编程的对象，能够在未来返回值。承诺的知识——它们是什么，它们如何工作，何时使用它们，以及它们为什么强大——已经被描述为每个 JavaScript 开发人员工具箱中的*必备*。

以下是 JS 开发人员认为值得学习并推荐给他人进一步研究的几个领域的例子:

*   *web 组件* & *web 套接字*
*   *渐进式网络应用* (PWAs)
*   *无服务器后端框架*(在节点中)
*   JavaScript 特有的*面试技巧*

### [](#the-fivefigure-salary-impact-of-strong-interview-prep)强势面试准备的“五位数薪资冲击”

我想强调最后一点。对于许多 JavaScript 开发人员来说，构建令人惊叹的应用程序并在他们的专业社区中获得知名度和尊重本身就是一种奖励，但是如果你花费了多年的时间努力工作以成为你所做的事情的佼佼者，那么你自然会希望自己的技能得到很好的补偿。随着 JavaScript 开发人员在职业阶梯上攀升，并开始接受多家公司的工作邀请，扎实的面试准备实际上可以产生很大的影响，不仅可以减少面试时间，还可以提高薪酬待遇和最终提供的机会。我直接从开发人员那里听说，这可以产生“五位数的影响”:因为你的面试失败，薪水会增加$$$。我们的课程[涉及编码面试](https://www.educative.io/track/ace-js-coding-interview)、[系统设计准备](https://www.educative.io/collection/5668639101419520/5649050225344512)和[数据结构和算法](https://www.educative.io/collection/5642554087309312/5663204961157120)白板练习，因此很受欢迎。

# [](#in-conclusion)总之

全球 JavaScript 景观是巨大的、多样的，充满了天才、想法和方法。这个领域的开发人员和工程师对于如何构建 web 和移动应用程序、使用何种方法和框架以及如何利用相关知识和工具来创造令人惊叹的东西有着不同的观点。但是和我交谈过的每一个 JavaScript 开发人员都有一个共同点:他们都热爱 JavaScript 本身，热爱不断的学习和实验，热爱帮助他人与他们一起成长。
