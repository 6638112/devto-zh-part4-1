# DEV 的工作列表如何改变了我对招聘的看法

> 原文：<https://dev.to/stevezieglerva/how-dev-s-job-listings-changed-my-perspective-on-hiring-df>

我最近用 DEV 的列表来帮助招聘。我们一直在努力寻找位于 DC 地区主要公司所在地附近的中级职位的候选人。我记得 DEV 创建了一个新的列表部分，并决定在那里寻找可能的候选人。这个过程让我看到了一些新的视角:

*   DEV 成员已经通过了我的第一关审核
    DEV 是一个很棒的社区。在我看来，会员们已经有了不断学习和实现他人在学习/职业成长中的价值的愿望。那些通过写博客做出贡献的人也表现出良好的组织和沟通技巧。

*   **初级也可以**
    我们本来是要找一个中级开发人员的。但是，作为一家咨询公司，ICF 是一个非常动态的环境，有各种各样的项目和机会供初级开发人员在工作中学习。我与我们的项目经理核实过，以确保他们在搬到 far 之前能够使用初级 dev。即使她的第一个项目没有使用最新、最前沿的技术，一个初级开发人员也可以学到很多关于软件开发、真实世界的项目和挑战，以及在团队中工作所需的软技能。

*   ICF 一直对远程友好，但通常是和以前在某个主要办公室工作过的有经验的开发人员(包括我自己！).我一直认为，在本地办公会给新员工带来一些安慰。即使你的同事可能是生态学家、癌症研究人员或社会工作者，他们仍然可以帮助新员工熟悉公司的组织和流程。虽然这可能有所帮助，但我不认为这是必需的。

总的来说，这是一次很好的体验，我很高兴 DEV 有了这个新功能。我真的很喜欢通过 Connect 的[联系人现在被默认勾选，以方便人们联系。](https://github.com/thepracticaldev/dev.to/issues/3088)
