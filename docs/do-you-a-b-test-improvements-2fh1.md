# 你 A/B 测试改进了吗？

> 原文：<https://dev.to/rick_viscomi/do-you-a-b-test-improvements-2fh1>

我很想听听社区的意见，你们都做了些什么来确保你们所做的改变能真正改善用户体验。

我特别好奇的是:

*   如果不做 A/B 测试，为什么不做？
*   你的开发是一个数据驱动的过程吗？
*   你使用服务器端还是客户端的 A/B 测试工具？
*   你用什么标准来判断实验是否成功？
*   A/B 测试如何帮助你理解网站变化的原因和影响？

我计划再写一篇关于 web 性能以及 A/B 测试如何帮助将 UX 指标的改进与具体的优化联系起来的文章。你的回答会对我有帮助！
