# 为什么 JetBrains 要创造 Kotlin？科特林创作内幕。

> 原文:[https://dev . to/Jay _ tillu/why-jetbrains-create-kotlin-the-inside-story-of-kotlin-creation-1135](https://dev.to/jay_tillu/why-jetbrains-create-kotlin-the-inside-story-of-kotlin-creation-1135)

你可以在 play store 上找到数百万个用 java 开发的应用程序。正如甲骨文所说 ***Java 无处不在*** 。你想从事移动应用程序开发，java 就在那里，用于服务器端开发的 java 就在那里，用于控制台基础应用程序的 java 就在那里，用于 web 开发的 java 就在那里，所以问题来了，我们为什么需要 Kotlin？所以让我们回到一切开始的时候。

## 科特林:不是产品而是问题解决方案…

* * *

虽然 JetBrains 在 2016 年正式发布了 Kotlin，但该语言面临着一个真正漫长的发展过程。科特林最早出现在 2011 年。当 IDE 皇帝第一次向世界宣布 Kotlin 计划时。尽管 Kotlin 如今被广泛使用，但 JetBrains 并没有将 Kotlin 用于销售。相反，他们让它解决自己的发展问题。

JetBrains 的开发者权益副总裁 Hadi Hariri 说，我们 70%以上的产品都是用 Java 编写的，其余的大部分都是用微软的 C#编写的。虽然 java 非常强大，但是使用 Java 意味着编写大量的模板代码。最终变得难以书写、阅读和维护。因为即使是最简单的代码也需要声明完整的类和对象等等，这让 JetBrains 开发人员非常头疼。他们只是想摆脱多余的重复性工作。

JetBrains 团队确实想使用一种更现代的语言，但是他们仍然有许多用 Java 编写的应用程序需要维护。将所有的 Java 产品重写为 C#或其他语言是没有任何意义的。因此，他们需要的是一种与 Java 兼容的语言，这样他们就可以保留所有的旧代码，并开始用新语言创建新功能。

他们还尝试了一些现有的选项，如 Scala、Groovy 和 Clojure。Scala 是所有版本中最好的，但是主要的缺点是，它没有 JetBrains 开发者所期待的那么快和简单。

所以这个团队决定创造他们自己的语言，拥有他们想要的所有特性。在创建 Kotlin JetBrains 团队时，我们主要关注三件事:

1.  快(至少和 Java 一样快)。
2.  与 Java 的 100%互操作性。
3.  简洁而富有表现力的代码。

JetBrains 没有在内部保留该项目，而是在 2012 年根据 Apache 开源许可开源了该项目。JetBrains 并没有直接从 Kotlin 在开发者中的使用中获利，但该公司希望从他们未来支持 Kotlin 的核心产品中赚钱。

2017 年 Google IO，Google 宣布 Kotlin 为 Android 官方支持的语言。从那以后，科特林的知名度与日俱增。Kotlin 也用于浏览器，未来通过 Kotlin Native，我们还可以创建跨平台的原生应用。

因此，Kotlin 的未来将令人惊讶地成形，如果您还没有，请查看并尝试一下。

请每天查看关于 Kotlin 和其他编程主题的新帖子。在那之前，继续编码，继续爱。
