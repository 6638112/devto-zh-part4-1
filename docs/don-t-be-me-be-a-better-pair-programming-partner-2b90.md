# 不要成为我。成为更好的结对编程伙伴。

> 原文：<https://dev.to/spencerlindemuth/don-t-be-me-be-a-better-pair-programming-partner-2b90>

最近，学校给我们分配了第一个自由形式结对编程项目的要求。回到小学，你将和谁配对的秘密在你体内燃烧。希望...还是非常希望*不要*跟身边的人成对。然而不像小学，你几乎不认识这些人。你才来了 2 周，每个人都被工作压得喘不过气来，而且时间还早，技能水平还相差很大。你开始思考，*“我从 2010 年大一开始就开始编程，已经写了数不清的基本 CLI 应用，这将是一件轻而易举的事”*但是你不知道你周围有谁有同样水平的经验，我当时并不知道，但这种思维方式是成为糟糕的结对编程伙伴的第一步。更好的结对编程的第一条建议是绑在一个栓柱上，在骑马的时候休息一下。

由于这个项目从一开始就显得如此有名无实，以及*“我是一个如此优秀的程序员”*，我花了很多时间思考我想在这专注的一周里完成什么。我发现了一个很棒的实时股票市场信息 API，它与我在空闲时间构建的一个应用程序想法完美地结合在一起。有了这种新发现的 API 和能量，我向我的合作伙伴提出了这个想法，他们同意了！完美！现在这些想法真的开始在我的脑子里打转。*“我应该先从什么开始呢？我将使用什么样的 gem 来将这些数据放入一个表中呢？在我们的程序中，实时股市数据的生命周期是怎样的？我到底需要什么？JSON？”...从来没有想过我的伴侣会有什么样的想法。*“股票是什么鬼？ETF？为什么有人想看一家公司 5 年的股票历史？”*。此时，我已经在脑子里写了 300 行虚构的代码，没有一秒钟去思考我的伙伴们发生了什么。更好的结对编程的第二个技巧是认识到你必须分享一个愿景来平等地为一个项目做出贡献。没有人会热心地帮助他们一直在搜索的东西。*

经过两天的疯狂编码，是时候开始把他们的 UI 挂在我的后端钩子上了，而且(没人会感到*惊讶！他们对我的宠物项目应该是什么样子的看法和我的不一样！(*喘息*)。*

“别担心，我能修好它！我今天会呆到很晚，做一些所谓的“微妙的”改变。”又是那个讨厌的“坏伴侣”的声音。更好配对的第三个秘诀是接受。从 2 号小费开始，3 号小费意味着你必须放手。如果你让程序看起来完全是你想要的样子，做起来完全是你想要的，整个概念完全是你想要的，为什么你还要有一个伙伴？接受你伴侣的观点也只是成功的一半。每个人都不一样。每个人的眼睛不一样，指纹不一样，笔迹不一样，编码风格不一样。真正接受你的伴侣(和人生伴侣，但那是不同的博客)是欣赏他们不同的做事方式。也许他们更喜欢 if/else 语句而不是 case 语句，或者他们更喜欢。atom to VSCode，但也许他们使用了一种宝石来制作您已经尝试格式化了 16 个小时的表格。

结对编程的目的是创造一种结果，这种结果是参与双方的表达，天生 make 更适合处理一项任务，因为它有 100%以上的脑力支持。代码的每一个变化都可能使你的视野提高 1%或 2%，但是程序的能力下降 5%或 10%。第四条建议是，不要害怕表达自己。你太棒了！给世界你所能提供的！但你的搭档也是！不要阻挡你旁边的人的光芒，因为你知道如果他们是撑着伞的人，那感觉不会很好。

让我们回顾一下:

1.  保持谦逊——今天聪明并不意味着明天聪明。

2.  分享愿景——朝着共同的目标努力比建立别人的目标更好。

3.  接受——你对他们来说是不同的，就像他们对你一样。

4.  合作——如果你有好主意，为什么还坐在这里看我的博客？

最重要的提示是**学点东西！**如果你没有从一个结对项目中感觉到你自己已经学会了一些关于编码*和*的东西，那么你的生活是完全错误的。

*   “如果你是房间里最聪明的人，那你就走错房间了。”
