# 如何用 GitHub 跟踪你的在线学习

> 原文:[https://dev . to/fmontes/how-to-track-your ' s online-learning with github-1 die](https://dev.to/fmontes/how-to-track-your-online-learning-with-github-1die)

我敢打赌很多时候你要解决某个问题你会想:**“我看了一些会帮我解决这个问题的”**但是你不记得在哪里看到的，也没有办法找到。

### [](#github-to-the-rescue)GitHub 来救援了

什么？GitHub？答案是肯定的。你可以创建一个 GitHub Repo，并使用这些问题来跟踪你在网上阅读、观看和收听的内容。

所以我开始使用 GitHub 的问题，根据官方信息是:

> 使用问题来跟踪 GitHub 工作中的想法、改进、任务或错误。

### [](#how-to-do-it)怎么做？

1.  创建一个 GitHub 账户(但我打赌你已经有一个了)。我的是: [fmontes](https://github.com/fmontes)
2.  [创建一个回购](https://help.github.com/en/articles/create-a-repo)，我叫:[学习-跟踪](https://github.com/fmontes/learning-tracking)。
3.  对于每一条内容，您都要保存[创建一个问题](https://help.github.com/en/articles/creating-an-issue)。
4.  差不多就是这样。

[![GitHub Issues List](../Images/ea931185756fd47764119511582aefd0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--3j_skFEv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/cjo5o054bph695f49yjf.jpg)

### [](#what-to-add-to-the-issues)要补充什么问题？

当我在网上消费一些内容时，我总是想记住某些部分，所以在我的问题中，我通常会添加:

1.  带有注释的代码片段
2.  文章引用
3.  带结论的说明
4.  当然是链接
5.  来源

#### [](#github-issues-support-markdown)GitHub 发布支持降价

这将允许您添加:

1.  代码片段
2.  评论
3.  引用
4.  链接
5.  形象
6.  桌子
7.  因此[比](https://guides.github.com/features/mastering-markdown/)多得多。

### [](#use-tags)使用标签

您可以根据需要为问题创建和分配任意数量的标签，您应该在问题中使用标签，以便以后进行筛选。

[![GitHub Issues Tags](../Images/7d336e73cd0c8ef8a0eac6ee5e0d053c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--0zaUUK2b--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/gpko613a8j3a2nairdwc.jpg)

### [](#search)搜索

你不仅可以通过标签进行过滤，GitHub 还有一个非常强大的搜索功能，可以在标题、标签甚至内容中找到关键词。不能再丢东西了！

### [](#conclusion)结论

我尝试了所有你能想到的笔记应用，但没有一个对我有用。通过这种方法，我看到了良好的效果:

1.  再也没有失去内容
2.  带注释的片段非常有用
3.  公开(你可以保密)让你的同事看到你在做什么
4.  提升你的开发者品牌，人们可以看到你在做什么😉
