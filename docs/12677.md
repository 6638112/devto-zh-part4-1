# 动机和开发者-背景是什么？

> 原文:[https://dev . to/rlxdprogrammer/motivation-and-developers-what-in-the-background-1 ob2](https://dev.to/rlxdprogrammer/motivation-and-developers-what-is-in-the-background-1ob2)

介绍

大多数软件公司正遭受着员工高度波动的困扰。这给他们带来了很大的困难，他们在不断招募新人和提高员工素质方面损失了大量资金。

另一方面，如果开发商找不到一个让他们感到舒适的地方，这对他们也是不利的。

所以情况似乎对每个人都不好，但是现在还没有解决的办法。

当然，我也不能解决它，但我花了一些精力来分析一下情况，以便对整个过程有一个了解。

当然，我的模型没有涵盖不同的人、不同的情况和不同的决定，但我试图涵盖最典型的职业道路。

这种分析的目的是帮助开发商尽快找到他们的最佳位置，并帮助雇主留住他们的员工。

这是根据我自己的经验编写的，但我也对其他意见感兴趣。

如果你开始在一个新的地方工作(大部分是新的公司)，你将以一个新手开始。这包括最初的几周和几个月。这是关于介绍你自己和探索你的新地方。在这段时间里，你与团队、你的新老板和你的新客户有了第一次接触，你必须展现出你最好的一面。另一方面，你需要了解新项目的工作流程和技术背景，并完成你的第一批任务。你需要证明你有足够的能力去那里工作。

这个阶段可以以三种不同的方式结束。

第一个是你或你的老板决定不再继续你的工作。如果你不能很好地适应团队或项目，或者工作场所不能满足你的期望，所以你决定离开，这种情况都会发生。

否则你就留在公司。但是你做这件事的方式有很大的不同。第一种方式是，你的老板觉得你比预期的要好。在这种情况下，你将成为焦点，你将获得几个成长的机会(新的责任，新的角色，等等。)，但这当然也意味着对你更高的期望。如果你接受了这个挑战，你就成为了一颗冉冉升起的新星:永远保持焦点，获得新的挑战和未来良好的职业机会。在每一个组织中都有这样的人，他们很快到达新的层级。但这种地位也有其黑暗的一面:大多数情况下，你会激怒已经在那里工作了几年、现在有这样机会的同事。

但也有可能是你“只是”满足了期望，你变成了一个完全普通的“灰色”同事。你是团队的一部分，你有所贡献，但在你职业生涯的这个阶段，你并没有真正向前迈进一步。暂时也没关系，因为在第一阶段你可以学到很多东西，你可以感受到缺乏高度责任感。有时候有机会从这种状态转变成一颗冉冉升起的新星，但这并不典型。通常只有当前的新星发生了什么事情(换到另一个地方等等)才会发生。).

过了一段时间，你会再次改变你的状态，对一些人来说，它会来得更快，对另一些人来说，它会来得更晚。从长远来看，成为一名冉冉升起的新星是令人疲惫的，而作为一名灰色同事，放松的魔力也不会永远持续下去。

从现在开始又有两个机会:找到你的舒适区或者变得不满足。最棒的是，同样的事情发生在冉冉升起的新星和灰色同事身上。

作为后起之秀，你会成长一段时间，但是一旦你达到你觉得承担更高的责任会不舒服的状态。对一些人来说，这是一个领导小型项目团队的职位，对其他人来说，这是一个首席执行官的职位，这取决于你的个性。在理想的情况下，你可以保留这个让你真正快乐的职位，这样你就可以在接下来的几十年里舒适地工作。

但也有可能(不幸的是，真的经常)你达不到这种状态，因为总是有越来越多的责任推给你(你可以保留对你来说舒服的责任)，你不同意公司的方向，或者你的薪水没有跟上你的新角色和新责任。当然，还有几个潜在的原因。在这种情况下，你将成为一名不满意的员工。

作为灰色同事，你也有两个选择:找到自己的舒适区，或者变得不满意。有几个人不想承担很高的责任，他们只是想做一份对他们来说还可以，不会给工作带来太大压力的工作。如果这样的人认为他们的位置很舒适(薪水是可接受的，项目满足他们的期望，有一个好的工作-生活平衡)，那么他们就找到了作为一个普通开发人员的舒适区。他们通常是过着紧张而平衡的生活的人，他们做他们的工作，但仅此而已，他们对此很高兴。

但是如果一般的开发人员想要达到更多(获得更高的责任等等。)，但是他们也没有机会变得不满足。如果其他条件不理想(工资、工作环境、工作与生活的平衡等)，也会发生这种情况。).值得一提的是，每个开发人员需要不同的动机。对他们中的一些人来说，工作与生活的平衡非常重要，对另一些人来说，拥有有趣的项目等更重要。

作为老板(团队经理等。)你需要了解你的员工和他们的工作动机。你还需要了解他们在职业生涯中处于哪种状态，他们未来的期望是什么。如果是这样的话，你可以制定一些措施来留住你的员工。

注意:如果环境在变化，舒适区的员工可能会变得不满意，也有可能发生相反的情况。但不幸的是，这并不典型。

从一个不满意的员工，有一条高速公路可以烧完。筋疲力尽对双方都不好:有筋疲力尽的开发人员是压抑的，消极的，没有效率的。他们只是让团队变得更慢，他们每天都在忍受上班的痛苦。从燃尽没有回头路。如果你到了这种状态，你能做的最好的事情就是尽快换个公司。但是最好在达到耗尽状态之前完成这一步。

离开公司后，你需要在一个新的地方重新开始，这显然比你在以前的地方开始更困难:你年纪大了，你的动力更少了，你想更快地完成所有事情，而这几乎是不可能的。

# [](#what-are-the-consequences-for-developers)对开发者有什么后果？

首先你需要确定什么样的职位能让你满意。也许你一开始就能发现，作为一个不需要承担很高责任的简单开发者，你是相当快乐的。在这种情况下，你有一个更容易的方法去你的舒适区。

在你工作的第一周和第一个月，你要以此为基础:如果你不想成为明日之星，就努力做到和其他人一样的水平，否则就尽力而为。如果在这段时间结束时，你已经看到它在公司根本行不通，不要犹豫离开，而不是很快达到精疲力竭的状态。

如果你正在成为一颗冉冉升起的新星，试着保持平衡，不要成长得太快，因为那样你就失去了到达舒适区的机会。而最难的:及时告知止损。

如果你是一个简单的“灰色”开发人员，只需要做预期的事情:不多也不少。试着为自己定位(项目、主题等)。)这让你很开心。

如果你是灰色开发商，如果你愿意，你仍有机会成为一颗冉冉升起的新星。在这种情况下，尽你最大的努力并保持耐心。

如果你到达了你的舒适区，试着保持它，试着避免那些会让你脱离这种状态的变化。

如果你变得不满意，让我们看看你是否还有机会找到你的舒适区。你可以和你的老板谈谈，你可以评估一下机会。但是如果你觉得这样的机会太低:趁早换个公司，不要浪费时间！

# [](#what-are-the-consequences-for-employers)对雇主有什么后果？

作为老板(团队经理、团队领导等)。)你需要保护你的员工不会达到精疲力竭的状态。如果他们没有达到不满意的状态，那就更好了，但是从那个状态还有一条路可以回来。如果你发现你的任何员工处于不满意的状态，支持他们找到他们的舒适区，以便能够把他们作为一个有价值的长期员工。

除了你的后起之秀，也要时刻关注你的“灰色”开发人员，理解他们的目标并支持他们实现目标。

如果你的后起之秀说“不”,接受他们的决定，这很重要！

也许还有最后一点:如果你在新同事面前发现这根本行不通，那就尽快辞职，否则你们两个都会有很多困难。在这种情况下，越早越好。
