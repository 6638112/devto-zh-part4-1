# 打破可访问性神话

> 原文:[https://dev.to/glitchgirl/accessibility-myth-busting-55hj](https://dev.to/glitchgirl/accessibility-myth-busting-55hj)

最近，我被邀请为一个当地的编码训练营做演讲。在这次演讲中，我讨论了许多关于可访问性和 SEO 的不同因素，我想花些时间来谈谈一些关于可访问性的神话。虽然有大量关于可访问性的错误信息和误解，但我还是要谈谈我职业生涯中遇到的三点。第一个是“残疾人不会使用我的网站”。其次，我听过“可及性丑”的神话。我也听说过“可访问性将使我的网站”这种令人困扰的说法。每当我听到这些——或主题的变化——我突然变成了困惑的数学女士迷因的迷因。这些神话表明，说这种话的人存在巨大的知识差距。通常情况下，我会尽量宽容地对待知识差距，因为大多数情况下，人们并不能真正帮助他们弥补知识差距。例如，我最近发现我对 web 浏览器实际工作方式的理解有很大差距，这对我的编码能力产生了负面影响。所以，一旦我发现差距，我做了研究，我觉得我的差距变小了。然而，当谈到可访问性时，它显示了一个更大的问题，即 web-dev/STEM 一般是如何教授的。如果只教你为健全人制造东西，你就不可能考虑为残疾人制造东西。应该在网络开发的基础阶段教授如何为每个人制作东西。这将有助于防止误解为每个人做网站。
这种知识差距的另一个问题是，这往往与能干主义密切相关。许多人认为，人们要么能够以“正常”的方式做事(即只使用鼠标操作电脑)，要么需要帮助。人们并没有想到能干主义的范围。当你使你的网站可访问时，你可以为每个人做得更好。慢慢被开发者越来越认可的一群人是视力有问题的人(盲人+低视力)。许多开发者意识到他们的网站需要对屏幕阅读器友好，他们的文本需要高对比度。然而，当你这样做的时候，你正在积极地影响你的 SEO*并使你的网站更容易被老年人阅读。因此，通过做出包容性的选择，你可能会扩大你的受众。
高对比度、易读、易导航网站的另一个特点是，它们倾向于更好的 UX/用户界面。根据我的经验，当你讨论让东西无障碍时，人们会立刻想到丑陋和笨重的东西，就像在现实生活中，他们想到的是拼凑起来的轮椅坡道。然而，好的设计包括每个人。这方面的一个真实例子是路缘上的切口，它们使轮椅使用者可以进入人行道，但也使推婴儿车或行李的人更容易。即使你身体健康，没有携带任何东西，走在人行道上也更省力。好的 UI/UX 也是如此。给你的网站一个清晰、易于理解的层次结构是一个好的设计，它让每个人都更容易理解，包括有学习障碍的人。

最后，我听到一些开发者抱怨说，让他们的网站无障碍会使它变慢。这是通常让我最困惑的一个，因为最容易访问的网站可能只是文本+ html，这将是闪电般的速度。通常，你的网站越不可访问，它就越慢，因为有额外的插件或脚本或华而不实的图片在运行。当你运行类似第三方弹出窗口的东西时，它会让你有能力的用户烦恼，它会搞乱屏幕阅读器，并且会让有学习障碍的用户困惑。发展中的许多其他黑暗模式也会有类似的影响。
总而言之，理解互联网是为每个人服务的，并据此设计你的网站通常会让你拥有一个更好的网站。让你的网站具有可访问性不会导致网站变慢。它不会导致一个更丑陋的网站。这也将增加你的潜在用户范围。遵循良好的可访问性实践也意味着你的网站对每个人来说都更容易使用。感谢你花时间阅读这篇文章，我希望你学到了一些关于 web 开发的新知识！

*好的 alt 标签可以帮助蜘蛛更好地阅读你的网站，并有望增加你的评分
