# 面试前的编码评估，太快了？

> 原文：<https://dev.to/garzo/coding-assessment-before-interview-too-soon-3epj>

最近，一家招聘机构向我推荐了一个 SRE 公司的职位。该机构向我提供了一个非常高层次的概述，他们在寻找什么，但没有详细的工作描述。此外，该职位没有在公司主页或任何招聘广告上刊登。这显然是“未上市”。

尽管如此，我同意提交的目的是进行讨论，甚至可能是一个介绍性的电话，我会与他们的招聘经理，一个担任团队领导的人，甚至是一名人力资源人员交谈，以了解更多关于角色，团队和挑战的信息，从而需要招聘一名新的网站可靠性工程师。

让我声明一下，我对技术评估并不陌生，对雇用有能力的技术人才作为有价值的贡献者的挑战也不陌生——在我的职业生涯中，我已经雇用、解雇、建立和培养了工程团队。

让我感到惊讶的是，第一次与这个组织互动时，我扬起了眉毛。没有一个电话是礼貌的自我介绍，没有一封电子邮件感谢我的申请被考虑，甚至没有一封对提交的简历的确认。

我收到了一封自动发来的电子邮件，邮件地址是 no-reply@ address，内容是关于一个域名的编码挑战，而这个域名甚至不属于我打算面试的组织。

这是我第一次与这家公司互动。

这封自动发送的电子邮件确实包含了一个直接发送给该组织招聘部门的地址，以防有人无法按时完成评估。我给这个地址发了电子邮件，感谢他们考虑我的申请，但要求有机会先与公司的代表或个人交谈，他们愿意更多地谈论工作细节、公司以及他们面临的挑战，我将为他们提供解决方案。我继续跟进，并保证一旦有机会，并且只有在有机会更多地了解我可能加入的角色和团队后，我才会乐意参与帮助公司评估我作为 SRE 的原始技术能力。

这是我对你的问题，开发者对社区:这是正确的反应吗？你会如何回应，你对在面试过程中，甚至在你未来的同伴联系你之前就使用编码评估有什么感觉？
