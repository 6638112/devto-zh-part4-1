# CSS 单位快速指南(像素、em、rem)

> 原文：<https://dev.to/prashantandani/quick-guide-to-css-units-px-em-rem-4lic>

下面是一些常用的 CSS 单元，以及它们各自的优缺点。

**像素(px):**
像素大小固定。1px =电脑屏幕上的 1 个点。这是一个绝对的测量单位

优点:
当我们想要开发像素完美的 web 应用程序时，像素是首选

*缺点:*
像素不能用来建立一个响应式网站。如果我们需要建立一个响应式网站，那么过度使用媒体查询将不得不使用，以实现一个响应式的设计，这可能会变得繁琐。这种方法也有助于减少视口的丢失。

什么时候用？像素派上了用场，尤其是当我们有固定的 div 时，比如页眉或页脚的高度保持不变，而与视窗无关。

**点(点):**

点数通常用于印刷在纸上的东西，即印刷媒体。
同一个像素。
固定度量且不可扩展。

**EM:**
EM 始终等于当前的字号。它本质上是可扩展的。

*优点:*
由于它是可扩展的，可以构建响应式网站。
如果字体大小等于 16px，那么 1em = 16px，0.5em = 8px，2em = 32px。

反对意见:
当我们有嵌套元素时，Em 会导致一个问题。

让我们在 100%有一个 16px 的根。假设有一个孩子应该有 12px，所以我们指定值为 0.75em。现在，如果我们在这个里面有另一个孩子，它将获得 12px 的 0.75em 的字体大小，即 9px。为了避免这种情况，我们需要再次指定它

`html {
font-size: 100%;
}
ul {
font-size: 0.75em;
}
ul ul {
font-size: 1em;
}`

当元素嵌套发生时，我们需要非常注意。

**百分比(%):**

百分比单位类似于 em。

*优点:*
顾名思义，百分之几的东西。如果没有指定，它将占浏览器宽度或高度的%。
对建筑流体布局有用。

*缺点:*
如果子元素的宽度是固定的，而父元素的宽度是%，那么会导致意想不到的布局。需要小心嵌套元素。
注意:当基础尺寸增加时，%和 em 自动增加，但 pixel 和 pt 保持不变。

**REM:**
代表根 EM。REM 仅与根大小相关，而 EM 与其最近的亲本相关。

*优点:*
Rem 总是考虑根的大小，而不考虑元素的嵌套。

*缺点:*
浏览器默认 Rem 大小为 16px。因此很难记住 rem 值。

`10px = 0.625rem
12px = 0.75rem
14px = 0.875rem
16px = 1rem (base)
18px = 1.125rem
20px = 1.25rem
24px = 1.5rem
30px = 1.875rem
32px = 2rem`

2.IE 10 中部分支持。

大众，VH:

这些单位分别是视口宽度和视口高度。1vw =视窗宽度的 1/100，1vh =视窗高度的 1/100。
是一个相对的度量。

注意:%和 vh/vw 之间的主要区别是 vw 单位根据视口的宽度来决定它的大小。浏览器计算浏览器窗口的视区大小，包括滚动条的空间，而用%表示，它只是 html 元素的宽度。如果我们设置 100vw，那么它将包括 html 元素+正文+滚动条宽度。

对于高度，如果我们使用%，这可能会导致一个问题，因为元素的%大小是由其父元素的高度决定的。只有当一个元素的父元素填充了整个屏幕的高度时，我们才能让它填充整个屏幕。
但是，
。测试{
身高:100vh
}

这里，无论元素如何嵌套，它都是相对于视口尺寸的。滚动条的问题不会成为问题，因为大多数网页没有水平滚动条。因此，在处理宽度时，单位%更合适。有高度，vh 单位更好。
