# 网络开发人员不断的学习周期

> 原文：<https://dev.to/bagwaa/the-constant-learning-cycle-of-a-web-developer-2boj>

我从 1987 年左右开始写代码，有一件事一直困扰着我，那就是我对学习的热情。我记得当我大约 10 岁的时候，我正在 Amiga 500 上学习 C，最近从我的本地[公共领域库](http://www.nostalgianerd.com/pd-software)购买了一份“North C Compiler”。

在互联网成为主流之前，公共领域图书馆是我们用来获取软件的方式，这些地方实际上只是中年人以很少的费用分发免费软件，以弥补他们分发的软盘的成本，这和基于 DOS 的公告板系统是 80 年代和 90 年代交换和购买软件的面包和黄油。

## 学习新的 web 开发堆栈和工具

作为开发人员，我们经常被要求学习一个新的堆栈或框架，我记得在使用通用 PHP3 多年后，我第一次看到 [CodeIgniter](https://www.codeigniter.com/) ，当时这是我第一次真正接触 PHP 框架，我被它让事情变得如此简单而震惊。

显然，回顾过去，很容易看到这些工具是多么简单和缺乏，但在当时，它们是领先的，并且是 PHP 社区中许多人第一次接触到 MVC ( [模型-视图-控制器](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller))等概念。

学习一门新的语言或脚本语言可能是一项艰巨的任务，但是我认为有一个学习某项技能的理由总是好的。大多数时候，开发人员倾向于发现自己处于这样一种情况，别人已经决定使用最新最棒的 JavaScript 框架，我认为这是我们最常学习的地方。

对我来说，这种方法是一种更被动的学习方式，是的，你需要学习，但你只是在你的工作或情况需要时才真正学习，这种方法没有错，我已经这样做了数百次，但我个人认为学习的方式有些不同，因此稍微改变了我的方法。

## 作为开发人员，我们为什么需要学习新技能

作为一名 web 开发人员，最重要的一点是你学习新技能的能力，我不仅仅指框架和语言，还包括一般的问题解决和沟通技巧，这些同样重要，但往往被忽视。

[![Books](img/9a13d5de881c443ae0d29b653fcd6027.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Jmv4LzhU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yi6osqbe4iy6ijl4962z.jpg)

web 开发世界的发展速度快得离谱，试图不断跟上最终会导致筋疲力尽，但由于我们周围的生态系统发展速度如此之快，所以学习新技能以在当今的就业市场中保持相关性非常重要，这包括全职工作、自由职业以及合同工。

作为一名全职自由职业者，最可怕的情况之一是醒来时知道自己的技能已经过时，在当前的市场上不受欢迎，这往往发生在开发人员被困在工作岗位多年，享受舒适和安全，而不是延伸和强迫自己学习的时候。

我工作过的许多角色都将促进自我发展作为过程的一部分，甚至为会议和学习材料提供预算，这是一个很好的开始，如果你有这个机会，那么你应该 100%地利用它，然而这只是开始，这是你需要建立公司学习文化的地方。

## 一种公司文化的学习

这是一个棘手的问题，想象一下这种情况，你为一家出色的公司工作，他们为你提供学习和会议的预算，但没有人有时间。每个人都觉得他们不能花两天时间去参加那个会议，因为这会使他们在交付新功能方面落后，或者更糟的是，他们的拉式请求会再两天不合并！

以我的经验来看，这种情况很常见，如今的公司好得多，都想做正确的事情，但公司内部往往有一种“努力工作，努力实现”的文化，很少有时间学习和自我发展。

在这种情况下，我认为你必须以身作则，并温和地说服其他人加入你，我记得有一次在一个大型手机零售商的大约 20 名开发人员的团队中工作，我设法说服我的经理产生预算，将我们所有人送到 PHP North West，在这 20 名左右的开发人员中，我们中实际上有两人利用了这一点。

这对我来说太疯狂了，公司愿意为他们的员工支付教育费用，但是没有人对此感兴趣，这是由于缺乏学习和自我发展作为公司文化的一部分造成的，这在很大程度上是一种推动代码、抛弃`divs`和努力工作的情况，在我的同龄人的心目中，根本没有任何学习的时间，即使是免费的！

## 我如何有效地学习

不同的人有不同的学习方式，但我认为我的方法对我的情况来说是两全其美的。我主要是一个视觉学习者，我订阅了大量免费和付费的在线学习资源，在那里我可以观看各种不同类型的截屏来学习一个新的主题。

我也喜欢阅读，当我想更深入地了解一个主题时，我倾向于阅读，所以作为一个例子，如果我想学习 Vue，那么我通常会[从文档开始(顺便说一下，这很好)](https://vuejs.org/v2/guide/)但是很快就开始学习一门课程，这样我就可以看到 Vue 的实际应用。

一旦我通过文档了解了一些非常基础的东西，并稍微玩了一下，我就会通过参加在线课程添加一点结构，这不是 100%必要的，但我觉得它帮助我更快地到达要点，因为内容通常更有结构。

我还通过其他人关于这个主题的博客帖子、播客、书籍甚至 twitter 来获取尽可能多的信息。

一个重要的方面是实际编码，看 8 个小时的课程完全没问题，但是如果你不应用它，你就不会记住任何概念。

## 学习概念而不是语法

我认为学习语法是次要的，这是你使用语言越多就会发生的事情，使用我的记忆来主动记住这些不同的语法变化对我来说似乎有点多余，主要是因为我可以很容易地在谷歌和文档中查找它们。

相反，我试图学习我所学的东西背后的概念，所以作为一个例子，学习 Vue 中的[作用域槽](https://vuejs.org/v2/guide/components-slots.html)或 PHP 中的[依赖注入](https://phptherightway.com/#dependency_injection)更多的是关于问题是什么以及我们如何解决它的概念，而不是关于记住语法的任务。

要记住的一点是，我已经开发和编程很长时间了，每天我使用谷歌，堆栈溢出并四处寻找答案或更优雅的解决方案，而不是专注于记住为什么事情是这样构建的概念，并在谷歌上或通过备忘单查找语法，直到它成为你肌肉记忆的一部分。

## 不要试图什么都学

这真的会让你绝望，试图学习 web 开发世界中的所有东西从定义上来说是不可能的，你绝对不能学习所有东西，我想这就是为什么我对术语“全栈开发人员”或最近的[“10x 开发人员”](https://twitter.com/skirani/status/1149302828420067328)有问题，这只是胡扯，并不存在。

不要试图学习所有的东西，选择一个你感兴趣的堆栈，并在你的学习计划中尽可能多地使用它，当然，在这个过程中学习其他不太相关的东西也不错，但我 100%建议坚持使用特定的堆栈。

对我来说，这个堆栈就是 PHP、Laravel 和 Vue——我发现这个社区很友好，我目前的大部分日常工作都在这里进行，然而这并不能阻止我不时地涉足 React 和 [Gatsby](https://richardbagshaw.co.uk/an-introduction-to-gatsby-js/) 。

[![Laptop](img/c03d6c5a797a854e266bdba18aa31db8.png)](https://res.cloudinary.com/practicaldev/image/fetch/s---EJCFjTM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6rza5x8fjqb6e0q4aqth.jpg)

我的观点是，不要给自己设定不切实际的期望，试图学习一切只会让自己失败，相反，专注或专注一点，每天都学习新的东西，你可以喜欢大或小。

这是一场马拉松而不是短跑，一致性才是王道。

## 不要拿自己和别人比较

环顾互联网，特别是 Twitter，很容易坐下来看，并假设其他人都已经明白了这一点，这就是冒名顶替综合症的由来，突然你发现自己怀疑自己与他人相比的能力。

我 100%做到了这一点，我们真的不应该和其他人比较，你在互联网上看到的大多数情况下是好的部分，你没有看到的是其他开发者经历的痛苦，让他的知识达到现在的水平。

我认为知识是一个旅程，而不是一个数量，如果有人在某个特定的主题上比你知道得多，这并不是说他们比你聪明，他们只是在这个旅程中走得更远，你可以很容易地在同一个地方通过每天学习一点来前进。

这也适用于你可能领先的人，如果我知道一些你不知道的话题，这只是意味着我在这条路上走得更远，你可以投入更多的学习时间，轻松超越我的知识。

所以，不要在当下拿自己和别人比较，而是专注于自己的学习之路，要知道你可以学到任何东西，只是需要时间和练习。

## 支付教育费用是一项巨大的投资

我知道很多开发者完全反对为课程付费，虽然我知道他们中的一些人可能非常昂贵，但现在可能只需 15 美元就可以购买一个课程，然而即使对一些开发者来说，他们也很难以这种方式放弃现金。

作为一名自由职业者，甚至在我成为自由职业者之前，我会以看待我的业务的同样方式来看待这个概念，我本质上是在出售我的专业知识以换取某种形式的报酬，所以我知道的越多，我就能为我的客户提供越多，所以如果我花 15 美元参加一个 30 小时的课程来学习 Vue，那么我认为这是一个巨大的长期投资回报。

当然，不是每个人都有能力支付教育费用，我理解这一点，但是你可以从大量的免费资源开始，在线文档和博客。

根据我的经验，付费教育有点像快速跟踪知识，当然有一些优秀的免费资源可以学习大量的东西，但一般来说，我发现付费课程在大多数情况下更加结构化，并有一套学习成果。

## 睡眠充足

作为一个 3 岁小孩的父亲，我可以 100%地说，睡眠不足会极大地影响你有效学习的能力，如果你整晚都在给孩子喂奶，那么老实说，只是睡觉，不要试图在你筋疲力尽的时候学习，你不会学到任何东西，你会发现自己一遍又一遍地倒带或重读页面。

[![Get plenty of sleep!](img/5e53d8c4b5d11d74a7b001804c775c90.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--PzQS37o9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/zsapj9o1aizhix08scs6.jpg)

我倾向于在晚上 10 点左右上床睡觉，再晚一点，第二天早上我就觉得像死了一样。我倾向于在晚上读书，或者在我儿子上床睡觉后看一些电影。

已经有很多关于睡眠对学习和生活其他方面的影响的研究，睡眠不好只会阻碍你的进步，所以尽可能多的睡吧。

## 结论

在一天结束的时候，你必须想要学习一门学科才能有效和高效地学习，从视频、书籍、博客和播客中学习不同的学习形式，我发现这些是最有价值的方法。

但是也要记住，完全可以按照自己的节奏进行，不要拿自己和别人比较，要有充足的睡眠，让学习成为一项你喜欢并且不害怕的有趣任务。
