# DevOps 为什么有用

> 原文：<https://dev.to/flippedcoding/why-devops-is-useful-4d5k>

没有听说过 DevOps，你就无法进入 web 开发领域。这就像每个人都知道的事情，但没有人真正理解。希望当你读完这篇文章的时候，你会知道 DevOps 是什么以及它为什么存在。我花了一段时间才明白它的意义，但现在我明白它为什么会成为一件事了。

## 什么是 DevOps

DevOps 是处理应用程序生命周期的一种方式。从代码库创建的那一刻起，一直到产品代码的第十次发布，它都坚持着这个项目。它结合了传统上保持独立的两个团队。开发团队和运营团队合二为一。

通常这意味着开发人员学习如何为他们编写代码的应用程序管理构建和部署过程。这意味着学习如何使用 Azure 或 AWS 等工具来管理负载，以及学习如何自动化部署过程。开发人员负责应用程序的每个部分，从编写代码到部署用户看到的产品版本。

## devo PS 为什么存在

每个项目的幕后都有很多流程在进行。您必须处理用户报告的错误，而不要制造更多的错误。还有就是每次都要经历部署过程的问题。是否有适当的测试来检查代码？你能绝对肯定你每次都以完全相同的方式做每件事吗？

有了 DevOps，您可以摆脱很多这些顾虑。整个过程可以自动化，从而保持构建和部署的一致性。当您知道您的部署过程是一致的时，它会让您有信心更频繁地进行部署。在部署一些小的错误修复之前，您不必等待很长的周期来完成。

您可以更频繁地进行较小的部署，因为您确切地知道每一步发生了什么。这将使客户在使用您的应用程序时更高兴，因为他们看到他们提出的请求有了更快的回复。

在大多数 DevOps 工具中，还强调监控系统统计数据。你将能够实时地对突然的变化做出反应，因为它们会被报告给你。如果您的应用程序获得异常高的流量，您可能需要升级您的服务器资源。您可以通过 DevOps 自动实现这一点。就像改变一个设置一样简单。

DevOps 使应用程序的服务器端部分更容易使用。一旦您构建了构建和部署管道，您的代码将自动通过测试并部署到开发或生产环境中，您不需要做任何事情或与任何人交谈。拥有这种访问权限改变了公司处理软件的方式。这需要一些时间来适应，但值得投资。

## devo PS 如何工作

就像 web 开发中的其他事情一样，使用 DevOps 也有一些最佳实践。他们概述了你需要实现的东西，以完全进入 DevOps，但不是每个人都做所有的事情。有些人只做持续集成和交付，而其他人可能做得和一些微服务一样好。以下是一些最佳实践。

*   **持续集成:**这意味着每次你提交代码并且它被合并到一个主分支中，它将通过你设置的自动化测试。您编写的任何单元测试都将在这里运行。如果提交通过测试，那么它将与主分支合并。如果没有，你会收到一封电子邮件告诉你失败了。通过测试后，会自动为您运行一个构建。
*   **连续交付:**一旦您准备好了一个构建，这个部件将获取这个构建并自动部署它。通常是持续集成后的下一步。您可以将其设置为将构建直接部署到开发或生产环境中。这一部分主要是为您自动化部署。
*   **监控:**在您的应用程序被自动部署之后，您可能需要观察某些统计数据。内存使用量很大，访问应用程序的用户数量也很大。监控确保您可以找到服务器问题和意外行为的原因。

还有一些，但这些是 DevOps 的骨架。一些开发人员可能会将基础设施作为代码和微服务包含在 DevOps 中，这很好。它使应用程序变得更好！如果你学会了如何处理至少上面的三个实践，你将能够处理大多数使用 DevOps 的项目。

当我第一次听说 DevOps 时，听起来很奇怪，但它是有意义的。为什么不让开发人员来处理部署呢？这是一个严肃的问题。你觉得这有什么大问题吗，或者这让事情变得简单了吗？如果设置正确，事情似乎会变得更容易，如果设置不正确，事情就有点糟糕了。
