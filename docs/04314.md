# 当我只想放弃时，我是如何坚持下去的。

> 原文:[https://dev . to/Jeremy/how-I-keep-go-when-I-just-want-to-quit-pec](https://dev.to/jeremy/how-i-keep-going-when-i-just-want-to-quit-pec)

我原本打算这篇文章是一篇关于我如何构建我的第二个 Rails 应用程序的技术文章。这个想法是从哪里来的，遇到错误和解决方案的英雄式描述，诸如此类。但是我不能写，因为我的心不在这里。所以我要讲一些不同的东西，我希望这能让那些需要听我讲的人听到。

你看，我已经在一个自定进度的 web 开发训练营呆了一年半多了。在那段时间里，我见过很多人退出，很多人在几个月内就飞过去了。我认识的大多数在我毕业时就开始工作的人现在都已经毕业了。学习编程对我来说并不容易。

当我开始编程时，我有一种感觉，我可以做到这一点，但我没有证据支持这一点。我从未见过命令行。我从未听说过 Python、Ruby、Go、Erlang、Elixir、C 或任何其他语言。我从来没有想过 Java 和 JavaScript 是两个完全不同的东西。须藤？哦，你是说菲尔·柯林斯的那首歌？

**权限被拒绝**

当我应该构建我的第一个程序(CLI web scraper)时，我完全迷失了。在我找到一个指导我如何做的导师之前，我尝试并失败了多次。学会依赖他人，尽管感到愚蠢或尴尬也要寻求帮助，这对我的成功至关重要。但是即使在构建了面向对象的 Ruby 程序之后，我仍然感到失落。我会看着我的同学解决对我来说不可能的编码问题。我很沮丧，很焦虑，几乎每天都在想放弃。但是在新兵训练营开始大约五个月后，有趣的事情开始发生了。我还是和以前一样被困住了，但是我请求帮助的次数少了一点。我自己找到了更多的解决方案，事情对我来说开始变得更有意义了。我慢慢理解了为什么我需要创建类，`attr_accessor`是做什么的，随着我对 Ruby 中的`self`的深入了解，我也对自己有了深入的了解。我从不同的角度思考问题。逻辑地，系统地，分解事物，推理出结果和解决方案。我感到自信，兴奋，有能力。“这个我能行！”

然后我不得不学习辛纳屈。

[![Frank Sinatra punching](img/bdc96f1e2c15dfea7329dab6b4dab6bc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rtoQxqih--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kghb6f9d29ftkllf8pgk.gif)

学习 SQL、ORM、Rack、ActiveRecord、MVC 架构、web 框架，让我感觉又重新开始了。焦虑马上就产生了，我脑海深处总有一个声音在说“你不能这么做。”但是，在我开始工作将近一年后，多亏了可靠的指导，经过许多许多次的开始、停止和重新开始，我已经开发了我的第一个[网络应用](https://dev.to/jeremy/a-year-ago-i-never-would-have-been-able-to-do-this-but-i-just-built-my-first-web-app-here-it-is-1534)。

**最好的还在后面，天气不会变好吗**

一旦我有了一个应用程序，我觉得它会一帆风顺。当我在 Rails 中重建我的 Sinatra 应用程序时，我遇到了一个全新的挑战。我花了一段时间学习 Rails 能做的所有事情，并习惯了它的强大。几个月后，我已经构建了一个完整的 Rails 应用程序，但仍然觉得自己并不真正理解 Rails。因为我为第二个项目重建了第一个项目，所以我真的想为第三个项目做一些不同的事情，这应该只是将 jQuery 特性添加到我已经构建的 Rails 应用程序中。然而，焦虑开始了，我做不到。一想到要从头开始构建另一个应用程序，我就会吓出一身冷汗。接下来的几个月毫无收获，然后我有了一个很想开发的应用程序的想法。我可以马上看到它在后端是如何工作的。我知道如何建立我的数据库，我的协会，我的模型。我坐下来，打开我的终端，做了一个新目录，然后运行`rails new`。这里可以看到成品[，这里](https://my-job-log.herokuapp.com)可以看到代码[。](https://github.com/palledorous/job-log)

我发现所有的担心、怀疑和焦虑都是徒劳的，因为我知道该怎么做。现在看起来我真的要完成这个训练营了，尽管我一直认为我做不到。

我是怎么走到这一步的？嗯，问题是，我可能不相信自己，但我相信可能性。每当我认为我永远无法学习一项技术或完成一项练习或建造一些东西时，我总是对自己说，“但我有可能做到。”

这是我能给那些可能正在经历我所经历的事情并且肯定会再次经历的人的最好的建议。如果你有可能做到，你应该试一试。如果你已经停止了，你就必须开始，或者如果你想放弃，就继续走下去。这是我唯一知道的方法。这可能会很痛苦，但你会学习，你可能只是意识到你将要放弃，而没有意识到你可以一直做到这一点。

我仍然看着其他人开发的应用程序，并认为我永远不会好到足以创建这样的东西。但是后来我想起来，今天对我来说很容易的事情，在一年半以前对我来说是不可能的。所以我一直在学习，因为谁知道一年半后我能做什么。

所以不要放弃。
