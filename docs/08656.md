# 构建我的第一个副业项目时我学到了什么

> 原文：<https://dev.to/pedrobertao/what-i-learned-building-my-first-side-project-2nen>

从几年前的第一份工作开始，我就一直想成为一名更好的程序员。通过想成为一名更好的程序员，你自然会寻找实现这个目标的方法。

不难找到逐步解释如何提高编程技能的列表。几乎所有这些指南都会提到建立一个辅助项目。

所以我决定建立一个加密跟踪器应用程序，你可以通过本地通知跟踪你最喜欢的加密货币的价值。

[![](img/ff5d07424335214f78e2c91c88437ed1.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--YEno4Wnd--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7q06adku6m65zbas1ytq.png)

以下是从中吸取的教训

## 1。你必须在舒适区之外处理问题。

这个项目是使用 React Native 构建的，我现在正在使用这项技术。这并不困难，直到我不得不处理本地通知(iOS/Android)，这是我以前从未做过的事情。我花了几天时间阅读和测试，最终开始工作。在那之后，后台任务在 RN 世界中变得非常模糊，这导致了我最初想法的更多问题。

我最终解决了这些问题，并且我更愿意在不久的将来处理与这些主题相关的任务。

## 2。你编码，你检查，你修复

自己构建东西有时会让你产生懒惰的代码和糟糕的语法。我已经不止一次地发现我的代码太糟糕了，即使可以工作，我也不想提交它。我会耐心地按照设计模式或最低语法标准进行重写。那是因为我知道我会希望别人看我的代码，并给我发送关于一般架构和 UI/UX 的反馈，而不是关于我的代码写得有多差。

在完成这个第一个版本后，我可以肯定地说，我在代码(我的或其他的)中寻找不一致和缺陷的能力显著提高了。

## 3。动机是棘手的，你必须设定微观和宏观目标

推迟这个副业项目提前发布的原因可能是因为我没有足够的动力继续下去。一开始，我试图一步到位地构建所有东西。当然，这并不奏效，让我一直在想这是否真的是个好主意。但与此同时，我不希望这个项目在“我没有时间”的炼狱中结束。因此，我改变了我的工作常规，增加了一些小的增量步骤(就像任何其他敏捷方法一样)，并逐渐构建了应用程序的核心。从那以后，每次我坐下来为这个项目编码时，我都会专注于一个微观任务，总是追求宏观目标。

这样，我设法在两个月内完成了第一个版本(我预计是一个月)，我不得不说，这是一个很大的解脱。

### 结论

建立一个兼职项目可能会有些压力，可能会导致一些头痛，但总体经验和学习是值得的。你应该建立一个副业项目，因为这将增加你的技术知识，你将面对和解决新的问题，你将有一些很棒的东西与社区分享。不要让你的想法在“我没有时间”的区域里消失。
