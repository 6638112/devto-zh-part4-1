# 如何在演讲中进行有影响力的演示

> 原文:[https://dev . to/3 sanket 3/how-to-give-impact-demo-at-a-talk-28ph](https://dev.to/3sanket3/how-to-give-impactful-demo-at-a-talk-28ph)

在会议上演示代码是至关重要的一部分。如果出了问题，可能会让你陷入困境。此外，演示应该让观众觉得有趣、易于理解和可行。我将解释我在演示时使用的一些技巧，到目前为止都很有效。

# 虽然很小，但它应该是一个观众能理解的实际应用🚗

演示应该与大多数观众所知道的实际场景相关。

而且，他们应该感觉到，他们可以不费吹灰之力就做到，为此，

*   尽量让它容易理解。你可以展示一个架构图来给出一个整体的想法
*   使用第三方服务/库，这样用户就可以尝试专注于核心概念，而不需要额外的支持代码
*   保持一个非常简单的文件结构。

# [](#code-from-scratch)从零开始编码🐣

我发现，如果我们从创建一个项目文件夹开始演示，观众会觉得没有什么神奇的事情发生。

如果你正在使用框架，是的，你可能不想花时间在长时间运行的搭建过程上。在这种情况下，如果你能从默认的脚手架文件本身开始，那就太好了。如果你默认需要一些工具，你可以创建一个包或者至少是一个独立的文件夹，这样用户就可以考虑第三方库，而不用关注它的实现。

# [](#use-thirdparty-services)使用第三方服务👨‍🔧

当我们进行实际演示时，我们可能需要第三方服务，如发送电子邮件、上传图像、提取特定业务的 JSON 数据等。如今，在无代码解决方案不断发展的时代，大多数这样的服务都很容易集成。因此，我们最好快速添加它们，并尝试使演示尽可能地真实应用。

这些服务大多有免费的配额限制，所以只要确保你不超过演示本身相同😉我经历过一次😅

# 观众应该觉得这是可行的😎

演示的真正成功是当你的观众尝试了它，并且能够实现你所教的东西。只有当他们发现可行时，他们才会尝试。显然，可以有不同经验水平的人。但是我们应该尽可能让它变得简单。

# [](#keep-some-contents-in-mind-to-fillin-while-waiting-for-long-running-tasks)在等待长时间运行的任务时，记住一些要填写的内容📄

演示期间可能会有一些任务需要时间，如停止和启动项目、部署、下载库等。在等待这些过程完成时，我们可以解释相关的概念。所以同样保留一些内容。

# [](#practice-practice-and-practice)练习，练习，再练习👩‍💻

实践只能让你写代码快，错误少。将会有许多你可能没有想到的场景。通过更多的练习，你会在实际演示之前碰到它们并弄清楚。此外，正如我在[准备你的第一次演讲](https://dev.to/3sanket3/preparing-your-first-talk-5emg)中建议的，你可以给你的朋友演示一下，并获得反馈。

# [](#keep-screen-recording-as-backup)保留屏幕录像作为备份📹

我们都知道，是编程！如果不幸的是，你遇到了意想不到的情况，并且不可能在有限的时间内摆脱出来，你可以通过屏幕录制来解释演示。

希望你觉得有用。请在评论中加入你的建议。我很想从你的经历中学习。
