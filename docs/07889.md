# 3 天开发者的神话

> 原文:[https://dev . to/realtoughcandy/这位三日开发者的神话-5gj7](https://dev.to/realtoughcandy/the-myth-of-the-3-day-developer-5gj7)

## 没有什么比计算机更令人费解或复杂的了。那么 web 开发可以在几天内学会的想法是怎么回事呢？

是训练营集体承诺在 8 周内把代码新手变成熟练的 MERN·斯塔克忍者摇滚明星大师吗？

我自己的 YouTube 小生里的人？

高调的高绩效者在 Reddit 上炫耀，坚持说这东西非常简单，也许如果你读了 MDN 文档，你就能理解 JaVaScrIpT 了？

很难归咎于任何一个地方。但事实是，我遇到的开发人员比我应该遇到的要多，他们责备自己在 X 周内没有学习 X 概念、工具或语言。

### 请记住，传统的计算机科学学位需要 4 年时间，外加学习网络开发的额外时间。

还要记住，作为开发人员，我们每次启动代码编辑器时，都试图控制和理解无限。换句话说？这东西很硬。

为了回应这种在 X 天/周内学习编码的热情，去年我发布了一个由 Tina Jensen 主演的视频。蒂娜是我的另一个自我，她只用了 3 天就学会了编码。

[![](../Images/34a31c67b05bf026fc1ab73f88363b36.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UwiO8UOQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/dif0zve35oidrn6zkl87.jpg)

蒂娜生涩、自信又健忘，她确信自己已经用新获得的开发技能为工作做好了准备。她没有想到，HTML 并不代表厌恶女人的超级文本，或者“找工作”并不意味着花 5 分钟浏览 Indeed.com 上的促销信息，然后就此收工。

[遇见蒂娜:](https://www.youtube.com/watch?v=T-ISBcgkjVs) [![YouTube Demo video](../Images/472058432c9be60d8ce42cae907ee7d2.png)](http://www.youtube.com/watch?v=T-ISBcgkjVs)

最终的信息是，这需要时间。它需要大量的练习，会导致很多挫折，激发巨大的梦想，当事情发生时给我们一种胜利感，并引发频繁的大脑融化。如果你问我，这一切都是值得的。

当然，总会有非常高的成就者在 X 天内学会如何编码。好极了。

对于我们其他人来说，如果需要一段时间也没关系。
