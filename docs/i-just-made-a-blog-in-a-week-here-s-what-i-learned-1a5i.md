# 我刚在一周内写了一个博客——这是我学到的东西

> 原文：<https://dev.to/xy2iii/i-just-made-a-blog-in-a-week-here-s-what-i-learned-1a5i>

你好，戴夫·托！我刚刚发布了我的博客。我是计算机科学本科一年级学生，我决定挑战自己，写一个博客。[我在我的博客](https://xy2.dev/making-a-blog-from-scratch-in-a-week/)上发表了一篇关于此事的博文——或者如果你喜欢在 dev.to 上阅读，那么请继续阅读:)

两个月前，[我和一个由我们学校最聪明的人组成的团队参加了一场法国 CS 比赛。比赛分为三个 8 小时的背靠背项目，因此得名 *24H IUT 信息*。](https://di.iut.univ-lehavre.fr/24hinfo/index.php)

在第二个项目中，我们有八个小时的时间来创建一个玩网络游戏的机器人——每个机器人将与其他团队的机器人进行比赛，获胜者将晋级。我们的组织不稳定，为了恢复上一个项目，我们损失了六个小时。临近截止日期，我们决定实现最简单的功能来避免取消资格:连接到服务器，发送一个随机移动(一个有效的字符串)，然后等待响应并再次发送。100 行 Python 之后，我们有了东西。

我们最终在这个项目上击败了超过一半的团队:八个小时后，超过一半的团队**没有工作原型**。每个断开连接、程序崩溃或直到最后才能比赛的团队都被取消资格。

为什么一半的团队在拥有最好的计算机系本科生的比赛中失败了？我仍然不确切地知道为什么，但是在制作这个网站之后，我可以提供一个很好的猜测。

# 获得经验

大学一结束我的计算机科学第一年，我就缺乏经验。当面对制作真正的软件时，我不知所措。我广泛阅读，知道软件工程与我们所学的理论非常不同，所以我开始写一个简单的博客。

为了学习，我做了一些限制:

*   我想单独完成*，而且要快:我给自己一周左右的时间。*
**   我想从头开始建立我的博客。我用 Django，剩下的我自己做:租我自己的服务器，运行我自己的软件。*   我希望这个网站能够合理地*快速*和*易于阅读*。*   最重要的是，我想采取*没有捷径*:没有从堆栈溢出中复制粘贴代码，并在深入研究之前对我使用的工具有一个很好的理解。*

 *# 我学到了什么

### 设计你的程序，不要写代码

在写任何代码之前，我问自己:*我希望我的博客有什么？我花了很长时间来回答这个问题，随着时间的推移，我的答案也在不断变化。我认为我最重要的资产是一个灵活的设计，我可以很容易地改变，并坚持我早期做出的保证。如果我没有一个设计，或者我想在每次遇到复杂的问题时切换框架，我会花更多的时间来实现这个目标。*

### 采取正确的权衡

有很多次在寻找解决方案时，我看到了一些“银弹”,这些“银弹”需要更多的工作来处理。我看到了许多“闪亮的技术”，尤其是在早期，我很想使用它们。最后我避开了这两个，每次都选择了“无聊的技术”:postgres，debian，django。如果你不断地问自己使用什么样的框架或数据库，就很难超越它开始编程。

### 根据需要编写尽可能少的代码

我从头开始编写我的站点，但是没有重新实现整个框架，因为这超出了我的范围。我总是寻找那些经过良好测试的库，我可以毫不费力地依赖它们。与此同时，如果有令人失望的 libs，我会毫不犹豫地放弃它们，寻找替代品。在寻找一个 markdown 解析器时，我遍历了几个解析器，直到找到一个符合我设计的。

### 部署快，提炼慢

拥有一个不可靠的产品远比没有产品好。但是不要忘记，这意味着你必须回头去改正你的错误。我花了大量的时间以最快的速度赶到一个工作地点——然而每当我抄近路时，我都会记录下来，然后回去修理几乎所有的问题。我的代码中仍有一些令人尴尬的部分，我会在问题出现时修复它们。

### 彻底

在我开始写这个网站之前，我确保很好地学习了 Django。在很多情况下，当我遇到问题时，我有一个可以接受的解决方案，没有妥协任何东西，这要归功于我的准备。当事情结束或不起作用时，在我的初始部署之后，我将其他一切都搁置起来，以确保特定的问题得到解决，有时会钻研源代码并进行调试，以确保我理解了问题，并确保它不会再次出现。

### 不放弃

我没有提到时间框架，但它对我的存在产生了重大影响。我每天早上 5 点起床，工作至少 8 个小时，甚至更多。当处于这种状态时，你会想把所有事情都搞定。我没有这样做的唯一原因是我在早期定义的目标，如果没有它，我可能会在遇到第一个障碍时就放弃并找到解决方案。

# 结论

我认为，我认识的许多计算机科学学生面临的主要问题是成为一个永远的学生:总是学习新的技术和框架，但直到多年以后才能够用这些知识做些事情。不管你什么时候有机会这样做，都做得很差，因为学生从来没有学会如何运用所学知识和进行批判性思考。

所以这就是我想要摆脱的。让我知道你的想法。如果你想看代码，我已经把它放到 GitHub 上了。*
