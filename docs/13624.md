# 走出开发人员的舒适区

> 原文:[https://dev . to/visco sho/走出舒适区——作为一名开发者——olo](https://dev.to/viscosho/stepping-out-of-the-comfort-zone-as-a-developer-olo)

在过去的几年里，我在我的个人生活和职业生涯中尝试了很多积极的变化，这让我尝试新的东西，其中之一就是作为一名开发人员接受更多的挑战。由于我是远程工作，这对我来说很重要，因为我不想感觉自己被困在职业生涯中。我没有遇到很多这样工作的开发人员，所以我感觉我正在错过与其他同事交流知识和丰富知识的经历。
我列出了一些帮助我走出舒适区、迎接更多挑战的事情，我想在这里分享一下:

*   我已经开始参加我所在城市的会议和聚会:这已经成为我的一件大事，让我能够结识优秀的人，进行更多的社交。此外，如果需要的话，我现在有机会寻求建议，不仅是技术方面的，还有职业方面的。
*   我已经开始做代码挑战:当我有空闲时间时，我会尝试做一些实用的代码挑战来帮助我提高技能，有时我认为我不知道如何做一些事情，所以我没有尝试过，现在只要我能，我就会挑战自己。这真的提高了我最近的编码技能。
*   我在社交媒体上关注其他开发人员:我过去认为社交媒体只是为了打发时间，现在我开始更多地将它作为知识和信息的资源，我也发现了一些出色的开发人员，他们是普通人，并帮助我认识到我们有时都在与代码和能力作斗争，这很好。
*   我已经开始承认我的经历:我曾经很害羞或者害怕说我是一个好的开发人员，我不认为我真的相信这一点，所以我对此非常非常谦虚，现在我正在努力变得更加自信。我现在能够毫不犹豫地说什么是我觉得舒服的编码，什么是我擅长的，关键是要明白你不需要 100%知道一切。
*   我正在参与一些我通常不参与的项目:例如，使用另一个框架，使用另一种您知道但通常不在工作中使用的语言，帮助朋友处理个人项目，指导某人，向另一个开发人员提供建议或寻求建议。
*   我正在花时间做其他事情:最后，我已经开始寻找另一个爱好，我真的喜欢编码，但我也相信它有时会有压力，所以我明白也喜欢与编码无关的其他事情是可以的，并享受它们，享受我的自由时间，不去想代码是放松的。

我真的很想尝试教学，但不在我的清单上，因为我还没有实施，这是一个过程中的工作。

我还能做什么来走出我的舒适区？
