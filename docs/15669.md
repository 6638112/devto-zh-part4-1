# 我如何成为一名软件工程师

> 原文：<https://dev.to/rahat/how-i-became-a-software-engineer-3pdl>

Career Karma 最近在 twitter 上发布了一些东西“你总是离完全不同的生活只差一个决定”。在过去的一年里，我反复遇到这样的情况，一个决定完全改变了我的生活走向。

几年前，我是一名没有太多方向的大学辍学生，在客户支持部门工作，非常讨厌我的职业发展方向。我最终回到了大学，开始学习计算机科学，但这并不足以满足我的生活需求。坦白地说，我正在努力获得一个别人比我更关心的学位。我仍然不能不关心它。我需要一些实际上能让我亲身体验与当今时代相关的技术的东西。单靠传统的大学教育是不够的，对我来说这不是最好的学习方式。

一年前，我做出了第一个决定，这个决定彻底改变了我的生活。我决定参加一个新兵训练营。我研究了几个不同的选择，最终选定了我最好的朋友找到的一个训练营。这是哥伦比亚大学的三部曲训练营。我喜欢这样一个事实:我从一个不知道我在 HTML 文件中使用的内嵌样式被称为 CSS 的人，变成了一个在 React 中构建令人敬畏的现代 web 应用程序的人。

随着我们在训练营完成的每个项目，我的信心都在增加。我学到了很多，我渴望更多的知识。除了训练营，我还开始参加 Udemy 课程、前端大师课程，甚至是传奇人物肯特·c·多兹的工作坊。我坚持尽可能多的学习知识，并且尽我所能的学习。我想学习在就业市场上流行的所有不同的框架和技术。

2019 年 3 月，我从训练营毕业。我准备好去就业市场了。我很快失败了。我在第一个月申请了数百份工作，只收到了少数公司的回复。他们中的大多数人只不过是打了一个电话或者没有通过代码评估。我的信心立即下降。我开始进一步沉浸在 Udemy 的课程中，思考“如果我学得更多，这些公司会雇用我，对吗？”这个想法很快就会受到挑战并被彻底推翻。

我开始在 Twitter 上宣传自己，并在 Twitter 上的不同技术社区中变得更加活跃。一家小型初创企业的创始人发现了我，他希望为自己的公司雇佣更多的开发人员。他对我的客户服务背景很感兴趣，似乎对我喜欢编程感到惊讶。我想他现在的员工只是不像我一样对编码感兴趣。我们在电话上交谈，他很快邀请我亲自发言，看看我是否合适。我对第一次现场面试的前景感到兴奋。

我很自豪地透露，我正在从事一个项目，其中包括了很多他们正在使用的技术。我用的是 React、GraphQL 和 PostgreSQL。然后，我在第一次面试中失败了，无法真正解释为什么我编写的任何东西实际上都在工作。在被问及有关技术的问题，并回答了或多或少的错误后，我被告知我可能不太适合这个角色。我立刻被击垮了，但他接下来为我做的事情是我今天真正感激的。他花了将近一个小时给我反馈和如何提高自己的建议。以下是他告诉我的内容的摘要:

*   我试图在同一时间学习太多的东西，但没有一样真正留在我的脑海里。

*   我太专注于他所谓的“性感标签”和现代技术中存在的抽象概念。

*   我需要回去重复基础，重建我的基础。

*   我需要加倍使用我的普通 JavaScript。

*   不使用 ORM 运行 SQL 查询。

*   忘记那些让开发生活变得更容易的事情，去经历那些痛苦的事情，比如用 Node 而不是 Express framework 构建服务器。

这些都是可行的项目，我知道我可以带着它们去做。他没有就此罢休，他还告诉我我做得对:

*   继续在 Github 上积极提交我的代码。

*   继续写关于我的旅程的博文，分享我所了解的事情。

*   继续活跃在 Twitter 上，在网上向他人学习。

我认为反馈会议持续的时间比面试本身还要长，我将永远感激我在这一刻失败了。我回去加倍下注在基础上。我专注于使用普通的 javascript。我重新学习了如何构建简单的节点应用程序。我使用原始 SQL 查询做了几个项目。与此同时，我让自己在网上保持活跃，并准备好与他人交往。我开发了一个小型的 React 简介研讨会，并以网络研讨会的形式进行了演示。这是我第一次在 React 中创建完整的项目，而不依赖 Create-React-App CLI。

当我做这些事情的时候，我在寻找聚会、黑客马拉松和会议来参加。我参加了一些不同程度的成功的黑客马拉松。我在耶鲁黑客精神健康黑客马拉松上建立了一个很好的联系，现在我正在做一个兼职项目，为大学生开发一个精神健康应用程序。这些联系和项目都提高了我的技能，但我需要一个带薪的工程职位来摆脱我的客户支持工作。

现在我们要做的另一个决定彻底改变了我的生活。我决定自愿参加一个名为“全栈纽约”的会议。问题是，我几乎没有。这是在斋月期间发生的，我不确定我是否可以一整天都斋戒，同时也在会议上工作。我忍气吞声，还是做了。

我在社交和与人交谈方面有很多麻烦，所以这对我来说是一个走出舒适区的好机会。我可以帮助并直接接触到演讲者，而不必穿过人群，并试图保持他们的注意力足够长的时间来建立联系。演讲者直接跟我说话，不需要我主动发起对话，因为我在帮助他们。如果你是一个内向的人，我强烈推荐这种方法。你将会工作，所以总会有一些事情要做，以避免任何尴尬的沉默，你不必努力开始这些对话。他们会马上过来找你。我和一位志愿者同事成了朋友，她后来带我去她的聚会上做了一次演讲，还把我介绍给了美国运通招聘新开发人员的人。

会议结束一周半后，一个周四的下午，我接到一个电话，问我能否在第二天去参加一个工作面试。我很害怕，感觉毫无准备，差点说不，这太晚了。就在那时，我做了一个彻底改变了我生活的最新决定。我告诉自己，自从上次面试以来，我已经做好了准备。我告诉自己，我可以回顾任何小细节，但我已经知道我需要知道什么来证明我懂 JavaScript。我知道如何在反应中工作。我知道如何使用 Node 和 Express。我必须向采访我的人证明这一点。

于是第二天的大部分时间，我都在经历一个漫长的现场面试过程。当时采访我的人给了我很多很好的反馈。我很乐观，但仍然很谨慎。我让时间流逝，并继续在其他公司试试运气。我最终陷入了一个困境，不知道是否会有什么事情发生在我身上。然后就发生了，我得到了一份工作。今天，在完成哥伦比亚大学的训练营三个月后，我很高兴地宣布，我已经接受了美国运通公司软件工程师的邀请。我将会和许多我喜欢的技术一起工作，为一个了不起的公司和一个了不起的团队一起工作。我能走到这一步，是因为我付出的努力，是因为我的朋友、家人和妻子给我的支持。我之所以走到这一步，是因为有人花时间从一次失败的求职面试中给了我切实可行的反馈。

不是每个人都幸运地得到这样的机会，我真的很感激。当我在训练营之后建立自己的时候，我开始帮助其他人，他们也开始学习更多关于编码的知识。它巩固了我的知识，并真正帮助我在面试中取得成功。我计划继续这样做，因为帮助别人最终也帮助我变得更好。我就是这样成为一名软件工程师的。这些方法中有些可能对你有用，有些可能不。要记住的最重要的事情是不要放弃，因为你可能会发现自己离彻底改变生活只有一步之遥。
