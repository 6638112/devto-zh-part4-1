# 无头的商业和它为什么摇摆不定

> 原文:[https://dev . to/Anthony cook/headless-commerce-and-why-it-rocks-2nkl](https://dev.to/anthonycook/headless-commerce-and-why-it-rocks-2nkl)

Headless 是电子商务世界的下一个大步骤，它本质上允许你独立于你的电子商务平台和后台建立面向公众的店面。

这带来了巨大的好处，包括性能、定制和可扩展性。大品牌纷纷涌向无头模式有很多原因，所以现在让我们来讨论其中的一些。

## [](#complete-control-of-urls)完全控制网址

在大多数电子商务平台上，你必须符合他们的 URL 结构和分类。一个很好的例子是 Shopify，他们所有的产品 URL 都以`/products`为前缀，所有的类别都以`/collections`为前缀，再加上事实上没有一个真正嵌套类别 URL 的方法，这意味着 SEO 很有挑战性。

让我们来看看一件印有菠萝图片的男士 t 恤的理想 URL 结构是什么样的:

```
/mens/t-shirts/pineapple-shirt 
```

我们的云甚至更深，就像这样:

```
/mens/t-shirts/blue/large/pineapple-shirt 
```

这对 SEO 来说很棒，并且清楚地显示了你在看什么，对人和机器都一样。

问题是，在许多 eCom 平台上，这根本不可能。再次以 Shopify 为例，我们能够达到的最接近这种理想结构的结构是:

```
 /collections/mens/products/pineapple-shirt 
```

不太好，对吧？我们不仅在每一个网址上都有*系列*和*产品*的字样，我们也不能嵌套超过一个类别，哎呀！

在我过去的经验中，没有一个 ecom 平台提供结构化 URL 的完美解决方案，但幸运的是 headless 解决了这个问题，因为你可以完全控制你的商店的前端，你可以按照你认为合适的方式构建你的 URL。这很好，因为我坚信，当涉及到如何组织产品数据时，没有一个放之四海而皆准的解决方案。

## [](#language-agnostic)语言不可知

当使用任何电子商务平台时，通常会附带一个模板工具来构建主题。这很好，但不幸的是，与成熟的编程语言相比，它们的能力有限。当然，它们对条件和变量有基本的支持，但是我可以保证总有一天你的模板工具不能以你想要的方式请求或操作数据。

有了 headless，你可以通过你的 plafroms API 请求你的数据，这样你就可以决定在你选择的编程语言中你想对数据做什么。

大多数平台都有传统的 RESTful API，但是有些平台提供了更现代的方法，比如 GraphQL。同样，这也是关于定制和以最佳方式打造商店的能力。

## [](#scalability)可扩展性

当你的商店与你的平台分离时，它允许你在任何你喜欢的地方托管你的前置网站。

这意味着您不仅可以控制缓存和内容交付等事情，还可以在许多服务器上复制您的商店网站，为您提供疯狂的可靠性和冗余。

同样值得一提的是，像 progressive web apps 和 AMP 这样令人兴奋的新技术在 headless 上的实现变得更加容易。

## [](#finally-make-it-your-own)最后，让它成为你自己的

想用微小的捆绑包和平滑的页面加载制作一个高性能的 react.js 应用吗？现在你可以，想做一个零膨胀和最小依赖的原生移动应用吗？现在你可以了！

Headless 真正做到了摇滚，它利用了 Shopify、BigCommerce 和 Magento 等平台的优势，打破了它们的束缚和限制。

如果您还没有，我强烈建议您研究一下 headless commerce，为您的用户提供更好、更快的体验。
