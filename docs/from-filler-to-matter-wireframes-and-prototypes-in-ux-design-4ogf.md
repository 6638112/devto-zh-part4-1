# 从填充物到物质:UX 设计中的线框和原型

> 原文：<https://dev.to/gymnasium/from-filler-to-matter-wireframes-and-prototypes-in-ux-design-4ogf>

设计语言对新手和专业人士来说都是令人困惑的。线框和原型是两个特别容易混淆的术语，尽管这两个在生产过程中有明显不同的作用。缺少这些关键步骤可能会导致产品不完整，用户体验不够一致。因此，理解线框和原型之间的区别并确保它们在你的项目中都有一席之地是至关重要的。

# 从一个线框开始

尽管线框跨越了一个保真度范围，但它们往往处于低端。这意味着它们是粗略的草稿，通常省略了细节和最后的润色，如复制和可点击的内容。换句话说，它们并不意味着是艺术品。一个成功的线框的关键是简单，因为它是成品的静态骨架。宽水平线和 Lorem Ipsum 可以作为文本的占位符；空白框可以替代图像和视频；箭头可以说明运动和流动。在项目过程中，这些代理将成为用户体验中完整的核心元素。

无论是通过数字草图工具设计还是在白板上头脑风暴，线框通过定义其结构、布局和[信息层次](https://theblog.adobe.com/a-beginners-guide-to-information-architecture-for-ux-designers/)的轮廓来形成你的概念。就像建筑的蓝图一样，线框出了更大的画面，并帮助你规划出实现它所需的步骤。它们也是在你的想法的第一次迭代中获得建设性批评的宝贵资源，这可以解决潜在的问题，并为设计师和开发人员节省时间。

虽然线框在设计和开发的开始阶段起着关键的作用，但它们最重要的功能可能是作为原型生产的基础。

# 计划有了雏形

原型在骨骼线框的顶部构建肌肉——功能和形式——并赋予其生命。这给了你一个真实的设计，以及可用性和交互性的水平。与低保真度的线框相比，原型是高保真度的模型，为最终产品提供了有形的测试驱动。

无论你的目标是方便用户测试还是向客户展示概念，原型通常都是通过使用像 [Sketch](https://thegymnasium.com/courses/GYM/005/0/about) 和 [Axure](https://thegymnasium.com/courses/GYM/008/0/about) 这样的软件来创建的。通过这些工具，您可以通过添加可点击性和动态内容等特性来模拟与原型的真实交互。这种交互性将线框变成了原型。它还有助于简化开发过程，并使从概念到执行的过渡更加容易。

然而，正如你将在 gym 的新课程中了解到的，原型不仅仅是高保真的模型——它们还是一种交流的手段。作为概念的证明，原型可以通过提供最终产品及其功能的清晰图片，在设计者和开发者之间架起一座桥梁。这种共同点促进了协作，并使团队有机会在错误产生实际后果之前识别和解决错误。根据最近在[体育馆举行的网上研讨会](https://gymnasium.github.io/cms/webinars/planning-before-pixels/)，花时间这样做可以使你的最终结果更加完美和有目的。

根据一项研究，原型的使用可以减少 25%的发布后错误报告。正如设计师约翰·梅达曾经说过的，“如果一张图片胜过千言万语，那么一个原型就抵得上一千次会议。”在它们的细节水平和帮助阐明目标的能力之间，原型是项目管理、降低风险和创建精炼产品不可或缺的资产。

# 那么，我应该是线框，原型，还是两者都要？

你的具体过程将取决于你的预算和时间表。然而，花时间使用线框和原型是确保你的用户体验被优化到尽可能无缝的一种方法。这两种资产协同工作，将您的想法从开始带到实现，并允许反馈和微调。尽管跳过实践环节直接进入细节很有诱惑力，但完善你的愿景并允许它发展将会给用户和团队带来更好的体验。
