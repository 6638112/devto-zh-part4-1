# 管理远程开发团队

> 原文：<https://dev.to/jetrockets/managing-a-remote-development-team-5cgl>

沟通 -听起来很明显，有效的沟通是项目成功的关键。对于地理上完全不同的团队来说更是如此。将所有团队成员联系起来，发展团队成员之间的相互信任和协作，可以增强责任感。有很多工具可以帮助集中沟通。在 JetRockets，我们依靠 Slack 来促进沟通，并让每个人都了解项目和关键决策。

**面对面的时间**——虽然通过聊天和短信交流是有效的，但真正看到和你一起工作的人也是非常重要的。把一个名字和一张面孔联系起来有助于你更好地了解这个人，甚至可能发展友谊。视频会议很容易消除时间和空间的障碍，建立团队友谊。我们还鼓励我们的客户每周至少与他们的产品团队进行一次视频电话会议，这样每个人都能更好地相互了解。

**结构化会议** -虽然会议的频率可能取决于特定项目的需求，但每次会议的结构应保持不变，以最大限度地减少浪费的时间和提高生产率。每次会议以简短的 5-10 分钟非正式聊天开始总是好的，但是之后，会议应该坚持一个明确的议程，每个成员都有机会分享他们的进展并提出问题。任何不需要整个团队在场的后续话题都应该下线，以避免会议劫持。

抛弃电子邮件——众所周知，电子邮件是最低效的交流工具。信息经常在无尽的线程中丢失，文档很难找到并且无法以协作的方式编辑，请求被忽略或遗忘。当涉及到项目管理时，不要依赖电子邮件，而是利用许多技术解决方案，这些解决方案可以对您的项目结果产生重大影响。

任务管理在尝试了许多不同的项目管理工具后，JetRockets 团队得出结论，Asana 是最方便有效的。我们已经开发了自己的方法，使用 Asana 进行有效的积压跟踪和冲刺计划。在任何给定的时间点，项目经理或客户都能够看到团队如何进展以及出现了什么问题。

**知识库** -维护并有效共享关于项目重要信息的综合数据库非常重要。它帮助新员工入职，培训客户，并提供主动支持。有许多工具可以用于此目的，但是只要所有团队成员都为知识库做出贡献，并且它是整个开发过程的一部分，即使是简单的 Google Doc 也可以做到这一点。

不要微观管理 -大多数专业人士不喜欢被微观管理。这打击了他们的士气，并导致管理者和员工之间缺乏信任。设定现实的期望，清晰地表达出来，然后让团队完成他们的工作。建立一个适合您的更新流程，并将其作为新员工入职流程的一部分。

**及时反馈** -透明度和客户参与是项目成功的关键。为了提高透明度，超越通常的“每周状态更新”。我们的客户可以完全访问我们的项目管理系统和我们的开发环境，他们可以实时看到工作的进展，并在必要时提供及时的反馈。

**随时待命** -高效的经理随时待命，并对下属做出快速反应。重要的是要保持一种开放的政策，让自己能够跨越多个时区，通过多种渠道——Slack、Skype、电子邮件、电话、短信——及时回答问题或解决问题。

总而言之，成功管理远程团队需要特别重视沟通和及时反馈。无论开发团队位于何处，保持专注和响应，创建授权和信任的环境，以及明智地使用技术都有助于使每个项目成功。
