# 你为什么使用 Rust 进行 web 开发？

> 原文：<https://dev.to/spacerockmedia/why-are-you-using-rust-for-web-development-2i91>

对于那些使用 Rust 进行 web 开发的人，不管是不是 WASM，你们为什么决定使用 Rust 而不是 Go？我假设你想要一种静态/编译语言，而不是 Python。但是我对为什么不使用脚本语言感兴趣。

在 Rust 做 web 开发有多难？让你慢下来的最大问题是什么？什么使得使用 Rust 更好？你是否发现使用 MVP/Production 的时间比使用其他语言(如 Go、Python 或 JavaScript)慢得多？

听听那些也尝试过围绕 Elixir/Erlang 构建堆栈的人的意见可能会很有趣。
