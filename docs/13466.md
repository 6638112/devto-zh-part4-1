# 设计连续构建系统——解析规范

> 原文：<https://dev.to/tryexceptpass/designing-continuous-build-systems-parsing-the-specification-2b83>

每个代码库都是不同的。执行环境、框架、可交付成果，甚至是 linters，都需要某种定制。创建灵活的构建系统需要一种机制来指定在管道的不同阶段要遵循的步骤。作为设计连续构建系统系列的下一章，本文将研究您想要向您的定制系统传达哪些指令以及如何解析它们。
