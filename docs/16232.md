# 什么时候为了知名度和 SEO 而交叉发布？

> 原文:[https://dev . to/spacerockmedia/when-do-you-cross post-for-visibility-and-SEO-1 o5f](https://dev.to/spacerockmedia/when-do-you-crosspost-for-visibility-and-seo-1o5f)

很久以后，我终于开了我的博客。这是一个令人生畏的过程，仅仅因为你认为你需要做多少工作。即使你对材料很了解。这也是为了帮助我更好地进行技术写作。

但是有一件事我看到了不同的答案，就是*什么时候*要跨岗。还有一些地方似乎比其他地方更早地想要交叉邮寄。Medium 似乎想要更新的内容，所以早点发布可能会有帮助。而 DEV 似乎不受此影响。

总的来说，据我所知，一旦你在你的网站上发帖，你应该等待一个星期才能在 DEV 等其他网站上交叉发帖。我觉得在 reddit 这样的地方交叉发帖很糟糕，让人们看我的内容对我来说更垃圾。否则，我可以在 linkedin 和其他来源上发帖。在社交媒体上分享应该马上进行，以获得你的主域名的流量。

关于什么时候发帖，可能在哪里发帖，你们还有什么其他建议吗(除了基于你的主题的社区特定区域，因为这可能变化很大)。
