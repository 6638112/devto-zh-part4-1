# 远程工作:2 个月

> 原文：<https://dev.to/ethanbray/working-remotely-2-months-in-2b2e>

我已经在 [MessageCloud](https://www.messagecloud.com) 工作了大约 3 年半。今年年初休息去旅游后，回到 75%偏远岗位工作。我大部分时间在家工作，每个月在办公室呆一周。

我们过去也有其他几名远程员工，但他们都是从现场开始的。我读了很多关于远程工作的利弊的文章，解决了孤独和缺乏沟通等问题，所以我觉得有所准备。

这篇文章将涵盖我们遇到的一些初期问题，以及我们为帮助应对这些问题而采取的解决方案。这篇文章很可能是详述我远程工作经历的系列文章中的第一篇。

### 使用视频站立代替松弛消息

以前，我会与每个团队成员单独交谈，讨论我们当天的计划。我很少需要询问更多的信息，因为在现场让我更容易被动地获取有关他们进度和工作量的信息。

一旦我开始远程工作，我就试着用 Slack 继续日常聊天。然而，Slack 并不特别有利于长时间的谈话。尤其是几个人同时在一个房间里。人们的回复经常会延迟，因为他们被其他人转移了话题，或者根本没有看到我在其他对话中回复了他们。

我们的解决方案是每天早上进行视频站立。这样，团队中的每个成员都有机会发言并引起每个人的注意。这些单立帮助我和我的团队及时了解每个人的工作和他们可能面临的阻碍。

### 让所有成员使用网络摄像头开会

当我们介绍站立会议以及我与其他同事的定期会议时，这个问题变得很明显。如果没有网络摄像头，很难知道一个人什么时候结束了讲话，或者在有两个以上的人参加会议的情况下，谁在和谁讲话。

感觉更好。在我看来，能够看到与你交谈的人是远程工作的一个被低估的要求。

起初，我想团队中的一些成员会对被要求使用网络摄像头感到不舒服，但由于这已经成为我们站立会议中的一个常规事件，每个人都一直在其他会议中使用网络摄像头。

### 详细的问题跟踪

在 MessageCloud，我们使用 GitHub 问题来记录我们的工作。然而，公平地说，有时我们在我们的问题中留下的细节并不冗长。我们不知不觉地依赖于一天中接收到的被动信息来了解我们正在处理的问题。

当团队成员开始就某个问题与我交谈时，我会很难立即理解他们在谈论什么。我经常会要求我的同事从头开始，并向我更深入地解释这个问题。这可能会有点令人沮丧，因为你花了更多的时间要求澄清，而不是实际解决问题。

我与团队进行了交谈，我们一起工作，更详细地记录我们的问题，并让他们了解最新的进展、阻碍因素和发现的额外信息。我们还没有经常这样做，但我们正在努力使它成为整个团队的一致习惯。

### 错过休息/午餐

虽然我远程工作，但我需要像在办公室一样工作，因为我的角色主要是团队领导。远程工作的一个令人惊讶的症状是，没有办公室休息时间的喧闹声提醒我，我经常错过预定的休息和午餐时间。当专注于一件特别棘手的工作时，我倾向于打开“请勿打扰”并全屏显示我的编辑器，隐藏时间并加剧问题。

在 MessageCloud，我们有一个基于围棋的 Slack 机器人，叫做[艾娃](https://non-aliencreatures.fandom.com/wiki/Ava_(Ex_Machina))。艾娃是用来启动詹金斯部署，以及有几个有趣的小功能。

我给艾娃添加了一小段代码，它会在休息时间给我发送一条关于 Slack 的消息。

### 明智地利用我的现场时间

尽管每个人都喜欢赞美远程工作，但确实存在一些问题。尤其是在一个没有很多远程员工和“远程优先”态度的公司。某些任务，比如事件风暴和帮助初级开发人员，面对面会容易得多。

在我呆在现场的那一周里，我专注于执行那些原本很难远程完成的任务。与领域专家会面，并与组织的其他成员面对面交流。

利用这段时间与团队建立联系，确保你对他们来说不仅仅是电脑屏幕上的一张脸，这也很好！
