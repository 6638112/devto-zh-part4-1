# 你在办公室遇到的 4 种类型的开发人员

> 原文:[https://dev . to/flipped coding/5-type-of-developers-you-meet-in-the-office-m8i](https://dev.to/flippedcoding/5-types-of-developers-you-meet-in-the-office-m8i)

在我们的职业生涯中，我们都在许多团队中工作。开发人员与许多不同的人一起工作，但是当他们遇到他们时，他们知道他们的个性。Web 开发和一般的编程吸引了来自各种背景的人，但你似乎仍然会遇到“同样的人”。

在开发领域有一些志趣相投的人，你几乎到哪里都会碰到他们。这些是我在合作过的团队中遇到的一些有趣的人物。欢迎在评论中添加到列表中！

## [](#the-cutting-edge-one)最前沿的一个

每个开发人员都在某种程度上了解他们的技术栈中发生的事情。还有一些人遥遥领先，就像他们在做预测。大多数团队中至少有一个这样的人。他们总是告诉你他们玩的一个新库或者他们已经开始使用的一个有趣的软件。还记得在你听说 Docker 之前他们就在谈论它吗？

如果没有别的原因，仅仅是你能从他们身上学到很多东西，前沿是很棒的。他们总是有一些他们正在做的副业项目，他们通常了解技术栈或 web 开发中的最新事物。

## [](#the-friendliest-one)最友善的一个

你不介意和这个人一起工作。他们和办公室里的每个人以及大多数客户都有很好的关系。大多数时候，你会看到他们试图帮助别人解决问题，或者只是和别人聊天。他们拥有任何默默无闻的开发人员的技术实力，也拥有交际花的天赋。

当你带着问题不知道去哪里的时候，一般都是从这里开始。他们是团队中让人们感到舒适的开发人员。它们让你觉得即使和同事一起工作，你也可以玩得很开心。即使是到了深入研究一些严肃作品的时候，你也可以指望他们说最后一句俏皮话。

## [](#the-eccentric-one)古怪的一个

他们只是不同而已。他们很聪明，但也有一些真正的怪癖。这个人通常是团队中比较聪明的人之一，他们有自己的做事方式。你可能会去他们的立方体，找到三个空罐奶油苏打，一块石头，显示器亮度调到 10，但真的就这些了。

大多数时候，他们提出的东西解决了团队面临的问题。他们能够从用户故事中提取任务，这是前所未有的。他们还想出了非常有用的方法来处理技术堆栈。他们是团队中知道表达式树如何工作的人，他们能够以不破坏项目的方式实现表达式树。

## [](#the-nonjargon-one)非行话者

他们知道自己在做什么，但是不要用行话问他们任何问题。例如，你可以让他们创建一个类，稍后在这个类中定义类型，但是不要告诉他们使用泛型。你会从这个人那里听到很多描述，直到他们掌握了行话。与此同时，对他们使用专业术语就像在说一种不同的语言。耐心点，准备解释很多。

这个人甚至不一定是初级开发人员。术语太多了，一些开发人员已经放弃了跟上不断变化的术语，他们只是描述他们在做什么。这可以使帮助他们和从他们那里获得帮助变得更有意义，因为他们在解释中变得非常详细。但是随着时间的推移，他们通常会学会编程术语。此外，在这个过程中，你会成为一名更好的开发人员，因为你必须真正了解自己在谈论什么，才能理解他们在谈论什么。

## [](#the-senior-dev)高级开发员

无论如何，他们似乎总是知道答案。即使他们不得不用谷歌搜索某些东西，他们似乎还是会用不同的方式搜索。有不同种类的高级开发人员，但是我将把重点放在大师级别的开发人员上。这些人不仅知道比你听说过的更多的代码，他们真的很乐意帮助别人学习。

他们完成了一行代码的修改，修复了部署的问题。他们能够参与任何项目，并在需要的地方提供帮助。《高级开发者》充满了从技术堆栈到职业道路再到一般生活的建议。有时没有什么比经验更重要，他们有丰富的经验，并且愿意分享。

希望你能对此一笑置之。这些是我在流浪期间遇到的一些人格。值得庆幸的是，大多数 web 开发社区非常友好，但是仍然有一些隐藏的问题。你呢？无论你走到哪里，你似乎都会遇到什么不同的性格类型？
