# 没人告诉你的演讲技巧:申请参加会议

> 原文:[https://dev . to/Sam Julien/speaker-stuff-no-one-tell-you-about-applying-to-conferences-l3n](https://dev.to/samjulien/speaker-stuff-no-one-tells-you-about-applying-to-conferences-l3n)

> 这是“没人告诉你的说话终极指南”系列的第一篇。第 2 部分是关于构建精彩的演示，第 3 部分是关于旅行技巧、信用卡等等。(别担心，他们在路上了。)本文原载于[我的博客](http://www.samjulien.com/speaker-stuff-conferences/)。

我讨厌技术文章中的“摇手”。你知道我在说什么。你试图学习如何用一个新的框架构建一个原型，或者使用一种语言的新特性，或者建立最新最棒的构建系统，教程是这样说的:

1.  做几件小事。
2.  ？？？(“这超出了本教程的范围…”)
3.  嘭完了！

那使我发疯！我在你这个愚蠢的网站上的全部原因是为了学习第 2 部分的所有内容！

我发现在关于技术职业发展的文章中也会出现同样的挥手动作。他们似乎总是这样结束:

1.  我不知道，发些微博什么的？
2.  ？？？(某事某事“努力工作”)
3.  国际演说家和创业大亨！享受你的特斯拉！

我成为“专业开发人员”已经有五年了(我的 7 年级 HTML 没有支付账单)，所以和你一样，我也在谷歌上搜索“如何成为一名演讲者”和“如何成为一名更好的开发人员”太让人沮丧了！感觉就像一个游戏正在你身边进行，你不知道规则。你看着其他人变得“成功”(或在社交媒体上看起来如此)，而你却感到停滞不前。

坦白地说，我最终不得不依靠朋友、同事和 Angular 社区成员的建议来获得帮助，从申请会议到贡献开源。我就是这样为自己推导出游戏规则的。

当我开始获得一点点成功的时候，我想写下一些“不言而喻”的建议。作为一个可能比你领先几个台阶的人，我将把我自己的经验融入其中。这或多或少是我开始写的[的延续，这是一个非常害羞的开发人员为了更好的工作而建立关系网的指南](http://www.samjulien.com/shy-dev-networking/)。

我将从技术演讲开始。前几天我意识到，我第一次接受会议是在一年多前，当我写这篇文章的时候，所以所有的教训和潜规则都在我脑海中记忆犹新。在那之前的几年里，我一直在当地的集会上演讲，但我仍然是会议演讲的新手。自从第一次被大会接受以来，我已经在哥本哈根、丹佛、赫尔辛基、伦敦、奥兰多和盐湖城发表过演讲，日程表上还有拉斯维加斯和德里。

在这篇文章中，我将介绍一些战略和战术来帮助你开始技术演讲。你还会注意到这三个主题在本文中出现了很多次:**移情**、**说服**和**坚持**。

我们在这里将变得非常实际，但首先我想花点时间来思考一下*为什么*我们首先要这样做。

## 等等——为什么我们又要这样做？

你是否感到在会议上发言是获得职业成功的唯一途径的压力？随着 Twitter 的出现，“科技名人”是真实存在的。这造成了很多不安全感和职业嫉妒。

在这里我想给大家一些建议和鼓励。大多数有才华、有经验的开发人员*从未踏上舞台*。他们可能害怕公开演讲，他们可能有更好的事情要做，比如对家庭的义务，或者他们可能只是不想做。这完全没问题。你不必遵循一些 Twitter 认可的通往职业成功的镀金路径。做让你开心的事。

与此同时，我和许多开发者交谈过，他们对我说，“我从来不会做演讲。我不够好。我没什么可贡献的。”这同样被误导了。让我告诉你一个小秘密:

最好的技术演讲者并不总是最好的工程师。

我认识很多比我更好的工程师，不管是因为经验、天赋，还是因为他们把自己锁在房间里每天编程 16 个小时(你什么时候睡觉！？).说话是关于讲故事、情感和分解复杂的概念。你可以成为一部活的编程信息百科全书，但是如果你不能用一种能引起共鸣和有趣的方式来解释它，那就没什么大不了的。如果我站起来给你读完整个 ECMAScript 规范，我会是一个技术上的老手，但却是一个糟糕的演讲者。

你的主题需要一定程度的经验吗？当然了。观众可以根据主题专业知识来区分初级和高级开发人员。但是这里的关键在于*框架*，而不是扬声器的性能。如果你是初学者，不要报名去做一个关于你如何成为某方面专家的报告。将演讲框定为一个故事，讲述你学到的一些非常棒的东西，或者你想出的一个主意。人*爱*真实性；他们喜欢当一个演讲者真实地展现他们自己，并能引起共鸣。

这也是一个公认的真理，你离学习的东西越远，你就越难解释。这使得初学者的视角非常有用。你越是成为某方面的专家，你就越是开始忘记那些让你这个初学者困惑的怪癖和粗糙的边缘。

顺便说一句，在你的职业生涯中，这仍然是正确的，也是在任何媒体中创造伟大的技术内容的关键之一。今天你可能是 JavaScript 或 web 开发的初学者，但五年后你可能是机器学习或 Swift 的初学者。保持“初学者的心态”带着惊奇接近新技术，让这种惊奇渗透到你的写作和演讲中。

我喜欢演讲，因为我喜欢教学。我一直都是，甚至在我成为开发人员之前很久。我喜欢打破复杂的概念，让其他人更好地工作或推进他们的职业生涯。我没有为名声或其他什么东西说话，我只是对它有一种天生的亲和力。你可能没有同样的亲和力。也许你想旅行，或者你是为了荣耀。没关系，只是对自己诚实一点。

这引出了我的下一点。

## 申请参加会议是一项你可以学习的技能

让我们把这个弄清楚。有些人似乎一夜成名。这通常是因为他们有争议或者有某种魅力。这也可能是因为他们在正确的时间出现在正确的地方，或者有其他的“关系”,比如家人或朋友。尽管你在推特上看到了现实版本，但是这些人是少数。绝大多数人都必须努力工作，并遵循一个系统(我正在教你的系统)，即使他们看起来是凭空出现的。

让我举一个我生活中的例子。

我有一个好朋友，他改编了视频游戏的原声音乐。这是一个利基人才，但他在这方面非常出色。几年前，他发行了一张基于塞尔达游戏的专辑，并迅速走红。突然间，成千上万的人喜欢、聆听和购买他的音乐。他在 Reddit 上做了一个 AMA，我永远不会忘记关于他如何一夜成名的评论。我忍不住笑了，因为他们不知道的是，很多年前，我和他放学后会一起玩，一起作曲。我们会花几个小时学习音频软件，尝试 MIDI 样本，并想出我们自己的视频游戏重新编排。在我的朋友看到任何商业成功之前，我已经看着他练习了十多年。然而，对普通大众来说，他“不知从哪里冒出来”，一夜成名。

大多数时候，这在技术上没有什么不同，尽管它可能不会花你十年的时间。这个过程是有离群值的，但不能以此为借口。这里有一个好消息:申请并在会议上发言是你可以学习的技能，并随着时间的推移不断提高。它们既不神秘也不神奇，它们不需要护身符或魔法书，也不需要一定程度的天赋。

哲学奠定了，让我们进入本质。

## [](#crafting-a-cfp-with-empathy)用心灵感应制作 CFP

当你为会议准备提案时，想想会议在寻找什么。有时他们会在 CFP 表格上直接告诉你，否则你可以回去看看以前的谈话，了解一下他们的风格。他们倾向于选择更高级的谈话吗？他们会选择一些软(催化)技能谈话吗？您可以在起草提案时使用这些信息。

> CFP 的意思是“征集论文”或“征集建议”这些通常包括基本的传记信息、标题和演讲描述(通常称为“摘要”)

让我给你一个对所有会议都非常安全的建议:会议想要卖票。这意味着他们希望从经过审查的演讲者那里获得可靠的、有教育意义的、有趣的内容。请注意，“审查”并不一定意味着“有经验”他们希望演讲者有吸引力，准备充分，并且不违反行为准则。

“但是等等，如果我从来没有做过演讲，我怎么能通过审查呢？”

有很多方法可以做到这一点。最好的方法是开始在当地集会上演讲，并*记录自己的演讲。不要担心质量。给自己买一个便宜的三脚架，把你的手机支撑起来。你可以随心所欲地想象，但你的想法是获得一些你演讲的录音，你可以发送到会议上来支持你的提议。你可以把它们上传到你自己的 YouTube 频道，然后开始建立一个公开的作品集，或者把它们保留下来，私下发给组织者。*

如果因为任何原因你不能录制，至少开始通过[Slides.com](https://slides.com/)、 [SlideShare](https://www.slideshare.net/) 、谷歌幻灯片、GitHub、[扬声器平台](https://speakerdeck.com/)(我的最爱)，或者其他人公开发布幻灯片。(我们将在本系列的下一部分深入讨论幻灯片。)

一般来说，社区参与也是开始接受会议组织者审查的好方法。以一种不令人毛骨悚然且有益的方式与社区成员和组织者接触，会比你想象的走得更远。在 Twitter、Gitter、StackOverflow 和其他论坛上回答人们的问题。推广别人的工作。积极热情。这些不仅仅是愚蠢的陈词滥调；他们真的工作！建立一个友好和乐于助人的名声。

“我不是专家，我如何想出一个话题？”

正如我们所建立的，你不需要成为一个专家来谈论一个你有经验的话题。但除此之外，让我告诉你一个行业秘密:许多伟大的演讲者提交他们想让 T1 了解的东西的建议。你没看错。他们会选择一个主题，学到足够的知识来写一篇关于这个主题的摘要，然后，一旦被接受，学到足够的知识来写一篇演讲。在准备过程中，最优秀的演讲者还会与主题专家一起审查他们的内容，以确保他们不会偏离主题或推广糟糕的做法。

你可以充分利用这一点。如果你注意到一个热门话题，并有一个关于如何呈现它的想法，提交一份关于它的报告。

好了，你现在知道如何开始考虑你的 CFP 了。让我们开始写吧。

## [](#good-cfps-solve-problems-great-cfps-tell-stories)好的 CFP 解决问题；伟大的 CFP 会讲故事。

当我们第一次开始应用于会议时，很自然地认为它们是非黑即白的、技术性的和纯逻辑的。这从工程师的角度来说是有道理的。事实是，人类是故事驱动的。好的会谈专注于解决问题。伟大的演讲讲述改变观点的故事。

也就是说，避免“经验教训”的谈话，除非会议特别要求。实际上，每个人都有这些课程。通常，虽然，有一个良好的 CFP 的胚芽在那里。吸取教训，把它变成话题。当你在开发一个复杂的用户界面的时候，你知道影子 DOM 是如何工作的吗？将“关于影子 DOM 的经验教训”改为“在复杂 ui 中使用影子 DOM”

不过，让我们更进一步。专注于将技术问题与实际结果联系起来。人们喜欢节省时间，节省金钱，减少错误，增加销售和客户，减少编写或维护的代码量。

以下是一些关注优势的好标题示例:

*   [如何通过规划你的升级来节省时间和金钱](https://www.youtube.com/watch?v=495KHr966e0)
*   异步管道如何为我们节省了一百万美元
*   使用 TypeScript 将您的错误减少 50%

后两个是我编的，但希望你能明白。这些谈话标题展示了一个特定的好处。你也可以通过令人惊讶、出乎意料或反其道而行之来吸引注意力:

*   你可能不需要 NgRx
*   “你能想象没有区域的未来吗？” (Zone.js 是 Angular 的一部分)
*   [浏览器版本已死](https://www.youtube.com/watch?v=XP89VxObKO4)

只是不要点击诱饵，否则我会亲自跟踪你，在你的车上装满披萨传单(优惠券甚至会过期)。

## [](#the-abstract-is-not-about-you)摘要不是关于你的。

这里还有一个没人告诉你的秘密:你的演讲摘要不是关于你的。这甚至与你演讲的技术细节无关。实际上是销售文案。是的，你没看错。谈话描述有两项工作:

1.  让组织者相信这是一个值得挑选的演讲
2.  说服听众参加你的会议演讲(或者稍后在线观看)

别担心。你不需要把你的抽象变得庸俗、噱头或者老套。不过，这的确需要有说服力。这意味着你的摘要应该关注演讲的好处以及听众将会学到什么。你的写作中应该有一股强烈的同理心。

让我们看一个例子:

* * *

我的演讲是关于使用 Jest 进行单元测试。我将讨论为什么使用 Jest，定义快照测试，并演示如何在 Angular 和 React 项目中设置它。我还将展示用 Jest 进行单元测试时要避免的常见错误。

* * *

你注意到了什么？它不仅枯燥乏味，而且在短短几个句子中多次使用了“我的”和“我”这两个词。这个例子描述了演讲的内容，但没有描述它如何影响听众。看看这段改写:

* * *

你讨厌编写单元测试吗？没关系，你不是一个人。不过，有一个好消息:Jest 使单元测试至少减少了 30%。在这次演讲中，您将了解 Jest 是什么，它为什么如此棒，以及它如何在去年将我的团队的错误减少了 15%。您还将学习如何在项目中设置 Jest，无论是使用 Angular、React 还是其他方式。最后，您将了解在快照测试中要避免的常见陷阱，以确保您不会浪费本可以用来编写新特性的时间。

* * *

有什么区别？这个例子有更多的能量和幽默。它也更具体地说明了谈话的好处。最重要的是，它使用“你”和“你会学会”来直接与观众交谈。

如果你觉得这不自然，不要为此感到紧张。你写得越多，你就越能找到自己的声音。通过 GitHub、Google Docs 或像 [Ulysses](https://ulysses.app/) 这样的应用程序来保存你的 CFP，这样你就可以很容易地重复使用短语或整个 CFP。

## [](#remember-its-a-numbers-game)记住:这是个数字游戏。

当我从事金融行业时，有人给了我一本尼克·默里的书[数字游戏](http://www.nickmurray.com/gon.htm)。这本书是关于金融销售的，但有一个奇妙的思维实验，在许多领域都有帮助，包括口语。以下是我对它的解释:

* * *

想象一下赌场里的老虎机。它看起来像任何其他吃角子老虎机，但这个是神奇的。这台吃角子老虎机有一个保证:在拉动 10，000 次后，它将(不是可能，将)支付 100，000 美元。

* * *

这些知识如何影响你对老虎机的看法？你可能会一直站在那里直到达到 10，000 次拉力，对吗？

正是不确定性让我们放弃。我们满怀热情地开始一个项目或目标，然后在中途的某个地方我们失去了热情。也许我们会被拒绝。也许我们看不到结果。我们放弃了，通常有一些借口或合理化(“我只是不想成为一个演讲者。”).

像接近老虎机一样说话。申请尽可能多的聚会和会议。如果你是一个正派的人，并且是一个相对有天赋的开发人员，你会成功的(不是可能，*将会*)。好消息是:这可能不需要你尝试一万次。

## 关于寻找申请地点和跟踪申请的简短策略说明

有几个不同的网站、应用程序和 GitHub 库可以跟踪各种语言和框架的会议，所以我不打算在这里重新发明轮子。然而，我想给你一些实用的技巧来帮助你完成这个过程:

*   创建一个 CFP 电子表格来跟踪你所有的提交。你可以随意[抄我的](https://docs.google.com/spreadsheets/d/13AveQ_Jp0kN32GI4wH_JtPx5XFixJze-pr4YH59Y1Rs/edit?usp=sharing)。我跟踪会议日期、提交截止日期、会议和 CFP 的链接，并记录我提交的内容。您也可以使用类似 Trello 的工具来实现这一点。工具并不重要，它必须工作。
*   创建一个单独的 CFP 截止日期日历。
*   将你的 CFP 论文摘要保存在谷歌文档或类似于 [Bear](https://bear.app/) 或 Ulysses 的云同步应用中。这将让您找到重用和重新利用全部或部分提交内容的方法。不要只提交一次报告！你可以每年写几个扎实的提案，以各种格式提交给多个会议。有时候，一个会议会是一个独特的演讲，但是他们会把它说清楚。(这些通常是每个领域的顶级会议，如 Angular 的 [ng-conf](https://www.ng-conf.org/) 。)
*   写简短的第一人称和第三人称的个人传记，并在存放 CFP 的地方放上一张完整的头像链接。你通常还需要链接到你的网站，你的 Twitter 和 GitHub 账户，以及之前的演讲，所以做一个文档，包含你所有的“自我元数据”,以便提交时容易访问。
*   不要回避 5-15 分钟的闪电谈话提交。这些并没有变得不那么重要、不那么有声望或不那么有价值。事实上，我发现更短的演讲更难写，更有趣。此外，你有更好的机会:许多人跳过闪电谈话提交。他们可能是你的敲门砖！

## [](#action-steps)动作步骤

这里有几件事可以帮助你开始你的演讲之路:

1.  创建一个 CFP 电子表格和截止日期日历。
2.  找到三个明年要申请的会议，并将这些信息添加到你的电子表格和日历中。
3.  联系两个当地的聚会来做一个演讲。
4.  写下你的简短传记，保存在你选择的应用程序中。
