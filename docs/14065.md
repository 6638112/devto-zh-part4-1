# 地图和字典的基本采访代码

> 原文:[https://dev . to/mortoray/essential-interview-code-for-maps-and-dictionary-5ajm](https://dev.to/mortoray/essential-interview-code-for-maps-and-dictionaries-5ajm)

映射是编码中的基本数据结构之一。第二到[列举](https://interview.codes/reference/essential-code-for-lists-and-vectors-in-an-interview/)，在面试问题中比较常见，经常使用。知道如何使用地图会帮助你在许多问题上避免悲伤。

很多算法的解决方法在于正确使用地图。你越了解如何使用地图，它就越容易应用。任何时候，当你需要组织数据或跟踪项目时，地图是最好的选择之一。我将在以后的文章中写一些这方面的例子。

## [](#terminology)术语

围绕[地图](https://en.wikipedia.org/wiki/Associative_array)的术语是混杂的。

Python 和 C#更喜欢用“字典”。在这里，字典是一个特定的 1:1 映射，其中一个键映射到一个值。PHP 使用术语“关联数组”。C++用的是“map”，也有一个“unordered_map”。在 Java 中，“HashMap”是他们的“Map”接口的常见实现。

最抽象的类型被正确地称为“地图”。但是，除非你面试的是语言设计职位，否则不要卖弄学问。绝不要在面试中争论名字。学习你的语言中使用的名称，并熟悉相关的术语。

> 💭一些语言将地图实现为不可变的数据结构。然而，本文中的措辞假设您使用的是可变地图。您仍然需要所有这些操作，但是您的代码结构会有很大的不同。关注要完成的事情，而不是具体的措辞。

# [](#basic)基本

了解执行这些操作的最常用方法。每种语言都提供专门的语法，但在面试中最好坚持通用的功能。对于不断发展的语言，一定要充实自己的知识，并注意不断变化的习惯用法。

*   **创建一个地图:**使用最常用的，或通用的语言。对于静态类型语言，您需要指定键和值的类型。
*   **添加一个带键和值的元素:**将键-值对加入映射的基本方法。
*   **通过键检查元素是否存在:**检查存在是我们使用 map 的主要原因。
*   **通过键获取一个元素:**这在不同的语言之间变化很大，周围的代码通常也会变化。
*   **获取地图的长度:**几乎所有语言的语法都不同。
*   **语言习语用于从地图中“检查并获取项目:**这种组合很常见，许多语言都有特定的方式来组合这两个步骤。

# [](#searching-and-removal)搜索和删除

一旦元素出现在地图上，我们就需要重新获取它们的方法。

*   **按键替换项目:**添加和替换值的语法可能不同。
*   **按值查找项目:**虽然这不是地图的主要用途，但按值搜索项目有时很有帮助，即使这样效率很低。
*   **Get by key or default value:** 常见的习语是查找一个键，并返回值，但如果键不存在，则返回一个默认值。大多数语言都提供了专门的方法。
*   **按键删除元素:**这将删除值及其键。给键赋一个空值是不一样的。该密钥应该已经不存在了。
*   **清除地图:**从地图中移除所有元素。

# [](#iteration)迭代

我们可能想为地图中的所有元素做些什么。这可能使用该语言的循环机制，或者它们可能是地图本身的特殊 for-each 函数。

*   **迭代映射中的所有键和值对:**每次迭代都应该访问键和值。
*   **迭代映射的键:**您只想访问映射中存在的键，而不是值。
*   **迭代映射的值:**您只想访问映射中存在的值，而不是键。
*   **按排序顺序迭代:**如果不同，则按排序顺序迭代对、键和值。这可能需要使用不同的排序映射类型。

# [](#fullmap-operations)全图操作

我们并不总是希望使用完整的地图，或者希望隔离某些项目。

*   **生成一个地图的浅层克隆:**新地图将包含相同的实例。
*   **将一个映射的值映射到一个新的映射:**对于静态类型语言，确保你可以映射到一个新的类型。
*   **合并两张图的内容:**知道哪些值优先。
*   **按关键字过滤项目:**创建一个地图，或它的视图，其中只有匹配特定关键字的元素才可见。
*   **按值过滤项目:**创建一个地图或它的视图，其中只有匹配特定值的元素才可见。

# [](#mutlimaps)多重地图

多映射允许多个值与同一个键相关联。这些是 1:N 地图。它们不像 1:1 地图那样常见，但对一些算法问题很有用。

*   创建多重贴图:如果你的语言不支持多重贴图，学习如何在普通贴图中使用数组。
*   **通过键添加一个项目:**如果键已经存在，您将得到与同一个键相关联的两个值。
*   **迭代匹配一个键的条目:**对于一个键，可能有多个值。
*   **删除匹配关键字的项目:**删除匹配关键字的单个项目，但保留其他值。这可能会删除第一个或最后一个，或者需要额外的值匹配。
*   **替换所有与关键字匹配的项目:**确保多地图中不再有带有关键字的项目。

# [](#with-different-types)同不同类型

使用多种数据类型作为键和值。不要只使用字符串键！在许多方面舒适地使用地图是很重要的。

并非所有语言都支持将所有类型作为键。它可能会限制您只能使用可哈希的类型，或者具有自然顺序的类型。了解你的语言支持什么。
