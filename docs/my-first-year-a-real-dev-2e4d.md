# 我作为“真正的开发者”的第一年

> 原文：<https://dev.to/johannarlee/my-first-year-a-real-dev-2e4d>

大约 360-几天前，我开始了我的第一个前端工程师角色。这一年充满了大量的学习、挑战以及个人和职业的疯狂成长。

虽然我从来不会说，“*编码是为每个人准备的”，*但我可以诚实地说，我相信对于任何希望进入一个稳定的职业生涯并有能力定期产生影响的人来说，发展可能就是这样。

我仍然有意识地确保不低估成为一名自我导向的学习者所需要的努力和自我激励——甚至是去新兵训练营或获得大学学位——但我不认为像*不喜欢数学*或*在*之前从未编码这样的事情应该成为不考虑从事技术职业的理由。下面我将把我过去一年的一些主要发现进行分类。

## 无论你是否被雇佣，你都是一名开发者

如果你没有得到报酬，我拒绝接受任何你不是开发者的想法。你也不必成为专家。这些心态都为进入一个需要大量人才涌入的领域制造了障碍。

非高级开发人员的视角对于任何开源团队或工作场所都是至关重要的。如果文档对任何人都没有意义，那么它也可能是一篇写得很差的日记。

你从一开始就能产生的影响是深远的。技术应该让生活变得更容易，现在是时候让整个行业*停止假装编码对那些不能花四年以上时间在教室里学习算法理论的人来说是不可理解的，而我每天的大部分时间都是实践学习和实现。

## 一些最好的开发人员技能不是技术性的

在过去的一年里，我读了很多很棒的博客帖子，这些帖子讲述了作为一名开发人员，你可以拥有的一些最好的技能实际上一点也不技术性。

例如，毫无疑问，同理心是一种可以通过在设计和编写应用程序时将最终用户的需求放在心上而习得的特性。但这也延伸到你经常接触的人，他们通常会讨论编码、技术行业和你的日常工作。

主动倾听、写作以及能够通过各种形式(视频、博客帖子、推文)进行教学等沟通技能对于确保科技行业更加欢迎不同背景的所有人非常重要。

在我看来，作为一个全面发展的人，拥有开放的思想和提升他人的意愿，远比在用低级编程语言编写复杂代码方面具有超级天赋更有价值。

## 休息是健康的

我绝对是那种在永远忙碌中找到目标的人。在过去的一年里，我开始了解并喜欢我的旅程的一件事是休息——无论是从 GitHub 上的绿色小方块，还是从体育活动中休息一周——休息是改善生活的最佳方式之一。你的头脑和身体需要有意义的休息。

我们很容易陷入资本主义社会的日常琐事中。我鼓励每个人时不时地叛逆一下，补上“你的时间”。你会带着一些之前隐藏的自我真相离开，甚至可能对你所做的事情重新燃起激情！

## 剩下的还没有写出来

娜萨·贝丁菲尔德在这些标志性的歌词中有一些正确的东西。我甚至无法想象五年后这个领域会变成什么样子。唯一确定的是改变，驾驭这种改变意味着对被证明是错误的持开放态度，并相应地适应。我期待着未来的变化和所有新的 JavaScript 框架，它们将在未来几年内打破我的项目。

*在四年制计算机科学学位不再是顶级编程公司的主要资格方面，正在取得一些进展。
