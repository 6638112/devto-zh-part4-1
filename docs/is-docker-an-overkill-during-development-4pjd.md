# Docker 在开发过程中是否矫枉过正？

> 原文：<https://dev.to/bythckr/is-docker-an-overkill-during-development-4pjd>

我是一个学 python 的新手。早些时候我使用 virtualenv，我搞砸了管理它。然后，我尝试了 Docker，每个项目都有自己的问题，但我的 16gb i5 X220 的风扇不是这样。Chrome，VS code & docker 对于笔记本电脑来说似乎太难处理了。

我认为更强大的至强处理器可以解决我的问题(新手注意！).我读过关于使用 VIM+ZSH+TMUX+宋承宪的专业人士的文章，我努力想办法摆脱 VIM。我意识到我承担的太多了，vim+tmux 对我来说似乎是多余的。我现在关注的是带有 vscode & bash 的 python。现在我读到了支持 SSH 的 vscode。所以，我开始重新考虑远程开发设置的想法。

我知道 100 美元可以买到一台惠普 z400。所以，我在考虑这个选择。但后来我读到了 Pipenv 的发展，并听说了其他选项，如 pyenv 和 conda。

所以，假设 pipenv，pyenv & conda 远没有 docker 那么资源密集。我正在重新考虑使用 docker 进行开发是否明智。
