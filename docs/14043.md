没有办法处理数据。这就是为什么你需要知道你的存储选项之间的区别。有些数据在一个会话中只使用一次，而其他时候，无论用户做什么，您都需要数据。你也不需要把所有东西都存储在数据库里。

你有几种方法可以将用户数据保存在他们的电脑上，我们将介绍三种最常用的方法。它们都很相似，主要区别在于使用时的大小写。

## [](#session-storage-amp-local-storage)会话存储&本地存储

这两个没有足够的不同来得到他们自己的部分。两者只有一个很大的区别，不过先说两者是如何相似的。会话存储和本地存储都是 Web 存储 API 的一部分。它们允许你在用户的电脑上本地存储数据，这样你就不需要总是调用服务器。

这类似于 cookies 的工作方式，但是会话存储或本地存储中的任何数据都不会被传递到 web 服务器。通过使用会话和本地存储，您总共可以获得 10MB 的空间，这比使用 cookies 要多得多。当您想要保存用户设置或首选项，但又不想将它们保存在数据库中时，可以考虑使用以下存储类型之一。

当您了解这种差异时，通常很容易决定是使用会话存储还是本地存储。会话存储与本地存储完全一样，只是当用户关闭他们正在使用的窗口或选项卡时，它会被清除。一旦网页关闭，会话存储中的数据就不再保留。另一方面，本地存储保存数据，直到您删除它。

本地存储更多地用于用户设置或长期保存的用户数据，因为它只有在您要求时才会被清除。如果您有一个计算值的应用程序，并且您希望临时保存这些数字，因为您知道用户一旦关闭标签就不再需要它们，那么会话存储将非常有用。

## [](#cookies)饼干

饼干是不同的，他们最近得到了不公平的声誉。cookie 类似于 web 存储类型，因为它也只是一段文本。主要区别在于，cookies 会随每个请求一起发送到 web 服务器。这带来了潜在的安全问题，因为如果没有安全措施，有人可以修改 cookie 值并访问数据库。

开发人员可以设置 cookie 的生命周期。如果没有任何过期设置，当您关闭浏览器窗口时，cookie 将被删除。否则，只要你让它呆多久，它就会呆多久。cookie 保存的数据不如这两种 web 存储选项多，因此它最适合用于较小的数据存储，并且只用于需要在请求中发送到服务器的信息。

当您需要快速使用非敏感用户信息时，这些存储选项中的一个将很好地工作。通过在浏览器中保存某些值，您可以提高应用程序的性能，并使其对用户更加友好。我是否遗漏了其他储物选项？或者你见过这些奇怪的用法吗？我敢肯定，有人用这些储物选项做了一些奇怪的事情，让你发笑。
