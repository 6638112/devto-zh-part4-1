# 后悔最小化

> 原文：<https://dev.to/alexgascon/regret-minimization-a8c>

最近我一直在读一本关于杰夫·贝索斯和亚马逊如何成为今天的互联网巨头的书，书名是《点击一下:Amazon.com 的崛起和杰夫·贝索斯》。我仍然在第 5/17 章，但到目前为止，我发现最好奇的是，每次贝佐斯做出决定时，不仅是关于业务，也是关于他的个人生活，他都会采取令人难以置信的分析方法，研究所有可能的结果和可能影响结果的方面。

体现这一点的方式之一是通过他所谓的“后悔最小化框架”。基本上，它是一个工具，通过分析哪一个将是他未来可能后悔较少的一个，来帮助他在一组选项中进行选择。在这本书里，当他不得不选择继续在一家金融软件公司做他的(真正的)高薪工作，还是离开它去开创一个互联网业务的时候。

> “我知道，当我 80 岁的时候，我不会后悔在年中拿走了 1994 年华尔街奖金。我都不记得了。但是我确实认为有一个机会，我可能会非常后悔没有参加这个叫做互联网的东西，而我对这个东西充满热情。我也知道，如果我尝试过但失败了，我不会后悔”

我最近不得不自己做出一个重大的个人决定(换一份新工作，搬到另一个国家，或者继续一份仍然是一个很好机会的工作)，即使我没有这样命名，“后悔最小化”肯定是影响我决定的最重要的因素。如果你想到这一点，这可能是做决定的最佳方式，因为它迫使你不仅考虑你将从每个选项中获得的即时满足，还考虑其长期后果。

一个看似无关紧要但对我来说是框架中最重要的部分之一的细节是他如何考虑好的和坏的结果。最后，很明显，事情不会总是如你所愿，确保你不会后悔一个特定的决定，即使它完全错误，这可能是你在做出选择时最需要考虑的因素之一。

决策是将对你的生活产生最大影响的技能，在这个过程中有一套清晰的步骤可以遵循，无疑会帮助你从中获得最佳结果。即使考虑到事情可能会出错，你现在做这个决定是否合适？以后想好了还能换一个选项吗，会有什么后果吗？最重要的是:10 年、20 年甚至 50 年后，你能接受这个决定吗？
