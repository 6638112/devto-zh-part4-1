# 获得开发人员工作的技巧

> 原文：<https://dev.to/devpato/tips-to-get-a-job-as-a-developer-4ic3>

大家好！

这是我的第一篇帖子！我希望有些人会发现这有助于他们作为开发人员找到第一份工作。我相信下面的建议会帮助你找到工作，即使你已经有了一份开发人员的工作。如果你有更多的建议，请在评论中添加，我会将它们添加到列表中！

1.  没人关心你是去了耶鲁，还是去了技术学校，或者你是不是通过看 YouTube 学会了如何编程。只要你知道你的东西，你是好的。如果一家公司关心你在哪里上学，而不是关注你的技能，你可能不想在那里工作。

2.  贡献开源。怎么会？github([https://github.com/](https://github.com/))是一个开始或者去 NPM 寻找项目的好地方。

3.  随时更新和完善你的简历。仅供参考:我免费审阅简历。在 Twitter 上给我发个 DM[@ dev Pato](https://dev.to/devpato)。

4.  在 LinkedIn 上完成您的个人资料。说真的，这就是科技招聘人员找到你的方式！

5.  活跃在 LinkedIn 上。分享你读到的，你的想法，你正在学习的，等等。因为前面提到的所有事情，招聘人员每周都会联系我几次。永远保持你的 LinkedIn 的专业性。

6.  联系招聘人员。招聘机构的候选人比直接向公司申请的人有更高的机会获得面试。

7.  有个人网站。

8.  在你的文件夹中至少有 3 个项目，并在你的简历中谈论它们。

9.  确保你写的每一行代码都在 Github 上。Github 是开发者和招聘人员的组合，他们也可以在 Github 上找到你！；)

10.  在 Stackoverflow 上回答和发布问题。

11.  在 Stackoverflow 上创建一个开发者故事。你是怎么做到的？转到您的个人资料->点击开发者故事选项卡。一旦完成，你会看到类似我的[https://stackoverflow.com/users/story/5262452](https://stackoverflow.com/users/story/5262452)的东西。这有助于招聘人员找到你，帮助你找到你想要的工作。

12.  面试时展示你对编程的热情。

13.  展示你学习新技术的愿望。

14.  活跃在推特上。推特？等等，什么？？？是的，推特！Twitter 是一种与开发人员见面的非正式方式。你永远不知道你将与谁互动，以及潜在的机会。

15.  数据结构和算法是最重要的东西！谎言！大多数开发人员永远不会使用疯狂的算法或疯狂的 DS。你还需要专注于学习最佳实践，如何将不同的工具如 NPM 软件包集成到你的代码中，API 等等。

16.  如果你在 Indeed 或 LinkedIn 上快速申请的 200 家公司都没有回音，也不要气馁。

17.  不要点击“快速应用”到职务。大部分贴着“快速申请”的工作都是假的。多花几分钟申请并正确回答表格上的问题，等等。

18.  不要在简历上撒谎。你写在简历上的所有内容都是面试官可以提问的。

19.  去聚会！聚会是一种很棒的社交方式，可以让你的名字出现在那里。还有，招聘人员去聚会:)

20.  参加黑客马拉松。如果你的学校或你所在城镇的某人正在举办黑客马拉松，请参加！即使你认为自己不够好，无法参赛。黑客马拉松并不总是关于竞争，而是关于学习、社交和享受快乐时光！另外，他们通常提供免费食物。

21.  在 Stackoverflow、LinkedIn 等网站上找工作。Stackoverflow 非常适合远程作业:)

22.  在 Stackoverflow 和 LinkedIn 上设置首选项，指定您正在寻找新的机会。

23.  找工作的时候在 twitter 上发帖。我们不止一个人会转发！

24.  担心你的绩点？该死，很抱歉大学不适合你。幸运的是，这并不意味着你不知道如何编码。事实上，我认识很多非常有才华的程序员，他们的平均绩点很低。

25.  找个导师。有一个导师总是好的，即使你是一个高级开发人员。导师可以帮助你做出决定，在你的职业生涯中走上正确的道路。

26.  做一些让你对自己的学习和目标负责的事情。

你不必做这些建议中的每一条。它们是关于如何找到一份开发工作的提示，而不是规则或公式。

祝你找工作好运！
