# 心理模型阴谋

> 原文:[https://dev.to/justzeros/mental-model-machinations-10o6](https://dev.to/justzeros/mental-model-machinations-10o6)

在过去的两周里，有两个想法一直在我的脑海中萦绕。他们现在互相碰撞，产生了几封电子邮件。

今天的邮件是关于第一个想法，它来自第四波系统思维。这是“心智模型”的概念，我们一直都在使用。

想法是这样的:*我们通过我们的心智模型间接地**看到现实**。*

神智不清，对吧？

想象你戴着标有“心智模型”的护目镜，通过它们观察一切。

例如，现在，我正坐在一个由布、钢、拉力、齿轮和液压系统组成的系统上。我的心理模型说我“坐在椅子上”，对此我毫不犹豫。

我还用我的手指快速按下一系列安装在矩形上的发光黑色塑料方块。我的心理模型说我在“键盘”上“打字”

我们都有心智模型，并且一来到这个世界就开始形成它们。

例如，我 1 岁的孙子还不会“打字”，但他会用手敲键盘，可能是模仿我。

我还不知道他对键盘的心理模型是什么，但有一天我们会分享同一个模型。

## [](#these-go-to-eleven)“这些去十一。”

由于其抽象的本质，软件开发将心智模型提高到 11。

我们不仅有关于物理现实(计算机、显示器、网络等)的心智模型。)，我们有围绕软件(应用程序、网站、数据库、web 服务器等)的心智模型。)

除此之外，还有所有关于构建软件(文件、架构、模式、框架中的代码)和过程(敏捷、看板、质量、测试、部署、分支等)的心智模型。)，而且很容易混淆。

如果心智模型是我们用来理解世界的镜片，为什么我们不多谈一谈呢？

# [](#what-do-you-discuss)你们讨论什么？

我注意到人们经常分享用心智模型创造的信息，但是他们很少分享他们正在使用的心智模型。

分享信息是好的，但这通常就像分享一个残酷的事实，比如“42”

心智模型是产生信息的“引擎”，比如“将数字 20 和 22 相加”

你可能注意到了，还有其他的心理模型可以产生“42”

不足为奇的是，当人们争论一个技术话题时，他们没有相同的思维模式。
