# 我从 Greg Mckeown 的书《本质主义:对更少的严格追求》中学到了 3 点关于软件开发的经验

> 原文：<https://dev.to/educostadev/3-lessons-i-learned-on-software-development-from-the-book-essentialism-the-disciplined-pursuit-of-less-by-greg-mckeown-5ecf>

在这篇文章中，我将分享我在阅读这本神奇的书时学到的一些经验。这本书既不是时间管理的秘方，也不是一本自助的书。而是关于如何成为一个本质主义者。一本教你如何最大限度地利用时间做对你来说重要的事情的书。我可以说这是我读过的最好的书之一，我全力推荐。书上说:

> "记住，如果你不优先考虑你的生活，别人会的."

阅读这本书让我清楚地了解我们在做无用的事情上浪费了多少时间。我从中吸取了一些教训，可以应用到软件开发中。

### 第一课:越少越好

真正的本质主义者奉行这一原则。不是如何完成更多的事情，而是完成正确的事情。有多少次我们只是想增加一个功能？或者设计并实现一些我们认为有用但用户从未使用过的东西。

通常我们做的一个决定会引导我们做出十几个未来的决定。在软件开发中，这个概念与敏捷原则'[少即是多](http://www.agilearchitect.org/agile/principles.htm#Less%20is%20More)'密切相关。

### 第二课:运用本质主义方法

### 探索和评价

花尽可能多的时间评估和探索各种选择。倾听用户和同事的意见，花时间提问、思考和辩论。探索不是路的尽头，而是选择重要事物的方法。

### 消除

消除那些你不确定是否能给你的产品或最终用户带来价值的活动和努力。有时候很难去掉一些东西，我们倾向于添加一些很酷的功能。但是记住少而精！

### 执行

消除障碍，使执行尽可能容易。开始工作前，花必要的时间建立必要的环境。CI/CD 管道应该已经准备好，旧的单元测试正在工作，用户的故事已经定义，等等。这将防止开发过程中的停顿，并使编码更有趣。

### 第三课:保持专注

避免任何与你正在做的事情或你打算实现的目标不一致的任务。把注意力集中在你计划要做的事情上。当一组不相关的类与你正在做的事情无关时，不要重构它们。或者宁愿报告一个 bug 而不告诉别人。
