# 我对编码没有热情

> 原文：<https://dev.to/tsimpson/i-m-not-passionate-about-coding-4949>

# 关于我

首先，我想简单介绍一下我是谁。我浏览开发人员的帖子已经有一段时间了，通过最终建立一个帐户(主要是为了给我遇到的但没有时间立即阅读的帖子添加书签)，一些评论开始出现在真正让我自己陷进去的帖子中。我是一名软件开发人员，从来不觉得自己已经准备好或者足够好，但是喜欢挑战和前进。我的志向不是编码本身，而是创造事物。原来，编码就是创造东西，只是不是我特别想创造的东西，但那是另一篇文章的内容。

# 激情

激情在开发社区中经常被谈论。你会经常看到“如果你想做 X 或 Y，你必须对编码有热情”或“如果你对编码没有热情，你就不能得到 X 或 Y”的帖子，或者更激烈的“如果我们看不到你对编码的热情，我们就不能雇用你。”没关系。反正那可能不是我想去的公司。激情永远是一个燃烧的话题，似乎是面试官和候选人的亮点和最重要的决定点。

但是...什么是激情，作为一名软件开发人员，什么样的激情是值得的？让我们谈谈那个。

激情有很多定义，在这种特殊的情况下，它仅仅意味着

> 对某事的强烈愿望或热情。

当我想到激情的这个定义时，有几件事特别浮现在我的脑海中(没有固定的顺序):

*   艺术，无论是素描、油画、写作、音乐等等。创造给人们带来快乐的事物。
*   帮助他人成长和成功，同样，帮助企业成长和成功。
*   教育。尽可能多地学习任何感兴趣的话题，无论是科技、科学、数学、文学、音乐、健康还是更多。继续教育和自学。
*   探索。探索未知领域的兴奋，你从未见过的地方，你从未爬过的山，你从未游过的海洋。或者也许..一个你从未涉足的职业领域。探索你不知道的东西，我想这很可能符合教育。

这些都是我热爱的事情。每一个都互相连接，同样的，也连接到软件开发。写代码不是我的激情，但是创造人们喜欢的或者可以用来改善他们或他人生活的好东西才是。我试图跟上新技术和最新趋势，但我总是发现这太费时间和单调。我不太感兴趣。但真正重要的是学习。学习和探索未知。漂亮、闪亮的新玩具并不在我的清单上，尤其是在开发中。更重要的是保持稳定并使用已知有效的工具，并且在这方面做得很好。所以我花了很多时间研究更普通的开发，特别是 JavaScript。我想知道来龙去脉，以及那些最容易被忽视或忽略的小细节，因为这对他们来说不方便，或者看起来没有真正的意义。当我投入时，我真的...*真的*潜进去。我沉迷于小事，直到我把它们分解到不能再分解的程度。

# 为什么

我想下一个问题是...**为什么**？这是我自己经常折腾的一个有趣的问题。为什么我不像其他人那样对发展充满热情？为什么与同龄人相比，我感到不成功或表现不佳？为什么要继续追逐我并不直接热衷的东西呢？

我喜欢挑战。在投身于这个世界之前，我一直在追求在任何一家公司达到我能达到的最高水平。当我最终到达那个平台时，很明显没有其他职位可以晋升，或者他们出于某种原因不允许我晋升，我继续前进。我的最后一根稻草是在不到 6 个月的时间里从一个简单的商店送货司机变成了助理经理。这很有趣，令人兴奋，而且....没有挑战性。一点也不。我学到了更多关于管理、培训他人(我在之前的心理健康和特殊需求领域已经做了多年)、运营等方面的知识。这是一个很好的见解。但是它没有挑战性。一旦我找到了模式，我就能够简化日常流程，从而减少时间并提高商店的销售额，这很棒。不幸的是，很明显，无论我做什么，他们都不会让我晋升到商店经理或区域经理或任何比我更高的职位。不过没关系。我厌倦了没有新东西可学，也没有梯子可爬，所以我提交了两周的通知，一头扎进了软件开发。

但是 Travis，如果你对它没有热情，你为什么要选择它呢？

因为新的经历，对自我教育的无尽需求，巨大的成长空间，让我们面对它吧...这是世界正在慢慢转变的方向。它还提供了关于如何更好地优化业务的新见解。它使我能够优化国会竞选中一个小团队的表现，在美国最重要的选举时期之一，减少了他们的工作时间，同时增加了志愿者注册和选民投票。这让我能够看透并看到缺陷，以及如何在我目前作为一名“一级技术支持专家”工作的公司中纠正这些缺陷。我一直在为这家公司重建一些网站，同时也在重建他们混乱的知识库，就如何优化和简化销售点系统最常见问题的基本故障诊断步骤提供建议。我已经能够深入了解他们公司内部网络安全的不足。

在多短的时间内我要走多远？我已经跟他们说得很清楚了，我不会留在这里做技术支持。我讨厌打电话。我一生都在为我的家人、朋友、同事和同事提供技术支持。一旦你学会了这些模式，就没有挑战性了。如果我们不能在我的时间范围内行动，没关系。他们还没准备好，所以我会继续前进。与此同时，我仍在积极寻找我的第一个真正的软件开发职位。这花了我一些时间，但我已经开始接受这一点，与此同时，继续提高自己，当合适的公司选择我的时候。

# 未来

那么未来会怎样呢？谁知道呢。我知道我想花至少 3 年左右的时间从事软件开发。我知道有几个不同的业务我想推出自己的。帮助改善其他企业很有趣，但看到我自己的“孩子”成长会更有趣。我知道我需要写更多，特别是通过我自己的编码过程，让公司更容易看到我思考、计划、处理和创造的方式。所以这是我的第一步。

# 你呢？

你呢？你热衷于什么？是软件开发吗？如果有，具体是什么？是别的原因吗？如果有，是什么？我完全相信有更多像我一样的人，他们对开发没有特别的热情，但对技术、教育等充满热情，不会让他们的代码或他们的公司因此而遭受损失。我想这是最让我恼火的部分。"你对编码没有热情，所以你可能写的代码很潦草或者不知道怎么写."嗯，我是新手，所以是的，我正在学。随着时间的推移，我将学会优化自己的代码，使其更具可读性，编写测试等等。但我也喜欢花时间追求自己真正的激情，其中编码可以很好地让我受益，所以我学习如何编写更好的代码并在某种程度上跟上新技术是非常重要的。
