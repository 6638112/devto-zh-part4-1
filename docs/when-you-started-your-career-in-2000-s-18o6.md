# 当你在 2000 年开始职业生涯时...

> 原文：<https://dev.to/portekoi/when-you-started-your-career-in-2000-s-18o6>

首先，为我的英语道歉。

我是一名法国开发人员，下个月我就 40 岁了，在 IT 界已经 18 年了。

在我的第一篇文章中，我想谈谈我在这个领域工作多年后的感受。

面对所有的新技术，对于许多在 2000 年开始编程的程序员来说，很难为一个新项目找到好的解决方案/语言/平台。

我还应该用 PHP 吗？还是另一种语言比如 C#，Python 或者 NodeJS？

我应该使用像 containers (Docker)、AWS 或 Azure 这样的云技术，还是使用专用服务器？

我应该使用微服务还是整体架构模式？

前面呢？我应该继续使用 Boostrap/Jquery 这样的经典 UI 还是继续 Angular，Vue。Js？

冒名顶替综合症呢？

从我的角度来看，许多问题并不只有一个答案。

### 我的事业

我来描述一下我的职业生涯。也许你今天也有同样的情况，同样的问题和感受。

关于编码，我在学校学过汇编和 C++。在我学习期间，我通过在 Word 上通宵达旦地编码，独自学习了 PHP 3、HTML 和 CSS(我知道...)然后，Web Expert 和 Dreamweaver 3。这个时候我拿了一个今天已经不存在的公司的共享服务器。

在我拿到高等教育文凭后，我在 2001 年左右开始了我的网络创业生涯。在这家公司，我学会了 ASP 3...ASP 3 不是面向对象编程的最佳语言，所以我丢掉了关于实体、抽象、继承、多态和其他许多概念。当你不练习时，你会忘记。

2005 年，我受雇于一家银行，没有人用新技术编程。所有其他程序员都在 DB2 / AS400 环境下使用 Cobol 和 RPG。
因此，对我来说，挑战在于将这个“旧”界面连接到网络上，通过学习 Cobol 和寻找解决方案，我成功地做到了这一点...

2009 年，在用 ASP / PHP 5 / Cobol 编写代码这么长时间后，我开始了向 C#的迁移，因为对我来说，这是最好的选择，但我的对象方法更像是过程代码。我避免学习 Silverlight 和 Web Form 技术，因为我不相信它们。(即使今天，我还得管理一个这方面的项目...)

2012 年，我还在银行，但一个联系人把我介绍给了他的一个朋友，因为他正在为一个新平台寻找一名网络开发人员:销售电子书和视频等虚拟信息，能够进行 A/B 测试，处理漏斗转换...我接受了挑战。

所以在白天，从早上 9 点到晚上 7 点，我在为银行工作。从晚上 9 点到凌晨 1 点，我在为这个客户工作...

我学习了营销策略，如何使用 Google Analytics，Adyen，Paybox 和 Paypal 支付网关，如何推出新产品，电子邮件策略，管理 Wordpress 等。

这不是我的个人资料，但我必须适应和学习。

所以今天，我在编程中学到了很多东西，但不仅仅是。

## 冒名顶替综合症来了...

而且，对我来说，这就是一个程序员(初级/高级)的核心:阅读、练习、学习。

但是对我来说，当你读得太多，学得太多，得到太多信息时，你就不能确定你为一个新项目选择的技术，因为:

1.  也许这项技术会像 Silverlight 一样消失
2.  你练习得不够，所以你的代码不会在规则中被创建(`Hello world`是不够的)
3.  这不是关于每个身体语言的最后一种技术/模式(目前是微服务)

和许多其他原因...

所有这些都让你感觉不到冒名顶替综合症。这就是我今天的问题:确保我每天做的事情仍然是好的。

当我与“老开发者”交谈时，他们中的一些人也有同样的感觉:我是个骗子，因为我并不掌握所有最新的技术。

这种综合症可能是戏剧性的:你每天早上醒来都想着有人会揭开你的面具，取代你。认为你今天所做的一切都是不正确的。最后，有时候，你会质疑自己的生活，这是最糟糕的感觉。

这是许多开发人员的问题:没有掌握所有最新的技术，感觉自己过时了。

## 这一切都是正常的

但我可以回答:你不能。你不可能掌握所有的新技术。对我来说，没有人能做到，因为你需要每天练习才能不忘记它，这是不可能的，除非你同时有 15 个项目，使用许多不同的平台...

对我来说，你必须选择一种你觉得舒服的语言。比如选择 C#或者 PHP，乖一点。如果你需要，请牢记 OOP 的基础并尽力而为。为 PHP 选择一个框架:Laravel，Symfony，Slim...但是不要试图去学习所有的知识。

你不掌握微服务还是孤军奋战？所有以前的项目都是以单一模式完成的，许多项目仍然存在，并且很容易改进(即使有时技术债务是巨大的)。

通过问你以下两个问题来选择一种语言/技术:

1.  我能用它完成这个项目吗？
2.  我可以自己管理这个项目，还是使用另一个开发人员来管理他？

如果你对这两个问题的回答是肯定的，那就去吧。别问问题了，尽力而为。

因为这对我来说是决定性的，也是最重要的一点:尽你最大的努力，即使你认为这对其他程序员来说不够好。

他们没有你的技能。

所以，如果你是一个像我一样的老程序员，相信你自己，相信你的能力，在你的项目上尽你所能。
