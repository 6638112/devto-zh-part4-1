这是推销你作为开发人员的技能的第二个更好的方法，而不仅仅是“雇用我！”或者大量申请工作。

在第一部分中，我们讨论了为什么“喷洒和祈祷”的方法——你发出数百份申请或者只是在 Twitter/LinkedIn/etc 上宣布你在找工作——通常是一个糟糕的策略。

不好，因为你用一个完全通用的推销来推销你的技能和经验。在这种情况下，你最终看起来就像其他申请相同职位的开发人员一样。

我们还讲述了如何研究公司，这样您就可以用了解他们的业务问题和技术挑战所需的知识来武装自己。这对于向公司证明你正是他们需要雇佣来帮助他们解决挑战的人是至关重要的。

但是你如何向公司证明这一点呢？你如何进行调查并推销自己，让你潜在的未来公司开始将你视为他们需要雇佣的人？

这就是流程的第二步的作用。

老实说，这本身就是一整本书——推销你作为开发人员的技能是一个巨大的话题。

但是我们在这里要介绍的是你的研究，弄清楚它如何与你的经验相吻合，并精心制作一个“推销”。

你获得推销的方式可以有很多种形式——电子邮件、申请表、和公司同事一起喝非正式咖啡等等。

但是形式并不重要，重要的是你能根据你的技能和经验告诉公司你能做些什么。

### 第二步，展示你如何帮助解决这些问题

**1。将你的经历与他们的问题相匹配**
现在你已经有了一份公司面临的问题/挑战的清单，创建推销的第一步是列出你的经验和技能，这些可以帮助你解决这些问题。

基本上，把你的经历和公司的挑战“匹配”起来。你如何利用你的技能和经验来帮助解决他们的问题？

显然，这将因经验水平而异，但即使是寻找第一份开发工作的人也可以利用学校经验、课外活动、实践项目、其他职业的经验等。你可能需要更有创造性，但是不要排除非开发人员的经验。

要考虑的一件事是:把自己想象成一个帮助解决问题的员工会更有帮助，而代码恰好就是你如何做的。当你这样想的时候，如果你没有丰富的开发经验，那就变得越来越不重要了。只要你能找到符合公司需要的过往经历。

**2。将它整理成简历格式**
不管你是否需要一份你所申请的工作的简历，我发现把上一步的清单拿出来，修改你的简历使之适合是很有用的。

这不仅是因为你有一份更好的简历，还因为它有助于你创造你的“销售卖点”。

所以，如果在你的研究中，你发现你申请的公司在过去的 6 个月中用户数量有了很大的增长，因此对业绩产生了很大的影响...您碰巧有过加速 API 以缩短响应时间的经历...在你的简历上，你应该写上:

> “-在核心应用流量增加 50%后，API 响应时间缩短 35%”

记住(无论是你的简历还是你的“推介”)，最好总是**量化**和/或**限定**事情。

从“为什么我建造的东西很重要？”。而不是“我做了什么？”而是“我为什么要这么做？这有什么关系？它有什么结果？”。

例如，不要放:

> -构建 React 应用程序，供客户查看和更新订单

请放:

> -构建了 React 订单应用程序，使收入增加了 5%，客户保持率增加了 4%

在你的简历中，你不需要只列出对公司有帮助的事情。毕竟，为每份申请制作一份完全**独一无二的简历有点麻烦。**

但是你一定要使用“为什么，而不是什么”的格式来描述你要包含的内容。这是因为，即使你的经历现在可能对公司没有直接好处，也不意味着将来不会。你希望他们看到你有解决问题的历史。

如果你在其他公司也这样做过，那么招聘经理或者任何正在看你的申请，亲自和你谈话的人，等等。将更容易看到你如何在他们的公司做到这一点。

**3。创建一个短间距**
下一步是创建一个短间距。我拍摄 1-2 段，主要是因为这感觉上足以推销你自己，但不会太长，以至于接收端的人失去兴趣。

这实际上应该很容易——所有这些步骤都建立在彼此的基础上，这没有什么不同。

从上面的简历中找出要点，然后选择你认为最能帮助公司应对挑战的要点。在简历中，你列出了你做过的事情，最重要的是，为什么这些事情很重要。你还加入了有助于解决公司(你有兴趣加入的公司)问题的东西。

通过这种推销，你实际上只是更清楚地表明了你的经历将如何帮助公司。如果他们还没有根据你的简历把这些点联系起来，那么这份简历会帮他们把这些点联系起来。

使用上面简历步骤中的示例:

> -在核心应用流量增加 50%后，响应时间缩短了 35%

将它转换成音高格式就变成了这样:

> 我从你网站上最近的新闻部分看到，随着你最近的产品收购，你与所有额外客户的流量大幅增加。这是我有很多经验的事情，我很乐意帮忙。在 ABC 公司，在 3 个月的时间里，我们的流量大幅增加，我能够诊断瓶颈并减少 35%的 API 响应时间。

**4。修改音调以适应环境**
最后，这应该是显而易见的，但你需要修改你的音调以适应任何环境。

如果一份申请需要一封求职信，我会把它放在那里。如果他们没有，我会在邮件正文中加入推介，并附上简历。

如果是在线申请，我会把内容放在一个自由格式的文本框中(他们通常有这些)。

如果是一次非正式的咖啡聊天，在现实生活中见面，我只是确保涵盖一些球场上的事情，但要自然地融入对话中，绝对不是一口气脱口而出。尽管我一直称之为推销，但如果你正在与某人面对面交谈，我不会像在销售会议或其他场合那样*字面上*陈述推销，哈哈。

正如我在这篇文章的开头提到的——根据具体情况，推销可能有多种形式，但关键是展示你如何通过将你的技能和经验与他们的需求相结合来实现价值。

### 模拟的例子

让我们将上面的策略应用到一个模拟的例子中——有一个虚构的公司(实际上可能存在),名为“Infotech ”,我们有兴趣为其工作。

在浏览了他们的网站和工程博客后，我们发现了以下内容:

*   由于客户的大量增加，他们遇到了扩展问题，因此在过去 3 个月中为缓存层和队列系统实施了 Redis，以处理更繁重的后台作业
*   他们为医生开发了一种新产品，可以自动为病人申请实验室
*   他们的一个大客户是北方州立医院

以下是我们的技能和经验列表:

*   在 CDE 软件公司，在 Redis 中实现了一个缓存层(编写一个包装器模块，帮助客户访问 Redis)
*   在 XYZ 医疗保健软件公司，与产品、UX 和最终用户密切合作，重新设计了 iOS 应用程序中的计费工作流程
*   该公司的一个客户是南方州立医院(一个地区性竞争对手)

幸运的是，我们的技能/经验非常匹配！

显然，事情不会每次都如此完美。这是意料之中的。

但是，如果围绕你如何创造价值进行一些创造性的思考，并进行一些创造性的研究，你会惊讶地发现，你经常拥有一种特定的技能或特定的经验，而这种技能或经验会转化为公司需要的东西(不仅仅是他们在工作描述中列出的东西)。

现在让我们来应用这个过程...
对于根据公司的挑战整理简历，这是我们的经验:

*   在 CDE 软件公司，在 Redis 中实现了一个缓存层(编写一个包装器模块，帮助客户访问 Redis)
*   在 XYZ 医疗保健软件公司，与产品、UX 和最终用户密切合作，重新设计了 iOS 应用程序中的计费工作流程

这是一个好的开始，但让我们进一步量化/限定它。为什么这种经历很重要，这些项目的结果是什么？

*   在 CDE 软件公司，在 Redis 中实现了一个缓存层(编写一个包装器模块，帮助客户访问 Redis)；**由于无需每次往返，核心 API 响应速度提高了 45%**
*   在 XYZ 医疗保健软件公司，与产品、UX 和最终用户密切合作，重新设计了 iOS 应用程序中的计费工作流程，**导致客户支持团队的呼叫减少了 20%**

那看起来好多了。

现在，根据我们对该公司的了解，我们来做一个简短的推介。下面是它可能看起来的样子(想象我们正在通过电子邮件发送我们的简历，这是电子邮件的正文，不包括一般介绍):

“在 CDE 软件公司，我在 Redis 中实现了一个缓存层，使核心 API 响应速度提高了 45%。当我在那里工作时，我们的流量在短短 3 个月内有了巨大的增长，我们真的需要扩大规模。我知道在医疗保健 IT 领域经常会出现这种情况，正如我从您的新闻栏目中看到的那样，您的客户最近有了大幅增长(恭喜！).性能和扩展可能是痛苦的，但幸运的是我在这方面有很多经验。

说到医疗保健 IT，我注意到您的一个客户是北方州立医院。当我在 XYZ 医疗保健软件公司工作时，我们的一个客户实际上是南方州立医院。我为他们重新设计了一个核心计费工作流程，我知道它有点类似于您的新实验室请求产品。我能够从最终用户那里学到很多他们是如何工作的，并获得了很多特定于这些工作流的领域知识，我知道我可以将这些知识带入 Infotech。

感谢您的考虑，我很乐意谈谈如何利用我过去的经验为 Infotech 创造价值。

你可以用完全不同的方式，用你自己的语气来写，但希望这足以表达你的意思。

### 如果你在那家公司已经有关系了呢？还需要卖吗？

如果你在公司已经有关系，你很可能会被自动带去面试(甚至可能跳过一些面试步骤)。但是面试过程还是一个销售过程。这不仅仅是一次面试，而是一次更大的销售。

得到一次面试机会，公司就买下了你的简历/申请/无论什么——你的“推销”——他们支付的是人力资源部 30 分钟的电话，他们的一个开发人员的一两个小时，等等。

但是在面试过程的最后，如果他们决定“购买”你在那里“卖”的东西，只要你在那里工作，他们就会支付一个开发者的工资。那可是一大笔钱。

所以这就是为什么我说，即使你不需要一开始就出售你的技能和经验来获得面试机会，同样的过程——研究公司发现他们的问题/挑战/真正的需求，然后展示你如何帮助解决这些问题——无论如何都是非常有益的。无论如何，你都会在面试中用到你所做的研究，你的经验，以及你的“推销”！

### 包扎

找工作的过程——发送申请，亲自与现有员工见面，等等。-整个面试过程都需要销售方法。

一个似乎经常只是列出我们知道的技术和我们做过的事情的问题。

没有量化/限定工作产生的影响，没有将其与公司当前面临的挑战/需求相匹配。

但是这里描述的过程更加具体和可行。我认为这会让你更容易被视为公司需要雇佣的人，因为他们可以**想象出**你将如何帮助他们以及你将帮助他们解决什么问题。

最重要的是，我的建议是避免**喷洒和祈祷**的方法，并根据你提供的商业价值来考虑你自己。

我认为你会有更高的成功率，避免找工作时被忽视的痛苦，最重要的是，你会得到非常适合你和公司的工作。
