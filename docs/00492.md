# 除了. NET - Prologue 什么都没有

> 原文：<https://dev.to/carlwatkins/nothing-but-dotnet-prologue-3i12>

# 昨天，我以开发者的身份参加了我的第一次用户组聚会。

这是一次我们聚在一起观看直播的活动。NET 大会主题演讲，吃点披萨，聊聊天，拿点奖品。大家都很开心。这不仅是一段美好的时光，也是微软的一次巨大成功，它将我与他们未来的. NET 计划联系在了一起。

# 那么我是谁？

我是个无名小卒。我只是一个在 30 多岁时转行，在社区大学完成计算机科学学位最后一个学期的人。我几乎是任何一个现在想进入开发游戏的人。我有一些一般知识，但几乎没有实践经验，我所拥有的专业经验分散在 JavaScript、C#中。NET 和许多其他技术。

# 为什么一个无名小卒可能很重要？

因为以上所有原因，我是越来越多年轻程序员的代表。有越来越多的开发工作，但没有足够的经验丰富的开发人员。这意味着越来越多的企业将不得不开始吸收初始培训的成本。微软和开发人员。NET 正在提供一个具有可转移技能和全栈解决方案的工具集。

作为一名崭露头角的开发人员，我很欣赏这一点。NET 可能会给我一套技能，可以跟踪桌面可执行文件、web 和移动应用程序。我在一个领域发展的技能只需很少的调整就可以转移到其他领域。所以，恭喜微软，你在你的平台上卖给我了？

# 接下来是什么？你在做什么?

这最后一个学期的挑战性明显低于之前的任何一个学期，你每天只能花这么多时间来找工作。我只是在看卡通片，但这不会帮助我发展技能，也不会引起别人的注意来帮我找工作。为了让我的时间过得更有成效，我将开始一个新的。NET Core 3 项目。具体来说，我将利用 Core 3 推出后可用的所有花哨技巧，将我的乏味、陈旧、基本的 HTML 个人网站改造成一个乏味、过度设计的 SPA 网站。

在这个过程中，我只是作为一个刚从大学毕业的初级开发人员，花时间学习这些新工具和设计，给出我的反馈和见解。

# 你为什么要这样做？

基本上是因为我无论如何都要做，所以我不妨分享一下我的经历？也许会有帮助或者通知别人？我也期待得到反馈。有时，我可能会向读者征求反馈，告诉他们我下一步应该做什么，或者我应该从哪里寻找解决方案。还有，我只是有点无聊。

# 这是教程吗？

绝对不行。我是一个专业的白痴，当我在这个项目中阅读我的观点、想法和理论时，应该考虑到这一点。我不喜欢把自己标榜成任何类型的专家，也不希望人们有其他想法。

我将张贴一些截图和我自己的代码示例。我可能会把整件事作为回购上传，以防有人真的对此感兴趣。这样做并不意味着它是好的代码/设计或者应该被复制/模仿。

也就是说，我会定期尝试链接到我在自己的研究中偶然发现的教程、文档和其他资源。然而，我会挑战检查我的来源，而不只是假设他们是有效的。那只是很好的练习。

# 你为什么让我问你这么多问题？

1.  我把它作为一种叙事手段，把整篇文章联系在一起。
2.  这是一个微妙的技巧，希望能让你对这个职位产生兴趣。
3.  这是权力的问题。

# 我应该什么时候期待这个系列的第一个正式帖子？

希望在这个周末之前。我有作业，工作面试等。这是一个个人项目，所以开发可能会不一致，但我会尽我所能至少每周发布一些东西来开始这个项目。

感谢阅读。
