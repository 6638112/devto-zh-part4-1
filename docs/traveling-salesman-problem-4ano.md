# 旅行推销员问题

> 原文：<https://dev.to/piyukore06/traveling-salesman-problem-4ano>

几周前，我听说了一个无法解决的数学问题。它激起了我的兴趣。我对此进行了研究，以下是我的发现。这篇博客文章献给所有的数学爱好者。

自文艺复兴以来，每个世纪都比前一个世纪解决了更多的数学问题，然而许多数学问题，无论是大的还是小的，仍然没有解决。旅行商问题是`hard`优化问题类中的经典算法问题之一，两个多世纪以来一直被标记为`unsolvable`。

*   问题陈述📜

给定 n 个城市，你可以选择哪条最有效的路线去参观每个城市，然后再回到你出发的地方

*   听起来简单，为什么无解🤔

这听起来很简单，尤其是当你在用电脑工作的时候。你所要做的就是检查每一次往返的距离，最短的一次就是你的答案。有`n`个城市，就有`(n-1)!`种可能。我们只计算一半，因为每条路由都有一条长度或成本相同的反向等价路由。所以是`(n-1)!/2`。所以你走吧！我们破解了？

> 这还不够好。出于显而易见的原因，这种解决问题的方式被称为幼稚解决方案。

*   最大的为什么？？

多年来，科学家们已经提出了更新的算法来解决这个问题。在计算机科学的早期，人们想出了涉及特定数量城市的场景的解决方案，但用单一算法(即，计算机可以遵循来找到解决方案的单一规则列表)解决每个旅行推销员问题的方法仍然很难实现。20 世纪 70 年代，数学家理查德·卡普发表了一篇论文，提出我们可能永远也找不到答案。他证明了这个问题是 [NP-hard](https://simple.wikipedia.org/wiki/NP-hardness) ，这意味着永远不会有算法来解决它😲
