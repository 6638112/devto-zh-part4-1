在采访中，阿里分享了她的经验，不仅是作为一名开发人员，也是一名有成就的老师。她强调了教学是检验和巩固你的知识的一种不可思议的手段。她告诉我，她的学生会问一些极具挑战性的问题，迫使她从基本原理中理解概念。

正如阿里所说，教学迫使你理解一门学科的原理。

阿里的故事鼓励我在我的社区和网上投资教导他人。

以下是采访中提到的一些事情:

### 1。你可以异步教学

教学不一定意味着教室。当然可以，阿里在那种环境下很舒服，但这不是唯一的办法。我从阿里身上学到了很多东西，但我们从未见过面。

无可否认，阿里已经变得善于将自己的教学能力和知识提炼到文章中，然后发布到网上。

阅读阿里文章的体验就像在教室里一样，因为你读的是一个非常擅长向他人传递信息的人写的东西。

这是任何人都可以效仿的(见:这篇博文)，也是你可以学习的技能。我现在正在练习，我会建议任何阅读这篇文章的人试一试。

还有，不一定是博文。如果你擅长演讲，可以建立一个播客或视频博客，如果你想做视频，那就去做吧。直播一些编码会话。做一些事情来上传你的经历，这样别人就可以向你学习。

### 2。为自己创建内容

如果你决定在网上分享你的经验并异步教学，你会在一开始就努力定义你的受众。

有一个简单的方法可以解决这个问题:为年轻版的你创建内容。

阿里分享了这个建议，它立即引起了我的共鸣。从字面上看，我所有的内容都是针对过去的我或者未来的我。

任何人都可以这样做。可以是技术性的，也可以是非技术性的。我写了很多关于 git 的文章，因为我每天都在使用它，但我经常忘记细节。我的文章给了我一个记住这些细节的地方。

这也有心理健康的成分(当你的工作是对代码进行推理时，心理健康是必不可少的)。专注于建立追随者，创建大量的博客帖子，或者每周流足够多的内容是令人疲惫和不健康的。不要为了别人的赞同而制作内容。

我播客上的其他几位嘉宾将为自己写作描述为简单地开源你的笔记。通过向开发者社区提供您的知识，这是一种更加健康的发展方式。

### 3。你的声音很独特

阿里在她的作品中很好地诠释了这一点。

在我们的聊天中，她提到了一些她觉得被遗忘的书呆子:星球大战，指环王，等等。因此，她欣然接受了自己的教学风格，她找到了机会，让那些不符合传统工程模式的人更容易理解和理解科技内容。

她的[链表指南](https://dev.to/aspittel/thank-u-next-an-introduction-to-linked-lists-4pph)是她颠覆主题能力的最好例子之一。

这是至关重要的，因为它驳斥了每个曾经考虑过写博客的人的想法，“有人已经写得比我更好了。”

好吧。随便啦。也许有人写了比你今天能写的更有技术含量或娱乐性的内容，但是没有人在他们的内容中捕捉到你的声音。

您的声音可能会传达给与现有内容无关的人，因此，尝试分享您的声音和观点对于开发人员社区的发展非常重要。

### 4。构建独特的视角

你是如何打造这种独特的声音的？多样的人生经历。

Ali 获得了计算机科学学位，她自学了 web 开发，并在训练营教书。如果你问阿里关于学习编码的意见，她会有很多经验可以借鉴。

如果你能让自己接触到各种各样的经历，你会发现自己吸引了别人看不到的联系，这就是你独特声音的本质。

效仿阿里，走多条路。

### 5。在你的社区开始教学和指导

Ali 分享了她在 DC 开发者社区的经历，并建议其他有兴趣分享知识的开发者在他们的领域中寻找社区。

如果你真的感觉到在一对一的教室式环境中与他人联系的驱动力，从在你当地社区寻找低承诺机会开始。

去聚会和咖啡店，在那里其他开发者可能会有问题。做 15 分钟的演讲。举办一个[提问开发](https://askadev.org/)活动。

尝试现实世界，同步教学可能会为你打开一个新的视角，它绝对会迫使你做你的研究，理解你所教的主题。

成为一名教师并不一定是一个严肃的承诺，对开发人员社区的贡献可能相对来说并不痛苦，但是这些行动将对你的职业生涯产生积极的影响。在这方面，阿里是一个很好的榜样。
