# 在公共场合学习&拯救文明

> 原文：<https://dev.to/manoeltlobo/learn-in-public-save-the-civilization-2j37>

> 或者“一个确实重要的东西的稍微有点吸引人的标题”。

创造这种写博客习惯的原因之一是养成在公共场合学习的习惯。尽管这个概念并不算新，但它被[王自如](https://twitter.com/swyx)在[的一篇博客文章](https://www.swyx.io/writing/learn-in-public/)中完美地呈现了出来。

这个习惯本身就相当于一整篇文章(甚至是一系列文章！)，但我想谈谈我最近发现的一些副作用。看了一段[张克帆吹](https://twitter.com/Jonathan_Blow) <sup id="fnref1">[1](#fn1)</sup> 、“[防止文明崩溃](https://youtu.be/pW-SOdj4Kkk)”的谈话视频，对其前提深感好奇(也有些害怕)。

在他的演讲中，Blow 谈到了过去文明发展的许多技术和设备。知识和工艺的奇迹吸引着我们，即使我们的技术相对先进。有些东西是我们今天无法实现的，即使我们拥有聪明的头脑和机器。

我所能理解的这种知识丧失的主要原因，正是传递这种知识的困难。向后代解释我们的方法和成就所依据的概念和背景是多么困难啊！没有这种共享，知识最终会被孤立。技术及其技术概念变得陈旧和枯萎。尽管如此，我们可以在终端中使用一个简单的命令，然后一切都会正常工作！除非它不会。

我们所有的工具依赖于一系列的抽象。这一系列的抽象可以被追踪到原子级别。使用这些工具是可以的，它们给我们的日常生活带来了很大的变化。但是这些是在很多层上的实现，可以隐藏起来。当工具失败或者我们需要与现成提供的略有不同的东西时会发生什么？

当我们不知道某样东西是如何工作的，而我们需要学习它的底层结构来解决问题时，我们最终会学到很多东西。如果我们分享我们所学到的，尽管看起来很小很简单，即使是为了我们未来的参考，我们也解开了这些信息，并为整个世界提供了一个备份(谢谢，互联网)。我们允许这种智慧蓬勃发展，被重新混合，并且服务于比仅仅解决你的问题更多的东西，停留在你的头脑中直到你忘记它。

我们现在有这种福气，能够在任何地方，任何人，只要用我们的指尖，就能找到我们想要的任何东西。我们可以从这个世界甚至其他地方获得关于一种疾病、一种材料或更多的最新学术发现。我们还可以找到如何同样轻松地修复那个讨厌的 webpack 配置。

我们开始分享和讨论我们所知道的怎么样？我们可能最终会拯救我们的文明。
