# 当你从视频教程中学到一些东西时要小心

> 原文：<https://dev.to/orkhanfarmanli/be-careful-when-you-learn-something-from-a-video-tutorial-4m07>

视频教程是学习新语言、框架、库等的好方法。有些人从视频教程中学得更好，有些人则不然。这是一个个人的选择，这意味着我们不应该总是逼迫初学者(比如我😅)在他们开始开发的积极性很高的时候阅读数百页的文档。我记得当我第一次尝试学习编码时，我从我的程序员朋友那里得到了一本 400 多页的 C++书。由于这样的编程入门，我的动机几乎消失了。

视频教程的问题是它们有时也会教给开发者一些不好的实践。渴望尽快收获的创作者，并不总是深入研究技术，尽快贴出教程。你知道有时候你在写代码的时候会觉得有些不对劲，但是你不知道是什么原因，因为你在这个问题上没有足够的经验。是的，我说的是那些时光。不管教程是免费的还是付费的。

我从 2015 年开始写代码，作为一名自学者，我看了很多教程。我注意到的是，教程的评论并不总是代表内容的质量。大多数时候，评论者都是初学者，他们对所学的主题了解不多，而且他们通常会忽略创作者的错误行为。这是因为他们感谢教程的存在，这是他们能负担得起或找到的提高自己的最好的教程。克服这个问题的最好方法是和已经知道你正在学习的东西的人讨论，这会给你一个很好的视角。你可以自己研究，但是和你认识的人辩论总是一个好的选择。(在教程中直接写信给创作者也是一种选择，但有时它可能会反击。)
