# 有趣且高效的会议技巧

> 原文:[https://dev . to/Valerie codes/tips-for-fun-and-productive-conferencing-34en](https://dev.to/valeriecodes/tips-for-fun-and-productive-conferencing-34en)

会议是一个很好的职业机会，无论是在学习资源方面，还是在与你所在领域的人建立持久的个人和职业联系方面。但是，如果你是这个领域的新手，或者对“网络”这个概念感到厌烦，那就有点令人畏惧了，所以我列出了一些想法来帮助你充分利用你的会议体验。Ruby 和 RailsConf 有一个学者/向导项目，为社区新人和更有经验的会议参与者牵线搭桥。下周我将第三次担任向导，所以我将分享一些我通过这个项目与我遇到的学者分享的和从他们那里获得的知识。
我写这篇指南时考虑到了我参加过的科技会议，但这条建议可能也适用于其他会议。

* * *

**制定战略**
对于一个多轨道会议，将会有许多会议同时进行，所以至少要对你想在什么时候参加什么会议有一个模糊的概念。请记住，在大多数会议上，演讲都会被录制下来，所以如果有研讨会和演讲的混合，你最好去参加你感兴趣的研讨会，获得实际操作的互动体验，然后在视频上观看演讲。把你想看的演讲列一个清单，这样当视频出来的时候你就不会忘记了。
如果你不想在每个时段都去听讲座，也没关系。你可以找一个安静的地方来处理和综合你到目前为止所听到的，或者在走廊上闲逛和聊天。

了解你的局限
会议让人筋疲力尽。如果你在下午晚些时候没有精力或注意力进行超级技术性的谈话，那也没关系。或者，如果你知道你宁愿以后再看视频，这样你就可以暂停和倒带。也许你不喜欢下班后的活动。或者也许你是，但你需要先打个盹。你可以随心所欲地来来去去，充分利用你的经历。这包括如果你感觉不到谈话在进行中，就停止。

放下笔记本电脑
我知道，你在参加一个科技会议，这是违反直觉的。但是不要在发言人提到的时候去查找或者试着记笔记，而是写下笔记或者在推特上直播。例外的情况是在工作坊中，你实际上是在练习中写代码。

如果你正在用手记笔记，在休息时突出重要的点来发微博。否则，在推特上直播你参加的讲座。确保记下或找到演讲者的 twitter 账号，如果他们有的话。如果有的话，也使用会议标签。这是一个与演讲者交流的好方法，他们会很高兴听到你印象最深的观点，这也是一个与其他与会者和整个科技 Twitter 社区交流的好方法。

**进行 1:1 和小组互动**
和成百上千的人一起参加同一个会议会让人感到害怕，所以确保你在小组中有时间。午餐时和某人聊天，或者找几个人一起喝杯咖啡或吃顿饭。随着你参加更多的会议，这将变得更容易，因为你将开始看到熟悉的面孔。
