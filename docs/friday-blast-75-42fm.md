# 星期五爆炸#75

> 原文：<https://dev.to/horia141/friday-blast-75-42fm>

[tensor flow . js-web 及更高领域的机器学习(2019)](https://blog.acolyer.org/2019/02/04/tensorflow-js-machine-learning-for-the-web-and-beyond/)-tensor flow . js 论文概述。令人印象深刻的是，它如何通过 WebGL 处理 JS 中的训练和推理！它在着色器和图形操作方面使用了表达非图形代码的“旧”技巧，性能相当不错！看起来 JS 和 GPU 之间的抽象层并没有那么大！

[机器学习研究中的斯文神话(2019)](https://crazyoscarchang.github.io/2019/02/16/seven-myths-in-machine-learning-research/) -这是一个非常有趣的阅读，尽管它确实将 ML 与神经网络混为一谈。有趣的是“每一个数据点都被用来训练一个神经网络”的神话，以及它是如何将支持向量机和神经网络的方法与输入分布的探索联系起来的。也就是说，两个*都有点*关注有趣的例子，通常在被评估的决策边界或表面的边缘。

[Supercookiess (2019)](https://www.johndcook.com/blog/2019/02/08/supercookies/) -除了常规 cookie 之外的“其他跟踪”方法概述。John D Cook 在这个主题上花了一些时间，但是可以说，不管是否启用了 cookies，我们都很容易被跟踪。

[AWS 架构简史(2019)](https://cloudonaut.io/a-brief-history-of-aws-architectures/) -你的公司卡在哪里了？无论如何，我很好奇有多少人大规模使用过 AWS 的大型“容器”或“无服务器”产品。我对他们的一些产品有混合的结果，看起来很神奇——大量的错误，大量不直观的性能行为。当人们不愿意从他们的*已知但有缺陷的*系统切换到*未知但有缺陷的*系统时，这是可以理解的。

[每个程序员都应该知道的 AWS 成本(2019)](https://david-codes.hatanian.com/2019/06/09/aws-costs-every-programmer-should-now.html) -非常酷的 AWS 定价页面合成。专注于基本计算(最便宜的是一个现代 4 核 CPU 每月 30 美元)、存储、带宽和内存。这差不多涵盖了它，因为许多其他服务都是用这些基本单位来表示的！
