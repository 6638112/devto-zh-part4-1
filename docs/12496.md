# 书评:重构 UI (2018)

> 原文:[https://dev.to/awwsmm/book-review-refactoring-ui-2018-1bjo](https://dev.to/awwsmm/book-review-refactoring-ui-2018-1bjo)

我喜欢这本书，因为它不乱:有一个标题页，目录，然后你会立即提出一些伟大的设计建议:

> *“从功能开始，而不是从布局开始”*

这是本书第一章第一节的标题。作者解释说*设计*不同于*美学*。当你在开发一个像应用程序或网站这样的产品时，最重要的是它的*功能*。首先决定你需要什么 UI 元素来满足这个功能，然后围绕它来构建你的产品。谷歌的主页只包含一个搜索框，但它是互联网上最受欢迎的网站之一。

我觉得我不能在一篇评论中准确地总结这本书，因为每一句话都充满了伟大的设计技巧。这本书只有大约 200 页长，而且——因为有很多图表——你可以在一个下午的几个小时内轻松地读完它。感觉就像坐下来和一个有经验的 web 开发人员聊天，听他们解释他们在职业生涯中积累的每一点设计建议。

这本书帮助我把我认为任何人能给出的最重要的设计建议用语言表达出来，那就是:“保持一致”。不是“一致”,意思是一切都必须是相同的，而是你应该选择一些不同的字体，一些不同的颜色，等等，并且在相同的上下文中以相同的方式使用它们。

作者称之为“视觉层次”，并建议你提前开发它，这样你就不会在开发你的网站或应用程序时不断为微小的设计细节而烦恼。为强调、正常和不强调的文本选择 2-3 种色调。使用较重的字体来强调重要信息。选择一种配色方案并坚持下去。明智地使用空格。避开相对大小。

依我拙见，对没有设计头脑的人来说，两个最大的“暗示”是:

1.  间距不一致或不合适
2.  布局不一致

我无法告诉你，在你网站的每一页，或者你的 Powerpoint 演示文稿的每一张幻灯片(可能除了主页/标题幻灯片)的同一个地方，你的公司徽标有多重要。看起来很不专业(我敢说“马虎”吗？).这本书强调了这些观点以及更多。

正如我说过的，很难用几句话来概括这本书。如果你对视觉设计感兴趣，尤其是网页设计，我保证你会爱不释手。我会把它推荐给任何从事 UI/UX、前端开发、网页或移动设计的人。实际上，我会把它推荐给任何曾经需要设计报告、制作幻灯片或设计小册子的人。有一些超越媒体的设计规则，这本书为设计专业外观的产品提供了一个很好的基础。100%推荐。
