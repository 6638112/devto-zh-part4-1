# 我作为软件开发人员的第一周

> 原文:[https://dev . to/bbarbour/my-first-week-as-a-a-a-software-developer-24p 7](https://dev.to/bbarbour/my-first-week-as-a-software-developer-24p7)

我写这篇文章的时候是周六早上，我承认，这仍然感觉超现实。周一，我作为社区品牌的 fullstack 开发人员开始了我的工作，在 NimbleAMS DevOps 团队工作。这是美妙的一周，我喜欢这里的工作环境。

我的团队很棒。用“好”来形容他们似乎是微不足道的。这不仅仅是职业礼貌和客套话。在我的一生中，我第一次感觉到每个人都为我的到来而兴奋。有一种难以描述的真诚和真实性。

也许是因为我在 IT 行业支撑了我的整个职业生涯，所以大多数人都变得麻木不仁，习惯了跳槽。在某种程度上，这不是一个容易茁壮成长的领域。这需要一种特殊的人来唤醒并保持高度的积极性来解决问题和与沮丧的顾客/客户争论。许多人在多年的压力下变得疲惫不堪。甚至我不得不多次在情感上脱离我的任务。大多数时候，我都摆出一副客户服务的面孔，进入自动驾驶模式。我希望有一段安静的时间来摆弄电脑，因为那是有趣的部分。

软件开发似乎有不同的平衡。毫无疑问，有些同样的东西在那里，只是剂量较小。只是梦想软件和学习新事物的行为在一天中变得活跃起来。对我来说，我将编写人们现在会真正使用的东西的想法是疯狂的。

我整个星期都在致力于研究和吸收尽可能多的关于我们产品和系统的信息。现在我只是一块人体海绵。我们的团队在他们的 AppExchange 上为 Salesforce 构建定制软件。事实上，我们的产品 NimbleAMS 是 AppExchange 上最大的应用和产品。

Salesforce 本身就是一头野兽。我发现自己对其能力的广度和深度感到震惊。即使在连续几天阅读和钻研细节之后，我也只是触及了皮毛。

到目前为止，我认为这可能也是一个不利因素。我真的不能以一种有意义的方式做出贡献，至少现在还不能。我知道这是我的第一周，但我之前学到的东西很少与 Salesforce 的工作方式相关。

在我的学习过程中，我主要练习 HTML/CSS、Javascript、React 和 Node。Salesforce stack *的*有 Javascript(否则我可能不会有这份工作。)Salesforce 创建了他们自己的整个前端 Javascript 框架，称为 Lightning Web Components，我必须深入研究。大部分后端代码使用 Salesforce 特有的编程语言 Apex(它基于 Java。)我要做的另一大部分工作是使用 C#。这两个我都不熟悉。

这是我必须学习的三件主要的事情，并且足够精通才能做出贡献。我的团队理解这一点，并致力于帮助我实现这一目标。我只希望我对 Javascript 的深层理解在这里会有用。不过，我可能只需要对自己有耐心。
