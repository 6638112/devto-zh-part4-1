# 一个 GraphQL API 来管理它们

> 原文:[https://dev . to/take shape/one-graph QL-API-to-rule-them-all-13f 7](https://dev.to/takeshape/one-graphql-api-to-rule-them-all-13f7)

### [](#with-mesh-all-the-services-you-depend-on-can-work-together-as-one)有了 Mesh，你所依赖的所有服务可以作为一个整体协同工作。

我们喜欢 JAMstack，因为它允许我们为我们的网站和应用程序使用最好的服务。然而，在服务之间组合和组织数据，然后确保数据保持同步，这可能需要开发人员投入大量精力。更不用说，我们必须了解每个 API 的特殊性，以维护和扩展应用程序。

**这就是我们建造[异形网](https://www.takeshape.io/signup)的原因。** Mesh 通过组合来自多个服务的数据、同步对该数据的更改以及从单个 GraphQL 端点获取对您的网站或应用程序至关重要的数据，帮助您轻松高效地开发 JAMstack。

## [](#compose)作曲

Mesh 的作曲简单而强大。现在，您可以将来自多个服务的数据组合并转换到一个 API 流中。将电子商务与分析数据相结合从未如此简单。
[![Alt Text](../Images/546e08219da6aa3879b6454c86eae819.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--WOPWTA8u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kt2gdm4ljja2fcmzyq6z.png)

## [](#sync)Sync

Mesh 保持服务之间的数据同步，因此一个服务的变化会反映到所有服务中。让每项服务成为事实的来源，并简化您的工作流程，因为您知道您的网格将保持一切同步。
[![Alt Text](../Images/6630df397ea1bfda940bd966b97ecb19.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--AwXX71FM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/t5e17qgs87p22lp671yp.png)

## [](#fetch)取

Mesh 通过为您的网站或应用程序所需的所有关键数据提供单个 GraphQL API 端点，使查询变得简单。与许多不同端点相关的混乱和错误修复已经成为过去。
