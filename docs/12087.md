## [](#why-it-matterspointleft)为什么重要👈

*   更好地理解你的用户
*   缩小您期望用户使用方式和用户实际使用方式之间的差距
*   营销数据分析

## [](#how-to-do-itpointdown)怎么做👇

### [](#on-client-side)在客户端

*   同步、发送和等待
*   异步，发送并运行
*   发送业务组件
*   发送单独的服务
*   为未发送的事件保留一个本地缓存，以便以后重新发送
*   本地缓存大小限制
*   API 调用超时/失败处理

### [](#on-server-side)在服务器端

*   日志收集，集中所有后端服务器的所有日志
*   根据请求基于预定义的事件数据结构生成事件
*   直接到数据库
*   通过消息队列发布

### [](#employ-a-thirdparty-service)雇佣第三方服务

*   直接请求
*   集成 SDK
*   共享数据库/独立数据库

## [](#data-definitionpointup)数据 definition☝️

### [](#event)事件

*   事件类型，发生了什么
*   CreatedAt，什么时候发生的
*   来源，在哪里发生的
*   有效载荷、参与人员/内容

### [](#result)结果

*   StatusCode
*   状态消息
