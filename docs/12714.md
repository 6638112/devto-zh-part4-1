# 静态网站生成的卖点

> 原文：<https://dev.to/megazear7/the-sales-pitch-of-static-site-generation-27fk>

我最近开发了一个名为 [Orison](https://orison.alexlockhart.me) 的静态站点生成器工具，为此我一直试图向人们“推销”静态站点生成器；包括开发人员、架构师和非技术人员。对许多人来说，术语“静态站点生成”是“小型博客”、“个人站点”和“琐碎用例”的同义词，而不是像“企业规模”和“无与伦比的性能”这样的短语。

我非常感谢 [Netlify](https://www.netlify.com) 为推广静态站点生成所做的一切。然而，我怀疑这个术语的命名是否会让人们望而却步。在技术文章、演示、博客帖子和个人讨论中，我经常使用术语“构建工具”、“构建过程”或类似的东西来描述静态站点生成器在 [JAMstack](https://jamstack.org) 架构中扮演的角色。

我很好奇你发现了什么。我错过了什么更好的词吗？我们是否应该关注“静态站点生成器”这个术语的更名？这有关系吗？

提前感谢您的任何回复。主观的人类经验通常是一个未被意识到的技术障碍，尤其是在这种情况下，我认为这是一个值得讨论的问题。
