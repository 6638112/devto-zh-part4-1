# 何时选择测试自动化框架而不是手动测试

> 原文:[https://dev . to/emailforadil/when-to-choose-test-automation-framework-over-manual-testing-4m9j](https://dev.to/emailforadil/when-to-choose-test-automation-framework-over-manual-testing-4m9j)

如何确定哪个框架适合您的组织？你应该选择测试自动化框架而不是手工测试吗？

在这个快节奏的世界里，人们需要从他们的产品和服务中获得便利、创新、安全、设计等等。这促使公司快速开发和测试软件，以便在这个竞争激烈的市场中茁壮成长。为了快速满足这些需求，企业利用自动化技术。

众所周知，手工测试是乏味的、耗时的，并且需要人力资源的高投入，因此为了避免所有这些问题，自动化测试是组织的首选。因此，让我们探索一下测试自动化框架，什么时候使用它而不是手工测试，以及如何选择正确的框架。

自动化框架是一套全面的指导方针，它为自动化测试脚本提供了一个执行环境，以产生自动化测试活动的有益结果。指导方针可能包括编码标准、测试库、测试工具、通用实践和预期结果的假设。但是，知道一个组织何时应该选择自动化框架而不是手工测试是很重要的。当一个开发团队在同一个应用程序的不同模块上工作时，混合使用不同的方法会造成混乱，从而增加出错的可能性。因此，对于组织来说，为其特定需求选择理想的框架并将其应用于每个模块是非常重要的。

使用测试自动化框架的一些关键好处或目标包括:

*可重用测试:*高效设计的测试自动化框架更容易维护，也更有用。

低成本维护:可重用的自动化测试导致低成本维护。

*无人工干预:*无需任何人工干预即可运行自动化测试的能力。

*简单透明的报告:*提供清晰、明确和有针对性的报告，适合不同信息或详细程度的不同类型报告的要求。

以下是通常使用的测试自动化框架的标准类型:
根据测试人员的需要和要求，有一些广泛使用的具有不同功能的测试自动化框架，它们是 QAs 的首选，包括:

**线性脚本框架:**这个框架包括以线性的方式记录和重放测试脚本，没有或者很少修改。这是自动化软件测试的第一代测试方法，并且具有最简单的框架方法。线性脚本框架主要用于测试 web 应用程序的用户界面，其中测试人员记录他们的一步一步的动作，并回放记录的测试。

**数据驱动框架:**该框架允许测试人员将测试脚本逻辑和数据存储在外部数据库中，如 XML、CSV、Excel 电子表格、文本等。这种类型的框架的最好的特性是，它减少了覆盖所有测试用例所需的脚本数量，并且它能够基于所使用的数据用代码修改测试场景。数据驱动框架有一个缺点，即外部数据源的组合可能会使软件工具崩溃，并使手动测试变得复杂。

关键字驱动的框架:在这种类型的框架中，电子表格上的表格根据指定的关键字决定测试脚本的动作。这个框架提供了与数据驱动框架相同的好处，还有一个额外的好处，那就是用户不需要有丰富的编码知识。该框架也是独立于应用的，即它不依赖于应用、编程语言或自动化工具。关键词驱动框架的主要缺点是时间和金钱的初期投入较高。

**模块化测试框架:**这种类型的框架将测试中的应用程序划分为孤立的模块，这些模块可以组合起来形成更大的测试脚本。模块化测试框架将用户与应用程序的交互分解成更小的测试，然后重新组合它们以实现特定的目标。它是可扩展的，并且具有高度的模块化，这使得维护更加容易和便宜。

**混合测试框架:**该框架是上述多个框架的组合，以利用各自的优势。因此，它结合了所有相关框架的优点，避免了缺点。大多数为现代软件开发周期开发的测试自动化框架都是混合性质的。
为了有效，这些自动化框架应该具备一些关键要素，例如:

*应该易于维护并可重用
应该可扩展以满足未来需求
具有可靠性和一致性
具有高效率
应该具有最大的测试覆盖率*

如何确定哪个框架适合您的组织？每个团队和每个项目都有自己的测试数据管理、测试行为规范、错误处理、报告等需求。在最终确定组织的理想框架之前，您应该问自己以下问题:

a)应用程序是否有复杂的流程？看一下测试要求。

b)用户体验如何，考虑应用中使用的技术？

c)确定与每个框架相关的成本。

d)您的团队能轻松地插入其中一个框架吗？评估你的团队在组织中的技能。

e)什么样的指导方针会给出自动化测试的预期结果？

f)在设计满足所有要求的方法之前，哪些工具最适合您的组织？

g)维护测试自动化的人员的技术熟练程度如何？

h)自动化需要哪些类型的报告？

最好创建一个记分卡来评估几个参数，包括脚本、集成、生成的报告、使用等。同时评估多种工具。测试自动化框架将建立实现自动化测试预期目标的指导方针。但是一个合格的、熟练的测试自动化专家应该参与自动化框架的设计和实现或者软件工具的选择。

在当今的现代软件开发方法中，测试自动化框架扮演着重要的角色。即使是中小型企业也期待从自动化框架中获益，以满足对其产品和服务的需求，并为未来的需求做好充分准备。
