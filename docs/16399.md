# 开始写博客

> 原文:[https://dev.to/jixion/getting-started-blogging-5lc](https://dev.to/jixion/getting-started-blogging-5lc)

我最近开始了一份新工作，一直在努力学习一些新的 JS 框架。在参加了一些课程后，我开始了一个项目，并想开始写关于这次经历的博客帖子。我发现开始写第一篇博文很困难。我在前进的道路上遇到了绊脚石，这阻碍了我实现我的目标。我想在这里分享这些绊脚石以及我是如何越过它们的。

开始写我想象中的第一篇博文，当时看起来并不困难。我从记录我所采取的步骤开始，在我意识到之前，我已经有了一个关于如何重复我所做的事情的完整的步骤列表。这不是一篇博客文章。我反复阅读这些步骤，心想这会让某人回到我现在的状态，但是没有包括任何关于我为什么选择这条路线的实际信息。我还意识到，步骤列表很快就会过时，对读者来说就没那么有用了。

需要做出一个决定。我需要扩展我所提供的信息，但我不确定如何前进。它感觉如此不堪重负，我放弃了。这是我第一次尝试在博客上记录我的学习经历，它结束得和开始时一样快。

我在这里遇到的一个问题是，我期望我可以坐下来立即成为一名技术博客作者。这个假设是完全不正确的。正如一位同事今天提到的，这些事情需要练习。这个问题正在通过写这篇博客来解决！这是我实践的开始。希望随着时间的推移，我会在这个过程中变得更好。

在与同事交谈后，我注意到的另一个问题是，我需要思考一下我想要传达的内容。在我的第一次尝试中，我认为唯一重要的事情是记录我所采取的步骤。虽然这可能有些用处，但我认为这些步骤背后的原因同样重要，如果不是更重要的话。我需要花一点时间在我希望我的帖子传达的信息上。这是我今天花了一些时间写这篇文章的内容。这里的想法不仅仅是记录这个过程是艰难的，哦，不，我放弃了，不能前进了。这也是为了记录问题，并提供一个故事，讲述我如何从这个过程中吸取教训，以及我可以做些什么来为下一轮做得更好。

所以，我就到此为止了。问题已经被克服，我现在有一个职位。不久将会有关于我如何着手创建我的最新项目的帖子。
