# 有多少人使用 Github 动作，为什么？

> 原文：<https://dev.to/akhiljp_dev/how-many-use-github-actions-and-why-13fe>

我真的很好奇，开发人员和 QA 工程师是否喜欢在 web 应用程序中创建功能性和可执行的代码块，而不是将其编写为 python 脚本？

例如，如果我是一名使用 GCP-APJ 和 AWS-US-east 存储图像的开发人员，那么我必须在内部创建一个脚本，与这两个云供应商 API 挂钩，以执行类似的功能，如上传新内容、删除图像等。

给小组的问题:
a)作为一名开发人员，你会使用 Github Actions 这样的系统来做这件事吗？如果是，为什么？
b)如果您不打算使用 Github 动作，那么您会将 Github 动作用于哪些其他用例？
