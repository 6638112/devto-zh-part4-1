# 脚本语言和编程语言的区别

> 原文：<https://dev.to/jay_tillu/difference-between-scripting-language-and-programming-language-4dmi>

首先，请记住，他们之间的确切区别是非常模糊的，在今天的条件下，它也不是那么重要。但是如果你想成为一名优秀的程序员，那么你应该清楚这两个术语。

两者的区别在于它们的编译过程。

## 脚本语言

* * *

脚本语言不需要编译，而是被解释。是指脚本语言使用解释器将其代码转换为本机代码。

这里还要记住，脚本语言比编程语言运行得慢。而且他们也不能直接访问硬件的底层细节。

例如，您的 C 程序需要一个编译器来将其代码转换为本机代码。但另一方面，JavaScript 不需要编译器，而是使用解释器将其代码转换为本机代码。这就是 JavaScript 有时被称为“脚本语言”的原因。

#### 脚本语言的应用

*   自动化某些任务(构建像 Gradle 这样的系统，使用脚本实现自动化)
*   从一组数据中提取信息

脚本语言的例子(这些语言通常在没有明确编译过程的情况下使用。但今天，其中一些可能会有所不同。)

*   Java Script 语言
*   左上臂
*   Perl 语言
*   计算机编程语言
*   服务器端编程语言（Professional Hypertext Preprocessor 的缩写）
*   VBScript 等。

## 编程语言

* * *

编程语言是需要编译器将它们的代码转换成本机代码的语言。与脚本语言相比，编程语言运行速度更快。此外，编程语言可以访问底层硬件细节。

**编程语言的例子**

*   Java 语言(一种计算机语言，尤用于创建网站)
*   我的锅
*   C
*   C++
*   斯卡拉
*   太棒了等等。

但是，在将这种语言划分为脚本语言或编程语言时，部署它的环境也是需要考虑的主要问题。这是因为我们可以根据需要为任何语言设计编译器或解释器。我们也可以为 C++，C 或任何其他语言设计解释器。

比如 Python 在没有编译器的情况下被广泛使用，但是 CPython 的主要实现需要编译器。JavaScript 也是如此。所以部署也是主要的考虑因素，因为我们可以根据需要为一种语言设计编译器或解释器。

我知道今天这两者之间的区别非常模糊，但是作为一个程序员，请记住它们之间的基本区别。
