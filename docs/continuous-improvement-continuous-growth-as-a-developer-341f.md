# 作为开发人员的持续改进/持续成长

> 原文：<https://dev.to/lewiskori/continuous-improvement-continuous-growth-as-a-developer-341f>

这个故事最初发布在我的个人网站上。
这个帖子已经来了很长时间，非常讽刺的是我决定在今年写更多，但是哦，他们说，迟到总比不到好。

我两年多前开始编程。这一切都始于一天下午偶然看了一些 C 语言教程。当时，这一切对我来说都很陌生，老实说有点挑战性。尽管如此，只要我有空闲时间，我就继续投入时间，并最终设法根据各种教程构建了几个项目。这个旅程并不容易，阅读这篇文章的一些程序员可以证明这一点。一路走来，我学到了很多东西(有些是艰难地学到的)，但所有这些最终都让我不仅成为了一名优秀的软件工程师，也成为了一个更好的人。这是旅途中获得的一些珍贵的金块。

### 1。入门指南

新程序员问自己的第一个问题是
***“我从哪种编程语言开始？我的上帝，他们这么多！”***
当你被如此多的选择轰炸时，这种想法可能会令人恐惧和不知所措。有些朋友会告诉你这样做，然后你上网看到大量的文章告诉你做一件完全不同的事情。虽然我自己不是建议方面的专家，但对我来说，解决这个问题的最佳方法是评估我的优先事项，以及我打算用我将获得的技能做/建立什么。在确定了我的需求后，选择学习哪些技术变得相对容易，获得了这些技能，所有那些抨击你学习这个或那个的人都变成了白噪音。你可以在这里和这里获得流行语言及其最佳用例的列表[。
你会发现，开始往往是最困难的部分，但只要有足够的毅力和勇气，一切都会像肌肉记忆一样](https://medium.com/@CarolPelu/5-popular-programming-languages-and-their-uses-22af241de35b)

### 2。渐进式学习

好了，现在你已经完成了你的第一个教程/在线课程，你看到你所有的朋友都在做这些令人惊奇的事情。这就是冒名顶替综合症真正出现的时候。不充分的感觉是一个真正的问题，也是我长期以来一直在与之斗争的问题。每次你都觉得自己处于群体的末端，你还不够好，不能进入这个行业。所以你做了什么，你过度劳累，最终筋疲力尽。我花了很长时间才走出这个区域，我决定采取一种更放松的方式来学习新东西。现在的目标是在我从事的每一个项目中学习一些东西，不管它有多小，它最终会导致一些大的东西，并且令人惊讶的是，在很短的时间内所有这些小事情积累起来会带来大量的知识和个人成长。希望这种技术能帮助到所有正在为之奋斗的人。当你不知道某件事时寻求帮助也很有帮助，并为你节省无数的时间，很多时候，小问题本可以由比你更有经验的人来解决。如果你曾经感到[被](https://stackoverflow.com/)困住，这里是一个询问的好地方。
*“瞧，我知道你是个非常骄傲的人，但如果你有困难，你需要让别人帮助你。没有人是一座孤岛。”*

### 3。参加聚会

一个人需要他人和整个社会的陪伴和支持才能茁壮成长

想要与他人互动是人类的天性，自然地，我们会聚集在与我们最相似的人周围。没有什么比一有机会就参加技术会议更能说明这一点了。
好处包括但不限于😁:
1。获取最新的行业趋势。
2。获得职业或个人的好机会。
3。这是结识新朋友和拓展人脉的好方法。
4。这是一个远离笔记本电脑，在真实世界中度过一段时间的机会。
5。通过分享你正在纠结的事情，你可能会找到一些有用的建议。

这个 [gem](https://www.meetup.com/) 是一个开始和识别当地聚会团体和社区的好地方，它并不是真正专注于技术聚会，而是各种各样的社区，从艺术和健身爱好者、烹饪俱乐部到图书俱乐部。如果你还没有听说过它，请看看它。

### 4。休息一下

休息一下让你有机会从问题的内部工作中抽身出来，让你从一个新的角度重新审视它。这也能让你的潜意识有机会解决问题——这能让你产生新的想法，而不是继续在现有的道路上跋涉。

休息也有助于减少疲劳，我个人发现，跑步或锻炼有助于保持平衡。
