* * *

这是一个关于 [Yesod](https://www.yesodweb.com/) 的系列:一个 Haskell web 框架，遵循类似于 [Rails](https://rubyonrails.org/) 的哲学。事实上，它非常固执己见，并且提供了许多现成的功能。

一本关于 Yesod 的好书可以在网上免费获得:[用 Haskell 和 Yesod 开发 web 应用](https://www.yesodweb.com/book)。这就是为什么这个系列将是对来自[回购](https://github.com/3v0k4/yesod-blog)的提交的评论，我们将使用它来开发一个超级简单的博客。

换句话说，这不是学习如何使用 Yesod 的好材料。然而，它有望给出该框架如何工作的概述。

## [](#giving-back)回馈

上周是非常特别的一周。首先，我有机会与数百名软件设计师共度时光:在过去的三年里， [SoCraTes 会议](https://socrates-conference.de)一直是我一年中最重要的事情。此外，帮助组织是我回报社区的方式。

上周还发生了另一件让我引以为豪的事。我为一个开源项目合并了我的第一个拉请求:我正式成为了 [Haskell Yesod 贡献者](https://github.com/yesodweb/yesod/graphs/contributors)！！

灵感来自于[上周的帖子](https://dev.to/riccardoodone/building-a-blog-in-haskell-with-yesod-json-api-40d1):

> 不幸的是，是的。Auth.Dummy 不支持通过 JSON 请求注册。所以，我们得自己打补丁。

在那里，我[问](https://github.com/yesodweb/yesod/issues/1618)维护人员是否对某个 PR 感兴趣，盯着屏幕看了几个小时，从代码库的其他部分复制/粘贴了一些代码，并提交了新功能以供审查。

特别感谢《Yesod》作者迈克尔的支持和鼓励！我期待着下一个拉请求。
