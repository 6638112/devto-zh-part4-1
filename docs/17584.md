### **背景/导语**

我在大学还有 1 年半的时间，所以我暂时不会申请职位，但我认为现在为我的理想工作做准备比毕业时更好。

我一直在做一些关于在 Mozilla 工作的研究，我看到了一些关于它的黑客新闻和 quora 帖子，这似乎证实了在那里工作可能是我的梦想工作。

我不确定的是我需要如何去计划和得到我想要的工作。我喜欢直接使用 Firefox、Servo 或 Rust，但我不确定在列表页面上找哪个位置，或者这是否不太重要？他们似乎也非常擅长远程工作。

我现在正在努力获得计算机科学学士学位，也一直在考虑攻读硕士学位，尤其是如果它能帮助我迈出第一步。

我正试图在空闲时间学习 Rust，到目前为止，它是我最喜欢的语言，这也是我研究 Mozilla 的原因之一(是的，我知道还有其他地方使用 Rust，但我真的很喜欢 Mozilla 带来的利他主义的感觉)。

我一直在清除[exercim . io](https://exercism.io/tracks/rust)、[阅读文档](https://doc.rust-lang.org/book/index.html)、[奥莱利出版的书](http://shop.oreilly.com/product/0636920040385.do)上的锈迹，并用它尝试了几个个人项目。

一旦我对这门语言有了信心，我就有了一些我热衷于参与的开源项目，比如 Smithay(见下文)，它们看起来很有趣，也能提供一些我可以展示自己能力的东西。我也可能开始尝试为 Firefox、Servo 和 Rust 做贡献，以更熟悉这些项目并提高我的技能。

## 目标

Smithay 的目标是在 Rust 中提供构建模块来创建 wayland 合成器。虽然不是一个成熟的合成器，但它将提供对象和接口，以一种通用的方式实现几乎任何合成器都需要的通用功能。

它支持[核心 Wayland 协议](https://gitlab.freedesktop.org/wayland/wayland)，官方[协议扩展](https://gitlab.freedesktop.org/wayland/wayland-protocols)，以及*一些*外部扩展，比如那些由[wloots](https://gitlab.freedesktop.org/wlroots/wlr-protocols)和 [KDE](https://invent.kde.org/libraries/plasma-wayland-protocols) 制作的

另外:

*   **记录:**史密斯公司努力保持其 API 和功能的清晰和详细的记录。已编译的文档在 [docs.rs](https://docs.rs/smithay) 上可获得发布版本，在[这里](https://smithay.github.io/smithay)可获得主分支。
*   **安全性:**史密斯公司将目标定为使用安全，因为生锈。
*   模块化: Smithay 不是一个框架，也不会受到约束。如果有一部分你不想用，就不要强迫你用。
