# 我应该继续去我的工具箱吗？

> 原文：<https://dev.to/jasonish/should-i-keep-go-in-my-tool-box-3n15>

在任何一天，我都可能会写 C、JavaScript/TypeScript、Rust 和 Go...由于我的工作，Rust 和 C 是必须的。如果你用 web 前端做任何事情，那么 JavaScript 是必须的。虽然我有一些应用程序在后台使用 Go，但我很难找到保持这种方式的理由。在我看来，在开发后端服务时，Rust 在开发人员生产率方面正在迎头赶上，JavaScript 完全有能力做到这一点，Node 也是如此。

鉴于我所有的 go 应用程序都使用 SQLite，并且需要 Cgo，我甚至没有真正利用 Go 的杀手级特性——可以交叉编译到任何其他操作系统/架构的静态二进制文件。
