# 语义 HTML 的重要性

> 原文：<https://dev.to/kayla/the-importance-of-semantic-html-1032>

##### 语义 HTML 可以对你的用户和其他开发者产生各种不同的影响。

## 什么是语义 HTML？

语义 HTML 是添加上下文并向浏览器解释元素应该如何显示的标记，而不仅仅是解释元素的表示。这进一步将元素表示的责任转移到 CSS 上。比如用`<strong>`和`<em>`代替`<b>`和`<i>`。`<strong>`和`<em>`提供文本背后的语境。`<b>`和`<i>`解释文本应该如何呈现。(查看 [Mozilla 的 HTML 元素参考页面](https://developer.mozilla.org/en-US/docs/Web/HTML/Element)，了解更多语义 HTML 标签以及如何使用它们。)

## 为什么？

### 一般都是工作少。

如果元素是可操作的，比如一个按钮，那么本地`<button>`元素中就包含了可访问性特性和功能。使用一个`<div>`来用 JavaScript 模拟一个按钮动作，通过将使元素可操作和可访问的工作转移到开发人员身上，使得开发人员的工作变得更加困难。使用语义 HTML 使您不必重新发明轮子，并且避免了使用原生元素时默认获得的关键特性。

这也意味着开发人员需要更少的工作来使元素易于访问或更好地用于搜索引擎优化(SEO)。辅助技术可以使用标签，比如`<nav>`和`<main>`，来帮助用户更有效地浏览网站。搜索引擎在对页面进行排名时会考虑语义 HTML。

### 比较好读。

在您的标记中包含语义 HTML 意味着其他开发人员将更容易阅读和维护。它通过描述元素预期包含的信息类型来赋予每个部分意义。此外，如果你忘记关闭标签，比你必须在一大堆`</div>`中搜寻更容易发现。

用语义 HTML 就行了。这对你和你的用户来说都更容易。🎉
