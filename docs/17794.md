# GraphQL 是 API 的未来吗？

> 原文：<https://dev.to/graphqleditor/is-graphql-the-future-of-apis-2i90>

开发 API 对于任何项目来说都是至关重要的任务。一个设计良好的 API 必须具有灵活性和可伸缩性，因为它总是随着时间而发展。在过去的几年中，对改进的迫切需求导致了数据图表的爆炸性增长。这对 GraphQL 来说是个好消息，因为许多公司已经开始在他们的项目中采用它。并非毫无根据:

*   **奥迪的 e-tron 汽车**附资料图
*   纽约时报的首页是基于一个数据图表
*   Airbnb 在他们所有的应用程序中都使用数据图表，包括网络和手机&

...这个清单还在继续。

### 数据图表的好处

图无处不在！那么数据图结构的主要好处是什么呢？首先，它通过更好、更清晰的结构提供了更容易的数据访问。

[![Graphs are everywhere](img/ccf6454f7f2cd3ffcf78b1f820659dc5.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--H8ai5fZZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3ilvrt7tf3ge8xcr7gx9.jpg)

这带来了使用数据图的另一个巨大好处，即更好地理解数据，这有助于更快的项目启动和更有效的整体协作，因为我们可以清楚地看到因果关系。值得一提的是，图结构更容易可视化，尤其是在**数据图/GraphQL** 市场上有现代工具可用，如 [GraphQL Birdseye](https://github.com/Novvum/graphql-birdseye) 或 [GraphQL 编辑器](https://graphqleditor.com/)。

[![GraphQL schema visualized with GraphQL Editor](img/541b2d4201f838bba7d137ec498fbf5f.png)](https://graphqleditor.com/)

好处不止于此，还不止于更快的开发、更高的代码自动化、跨多个应用的更一致的用户体验等等。

### 前途一片光明

毫无疑问， [Apollo](https://www.apollographql.com/) 中的人，最流行的 GraphQL 实现，完全理解数据图结构的好处。幸运的是，越来越多的人开始看到数据图表提供的品质。最近，大新闻出现了，[“Apollo 融资 2200 万美元”](https://www.businesswire.com/news/home/20190612005247/en/Apollo-Raises-22-Million-Give-GraphQL-based-Data)的消息，这表明风投们也注意到了 GraphQL 技术中的商机，这让人们更加相信 GraphQL 可能是 API 的未来。你怎么想呢?
