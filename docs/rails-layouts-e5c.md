## Layouts

布局允许你用你的应用程序做很多事情，但是为了简单起见，让我们分解一下布局影响你的应用程序的三种简单方式。

### 默认布局

这是将在整个应用程序中持续存在的布局。默认布局可能包含主页链接、登录/注销链接和帮助链接。这可能看起来像这样。

```
<%= link_to 'Home', root_path %>
<%= link_to 'Login', login_path %>
<%= link_to 'Help', help_path %>
<%= yield %> 
```

### 渲染一个布局

就像您可以呈现视图页面一样，您也可以呈现特定的布局。例如，您可能希望登录页面的布局与主页的布局不同。渲染布局使用与渲染特定视图非常相似的语法。

```
render layout: 'login' 
```

### 控制器 vs 方法

默认情况下，控制器会查找与控制器型号匹配的布局文件。例如，对于您的用户控制器，Rails 将在您的 layouts 文件夹中查找用户视图文件。然而，如果你想为你的方法之一使用一个特定的布局，你仍然可以用我们在上面看到的渲染布局命令来指定。

### 向前移动

布局是保持代码干燥的一个很好的方法，因为像“主页”这样的链接几乎可以在你的应用程序的每一页上找到。它们还有一个好处，就是允许你加入链接，帮助你快速导航和测试你的应用。虽然看起来没什么大不了的，但我非常欣赏这种无需手动输入 URL 就能直接导航应用的能力。
