# [](#a-brief-history-lesson)简短的历史课

当微软推出。NET，他们决定他们不是在构建一个单一的语言，而是一个托管的框架，不同的语言可以编译运行。这是有意义的因为。NET 是为了解决“DLL 地狱”,并通过为人们提供一种更智能的替代方案来简化托管代码的互操作性。

这意味着你可以用 VB 编写代码。如果你习惯于 Visual Basic，你可以试试新的 C#编程语言，如果你更喜欢 C++或者 Java 风格的语法。这也给了微软引入新语言的空间，比如 [F#](https://fsharp.org/) 。

。NET 在解决 COM 的困境和提高开发人员的生产力和应用程序质量方面取得了巨大的成功。

# [T1。当今世界的网络](#net-in-todays-world)

微软继续用 Blazor 进行创新，他们采用编译语言并生成 web 汇编，允许开发人员用 JavaScript 之外的语言编写 Web 应用程序。这仍然是一项新兴技术，还处于早期阶段，但持续的奉献和承诺是存在的。
