## [](#rfc-for-deprecating-eventing-from-ember)[RFC 为从余烬中脱颖而出💬](https://github.com/emberjs/rfcs/pull/528)

[@ggayowsky](https://github.com/ggayowsky) 询问您对 Ember 的**事件**模式的评价。动机源于 Ember 不再使用事件，感谢为[辛烷](https://emberjs.com/editions/octane/)所做的工作。通过弃用它，我们可以[简化 Ember 并遵循 2019-2020 路线图](https://github.com/emberjs/rfcs/blob/2018-2019-roadmap/text/0519-2018-2019-roadmap.md#continue-simplifying-ember)。

特别是，Gerald [建议我们目标弃用](https://github.com/ggayowsky/rfcs/blob/deprecate-evented-mixin/text/0000-deprecate-evented-mixin.md)[`Evented`mixin](https://api.emberjs.com/ember/release/classes/Evented)和来自 [`@ember/object/events`](https://api.emberjs.com/ember/release/modules/@ember%2Fobject#functions-events) 和 [`@ember/object/evented`](https://api.emberjs.com/ember/release/modules/@ember%2Fobject#functions-evented) 包的方法。如果你仍然需要使用事件模式，你可以使用一个[支持良好的第三方库](https://github.com/ggayowsky/rfcs/blob/deprecate-evented-mixin/text/0000-deprecate-evented-mixin.md#transition-path)。

更多关于 eventing 的信息和 Ember 的弃用计划，请今天查看 RFC 。请务必提供您的反馈和帮助！
