# 我的第一次创业:组织代码片段

> 原文：<https://dev.to/embluk/my-first-startup-organising-code-snippets-185f>

作为我在 Dev.to 上的第一篇文章，我想我应该谈谈我的第一个放归自然的产品！🤓

# 这一切是如何开始的...

所以在我大学二年级的时候，我很沮丧，因为没有一种简单的方法来保存有用的代码，这样我就可以在将来使用它们；它可能是 nginx/apache 的配置模板，这是我最喜欢的解决 JavaScript 问题的方法，或者我只是想以自己的格式保存 StackOverflow 帖子。

我对目前的解决方案并不满意，因为它们大多专注于 HTML、CSS 和 JS 沙箱。我不想那样。我只是想要一个简单、容易的方法来组织代码片段。所以我决定建立自己的平台。我第三年的开始是我开始和一个朋友一起做建筑。

# 另一个 Pastebin？

有些人可能会问为什么要构建另一个 Pastebin 类型的服务，我想解释一下为什么我认为我的项目是不同的。我想建立一个平台，专注于组织有用的代码。我们学的都不一样，品味也不一样。例如，我们喜欢不同的编码风格和编程技术。那么，为什么没有一个地方来存放所有这些东西呢？

我想以 Pastebin 的想法为基础，以自己的方式进行迭代，并为这样一个平台添加更多的社交元素。我的服务与众不同的一点是，它使用并排的代码和注释编辑器来帮助你记录/解释你的代码片段。我想让它成为一本代码参考书籍，因为每个人的学习方式不同，这是重点。

# 那么，我可以看看吗？

你感兴趣？太好了！

但在你看一看之前，这是我忙碌的一年，我在 2018 年启动了这个项目，但有一段时间没有时间从事这个项目。我没有抛弃它，它上市后我一直让它运行。我们推出了一个最小可行的产品，不是所有的功能都实现了，我想添加更多的功能，使它与其他代码片段保存平台区分开来。

所以现在我已经为它作序了，如果你好奇的话，请看看，也许对某人有用:[https://CodeClippet.com](https://CodeClippet.com)😆🤘

# 技术决策和挑战

我开始 CodeClippet 是基于我在大学的一个模块中获得的知识，这个模块是网络和数据库技术。CodeClippet 是使用 LAMP 堆栈构建的。自从在大学使用 PHP 以来，我一直很喜欢它的易用性和支持，所以有很多关于从头开始构建 web 应用程序以及如何扩展它们的资料。

CodeClippet 是我从头到尾(MVP)开发的第一个严肃的网络应用，所以很多东西对我来说都是新的；我正在学习如何管理服务器、数据库、设置环境来托管和开发后端和前端功能。我几乎什么都做了。这在很大程度上是一种边学边做的体验，我很享受这种体验，因为我刚刚潜入了自己尚未探索的技术。

回顾过去，我对我选择用来构建 CodeClippet 的技术很满意。这给了我一个很好的基础，让我在未来做出伟大的事情！

真正具有挑战性的一件事是设计数据库，我们做了大量的研究，以确保我们遵循最佳实践，就像我们对其他所有事情所做的那样。CodeClippet 实际上有自己定制的 PHP 框架，试图帮助我从底层了解像 Laravel 这样的框架是如何工作的。目前，我正在开发基于 SwoolePHP 的框架 v2。
