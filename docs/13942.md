### [](#angular-cli-sdk-and-angular-40)**Angular CLI、SDK 和 Angular 4.0**

*Hans Larsen 展示即将推出的 Angular 产品并讨论 SDK*

作为核心 angular 团队的一员，Hans 一直致力于 SDK——CLI 的下一个版本。他说，他的工作包括帮助开发者提高效率，让他们管理自己的应用和项目，甚至在出现在浏览器之前。他还希望改进界面。

Hans 进一步解释说，习惯当前 CLI 的个人将能够轻松使用新版本。默认功能将保持不变，升级路径将是直截了当的，以支持 Angular 4.0。话虽如此，SDK 将作为一个蓝图库完全重新设计。对于 Angular 社区来说，这些都是非常令人兴奋的消息。

### [](#angular-angularjs-and-ngupgrade)**棱角分明，棱角分明，Ng-upgrade**

*Pete Bacon Darwin，Angular Core 团队介绍升级 AngularJS 应用的最佳方式*

作为一名仍在从事最大的 JavaScript 框架之一的贡献者，他谈到了如何减缓 AngularJS 的开发，以便允许向 Angular 迁移。AngularJS 中许多人开始喜欢的部分仍然存在于 Angular 中，这是帮助开发者走向 Angular 的更多理由。

使用 ngUpgrade 可以轻松升级，它允许 angular 和 angularjs 同时运行。这样，开发人员可以继续保持客户的正常运行时间，同时一次移动一部分代码。NgUpgrade 是转换到 Angular 的许多潜在解决方案之一。

回到 Angular 推出的时候，Pete 谈到了是否有更好的方法让社区为迁移做好准备。从他的角度来看，由于 Angularjs 的现有架构，没有什么可以做得不同，因为 Angular 现在相比之下要先进得多。

采访结束时，Pete 描述了 Angular 社区和不同的框架走到了一起并融合了一些想法。他认为，随着 JavaScript 和现代网络的不断发展，比较和讨论想法对解决方案有很大帮助。

### [](#angular-directives)**角度指示**

迈克·布罗基和他关于角度指令的想法

Mike 描述了如何成为 Angular-CLI 核心团队的一员，并为 Angular 社区做出更大的贡献。在这次采访中，他详细地谈到了指令，从指令被低估和应该被提升的事实开始。他希望在未来，人们会更加意识到如何有效地使用指令。他解释了为什么它仍然是 Angular 的重要组成部分，甚至超越 Angularjs。

有了指令，开发人员可以将行为添加到他们的应用程序中，而不必分离组件或将逻辑添加到组件中。这可以在许多情况下实现。

Mike 还讨论了何时不使用指令，并对主机绑定和主机监听做了一些扩展。然后，他对指令还能完成什么进行了分析，比如提高可访问性或用于从现有组件访问模板。Mike 还描述了不同类型的指令之间的差异。

[https://www.youtube.com/embed/8_rtU5Lcd6U](https://www.youtube.com/embed/8_rtU5Lcd6U)

### [](#angular-community-open-sourcing-projects)**棱角社区+开源项目**

*克服成为开发人员的挑战:Justin Searls 分享他的经验*

Justin 讨论了学习 Angular(来自 ember)是什么感觉，以及框架是如何演变的，变得更加强烈和“健壮”。虽然这是一件好事，但他提到了为创造性留有余地的重要性，因为这是他当初选择探索 javascript 的原因。用他的话说，“我们需要为创造力和分歧创造空间，而不是走上解决应用程序的单边道路”。这可以通过 babel 之类的工具轻松实现，它允许开发人员尝试不同的语言特性。

Justin 阐述了作为开发人员的非技术方面。他谈到了作为一名开发人员的挫败感和不安感、缺乏动力以及其他障碍。然后，他讨论了如何开始小的，创造性的项目和开源思想可以帮助疏导这些负面情绪，并把它们变成积极和适用的东西，而不是仅仅无法忍受。他鼓励开发人员提出他们的想法，即使他们不确定结果，并给出了如何以有效和谦虚的方式获得认可的技巧。总而言之，Justin 致力于为行业提供一个模型，使关于弱点和缺点的讨论正常化，不管你是谁，你来自哪里，或者你作为开发人员有什么背景。

[https://www.youtube.com/embed/NwsPydCF-9A](https://www.youtube.com/embed/NwsPydCF-9A)
