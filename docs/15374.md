# 我对免费代码营的看法

> 原文:[https://dev.to/tuxhedoh/my-thoughts-on-freecodecamp-55p7](https://dev.to/tuxhedoh/my-thoughts-on-freecodecamp-55p7)

如果你没有关注这个系列，它基本上是一个 100 天代码的日志，在其中我希望完成 FreeCodeCamp 课程，我开始完全重置我的进度，从而迫使自己再次经历他们的 HTML 和 CSS 学习路径。我相信在我第一次做了之后，他们添加了可访问性和响应部分，甚至可能添加了 CSS 网格和 Flexbox 的内容。我认为 HTML 和 CSS 的基础知识非常有用(但是我有过一些实践，我不知道如果他们从零开始会有什么感觉)。内置的 IDE 和 live page 是一个不错的选择，尽管我希望它能内置一些智能感知。

当涉及到可访问性和响应部分时，我很难适应课程。他们似乎只是提出了一个非常基本的想法，并用一个练习来展示它。在他们提到的一点上，在上次练习中，我们讨论了一些工具来帮助你(我相信是关于 HSL 和色盲的可访问性的练习)，我回去了。没有讨论任何工具。我知道 FCC 不是这个游戏中的唯一玩家，但我绝对认为这是他们程序中的一个薄弱环节。

当我开始一天的工作时，我希望能够进入 Flexbox 部分，但是我已经成功地完成了项目部分的所有工作。我还没有点击过致敬页项目，但是我已经做过一次了。如果我从来没有做过，我不相信我会准备这样做。把这些项目放在材料的这个点上是否服务于平台？也许是的，它让人们扩展他们的阅读材料，搜索新的部分，并参与论坛-阅读-搜索-提问。

FCC 论坛铺天盖地，我很少觉得我的问题可以在那里得到回答，我是技术人员，我无法想象一个新人会有什么感觉。
