# gimp，git 和管理攻击行为的神圣规则

> 原文:[https://dev . to/prahladyeri/the-gimp-the-git-and-the-sanary-rules-governing-offensive-behavior-4c B2](https://dev.to/prahladyeri/the-gimp-the-git-and-the-sacred-rules-that-govern-offensive-behavior-4cb2)

*(最初发表于[prahladyeri.com](https://prahladyeri.com/blog/2019/07/the-gimp-the-git-and-the-sacred-rules-that-govern-offensive-behavior.html))*

昨天，我在阅读关于 GIMP 问题跟踪器的激烈辩论，这可能也是当天讨论最多的话题。

冒犯行为是一个非常严肃和微妙的话题，需要小心处理。更重要的是，应该制定规则，客观地确定一个术语/句子是否具有冒犯性，因为任何留在未决定的灰色区域的东西都会成为地盘争斗和伤害情绪的温床，就像昨天发生的那样！

规则只不过是根据预定义的逻辑列出了在给定的领域中什么是(什么是)可接受的，什么是不可接受的。它们不一定是完全客观或预先确定的，有时它们的结果可能取决于上下文，有时它们也可能是主观的，会带来创作者的偏见。考虑到神经语言编程的进展，人工智能为我们自动决定这些规则的那一天不远了。因此，现在讨论和解决这个问题是很重要的。

根据这些成文规则的受欢迎程度，它们可能会被社会广泛接受，但这并不意味着它们是完美的，因为它们总是随着时间的推移而不断演变。在古代，它们被称为“汉谟拉比法典”，然后在中世纪开始被称为“十诫”，现在，它们被普遍称为 CoC(行为准则)。但是它们的共同点是支配人类行为的规则，正如我所说的，这些规则是不断进化的，有时是主观的。

但这并不意味着他们没有客观的一面。我们必须做的第一件事是毫无疑问地以确定的方式确定那些普遍冒犯每个人(或至少大多数人)的客观词汇是什么。如果一个人工智能决定这一点，这正是它将如何进行。让我们称之为客观黑名单(OBL)或什么的，这个名单可能包括像“你**呃”、“sl**”、“id**t”、“wh**e”等词。毫无疑问，说出这样的话会在精神上伤害大多数说英语的人。除了在某些情况下，比如关于抑制网上攻击性行为和语言学研究的元讨论，这些词可以在大多数互联网论坛上被自动屏蔽，我不认为很多人会有任何问题。

但另一方面，像 gimp、git 这种主观上带有攻击性的词呢，应该如何看待？我称它们是主观的，因为只有一小部分人觉得它令人不快(就像那个链接帖子上的一些人)，而不是所有人。就我个人而言，我甚至不知道 gimp 实际上是指 linux 画图应用程序之外的其他东西，直到我昨天查看了韦氏词典！

我不是语言专家，但从那次讨论和[Reddit 上关于同一主题的这篇文章](https://old.reddit.com/r/linux/comments/c9gvgf/consider_renaming_gimp_to_a_less_offensive_name/)中我了解到，gimp(跛足或残废的人)的另一个意思只被一小部分英国/欧洲英语使用者使用。大多数美国或亚洲说英语的人可能甚至不知道这第二个意思，gimp 对他们来说只意味着一个 linux 应用程序。所以冒犯的问题从一开始就不会出现。

在这一点上，它归结为“个人与集体自由”。像 gimp 这样受欢迎的开源项目的名称更改并不是一件小事。您不仅必须在构建过程中更改名称(编译命令、make & config 文件、翻译文件等。)，而是所有像 ubuntu，debian，fedora，suse 等各种发行版。也必须在他们的存储库中改变它。花费如此大量的时间和精力来让一小部分人高兴，而这些人又碰巧主观上以项目的名义犯了罪，这合理吗？

此外，如果我们允许这种情况发生，火车停在哪里？说到翻译，为什么只有说英语的公众才有冒犯的奢侈！gimp 被翻译成其他几种语言，为什么说法语、西班牙语、俄语、阿拉伯语和印地语的公众没有发言权呢？我敢肯定，如果你考虑所有这些语言，在所有这些 linux 包中会发现大量类似内容的攻击！

即使你坚持使用英语，其他 linux 程序如 git 和 fsck 又如何呢，因为前者实际上是一个令人不快的词，而后者在某种程度上类似于它(就像 GIMP 缩写一样)？我很久以前就知道 git 的意思了(可能是在哈利波特的某本书里)，而 GIMP 直到昨天还是我的首字母缩略词。而我第一次碰到版本控制系统的时候，我就有点恼火，什么样的人会把一个程序命名为“git”。但即便如此，我也从未觉得这个词有任何冒犯或困扰。随着时间的推移，我开始在几乎每个项目中使用 git 和 github，我忘记了 git 的另一个含义，只是在最近遇到它时才想起它(就像今天！).

但是我，我是一个程序员，我能理解一个非技术人员的感受。特别是如果你是一个英国人，我敢肯定，如果有人在谈话中随便叫你“git ”,不管你是不是程序员，你都会生气。但是，您真的会对这个词感到生气，并在 git 论坛上要求他们重新命名这个项目吗？所有这些努力值得花在改名上吗？

最重要的是，如果整个世界都开始对技术上下文中使用的一些词感到不快，你能想象仅仅在名称更改上就要浪费多少小时的编码、测试、文档、构建和项目维护吗？T3】
