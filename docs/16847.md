# 我经常安装的 6 种工具

> 原文:[https://dev.to/0xkoji/6-tools-i-always-install-7dm](https://dev.to/0xkoji/6-tools-i-always-install-7dm)

### [](#1-xcode)1。Xcode

基本上我都是用 Xcode 给`openFrameworks`写代码，有时候会写`Swift`。此外，很少使用从源代码构建一个库

[https://developer.apple.com/xcode/](https://developer.apple.com/xcode/)

### [](#2-homebrew)2。公司自产自用

Homebrew 是一个免费开源的软件包管理系统，它简化了在苹果 macOS 操作系统和 Linux 上安装软件的过程。

我不能没有这个 lol。我也在我的 Ubuntu 机器上使用这个。

[https://brew.sh/](https://brew.sh/)

### [](#3-visual-studio-code)3。Visual Studio 代码

我试过 sublime，atom 和 bracket。最终，继续使用 vscode。
我认为最大的原因是`code`指挥。实际上，我们可以添加一个命令来从`iTerm`/终端打开一个应用程序，但 vscode 不需要任何设置，这很容易。我喜欢不需要安装任何插件的 markdown 预览功能。

[https://code.visualstudio.com/](https://code.visualstudio.com/)

### 4。伊特雷姆+欧米-zsh

这不是必须的，但是我更喜欢 iTerm 而不是 Terminal。

[https://www.iterm2.com/](https://www.iterm2.com/)
T3】https://github.com/robbyrussell/oh-my-zsh

### [](#5-clipy)5 个。克利希

macOS 的剪贴板扩展应用程序。

这对我来说非常有用，因为我在这个应用程序上有许多片段。另外，我用朋友的机器帮他们的时候，总在想为什么你们没有 Clipy lol。

[https://clipy-app.com/](https://clipy-app.com/)
T3】https://github.com/Clipy/Clipy

### [](#6-boostnote)6。Boostnote 注意

一个用于 Mac、Windows 和 Linux 应用程序的开源 markdown 编辑器。

我总是用 markdown 而不是 txt，因为我的笔记包括一些代码。这个应用程序没有通过互联网分享功能，所以基本上是本地使用。
