# 一切都是相对的

> 原文：<https://dev.to/atila/everything-is-relative-1n73>

根据行为心理学，有可能迫使任何个人获得某种行为，这种行为有一个触发器、一个动机和一个目标。一旦你有了目标(例如:让一只狗流口水🐶)，你规定什么是触发器(例如:铃响了🔔);第二，你提供了一个动机(例如:一块多汁的肉🥩).现在，只需要同步触发器(🔔)和动机(🥩)一段时间后，动机可以被移除，触发器将保持运行(🐶).

现在，这个轶事和 web 开发有什么关系呢？

在新工具出现的速度下，随着范例的减少和最佳实践的替换，让我们自己质疑我们做事的动机是很重要的。这种情况下，我是不是应该再也不用像素了？我真的应该事事都去`rem` / `em`吗？

在软件开发中，我们经常被提醒没有灵丹妙药。在我看来，这还不是这条规则的例外。

## 坏边的像素

HTML 已经有 26 年的历史了，从那时起像素就作为一种布局测量单位出现了。我不是说像素是随着网络出现的，但是我们在网络中使用的像素并不完全是我们以前所知道的像素。正是在这个小字上，我们对他们的信任应该开始消退。

### 像素不是绝对的

不是网上的那些。事实上，它们的大小因多种因素而异:屏幕分辨率、浏览器渲染器和设备渲染器——如果 3 个对你来说不算多，运行每种最常见类型的组合，看看我们手头有多少个案例。

简而言之，`1px`并不是每次你的 <abbr title="user interface">UI</abbr> 加载时都是相同的距离。总而言之:我们能在多大程度上依赖一个非绝对的“绝对单位”？

## 字体基础单位

内容应该是应用程序中的一等公民。不管你的销售团队告诉你什么，我们都在从事传递信息、**内容**的业务。有鉴于此，我们的容器和盒子相对于文本来调整大小是完全合理的。我们希望我们的内容舒适地被我们设置的框和边框所包围。

幸运的是，`CSS`为我们提供了一些很好的工具来完成这个计划，我将节省一些时间，只提到 3 个主要的工具:

*   `ch`:基于字符`0`(零)长度的印刷度量。
*   `em`:基于字符`m`长度的排印度量。
*   `rem`:基于**根**元素的`em`的 web 度量。

### `em` vs `ch`

既然我们决定使用基于文本的单元，我们需要选择哪一个以更加一致和可靠的方式完成工作。为了做出选择，首先需要理解它们是如何定义的。

### 的宽度`m`

“为什么是`m`？”可能是第一个问题。当创造一种新的字体时，设计师使用(或*使用*)一个方形块(*字距调整*)作为参考，以确保每个字符与其他字符相比是按比例设计的。`m`是我们整个字母表中唯一会触及这个盒子左右边界的字符。因此，`1em`实际上是一个字符的最大宽度，不管它属于哪个字体家族。

### 绝对零度

`0`不触及任何*字距调整*的边界。事实上，数字零往往是字体中所有字符(字母数字)的平均宽度。因此，`ch`是对特定字体字符长度的估计，因此得名(`character`)。

因为零是平均值，所以它成为比`em`更可靠的单位。如果内容的字体系列发生变化，`5ch`将继续保持 5 个字符的长度，而`1em` / `1rem`可能会比以前有所变化。简而言之，`Times New Roman`中的`1ch`打印的距离与`Helvetica`中的`1ch`不同，也与`monospace`字体不同。

## 百分比呢？

百分比的使用非常强大，但也很棘手。根据要添加值的 CSS 属性，百分比与不同的度量相关。例如，对于填充和边距，百分比与包含框的宽度有关(不管它是垂直还是水平填充/边距)。

```
.wrapper {
  width: 10rem;
  height: 3rem;
}

.child {
  padding-top: 10%; /* 1rem */
  margin-top: 10%; /* 1rem */
} 
```

此外，如果您打算为`width`、`height`及其派生设置一个百分比值，分数将对应于其容器，因此`height: 50%`是元素容器高度的一半(假定它在解析时有一个可测量的长度)。

```
.parent {
  width: 40rem;
  height: 20rem;
}

.child {
  width: 50%; /* 20rem */
  height: 70%; /* 14rem */
} 
```

在某种程度上，我喜欢将`%`视为视口大小的`em`等价物。它们都非常强大，但是它们也需要对 CSS 呈现/解析如何工作有一定程度的理解，以及对 UI 元素如何集成和相互交互有更具战略性的看法，因为它们依赖于插入它们的上下文。

## 视口基础单位

另一方面，视口更加直观。他们的规范是`CSS3`等级的一部分，所以他们很新，尽管他们的学习曲线对他们的使用有很大帮助。

*   `vh`:视口高度。`1vh`是实际视口的**高度**的`1%`。
*   `vw`:视口宽度。`1vw`是实际视口的**宽度**的`1%`。
*   `vmax`:视口最大。`1vmax`是`1%`视口的**最大尺寸**
*   `vmin`:视口最小值。`1vmin`是视口**中最小维度**的`1%`

您可能已经意识到，视口单元不依赖于它们所插入的上下文。所以，为了保持单位间的比较:如果`%`是视口的`em`，`viewport units`是`rem`。

## 那么我们应该用什么呢？

这其实不是一个*用什么*的问题，而是一个*什么时候*该用*哪个*的问题。决定测量装置可靠性和使用案例的因素是什么。例如，`Celsius`定义为水的冻结温度，`kilocalorie`是`1ºC`加热水的`1Kg`所需的能量，等等。我们离量表的定义越近，它就越可靠。

简而言之，我们应该努力在我们正在创建的布局上找到信息，一旦我们做到了这一点，最好的单元将会显示出它更有可能提供可靠的信息度量。

让我们来看一个简单的例子，如果你的文章是基于文本的(例如:一篇博客文章)，你可以把你的选项缩小到`rem`、`em`或`ch`。现在，正在讨论的特定元素依赖于它的父元素吗？(如果是，`em`或`ch`大概)。你需要它在你转换字体的时候是可伸缩的吗？(如果有，肯定是`ch`)。当然，这种情况过于简单了，但我相信它会提供一个合理的依据。

在这篇文章中，我们回顾了如何决定在 CSS 布局中采用相对单元。然后，我们重新评估未来的选择，以确保我们仍然使用最好的工具。

记住，没有什么灵丹妙药，不要羞于利用你所掌握的每一种资源。为你所展示的东西找到什么是一等公民，并使上下文适应它。
