# 无障碍工作及其对未来人工智能的影响

> 原文:[https://dev . to/this dot media/accessibility-work-and-its-impact-on-future-ai-41mg](https://dev.to/thisdotmedia/accessibility-work-and-its-impact-on-future-ai-41mg)

如果您曾经因为所用字体的大小或颜色而在阅读时遇到困难，或者在大声的音乐会上很难听到旁边的人，那么您就有许多残疾人每天都要面对的相同的无障碍体验。

[![](../Images/4342983334ad35920925caf883bc762e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2_1cyKWh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2Am2s9bnC5zPVhM6C6UvzJGQ.jpeg)

那些有视力、听力和运动障碍的人很难在网上获取信息。主要问题是大多数网站在设计时没有考虑到这些用例。

根据世界卫生组织的数据，全世界大约有 10 亿人患有某种形式的残疾。这意味着高达 15%的世界人口无法有效地使用网络上的大量网站。

我们如何确保所有用户都能平等地访问内容？开发者在创建网站时，如何在网络上包容性地表达自己？

无障碍工作，也称为 a11y，致力于改善残疾人的网络体验，以确保网络内容和体验对每个人都是平等的。

谷歌 Chrome 的开发者支持者 Rob Dodson 是致力于改善网页可访问性的支持者之一。

从根本上说，可访问性被定义为一个人能够访问某些东西的能力，无论是信息、物理访问还是舒适工作的能力。当我们谈到网络的可访问性时，我们通常指的是技术访问，包括阅读网页、在线预订机票或从网站订购食品的能力。无障碍工作的目标是帮助人们与他们的环境互动，并消除人们在这样做时可能遇到的障碍。

对于没有残疾或损伤的人来说，很容易把视力这样的东西视为理所当然。如果你能够有效地看到你的电脑屏幕和消费信息，你就不会考虑如果你看不到你正在浏览的网站，你的体验会受到怎样的影响。

大多数可访问性问题源于用户体验设计，甚至可以像网站的颜色对比一样简单。如果页面上的文字与背景颜色的对比度很低，这不仅是一个访问问题，也是一个可用性问题；任何浏览网页的人都很难看清内容。

[![](../Images/983a4ed27f4bedd6028fcf28f9ba1ef0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rXNddU9A--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2Asx1IjFSJOOh9qRv4va5KWA.gif)

当设计网站时，用户效率也很重要。当考虑有运动障碍的用户或活动范围有限的人的体验时，我们为什么需要考虑可访问性就变得更加明显了。这些用户可能只能移动手指来激活开关设备，或者使用类似的吸气和吹气设备来导航页面。对于这些用户来说，跳过一个菜单，少点击几个按钮就能使用网站，这是高效导航的基础。

但是，我们必须超越帮助每个人平等访问网络的简单目标。可访问性是构建网络未来的关键因素之一。如果我们看看今天的许多尖端产品，我们开始意识到，许多如亚马逊 Alexa 或谷歌主页开始作为无障碍技术。随着人工智能越来越融入我们的日常生活，我们经常想当然地认为我们所依赖的文本到语音转换功能曾经只用于盲人的计算器。

[![](../Images/cf3935f0a06d5793ea28d1c2c8dad280.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gZ8bBbbG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2A7Y02PojTJrHDEcWSYKNhLw.jpeg)

在遵循易访问性如何加速技术进步的概念时，我们必须注意到在开发网站时语义的重要性。为网页添加语义不仅会通过提高屏幕和盲文阅读器有效工作的能力来改善今天视力受损或失明用户的体验，还可能有助于为未来人工智能理解我们的页面并与之交互奠定基础。当我们构建人工智能时，我们必须训练它们，并教会它们作为一个网页意味着什么，这伴随着增加语义。

语音交互是一种越来越依赖于与数字助理和人工智能通信的技术，也是一种辅助技术。与直接操作开关设备或屏幕阅读器相比，语音交互使有运动或视觉障碍的用户能够更容易地访问内容和执行某些操作。就像语音交互技术可以帮助盲人导航他们看不见的周围世界一样，它也可以帮助告诉用户在他们无法阅读或理解母语的外国该去哪里。

很快，人工智能将能够在我们的家里帮助我们。像这样的想法今天正在进行测试。如果亚马逊 Alexa 能够可靠地帮助用户在项目工作时修理厕所或安装管道，我们将成功地跨越到未来。随着更多的语义内容可供人工智能使用，这种类型的帮助可能会更快地实现。

当开发人员努力理解可访问性的重要性时，像 WAI-ARIA 和 T2 这样的机构正在致力于提高网络标准。目前，像 [aXe](https://github.com/dequelabs/axe-core) 和 [WAVE](http://wave.webaim.org/extension/) 这样的工具可以用来帮助审核一个网站，看看它是否符合这些标准。

尽管仍处于标准化阶段，JavaScript 的早期可访问性工作仍有希望在可访问性社区产生巨大影响。像 HTML 的惰性属性这样的项目将使构建可访问性模态对话框变得更加容易。对有运动障碍的用户来说，关注并宣传的重要性是关键。
