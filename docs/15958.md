# 软件 Peeps 的开发技巧

> 原文：<https://dev.to/akintunde102/development-tips-for-software-peeps-317h>

这将是我在[发展到](https://dev.to)的第一篇文章。

虽然在这里真的很好，但我相信如果我能通过倾吐我认为有助于提高开发人员生产力的某些技巧来增加一些价值，那会更好。

我会强调时间管理，在自己的舒适区呆足够长的时间来跳出它。

他们来了！

*   ## 当心通勤，它偷时间。

    我们中的一些人已经知道这一点。对于那些我没有观察到的，请注意。投资一个快速的交通系统，住得离工作地点近，或者更重要的是，远离或半远离工作地点，总比在路上浪费无尽的时间要好，因为你本可以做一些事情。这样的话，我们可能每周都能为自我发展节省一些额外的时间。

*   ## 通勤偷走了力量、注意力，然后是时间和生产力

    虽然通勤可以帮助伸展骨骼，放松身体螺栓，提高心血管效率，但有时它只会因为压力和最终的疲劳而把事情弄糟，这些压力和疲劳可能来自交通堵塞。我尽可能地减少通勤，尤其是在繁忙的城市里。我希望每个开发人员都能想出办法来解决这个问题。

*   ## 不要过度思考一个项目

    有些人选择“就这么做”而不是过度规划一个项目或一个想法，直到“他们的想法从可能的执行中消失”，这是有原因的。这就是我们通过它过度思考事情的结果。如果你想成为一名真正有效率的开发人员(或者其他人)，最好是在你动手之前就开始一个兼职项目，而不是过度规划。当你发现计划过程耗时太长时，我建议你“就这么做”。虽然让一个软件尽可能具有高性能是件好事。我相信有一个不完美的项目比没有要好，因为一个不完美的项目是可以改进的。虽然这是一个很好的技巧，但我只把它应用到我的业余项目中。生产代码完全是另一回事。

*   ## 呆在你的舒适区足够长的时间，以便跳出它

    仅仅因为你的项目用另一种语言编写得更好，并不意味着在你目前的技能范围内它是不可能的，至少不是一直如此。如果对你来说是这样的话，至少有一个包含项目的投资组合比没有或只有少量项目要好，因为你想构建有史以来最有**性能**的代码。如果你有时间学习新的东西，很好！如果你不这样做，我建议你使用你现在的技能，以后有时间的时候再学习，然后回来改进你所做的一切。
