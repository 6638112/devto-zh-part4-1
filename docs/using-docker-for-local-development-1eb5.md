# 使用 Docker 进行本地开发

> 原文：<https://dev.to/allentv/using-docker-for-local-development-1eb5>

本地开发包括设置和试验许多工具，以达到应用程序可以开发的程度。使用正确的工具进行试验需要时间，而且还冒着污染机器全局空间的风险，这反过来会导致其他软件停止工作，因为一个共享库被另一个更新了。类似的情况叫做“DLL hell”在微软世界也是相当知名的。集装箱化是一个很好的解决方案，几乎没有负面影响。

Docker 在容器领域是一个著名的玩家，当有人想到容器时，他在思想空间方面具有世界范围的优势。DevOps 文化的兴起发挥了巨大的作用，Kubernetes 等工具获得了广泛的采用。对了解 Docker 如何工作的工程师也有巨大的需求。所以投资时间学习 Docker 是完全值得的。

为了改善本地开发体验，请根据您的环境安装来自 https://www.docker.com/products/docker-desktop 的 Docker。下一步是创建一个 Docker 文件，定义应用程序所需的定制环境和工具。这一步将需要一些时间，因为我们需要试错，以建立一个优化的形象。构建完映像后，您现在可以生成 docker 容器，这是您的应用程序的运行时。容器可以从您的机器装载资源到您的容器中，非常类似于在虚拟机(VM)中装载外部资源。微调选项，如控制 CPU 和内存等容器可用的硬件资源，提供了经验丰富的工程师感兴趣的额外功能。这样的选项也很有意义，特别是当应用程序需要扩展，并由 Kubernetes 等工具在集群上自动管理时。清楚地定义硬件资源也有助于管理硬件利用率，特别是在钱包方面，因为更好的利用率将转化为更低的运营成本。

docker 容器提供的主要优势是资源隔离和崩溃时轻松恢复的能力。容器由操作系统通过一个进程来管理，在崩溃时，只有进程被终止，而不会影响系统的其余部分。这在处理将遗留软件系统迁移到新版本时非常方便，并且业务涉众期望两个系统并行运行，直到遗留版本退役。两个版本之间的隔离以及监控并行工作的能力对于大型软件迁移项目的成功非常有用。

如果你认为这是太多的工作，它不是。整个 docker 设置不到 5 分钟。你可以在 docker hub 上找到预建的 docker 图像，这是一个公共注册表，任何人都可以托管他们创建的 docker 图像。Docker 文件管理和多个容器的串接会变得非常难以维护。在这种情况下，您可以使用 docker compose 工具来管理多层堆栈。docker 周围的社区非常支持，世界各地都在举行当地聚会/活动，今天就开始是个好主意！
