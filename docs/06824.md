> 原文:[https://dev . to/setevoy/AWS-cloud craft-an-existing-infra structure-s-map-generator-499 a](https://dev.to/setevoy/aws-cloudcraft-an-existing-infrastructure-s-map-generator-499a)

[![](../Images/9cf5c5b689deb4884e18a510f67d47cf.png)](https://res.cloudinary.com/practicaldev/image/fetch/s---dkzETw5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://rtfm.co.ua/wp-content/uploads/2014/11/aws-logo-square-02-e1417012834176-1.png) 我暂时使用了 [CloudCraft](https://cloudcraft.co/) 服务——非常方便。

几天前，我获得了其所有功能的完整订阅(顺便说一句，前两周是免费的)，其中我想尝试创建一个现有 AWS 帐户的地图及其所有资源和关系/通信的能力。

订阅费用为每月 49 美元。

一般来说，使用 CloudCraft 可以绘制类似 next 的地图(来自[AWS:миграцияrtfm 2.3—интрастттрадляrtfmисозданеcloud formationшаблона—VPC，subnets，EC2](https://rtfm.co.ua/aws-infrastruktura-dlya-rtfm-i-sozdanie-cloudformation-shablona-vpc-subnets-ec2/#i) post， *Rus* ):

[![](../Images/9db66045aac084c2d515b7ac9663fd35.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2017/10/rtfm_migration_map_2017_0.5.png)

在本帖中，我们将把 CloudCraft 与现有的 AWS 帐户相集成，并用该帐户中已创建的资源生成一个新地图。

对 AWS 账户的访问将使用 CloudCraft 的[外部 ID](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create_for-user_externalid.html) 和 AWS [交叉账户 IAM 角色](https://docs.aws.amazon.com/en_us/IAM/latest/UserGuide/tutorial_cross-account-with-roles.html)进行配置。

### [](#cloudcraft-and-aws-account-integration)CloudCraft 与 AWS 账户整合

转到[https://cloudcraft.co/app](https://cloudcraft.co/app)，选择了*直播*标签:

[![](../Images/d26a1bc48c1db48b91d3b752ef669478.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_135844.png)

点击*添加 AWS 账户*:

[![](../Images/b1268990eec6686fe34cab6d276d2f85.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_135953.png)

只需按照 CloudCraft 的指示一步一步地操作:

[![](../Images/14c07246e8f1d31a9e937b3f56884ca0.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_140038.png)

点击“*打开 AWS IAM 控制台，进入创建角色页面*”链接，您将被重定向至 AWS = > IAM，这里将填写所有内容:

[![](../Images/fc831bf2608f2ab87218efd7118f9c75.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_140131.png)

点击*权限*——这里是已经检查过的`ReadOnlyAccess`:

[![](../Images/99e9bf728736e3fa671f7a390039c3d0.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_140235.png)

跳过标签，保存角色:

[![](../Images/0e9932b42a9f520bb4d290812edb9a9a.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_140335.png)

复制它的 ARN:

[![](../Images/f5af81fe24fa66078d9d93fcc04d44eb.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_173359.png)

将它放入 CloudCraft:

[![](../Images/9417a8f59a77939a092cc2de2f6c99c3.png "AWS: CloudCraft - генерация карты существующей инфрастуктуры")T2】](https://rtfm.co.ua/wp-content/uploads/2019/08/Screenshot_20190821_173605.png)

点击*保存 AWS 账户*。
