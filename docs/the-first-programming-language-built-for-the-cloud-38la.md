# 第一种为云构建的编程语言

> 原文：<https://dev.to/educostadev/the-first-programming-language-built-for-the-cloud-38la>

第一个面向云的编程语言设计即将成为现实。它已经设计，原型，验证和记录了几年，它的名字是 EcstasyLang。

你可能会想，它解决了什么其他编程语言没有解决的问题？这是一个很难回答的问题，但我们讨论了 EcstasyLang 解决了哪些挑战，以及它是如何解决这些挑战的。让我们来看看一些主要特征。

## 可伸缩

这种语言被设计成可扩展的。一种编程语言的可伸缩性意味着它必须能很好地适用于小型和大型项目。大型项目和大型团队在一年或几十年的时间里维护着庞大的代码库。从运行时的角度来看，它必须在受限制的设备中工作良好，并且能够使用较大服务器的资源。

EcstasyLang 是一种强类型语言。这最大限度地减少了代码更改带来的潜在意外，代码更改只会被检测为运行时故障。另一个好的设计重点是可读性。这种风格鼓励使用垂直空间、显式结构和丰富的文档作为代码的一部分。它还包括冷却支架。默认情况下，版本控制和升级、模拟和测试以及包括条件依赖在内的依赖管理是语言的一部分。

该语言还被设计成编译成本地可执行格式，在运行时环境中执行或嵌入到其他语言中。内存管理是由语言运行库自动完成的。它的虚拟机被设计为高效地支持 TB 甚至 Pb 的应用程序内存，而不需要任何“停止世界”的暂停。

在客户端从低功耗设备连接到应用程序的云环境中，可扩展的语言非常重要。应用程序会随着时间的推移而增长，并且可能会有数百万甚至数十亿的连接客户端。

## 安全

该语言在设计上也是安全的，没有复杂的层或运行时逻辑和安全检查。使用容器模型。与操作系统容器不同，摇头丸容器在容器内没有表面积。在容器中运行的代码不能访问操作系统或硬件资源。

云中的客户端和服务器绝不能信任它们托管的代码。同样，开发人员永远不应该相信不首先假设这一点的语言。

## 便携

这种语言有一个定义良好中间编译。它可以针对几乎任何操作系统和硬件环境。它旨在支持编译为 Web 程序集，以支持浏览器部署，并可移植到客户端。它还被设计成支持应用程序的可移植性，应用程序可以从一个服务器提升到另一个服务器。

可移植的语言是有用和重要的。代码需要跨各种云提供商和开发环境运行。应用程序通常需要在云中迁移。

## 可重复使用

开发人员需要能够重用他们的工作。他们越来越依赖于其他人开发的组件、库和服务。没有明确的可重用性设计，事情会变得一团糟。

摇头丸支持模块化编程。模块是一个重用单元，可以简单到只有几行代码，也可以复杂到有数百个代码。模块可以被签名，甚至完全加密。模块存储库允许管理模块，并使其在线或离线可用，支持按需缓存和下载。

## 结论

在过去，编程语言被设计用来解决许多问题。今天，新的语言在解决具体问题上更加专业。为云环境设计语言是受欢迎的，我祝愿它成功。进入官方的[博客](https://xtclang.blogspot.com/)和 [GitHub](https://github.com/xtclang/) 库你可以找到更多关于这种语言的信息。推特账户 [@xtclang](https://twitter.com/xtclang) 可以让你了解关于这门语言的最新消息。
