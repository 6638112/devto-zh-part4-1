大约 14 年前，我开始自学软件，因为我想制作一个 MMORPG 游戏。快进 3 年，我开始处理大量的数据集，开发了许多用 Python 编写的工具，用于获取、转换、验证和显示数据。
所以我说我有 4 年的准备，只有 10 年的实际经验。
我不想提这个来吹牛，因为我觉得自己像个学生，每天都在学习新的东西。
我只是想提一下数字，这样你就可以相信我有资格说出我将要说的话。

# [](#step-1-functions)第一步-功能

首先，你开始学习如何编写软件，因为你有一种强烈的愿望去构建一些东西。也许是一个游戏，也许是一个待办事项应用，也许是一个 Twitter 替代品。

但是你不知道如何编写任何软件，即使你可能知道“变量”是什么意思，或者“if 条件”，或者“for 循环”。在你写了 100 多行代码之后，你如何把所有这些联系起来？你如何重用代码，这样你就不必复制&粘贴相同的功能？如何从文件中读取配置？app 关闭后你的“状态”存放在哪里？如何让多个“东西”同时运行？如何显示应用程序内部发生的“事情”？

所有这些小问题都需要花很多时间去学习，以至于你可能会完全忘记你从哪里开始，以及最初是什么启动了这段旅程。

但是如果你克服了所有这些，你肯定有一个坚实的基础，你有潜力构建任何软件系统。

# [](#step-2-libraries)第二步-库

你写了几个项目，你开始在所有的项目中看到模式。所有的应用程序都在做不同的事情，但在内部，它们以非常相似的方式做。这些被称为“设计模式”,只是你可以在类似情况下应用的模板。

你可能听说过诸如“过程的”、“功能的”或“面向对象的”之类的东西，但是只有在这一点上，你才开始理解它们的含义。

你开始习惯使用多重继承、单例、原型、装饰器、迭代器、事件发射器、同步/异步、线程、纤程——这取决于你使用的编程语言。

# [](#step-3-systems)第三步-系统

你学习一些编程语言。你永远不会精通每一种语言，但是你可以写出地道的代码。

你开始考虑多个软件系统的架构，其中许多是用不同的编程语言编写的，用不同的协议通信，有些很慢，有些偶尔会崩溃。

如何存储海量数据？如何调试每分钟推送数百万行日志的实时应用程序？

你可以编排、设计和维护复杂的系统。

# [](#step-4-meta)第四步-元

你开始反思“我们为什么要写软件？”软件是由人制造的，也是为人服务的...
软件作为一种生活哲学🧘‍♂️

“开发人员的直觉”可能会引导您更喜欢某些框架，原因对其他人来说不是立即可见的，因为您总是可以找到或构建替代方案。

一些例子包括:
也许框架或者库对于你的特定用例来说太庞大了，所以如果你导入了这个庞大的东西并且你使用了它的 5%,也许你可以构建你自己的？
也许许可证限制太多，可能会影响项目的未来？
也许你不喜欢图书馆把你所有的数据放在他们的云中，你想要数据解放？

当你有一个新项目时，你并不关心具体的语言或框架。编写代码或设计系统不再是问题，你已经知道如何去做。

你要先搞清楚“这是想解决什么问题”。然后你想知道“展示正在发生的事情的最相关的方式是什么”和“人们如何与之互动”。然后你想知道一个高层次的技术特性列表。只有在最后，你才开始关注架构和软件框架。从最抽象的到真正具体的。

# [](#disclaimer)免责声明
