# 推销你的解决方案，而不是你自己

> 原文：<https://dev.to/raddevon/sell-your-solution-not-yourself-1mgl>

我的一个读者最近给我发了一封电子邮件，询问他们对 web 开发组合的反馈。它非常美观，展示了一些很酷的项目。

唯一需要一些爱的部分是项目上面的介绍性文字。“我是一名有抱负的网络开发学生。我从事 JavaScript、HTML 和 CSS 工作。我喜欢写漂亮的代码，喜欢学习新技能。”

## 为什么这不是很有效

想象你是一个电子游戏收藏家。你偶然发现一个定价很低的非常罕见的游戏，好像卖家并不知道他们有什么。你买下来带回家。厉害！🎉

问题是，它的正面右侧有所有这些最初购买的商店留下的讨厌的价格标签。现在，即使你买了很多东西，**你也有一个问题**:你需要在不损坏盒子的情况下去掉那些贴纸，以实现你的物品的全部价值。

你去五金店问问他们有没有去除贴纸的产品。幸运的是，他们会带你去一个有三种选择的过道。你决定阅读瓶子上的内容来帮助你决定买哪一个。

第一个瓶子的顶部有一个大标志“Acme ”,这是生产该产品的公司。除了那个标志，这是你能在瓶子上找到的唯一文字:

> 我们 Acme 热衷于生产每个人都能负担得起的优秀产品。我们的 R&D 部门一直在努力开发人们会喜欢的新产品。

第二个瓶子上贴着“远离粘连”的标签上面写着，

> 去除各种轻型粘合剂，不留任何残留物。

第三瓶贴着“游戏清洁剂”上面写着

> 恢复您收藏的游戏的全部荣耀。去除难看的贴纸，而不损坏游戏或其包装。

如果有人想雇佣一名网络开发人员，这意味着**他们有一个需要解决的问题**，就像*你的新游戏*有一个问题——贴纸。如果你给他们的推销没有提到他们希望解决的问题，他们就没有任何理由雇佣你，就像*你*没有一个好的理由去购买 Acme 产品来去掉你的标签，因为你甚至不清楚它是做什么的。

如果你在第二瓶上写下类似标签的内容，说明*确实解决了问题，但非常笼统，你赢得客户的机会会大得多。感觉不像是瞎猜，更像是潜在的解决方案。*

如果你能让**更具体地**解决你客户的问题，就像第三瓶酒一样**，你就会看起来像一个绝对的本垒打⚾** 。尽管有你正在解决的问题的人没那么多，但做*做*的人会付给你**比多面手多几倍的**报酬，因为这让人感觉你了解他们和他们正在处理的具体问题。

这可能需要你做一些工作。如果你像大多数新开发人员一样，你可能没有太多考虑过**你为谁工作**和**你为他们解决什么问题**。我会敦促你做这项工作，这样你卖的东西才能让你的潜在客户真正感到兴奋。这就是**将你从成千上万更便宜的选择中分离出来的原因**，让你作为一名网络开发人员过上舒适的生活。
