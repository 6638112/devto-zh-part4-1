# 6 种帮助你继续从事 JavaScript 项目的方法

> 原文：<https://dev.to/jsmanifest/6-ways-to-help-keep-you-going-when-working-on-javascript-projects-13ah>

在[媒体](https://medium.com/@jsmanifest)上找到我

当你编码的时候，你可能会感到高兴、悲伤、疲惫或者没有动力。其中一些会削弱您有效编写 JavaScript 应用程序的能力。但是有一些方法可以处理这种行为(根据我的经验),让你一整天都编码而不会感到不知所措。

在周末，我平均每天花 15 个小时用 JavaScript 编码，或者至少参与一些 JavaScript 相关的活动，比如阅读图书馆的变更日志或者在 YouTube 上听 JavaScript 相关的播客。虽然长时间做这些是不健康的，但我想说的是，我从来不会感到无聊，T2 总是觉得一天结束得太快，我相信这是一些开发人员在他们的生活中需要的——保持专注，不要分心、无聊或不知所措地编写 JavaScript 应用程序。当周末结束时，我*总觉得我需要更多的时间*。

这篇文章将列出一些让 *me* 长时间编码的事情，目的是帮助阐明 *you* 如何利用这些技巧来提高你对任务的专注度。

话虽如此，这不是一篇我说“请像我一样每天做 15 个小时的代码”的文章。你不需要一天 15 个小时编码。你只需要避免感到没有动力、绝望或不知所措。我们都是成年人，我相信你会做出最好的决定:)

当你不工作的时候，不要像我一样在睡觉前还在编码。我有我的理由，但我很高兴我能够编码，而且从不感到无聊，这恰好是我在正确的时间需要的东西。多陪陪家人，朋友，吃好！

这里有 6 种方法可以帮助你继续从事 JavaScript 项目。

## 1。至少有一个你认为对你的未来有益的正在开发的 JavaScript 项目

[![10-10-10 rule](img/b14df731bd0f57a84b4dcb0d409ea31c.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--1jL_-X1P--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/10_10_10_rule.jpg) 
照片由阿维·理查兹在 Unsplash 上拍摄

我喜欢使用 10/10/10 规则来控制我，并激励我在更长的时间内编码。我总是喜欢有一个项目，当我不工作时，我会积极地开发这个项目，我相信这个项目最终会对我的未来产生很大的影响。

但是你怎么知道你的项目是否对你的未来有益呢？

虽然 10/10/10 规则的定义因地而异，但以下是我一直遵循的 10/10/10 规则:

从现在起的 10 天内，这个项目值得所有的麻烦吗？从今天起 10 个月后怎么样？10 年？如果它满足所有这三个标准，这应该足以让你站起来编码，就像你的生命依赖于它一样。到了睡觉的时候，你会觉得明天要更加努力地编码，因为你低估了今天。

## 2。收听 JavaScript 播客

[![listening to javascript podcasts](img/f71e6b804ff2e90890be6aa38c510594.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--OjqIdfzU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/listening_to_podcasts_in_javascript.jpg) 
照片由 Juja Han 在 Unsplash 上拍摄

我喜欢在编写代码的时候在后台听 JavaScript 相关的播客。这意味着我只需打开播客，让它在后台运行，同时编写 JavaScript。

在我开始工作之前，我经常喜欢头脑风暴一下我要写的代码，这样我的大脑就已经为写作做好了准备，这样我就可以自信地对后台播放的播客给予更多的关注。

我喜欢在后台听播客的原因是因为它有助于我了解 JavaScript 社区正在发生的事情，并且我最终总是会学到关于*人*、技术、代码中的最佳实践等新东西。这在某种程度上填补了错过社会经验的空白。这对某些人来说或多或少很重要。

播客过得很快，通常当播客结束时，我会觉得很有效率，因为现在我脑子里已经有了所有这些新的有价值的信息。这让我对自己的进步感觉很棒。这让我兴奋。你知道当你终于在健身房取得进步，而现在你不想停下来时那种可怕的感觉吗？保持势头。

我听的播客通常能激发我的好奇心。我一边听着肯特·c·多兹和阿里·斯皮特尔的音乐，一边编写代码，学习关于生活的宝贵见解，比如当冒名顶替综合症蔓延到人们身上时，他们如何应对，哪些网络社区是有毒的(实际上非常重要。让自己充满积极的能量，让自己在*、她是如何成功的等方面一直保持动力和坚强。在听这些播客时要记住的一件重要事情是，这些人也是 JavaScript 开发人员。他们经历的任何事情都有可能成为你的经历。*

 *我听过的其他一些著名的是肯特·c·多兹和丹·阿布拉莫夫之间的对话。像这样的播客揭示了一些你在其他地方找不到的有价值的信息。

## 3。听好听的音乐

[![listen to good music while writing in javascript](img/bb6a465746ea47fd1b7ca7360b1e3aec.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--O7q1MbPk--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/listen_to_good_music_while_writing_javascript.jpg) 
照片由斯坦纳·恩格兰德在 Unsplash 上拍摄

这听起来可能很傻，但是当我听让我感觉“很糟糕”的音乐时，我会应用更好的编码实践，这使得编码成为一种很棒的体验。

在你编码的时候听音乐可以产生很多积极的效果。例如，2005 年《音乐心理学杂志》的一项研究表明，软件开发人员在听音乐时体验到了更好的情绪、质量和效率。它还提到音乐可以改变你的情绪。

听着悲伤的音乐，我从来没有过很棒的编码体验。只是把这个放在外面。

## 4。总是在附近喝杯咖啡

[![always have a cup of coffee nearby when writing javascript](img/18c01e48af5d2af19555119a0c9ff70e.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--OFuTPPCJ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/coffee%2Bwith%2Bjavascript.jpg)
Unsplash 上 BRUNO CERVERA 的照片

Coffee 在几个方面帮助你更有效地编码。首先，随着年龄的增长，随着时间的推移，喝咖啡被证明能够提高记忆力和认知功能。这意味着，随着我们能够更快地解决问题并全天专注于任务，我们在编写 JavaScript 应用程序(或*任何东西*实际上)方面的性能变得更强。

现在当我边喝咖啡边编码时，我发现自己编码的时间更长了。这有助于我打字更快，而且明显有助于避免在编码过程中对我的个人生活产生随意的想法。

这有助于控制我的注意力和心态。咖啡消费中的这种效应背后有一些科学依据。

例如，[咖啡可以帮助暂时缓解恼人的头痛](http://www.webmd.com/migraines-headaches/guide/triggers-caffeine)，因为它可以减少体内的炎症。但是有时候我喝咖啡会头疼(T2)。在这种情况下，我可能会想到脱水是原因(*顺便说一句，当这种情况发生时，水总是有助于消除头痛)。咖啡[刺激多巴胺的释放，改善情绪，帮助稳定你的情绪](http://thebrain.mcgill.ca/flash/i/i_03/i_03_m/i_03_m_par/i_03_m_par_cafeine.html)所以我一整天都感觉很好。这也有助于防止随机的个人想法进入编码。这是一种天然的 adderall。*

## 5。每 30 分钟喝一次水(咖啡仍然适用)

[![drink water while developing in javascript](img/9a29a04e9397e15fe9415558d4340afc.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--6fazUyco--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/water%2Bwith%2Bjavascript.jpg)
Unsplash 上斯蒂夫·约翰森的照片

在我的工作中，人们知道我喝很多水，因为我总是带着这些瓶子再装满。我每天至少喝三杯。

水帮助我保持良好的精神健康和幸福状态，这是 T2 和经常喝咖啡的重要组合！这可能是整篇文章中最有效的习惯，可以帮助你更长时间地编码。我会称之为我的秘密，但那是愚蠢的。我们都应该每天喝大量的水和至少一杯咖啡:)

## 6。坐在有阳光的窗户旁边

[![sit next to a window with sunlight when writing javascript](img/e71ac9bdab69685335ff7fe028c7448e.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--RFeComx_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/6-ways-to-help-keep-you-going-when-working-on-javascript-projects/sit%2Bnext%2Bto%2Ba%2Bwindow%2Bwith%2Bsunlight%2Bin%2Bjavascript.jpg) 
照片由伊森·赛克斯在 Unsplash 上拍摄

我的桌子就在我房间的窗户旁边。当我早上醒来时，阳光透过窗户射进来。我的电脑就在它旁边。就好像我的胸口有一扇窗，阳光从里面倾泻进来。 *JavaScript，我来了！*

尽量不要在黑暗的地方编码。当墙壁和桌子是黑暗的，或者当墙上没有任何植物或图片时，[这对你的健康和生产力有不利影响，会导致抑郁和焦虑](http://journals.plos.org/plosone/article/file?id=10.1371/journal.pone.0155614&type=printable)。
