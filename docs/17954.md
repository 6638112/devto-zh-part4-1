# [](#pick-a-marketable-stack-to-learn)挑选适销对路的筹码进行学习

软件开发是一个令人难以置信的庞大学科，你可以学什么/做什么没有限制，但是最重要的是，你要确保你的技能在市场上得到应用，这样你才能维持自己。现在有大量的学习资源可以教你加入职场需要知道的东西(JavaScript，React，HTML，CSS 应该是一个好的开始，Git 是必须的)。

# [](#dont-get-spooked-by-gatekeepers)不要被守门人吓着了

会有人告诉你“太晚了/你做不到/太难了/等等”。不要被这种说法吓到，你可能无法获得一个高级职位或挑选你的薪水，但如果你能解决问题，善于沟通，并展示你的学习能力，有很多企业可以从你的工作中受益。请记住，软件开发最终是实现解决方案，其他人可以通过使用代码作为媒介而受益。如果你能帮助其他人，他们也会帮助你，即使你不是世界上最好的程序员。

# [](#put-you-prior-experience-in-perspective)把你以前的经历放在一个角度来看

交流、研究、谈判的能力，尤其是如果你能领导其他人，可以成为触发你职业生涯的巨大杠杆。如果你习惯于解决难题，要有责任心，就像他们说的，“拥有你的使命”，这对潜在雇主来说是一笔巨大的财富。从雇主的角度来看，重要的是你能为他们提供多少价值，而不是你是一个多好的开发者。我知道这将激怒一些人，但这是一个关于世界现实的铁的事实，这并不意味着你不能随着时间的推移获得更好的机会。

# [](#get-used-to-uncertainty)习惯不确定性

技术一直在变化，这就是为什么我不相信那些说除非你年轻时开始，否则你做不到的人。新的东西一直在发展，但如何使用它却没有规则。如果你踏上了这个旅程，你将不得不不断学习和更新你的技能，所以不要认为知道任何给定的堆栈是任何事情的保证。

# [](#create-a-nice-showcase)打造一个漂亮的橱窗

学习编码的最好方法是通过编码，没有足够好的代码，除非它解决了现实世界中的一些实际问题。如果你开发了一个有用的软件并向全世界发布，没有比这更好的展示卡了，即使已经有类似的东西了。您可以根据个人喜好或用途，对某些工具进行一些变化或调整，制作您自己的版本。世界上有足够多的人可以保证，相对于同一工具的其他版本，有人会更喜欢您的工具版本。

# [](#learn-from-the-best)向最好的人学习

通过跟随正确的人和投资正确的资源，你可以节省大量的时间并且学得很快。你只需要一个 Twitter 账户，就可以找到真正了不起的人，从他们身上你可以学到很多东西，并让自己了解技术趋势(我就是这样做的！).他们中的许多人会非常乐意回答你的问题，并免费提供非常有用的建议。如果你知道业内有人愿意指导你或提供一些指导，那会有很大的帮助。如果你开始与这些人交往，你可以大大加快学习速度。一旦你找到合适的人，你会发现他们中的大多数人可以提供高质量的教育，而你从传统渠道支付的价格只是一小部分。

# [](#be-humble)要谦虚

请记住，世界上一些最聪明、最有智慧的人都是软件开发人员，你只有保持谦逊并知道自己在行业中的位置，才能赢得他们的尊重。无知并不可耻，因为这个世界上的每个人生来都是无知的，所以不要害怕提出问题，并在专门为像你这样的人设立的论坛上引起注意。粗鲁和自负在这门手艺中是不受欢迎的，尽管一些受欢迎的角色可能会以那种方式出现。避开有毒的人，守门人，以及任何告诉你你不属于这里的人。

希望有帮助！

根据我的经验，按钮应该传达动作。尽管按钮显示 Following，但这并不表示它应该做什么。如果我们从 Twitter 的 UX 获取一个页面，当它显示之后的*时，将鼠标悬停在按钮上，将文本更改为*取消关注*，这是预期的动作。所以我认为潜在地显示当前状态是可以的，但是最终在按下之前，它应该传达将要发生的实际动作。*
