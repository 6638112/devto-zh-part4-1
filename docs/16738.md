# 拥抱基地

> 原文:[https://dev.to/kranzky/embrace-the-base-20m6](https://dev.to/kranzky/embrace-the-base-20m6)

从根本上说，Web 和应用程序开发是一个使用三个主要组件和两个网络连接的过程。

这里有一个图表:

```
client <-internet-> server <-intranet-> store 
```

如今,`client`是一个反应式应用程序,`server`通常是 GraphQL 或一个由 web 框架或 Lambda 等无服务器架构支持的老派 REST API,`store`通常是某种数据库，有时是 NoSQL 类型的。

这两个网络连接是致命的。与每个做内部工作的组件相比，它们真的非常非常非常慢。

这些天我正在试验我自己的 omakase 框架，它专注于`client`和`server`之间的实时数据同步，而不是 API，它将许多应用程序逻辑从`server`移动到`store`，这毕竟是事实的基本来源，并且已经进行了超级优化来扮演这个角色。这极大地简化了`server`组件，因为它只需要充当认证、路由和呈现层。

所有这些的诀窍是在`store`中也实现单向数据流。所有的查询都是只读的，数据变更只能通过添加 changelog 事件来实现，允许数据被操纵，就像我们对`git`所做的那样，并且很容易确定特定的`client`更新数据所需的最小变更集。

所有这些都是一种冗长的说法，我认为开发人员应该将更多的注意力放在数据库上，因为那里仍然有许多潜在的潜力等待被释放。
