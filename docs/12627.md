#### CSS&SCS

##### [用自定义属性管理多个背景](https://css-tricks.com/managing-multiple-backgrounds-with-custom-properties/)

很好地了解了如何使用 CSS 自定义属性作为 CSS 值的一部分，而不是整个值。此外，属性可以由其他属性组成，因此这确实为您提供了一些在 CSS 中管理和组合值的惊人工具。

##### [如何用 CSS 网格构建日历](https://zellwk.com/blog/calendar-with-css-grid/)

关于用 CSS 网格构建日历布局的一步一步的教程。确实很好地展示了自动布局算法是如何为您所用的。

##### [关于 CSS 边距你需要知道的一切](https://www.smashingmagazine.com/2019/07/margins-in-css/)

超级深入研究 CSS 利润，包括大量关于可怕的利润崩溃的时间。对于新开发人员来说，边距折叠是 CSS 最困难的事情之一，本文不仅深入探讨了它以及如何避免它，还解释了它背后的“原因”。

##### [位置粘和表头](https://css-tricks.com/position-sticky-and-table-headers/)

简短、甜蜜、切中要害，但它将帮助您在开始使用`position: sticky`处理表格时避免一些挫折和头痛。你应该开始使用`position: sticky`吗？除非你必须支持 IE，否则我会说答案绝对是肯定的，因为它解决了很多以前处理粘性元素的痛苦。

##### [用纯 CSS 进行多行截断](https://css-tricks.com/multi-line-truncation-with-pure-css/)

一个非常非常聪明的把戏。在大多数情况下，使用 JavaScript 可能更好。但是，如果你发现自己在一个环境中工作，你真的想在 CSS 中保持这一点，你不介意在你的内容的结尾和“...”之间有一点额外的空白，这看起来是个不错的搭配。

#### JavaScript

##### [JavaScript 生成器初学者工作指南](https://www.freecodecamp.org/news/beginners-working-guide-to-javascript-generators/)

你在代码中使用生成器了吗？我没有…至少有一部分是因为它们对我来说还不够直观。这个指南非常容易理解，尽管肯定还有一些不直观的东西(传递给生成器的第一个值被忽略，但后面的值不会？什么？)

##### [JavaScript 地图和集合解释](https://www.freecodecamp.org/news/javascript-maps-and-sets-explained/)

对我来说，文章中列出的简单对象上的映射的价值主张似乎很弱(映射给你一致的迭代顺序的事实是一个更强有力的论据)，但是集合是一个非常有用的工具。

##### [用 Vue 构建一个互动且不受干扰的形式](https://medium.com/vue-mastery/building-an-interactive-and-distraction-free-form-with-vue-bfe23907e981)

非常好看的长篇教程。从一组需求开始，进入概念大纲/验证阶段，然后构建完整的实施，解释整个过程中使用的 Vue 功能。伟大的阅读。

##### [在 React 中创建自定义的模态与上下文、门户和挂钩](https://dev.to/treyhuffine/build-a-modal-using-react-context-portals-and-hooks-27g2-temp-slug-9061738)

一篇与上一篇形式相同的短文。构建一个东西(在这种情况下是一个模型)并使用这个过程来探索一些概念，这次是在 React 中。

#### 其他牛气

##### [通过阅读源代码提高你的 JavaScript 知识](https://www.smashingmagazine.com/2019/07/javascript-knowledge-reading-source-code/)

虽然标题和例子是 JavaScript 特有的，但我确实认为这对于开发领域的任何人来说都是一篇非常有用的文章。毫无疑问，学习新技术的最好方法之一是通读并试图理解他人的代码。尤其是写得好的代码，但老实说，任何功能代码。随着开源软件的扩散，你已经有了大量的例子。因此，选择一个你正在使用的库或框架，通读它，建立你的理解。这篇文章将为你提供一些技巧的指导，你可以用这些技巧来让你的学习变得更加平易近人并优化你的学习。

##### [击败技术学习曲线怪兽](https://www.freecodecamp.org/news/beating-the-technology-learning-curve-monster/)

我希望这篇文章有更多的内容，但是我真的很喜欢他介绍的不同学习类型之间的分类(顺序与扫描运行)。我觉得我对新技术的方法是做足够多的连续工作来开始理解全局，然后开始一个项目，并在每次遇到问题时扫描并运行。

##### [关于倦怠的坦诚对话(音频)](https://changelog.com/jsparty/83)

这是最近 JSParty 的一集，三个 JS/frontend 世界的了不起的人都以不同的方式谈论他们与倦怠的斗争。

##### [组件驱动开发指南(CDD)](https://itnext.io/a-guide-to-component-driven-development-cdd-1516f65d8b55)

关于如何使组件成为开发过程的核心的优秀指南。由于组件是当今前端开发中最大的主题之一，无论您选择了什么技术，它都会对您产生很大的影响，因此非常值得一读。

##### [输出:HTML 的原生 live region 元素](https://www.scottohara.me/blog/2019/07/10/the-output-element.html)

对另一个我不知道的语义 HTML 元素的有趣观察。`output`元素是一个原生的“live”元素，这意味着屏幕阅读器会自动期望它更新内容，并且会观看它并宣布更新。酷！
