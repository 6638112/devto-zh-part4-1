# 工作时间，朝九晚五？

> 原文:[https://dev.to/petercour/work-hours-9-5-4ald](https://dev.to/petercour/work-hours-9-5-4ald)

在办公室里，开发人员通常工作时间是朝九晚五。这样效率最高吗？

如果你看看著名程序员的工作时间，[和](https://ivan.bessarabov.com/blog/famous-programmers-work-time)并不重叠。

[![](../Images/2c036ed1aef2f91faec3863ea4b8ce26.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--OCZ7vW78--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://proxy.duckduckgo.com/iu/%3Fu%3Dhttps%253A%252F%252Fthumbs.gfycat.com%252FAnguishedWeepyBluebottlejellyfish-mobile.jpg%26f%3D1)

一个人在 07:00 到 20:00 之间工作最多。又是 10 点到 17 点。Brad Fitzpatrick 有一个有趣的时间表，他在 09:00 到 15:00 之间不做项目。或者他可能一天 24 小时都在工作。

[![](../Images/b6b0d918a6dbd139ab936623e27936a8.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Ih979m_Z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://proxy.duckduckgo.com/iu/%3Fu%3Dhttps%253A%252F%252Ftse4.mm.bing.net%252Fth%253Fid%253DOIP.2r8Pmx0BfjHw0O799x9hHAHaEO%2526pid%253DApi%26f%3D1)


它们似乎都有一个提交次数最多的高峰期。它慢慢积累到那个峰值时间。这代表了正态分布。所有的似乎都遵循正态分布，只是它被移动了。

这就意味着程序员最高效工作的最佳时间是不一样的。因此，当你让每个人工作同样的时间，你不会得到最佳的结果。

我的第一个软件经理说，你能从人们身上得到的最多是每天 5 个小时的实际工作。她还说，程序员需要学会分辨什么时候他们能做的最有效率的事情是往窗外看。
