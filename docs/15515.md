# 你问商业人士问题，但你这样做没有尊重

> 原文：<https://dev.to/rsedykh/you-ask-business-people-questions-but-you-do-it-without-respect-4io7>

好吧，更准确的标题应该是*“如果你是一名自由开发者，如何与商务人士就他们的项目进行交流”*，但我无法抗拒。:-)

当你开始自由开发者生涯时，第一个大项目是改变游戏规则的。大项目带来更多的钱，也不可避免地带来更多的问题。如何避免因沟通不畅而闹僵？

需要很多年才能看清事物的本质。在那之前，你会在理解客户需求、低估任务、错过最后期限、信任和金钱方面遇到麻烦。这是过程的一部分，但是你可以通过更好地交流你对新项目的理解来加速学习。

我根据自己评估一百多个网络和移动项目(从一周到五年的开发)的经验，编制了一份清单。

* * *

第一步，**准备**。

你应该考虑接这个项目吗？客户会同意你的条件吗？为此，您应该回答以下问题:

1.  我能做到吗？
2.  我需要多少时间？
3.  如果出了问题(总是这样)，我要收多少钱才能完成这个项目？

在这一步，你可能不能花太多时间收集信息，但你肯定可以与潜在客户通话。你该怎么办？

打电话之前，做好准备:阅读所有文件(如果有的话)，了解一些类似的项目。

然后，按照算法:

1.  询问项目的目标。
2.  询问边缘案例，询问客户为什么这样看问题，原因是什么。
3.  询问您在阅读原始文档时留下的任何问题。
4.  告诉客户你对一个项目的设想，仔细倾听他们的反应。
5.  尝试找出技术/设计/业务的潜在问题。
6.  更新你的项目愿景，与客户达成一致。这将是你要评估的项目。
7.  讨论网站/应用/服务的哪些主要部分你应该首先彻底评估，把其他的都放在一边。(始终从项目的关键部分开始)。

打完电话后，立即记下所有内容。然后，如果你能等到第二天——就等着，让事情平息下来。你会有一个更通畅的视野，客户可能会给你发送一些额外的信息。

尽管如此，你想达成这笔交易吗？

第二步，**评估**。

按照以下模板，用您的估计写一份回复:

1.  撰写项目描述。
2.  写下项目目标。
3.  写下你为什么喜欢它。
4.  写下项目的特点。提及潜在的问题。
5.  用一般人能理解的项目结构创建一个思维导图。
6.  如果你有足够的时间，可以写一些基本的用户故事(见[备忘单](https://medium.com/@rsedykh/example-canned-user-stories-and-tasks-717c73bc4046))。
7.  写下你认为你将如何构建它，主要的里程碑是什么，你的发布策略是什么。
8.  写下完成一项任务所需的资源。
9.  根据任务和/或资源，编写时间/预算估计。不要把事情过于简单化，但也不要悲观。

这种回应的密度取决于你有多喜欢/需要这个项目。

如果可能的话，总是通过视频电话或面对面的方式给出你的回答。

* * *

几个迭代之后，你得到了一个你和你的客户都有信心的大项目，它可以成为你专业技能和职业生涯的一大步。
