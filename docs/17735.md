## [](#old-friends)老朋友。

十五年来，我和 Wordpress 一直是朋友也是敌人。在那段时间里，我们经历了起起落落。但在某种程度上，我认为我们决定更好地关注我们一起做的好事，而不是让我们分开的斗争。从这个角度来看，我们很像我在线出版生活中的米克·贾格尔和基思·理查兹。

谈到使用 Wordpress，我已经做了很多。博客很多。拥有 Woocommerce 的电子商务网站。会员门户和预订平台。附属网站从各种数据流中获取信息。你说得出的，我都用 Wordpress 做过。

## [](#a-new-client-site)一个新的客户端站点。

我最近做的客户网站是一个运动按摩师的预约和管理网站。网站建设很顺利，客户非常乐于助人和热情。当我们向公众发布时，有大量关于它是如何容易注册和使用的精彩引用和证明。然后好戏开始了。

黑客们几乎立即开始行动。我注意到一些可疑的用户注册，并在关键时刻抓住了一个后门收购的企图。当我着手建立这个系统时，这并不是我的计划。当全世界的脚本小子都试图接管你客户的网站时，谁还想每天玩打地鼠游戏呢？

**回答:没人。**

所以我加倍了我通常的安全措施，列出了 5 个有助于提高你的 Wordpress 网站安全性的提示。

## 提高 Wordpress 安全性的五大技巧。

### 1)做你的更新

这是我的第一条，确保没有人利用你对 Wordpress 安全性的懒散态度。如果可以的话，设置 Wordpress 在有新版本时自动更新主核心安装。如果你明智地使用插件，并且没有在网站上进行太多的定制开发，那么它应该会治愈更多的疾病。(保持定期备份以防万一)。

不要忘记定期检查并更新您的插件，以获得更新版本。这是一个战区，如果你不及时了解最新情况(有时是每天)，就会有一大群混蛋等着利用你的弱点。不要让他们(或他们的机器人军队)满意。

### [](#2-install-security-plugins)2)安装安全插件

但是你刚才不是说不要沉迷于插件吗？下定决心。

Wordpress 进行一次基本的安装就已经够麻烦了，不要紧，一旦你发疯似的开始安装几十个插件，然后你不得不维护并保持更新。当涉及到安全插件的问题时，我会建议一个例外。在这一点上，我已经采取了全力以赴的方法，目前正在使用 [Wordfence](https://www.wordfence.com/) 和 [iThemes Security](https://ithemes.com/security/) 的服务。

两者各有所长，考虑到我所面临的协同攻击，我认为最好是充分利用两者的优势。Wordfence 有一个优秀的文件扫描功能，而 iThemes Security 有许多不同的选项来改变“正常”的 Wordpress 配置。这两个插件都有高级选项，这将有助于进一步增强您的安全性。有一个为你的整个业务提供基础的 Wordpress 网站吗？这项投资是绝对不需要动脑筋的。

### [](#3-enforce-strong-passwords)3)实施强密码

我知道记住一个简单的一个单词的密码更容易，自从你有了第一台电脑后，你每次登录网站都会用到这个密码。我知道让你的用户这么做很诱人，但是，用席琳·迪翁的话说，要三思。有专门允许您设置密码策略的插件，并且在其他与登录相关的插件中通常有选项来扩充您的密码复杂性规则。使用它们。一直都是。

如果你已经安装了上面的一个安全插件，你会发现那里也有实施强密码规则的选项。就像我说的，安全带和背带。严厉一点，让你自己在未来免受一些严重的痛苦。为了更容易记住 20 个字符的随机密码，我最近开始使用[1 密码](https://1password.com)。之前用谷歌 Chrome 密码管理器浏览过之后，我花了一点时间来适应，但我很高兴我现在做出了改变。

### [](#4-disable-xmlrpcphp)4)禁用 xmlrpc.php

听说过这个简单的 Wordpress 时代的遗留问题吗？可能不会，除非你坐下来一个文件一个文件地检查你的 Wordpress 安装文件。它基本上允许远程访问你的网站，这在过去很有用，但在 Wordpress 的现代版本中几乎完全被否决了。一天晚上，我发现它正受到来自世界各地的 IP 地址的疯狂攻击，因为黑客试图访问我网站的管理员账户。

(我明智地决定不仅仅将主管理员用户名命名为“admin ”,这是另一个很好的 WP 安全提示。)

如果您不需要它，您可以用一个专用插件禁用 xmlrpc.php 文件(这不是我的第一选择——原因见上文),代码在您的。htaccess 文件或者——我的选择——通过 iThemes 安全插件上的另一个设置。

时间会证明这是否会有很大的不同，但在几分钟内试图从来自蒙特利尔、芬兰、意大利、法国等地的代理 IP 访问该文件的可疑活动足以让我快速穿过它。

### [](#5-dont-allow-users-to-self-register)5)不允许用户自助注册

这并不总是一个选项，取决于你的网站的性质。如果它是一个免费注册的会员门户网站，那么它真的不在考虑之列。多年来，当我能够关闭用户在博客上的注册，或者让网站管理员手动添加新用户时，我已经陷入了一种虚假的安全感(原谅我的双关语)。

这一次，我需要 Joe Public 能够自己注册和预约，所以是时候收紧旧的约束并尽可能地限制它们了。

如果你不能阻止他们自己注册，限制仪表板访问、强制使用强密码(又来了)以及密切关注谁在注册你的网站是绝对必要的。为了更严格的控制，你可以使用验证码(让每个人都讨厌你)或者让用户点击电子邮件中的确认链接来确认他们是他们所说的那个人。两者都不是万无一失的，但是生活到底是什么呢？

## 所有这些安全努力值得吗？

底线和我的主要收获是，在不使你的网站变得完全不可用的情况下，尽可能多的结合安全措施。任何负责过任何描述的网站，而不仅仅是 Wordpress 安装的人都会知道，你不能相信网上的人会玩得很好。

无论是在他们的脸书页面上发布令人厌恶的内容，用充满仇恨的评论和信息骚扰人们，还是在论坛上成为一个普通的老讨厌鬼，互联网似乎真的让一些人表现出了最坏的一面。

这还是在你到达我们试图阻止的黑客网站之前。有些人这样做只是为了好玩。有些是因为他们可以。有些人实际上有一个游戏计划，而不仅仅是破坏或侵入别人的空间。尽你所能不让他们赢，毁掉你所有的努力。

清理工作会非常痛苦，而且是对时间、金钱和焦虑的巨大浪费，预防总是比治疗好。祝你好运。
