# 为什么我们应该在“可访问优先”的环境中默认使用制表符而不是空格

> 原文：<https://dev.to/alexandersandberg/why-we-should-default-to-tabs-instead-of-spaces-for-an-accessible-first-environment-101f>

我相信你以前也遇到过关于缩进的“制表符对空格”的古老的、基于观点的争论。这种情况一直持续至今，双方似乎都有合理的理由:

**标签页...**

*   较小的文件，因为它只有一个字符
*   这就是它们的用途
*   您可以自定义自己的缩进宽度(👈记住这一条)

**空格...**

*   为了避免空白敏感环境中的问题
*   因为你知道你会得到什么，而一致性是关键

但是，这不仅仅是个人喜好。 [u/ChaseMoskal](https://www.reddit.com/user/ChaseMoskal) 最近在 Reddit 上发布了一篇非常有见地和令人大开眼界的帖子，标题是[“没有人谈论使用制表符而不是空格的真正原因”](https://www.reddit.com/r/javascript/comments/c8drjo/nobody_talks_about_the_real_reason_to_use_tabs/)。

## 使用制表符而不是空格的一个真正原因

在他的帖子中，Chase 解释了他自己在工作场所使用空间的经历，以及它对有视觉障碍的同事产生的负面影响。

他的一位同事使用标签宽度`1`，以避免使用大字体时出现巨大的缩进。

他的另一个同事使用的标签宽度是`8`，因为这最适合他使用宽显示器的情况。

> 我看到人们说“制表符让我们定制我们的制表符宽度”，好像我们这样做是“为了好玩”——Chase

通过在我们的项目中使用标签，我们允许对有视觉障碍的人来说非常重要的定制。定制标签宽度对大多数人来说似乎是不必要的，但有些人依赖它——我们*不能忽视这一点。*

## 先通达

毫无疑问，我们不能让每个人都“转变”到一边或另一边。每个人都有自己的偏好，我们应该庆祝这种自由——我们有能力做出自己的选择。

但话虽如此，我们也应该确保将*每个人*都包括在内，这意味着尊重和使用像制表符这样的可访问工具。

就像“移动优先”的设计一样——确保每个人，无论使用什么设备，都能对你的产品有很好的体验——我们应该通过确保每个人都有相同的工作能力，无论是在工作场所还是在开源项目中，来创造一个“T0”可访问优先的环境。

我认为蔡斯在他的帖子中说得很好...没有任何反驳的理由甚至比重要同事的可接近性需求更重要。

## 结论——可及性第一，个人喜好第二

通过默认选项卡，我们确保每个人都有与他人一起工作的美好体验，不管他们自己的能力如何。

如果你喜欢空格，可以使用自动格式化工具，在打开文件时将制表符转换成空格(所有现代编辑器都有这些功能)。只要确保在其他人处理您的代码之前，空格被转换回制表符。

> 只有确保每个人都有同样的能力，我们才能最大限度地发挥我们的综合潜力❤️
