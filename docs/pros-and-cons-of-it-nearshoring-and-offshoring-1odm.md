# IT 近岸和离岸的利与弊

> 原文：<https://dev.to/ideamotive_co/pros-and-cons-of-it-nearshoring-and-offshoring-1odm>

**离岸外包？近视？利弊如何，如何选择最佳方案？**

IT 市场从来都不是一个简单的市场——可用的专家数量有限，而对他们知识的需求却在飙升。这个问题在欧洲很普遍，根据欧洲委员会的报告，2020 年将会出现高达 90 万熟练 ICT 工人的短缺。

世界经济论坛也强调了这个问题，因为在第四次工业革命的时代，对熟练程序员和软件工程师的需求正在激增。WEF 专家指出，到 2022 年，54%的员工将需要再培训，以便留在市场上并提供所需的劳动力。

这种情况的自然后果是信息和通信技术工人的成本上升，特别是优秀的工人。软件工程师的工资甚至每年上涨 25 %, devo PS 甚至更高。如此多的企业受困于对技能的需求和有限的资源。这就是外包 it 流程变得越来越普遍的原因，无论是简单的还是复杂的，比如[软件开发外包](https://ideamotive.co/blog/pick-custom-software-development-company/)。

# IT 外包——是关于什么的？

在外包的基础上，有一种想法是将业务流程从公司中拿出来，并将其转移到专业的服务提供商那里。外包会计和法律业务很常见，专门从事这些业务的公司赢得了市场的尊重。毕马威或普华永道是国际巨头提供复杂外包服务的最好例子。

根据[全球行业分析](http://www.prweb.com/releases/telemarketing_services/procurement_bpo_market/prweb12627933.htm)报告，到 2020 年，全球外包市场将达到 2200 亿美元左右。这一估计不包括基于云的流程，这基本上是外包特定需求的顶级技术形式，无论是交付内部工具还是提供计算能力。

软件开发外包本身通常分为离岸和近岸。

# 什么是离岸？

离岸外包基本上是一个地理术语，意味着要做的工作被转移到…嗯…很远的某个地方。它通常用于从发达经济体向新兴经济体转移的过程中。离岸外包既包括需要非熟练劳动力完成的简单任务，也包括由超级熟练的专家团队完成的复杂任务。这基本上不是这个想法的核心。

离岸外包通常伴随着经济差异带来的大量节约。这是一件很自然的事情——离岸员工以他们认为公平的价格提供高质量的服务。这通常比当地劳动力认为的要少。

# 什么是近视？

近岸和离岸是同一个概念，但是地理位置发生了变化。在这种情况下，公司将业务流程转移到附近(或至少是邻近)国家的合作伙伴。因此，一家公司既可以是离岸公司，也可以是近岸公司，这取决于与它合作的客户。

# 什么是 Onshoring？

在考虑地理和外包的关系时，离岸外包是最后一个术语。Onshoring 将流程转移到同一个国家的外包公司，但具有 IT 外包服务的所有优势。

**举例:**

一家公司可以是陆上、近岸和离岸服务提供商，这取决于客户的地理位置。所以 Ideamotive，一家波兰软件公司，是:

*   波兰电子商务市场领导者的 IT 国内合作伙伴
*   一家德国零售公司的 IT 近岸合作伙伴
*   美国科技巨头的 IT 离岸合作伙伴

# 离岸、近岸——利弊

考虑到所有这些支持仅指合作伙伴的地理位置，而且在互联网时代，它只对制造业务有影响，大多数利弊都是相同的。但不是全部！

# 专家知识

外包服务提供商只关注他们的业务，即他们提供的服务。另一方面，大多数公司在其他地方赚钱——制造、销售或提供另一种服务。因此，任何辅助角色，无论是会计、营销还是 IT，都是有成本的，由和他们一样优秀的人来提供——一个团队通常限于几个人。

一个服务提供商拥有数十或数百名专家，他们面临着为多个客户工作的诸多挑战。内部团队比外包团队更好的机会是有限的(老实说，这是不可能的，除了像银行 IT 安全这样的小案例)。此外，IT 外包提供商正在寻找该领域的超级明星——提供他们的服务很重要，为多个合作伙伴工作证明高薪是合理的。因此，毫不奇怪，在像 Ideamotive 这样的软件公司中，有顶尖的 UX/用户界面设计师 T1、T2、网络开发人员 T3 和 T4 的移动开发人员 T5，这些人在内部团队中是看不到的。

# 不排斥

一家有多个合作伙伴的公司可能会随着时间的推移感到例行公事，并开始把所有的客户都视为一样。这是一个巨大的挑战，尤其是考虑到每个公司都是不同的，即使是标准的服务也需要根据客户的需求进行调整。

但这是特定 IT 外包公司的劣势，而不是 IT 外包本身的劣势。然而，一些公司对此感到不舒服。

# Timezones

这可能是一个优点，也可能是一个缺点，取决于具体情况。说到 it 离岸外包，这些公司通常在同一个时区，或者相差不超过一个小时。考虑到这一点，很容易安排一个电话或视频会议来讨论任何问题。与周边国家，也很容易满足，无论是在中立的地面会议中心或在总部的任何合作伙伴。

说到离岸外包，这更是一个挑战。虽然项目经理经常在半夜接到电话，但是联系技术领导可能会很麻烦。由于时区的原因，安排临时呼叫几乎是不可能的。

另一方面，时区带来了一系列意想不到的好处。对于一个 IT 离岸合作伙伴来说，晚上要求什么，早上得到什么是很常见的——他们的一天刚刚过去。例如，如果你在美国，而你的 IT 团队在欧洲，你知道当你来工作时，他们几乎已经工作了一整天！

# 语言和文化障碍

随着英语成为通用语，语言障碍似乎消失了。但不是所有人。除了语言本身，在工作方式、坚持时间表或沟通方式方面，差距甚至更大。

最好的例子就是北欧人的守时，而南欧人则比较放松。这些文化差异在与离岸和近岸公司合作时都可以看到。但近岸型团队通常由具有已知思维模式的人组成——波兰人工作灵活，总是让他们的德国伙伴感到震惊，但很少让人感到意外。和远方的伙伴一起工作总是一个谜。

# 总结

考虑到所有的利弊，通常 IT 离岸外包是最好的选择。当需要紧密合作时，了解合作伙伴并在同一时区内联系是一个很好的选择——这也是 it 外包的基本要点。此外，尽管地理距离很短，但经济差异可能很大。

这就是为什么东欧(尤其是波兰)成为主要的 IT 外包目的地。[据 Business Insider 报道](https://www.businessinsider.com/which-countries-have-the-best-computer-programmers-2016-8?IR=T#2-russia-9)波兰拥有世界第三的程序员，让日本相形见绌，将美国或英国远远甩在后面。
