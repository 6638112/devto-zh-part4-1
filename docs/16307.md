## [](#part-1-getting-started-with-the-command-line)第 1 部分:命令行入门

对于任何人来说，命令行似乎是一个令人望而生畏的工具，他们越过他们最喜欢的软件开发人员的肩膀，看到他们锤打充满下划线、破折号和奇怪的特定十进制数的命令。它是一个强大的工具，可以用来横向移动、操作和配置计算机的各个方面，但它也有很多实际用途。命令行是你计算机的光剑:过去的一个优雅的工具，如果使用正确，可以在瞬间解决棘手的问题。

当你作为一名开发人员开始工作时，通常只需要根据需要将几行代码复制并粘贴到终端中，以安装 Node(或 Python 或 Ruby)或为你的站点运行本地服务器，如果这是你到目前为止所做的，那么本指南就是为你准备的！

当您变得更高级一点时，您会发现大多数开发人员使用终端做很多事情。随着您获得更多的经验，它出现的方式之一是在对 git 存储库进行更新时。有*个*图形工具可以推送到不需要命令行的 git 库，但是如果你想融入你的开发团队中的聪明女士们，这是一个知道如何使用的很棒的工具。

除了基本的 git，我每天使用命令行的方式是 [Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli) 。我可以很好地了解我的 dynos 的执行情况，它们遇到了什么错误，并且我可以进行更新——所有这些都可以从一个清晰的界面上进行。我的下一篇文章将专门讨论如何使用命令行来使用 Heroku。

本教程将把这个“东西”称为终端:

这是什么东西？从技术上讲，在我的例子中，它是运行一个名为 Bash 的 shell 的 macOS 终端。Bash 让我能够访问我需要的其他命令——特别是作为操作系统的一部分提供的命令，以及我可能已经安装的其他命令。Bash 是一个命令行接口，或者简称为“命令行”

所以从技术上讲，当你完成本教程的下一部分，并且你正在使用 Heroku 命令行界面(CLI)时，你将使用 CLI (Bash)来使用 CLI (Heroku)。

幸运的是，语言的目的是被理解，所以我将参考通用命令行，让我们运行我们想要的任何东西'终端'，当我们到达 Heroku 特定的命令时，我将称之为' Heroku CLI '

## [](#getting-to-the-command-line-on-mac-and-windows)在 Mac 和 Windows 上使用命令行

在 macOS 和 Windows 上，进入命令行应该只需要点击 2-3 次。

在 macOS 中，命令行程序被称为“终端”。打开 Launchpad，在顶部的搜索栏中键入“终端”。同样，您可以使用 Spotlight 通过单击屏幕右上角的放大镜图标来查找终端，并执行相同的搜索。如果您喜欢浏览文件系统，您可以在“其他”文件夹中找到终端。

在 Windows 端，您可以通过打开名副其实的“命令提示符”程序进入命令行。和 macOS 一样，Windows 有一个系统范围的搜索，你可以用它来快速打开它。同样，你可以在“应用程序”菜单中找到命令提示符。本文主要关注在 macOS 上使用命令行，但是大多数命令都是通用的，可以在两种操作系统上工作。

> 一点历史

直到最近，大多数命令，如您将在本教程的其余部分看到的那些命令(Unix 命令)，在 Windows 中不能工作或不能以相同的方式工作。然而，最近对 Windows 10 的更新使事情变得更加相似。如果你有一个旧的 Windows 系统，你仍然可以通过获取微软的免费软件 Windows Powershell 来获得这些功能。

## [](#navigating-directories)导航目录

这里有一些基本的命令，可以用来轻松地浏览目录。

### [](#-raw-ls-endraw-)`ls`

`ls`是 list 的简称。顾名思义，这将列出您已经打开的目录中的所有文件和文件夹。如果你在主目录下运行这个，你可能会看到“应用程序”、“桌面”、“下载”等等。

[![LS command showing The contents of my home director](../Images/e1a9092f46901bd19420b98567ca301c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--oJF3JtQz--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://tobycreative.s3.amazonaws.com/heroku/tobyfee_%25E2%2580%2594_-bash_%25E2%2580%2594_80%25C3%259720.png)

*我的个人目录的内容*

### [](#-raw-cd-endraw-directory-name)`cd`【目录名】

`cd`代表更改目录。您可以使用它在上面列出的目录之间移动。比如`cd desktop`会带你去你的桌面。您可以运行`cd`命令，后跟运行`ls`时弹出的任何文件夹名称。

### [](#-raw-cd-endraw-)`cd ..`

如果您导航到某个目录并想要返回，请使用`cd ..`向上返回。在这种情况下，从桌面运行`cd ..`会将您带回主目录。如果你想从任何地方回到主目录，`cd`会带你去那里。

### [](#wait-where-are-we-right-now-raw-pwd-endraw-)等等，我们现在在哪里？`pwd`

`pwd`代表“打印工作目录”,意思是在我们的终端上打印出来。这将显示您当前工作的路径。

[![the output of the pwd command showing a file path](../Images/9c8a7927ec67fa4ece92d1cd8a0011cc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Az1RgKcu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://tobycreative.s3.amazonaws.com/heroku/virtualpet_%25E2%2580%2594_-bash_%25E2%2580%2594_80%25C3%259720.png)

我的 macOS 终端上的标题栏显示了我当前所在的目录，但是 pwd 仍然是一个有用的命令，因为它打印了整个路径。

## [](#looking-into-a-file-with-raw-less-endraw-)用`less`看文件

一旦我们通过浏览上面的命令找到一个文件，我们通常会用我们通常使用的任何应用程序打开它。事实上，键入`open [file name]`将会用该类型的默认应用程序打开文件。但是为了坚持使用命令行，并确保我们可以处理一些更高级的情况，让我们使用`less`。

我喜欢以解释为什么这个命令有它的名字来开始这些解释。遗憾的是,“更少”之所以被命名为“更少”,是因为它是“更多”命令的显著改进版本，所以就像“终端”中的许多概念一样，它是一个没有“意义”而更像是“起源故事”的术语😄。

如果我们运行`less [file name]`，我们的终端会改变。我们不再看到我们以前的命令列表，而是整个屏幕充满了我们输入的文件名的内容。我们也不能再输入像`cd`这样的命令了。

[![The less interface showing some javascript](../Images/84a084ee921cf88caf85b40a93f871b0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--1Xmrdyp7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://tobycreative.s3.amazonaws.com/heroku/bin_%25E2%2580%2594_less_%25E2%2597%2582_man_ls_%25E2%2580%2594_80%25C3%259720.png)

这是我们以前没有遇到过的:一个改变终端的命令，以显示终端版本的“全屏”界面。而在 less 中，有两个关键点你需要记住:(如果你是一个有经验的 Linux 用户，正在阅读这篇文章，谁想告诉我更多的关键点:你到底为什么读了这么多这个非常基础的教程？？？)

箭头键:让你在文件中上下滚动(空格会让你向前跳一整页)

问:退出`less`并回到命令行

此外,/键非常有用，可以让您在文档中找到文本。

## [](#getting-help)寻求帮助

### [](#-raw-man-endraw-some-command)`man`【某命令】

作为程序员，我们经常通过谷歌搜索找到答案，而在 Stack Overflow 这样的网站上，解决问题的方法通常是一条很长的命令，比如“哦，只要运行`bundle exec jekyll serve -lo`就能完美运行。”但是这些单独的命令做什么呢？

好消息是每个命令*应该有一个描述它做什么的文件*，我们通过键入`man [command]`就可以到达那里。

[![alt_text](../Images/84a084ee921cf88caf85b40a93f871b0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--1Xmrdyp7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://tobycreative.s3.amazonaws.com/heroku/bin_%25E2%2580%2594_less_%25E2%2597%2582_man_ls_%25E2%2580%2594_80%25C3%259720.png)

*手册页为`ls`*

就像我们在上面运行 less 时一样，man 的界面是全屏的，并使用与 less 相同的键盘命令(箭头滚动，空格前进一页，q 退出)。

**好吧，但是现实检查**:手册页通常是为了给那些非常了解命令如何工作的人而不是初学者增加细节。最好直接跳到手册页的示例部分。我经常在谷歌上搜索一个命令，看一些为从未使用过它的人写的关于它如何工作的帖子。

### [](#getting-help-with-a-command-thats-hard-to-google)获得难以谷歌到的命令的帮助

这有点傻，但像 curl 这样的命令可能很难谷歌到，除非你想要发型设计技巧。在这种情况下，谷歌一下“卷发男”就能找到你想要的。我觉得这个结果比从命令行运行 man curl 对初学者更有帮助。

请记住，您通过 google 得到的答案可能不适合您的命令版本，甚至不适合相同的操作系统。一旦你在 googled 上搜索了足够多的信息，基本上知道了这个命令是做什么的，或者应该做什么，如果你需要更具体地了解它的功能，可能是时候回到手册页了。

## [](#more-useful-tips-tricks-and-commands)更多有用的提示、技巧和命令

### [](#using-the-up-arrow-key-to-access-previous-commands)使用“向上”箭头键访问之前的命令

您可以使用向上箭头键轻松地滚动查看之前的命令。这在各种情况下都会有所帮助。如果你深入到一个文件系统并想退出，你可以输入`cd ..`并使用向上键多次访问该命令，直到你回到你想要的目录。

### [](#using-tab-to-autocomplete-file-names)使用“tab”来自动完成文件名

你肯定不希望每次导航到超长的文件夹名目录时都键入`cd extremely-long-folder-name-here`，对吗？幸运的是，您可以跳过繁重的工作，只需在目录名的开头键入，然后使用 tab 键填充其余部分。在这种情况下，您可以键入`cd extre`并按 tab 键来填充其余的内容。如果您有多个名称相似的文件夹，这显然需要一些操作，但它仍然是一个有价值的工具。

### [](#using-raw-open-endraw-to-view-the-contents-of-a-directory-in-a-finder-window)使用`open .`在 finder 窗口中查看目录的内容

想象一下，你深入到一系列文件夹中，遇到一个需要帮助的文件夹。有些文件需要重命名，需要添加一个空白文档等等。这些都可以在终端中完成，但是如果您还不是使用命令行的高手，这可能会非常痛苦。一些任务在 Finder 中更容易，如果你简单地运行`open .`，它可以弹出打开你所在的目录

## [](#putting-it-all-together)把所有的东西放在一起
