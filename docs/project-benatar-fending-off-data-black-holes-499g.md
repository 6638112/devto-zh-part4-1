## **在一个不太远的星系里**

web 最初是由通过链接互连并被搜索引擎发现的中小型网站组成的集合。但是随着平台的演变，一些杠杆比较大的企业开始扩张势力，消费越来越多。

在相互关联的实体星系中，一切都产生引力。重力是一个平台的力量，通过巨大的流量、社会和经济价值以及随着时间的推移而加强的击败竞争的护城河来实现。如果流量和数据在网站中流动，用户在轨道上运行，那么每个网站都有引力。

有些平台*重力*很大。谷歌、脸书、推特、维基百科、Reddit 等。想起来了。

这种重力对用户来说真的很好。脸书连接世界的总使命在理论上是崇高的。在开发者生态系统中，StackOverflow 和 GitHub 的引力帮助软件开发者完成如此多的工作，并通过熟悉的目的地、工具和外形来组织我们。

任何星系或太阳系都有自然引力。但是当任何空间物体积累了太多的重力时，它就会变成一个黑洞。

随着数万亿美元的网络公司成为常态，这些平台中的每一个都从尽可能多的数据中受益，它们开始吞噬自己的太阳系，破坏竞争，并将自己的意志强加给它们身后留下的一切。他们消费我们的数据，他们消费*开放网络*，他们把以前开放的标准变成*他们的*标准。

我们对任何平台都没有特别的蔑视，但我们越来越担心平台与其轨道用户、开发者、合作伙伴等之间的健康关系。谷歌将越来越多的流量保存在自己的服务器上，而不是通过独立的网站。我们可以继续举类似的例子。

数据黑洞显然是一种模式。组织实际上有承担这一角色的信托责任。

## **不过先介绍一个更小的项目**

支持类似于 DEV 的成熟平台的增长是一个令人兴奋的未来优先事项。但并不是每个网站都应该渴望服务于如此广泛的使命和广泛的利益相关者。

因此，这个宏大计划的一个关键要素是支持能够与大中型平台(DEV 和其他)交互的小型平台的开发。这在实践中意味着数据的输入和输出必须简单；确定规范的 URL 和内容所有权的其他逻辑方面必须简单易懂；以及不同星球(DEV，Medium，future DEV 实例等)之间的传送过程。)一定要流畅简单。

## # [projectbenatar](https://dev.to/t/projectbenatar) <button name="button" type="button" data-info="{&quot;className&quot;:&quot;Tag&quot;,&quot;style&quot;:&quot;full&quot;,&quot;id&quot;:27699,&quot;name&quot;:&quot;projectbenatar&quot;}" class="crayons-btn follow-action-button whitespace-nowrap c-btn--secondary fs-base " aria-label="Follow tag: projectbenatar" aria-pressed="false">跟随</button>

Project Benatar 的任务是创建支持独立平台和兼容实例开发的工具。这是一项事业，将使它更容易建立一个独立的网站，兼容开发，相关的实例，和其他平台，在一定程度上，我们可以创建集成(媒体，Twitter 等)。).这是一个开发 API 的项目，用于创建、更新和删除数据，以及以有用和相关的格式导出数据。所有这些功能都与向最终用户和内容创建者提供更多控制的最终目标有关。

因为这与开发平台的底层机制相关，我们将努力帮助创建所有相关的 API。我们已经开发了一些功能，包括我们的[发布 API](https://dev.to/api) 、我们的[数据导出功能](https://dev.to/settings/misc)、我们的 [RSS 发布功能](https://dev.to/settings/publishing-from-rss)，以及其余的工具。但这是一个长期项目，我们将努力使我们在这场运动中的作用尽可能有用和成熟。

本质上，**我们希望个人网站在生态系统中发挥更强大的作用**，同时我们仍然从社区平台创造的价值中受益。

这个项目将建立在 [POSSE](https://indieweb.org/POSSE) 和分权原则的基础上，但主要围绕这些想法可以影响*我们的*社区、生态系统和项目。随着它的成长，它可能会有更广泛的理想，但我们希望对它持务实态度。这将是降低这些概念的复杂性和创造最佳用户体验的努力。

### [猪八戒道:](#posse)

> POSSE 是 Publish (on your) Own Site，Syndicate Elsewhere 的缩写，这是一种内容发布模式，首先在您自己的域上发布内容，然后将副本联合发布给第三方服务，并通过永久快捷链接返回到您网站上的原始内容。

作为我们已经产生的吸引力的一个标志，这实际上是一个我们不会自己承担*的项目*，但是我们将与相信这些价值观并且已经拥有支持这些理念的产品的合作伙伴一起发起这项倡议。

我们很高兴我们在 **Stackbit** 的朋友们加入到这场运动中来，这是一家专注于使现代网站栈变得可访问和简单的初创公司。他们已经在这个领域开发软件，我们觉得通过一个共享的计划将我们的努力结合起来将会使整个网络受益。
