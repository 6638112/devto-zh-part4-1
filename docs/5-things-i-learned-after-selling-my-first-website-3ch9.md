# 卖掉我的第一个网站后我学到的 5 件事

> 原文：<https://dev.to/abdulisbomb/5-things-i-learned-after-selling-my-first-website-3ch9>

前言:首先，我想说我不是一个专业的网站开发人员，我只是一个刚刚毕业的人，想做一个项目来建立我的投资组合。请自行阅读我只是想分享我个人从我的经历中学到的东西。

当我刚从大学毕业时，我知道要成为一名成功的 web 开发人员，我需要一些项目。我对网页设计了解不多，但我知道我理解核心概念。我联系了当地的一个历史协会，当时我的女朋友是志愿者，他们决定让我试一试。话不多说，以下是我在网站工作后学到的一些东西。

## 1。我的第一个网站不一定是我最好的网站

这可能听起来很傻，因为当然，你有希望在未来成长和建立更酷的东西，但这是我在开始我的第一个网站时必须学习的东西。我想实现这么多很酷的功能、设计元素，并添加一些花哨的功能，以至于我开始忽略客户真正想要的是什么。在绞尽脑汁思考如何做这些事情之后，我回到了我的客户想要的基本问题上。这是一个简单的静态网站，具有一致的设计语言。理解这一点后，我努力创建一个基本的响应网站，然后我只是建立了小尼斯有东西在上面。

## 2。如何有效地与客户沟通

当我第一次签约从事这项工作时，我知道我将不得不面对的一个挑战，就是能够与那些没有与自由开发者合作经验的人交流。很多时候有多个联系人，每个人都有不同的意见，讨论需要几个月才能实现的功能，在设计选择上摇摆不定，以及任意的截止日期。从这件事，我知道我需要起草一份合同，并确定我为我的客户做了什么。通过这样做，我能够设置一个指定的联系点，管理项目的期望，并更好地了解我需要做什么。

## 3。如何挑战自我，让自己走出舒适区

当建立网站时，我知道我想建立它，这样客户就可以有一个很酷的新网站，但我也真的想把这作为一个学习新东西的机会。我知道我的客户想要一个静态网站，所以我开始尝试 Jekyll。我发现它很好地实现了我想要实现的目标。通过尝试 Jekyll、Netlify 和 Firebase 等新工具和技术，我能够挑战自己学习一些新工具，而不会让自己不知所措。现在，每当我继续构建项目时，我都把它们视为一个尝试新事物的机会，而不会让我不知所措。我明白了在尝试学习新事物时感到不舒服是正常的。

## 4。我应该如何向我的顾客收费？

这是我在听
[Syntax.fm](https://syntax.fm) 播客时真正了解到的。有一段时间，当我在做这个项目的时候，我太沉迷于开发的乐趣了，我完全忘记了为这个项目收费。我必须决定是按小时向我的客户收费，还是按约定的金额收取一次性费用。在这个时候，我正在做一份全职工作，只做这个自由职业者网站作为购买任天堂 Switch 的兼职(并建立我的投资组合)。我知道我不可能把所有的时间都投入到这个项目中。我最终为这个项目收取了一次性费用。尽管如此，我真的希望我能计算出我将为这个项目支付的税金。当我马上花掉这个项目的钱时，我确实受到了打击。

## 5。我的工作什么时候结束

一旦我完成了这个项目，我就有点被它搞得焦头烂额了，我知道即使我从中得到了很多乐趣，但隧道的尽头有一盏灯，我想把它运出去。这是我仍在不断学习的东西，也就是说，我什么时候完成这件事。幸运的是，当我建立网站时，我已经把它建成了一个静态网站，所以它不需要不断地修补，尽管我的客户仍然会问我是否有兴趣继续更新新合同，并为这个网站开发新功能。这是我很有兴趣从其他开发网站的人那里听到的事情，那就是**我什么时候完成这件事？**我何时可以停止支持，或者如何处理自由职业项目的移交。
