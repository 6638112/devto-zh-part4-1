# 键盘自动化到极致

> 原文:[https://dev . to/fischgeek/put-extra-keyboards-to-good-use-2d7i](https://dev.to/fischgeek/putting-extra-keyboards-to-good-use-2d7i)

对于那些同样热衷于通过任何必要的手段让我们的计算机为我们工作的人来说，你需要的一个工具是 AutoHotkey。

虽然我可以写一篇关于 AHK 奇迹的长文，但这不是我来这里的原因。我想和你分享一个非常强大的多键盘设置。我个人并不认为这是我的功劳，但是我们都有自己的实现方式。

要点是，我的桌面上有 4 个键盘。1 个主要和 3 个额外的热键映射。这可以通过使用 LuaMacros 和[自动热键](https://www.autohotkey.com/)来实现。我这里有一整套的[指令](https://paper.dropbox.com/doc/Multi-Keyboard-Setup-B1n9J3ll7QBIczL9cJLVr)，但是我会对正在发生的事情做一个简单的总结。这是一个快速的图像。

我先说说我的游戏手柄键盘。我不把它用于游戏，而是专用于 Visual Studio 键位映射。这个游戏手柄有三个配置文件，每个配置文件有 14 个键。我将第一个配置文件用于基本的 Visual Studio 快捷方式，如构建、重新构建、格式化文档等。，5 专用于应用布局。我使用另一个概要文件来启动解决方案。

我的第二个全尺寸键盘主要用于启动东西(应用程序、文档、文件，你能想到的)。例如，如果我想启动或显示 Slack 窗口，我只需按下键盘上的 S 键。

但是，如果我想在特定的工作区上启动或显示时差窗口，该怎么办呢？嗯，它实际上也能打断关键和弦！所以，S+1 会显示或启动 Slack，然后切换到第一个工作区。S+2 将为第二个工作空间做同样的事情。我甚至可以添加更多的键来针对特定的 DM 或渠道！

我也经常使用远程桌面，所以 R+(1-9)将启动一个保存的远程桌面会话。

我使用的最后一个键盘是用来操作窗口、文件、屏幕上的文本等等。这里有一些简单的例子:当我有三个显示器时，我有时会忘记我的鼠标。我可以按下 1、2 或 3，鼠标就会移动到相应显示器的中心，而不是像猴子一样去寻找或摇动它。

按 S+D 会将我的下载目录分类到相似文件的子目录中，并删除停滞的文件。

S+U、S+L 和 S+T 将分别区分当前选定字符串的大小写。

还有更多！我希望你从中有所收获！
