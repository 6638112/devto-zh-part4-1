以下是每个开发人员在 Visual Studio 代码中应该知道的 10 个技巧:

*   命令选项板
*   禅宗模式

*   标签包装
*   Git 集成
*   新的主题和插件
*   VS 代码中的命令行
*   同时编辑多行
*   排除文件夹
*   转到定义

## [](#command-palette)命令调色板

命令调板允许您通过键入关键字来访问各种可用的命令，而不是浏览菜单。

使用组合键**Ctrl**+**Shift**+**P**可以打开命令面板。然后，您可以键入相关的关键字，在下拉窗口中查看命令列表。

## [](#zen-mode)禅模式

禅宗模式是一种不受干扰的观点。窗口中所有多余的工具箱和工具栏都将被移除，让您可以完全专注于代码。

通过选择**视图** > **外观** > **切换禅模式**来切换禅模式。

此外，您可以在 Zen 模式下居中对齐您的代码，体验类似于文档编辑器的视图。通过选择**视图** > **外观** > **切换居中布局**来跳出居中布局。

[![Centered Code in Zen Mode](../Images/02e02765bcc9781ce7467a3d71d3cb30.png)T10】](https://res.cloudinary.com/practicaldev/image/fetch/s--CfzaSX4l--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://blog.syncfusion.com/blogs/wp-content/uploads/2019/06/Centered-Code-in-Zen-Mode.png)

## [](#split-view)分割视图

如果您擅长多任务处理，并且您正在同时处理同一个项目的两个不同文件，或者需要检查两个文件之间的差异，请转到拆分视图。

无论您喜欢垂直还是水平布局文件，拆分视图都可以根据您的偏好进行定制。

您可以通过选择**视图** > **编辑器布局** > **拆分**来实现拆分视图。

## [](#git-integration)Git 集成

Visual Studio 代码附带了 Git 集成，允许您提交、提取和推送代码更改到远程 Git 存储库。

在 Visual Studio 代码中打开属于 Git 储存库的文件。编辑器将显示工作副本和远程存储库中的当前文件之间的差异。没有互联网连接，这将无法工作；需要一个人与 Git 存储库保持连接。

## VS 代码中的命令行

VS 代码带有一个集成的命令行终端。在 Windows 上，此终端显示为命令提示符。在 Mac 和 Linux 上，它显示为 Bash 提示符。无论哪种方式，终端都从当前项目的工作目录(如果加载了一个项目)开始，或者从您的个人文件夹(如果没有加载项目)开始。

命令行界面允许您控制如何启动编辑器。您可以通过它的选项打开文件、安装扩展、更改显示语言和输出诊断。

它还支持拥有多个独立终端的能力。只需点击右上角的 **+** 图标即可生成更多终端实例，或者点击垃圾桶图标关闭当前终端。

## [](#edit-multiple-lines-simultaneously)同时编辑多行

仅仅复制和粘贴代码有点过时了。在 VS 代码中，可以通过将光标添加到不同的位置来编辑多行。如果您打算在不同的行上使用相同的代码，这将非常有用。你可以按住 **Alt** 和**点击**在你想要输入或编辑代码的地方添加一个光标，而不是在所有地方复制和粘贴代码。添加光标后，编辑代码以查看光标添加位置的变化。

## [](#exclude-folders)排除文件夹

您可以使用“排除文件夹”选项移除不需要的文件夹，如 node_modules 或您不想在 Visual Studio 代码中打开的其他文件夹。

要排除一个文件夹，进入**文件** > **首选项**，在搜索设置中搜索**文件. exclude** 。您可以添加不希望 Visual Studio 代码打开的文件夹的模式。

## [](#go-to-definition)去定义

在编程或编写脚本时，您经常会遇到您不认识的变量或方法。您可以花几分钟时间搜索正确的文件，或者按住 Command(在 Mac 上)或 Ctrl(在 Windows 上)并单击变量或方法名称。VS 代码将带你找到它的定义。
