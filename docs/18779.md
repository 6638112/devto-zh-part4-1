# 如何在区块链上安全发送文件[简单方法]

> 原文：<https://dev.to/walterion1/how-to-send-file-securely-on-the-blockchain-the-easy-way-1fam>

## 为什么

随着我们不断听到像 Dropbox 或 WhatsApp 这样的大公司的数据泄露消息，我们越来越难以继续信任他们的文件和个人数据，这让我们思考如何找到更好的方法。我们认为保护数据安全的最好方法是一开始就不要拥有它们。这就是为什么我们选择无服务器解决方案，换句话说，区块链。任何因数据泄露而身份被盗或损失金钱的人都知道集中式应用程序会带来个人风险。

## 如何

对我们来说，最具挑战性的部分是设计一个看起来不像区块链应用的区块链应用！我们希望尽可能平稳地从集中式解决方案过渡到分散式解决方案。这就是 Blockstack 在比特币区块链上帮助我们提供安全和分散的认证服务的原因。

你的共享文件被分成几部分，加密，然后上传到你选择的一个随机集线器。它为您提供了一个私有共享链接，起到了钥匙的作用。没有此链接，任何人都无法访问该文件。与传统的集中式服务不同，BlackHole 不知道谁、谁、什么以及文件在哪里共享！这就是我们将它命名为🕳黑洞的原因

## 技术货色

首先，我们需要一个开放的协议，任何人都可以查看和检查。为此，我们简化了协议，但没有让任何事情拖慢它，因为我们想要快速的体验。正如我所说的，它会将文件分割成多个部分，但我没有说它有一个智能压缩层。为什么聪明？因为它会查看内容并估计它对压缩有多好。压缩后，它会用随机的 256 位密钥加密每一部分，并以多线程的方式上传(默认为 4 个连接)。
协议完成后，我们通过增加一层安全认证和一个 Gaia 提供商，使其与 Blockstack 兼容。

Gaia 的去中心化方法侧重于用户对数据及其存储的控制。如果用户可以选择使用哪个 Gaia hub 提供商，那么这种选择就是支持用户控制的应用程序所需的全部去中心化。

我们用拉撒路和 FPC 编译器制作了它，这是一次非常有趣的经历。您可以使用具有 AES-NI 汇编支持的 256 位密钥的行业标准 AES CBC，并为 Mac 和 Windows 轻松编译它。

## 结果

黑洞是完全原生的 app 因此，它的系统使用量少了很多(10MB 的内存),用户体验也很好。本机还提供了多层加密，性能卓越。黑洞存在于你的菜单栏中，让你只需点击一下就能发送文件。
