# 学习 Web 开发的 7 个简单步骤#fornewbies

> 原文:[https://dev . to/danijelajs/7-easy-steps-to-learning-web-development-fornewbies-4l9a](https://dev.to/danijelajs/7-easy-steps-to-learning-web-development-fornewbies-4l9a)

我写这篇文章是为了帮助我的新手朋友。我想分享一些事情，我希望我在开始编码时就知道。

## 1.带着激情学习

不要因为你从别人那里听说某个无名氏是个程序员，他正在实现自己的梦想，就去学习如何编码。人们通常只看到成功。他们看不到实现目标的艰难。如果你对每一个小细节都没有激情和兴奋，编程可能会很“无聊”。即使你是，有时也是一种挣扎。对我个人来说，**编码就是生命**。我无法想象没有编码的日子。无论你在生活中做什么，都是因为你想做，因为它让你快乐。我的人生格言是:“要么 100%做，要么干脆不做”。

## 2.做笔记

做笔记很重要。这对我个人非常有帮助。当你学习新的东西时，你通过写下来来承认它。你会记住更多。试着用你自己的话，按照你理解的方式写下来。因为当你回到它的时候，它会触发你的记忆，你不必重复整个课程。

## 3.加入编程社区

最重要的步骤之一就是这一步。试着在网上找一个好的编程社区，或者更好的是，在你所在的地区找一个。和志同道合的人联系会对你有很大帮助。你可以在 Twitter 上尝试#100daysofcode 或#301DaysOfCode 这样的编码挑战。你会学到新的东西，也许会认识新的朋友！再加上无论何时你需要帮助，你都会找到愿意帮助你的人！

## 4.你不必记住所有的东西

当我开始编码时，我总是强调要记住每一个小细节。我会花两天时间学习一门课。后来我意识到熟能生巧。你建造的东西越多，你就会变得越来越好。所以不要压力太大！只是练习。

## 5.从不同的资源中学习

从不同的资源中学习是非常重要的。当你完成一个 3 小时的 YouTube 教程时，你还没有完成学习。一个人不会把所有的“秘密”都告诉你，很可能不知道某个编程语言的每一个小细节。但是有人知道那些缺失的“片段”，也许他/她是一个更好的老师。

## 6.一致性是关键。

编程是一项需要练习和坚持的技能。一周 2 天编码是学不会编码的。信息太多了，你不会记得你从哪里停下来的。这就像走一步，退两步。最好加入一些编码挑战。这样你就会养成每天编码的习惯，也不会想太多。

## 7.艰难的方式，是最好的方式

我在标题里说了“7 个简单的步骤”吗？哦，我撒谎了，抱歉😜有时候学习的最好方式是艰难的方式。因为强硬手段会让信息变得难以消除。我不建议你在学习 HTML 和一点 CSS 的时候，马上开始探索 Bootstrap。因为你以后会有问题，在应聘工作的时候，你会不知道如何做一些基本的事情，比如如何搭建一个固定的 Navbar，如何修复一些基本的 bug。也不要尝试一下子学习 2-3 门编程语言。也许有可能，但我不推荐。:D
