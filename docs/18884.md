# 学习如何学习编码

> 原文:[https://dev . to/mthacker 09/learning-how-to-learn-to-code-2h 0i](https://dev.to/mthacker09/learning-how-to-learn-to-code-2h0i)

是的，你没看错...本周早些时候，我向 Dev.to 发表了我的第一篇文章，讲述了我是如何开始编程之旅的。在那篇文章中，我提到了在几年的时间里，我曾数次选择和放弃编码。我讨厌去想如果我坚持下去，我现在会走多远...所以我试着不去关注这个👨🏻‍💻。

我决定利用我的 Dev.to 帐户来创建文章和教程，面向那些和我学习 im 方式相似的人。

这是干什么用的？？在我第一次尝试编码时，我遇到的主要障碍是我可以学习这些材料，但我所学的课程从来没有给出一个理由，说明我所学的东西为什么或如何适用于我成为一名开发人员的长期目标。是的，我能理解什么是数组，什么是对象，什么是函数。我理解每一个的技术定义，但是它如何适用于我的目标呢？它在最后的画面中扮演了什么角色？

在我的职业生涯中，我越来越善于理解如何学得最好；这个方法就是解构。当我看到最终结果时，我学得更好，我把它分成更小的部分，并学习这些部分如何一起工作。我继续这个过程，直到我下降到最基本的。我要说的是，这个过程并不是我每次都遵循的确切过程，但它确实给出了我学习风格的大致想法。

我想利用这个博客作为一种手段，为那些需要更大图片的学习者创建教程；去学习，不仅仅是如何，而是为什么。

希望其他人也能从中有所收获！

编码快乐！
