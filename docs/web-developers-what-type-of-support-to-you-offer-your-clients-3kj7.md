# 网络开发人员，你们为客户提供什么样的支持？

> 原文：<https://dev.to/mrispoli24/web-developers-what-type-of-support-to-you-offer-your-clients-3kj7>

我发现自己越来越需要向现有客户提供某种月度支持计划。到目前为止，我一直做的事情要么是临时的，要么是按小时计价，但是现在我需要更正式地做这件事。

我提出的一些想法是:

*   电话支持时间有时间限制
*   有时间限制的电子邮件支持时间
*   正常运行时间监控(利用节点 ping 之类的服务),并针对停机提供 SLA。
*   错误监控(利用类似 sentry 的服务),有固定的修复关键错误和错误的时间，超过后按比例折扣。
*   管理他们的 DNS 和主机
*   分析报告
*   Lighthouse reporting 有固定的小时数，用于在适用的情况下提高性能。
*   支持计划订户的所有工作的小时和项目折扣费率。

以前，我不想要托管和管理网站带来的所有麻烦。然而，我已经接受了这样一个想法，无论如何，当出现问题时，我最终会是第一个打电话的人，所以最好是尽早发现这些问题，并被授权去解决它们。我还想公平定价，提供最大的价值，让我的老客户群保持满意。

我很想听听其他人是如何处理和/或定价这类服务的。你会增加每月的费用吗？你允许翻车吗？这样做时，有哪些陷阱是绝对应该避免的？有恐怖故事吗？
