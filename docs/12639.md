对任何企业来说，最大的挑战是在几个正确的选项中做出正确的选择。在线投资组合平台 Behance 的联合创始人 Scott Belsky 在他的以下引言中总结了设计选择的挑战

" UX 的经验法则:更多的选择，更多的问题."

A/B 测试是一种行之有效的方法，可以解决为 web 界面做出正确设计选择的难题。现代统计学的先驱威廉·希利·戈塞在 1908 年首次实施了 A/B 检验。A/B 测试的主要目的是分析和选择几个数字设计变量中最合适的选项。这是一个非常关键的过程，确保 UX 研究服务的有效性，以实现最大限度的用户采用。

这篇博客文章评估了使用 A/B 测试方法产生数字设计的利弊。

了解 A/B 测试和所需资源

A/B 测试是一种 UX 研究方法，涉及比较和分析某个网页或用户活动的两种或多种变化。UX 团队在从头开始设计或重新设计 web 界面时可以遵循这个过程。

它包括向不同的用户呈现不同的设计变化。由于这些变化被展示为实时和功能性的网站，UX 的研究人员可以实时研究用户行为。用户研究顾问可以密切观察用户与设计的交互，以记录可用性指标。

早在 2000 年，谷歌的工程师们就成为了首批进行 A/B 测试的团队之一。运行测试的目的是确定搜索结果页面上的最佳结果数量。
