我曾经得到一份新工作，在那里他们谈论他们正在招聘的盛大 v2 重写项目，由于我在 React 方面的经验，我将帮助领导这个项目。一个月后，他们修改了项目，从完全重写到一个单独的代码库中的新特性。两个月后，他们把新的代码库整合到旧的应用程序中。三个月后，他们说这很好，并要求我开始做数据库管理和 Rails 票——这两项我以前都没有做过，我的头衔是“高级*前端*开发人员”

四个月过去了，我意识到我比生命中任何时候都更有压力。我在早上上班前有身体压力反应。我被微观管理着，我觉得没有时间去学习，我在 1:1 中得到了如此强烈的负面反馈，以至于我开始怀疑自己的能力。

我在公司工作的第五个月前就提出了辞职。在告诉他们我将在 2 周内离开后，我被告知回家，不要回来。在我有机会和我的远程队友说再见之前，我对电子邮件和 Slack 的访问被切断了。

我职业生涯中最令人满意的时刻之一是，6 个月后，我在那里的一位同事告诉我，“你当时退出是明智的。”

这是一个很棒的问题。答案取决于采取哪种观点。

## 开发者

如果你在开源上工作

*   这意味着你的贡献会比任何简历更容易被看到。
*   这意味着当你换工作时，你不必离开你热爱的项目
*   这意味着如果公司决定停止开发它，它也不会消亡

开源很棒，因为你可以免费使用它(大部分时间)

*   这意味着在工作中使用它不需要管理层的批准
*   这意味着你可以免费在线学习(与专有软件和 PaaS 相比)

您可以阅读源代码并向作者学习——查看真实的代码，而不是教育示例。

## 公司

开源很棒，因为它是免费的(大部分时间)。有时可能需要向开发者支付费用来支持基于开源的解决方案，但是总的来说，无论如何这在经济上是好的。

开源可以是一个安全的赌注，因为当你使用专有软件时，你依赖于其他公司

*   这可能会停止支持产品
*   在提供新功能、错误修复
*   可能会因为法律(例如，美国的制裁)而被迫停止与您合作

另一方面，使用开源软件，你是安全的，因为如果你真的依赖一些软件，你可以使用它。

你可以把开源当做 PaaS 来卖，几乎不费吹灰之力就能给你立竿见影的收益。这就是 AWS 为托管的 MySQL、Redis、PostgreSQL 等所做的事情。

如果一家公司生产开源软件，这对开发者来说更有吸引力，因为他们将有机会使用开源软件。

开源的东西可能会给你免费的测试人员，有时甚至是免费的劳动力。一些公司使用开源作为招聘过程，而不是他们要求为他们的开源项目做贡献的家庭任务。

## 人道

开源对经济学有很大的影响。开源和自由软件推动了大量的增长，许多 IT 业务的存在是因为存在开源解决方案，没有它他们将无法持续。例如，Linux 和 nginx 占整个互联网流量的 80%。Git 是 Github 成功的一部分。等等。

开源使科学研究成为可能。不需要为软件付费使得研究更便宜。

公司没有必要生产相同的专有产品，没有必要浪费时间和资源。相反，他们可以开发一种产品——大公司为 Linux 和其他大型开源软件做出贡献。想象一下，每家公司都会支持自己蹩脚的操作系统。

## 维护者

最有趣的部分。通常维护人员从他们的工作中什么也得不到(除非这是他们的日常工作)。

有非常著名的开源维护者，例如，Linus Torvalds，Yukihiro Matsumoto 等。，这得到了很多名声上的赞扬(可能因为地位，钱多一点)，但是大多数开源维护者什么也得不到。没有名气。没钱。

有一种奇怪的想法，认为开源会让你出名或者受人尊敬。这是可能发生的，但前提是你要做一个非常受欢迎的项目。

有一个长期存在的问题是如何使开源软件可持续，例如，我们应该制定特殊的许可证，这将迫使大公司捐赠回来。

或者，我们可以从税收中支付开源维护者，并为科技公司增加税收(就像我们支付道路税一样——如果你有车，你应该支付道路税)。

Github 上有这个捐赠按钮，但我不认为这是一个解决方案。如果你是一个社会人，你可以用它获得金钱，否则...
