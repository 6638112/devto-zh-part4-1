无论您是在区块链空间、Web 3.0 空间、云计算空间还是实际空间中进行构建，都越来越希望让用户能够更好地控制他们的数据。就连大型科技公司也在制定数据标准，以帮助实现数据可移植性(即使这可能只是表面现象)。虽然有多种方法来处理用户控制的数据，但 Web 3.0 空间(包括区块链解决方案)通常专注于非托管的完整控制系统。这些系统需要一定水平的技术知识，它们要求用户跳出与应用程序交互的正常流程，并且它们要求用户承担他们不习惯承担的负担。如果用户希望这样，这完全没问题。但是，对于大多数希望对自己的数据进行某种控制，但又不想承担额外责任的用户来说，该怎么办呢？

先说问题中的额外责任。对于大多数去中心化的支持者来说，给用户控制权的想法意味着给用户他们的加密密钥和身份信息，并告诉他们要妥善管理，否则就会失去一切。对一些人来说，这是正确的方法。有人想要那种程度的控制。但你的父母可能只是想知道他们的数据在哪里，如何被使用，以及如果他们想要的话，他们如何才能获得这些数据。我们是不是应该强迫他们承担起管理钥匙和钱包的责任，把纸片塞进保险箱，这样他们就不会丢失所有的东西？大概不会。但是我们也不应该排除这个选项。

给用户控制权是分层次的。有一天，妈妈和爸爸可能会决定他们真的想省去中间人，即便利层。如果他们做了那个决定，应该有办法让他们达到这个目标。应该有一种方法让他们说“我现在准备好管理我的身份和加密密钥了，请。”不过，在他们做出决定之前，应该有一种方法可以帮助他们管理自己的身份和加密密钥，而无需让他们经历太多的麻烦。而不完全集中解决方案。因为，如果解决方案是完全集中的，该解决方案的失败将导致他们无法访问他们的数据，这意味着他们实际上根本没有控制权。
