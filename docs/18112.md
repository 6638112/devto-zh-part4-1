# CSS 和 JavaScript 只是渐进增强

> 原文:[https://dev . to/celestial cinnamon/CSS-and-JavaScript-are-just-progressive-enhancement-akf](https://dev.to/celestialcinnamon/css-and-javascript-are-just-progressive-enhancement-akf)

我不是一个很好的作家，但我一直热衷于跟上网络上出现的事情。当人们在讨论 CSS-in-JS 的时候，我在场。阅读人们对是否使用 React 或 Vue 或 Angular 或其他任何东西的意见也很有趣。当比我更有经验的人谈论基础和框架时，对我来说也是一次很好的经历。

我从事 Web 开发已经 3 年了，我可以说这些问题对我来说很重要。它们要么让我的生活变得更轻松，要么让我头疼，因为我试图在最后期限前完成那些我无论如何都不会完成的任务。但是有一点我注意到，每当人们找到我，发现我是以创建网站为生的，他们并不真正关心我们使用什么或我们如何做事。他们在网上是因为网上的东西。这是我在其他开发人员和设计人员的博客中多次读到的内容。

重点是提供高质量的内容而不是工具。是我们用户关心的。但我想说的是语义，HTML 标签的正确用法。

我偶然发现了蒂姆·伯纳斯·李爵士在 LCS 35 周年纪念日的演讲的一份文字稿，[](https://www.w3.org/1999/04/13-tbl.html)网络的未来。在那里，他提到了一个*文档和数据*，其中*文档*是我们人类消费的东西，而*数据*是机器人用来理解内容的东西。作为有知觉的生物，我们人类有能力理解文本内容，或任何真正的内容，并理解它是关于什么的。但是机器人，虽然他们有能力阅读更多的内容，但他们没有能力理解这些内容。这让我想起了伯纳斯·李爵士发明的东西:超文本标记语言。它标记了我们人类可以理解的内容(文档)，这样机器人就可以真正理解它并利用它做事情(数据)。

有各种文章、视频和播客讨论语义 HTML 有多重要。

*   它需要更好的搜索排名
*   它导致更易维护的代码
*   更容易设计风格
*   它**更容易接近**
*   它使您的代码经得起未来的考验

然而，没有多少人买那个。仍然有人满足于在适当的时候使用`<div>` s 而不是`<article>` s 或`<section>` s。

最近偶然看到蒂姆·伯纳斯·李爵士的一篇文章， [<cite>给年轻人的答案</cite>](https://www.w3.org/People/Berners-Lee/Kids.html) ，他在文章中回答了学生们经常问的关于项目或作业的问题。其中一个问题是<q>“是什么让你想到了万维网？”</q>他的回答是:

> 因为欧洲核子研究中心的人来自世界各地的大学，他们带来了各种类型的电脑...实际上，我编写了一些程序，从一个系统中获取信息，并将其转换，以便可以插入到另一个系统中。不止一次。当你是一个程序员，你解决了一个问题，然后又解决了一个非常相似的问题，你经常会想，“难道没有更好的方法吗？我们就不能永远解决这个问题吗？”这就变成了“难道我们不能转换每一个信息系统，让它看起来像是某个虚构的信息系统的一部分，每个人都可以阅读吗？”这就是万维网。

网络是人类最伟大的发明之一，因为它打破了界限。它是全球性的，不惧怕操作系统、设备和机器的差异，甚至不惧怕使用它的人。到处都是*而*只是工作*。然而，当你想一想，我们仍然处于一个碎片化的网络空间——人们有残疾，在不同的环境下操作，使他们的浏览体验不同；我们使用不同的设备，有人使用台式机和笔记本电脑，有人使用平板电脑和移动设备，还有人在智能电视和其他设备上浏览网页；在这些设备上，我们还有另一种浏览模式——阅读器模式，如 Firefox、命令行浏览器、屏幕阅读器和其他类型的东西。*

 *为了维持网络作为一个开放平台，聪明人一直在设计策略来打破这些界限和分裂。我们有了响应式网页设计，现在又有了内在网页设计，来解决设备差异的问题。我们也有 WebAIM 和大量的博客、视频和播客来敦促世界各地的开发者让他们的网站对每个人都是可访问的，不管他们是否残疾。

我认为我们的问题之一——我自己也有，只是现在才意识到——是我们把网络当作 HTML + CSS + JS，而实际上，它只是 HTML。让我解释一下我的意思。我们试图以一种能够响应和适应设备宽度的方式来设计站点。但是 CSS 并不是无处不在！就在两年前，我使用的手机使用的键盘就像人们在 2003 年使用的键盘一样。它有一个 WAP 浏览器，不能连接互联网，出于好奇，我把它拿出来，在上面测试了我的作品集。我的 CSS 和 JavaScript 没有通过，因为它的 WAP 浏览器甚至没有获取(我认为)CSS 和 JS 文件。它拥有的是我用 HTML 输入的文本和图片。这让我想到了人们以不同方式浏览网页的其他情况，比如在聚合网站中，出现在搜索结果中的网站，命令行浏览器，以及所有其他我们无法控制内容外观和行为的地方，在这些地方，我们唯一能控制的是内容的实际内容，而让上下文决定其他一切。

我想这就是我所说的 CSS 只是一个渐进增强的工具。CSS 并不适用于所有人。JavaScript 也是如此。我们忘记了，至少我忘记了，有些人过着不同的生活。我们只是把 CSS 和 JS 放在那里*给可以从中受益的人*，而不是每个人。所以我认为我的观点是使用语义 HTML，这样内容可以在任何地方出现。通过这种方式，我们这些负责任的开发者确保了网络的普及，并且继续超越任何阻碍它的界限。毕竟，如果我们有时间讨论用户不关心或者甚至不知道存在的东西，那么我认为我们有几分钟的时间来决定哪个 HTML 标签最适合一段内容。*
