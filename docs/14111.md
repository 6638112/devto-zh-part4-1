# 网络浏览器是企业的最佳选择吗？

> 原文:[https://dev . to/openlow code/is-web-client-optimal-for-enterprise-1o4j](https://dev.to/openlowcode/is-web-client-optimal-for-enterprise-1o4j)

在 20 多年构建、维护和管理企业应用程序的过程中，我对许多主题都有强烈的看法。举一个例子，我肯定喜欢强类型语言，因为它们会简化维护，并允许典型的开发人员更快地升级。在开源解决方案中使用标准技术对于企业来说也是显而易见的。您希望应用程序在 10 年后仍能运行，而不是永远支付许可费。

所以当我开始构建 [Open Lowcode](https://openlowcode.com) ，一个开源的低代码平台的时候，很多选择是显而易见的。该应用程序将使用 java 编写，并使用 SQL 实现持久性。它将少用开源库，其中大部分来自 Apache。您可能会发现这些选择平淡无奇，但是平淡无奇正是企业应用程序艰难生活中所需要的。

现在，我必须决定使用哪个客户端，这是最难的一个。当然，我想要一个瘦客户机:企业应用程序维护非常频繁，比方说一个补丁一个月两次。我不想每次都重新安装客户端。接下来的问题是:我应该使用 web 客户端，还是开发另一种瘦客户端？

Web 客户端起初似乎是一个显而易见的选择，互联网巨头已经在各种框架和技术上投入了大量资金。然而，由于种种原因，我最终选择了一条不同的路线。

我选择使用 [javafx](https://openjfx.io/) 开发一个特定的瘦客户端(企业 app)。客户端显示由完全适合企业用途的高级原语组成的页面。这包括著名的类似电子表格的界面来操作表中的数据，这是企业软件的“杀手级部件”。据我所知，我的研究可能还不完整，这种小部件并不真正可用，或者用网络技术制作成本很高。

其他优势包括将客户端和服务器之间发送的数据限制在最低限度的可能性，或者在军事环境中工作时实施专门的安全工具。我预计 javafx 在未来几十年将保持稳定。一个潜在的缺点是在客户端计算机上安装软件，但这可以通过使客户端安装简单并拥有自动更新程序来解决。

尽管如此，这个选择还是我最犹豫的一个，想听听大家的看法。
