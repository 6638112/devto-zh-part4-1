# 管理开发运维项目的 8 个最佳实践

> 原文：<https://dev.to/mskshahrukh/8-best-practices-for-managing-devops-projects-37m8>

DevOps 遍布整个软件行业。对 DevOps 的宣传是真实的，也应该是真实的，因为 DevOps 简化并加速了开发过程。情况是这样的，大多数领先的科技公司都无法想象 2019 年没有 DevOps 的工作流程。它已经成为一个组织内部 It 结构的一个组成部分。

对 DevOps 的高需求也增加了市场对 DevOps 工程师的需求。有经验的 DevOps 工程师并不真的需要马上找工作，他们每月至少会接到一些新公司的面试电话(鉴于他们有很好的在线作品集)。但是只有经验丰富和熟练的 DevOps 工程师会被大多数公司进一步追求，因为他们知道管理 DevOps 项目的最佳实践。

正确的技能有助于为项目构建正确的开发运维流程。这些技能是随着时间的推移而获得的，但有一些最佳实践是每个 DevOps 工程师都应该知道的，我们将在下面讨论。

当成为一名 DevOps 工程师，甚至是任何类型的开发人员时，一个人应该始终保持好奇，想知道还能做些什么来以更优雅的方式完成同样的任务。这种技能在 IT 行业至关重要，大多数 DevOps 工程师已经知道这一点，不管他们是否获得了[最佳 DevOps 认证](https://www.devopsu.org/certifications/)，这种技能都会随着时间的推移而发展。虽然 DevOps 工程师拥有某种 DevOps 认证也很重要。

在 DevOps 中管理项目的 8 个最佳实践

DevOps 的主要目标是以更少的开销使开发过程更快、更可靠。自动化是任何 DevOps 实施成功的关键，这包括部署前的自动化测试。

以下是管理 DevOps 项目的 8 个最佳实践。

1.  **从一开始就实现自动化**devo PS 的主要关注领域之一就是自动化。良好的 DevOps 流程在很大程度上依赖于持续集成和持续部署。正如在任何遵循敏捷开发的项目中一样，有许多变更需要非常频繁地部署或发布。每天都会推送大量的代码变更。有必要在这里实现某种自动化，这样这个过程就不用手动完成了。你可以通过在线获得[敏捷认证培训](https://www.devopsu.org/training/certifications/agile.html)，或者通过观看 YouTube 视频或咨询专家来学习。
2.  **可靠的测试策略**在部署之前，每个代码块都需要进行测试。相同类型或相似功能的测试被分组到由开发人员自己编写的测试用例中。作为测试策略的一部分，DevOps 工程师还应该提供正确测试类型的定义。从一开始就有一个清晰的测试策略有助于消除未来的主要错误。任何新特性的部署只有在所有测试用例以及单独的测试都通过之后才会发生。
3.  **关注协作团队** DevOps 是关于参与整个开发阶段的所有各方的协作。顾名思义，DevOps 本身是开发和运营两个词的结合。这突出了 DevOps 项目中协作的重要性。为了实现成功的开发运维工作流，各个团队和客户都需要保持同步。
4.  **使用正确的工具**部署、开发和测试应该使用正确的工具。就需要使用什么工具而言，每个项目都是不同的。有一些常见的工具，如代码编辑器和在线存储库管理工具，它们在不同的项目中可能是相同的。但从本质上讲，每个项目本身都是独一无二的，都有一些独特的特征。在进行任何开发之前，都需要了解这些特性，并且要记住，必须为特定的项目选择合适的工具。
5.  **应用程序监控**在极少数情况下，应用程序会意外崩溃。大多数情况下，这与代码中的错误无关，而是一些被忽略的边缘情况。大家都很熟悉 Windows OS 中的蓝屏，上面写着“出错了”。微软是一家价值数十亿美元的公司，如果它能发生在他们身上，它也能发生在你身上。在这种情况下，最好的方法是进行某种应用程序监控，这可以让您在应用程序的关键功能或整个应用程序崩溃时立即知道。
6.  项目的鸟瞰图(仪表板)在大型项目中，有很多事情要做，很多人参与其中，事情往往会聚集在一起。如果没有一个合适的仪表板来管理每个单独的任务，整个项目将变得混乱不堪。对整个项目有一个粗略的鸟瞰图是非常重要的，可以看到正在进行什么，需要做什么，团队在哪里陷入困境。这种任务和资源管理仪表板的一个例子是吉拉。吉拉是大多数公司使用的主要任务管理软件。虽然有些公司自己制作仪表盘用于管理。
7.  **减少开销**开销是可以减少的额外时间。在软件开发中，如果你没有遵循敏捷过程，那么会有很多开销，比如在部署/发布之后召开会议来收集新的需求，从测试团队那里得到响应，等等。敏捷开发将这种开销减少到一个极限，比 DevOps 发挥作用时还要少。开发自动化和连续性的一些概念，这在很大程度上减少了开销。在 DevOps 中，DevOps 工程师的工作是找到并尽可能减少开销。
8.  **获得评审**从开发团队和客户那里获得评审在软件开发过程中的所有参与方之间建立了纽带。评审有助于消除客户和开发团队之间的误解，如果需求没有被正确理解，还可以减少浪费精力的机会。在 DevOps 项目中，在小的变更后获得评审总是更好的，这样任何不规则或误解都不会导致大规模的 bug 或缺乏功能。
