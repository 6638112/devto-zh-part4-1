我对你在这里提到的一些观点感到内疚，尤其是第 1、2 和 3 点，但我正在努力😌

如果我可以补充一件事，那就是:

不要被你的角色或职位所限制。

仅仅因为你的头衔表明你是前端工程师、后端工程师等等，这并不意味着你应该把你的团队或项目局限在那个特定的领域。

当出现新功能、出现问题，并且你们都提出了最佳解决方案时，请参与团队讨论。

相信我，你会对这个项目有更深的了解，它的整体架构就像 API 是如何构造的。因为即使你在这方面没有太多的经验，你也可以提供一个不同的视角来看待这个问题。

你的建议可能不是解决方案，但你的洞察力可以帮助你的团队成员以不同的方式看待问题，他们可以在此基础上提出更好的解决方案。

你好。

不久前，我和一位医生朋友进行了一次谈话。他抱怨说，他应该一直坚持学习，因为这项工作极其复杂，医疗设备、技术和可用的药物发展非常快。

我回答道:

> 良好的..至少你的硬件没那么经常换。诚然，你所在部门的知识进步很快，但人体保持不变，你需要抛弃五年前学到的关于疾病的知识并不常见。

我的朋友，这种事不断发生在我们身上。

我从事编程已经超过 30 年了(从我 10 岁开始)，我已经多次面对知识过时的现实。

我记得我用过 BASIC，QUICKBASIC，Turbo PASCAL，Delphi，ASP，...以及其他很多现在用的不太多的技术或者计算机语言。你今天所学的东西过几年就会过时...

但是如果你专注于理论，过时的可能性就会减少。

我知道这个行业正试图通过在线学习和认证来贬低大学。但是如果你想获得永不过期的知识，你可能还需要去上大学，或者至少买一些经典的理论书籍(数据库理论、编程范例、操作系统等等..).

一个很好的来源是麻省理工学院的开放课件。

但是..嘿..你可能不需要那个。这取决于你的期望是什么，你的人生规划是什么。

如果你想将编程技能应用于某个问题领域，那么这只是一个寻找相关信息的问题。

我们都需要认识到，不存在像万能程序员这样详细了解所有行业分支的概念。我们最多可能对每一件事都有一个大概的了解，而现在每个话题都是关于什么的，但并不包括所有的事情。

做一个*全栈程序员*已经很难了，几乎不可能深入了解*所有栈*。

如果你学会了信息学和计算机科学的所有知识，那么你会意识到你老了，可能什么都不用了，失去了享受生活的时间。

所以不要期望太高，试着通过做你需要的事情来学习，并不时地阅读一些理论。你很快就会发现，让程序员更有价值的不是他们关于信息学的知识，而是他们在问题领域(软件将要使用的分支)获得的知识...)

也花点时间阅读一下这些[编程范例](https://en.wikipedia.org/wiki/Programming_paradigm)中的一个是关于什么的。不是很深，只是为了看描述。

试着阅读科技新闻，如果一些概念对你来说是新的，试着阅读基本的定义。

祝你好运，放松，享受学习！
