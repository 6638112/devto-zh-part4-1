# 如何掌握数据库设计？

> 原文：<https://dev.to/madhu0/how-to-master-database-design-36jp>

这是我在 dev.to 的第一篇文章(可能是任何开发者社区中的第一篇)，我感到紧张。我不是一个好作家，我的英语也不好。所以请容忍我和我的英语。

在过去的 2 年里，我一直在使用 React 和 JavaScript，觉得是时候切换到后端了。所以我开始接受后端采访，因为我熟悉 java、python、SQL 和 mongodb。在一次面试中，我被要求设计 Whatsapp (DB 模型和 Rest Apis)。开始写模型(类)，想出了 4-5 个模型(在我看来是很少的)。并且我也困惑于一个方法应该在哪个模型中。然后我决定学习数据库设计，最好的方法(至少对我来说)是从头开始设计和实现一个应用程序。我谷歌了一下，发现了一些应用程序的想法，但我只是想先在这里检查一下。我走的方向对吗？我遗漏了什么吗？也欢迎任何学习设计的应用程序想法。
