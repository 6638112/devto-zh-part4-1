# 4 个伟大的自举方案

> 原文:[https://dev . to/flipped coding/4-great-bootstrap-alternatives-5602](https://dev.to/flippedcoding/4-great-bootstrap-alternatives-5602)

当 Bootstrap 发布时，web 开发社区为之疯狂。它使得创建一个响应性前端变得如此容易，以至于几乎每个人都在某种程度上采用了它。现在，如果你环顾互联网，你会注意到许多网站看起来非常相似。感觉 Bootstrap 成了互联网的模板，这是可以理解的。

一个设计师团队可能需要一两年的时间才能做出与 Bootstrap 相当的东西，许多公司不愿意花时间或资源去重新发明轮子。你再也不用担心这个问题了，因为有很多很好的方法可以替代 Bootstrap。最好的部分是，他们中的一些人甚至不需要 jQuery！这里有一些我修改过的其他 CSS 框架。

## [](#bulmacss)[Bulma.css](https://bulma.io/)

这是我目前最喜欢的。它不需要 jQuery，有大量预构建的组件，您可以使用 mixins 和变量对其进行定制。你将不得不对你的表单做更多的工作，因为这是纯粹的 CSS，所以记住这一点。最多，你会花 10 分钟来弄清楚这个框架是如何工作的。另外它叫布尔玛。还有其他 DBZ 书呆子吗？

## [](#materialize)[物化](https://materializecss.com/)

我接触过的很多客户都有这个框架的项目。它给你的项目更多的工业感觉。我用过的大多数应用程序都处理单页调度或仓库管理之类的事情。在你习惯它之前，它有点棘手，但是关于它的文档非常好。与 Bootstrap 类似，您必须小心任何 jQuery 问题。

## [](#cardinalcss)[红衣主教](https://cardinalcss.com/)

如果你需要一些东西来给你一个基本的移动优先的布局，这是完美的。CardinalCSS 没有加载大量预构建设计元素的框架，因此真正让页面变得生动起来要靠你或你的设计团队。这更像是响应能力的骨架。它去掉了制作移动响应网格的复杂部分，而不需要添加许多样式。

## [](#foundation)[基金会](https://foundation.zurb.com/)

目前，这可能是 Bootstrap 最强有力的竞争者。有一些课程和证书可以让你成为一名正式的基金会开发者。许多大公司，如亚马逊和 Mozilla，已经在生产中使用它。这是另一个框架，它使用某种 JavaScript 让某些元素工作，但是它有大量的文档供您阅读。

这些只是 Bootstrap 的几种替代方案。它们比 Bootstrap 提供了更多的微调，因为它们更轻量级。使用不同的框架也会给你一个机会让你的网站在 Bootstrap 的海洋中脱颖而出。尝试其中一个，让我知道你的想法。

你知道一些其他的自举替代品吗？有这么多，很难决定哪一个是最好的。
