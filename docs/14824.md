# 如何为你的队友举办远程研讨会

> 原文：<https://dev.to/spukas/how-to-do-a-remote-workshop-for-your-teammates-5297>

研讨会是一个很好的学习过程，通过让参与者积极参与讨论和练习，向他们展示您的想法，并通过应用概念提供实践的可能性。车间可以是本地的，也可以是远程的。虽然本地设备可以让你直接与参与者交流，但远程设备更具可扩展性，不受位置限制，压力更小，因为你不需要站在一群人面前。在本文中，我将分享我在为您的团队成员准备一个关于某个技术主题的远程研讨会时应该做和不应该做的事情。

# 犹豫

决定是否参加研讨会，你可能会遇到每个人都会遇到的问题；自我怀疑和不确定性，如果你有足够的资格解释你的概念。但是请记住，你不需要成为这个话题的专家，你需要比你的公众知道的多一点，当然，提前做一些准备。在参加研讨会之前，我总是感觉压力很大，吓得屁滚尿流，但这种经历是值得的。我对犹豫的两点看法是:感到恐惧，但还是要去做。

# 对你有什么好处？

测试你的知识的最好方法是试着解释和教别人。通过这样做，您将:

*   了解主题
*   获得声誉
*   训练你的沟通技巧
*   学习如何表达想法
*   教导他人的成就感

这些对我来说都是极好的卖点。

# 准备步骤

#### 找到题目

没有一个你想呈现并教给别人的想法？想起你最近学过的新话题或感到兴奋，或者也许有你想学的东西？如果你没有想到什么，你可以随时问你的队友，他们想学什么。

#### 研究它

研究题目。例如，如果是一些框架或库，官方文档是一个好的开始。在网上做一些指导，阅读博客文章，写下你想从这个主题中提出的东西的笔记。

#### 概述

写一份副标题计划。它会在这个过程中指导你，帮助你坚持下去。从概述开始:

*   先写标题(即如何调试 JS)
*   想一想标题(例如，引言、调试 JS 的五种方法、结论)
*   通过提问每行的什么，为什么，如何来找出副标题

之后，你应该有一个相当不错的计划来遵循。

#### 合适的工具

我只尝试过几种媒体工具，这就是我对它们的看法:

*   对于铸造和分享屏幕:谷歌会议(公司 acc。)、Google Hangouts(个人 acc。)，并缩放。虽然第一个是简单的设置，变焦有记录的可能性优势。如果您仍然喜欢使用 Google infrastructure 来共享屏幕，您可以使用 QuickTime Player 或第三方工具来录制它。
*   演示文稿:Keynote (Mac)和 Google Slides(来自 Google Drive)。我更喜欢后者，因为它将幻灯片保存在云中，可以跨平台使用，并且有一个友好的用户界面。

#### 简单幻灯片

使幻灯片简单。这是清单:

*   写短标题
*   添加图像；有些人更喜欢视觉学习
*   利用要点，每张幻灯片只添加几个要点
*   制作动画；在解释时，一个一个淡入要点或图片
*   不要阅读幻灯片上的文字；它们应该只是你想说的一个参考
*   添加演示者备注，以应对您陷入困境的情况。但是只添加简短的参考文献，这样你就可以思考并记住你想说的内容。如果你添加了太多的文字，你最终会全部读完。

#### 例子

添加你所解释的概念如何应用的例子。我建议使用您日常工作或产品代码库中的样本。参与者将熟悉示例的内容，并有兴趣了解这些建议如何改善日常工作。

#### 任务或练习

为了吸收新信息，允许参与者应用它。准备一个有任务或活动的环境。最好使用您的团队日常工作的应用程序。

# 要避免什么

我在演讲过程中犯了很多错误，并通过艰难的方式认识到这些错误。以下是您下次研讨会需要避免的几点:

#### 讲课

不惜任何代价避免讲座。在我做了 45 分钟的报告后，每个人都在鼓掌，但不是因为报告的内容，而是因为它完成了。材料太枯燥，例子很难，参与者没有机会尝试，我一直在发言。灾难。

#### 独白

尝试让公众参与对话和讨论。提问他们是否觉得有用，他们是否理解这个概念，他们对这个概念的看法。不时询问他们是否还在关注你，也许他们有问题或想补充什么。

#### 单调的声音

听单调的声音会催眠你的听众，他们会失去注意力。如果你像我一样难以通过演讲表达自己的情绪，想想这个话题，想想你是如何兴奋地与他人分享。

#### 过于严肃

在你的演讲中加入一点趣味会营造一种好的气氛，有助于打破与听众之间的僵局。添加有趣的图片或有趣的事实来放松观众。这会让你的演讲变得愉快而难忘。

# 总结

总而言之，参加研讨会是一次丰富的经历。你将提高你的公共演讲技巧，思维流动，如何用一种容易理解的方式解释一个困难的话题。如果你从未这样做过，希望这篇文章是一个好的起点，我想鼓励你去尝试去做。之后，收集反馈，下次做得更好。
