# 我如何管理我的项目、文件夹和文件

> 原文：<https://dev.to/cecilelebleu/how-i-manage-my-projects-folders-and-files-38d3>

本周我想分享我的项目文件系统，我是如何管理它的，以及我为什么这样做。我的意图不是说“这是最好的文件系统”或“你应该这样管理你的文件”，因为不同的人需要不同的方法，而且很少有真正的万能方法。也就是说，这是一个适合我的系统，是我在过去 5 年多的时间里开发的。用吧，修改一下，觉得有用就留个赞！

作为一个背景，我在成为开发人员之前是一名设计师，从 2014 年左右开始从事这项工作，当时我开始画字母，学习平面设计和品牌识别设计。这些年来，我一直在开发(因此不得不管理)非常不同类型的项目，包括设计一个关于网络女巫的[精彩纸牌游戏](https://cecilelebleu.com/hexadecimate/)，以及帮助设计一个[日记应用程序界面设计](https://pureformstudio.com/diarly)。

自从我成为一名开发人员以来，我已经修改了我的文件夹系统，以更好地满足开发人员的需求，比如更容易通过命令行和 IDEs、Git 等进行访问。

不多说了，让我们开始吧。我保证——这并不复杂。我们将从上到下，从高层到细节进行处理。*我在 Mac 上，但是这个系统比较基础，可以在任何 OS 或者环境下实现。*

## 我的项目管理文件系统

首先，最重要的文件夹:`allProjects`。这里，你猜对了，都是我的项目。这个文件夹在两个云中不断备份(以防地狱之火淋在一个云上，另一个云上，或者我的本地电脑上)。这是我打开 finder 窗口、命令行和编辑器时看到的主文件夹、个人收藏文件夹和第一个文件夹。

在那里，有三个文件夹:`+Archives`、`+Lab`和`+Study`，接着是大约 10 个项目文件夹。让我们详细看看三个主要文件夹中的每一个。我在名字前放了一个`+`符号，这样他们总是会出现在最前面。

```
v allProjects
|  > +Archives
|  > +Lab
|  > +Study
|  > ActiveProject
   ...
|  > AnotherActiveProject 
```

*   `+Archives`包含所有已完成、暂停或放弃的项目。在里面，你会找到每年的文件夹，从`2014`到`2019`。其中每个都包含一个文件夹，分别对应每个已经成功完成并交付的项目，或者任何被放弃的个人项目，或者最常见的，*暂停，直到我知道怎么做*。
*   包含了我想象中所有疯狂的生物。想法、半成品、开发初期的应用程序以及只画了几个字母的设计。我倾向于将每个想法或创作放在自己的文件夹中，以保持整洁。
*   同样重要。和其他开发者一样，我一直在学习。在这里，我有一个文件夹，里面有我正在上的每个*平台*的名字。我们走吧...[按照](https://dev.to/perborgen)的 [Scrimba](https://scrimba.com) 进行。在我的`Scrimba`文件夹中，我有一个每个课程或路径的记录文件，我在那里做笔记、代码片段、总结等等。如果是一个特别大的球场，我可能会有一个文件夹，里面有多个降价文件。每当我学习的时候，我都会把编辑器和浏览器并排放在全屏上，边走边记笔记。

我提到在三个`+`文件夹旁边还有其他文件夹。这些都是我正在进行的项目。这是我的主要工作领域，我的优先事项。无论是客户项目、与我的合作伙伴的项目，还是我自己的项目——如果我正在做，它就在这里。这是显而易见的，所以我马上就知道我在做什么。当一个来自`+Lab`的小生物长到足够大的时候，我会把它移到这个野生区域。如果这些项目中的一个有一段时间没有更新，如果我对它感到厌倦，或者如果我不先学习其他东西就无法继续，我会把它移到`+Archives`。此外，如果我决定要重新开始工作，我经常把项目从档案室的拿出来，然后放回这个主要区域。

## 单个项目文件夹

大部分时间，我都在我的主要工作区域做项目。但我也在`+Lab`区玩耍，每天花几个小时在`+Study`区。

每个项目文件夹都有自己的想法。没有两个项目是相同的，因此项目文件夹往往会有所不同。如果是开发项目，我通常会在项目文件夹中建立一个 git 存储库。但是对于设计项目来说，git 是一个大禁忌——文件大小往往会大幅增长。有些可以连接 GitHub，但是真的要看项目性质。一些项目文件夹只有一个或两个文件，其他项目文件夹可以有多个文件夹级别和数千兆字节的文件。
