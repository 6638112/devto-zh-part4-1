### 精！

所以你决定学习编码，但是你的情况使得你不可能加入一个新兵训练营或者任何其他的学习平台。您对要学习的最佳语言、库和框架的搜索通常会带来更多的困惑。

嘣！你在网上寻找有用的信息来帮助你学习。

作为一名“初级开发人员”，如果你与行业无关，没有人或机构会感到羞愧。你承认你所有的失败。你不知道如何问正确的问题，也没有人能帮你解决问题。虽然你掌握了正确的信息，但是因为你是这个行业的新手，所以很难使用这些信息。

因此，我认为开发人员生态系统必须设计学习系统，帮助初学者学习最好的方法，通过学习编码成就伟大的事业。

目前，有几个成熟的范例对学习编码很有用，但这对于行业中的职业发展来说很少见。

我认为应该有结构良好的系统，如:

# 职业驱动发展(CDD)

我选择将我将要讨论的过程命名为职业驱动开发(CDD)，以实现与测试驱动开发(TDD)的明显类比。但是 CDD 不是一种编程范式，它是一种学习方法，在这种方法中，学习编码是由

1.  自我推销
2.  好奇心
3.  帮助他人
4.  建筑组合
5.  合作
6.  和网络

作为一个过程，以定位初学者更好的职业生涯中的行业。

学习编码很难，在克服了艰难的学习挑战后，在行业中的职业发展应该不难。

需要一个职业驱动的框架来帮助初学者专注于成为世界级开发人员的重要事情，无论她是在新兵训练营学习还是自学。

我相信这应该像测试驱动开发一样进行，也就是说，提供了一个结构良好的方法——学习->展示->你的技能->重复。
