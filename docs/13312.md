# 当我第一次参加黑客马拉松时发生了什么

> 原文:[https://dev . to/aliyalewis/what-happed-when-when-I-entry-my first hackathon-54ge](https://dev.to/aliyalewis/what-happened-when-i-entered-my-first-hackathon-54ge)

过去的这个周末，我参加了“黑客冻结:黑客马拉松”，这是我第一次参加黑客马拉松！我知道黑客马拉松已经有一段时间了，但直到周五早上，也就是活动信息之夜的 10 个小时前，我才做出承诺。在参加了信息发布会并了解了我的团队成员后，我非常兴奋地看到我们的项目会有怎样的结果，但也非常担心我能为只有几周编码经验的团队做出多少贡献。

星期六的早晨来了，我前一天的兴奋被对这一天的焦虑所取代。在去黑客马拉松的路上，我一直告诉自己:

“即使只是一件事，你也可能有所贡献。”

"即使你不能做出贡献，你仍然会学到一些东西."

“无论结果如何，这都将是一次很好的经历。”

我走进去，融入一点点，和我的队友见面，把我的大脑切换到“代码模式”,试着不去想那种我将成为团队中最弱的一环的感觉。一旦团队就我们的 web 应用程序的概念达成一致，我们很快就遇到了第一个问题——我们中的两个人只懂 Ruby，其他三个人懂 JavaScript。我喜欢我们小组的一点是，我们沟通得很好，并努力利用彼此的技能。我们最终将工作分为前端和后端，我和另一名 Flatiron 学生用 Ruby 构建后端，一人设计我们的徽标，另外两人负责前端。

后端工作对我来说是一次宝贵的经历，因为我刚刚完成了一个为期一周的专注于 Ruby on Rails 的项目。我把我在过去三周里学到的一切都用在了帮助我的团队构建我们的应用程序上。在校外的环境中，和我刚认识的人一起，为一个我从未去过的活动使用我学到的技能，感觉真的很好。当从头开始构建项目时，我对编码充满信心，但这次黑客马拉松向我展示了我有足够的基本技能来做出重大贡献，并且我应该更有信心地参加未来的黑客马拉松。

我感谢我的团队(大喊团队的朋友们！)清晰有效的交流，互相支持，愿意和以前不知道的事情一起工作/学习，以及拥有如此好的态度。他们真的让我的第一次黑客马拉松体验难忘，因为他们，我期待着下一次！

在下面告诉我你的黑客马拉松经历吧！
