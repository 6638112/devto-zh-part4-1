# 开发者的终极游戏

> 原文：<https://dev.to/asim/the-end-game-for-developers-1k5e>

我经常想知道的是，我们是如何陷入软件开发的史前阶段的。从消费者体验的角度来看，技术已经有了很大的进步，但是作为开发者，我们感觉像在以蜗牛的速度前进。

今天的语言基本上没有变化。不断出现的框架提供了与之前的框架相同的经验。我们时常开发新技术，创造“范式转变”,最终让我们倒退 10 年，因为我们必须为新平台再次构建相同的工具。

作为一名开发人员和前系统工程师，我很累。我厌倦了老一套。我厌倦了建造和以前一样的东西。我厌倦了不断地重新发明轮子，因为我们决定使用不同的技术堆栈，或者因为我们想学习 docker、kubernetes、Go 或任何下一个新的炒作的东西。

最终，我觉得软件开发处于停滞状态。世界上只有 2400 万开发者的原因是我们没有进步。我们没有进化。我认为这种情况必须改变。

我认为开发者的最终目标是将每一个拥有移动设备的人都变成开发者。这要求我们设计编程语言和工具的方式发生根本性的改变。这是没人敢采用的基本原则方法。

开发者的最终游戏是什么？让我们知道你的想法。
