什么是代码重用？

代码可重用性是使用现有的代码来构建新的软件应用程序。现有代码可重复用于执行相同的功能，或可重新用于执行类似但略有不同的功能，以提高效率、节约成本、遵守标准并提高整体质量。
web API、web 组件、微服务等技术的可用性假设并允许可重用性。然而，对于 web 或移动应用程序开发人员来说，最好地利用可用的技术，预先设计 web 或移动应用程序的心态是很重要的。

以下是优化代码可重用性的五个考虑因素:

*   选择合适的**前端 web 组件架构**——基于组件的前端架构是 web 开发中的一种新时尚。有反应了吗，棱角分明的 VUE。JS 将这种方法用于其他趋势分析框架。
    [外部链接](https://www.udemy.com/front-end-component-architecture-react-and-angular/)

*   使用**面向对象的方法**——虽然面向对象编程有着悠久的历史，但这种方法和设计方面与使用类似任务的元素来规划 web 或移动应用程序相关。好好讨论一下货币的概念:
    [外部链接](https://www.thesoftwareguild.com/blog/why-object-oriented-programming-is-alive-and-well-in-2019/)

*   使用**模块化逻辑**进行设计——每个代码模块都应该细化为一个单独的职责。这是组件级结构化编程的思想。在完整的应用程序设计中，考虑最小的任务——它仅仅是一个工具或组件，作为一个实体是独立的。关于注意事项的初级读本可以在[这里](https://www.geeksforgeeks.org/modular-approach-in-programming/)找到。

*   使用**代码模板**与复制、粘贴然后修改代码。有开源选项和提供选项的认证产品。模板提供了效率。人们应该考虑:能力；认证；模板结果的所有权/定制。模板从 API 列表生成到具有不同特性和限制的代码生成器各不相同。

*   首先规划**文档**——促进应用程序代码的结构化和模块化方法。关于如何创建优秀文档的概述可以在[这里](https://www.oreilly.com/ideas/the-eight-rules-of-good-documentation)找到。
