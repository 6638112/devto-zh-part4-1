当你想到**风格**这个词时，你会想到什么？你考虑过你的风格吗？大多数人将他们对风格的想法局限于他们身上的衣服或一些设计趋势。也许，如果他们想得更多一点，他们会考虑他们风格的更完整的图片——他们沟通的方式。

# 理解风格

我们的风格是我们选择的向外界反映自我的所有方式的集合。这不仅仅包括我们选择什么颜色或使用什么词语。表达我们的价值观、情感、兴趣、信仰、知识、地位和技能的一切都是风格的累积。

风格很重要。精心设计的风格可以极大地提高你与他人沟通的能力；然而，我们如何沟通可能需要根据环境和受众进行调整。一件白大褂和一个听诊器可能有助于给提供医疗建议的人带来可信度，但对于一个卖拖拉机的人来说，穿着它可能会显得很奇怪。使用俚语和表情符号在漫画中写文章可能不会获得大学奖学金，但在给你的侄子发短信时，这可能是有趣和令人愉快的联系。

## 编码

编写软件在很大程度上是交流技术过程的艺术。软件是用编程语言编写的，这绝非偶然。很多时候，你如何编写代码可能比计算机实际执行的指令更重要。

据说**软件开发主要是读代码，而不是写代码。试图找出一个 bug 或者新特性如何适应现有的代码库，首先要阅读现有的代码。代码越容易理解，这些任务就能越快完成。正因为如此，组织和语言建立了风格指南来遵循。这些指南创建了一致的模式，有助于创建内聚的代码库。**

就像一个人的衣服或打扮一样，代码库的风格可以是代码质量及其创建者能力的一个重要指标。如果你是一名经理或招聘代理人，在查看某人的工作时，你会更信任谁:

人甲

```

let thing2 = p(1, 2, 3, 4, "connect");
console.log(Thing1 + " as compared to " + thing2);
```

读者，也许你在想，“我只是为自己写这些小的个人项目，没有人会看到我的代码。”人们很容易将某些事情视为私人的，并让其质量下降，但个人的小胜利会累积成大的公共胜利。当你写的代码被其他人使用时，你的风格已经从你实现的私有改变中成熟了。

即使你真的只会写一些东西给他们看，你仍然需要理解它。一个例子中的人的功能在那一刻对创造者来说可能是有意义的，但是一年后会立即有意义吗？当然，很容易找到`p()`的定义并理解它的作用，但是这增加了时间，即使只是一点点。如果需要对每个编写的函数都进行这样的处理，那么就需要增加时间来进行修改。预先设计一个清晰简洁的风格会节省很多时间。

## 提高

> "在风格问题上，随波逐流，在原则问题上，坚定不移."托马斯·杰斐逊

当开始一门新的语言时，很难知道什么是好的风格。有一些通用的规则总是适用的，比如一致的大写化或命名良好的函数。像 Python 这样的语言强制使用空白样式，而其他语言则允许你随心所欲地书写。谷歌有一个风格指南库，适用于多种语言。他们中的许多人甚至还带着棉绒。把这些当做你的私人造型师。它们作为一组规则工作，代码编辑器使用这些规则来自动更正任何样式问题。

也有很多优秀的书注重风格。干净的代码是一个很好的资源，特别是对于面向对象的语言。

如果你已经在做一个在代码风格上不太好的项目，那么去修正它永远不会晚。重构是对代码风格进行大规模修复的最佳时机；然而，您编写的下一行代码可以让代码库步入正轨。这可能是最小的错误，但每一个小变化都可能是一个胜利。
