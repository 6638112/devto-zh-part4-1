# 如何申请工作已经使我成为一个更好的开发人员。

> 原文:[https://dev . to/j _ Riordan/how-applying-for-jobs-have-make-me-a-better-developer-56d 8](https://dev.to/j_riordan/how-applying-for-jobs-has-already-made-me-a-better-developer-56d8)

上周的这个时候，我正处于狂热之中。

我终于获得了我的第一次 dev 面试。几乎同时错过了。尽管如此，时间已经确定，公司看起来很棒，期待开始了。

* * *

回到几周前，我最近搬到了蒙特利尔，目的是在技术领域工作，并被充满活力的数字艺术文化所包围。一个应该很容易，另一个就没那么容易了。

我胆怯地开始找工作。知道这需要很长时间。我等了很久才开始申请，部分是因为害怕失败和被拒绝，部分是为了安抚我内心的完美主义者。再过几周算什么？

“我的作品集还没有完成。x 不对。我穿着奇怪的袜子"

你最不应该相信的声音，却变成了最响亮的声音。

谢天谢地，我已经同意了 Coding Cardio 的在线模拟面试和投资组合评估，这成为了我准备好的具体日期。

我不知道你是怎么想的，但是作为一个纯粹的自学者，当我把一些超出我控制范围的事情，而且是别人的时间或期望，这真的给了我一个动力去完成最后期限。

经过几个星期的缓慢进展和最后一刻的匆忙，我做了模拟面试，如果你真的想看，你可以在这里看，得到了积极的反馈。有很多地方需要改进，但是已经有很多优点了。在我看来，这是一个胜利。

* * *

现在没有借口了。开始申请吧！
*但是 x 概念呢？我还是不太懂。”“这个课程看起来很棒。如果我那样做了，我会准备得更充分”*

**号**

什么是准备好了？你总还有别的事可以做。想象一下，如果我们在过马路的时候这样做。我们一生都在看着汽车驶过。(如果你来自一个“乱穿马路”完全正常、斑马线很少出现的国家，这个类比可能更适用。)

对极了。申请被寄出去了，就像没有明天一样。第二天我做了几个，也许第三天也做了一个。动力消失的速度之快令人惊讶。一天过去了，一无所获。二变成三。很快已经一周了。

*“我做错了什么？”"现在应该有人回复了吧？"*

然后最后一封邮件！

成功！

*”。。。我们已经决定不再继续处理你的申请。。. "*

不太成功的成功。

在接下来的几天和几周内，又有几封这样的信来了。直到最后 AngelList 提到一些公司对我“感兴趣”。我很确定这没有什么意义，但是我收到了几封邮件，让我这周打个电话。

电话从未打过。一两封电子邮件化为乌有。

* * *

在这一点上，我不得不停下来钦佩所有的开发者，自学的或者其他的，他们已经做了 3，4，5，甚至 6 个月或者更长的时间，并且仍然坚持不懈。你们都鼓舞了我。

这给了我新的信心，以及来自蒙特利尔当地 freeCodeCamp meetup 的一些很好的建议。我又开始狩猎了。

我也开始钻研更多的后端技术，并主要应用于初创公司。这是为了理想地结束我不得不做一点事情的位置，并希望成为一个更全面的开发人员。

不到一周，我又有了一些兴趣，但什么也没发生。

几周前的一封邮件。首席技术官非常忙，没有时间接受采访。我能在这个星期的某个时候来办公室聊一聊吗？

那还用说！

奇怪的是，两天多来我没有听到他的任何消息。奇怪。

我没有最强的精神决心，虽然这是另一个时间的主题，但我很快就认为他们对一些东西有更好的看法，GitHub，portfolio，LinkedIn，无论什么，并认为我可能不够好。消极情绪增长很快。

我能做的唯一一件事就是继续做项目。

原来我两天前在 AngelList 上更新我的电子邮件时，我没有确认我的新电子邮件。我通常会在平台上收到邮件通知，但现在他们陷入了电子邮件的困境。

他两天前就回复了，他给出的时间是今天*。**两小时前。***

 *可能说了一些脏话。

我立即给他发了电子邮件，解释了我的错误，向他道歉，并说我很乐意在他方便的时候来。

我们把时间定在星期二 10 点。现在是星期四。

***终于，发生了！*T3】**

* * *

所以我们在这里，4 天来学习你知道你不知道的一切。呀！

这永远不会发生，所以我做了每个开发人员最擅长的事情。

我列了一份清单。在一个盒子里。

**1。我开始面试自己，看自己会在哪里跌倒。**

*   在我知识贫乏和知识丰富的地方什么都没有。
*   花时间去克服弱点，强化优点。
*   在我内心深处，我知道我应该做的事情，但却从来没有真正去做。
*   我在网上搜索所有最好的面试问题。[下面是几个](https://developerhandbook.com/react/10-react-interview-questions/)。*这也许是我做过的最好的事情之一。*

这些问题指出了我从未想过要问的事情。

你知道 React 有超过 20 个不同的渲染库吗？一个 webVR 渲染器？画布渲染器？浏览器控制台渲染器？

或者说[片段](https://reactjs.org/docs/fragments.html)是什么？

驱动虚拟 DOM 的[差分算法](https://reactjs.org/docs/reconciliation.html)是如何工作的？

我们到底为什么要写[超(道具)](https://overreacted.io/why-do-we-write-super-props/)？

redux 到底是怎么工作的？我需要它吗？

为什么有这么多样板文件？现在我们有了钩子和上下文，你还需要它吗？他们是做什么的？

这就是我所说的热情。

React 是巨大的。这(显然)仅仅触及了表面。更重要的是，我学得越多，就越兴奋。我理解得越深，我的疑问就越多。

我相信任何一个正派的 React dev 都会知道我提到的大部分事情，如果不是全部的话，还有很多意见。我不想告诉你应该学什么，或者做什么，只是告诉你什么对我有用。

**2。我开始钻研我认为应该“多等一会儿再尝试”的技术。**

真是胡说八道。

React native 在他们的堆栈中，还有一个无服务器的后端。搭建 React 原生 app？不要介意我这样做！

在那四天里，我想我学到了更多关于 React 和 web dev 的知识，然后是过去的四周。也许甚至几个月。

更好的是，我对面试一点也不紧张，我超级兴奋！我迫不及待地想和一个有经验的人谈论我知道的和不知道的一切！

* * *

面试进行得非常顺利，这更像是一次文化契合的谈话，但我和首席技术官相处得非常好，一切看起来都很好。

这让我更加失望，因为尽管有各种建议，我最终还是屈服了，给出了一个期望薪资的数字，但我没有听到一点声音。他在采访中告诉我，他们需要这个来推动进程。我没有别的选择。
他们打算在那之后发起一次技术挑战，但什么也没发生。

然而，这并不是这篇文章的主题。

这种事情总是会发生的。也许招聘过程或现状是扭曲的或错误的，也许不是。还是那句话，下次再说。

我想让你看到的是，这一切促使我更加努力地工作，不断学习。

***以比以往更大的决心。*T3】**

我甚至开始写博客。

* * *

通过实际开始找工作，你会发现所有你不知道的事情。

你看到你的弱点，你知识上的差距。重要的事情变得显而易见。你完成那些次要的项目。

最重要的是，你向职业世界敞开心扉，向那些做你喜欢的事情的人敞开心扉。

一旦你这样做了，你就打破了那扇自你第一次制作《你好，世界！》以来就一直笼罩着你的门出现在浏览器里，都是几个月或几年前的事了。

你永远不会准备好。你已经**是**。*
