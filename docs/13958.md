# 使用一个 CSS 框架应该了解多少 CSS？

> 原文：<https://dev.to/bholmesdev/how-much-css-should-you-know-to-use-a-css-framework-38j3>

我想知道这是我第一次通过 CSS 框架学习 web 开发。我从早期的 [Sass](https://sass-lang.com/) 开始研究引导程序布局，学习正确的覆盖来按照我的意愿弯曲按钮和列布局😈

回过头来看，我想知道这在我建立自己真正想建立的网站的过程中起了多大的作用。我也注意到其他刚接触网络开发的大学生也有类似的困扰。尽管他们有使用 React 的编码能力，但是他们完全不知道如何使用 flexbox 来代替笨拙的引导网格。我已经[给出了我自己的想法](https://dev.to/bholmesdev/let-css-frameworks-empower-you-not-control-you-3hm0)关于 CSS 框架如何可以和普通 CSS 一起使用，而不会觉得被模板所束缚。然而，这需要的经验水平对一些人来说可能是遥不可及的，或者根本没有必要。

所以，我把问题延伸到你身上！在使用一个 CSS 框架之前，理想的 CSS 体验水平是怎样的？这对初学者意味着什么？
