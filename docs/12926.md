大家好，我是意大利的软件工程师 Valerio，也是 Inspector 的创始人，Inspector 是一款开发者工具，现已在产品搜索网站上发布:[https://www.producthunt.com/posts/inspector](https://www.producthunt.com/posts/inspector)

作为产品负责人，我切身体会到应用程序问题不仅仅是技术问题。对于软件驱动的公司，这可能会对用户在入职过程中的体验产生负面影响，或者影响敬业度。

我几乎每天都发布新的代码变更，不幸的是，很难预测每次发布后可能发生的所有问题。此外，用户不会花时间报告错误，他们最终会停止使用我们的应用程序，寻找另一个更适合他们需求的应用程序。

在我参与的大多数项目中，50%的用户缺陷是由简单的代码错误引起的，通常是在后台执行的代码，要知道是一切正常还是出了问题就更加困难了，而且应用程序越长，避免事故就越困难。

感谢 Inspector，我不再需要花费大量时间手动搜索日志中的错误，试图预测用户的投诉，因为一个自主工具正在为我做这项工作，冒泡任何可能给用户带来问题的东西。

我可以信任 Inspector 全天候为我调试应用程序，并实时提醒我任何问题或性能下降，以便我可以立即解决它们。与此同时，我可以把时间花在更有成效的任务上，而不是担心用户会因为应用程序不能按预期工作而来找我。

我构建 inspector 时有一个想法:“简单明了”。在最初的几个月里，我从我的用户(也包括付费用户)那里收到了很多积极的反馈，现在我将与更多的观众分享这个解决方案。

Inspector 以 composer 包的形式发布，您可以像安装任何其他依赖项一样在几秒钟内安装到您的应用程序中。
