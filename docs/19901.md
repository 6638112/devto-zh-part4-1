# 一个朋友兼工程师如何改变我的生活

> 原文:[https://dev . to/jeffpereira/how-a-friend-and-engineer-change-my life-2j5l](https://dev.to/jeffpereira/how-a-friend-and-engineer-changed-my-life-2j5l)

由于这是我写的许多关于工程师生活的文章中的第一篇，我将先简单介绍一下我的文章内容。我将写关于我的软件工程生涯的技术和非技术方面。即使你不是技术人员，我也希望我的一些著述能对你有所启发。无论你是走向世界，成为一名有抱负的软件工程师，还是仅仅让你感觉良好一分钟，我都希望能与我职业生涯中的想法和经历联系起来。

作为自我介绍，我叫杰弗里·佩雷拉。我是一名 29 岁的软件工程师，就职于一家名为 Malwarebytes 的小型安全初创公司。在过去的两年里，我一直在全职编写 Ruby on Rails 应用程序。我既不是初级工程师，也不是高级工程师，我只是一个普通人，尽力写出最好的代码。在最近的一次 Rails Conf 之旅中，我受到启发，开始写我作为一名开发人员的经历，所以这里是我的一点娱乐尝试。除了我的介绍，这第一篇文章将作为我如何进入软件工程的故事。它从我最好的朋友的故事开始。

在你的一生中，总会遇到一些人，他们会让你的生活变得更好。尽可能让这些人靠近你，因为如果他们能做到一次，他们很可能会一次又一次地这样做。就我而言，这个故事开始于我高中三年级的时候，在那里我遇到了约翰，这个家伙后来成了我最好的朋友。我们的友谊并没有马上变成什么，但是有一天他无意中听到我在找工作，告诉我麦当劳在招人，他就在那里工作。在我与约翰共事的几个月里，我们成了更好的朋友。在这一点上，我一点也不知道，这将是我唯一的朋友，将推动我做出选择，这将永远改变我的生活。

快进到我 20 岁出头的时候，我是一名兼职大学生，一个学期上两门课，在百思买商店工作，拿着微薄的薪水走进一份我讨厌的工作。一直以来，John 都是一名 IT 支持人员，比我更有能力。这让我开始意识到我的生活在做错事，我不再把约翰当成朋友，而是一个我想成为的人。最终，约翰在一家小信用机构找到了一份集成工程师的工作，大约一年后，他拍了拍我的肩膀，说是时候了。他为我打开了第一份 IT 工作的大门。他不仅为我打开了大门，我在那个位置学到的一切都是因为约翰是我的导师和教练。在那里工作对我来说是一种全新的体验，我很喜欢，那时我知道在那里工作绝对适合我。最终，John 成为了这家公司的 Rails 开发人员，而我则是一名集成工程师。有几个人从我的部门过来，并成为了一名开发人员，而我则留在了这个部门的内部。

当 John 不仅迅速成为一名优秀的开发人员，而且成为一名伟大的开发人员时，我却在努力将这些碎片重新组合在一起。我不明白为什么他能继续成为这位伟大的 Rails 开发人员，而我却被困在了集成部门。我又一次向约翰寻求建议，这次他对我说的话少了很多，这是一种严厉的爱。他基本上告诉我，发展需要是我想要的东西，我需要自己尝试并弄清楚。这种说法相当坦率地激怒了我。为什么约翰不能把他知道的教给我？为什么 John 不能像我在 IT 行业的入门级工作一样，为我的发展打开大门呢？在思考了所有这些自私的想法后，我意识到这意味着什么。自学如何成为一名程序员是关于奉献、灵感和痛苦的。约翰经历了这个权利，因为他的考验和磨难，他继续成为伟大的人物。这不是你可以插手的事情，你必须自己去经历。在这一点上，我决定像个男人一样，买一些书，参加一些在线课程，并开始构建自己的应用程序。我自己构建的原始应用程序仍然在 Github([https://github.com/jeffpereira/pinteresting](https://github.com/jeffpereira/pinteresting))上，它糟透了。它很难看，有很多错误，而且大部分代码都非常简单，甚至很糟糕。许多人问我为什么不在它上面工作，为什么不把它拿下来，这样人们就看不到它，只是告诉他们它在那里提醒我我来自哪里。

又过了几年，John 现在是一家初创公司的工程团队的团队领导，这家公司看起来即将成为一家大公司，我在 Malwarebytes 找到了我梦寐以求的工作，那里的氛围很好，我周围的人每天都在激励我，帮助我成为一名更好的开发人员。

此时你可能会问自己，这是一个迷人的故事，但为什么它很重要？对此我说，我们都应该努力成为别人的约翰。我们都应该努力激励他人变得更好，或者更努力地工作。总有一些人需要被推向一个方向，在那里他们可以成为一个更好的人，随着世界上更多的约翰，我们作为一个人只会变得更好。我仍然每天与约翰交谈，他仍然是我灵感的来源，也是我渴望成为的人的来源，我仍然有那个项目向我展示我已经走了多远。
