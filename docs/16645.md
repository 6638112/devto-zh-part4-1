# 关于公司内部无障碍的思考

> 原文:[https://dev . to/just sharkie/musings-on-accessibility-within-companies-4939](https://dev.to/justsharkie/musings-on-accessibility-within-companies-4939)

我找工作已经有一段时间了。开始觉得时间长得令人恶心。

在这段时间里，我开始注意到一些事情。其中之一就是我的一些本地公司似乎并没有真正理解可访问性，以及它对网站开发和设计的重要性。

# [](#note)注

我不会对任何公司指指点点，或暗示任何事情。这只是我通过自己的研究、工作面试等注意到的一些一般性的东西。

我也不是无障碍专家！我确实相信这是非常重要的，并且总是努力去提倡它，但是我肯定不知道所有的事情，我也不会假装知道。

此外，这是基于 [WCAG 指南](https://www.w3.org/TR/WCAG20/)，因为这些是我非常熟悉的，也是我最喜欢的。

# [](#keyboard-navigation)键盘导航

我找工作/申请过程的一部分是浏览公司网站。这让我对公司的核心价值有了一种感觉，我可以深入研究代码，看看后台发生了什么。

大多数情况下，我都是在笔记本电脑上做这种事情。我也是一个懒婆娘，只要有可能就用键盘浏览网站。我浏览过的网站 100%都不能用键盘导航。总有一个点会断裂，我不能再移动了，或者不能够到我想按的按钮。没有一个是可以通过键盘操作的！我不断震惊于有多少公司网站不关注这一点，或者只是认为这无关紧要。通常它可以通过简单地正确构建您的 HTML 来解决，显然在开发过程中没有人考虑它。

这可能不是最大的问题，但我相信这应该是一个问题，是你在编写网站代码时要考虑的问题。实现起来真的没那么难。

[链接到键盘 WCAG 法则。](https://www.w3.org/TR/WCAG20/#keyboard-operation)

# [](#colour-contrasting)颜色对比鲜明

哦。我的。上帝啊。为什么没有人考虑色彩对比？

澄清一下，我不是色盲。但是我总是希望有足够的对比度，这样任何人都可以清楚地看到一些东西。

显然，没有多少公司在寻找这样的机会。

我参加了一个面试，有几分钟时间看一个原型网站，并给出我对设计的意见。有*多个*部分，在中灰色背景上有深灰色图像。同样的配色方案在巨大的页脚与微小的文本链接。有醒目的红底白字。所有这些对我来说都很难读懂。一些链接我不得不问他们说什么！想象一下对于一个视力有问题的人来说会是什么样子。或者视力下降的人。

他们不可能阅读网站 90%的内容。

在这一点上，有很多网站可以让你检查你的颜色是否符合可访问性指南，是否有足够的对比度。请利用它们。他们是来帮忙的。

[链接到色彩对比 WCAG 法则。](https://www.w3.org/TR/WCAG20/#visual-audio-contrast)

# [](#no-javascript-no-problem)没有 JavaScript？没问题？

另一个 WCAG 指南指出，你的内容需要以不同的方式显示(例如，更简单，没有 JavaScript 等)。)

我见过的大量网站充斥着 JavaScript，没有它基本上无法阅读...有点可笑。考虑到不同 web 浏览器中的所有小的复杂性，以及大量不使用 JavaScript 浏览或在糟糕的互联网上浏览的人，您可能会认为您应该考虑使您的站点可读性更好，并且至少具有适度的功能，而不需要所有的花里胡哨。同样，我认为这是一个人们没有意识到这应该是一个问题的例子，也是我们在编写站点代码时考虑的事情。

[链接到内容 WCAG 法则。](https://www.w3.org/TR/WCAG20/#content-structure-separation)

# [](#time-crunch)时间紧缩

慢点。下去。你的。滑块。

我注意到网上到处都是这个。人们似乎认为幻灯片需要快速移动，这样人们才能在向下滚动之前看到所有内容。但是，它移动得太快了，在移动到下一个内容之前，根本没有时间去阅读它！我学到的一个技巧——你能在内容转换前读两遍吗？很好，现在再加一两秒钟。作为设计者/开发者，你已经习惯于阅读这些内容，以至于你可以比从来没有看过的人至少快一倍。

这不完全符合时间限制规则，但我也喜欢把它放在那里。它阻碍了人们阅读你的网站，他们实际上得不到他们需要的信息，因为事情发展得太快了。

虽然我也会对一些工作申请的时间限制发表意见。我是一个快速写作的人，通常可以在 10 分钟内写好，但对于一个人来说，回答 5 个问题并填写每个字段的时间太短了。加倍时间。友善点。

[链接到时间限制 WCAG 法则。](https://www.w3.org/TR/WCAG20/#time-limits)

# [](#mobile)移动

这实际上并不符合任何规则，但我无论如何都需要将它包含在内。

可能超过一半的访问你网站的用户会在手机上浏览。那是很多人。这显然会因用户群的不同而不同。

请，请让您的网站响应。

通常，我会在笔记本电脑上浏览潜在的公司网站。但有时，我会在手机上查找。有时我会测试它们，看看它们是否有反应。至少有一半的时间，他们没有。通常，是滑块、视频和大的复杂的文本部分被破坏了。最终，它们被破坏得无法在我的手机上浏览。

在这个时代，我真诚地认为这是一个悲剧。但是，嘿，至少它为潜在的面试带来了一个很好的话题，让我展示一个我对公司非常有价值的地方。

# [](#conclusion)结论

不是每个公司都这样，也不是所有的公司都在我上面列出的事情上失败。可能是因为我喜欢申请较小的公司，这些公司可能没有最大的网络开发预算。可能这就是他们想把我这样的人带进公司的原因！

我只是发现有趣的是，我经常注意到事情不符合/不像它们应该的那样工作。这是我脑子里根深蒂固的东西，所以看到那些甚至不去想它的人有点让我震惊。

就这样吧！当你编写自己的网站时，请记住所有这些事情，它们真的不需要花太多时间来实现，并且对一些人来说是无价的。
