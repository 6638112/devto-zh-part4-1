# 重温 2019 年多伦多 SmashingConf 的第一天

> 原文:[https://dev . to/solace devs/recaping-day-1-of-smashingconf-Toronto-2019-40cb](https://dev.to/solacedevs/recapping-day-1-of-smashingconf-toronto-2019-40cb)

我很高兴今年能参加多伦多的 SmashingConf。如果你想了解前端和 UX 开发，SmashingConf 是个不错的地方。我和我的同事布莱恩·赫西非常兴奋，我们早上 8 点就出现了，焦急地等待开门。

我在左边，布莱恩在右边

第一天，七位演讲者发表了引人注目的演讲，涉及从动画和前端代码架构到包容性设计和数字政策的方方面面。我在这里总结了他们谈论的内容，并在媒体上写了更多。

*   **让我们建立一个设计系统:**[Brad Frost](http://bradfrost.com/),*Atomic Design*一书的作者，同时也是 Style Guides 播客的主持人之一，他首先讲述了如何建立一个设计系统。他分享了一些关于代码架构、模块化和组件重用的伟大想法；用一对名为 [io](https://patternlab.io/) 和 [Story UI](https://storybook.js.org/) 的前端开发工具演示关键概念。在我最近重新设计[Solace 文档网站的经历中，通过我们使用的软件来设计和编码是至关重要的，这也包括我们想要集成的](https://dev.to/solacedevs/re-designing-documentation-website-from-planning-to-launch-ba9-temp-slug-5286738)[第三方软件](https://dev.to/solacedevs/integrating-the-algolia-docsearch-into-madcap-flare-based-customer-documentation-32l1-temp-slug-7562892)。
*   **让我们从头开始构建一个 Vue 应用:**然后我们听取了[萨拉·德拉斯纳](https://sarahdrasnerdesign.com/)的发言，她是 Netlify 的开发者体验主管，也是 [Vue.js](https://vuejs.org/) 团队的成员，Vue.js 是一个用于构建用户界面和单页应用的开源 javascript 库。她向我们展示了如何使用 Vue 构建一个功能性的应用程序，包括路由、服务器端渲染和使用 Nuxt.js 和她的 Vue 片段扩展的动画。我对她如何使用自己开发的自定义 Vue 片段和 CSS Grid generator 等其他工具来加快工作流程很感兴趣。非常酷！
*   **JAMStack:愚蠢的名字，严肃的东西:**然后[菲尔·霍克斯沃斯](https://twitter.com/philhawksworth)， [Netlify](https://netlify.com/) 的开发者关系主管，澄清了 [JAMStack](https://jamstack.org/) 的意思(Javascript+API+预渲染标记)，并演示了如何使用 WebHooks 和 API 来触发事件、生成通知以及创建和自动化开发环境。他的关键信息是，您应该简化技术堆栈，将站点从部署环境中分离出来，并尽可能多地实现自动化。
*   **搭建桥梁，而不是围墙:**经验丰富的 UX 顾问 [Jenny Shen](https://jennyshen.com/) 做了一个精彩的演讲，讲述了文化差异如何影响产品和界面设计。她明确表示，仅仅翻译页面上的文字已经不够了——为了增加信任和参与，设计师必须理解并考虑到各国文化、习惯和其他社会差异的[。](https://www.hofstede-insights.com/country-comparison/)
*   **让我们从头开始创建一个网页动画:**插画师和动画师[克里斯·甘农](https://gannon.tv/)向我们展示了用 SVG 代码创建动画的过程。看到他在编码时所做的决定，以及代码中即使是最小的细节也能对输出产生重大影响，这是非常有趣的。
*   **数字政策和标准康复时间:**[《T4》一书的作者 Kristina Podnar](https://www.kpodnar.com/) 解释了数字政策和标准这个经常被忽视的问题是如何在创建一个好网站的过程中发挥重要作用的。她谈到了一些常见的挑战，以及如何使用政策和标准来增强网站的健康。
*   **数字设计的真实性:**最后但同样重要的是，[史蒂文·胡伯](https://www.linkedin.com/in/shoobe01/)(他写了一本关于移动设计的书，即奥莱利的[设计移动界面](https://www.amazon.com/gp/product/1449394639))详细介绍了为移动设备设计时面临的独特的可用性挑战，例如人们不是从移动屏幕的顶部开始，而是从中间开始。

这是一个信息丰富的一天，如果你和我一样觉得这些话题很有趣，我想你会从我对他们在媒体上的演讲[的详细描述中受益匪浅。](https://medium.com/@arshrai.ca/day-1-smashingconf-toronto-2019-97596bddc69e)
