有时候，完成工作最难的部分并不是最具挑战性的部分。是无聊的部分。你知道的。写下拉取请求或导航到票据等。有时候很难摆脱这些东西。如果可以，那太好了。如果不能，还有[阿尔弗雷德](https://www.alfredapp.com)。

在使用了 iOS 上的快捷方式，并发现我喜欢制作工作流程来自动化我繁忙的工作之后，我决定为 macOS 寻找一个类似的工具。macOS 上的 Automator 是一个很好的开端，但缺乏快捷方式的易用性。多年来，我深信阿尔弗雷德是一个适度改善聚光灯搜索，因此不值得的钱。

## [](#first-servings)🤵第一份

在安装了 Alfred 并开始配置之后，我知道我错了。Spotlight 在搜索文件方面做得不错，而 Alfred 是一个执行多种不同类型操作的工具，其中一种恰好是超快速的文件搜索。Alfred 是高度可配置的，浏览其令人疲惫的偏好面板有助于您了解可用操作的数量。

以下是我最喜欢的一些。这些是默认内置的，我几乎每天都在使用它们:

*剪贴板历史:*如果你曾经不得不同时处理复制和粘贴两个东西，这很好地解决了这个问题。在输入关键字后(默认为`clipboard`，我把我的换成了`pb`，或者按键盘快捷键(`Alt + Command + C`，你会得到一个你最近复制的东西的列表。`Command + 3`粘贴最近复制的第二个的*。Alfred 甚至很聪明，不会将 1Password 或 Keychain 中的密码存储在剪贴板历史记录中。*

*片段:*`Command + 3`退回你第二份拷贝的原因是因为片段。它们是第一个结果。如果你曾经在填写具体细节之前不得不复制粘贴一个模板，你就知道再做一遍有多烦了。片段是解决这个问题的方法。您可以将模板粘贴到代码片段中，然后使用关键字`snip`进行搜索。这方面的一个例子是使用“合并请求”模板。我可以打开阿尔弗雷德`Command + Space`，然后输入`snip merge`并按回车键。该字段将被填充。

*网络搜索:*默认情况下，Alfred 会搜索许多热门网站设置。你可以轻松搜索`IMDb Avengers`找到更多关于《复仇者联盟》这部电影的信息。T4 的好处在于你可以添加自己的自定义搜索。例如，我让我的工作 Wiki 设置使用`wiki issue login fails on error`来搜索包含该文本的任何问题。您可以使它更加强大，但是我们将在下面进行讨论。

想要添加新的网络搜索吗？只要去你想搜索的网站，输入`{query}`并进行搜索。用`{query}`替换结果页面上 URL 的`%7Bquery%7D`部分，然后将该 URL 作为新条目添加到搜索列表中。这里有一个例子，我在大约一分钟内添加了一个 Python 搜索:

*网络书签:*类似于网络搜索，Alfred 可以搜索并打开你的书签。更好的是，它可以在书签所在的浏览器中打开它们。如果你处理的是十年前就该被清除的糟糕的内网网站，这就太神奇了。快速搜索(或者如果你设置了关键字，后面跟一个)`þe olde SVN`会打开一个 Chrome 的实例。不幸的是，书签搜索只能在 Safari 和 Chrome 中使用。你可以用 Firefox 打开网页，但是我们将在下面讨论。(悬念杀了你了吗？)

*其余:*计算器、字典、通讯录都比 Spotlight 有所提升。Alfred 还包括控制 iTunes、整个 macOS 系统的操作，甚至可以直接运行终端(Terminal.app、iTerm *或* Hyper，理论上更多)命令，打开一个新窗口并显示结果。很漂亮，是吧？让我们来看看最酷的部分:工作流。

## [](#%F0%9F%8F%97-workflows)🏗工作流程

到目前为止，一切都很顺利。然而，工作流程将这款应用从美好的必需品变成了提高生产力的药物。顾名思义，工作流是一种尽可能容易地执行一个或多个操作的方式。

有五个部分:触发器、输入、动作、实用程序和输出。触发器和输入在移交给动作或实用程序(也可能触发其他动作或实用程序)之前启动工作流，最后以可选输出结束。

这似乎很简单。那么，为什么要大惊小怪呢？这五个部分都提供了许多不同的选项。

🚦*触发器*可以是热键、其他 Alfred 功能或外部触发器(这通常意味着使用 AppleScript。期待在另一篇文章中看到更多！).想想点击`Command + Alt + V`或者运行 Bash 命令。这些甚至不需要你*扣动扳机。您可以使用类似于`fswatch`的工具在文件更新时运行它。*

⛽️ *输入*是基于关键字的动作，可能有一个查询。当您尝试使用类似于聚光灯的阿尔弗雷德时，请使用这些。“关键字”适用于简单的操作，如输入`keyword [query...]`。例如，我设置了一个`git jakew/dotfiles`来打开 github.com/jakew/dotfiles,，这使得进入 Github 上的项目变得非常容易。

输入的另一个重要选项是脚本过滤器。这个允许你运行一个脚本并直接在 Alfred 窗口返回结果。它可以运行 Python、PHP、Ruby、Perl、AppleScript 甚至只是一个 Bash 函数。以正确的格式返回 JSON 或 XML 允许 Alfred 显示定制的结果列表。例如，我有一个 Python 包装器，它返回我在组织的问题跟踪器中拥有的所有票据。通过键入`issues,`，列表被填充，并且选择一个选项在我的浏览器中打开问题。(不幸的是，我的问题工作流程使用专有软件，所以我不能发布它。😫你要知道这会让你大吃一惊。)

🏃‍♀️ *动作*通常是工作流中*做*的部分。例如，他们可能会打开特定的应用程序、文件或 URL，或者运行系统命令。出于自动化目的，操作也可以运行 AppleScript 脚本。您也可以将它与使用`x-callback-url` s 的应用程序结合起来(这本身是另一篇文章的主题。)

🧰 *工具*是工作流的逻辑。如果查询是这样，那么就这样做。分裂论点。把 X 转换成 y，诸如此类。一个额外的实用程序是“Debug”，它向您显示工作流对象中的当前值。

📺最后，*输出*是您从工作流中获取内容或通知的方式。它可以像 macOS 通知或声音一样简单，也可以自己触发组合键。输出还可以在光标当前等待的地方写入文本，或者将其结果放入剪贴板，以供粘贴。更疯狂的是:一个输出也可能调用一个“外部触发器”！还记得我们上面提到的那些吗？是的，你可以让工作流调用工作流。😱

## [](#sharing-is-caring-sharing-automation-is-doubly-so)❤️的分享是关爱；共享自动化更是如此。

“好吧，你已经说服我了，”你可能会想，“但是我想坐下来制定所有这些工作流程吗？”简单的回答是:你不需要。一大群人已经做了很多，并渴望分享！你可以加入阿尔弗雷德论坛，甚至分享你的工作流程。

你也可以从阿尔弗雷德的博客中获得工作流程，甚至在未来关于有效开发的文章中。这是我已经做过的一个例子:

*   *粘贴为代码块*:当“粘贴”时，你的剪贴板的内容用三个重音符( ````` )包围粘贴，创建一个代码块。像这样粘贴对 Github/Gitlab 上的 Markdown 和 Slack 都有帮助。

如果这篇文章说服了你去获得 Alfred，或者如果你已经有了它，并且你已经做了一些你自己的工作流想要分享，我很乐意在下面的评论中看到它们。

## [](#conclusion)🏁结论

阿尔弗雷德提高了我的工作效率。只要我不把一整天都花在工作流程上而不是工作上。阿尔弗雷德可以做很多事情，所以你不必做。它的社区充满活力，提供了大量的资源，你应该去看看。当然，你的工作流程是你自己的，所以开始尝试吧。
