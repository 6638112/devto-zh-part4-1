# Youtube 上为开发者提供的最新频道

> 原文:[https://dev . to/Accel 33/the-most-updated-channels-for-developers-on-YouTube-27e 8](https://dev.to/accel33/the-most-updated-channels-for-developers-on-youtube-27e8)

这是我在 Dev.to 上的第一个帖子，所以可能不会很好，我会尽力而为。无论如何，我的帖子是要分享哪些最好的 YouTube 频道，我认为这些频道可能对新手和希望保持软件开发趋势不断发展的老手有最大的影响。

我自己最近才开始更新自己，我想说大约六个月前，试图跟上技术的趋势，如 Vue.js，MongoDB，Node.js，我的情况是 Firebase。但是仍然很难跟上时代的步伐，尤其是所有这些东西开始形成势头，像 PWA、Typescript、Svelte、Components、Hooks、反应式编程这样的技术。

到目前为止，我一直在尝试一些技术，其中之一就是每天都有一个关于这些技术的教程，并上传到 GitHub 上。尽管有时我的大脑是模糊的，不能理解它的一些部分，我只是浏览并完成它，假设一点一点地，小块的编码语法会被理解。

为此，我将每天的 nano 项目建立在一个视频的基础上，我每天选择的视频在我即将提到的频道之间有所不同。直奔主题，这是我绝对推荐的 YouTube 频道。这是不断上传新内容的频道，有非常清晰和简洁的方式来解释编码概念，并跟上最新的趋势。

* * *

# [](#traversy-media-by-brad-traversy)旅行媒体。作者布拉德·特拉弗斯

[![](../Images/429a24ff0632fba57deeba903558e3e7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gqCjxyna--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qd04n2wwbbamdmyiysw1.png)

我认为 Brad 的频道是我个人最喜欢的，尽管有些人告诉我，Brad 不再费心去深入解释他的问题，比如“为什么要实现这个或者这个有什么作用”。但偏偏他们为了 begginer 的教程把更高级的视频搞混了。根据我的经验，他非常完整，他解释事情的方式也很完美。对我来说，这可以归结为布拉德如何通过他的视频让你感到非常舒服。我喜欢在学习代码时有这种感觉。

* * *

# [](#academind-by-maximilian-schwarzm%C3%BCller)Academind。马克西米连·施瓦茨米勒

[![](../Images/7a6a39b0d7c02a4429921e96c659bcdd.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--boGv6WGX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qfpxk56k52hvso5oo6d7.png)

马克西米利安在教学方面绝对是个天才，到目前为止，我发现他的 YouTube 视频，尤其是他的付费课程超出了我的预期。这家伙深入基础，让未来的开发者理解每个概念。最高水平的技术知识来自马克西米利安。

* * *

# [](#the-net-ninja-by-shaun-pelling)网络忍者。作者:肖恩·佩林

[![](../Images/a634c7787271283b558bbdac7302cfb7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QIYBUh7A--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/jk7wveye5qszkec79u2x.png)

Shaun 是一个了不起的、非常有经验的编码导师，你不禁会觉得有一群自己的朋友在整个学习过程中指导着你。它两全其美，在整个学习过程中具有良好的友好感和良好的超前知识。

* * *

基于这一点，你可以开始朝着你的目标迈出一些额外的步伐。我希望这能进一步帮助其他开发人员。真的迈出了这一步，这是一种解脱。我希望在不久的将来我能找到更多的方法来帮助开发者社区，我知道我非常感谢你们每一个人每天所做的贡献。快乐编码。
