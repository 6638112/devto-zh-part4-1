# 如何教一个非技术人员测试一个 app

> 原文：<https://dev.to/drbragg/how-to-teach-a-non-tech-person-to-test-an-app-4f5n>

我的公司每年夏天都会雇佣一些实习生。有时他们受雇于与他们专业相关的特定部门。其他时候，他们轮换部门，在需要的地方提供帮助。其中一名实习生最近被分配到我的部门，帮助我“测试”一个我们即将发布的网络应用。这太棒了，但我不知道如何教一个科技经验有限的大学生如何“测试”一个网络应用。

你会如何教别人测试你的 web 应用程序？

你会给他们什么样的指点或指示？

当测试人员不是开发人员时，你会要求什么样的错误报告？

到目前为止，我已经要求她:

*   像普通用户一样浏览应用程序。
*   点击一切！
*   尝试使用不同尺寸的浏览器，包括你的手机。
*   如果事情没有如你所愿，请告诉我。
*   如果有什么不对劲，请告诉我。
