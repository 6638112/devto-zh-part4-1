# 在 JavaScript 中使用 CLIs 生成项目是个好主意吗？

> 原文:[https://dev . to/leoat 12/is-it-a-a-good-idea-to-use-clis-to-generate-projects-in-JavaScript-4en 9](https://dev.to/leoat12/is-it-a-good-idea-to-use-clis-to-generate-projects-in-javascript-4en9)

当我使用 Angular CLI、React Native CLI 或 VueJs 生成项目时，总是有些麻烦，因为有时他们在生成项目之前会问一些问题，作为初学者，你不理解他们所问的一半内容，而且当项目生成时，有许多文件你不知道他们做了什么。

我不是 Javascript 的完全初学者，使用 jQuery 多年，甚至在一些个人项目中使用 Angular 2+。我当然比所有配置文件的含义更了解 Angular 基础知识，比如组件、模块、服务、依赖注入、双向绑定等等。

在一个项目的开始，所有的事情都是必要的吗？对于一个想要快速开始的真正初学者来说，这不是他们需要克服的又一堵墙吗？
