# “完美”的编辑？

> 原文:[https://dev.to/firatcansucu/the-perfect-editor-326a](https://dev.to/firatcansucu/the-perfect-editor-326a)

我在 Atom、Sublime Text 和 Code 之间跳来跳去；甚至试过 emacs 和 vim。还是没找到那个‘真命天子’。原因如下。

1-速度
2-可扩展
3-维护者
4-学习曲线
5-跨平台
6-存在和开源项目(更喜欢自由软件，但不要超越我们自己)

##### [](#im-not-qualified-enough-to-comment-on-the-emacs-vim-so-im-going-to-consider-them-as-one-here)我没有足够的资格来评论 emacs / vim，所以我在这里就把它们当做一个。

## [](#1-the-speed)1。速度

#### [](#emacs-vim-gt-notepad-sublime-gt-vs-code-gt-atom)Emacs / vim >记事本++ /崇高> VS 代码>原子

程序的速度取决于编写它的技术。那些使用 JS 框架的，比如 Electron，往往比其他的要慢很多。好吧，对于开发者来说，使用 Electron 来开发一个跨平台的应用程序可能会容易得多，但是你这样做会降低速度(至少到目前为止是这样)。

## [](#2-expendable)2。消耗品

你可以用 elisp 做 Emacs，用 Python 做 Sublime，用 Typescript 做 VS 代码等等。如果你愿意的话，我目前使用的所有编辑器都是可以牺牲的。他们依靠他们背后的社区来使其扩展。我记得过去围绕崇高文本的大肆宣传，“用 Python 扩展，太酷了！”。现在大炒作的是 VS 代码。他们可能都有自己的局限性，但最终，他们的表现取决于他们背后的社区。所以，它也必须“受欢迎”。

另外值得一提的是: [emacs](https://www.reddit.com/r/emacs/comments/7b6erv/weird_uses_of_emacs/)

## [](#3-the-maintainers)3。维护者

你可以说我多疑，但我想拥有一个不受大公司控制的编辑器(你被排除在 Mozilla 之外，爱你)。
这就是我选择的原因；

学习 vue.js over React
学习 Python over GO
使用 Firefox over Chromium

“一”必须由个人来维护，当[时刻到来](https://lists.gnu.org/archive/html/emacs-devel/2008-02/msg02140.html)时，他们愿意把它交给其他人。而不是像[这个](https://www.binpress.com/vim-creator-bram-moolenaar-interview/)(问题 8)。
