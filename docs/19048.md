# 带头！

> 原文:[https://dev.to/sandordargo/take-the-lead-4b8d](https://dev.to/sandordargo/take-the-lead-4b8d)

二月中旬，我的一位同事 Bob 在聊天中告诉我，他还没有拿到年度绩效评估。截止到一月底。他很惊讶，我居然完成了它。那是因为我对我的一对一和评论很有进取心。在我去度长假之前，我确保我的经理已经安排好了。当不得不改期的时候，我会处理的。

我告诉鲍勃，他应该查看我们经理的日程表并发出邀请。

他不会生气吗？-他问。

一点也不！

我的理论是，只要你不责备他们，或者你不提出你不得不代替他们做一些事情，大多数经理都不会被冒犯。

经理工作很多。也许在某些情况下，他们的附加值值得怀疑，他们可能是[](https://twitter.com/iamdevloper/status/1094919960030208000?lang=en)*，但他们至少花了很多时间让自己忙碌起来。*

 *你和他们的优先权几乎总是不同的。句号。

这里没有惊喜！如果你的同事突然出现在你的办公桌前，问你一些事情，而你对此并不高兴，很可能你们的优先事项不同。您想要为您正在处理的项目交付一个新的特性，而他正试图调查一些与您根本不关心的应用程序监控相关的问题——至少那个星期不关心。你的重点不同。

对你的经理来说就有点微妙了。他需要完成你的绩效评估。他需要签署你的训练计划。组织和准备并不是他的首要任务。

但是你可以帮助她！

查看她的日历并安排会议！不要忘记包括议程！你还记得吗？没有议程，没有会议！

她很可能会幸福。你刚刚从她的清单上勾掉了几项任务。

当我还是一名大学生时，我想获得奖学金，这样我就可以在欧洲议会实习一个月。我需要几封推荐信，其中一封来自我的大学。我想了一会儿，决定问问我们的系主任。他给了我一个快速的回复，表示他对此没有问题，但我应该准备好这封信，这样他就只需要签名了。

我很惊讶。

那是第一次，但不是最后一次，我不得不自己写推荐信。

这其实很好，因为你知道你为什么需要那封信，你知道你想在信里写些什么。你在控制，你开车！

如果你能对你的培训计划、你的绩效评估做同样的事情，不要把它视为一个障碍。这是应该走的路！如果你能提出建议，你以后可能会得到更好的评价，你可能会参加更好的培训。

写下你的评估，写下你的计划，准备好你能准备的一切！发送邀请时不要忘记分享它！你的经理会更高兴，因为她不用准备那么多了。当然，不要期望一切都照原样接受，但是你设定了基调，你设定了基线。

你也会看起来像一个积极主动的人。你是。双赢！

如果有一些不需要开会就能解决的话题，试着去做。提前发送你的想法会议应该被安排好，也许你会收到一个简单的回复。*我喜欢！批准！*我的训练计划发生了这种事。我们过去常常在年度目标设定结束时讨论这个问题，这通常为时已晚，无法在最好的培训中获得一席之地。所以几年前我改变了策略。我在年初发送我的列表。一份关于我想参加什么培训以及为什么要参加培训的结构良好的请求。我的经理通常很满意，我们只做了几处小改动。今年，我们甚至不需要开会。他对此非常满意，并且没有亲自见面就批准了。给他少干半小时或一小时的活！

## [](#conclusion)结论

对你的经理积极主动。他们中的大多数人会喜欢它。他们会理解你不是额外的负担，相反，你减轻了他们的负担。当然，你了解你的经理，如果她不是那样，你应该能够很快理解，不要强迫她。在这种情况下该怎么做是以后的事了。

如果你的经理喜欢你积极主动，好好利用它！准备您的培训计划、一对一培训的主题、绩效评估的材料。与让你的经理设定基线相比，如果你设定了基线，你就有更好的机会得到你想要的东西。

如何准备绩效评估是本系列下一篇文章的主题。
