### [](#kibana-as-a-data-visualization-engine)**作为数据可视化引擎的 Kibana**

**基巴纳**的特点和能力是*数不清的*。但是 Kibana 可以被描述为一个访问 **Elasticsearch** 数据并理解它的强大接口。

在大多数情况下，Elasticsearch 被用作主要的数据存储。但是 Kibana 可以在 ***一个集中的地方*** 收集多样的数据，并提供各种引人注目的 ***可视化*** 来分析这些数据。

随后，你可以将所有创建的可视化效果集中在一个单独的**仪表盘**上——这是 Kibana 最显著的**特性。这些仪表板的主要目的是实时了解您的数据，高效地交流信息并在团队内部共享。**

此外，由于 Elasticsearch 是 Elastic Stack 的核心，处理数据应该是快速且非常简单的。一般来说，弹性堆栈的所有产品在配置所花费的开发时间方面都是有效的。

### [](#popular-use-cases)**流行用例**

Elasticsearch 和 Kibana 联合工作的最常见用例是*日志分析*，报告 KPI，收集服务器指标，安全分析，各种*商业智能任务*等等。

Kibana 广泛应用于众多行业:金融、营销、教育、能源和公共服务、政府、医疗保健、制造、商业、媒体和娱乐、软件和技术、电信、旅游等等。

世界上最大的公司使用 Kibana 并从中受益。这不是没有原因的。[数百个用例](https://www.elastic.co/use-cases/?r=dt4)证明了 Kibana 和 Elasticsearch 使组织能够做出数据驱动的决策。

### [](#opensource-nature-of-kibana)**开源性质的基巴纳**

[**Kibana**](https://www.elastic.co/products/kibana/?r=dt4) 是一款**开源产品**，这个事实有利也有弊。一个巨大的优势是产品在不断发展，变得越来越好。周围的**社区**确实是*生机勃勃*活跃*活跃*。许多开发人员正致力于开发 Kibana 作为分析和可视化产品的附加功能。一个缺点是，这样的开发使得开发者很难跟上每一个新的重大更新，并使他们的产品与最新版本的 Kibana 兼容。

### [](#plugins)**插件**

在我测试过的插件中，有一个与其他插件明显不同。我将介绍它最值得注意的特性，这些特性可能会改进您的数据驱动策略。

### [](#reporting-tool-for-elasticsearch)**弹性搜索的报告工具**

[**Flexmonster**](https://www.flexmonster.com/?r=dt4) 实现了很多人渴望在现有的 Kibana 小部件中看到的特性，即数据**旋转**。

当然，有很多方法可以在图表、图形和地图中可视化您的数据，但您并不总是需要这些类型的演示。有时，在发送到图表之前，您需要对数据进行切片并以交互方式进行探索。

这个插件的主要目的是让你能够从隐藏在 Elasticsearch 文档中的数据中提取洞察力，通过在一个网格上的摘要视图中显示它。

此外，由于其设计，它看起来像一个原生模块。

是什么让这个插件吸引了**分析师**？

*   通过用户界面提供所有主要的**报告功能**。我特别想提到用户体验——当使用这个工具时，一切都显得直观而清晰。

*   能够以符合您需求的方式**个性化**组件。

*   **互动性**。在字段列表和网格中，您都可以拖放字段来更改要在报表中显示的数据部分。每次你这样做，你可以看到一个完全不同的数据图片。

*   选择**聚合函数**。

*   **排序**。只需点击几下鼠标，就可以对网格上的值和层次结构的成员进行排序。

*   **过滤**。可以按成员和值过滤数据。

**数据透视表**。组件有两种模式——图表和网格。您可以通过工具栏在它们之间切换。这很方便，因为您可以在短时间内在多个视图中呈现相同的数据。

而**开发者**可能会对什么感兴趣呢？

*   轻松连接到可用的 Elasticsearch 索引，无需浪费时间进行长时间设置。

*   能够保存所有结果。报告可以存储为 JSON 文件，稍后通过 UI 上传到组件中。

*   能够通过用户界面导出报告。最流行的格式都有: **PDF** ， **Excel** ， **CSV** ， **HTML** ，图片。

这只是对组件特性的表面分析，从我的角度来看，乍一看很有希望。
