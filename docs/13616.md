# 云平台变得越来越复杂

> 原文:[https://dev.to/alekssk/the-zero-clicks-revolution-1ffg](https://dev.to/alekssk/the-zero-clicks-revolution-1ffg)

如今，云提供商在不断争夺前端开发人员的注意力。酷的新技术几乎每天都在涌现。通常，这些技术可以重新打包，并添加到平台不断增长的软件即服务选项列表中。

有些人不赞成这种趋势，因为看起来好像提供者正在从开源社区的免费劳动中赚钱。虽然这是一个有效的观点，但这不是本文的主题。

我想提请大家注意大多数知名云平台的 SaaS 投资组合不断增长的另一个副作用。

复杂。

毫不奇怪，增加的选项越多，平台就变得越混乱，越难导航。然而，开发这些平台的人和那些从一开始就已经参与其中的人很难注意到这一点。在任何给定的时刻，这些人完全熟悉所有现有的，比如说 30 个选项，对他们来说，新的第 31 个选项在拼图中的位置是显而易见的。

然而，对于一个新人来说，随着每一个新的服务加入到这些平台中，学习曲线变得越来越陡。有越多的提议，就越有可能有不止一种方法来完成给定的任务。对于今天决定使用云平台的初级开发人员来说，他们总会怀疑某个给定的方法是否正确，或者它是否是一个好方法，或者它是否已经被新的东西淘汰了。

这种趋势已经开始变得明显，而且随着云平台的复杂性不断增加，这种趋势只会变得更加明显。

好消息是，这为小企业进入一个被许多人认为难以进入的市场创造了一个好机会。我称之为“零点击革命”。

**想象一下零点击即可设置云基础设施的情景。**那不是很好吗？这个概念很简单，从用户的角度来看并不复杂。此外，这是一个大型云平台无法企及的优惠，因为他们越是致力于扩展其服务组合，启动并运行某些东西的点击次数就越多。

属于“零点击革命”的一个例子是一个简单的在线工具 Crud。它将自己描述为前端开发人员的即时 CRUD REST API 端点的提供者。

Crud 允许前端开发人员节省时间，否则他们会浪费时间为他们的原型创建后端，而不是专注于他们真正关心的工作，前端。这可以通过零点击来实现。

当您访问[crud 网站](https://crudcrud.com/)时，一个唯一的 REST API 端点会自动生成并显示给您。零点击。在这一点上，你已经准备好了。端点启动并运行，由 Crud 后端管理。你可以将它复制到你的原型或概念验证应用中，并立即开始使用它，无论是在普通的 JavaScript、React、VueJS、Angular 还是任何其他前端开发框架中。

Crud 使前端开发人员能够通过零点击来设置快速而肮脏的 CRUD 后端。不需要注册，不需要账户，不需要在众多混乱的菜单和选项中导航。开门见山，毫不犹豫。大型云提供商无法企及的体验。这就是“零点击革命”。
