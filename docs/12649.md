# 代码 100 天- R2D18 和 19

> 原文：<https://dev.to/lindakatcodes/100-days-of-code-r2d18-19-4j21>

这是一篇来自我个人博客的交叉帖子。欢迎下方评论！

这周我发现了两个关于 Gridsome 的新东西！

首先——它不会告诉你什么时候它自己失败了(Vue 无声地失败)。

第二——如何改变第一件事。

所以，在过去的两天里，我一直在努力找出为什么我不能在我的博客代码上运行开发命令，这是使用 Gridsome 设置的。对我来说毫无意义。我最后一次用它的时候，它成功了！但是现在，当我运行 develop 命令时，它已经完成了 43%的构建工作....别动。通常在一个不同的文件名上，所以当它冻结时，它似乎与它列出的任何特定文件都没有关系。

我删除了。缓存文件夹，就像文档中建议的那样，以及卸载和重新安装节点模块。没有变化。我尝试了旧版本的 Gridsome，以防新的变化导致它。没什么。我在我的台式机上试了一下(因为我在我的笔记本电脑上发现了这个问题)，那是我最后一次成功运行它的地方。同样的问题。

Wtf？

于是我崩溃了，找到 Gridsome 不和谐组，在求助频道问了我的问题。幸运的是，一个用户拯救了我！他们有一个错误信息的截图(我从未见过)，显示了导致所有问题的文件。

当我最后一次在我的博客上工作时，我大部分时间都在评论一个工作不正常的页面(我想保留代码，这样我就可以继续工作，但它本身并不工作)。我没有意识到这一点，但我没有真正运行开发命令后，我这样做了。它建得很好，所以我没多想。但是我注释掉它的方式是，我让模板标签打开(注释掉结束标签)，这显然是它讨厌的。我也想！😝

因此，我将结束注释标记上移，这样模板标记就结束了，并了解到模板中也必须至少有一个元素。所以我再次移动了注释标签，只留下了一个(看起来)空白的布局元素——神奇的是，develop 命令又开始工作了！

所以。该死。**感恩！**我在故障排除方面相当不错，但是使用了一个我不熟悉的框架，该框架使用了我不熟悉的 JavaScript 框架，并且没有明显的错误消息....我很失落。很高兴我找到了不和谐小组并得到了回应！

* * *

因此，接下来的事情是学习如何让 Vue 在出错时向我显示错误信息，这样我就可以更好地自己解决问题了！Gridsome 文档中的故障排除部分提到了一行代码，可以插入来显示错误，而不是静静地失败....但是没有提到代码的去向。

对我来说幸运的是，我能够很快地解决这个问题！为此，我开始点击项目中的 Gridsome 配置页面，直到找到一个看起来不错的页面，然后查找该页面 API 的文档。哈哈😉瞧，我发现了一个看起来非常熟悉的命令！

所以，在 gridsome.server.js 文件夹中，在 module.exports 函数内部，我写了这行代码:

```
api.chainWebpack(config => { 
  config.mode('development') 
}); 
```

保存并再次运行 develop 命令后，一切仍然正常！当我停止了服务器，移动了问题文件中的一个注释标签，并试图再次运行 develop 时，我得到了错误消息:构建失败！

所以现在，如果我的文件中有未来的错误，我将能够实际看到错误消息，这将在故障排除中帮助**很多**！🙌学习新事物万岁！
