# 为什么可访问性很重要

> 原文:[https://dev.to/lhoffmanwg1/why-accessibility-matters-39nl](https://dev.to/lhoffmanwg1/why-accessibility-matters-39nl)

什么是无障碍设计？这是一种设计实践，如果做得正确，会使每个人，包括残疾人，都能平等地访问网络。它使残疾人能够访问、理解、导航、与网络互动并为网络做出贡献。

《美国残疾人法》将残疾人定义为身体或精神有缺陷，严重限制一项或多项主要生活活动的人。在这种情况下,“残疾”是一个法律术语，而不是医学术语。《美国残疾人法》是一部民权法，禁止在公共生活的所有领域歧视残疾人。然而，在美国，在公立学校等政府机构之外，合理平等地获取数字信息不受法律保护。所以这取决于你。

设计应用程序时要考虑的一些障碍包括视觉、认知、听觉、运动和智力障碍。如果没有你深思熟虑的设计，将会有相当数量的人被质疑，或者可能完全无法访问你的作品。

为什么要让你的作品无障碍？除了增加你的影响力，这是正确的做法。除了支持残疾人融入社会之外，无障碍设计也有利于非残疾人和其他得不到充分服务的人群，如老年人和农村或发展中国家的人。就其核心而言，网络旨在创造公平的竞争环境。所有人都可以使用它，不管硬件、语言、能力、社会经济和位置如何。为可访问性而设计就是这一目标的实现。

但是什么是无障碍设计呢？让设计对残疾人来说更容易理解需要考虑很多因素。但是，有一些具体的指南可以提高应用程序的可访问性。

例如，使用语义上有意义的 HTML。这是使用给内容增加意义的元素的实践。一个`<p>`标签是有意义的。它表示一个段落。人们知道什么是段落。标签来包含你的导航条，而不是使用`<div>`来通知屏幕阅读软件这个元素是一个导航条。语义 HTML 有助于为应用程序的残障消费者设定期望。使用语义 HTML 的副产品是改进 SEO。谷歌的算法分析标记和标签，以了解你的网站是什么。你的 HTML 越有语义，评价就越准确，对你的 SEO 越有利。

一致地使用带有描述性文本的图像“alt”属性(用于文本到语音的转换)；

`<img src="image.gif" alt="Smiley face" width="42" height="42">`

选择低视觉友好、高对比度的配色方案，可以决定您的网站是否适合视觉障碍用户。

尽管以下所有考虑事项在许多情况下都有帮助，但对于特定的残疾尤其重要。

视力受损的用户

*   为图像提供文本替代，例如替代文本、图表等图形中包含的信息描述
*   语义 HTML:

    `<article>,<aside>, <details>, <figcaption>, <figure>,<footer>, <header>, <main>, <mark>, <nav>, <section>, <summary>, <p>,<time>`

*   高对比度配色方案

*   大型或可扩展类型/图像，高达 400%

*   键盘导航

*   色盲的用户颜色、带下划线的文本、符号和纹理的使用以及有限的调色板

*   视频抄本

*   技术:盲文键盘，屏幕阅读器，语音识别(响应语音命令)，屏幕放大器

发作

*   没有或可选地有频闪/闪光效果

发动机

*   大的可点击链接/区域
*   语音识别(响应语音命令)
*   所有功能都可以通过键盘导航(鼠标微调马达)实现
*   支持键盘和鼠标的替代输入

声音的

*   为视频提供文本副本
*   闭路字幕

认知/智力/阅读障碍

*   明语
*   持续维护的站点
*   图表

万维网联盟(W3C)已经建立了网页可访问性倡议，概述了评估和提高应用程序可访问性的原则和方法。它特别关注用 Ajax、HTML、JavaScript 和相关技术开发的动态内容和高级用户界面控件。

W3C 的网页可访问性组件

*   内容(文本、图像、声音、定义表示的标记)
*   浏览器和媒体播放器
*   辅助技术
*   用户的知识/视角
*   开发商
*   创作工具
*   可访问性评估工具(CSS 和 HTML 验证器)

网站内容可访问性指南(由万维网联盟牵头)，内容开发者指南。解决与内容相关的四个问题:

*   可感知性
    文本替代、字幕、替代演示、增强的视听能力

*   可操作性
    键盘访问，有足够的时间让用户处理内容，不会引起痉挛，有助于导航和搜索，键盘以外的输入

*   可理解性
    使文本可读和可理解，使内容以一致和可预测的方式出现和表现

*   稳健性
    避免和纠正错误

越来越多的服务提供商定期提供可访问性审计，以发现问题并建议补救措施。使用自动化工具和由可访问性专家进行的有知识的最终用户评估。和最终用户测试。在中设计可访问性总是比回头补救更便宜。
