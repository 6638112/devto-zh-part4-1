# 有没有可能靠自己(而不是通过在公司工作)获得相关的行业经验？

> 原文：<https://dev.to/sloan/is-it-possible-to-get-relevant-industry-experience-on-your-own-not-through-working-at-a-company-213p>

这是一个不愿意透露姓名的会员发来的匿名帖子。请仔细考虑你的回答，因为这些通常是很难写的帖子。如果你想留下匿名评论，请发电子邮件至。

大家好，我正处在人生的一个关键点上，如果能得到软件开发行业有经验的人的帮助，我会非常感激。我一直很担心这件事，所以真的，谢谢你事先的建议。这可能有点长。对不起

首先，简单介绍一下背景:我 29 岁，几年前从工业工程专业毕业。当我还在大学的时候，我也对编程感兴趣，特别是 Android 开发。我的课上没有教编程；我只是在为我的第一部 Android 手机扎根的过程中偶然产生了这个想法，这是我喜欢做的事情。一段时间以来，我一直梦想将自己的想法转化为应用程序，并将其应用到我的业务中(首先是 Android 应用程序，如果一切顺利，可能会是 iOS 应用程序)。我是一个更有创造力的人，所以我想基于我的想法发布我自己的应用，而不是客户的工作。).显然，我希望在这方面有一个成功的企业。我参加了几门在线 Java 和 Android 课程，现在我已经知道了足够多的知识，可以开始构建我的第一个应用程序想法。顺便说一句，我在很大程度上还是个初学者。因此，它们可能没什么新奇的，但它们会是一个有用且实用的产品，随着我继续发展我的 Android 知识，它们有可能得到改进。

小提示:我也喜欢工业工程，设计和优化流程等等，并希望将这些知识应用到我的业务中。

好吧，所以我目前有一份非编程的工作，大致支付我的个人账单。我可以在家兼职做，因为这是自由职业，这很棒，在我的国家是一个奇迹。这让我有很多空闲时间开始构建东西，并继续学习 Android 和更多 CS 的东西。这也是一份非常没有压力的工作，当试图将其与其他活动结合时，这非常有帮助。我也在焦虑、强迫症和抑郁症中挣扎，所以这种工作对我的心理健康来说是一个非常积极的改变。我非常愿意遵循这条特定的道路，在这条道路上，我继续支持自己的工作，同时开始发布应用程序并从事这项工作，直到我真正开始赚足够的钱，并最终可以将此作为我的全职工作。

我的问题是，我对之前描述的路径有一个主要的不满。我有一些工作经验，但是是在与软件开发行业无关的领域。今年早些时候，我在一家真正的软件开发公司做了短期实习，他们在那里做客户工作。他们应该在 Android 开发方面进一步培训我，但这几乎是一个骗局；我没有接受任何形式的培训，我真的对他们的文化和我从环境中感受到的一切感到不舒服，所以我结束了实习，因为这感觉像是浪费时间，也感觉不太对。但是，当我在那里的时候，我能够从其他员工的交谈中了解到一些事情。这些人似乎对他们的过程有非常清晰的定义和微调。他们使用许多不同的工具来做一些事情，从沟通、跟踪项目任务、跟踪时间、管理他们构建的 API 等等。很明显，我对这些工具及其遵循的流程一无所知。我仍然不知道他们到底在做什么。我只能注意到他们有几个过程和工具。我不禁感到不知所措和焦虑，因为我对所有这些流程和工具知之甚少，完全没有这样的行业经验是多么的不利。

我知道拥有一些相关的行业经验会增加你生意成功的机会。显然，我希望围绕我的应用程序开展业务的尝试取得成功。正如我之前所说，我首先描述的道路是我真正想走的道路。但在软件开发公司的这段经历让我想到，如果我想拥有一个成功的应用业务(或任何其他软件业务)，我绝对需要在另一家做这种业务的公司工作一段时间，以便获得一些行业经验，然后最终过渡到自己的业务。我有一个想法，如果我不去另一家公司工作，我将永远不会获得我后来成功所需的现实世界的行业知识，就像这是唯一的方法一样，如果我不这样做，我的应用程序业务梦想将很早就失败了。问题是，由于非常强烈的个人原因，我不想再在另一家公司做员工了。因此，我有这样的想法，如果我完成了上述我想要的原始计划，我将无法获得与在公司工作时相同质量的知识和行业经验，我最终会失败。但是，正如我所说的，我真的不想再在一家公司工作了。

我也有一部分认为，如果我坚持走我想走的路，我仍然可以靠自己获得同样水平的行业知识。在某个时候，我实习的公司的创始人和/或员工必须坐下来定义他们的流程，对其他公司正在使用的工具和流程进行一些研究，试用它们，并随着时间的推移不断测试和优化它们。也许他们也是从零开始，因为创始人开始时还很年轻。所以我也必须遵循同样的过程，直到我找到自己选择的工具和过程。这肯定会比我在其他地方工作花费更长的时间，在那里我可以看到已经存在的工具和过程的例子，这可能会导致更多的挫折和反复试验。但这就是为什么我会得到目前工作的支持(同时过着有点节俭的生活)，直到事情进展顺利，足以让我能够完全过渡到我的应用业务。我还认为，我的工业工程背景在这方面可能非常有用，事情可能会变好。

如你所见，我对这个行业了解不多...所以我想知道那些真正了解的人的意见。去另一家公司工作真的是绝对必要的吗？(我知道这是最理想的做法...但正如我所说，我有非常强烈的个人理由反对这一点。)还是我有点太执着于这个想法而我的第二个观点也是可行的？在那里，我沿着我想要的道路前进，最终仍然可以获得想要的行业知识水平和质量，使我的企业取得成功。

非常抱歉发了这么长的帖子。只是我一直对此很焦虑，很执着，很压抑。我真的需要和比我了解更多的人讨论这个问题。我非常感谢你阅读并给我任何你能给的建议。
