### [](#how-to-leverage-kubernetes)如何利用 Kubernetes？

[![](../Images/f1ebf413e55dc4c3e7d32a4959fd3ad9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--05Erg9g9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://greenido.files.wordpress.com/2019/06/screen-shot-2019-06-17-at-3.53.27-pm.png%3Fw%3D696)

我们不仅谈论新项目，还谈论传统应用以及如何在混合世界中使用它。对安全性、可见性、网络等有什么影响。

当你把系统的一部分改造成新的世界时，如何让 k8b 满足你的需求。K8B 是一项新技术，它只有五年的历史，但发展速度是惊人的。越来越多的公司和开发者参与其中并使用它。

我们今天看到的是，只有大约 30%的客户在生产中使用 k8b。然而，三年后，我们将看到这一数字跃升至 85%左右。最重要的是，你没有错过任何火车。但是，你确实需要计划并看看 k8b 能在你的公司中做什么，因为它看起来将成为管理容器的主要技术。

### [](#cloud-native)*云原生*

[![](../Images/01b4f51facfe9b96b61251a63eee2976.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jFVfxZ95--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://greenido.files.wordpress.com/2019/06/screen-shot-2019-06-17-at-3.53.46-pm.png%3Fw%3D696)

Cloud native 是关于容器、自动化和微服务的。

当我们考虑采用 it 并将其融合到当前应用程序中的步骤时，企业面临许多挑战。然而，我们在市场上看到的许多不同行业的公司中，有一半已经在使用 SAAS 或 PAAS 解决方案进行开发运维。此外，在接下来的 3 年里，几乎每个人都会以某种形式出现。

### [](#hybrid-world-multicloud)*混合世界/多云*

[![](../Images/118cef6df879a774970e72dbe021887c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vnhMWg3M--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://greenido.files.wordpress.com/2019/06/screen-shot-2019-06-17-at-3.53.57-pm.png%3Fw%3D696)

几年前，我们听说云将无处不在，所有内部部署都将迁移到云中。

我们今天知道，我们生活在一个混合的世界，在许多行业中，这将是可预见的未来的规范。如何在如此复杂的世界中保持高效？利用它的选项/工具/方法是什么。

你可以看到像 [Anthos](https://cloud.google.com/anthos/) 这样的项目正在被发布，以帮助公司弥合这一差距并维持他们的混合世界。

### [](#security)*安全*

[![](../Images/0509726ebb41c8d10a6fbf491c5ac8a6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BUE2rmIc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://greenido.files.wordpress.com/2019/06/screen-shot-2019-06-17-at-3.54.10-pm.png%3Fw%3D696)

这里有很多机会和挑战，但归根结底是“如何让您的开发人员保持他们的速度和您必须拥有的安全级别？”

令我惊讶的是，只有%29 关注第三方软件，尽管它(在大多数情况下)包含 80%或更多来自你的代码。这是一个需要改变的焦点。

安全性不仅与漏洞有关，还与合规性和许可证有关。我们如何在开发人员构建应用程序时向他们展示正确的方法(使用正确的工具，例如 ide 的 X 射线插件),另一方面，让安全经理清楚地了解生产中正在发生的事情(例如来自发行版和其他来源的仪表板)。
