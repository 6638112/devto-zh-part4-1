# 为什么使用 ReactJS

> 原文：<https://dev.to/coolanant/why-use-reactjs-4m57>

> 在本文中，我不会详细解释每件事，但从开发人员的角度来看，我将解释 ReactJS 与其他旧平台的不同之处，因为当我们询问时，许多开发人员仍然不知道或不清楚使用 ReactJS 的目的。
> 
> React.js 是什么？React 是一个 JavaScript 库，用于构建用户界面，即客户端界面。React 于 2011 年在脸书创立，最初于 2013 年 3 月发布。
> 
> 它用于创建**spa(**单页应用* *)* * —无需页面重新加载，无需额外的等待时间。它只是你访问的一个网页，然后加载不同的组件。
> 
> React 允许您将组件定义为类或函数。被定义为类的组件目前提供了更多的特性，这些特性将在本页详细描述。它有助于努力实现*可伸缩性、代码可重用性*，并保持你的代码*有条理。*
> 
> 它提供了**虚拟 DOM** (文档对象模型)，即它在将最终网页渲染到浏览器之前执行必要的更新，这也是 React 能够创建快速、可扩展的 web 应用的核心原因之一。
> 
> 它为我们提供了新的功能，如**jsx**(JavaScript 的语法扩展)**状态管理**(使用 redux)将所有更改存储在一个部分中，并在 **props** 的帮助下将自定义数据传递给特定的 UI 组件。
> 
> **易于迁移—** 从旧版本技术迁移到新版本技术时，在执行任务所需的工作量和时间方面会出现一些问题。
> 
> React 是一个轻量级的 JavaScript 库。React 代码可以添加到现有基础设施的任何地方，而不用担心关闭系统进行维护，也减少了重新发明轮子的依赖。
> 
> 谷歌的角状框架；例如，在其第一代 Angular 1.x 和未来的 Angular 2 之间面临着非常令人担忧的兼容性问题。有意转向新版本的公司不得不投入时间对开发人员进行最新技术的培训。
> 
> 有许多 React UI 框架，它们提供了一个非常棒的用户界面，代码更少，网页加载速度更快，T2。
