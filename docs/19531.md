# 简介

在我参与的 meetup 中，我为我的博客读者遇到的一个问题是。

这将是“他们如何开始自己的技术博客，他们应该在博客中写些什么”。

读了约翰·索梅兹的书[《软技能:软件开发人员的生活手册》](https://www.amazon.com/Soft-Skills-software-developers-manual/dp/1617292397)。

那是大约一年前的事了，现在我已经完成了第二次尝试约翰·索姆兹的[简单程序员](https://simpleprogrammer.com)的[博客文章课程](https://simpleprogrammer.com/blog-course)。

这有助于解决我写文章时遇到的一系列问题，所以这里有一些我在博客课程中学到的经验，这些经验对我帮助很大，也适用于个人。

# 文思枯竭

找到有趣的话题来写很难。我的“过程”是我的直觉的古老用法。

每当我在空闲时间上网时偶然发现有趣的文章、播客、视频。

这激励我根据我从中学到的东西来写我自己的文章。

现在我学会了建立一个主题列表，让我的生活更容易写博客。

# 关注你的博客

博客课程教会了我如何为我的内容的特定读者定位我的博客，这让我想到了主题**创业**、**网络开发** & **Python** 。

直到今天，我已经写了一些文章，比如我的[用 Flask 构建 Restful API，Postman & PyTest 系列](https://www.maxongzb.com/building-restful-api-with-flask-postman-and-pytest-part-3-read-time-20-mins/)和[给初级开发人员去创业的提示](https://www.maxongzb.com/tips-for-junior-developers-heading-to-startups-reading-time-4-mins/)

# 为您的博客增加流量

我在课程中第一次尝试为我的博客增加流量。我学会了寻找我感兴趣的博客来评论他们的博客。

这个建议让我在 [Dev 社区](https://dev.to/)创建了一个账户来评论文章。

这真的很酷很有趣，就像[2019 年 React 开发者路线图](https://dev.to/javinpaul/the-2019-react-developer-roadmap-54ca)或[我是 Joseph Jacks，OSS Capital 的创始人和普通合伙人，问我任何问题！](https://dev.to/asynchio/im-joseph-jacks-founder-and-general-partner-of-oss-capital-ask-me-anything-4c52)

现在我认为把时间花在开发社区上比把时间浪费在 facebook 上要好得多。

# 结论

我希望它能激励你开始你的博客之旅或提高你的博客技能。

成为一个更好的开发者、沟通者，建立自己的个人品牌，在撰写自己热爱的文章的同时获得更高的薪水。

我多么希望我遇到了一个一步一步开始你自己的技术博客的指南。

当我开始之前，我已经创建了我的博客，以减少我遇到的错误数量。

我把建立技术博客的想法带回家，以帮助我成为一名更好的开发人员。
