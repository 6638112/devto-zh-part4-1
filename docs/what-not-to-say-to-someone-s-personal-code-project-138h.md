# 对某人的个人代码项目不该说什么

> 原文：<https://dev.to/vanessaz/what-not-to-say-to-someone-s-personal-code-project-138h>

虽然我在过去的十年里一直在科技领域工作，但我不是一名程序员。我能理解代码——如果我完全不能理解我看到的代码的头或尾，我永远也不会得到“质量保证工程师 I”的头衔——但是我不会写原始代码。我的编码能力的范围是阅读别人写的东西，然后根据我的需要进行调整。

前几天，我意识到我开始了职业生涯的一个新阶段。除了我参加过的一些培训之外，接下来还有一些演示，我还在工作场所的三个不同的委员会中担任领导角色。换句话说，我真的应该有一个网站。

当我思考如何做到这一点时，我考虑使用像 Wordpress 或 Drupal 这样的 CMS，但最终决定这将是我学习新技能的一个好机会。所以，我决定完全编码我的网站，并使用 CSS 样式，这是我从来没有做过的。投资组合网站主要是静态内容，所以我不会做任何太花哨的东西，但这将是我第一次完全由自己制作的东西。

当我和朋友讨论这个项目时，有一个互动很突出。当我告诉我的一个朋友我的计划时，他说，“你知道有一些平台可以为你做这件事，对吗？”

这不是你对谈论个人项目的人说的话。我当然知道！在我的上一个角色中，我支持了其中的一些。你不会说“你知道你可以在塔吉特买，对吧？”对那些告诉你他们想学习如何用陶轮制作咖啡杯的人。

在那一刻，我感到激烈和保护，但回想几年前当我对自己不太自信时，我很容易看到自己道歉并停止项目，反而做得更多(也许？)使用完整 CMS 的高效方式。请不要成为某人选择不追求做或学某事的理由，仅仅是为了学习它的意图。你永远不知道它是否会引发更具创新性的东西。
