# Fedora vs Ubuntu 对程序员来说 Ubuntu 还是 Fedora 更好的发行版？

> 原文：<https://dev.to/prahladyeri/is-ubuntu-or-fedora-a-better-distro-for-programmers-4lao>

你应该使用哪一个发行版作为开发工具需要考虑很多。这不仅仅是你的偏好或者你喜欢桌面图形用户界面，打包系统或者工作流程的其他方面，还有很多其他的东西。例如，该发行版的更大的生态系统和你得到的支持种类非常重要。

我个人是用 Ubuntu 做日常驱动的，这里有点偏！我最喜欢 Ubuntu 的一点是它简单易用。也许 Red Hat 和 openSUSE 这些天也改进了很多，但至少在 2010-2014 年，Ubuntu 在硬件支持和桌面技巧方面占据上风。如果你有一台笔记本电脑，配有一种特殊的触摸板或由 NVidia 或 Broadcom 等“对 linux 不友好”的公司制造的 WiFi/显卡，它很可能会放弃使用 Ubuntu 以外的任何其他发行版。

即使在他们确实工作的情况下，他们也留下了许多需要改进的地方。以 Debian 为例，许多默认设置在桌面上根本无法使用。例如，字体抗锯齿通常被设置为关闭(至少在我上次尝试的 Debian-XFCE 上是这样)，让你的桌面字体看起来很奇怪。第二，他们甚至没有提供一个 GUI 来控制你的触摸板设置，他们希望你去谷歌一下这个问题，自己摆弄一下拜占庭/etc/* config 文件！

不是说我做不到，但是当 Ubuntu 默认给我一个配置好的、打磨好的桌面时，我为什么要做呢？

来到 Fedora，桌面从来没有任何问题(特别是在 GNOME 版本中)，但我不知何故不太喜欢他们的打包系统(yum/dnf)。Ubuntu 的 apt 安装无缝地工作，并带有特殊的工具，如 synaptic 和 aptitude，以处理高级场景。公平地说，我并没有花太多精力去学习 dnf，但是我看不出它有什么意义，除非我能从中获得任何长期的好处。

Fedora 的另一个经常被吹捧的优势是，他们提供了一个“前沿”发行版，这意味着所有的应用程序都应该是最新版本。但是坦率地说，我从来没有感觉到需要一个超级最新的软件包，而且在几个星期的时间里，新的版本最终会渗透到所有的发行版。对我来说，运行一个稳定安全的应用程序版本远比拥有最新版本更重要。

最后，我不喜欢 Ubuntu 的是他们的支持！bugs.launchpad.net/ubuntu 的官方 bug 追踪器在解决问题时非常糟糕。那里的维护者的态度和政府一样。员工！但值得庆幸的是，在其他一些地方，比如 [askubuntu](https://askubuntu.com) 和 [ubuntuforums](https://ubuntuforums.com) ，你可以找到理智的声音。
