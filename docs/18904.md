我很好奇。有很多云服务被描述为**无服务器**，但没有一个单一的、被一致接受的行业定义。一些常见的想法包括:

*   *小额计费(按使用付费)* -如果您不经常使用，则无需付费
*   *事件驱动/代码* -由事件触发的小代码足迹
*   *自动缩放* -按需自动缩放
*   *无需配置/设置* -无需设置虚拟机或配置集群

一些例子包括 [Azure 函数](https://docs.microsoft.com/azure/azure-functions/functions-overview?WT.mc_id=dev_to-blog-jeliknes)、 [AWS Lambda](https://aws.amazon.com/lambda/) 、 [Google Cloud 函数](https://cloud.google.com/functions/docs/)、 [OpenFaaS](https://www.openfaas.com/) 和[Apache open whish](https://openwhisk.apache.org/)。

我很好奇，根据你的知识，这些服务有什么共同点使它们变得特别？与我们已经有一段时间的“完全受管”选项相比，什么可以将服务定义为“无服务器”？
