# 如何逃离教程的炼狱，成为一名高效的开发人员

> 原文：<https://dev.to/remus432/how-you-can-become-a-productive-developer-by-escaping-tutorial-purgatory-12ok>

旧网络的时代已经结束了。

数十种新技术已经兴起。作为一名开发人员，你应该知道哪些技术可以增加你的价值。

跟上技术领域的学习是你必须面对的一个真正的挑战。

一切每天都在变化。

对于自学成才的程序员来说，当你被拉向各个方向时，会变得很沮丧。你很容易失去对自己处境的控制，被外面的大量信息淹没。

对于自学成才的开发人员来说，教程是最可靠的学习资源之一。

基于教程的平台 Udemy 之所以如此受欢迎，是因为它通过低价提供高质量的教程吸引了全世界的开发者。

Udemy 包含了大量有价值的教程，但是这里有一句谚语“任何东西太多都不好”。

教程也是如此。你看多少教程和你实际投入多少实践之间必须有一个平衡。如果你没完没了地看教程，却没有作为一个 web 开发人员取得什么成就，这是教程炼狱的一个强烈迹象。

您越早发现症状，就能越早治疗它，并重新成为一名高效的开发人员。

在这篇文章中，我们将探讨教程炼狱的可能原因，如何解决它，以及什么时候应该真正使用教程。

* * *

# 精神短路

观看一个教程恢复到观看一个开发者在你跟随的时候编码一个项目。

你必须注意代码的内容和方式，同时注意他的解释，以便理解你写的代码行背后的逻辑。

有时你不得不重新看教程，因为你没有时间去理解和跟随代码。

然而，教程是一种便捷的学习方式。

可能太方便了。教程太方便了，以至于成了一种商品。你开始一个接一个地看教程，因为为什么不呢？

我的炼狱教程源于对无所作为的恐惧。我想感受到我实际上是富有成效的，我正在取得进步。

教程只是一个创可贴。

我害怕投入到一个真正的项目中，因为我知道我必须面对决策的负担，并且因为我不知道的简单错误而浪费无数的时间。

我放弃了作为教程编码员的责任，我可以被动地观看，但仍然让我感到有效率。

但是教程永远不能为你在现实世界中的未来做准备。

你必须去掉创可贴，接受现实。你有时会挣扎，但即使是高级开发人员也会在他们的项目中挣扎。为什么你的情况会不同？

教程为你提供基于特定背景的信息，并限制你的创造力。你知道如何使用信息，但不知道何时使用。了解使用概念的背景对于理解概念至关重要。

我的大多数项目都是“跟进”项目。

这些项目功能完善且完整。代码就像一首诗，网站运行流畅。但是，这真的证明了我作为开发人员的能力吗？我在简历中使用它真的有价值吗？

在现实世界中，斗争是不可避免的。学习编码不能走捷径。如果我在做项目的时候不挣扎，我能期待什么样的成长呢？

匆忙也可能把你推进炼狱教程。

你想在 3 个月内成为一名程序员，找一份工作，甚至是一份偏远的工作，开始以你喜欢的工作谋生。

web 开发的想法是如此酷的一份工作，它已经征服了我。我迫不及待地想学完当前的语言，这样我就可以继续学习另一种语言了。

从一个教程跳到另一个技术，实际上会让你停滞不前。

你不应该试图浏览一种语言，然后再去学另一种语言。改变你的策略，做一个缓慢但高效的学习者。选择一门语言并坚持下去，直到你能从中找到实质性的东西。

> "只要你不停下来，你走得多慢都没关系。"—孔子

# 头朝下跳水

关于自己编码最可怕的事实是，你必须承担责任，花无数的时间来解决 bug。

你不是被动地看着某人编码并为你做所有的决定。

你必须经历一个网站开发者的思维过程，自己做决定。比看一个教程花的时间多，但是成果丰硕。

> “对于我们在做之前必须学习的东西，我们通过做来学习。”—亚里士多德

我坚信刻意练习是最好的学习方式。

仅仅练习并不能维持真正的成长。刻意练习的重点是强化你的弱点，让你加强你的神经连接。

因此，学习代码的最好方法是通过项目实践。

你的项目不一定是像 todo-app 这样的通用项目。你可以为疯狂、愚蠢的项目而奋斗。你越喜欢你的代码，你的学习经历就会越好，这也会增加你的投资组合。

编码是创造力和逻辑交叉的地方。

你可以把如此惊人的创造带到餐桌上。你只需要耐心地玩，直到它点击。你的直觉会告诉你。你将在一个项目上工作，一切都可能会井井有条。

项目可以让你把脚浸入现实的冷水中。它不会总是充满乐趣和游戏。为了完成你想做的任务，你必须完成那些你不想做但又必须做的无聊任务。

但是如果你对项目没有任何想法呢？你从哪里开始？

你能用你已经拥有的知识创造什么？

# 余额

获得项目创意的先决条件是了解一门语言能做什么和不能做什么。

例如，有了 CSS，你现在可以完成许多只有 JS 才有可能完成的交互。CSS 在网页设计方法的多样性方面很丰富。

但是如果你不知道一门语言能做什么和不能做什么，你怎么开始学习呢？

一个月前，我开始重新学习 React，然后我继续学习 Redux 和 React 路由器。

我一无所知，但我知道我需要从某个地方开始。我去了 Redux 的文档页面，在浏览过程中，我做了笔记。我学 React 路由器的时候也是这样。

在我完成了基础工作后，我开始构建小项目。在几个小项目之后，我继续将我所学到的一切融入到一个大项目中。

我只能从教程中学到这么多。在我完成这个项目后，我学到的东西比通过教程学到的要多得多。我把我的皮肤放在游戏中。

学习编码需要理论和实践之间的平衡。

我学习编码的方法是收集基础知识来建立基础，然后在进入一个大项目之前将基础知识付诸实践，这个大项目将把你学到的所有东西都集中到一个地方。

你绝对应该使用教程来学习你缺乏知识的主题的基础，但不要学得太多。

使用 80/20 法则:80%的时间应该用于项目，20%用于辅导。你会收集到足够的信息来开始，然后你会带着新的知识和经验来到另一端。

通过这种平衡，你将成为一个有效的网络开发人员，能够适应新的环境。

记住，为了逃离教程的炼狱，你必须修正你的心态。你陷入炼狱教程的根本原因可能与我不同。

> “你不会变成你想要的样子，你会变成你相信的样子。”—奥普拉·温弗瑞
