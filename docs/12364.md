# 不要让游戏化拖垮你

> 原文:[https://dev . to/charlesdlandau/don-t-let-gamification-bring-you-down-27di](https://dev.to/charlesdlandau/don-t-let-gamification-bring-you-down-27di)

<figcaption>Photo by Kay on [Unsplash](https://unsplash.com/photos/V3qzwMY2ak0)</figcaption>

我最近有一次对话，主题是[“你不是你的代码。”](https://dev.to/josegonz321/comment/38f3)这是一个我来`dev.to`之前从未听过的表达，我认为这是一个极好的提醒。总之，对话是关于机器学习模型以不太完美的准确度执行是如何没问题的。这让我意识到，游戏化平台(如 Kaggle、CodeWars、HackerRank 和 DeepRacer)可以改变我们对自己的看法，如果我们允许的话。*就像任何其他竞技游戏一样。*

事实上，如果你在生产中有一个以前没有的可预测的、增值的模型，那就是一个巨大的胜利。如果你能达到 0.9 到 0.99 的精度，那也是一大胜利。同样，如果你能在 CW 上打败一个 6kyu“简单”的形，那就太棒了！如果你能在“大师级”1kyu 中击败另一个版本的形，那就太棒了！

这些平台为我们提供了一个竞争、兴奋和展示技能的场所，这太棒了。保持正确的观点也很重要——你不是你的代码。如果你输了一场赛马比赛，你仍然可以做你自己。不要让你自己认为你不是一个开发者，除非你能打败大师级的卡塔，或者你不是一个数据科学家，因为你在 Kaggle 上没有赢得最高奖项。这不是那些服务的目的。
