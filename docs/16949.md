# 成为创客的软件开发人员-第 8 周

> 原文:[https://dev . to/Leon levitate/becoming-a-a-software-developer-at-makers-week-4-21pe](https://dev.to/leonlevitate/becoming-a-software-developer-at-makers-week-4-21pe)

我们已经在 Makers 度过了第 8 周。今年早些时候，当我决定报名参加一个软件工程训练营时，我知道我会经历一个旋风般的夏天。在现场的第一天，我们被提醒了两个要点。首先，做好准备，除了创客，你几乎没有空闲时间。其次，面对不适，相信过程。

听起来很合理，也很让人安心。我已经迫不及待地告诉我的一个好朋友，我不能再参加她在康沃尔的 3 天婚礼了，因为婚礼将在我们小组项目的中期举行。她出乎意料地接受了，并且非常支持我加入创客的决定。现在我已经被赦免了几项夏季活动，从这里开始一切都应该一帆风顺了！

我怎么会在这里？我之前在一家全球软件公司担任咨询运营经理。我戴着几顶不同的帽子，应付着许多不同的责任和日常挑战。幸运的是，我接触了所有的团队，并且总是发现自己被坐在我后面的开发人员的工作所吸引。他们是构建和开发我们的企业架构工具的天才，这似乎是所有奇迹发生的地方。我有技术才能和毅力成为他们中的一员吗？这让我开始了成为一名软件开发人员的旅程。

在 Makers，每周我们都会学习新的工具和技能，来使用 Ruby 或 JavaScript 构建一个项目。上午的时间都花在了研讨会、实践练习和自主学习上。下午是我们开发测试驱动开发(TDD)、行为驱动设计(BDD)和结对编程的专业知识的地方。

以下是我在校园生活中得到的三个重要收获:

### [](#test-driven-development-is-a-big-deal)测试驱动开发是一件大事

我花了几天时间来理解为什么我们应该在应用程序代码之前写单元和特性测试。这肯定会增加开发时间并潜在地阻碍我的进度吗？哦，我错了。每天应用这个过程让我明白了 TDD 和 BDD 对于一个开发人员来说是多么的重要。它允许我们以敏捷的方式工作，并以用户需求和体验为出发点开发我们的产品。以迭代的方式工作并定期运行测试也给了我们信心，我们正朝着正确的方向前进。在我们继续编程之前，我们有能力识别错误发生的时间并解决它。写了一个失败的测试，然后应用代码让它通过绿灯，这种内啡肽的冲动有很多可说的。

### [](#pair-programming-is-great-for-the-soul)结对编程对灵魂很有好处

作为一名新的开发人员，我每天都从与不同成员的合作中获得了巨大的价值。当屏幕上只有一双眼睛时，解决问题会变得棘手，而且常常令人沮丧。结对可以让我们讨论策略、思考想法并相互分享最佳实践。到这个周末，你也会少一些白发。采用驾驶员-导航员角色，并定期切换，意味着我们从最终提高代码质量的双重视角中受益。这给了我们独自完成周末项目的弹药和信心。当我和我的搭档在课程开始时设定我们的期望时，结对编程效果最好。我们同意时间盒的事情，例如在寻找额外的资源之前，我们应该在一个问题上花多少时间。在会议期间，思考结构和方法是否符合计划是很重要的。我们是继续我们的工作，还是花些时间独自探索？哦，别忘了庆祝成功。远离屏幕击掌和短暂休息有助于激励和提高工作效率。

### [](#mindset-is-everything)心态决定一切

这也许是最关键的一点。制造商热衷于采用增长的思维方式，他们在这方面会给你很好的指导。在我们作为开发人员的职业生涯中，总会有新的技术、框架和语言需要掌握。当学习新概念时，很容易陷入几个兔子洞，但重要的是找到足够学习的“甜蜜点”,在不被额外的层淹没的情况下前进。从技术的角度来看，是的，编程需要你在很长一段时间内付出努力。没有人天生就具备 HTTP 流量、数据库和算法方面的知识，但通过整体思考并理解随着时间的推移，通过沉浸，矩阵很快就会变得清晰，你会收获更多。当我陷入一个问题(一天几次)时，我不会陷入“这太难了”的固定思维模式，而是问自己“我是不是比昨天更好的开发人员？”。通过定期的自我反省和评估，它提醒我，几周前我可能觉得无法理解的话题，我现在正在轻松地完成。技术令人激动——拥抱终生学习和成长的承诺！
