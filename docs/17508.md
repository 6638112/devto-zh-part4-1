# RFCs 视图

## 什么是 RFC？

“RFC”(征求意见)过程旨在为新特性进入框架提供一致且可控的途径。

许多变更，包括 bug 修复和文档改进，都可以通过正常的 GitHub pull request 工作流来实现和审查。

虽然有些变化是“实质性的”，但我们要求这些变化通过一点设计过程，并在 Vue [核心团队](https://vuejs.org/about/team.html)和社区中产生共识。

## RFC 生命周期

RFC 经历以下几个阶段:

*   **待定:**RFC 作为 PR 提交时。
*   **活动:**RFC 请购单合并并实施时。
*   **登陆:**当 RFC 的提议变更在实际版本中发货时。
*   **拒绝:**RFC 请购单未合并即关闭。

[待定 RFC 列表](https://github.com/vuejs/rfcs/pulls)
