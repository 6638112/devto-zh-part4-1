# 作为开发人员，我选择和管理软件和硬件的原则

> 原文：<https://dev.to/orkon/my-principles-of-choosing-and-managing-software-and-hardware-as-a-developer-3lha>

嘿！欢迎来到一个非编码的帖子，在这里我分享我如何组织我的软件和计算机。这可能是我的第一篇与编码无关的文章！请分享您的想法，让我们开始吧:

## 1。使用默认配置

当我安装新软件或获得新硬件时，它必须开箱即可满足我的需求。我知道许多人喜欢配置他们的软件，甚至从源代码开始构建。它不适合我:例如，多年来我一直使用默认的 Ubuntu 版本，没有定制任何东西。所以从 2008 年开始，我再次经历了 Gnome，Unity，和 Gnome 的日子。我觉得它节省了我很多时间，因为我没有试图改变系统或迁移到新的系统。这些年来，我所做的只是适应微小的变化。

所以，默认选择适合自己的产品也是必不可少的。通常，如果你需要改变产品很多，它可能不是一个很好的适应，会花费你很多时间。

此外，我会提前检查我计划使用的软件是否与我将要购买的硬件兼容和工作良好。

## 2。为所有重要文档建立备份

这就是我四年前所做的:我确保我的文档文件夹中的所有东西都定期备份。即使我的硬件损坏或被盗，我也可以随时访问备份。有多种方法可以设置备份:您可以使用 Dropbox 或其他服务。我选择了不同的方式:我从 Synology 购买了一个网络连接存储系统，在那里我有两个 RAID 硬盘。Synology 提供了一个备份程序，你可以在你的系统上运行，自动备份每个文件。我的 NAS 没有连接到互联网。

此外，我还做了以下工作:

*   我写的所有代码都存在在线服务中，比如 GitHub 或 BitBucket，还有我的工作电脑
*   我把我用手机拍的照片备份到谷歌云(帮他们做机器学习，你懂的)。我将最重要的照片备份到 NAS。关于其他的，丢了我也没事，但是留在 Google 上，享受自动同步也很方便。

## 3。使用密码管理器

我用的是 [KeePassX](https://www.keepassx.org/) ，本地存储加密的密码。我只在我的主计算机上创建密码，它会立即备份到 NAS。我在手机上保留了一份备份，以便在需要时可以访问服务。

## 4。不要自动安装和配置您当前使用的软件

我不会自动安装，也不会备份我使用的大多数工具的配置。我不这么做的原因是，如果我的电脑坏了，我需要安装一台新的，很可能，我不会需要再次使用我多年来安装的所有软件。通常，有更好的方法或更好的软件来解决特定的问题，所以我想在下次设置系统时发现它。每天都有问题被解决，错误修复被发布，很有可能你今天自动化的任何东西都将过时。

一个例子是:NVIDIA 驱动程序过去在 Linux 上很难设置和配置。我本可以花时间自动完成设置，并尝试以某种方式简化它，但 Ubuntu 和 NVIDIA 的开发人员最终取得了良好的进展，因此驱动程序可以开箱即用。

## 5。以上 4 条规则都有例外

世界并不完美，有时我不会使用密码管理器或忘记备份文档。有时，有一个优秀的软件，我会坚持一段时间，并且不会经常更改，在它身上投入更多时间是有意义的，例如，设置您的主编辑器或终端。尽管如此，我还是尽量减少花在这上面的时间。

所以总结一下，我把我的数据存储在多个地方，所以全部被销毁的几率比较低。我尽量减少我花在配置和调整软件上的时间。相反，我专注于寻找开箱即用的软件。随着时间的推移，我从切换到更新更好的工具中受益，因为我没有在当前的工具集上投入太多。此外，如果我需要恢复我的设置，它通常需要很少的时间。
