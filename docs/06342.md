# [](#superpower)超级大国

面对现实吧，我们开发者生活在一个非常奇怪的世界里。向 95%的人解释你做了什么，他们可能会在你说完话之前变得呆滞无神。对大多数人来说，你每天用电脑做的事情看起来像是血腥的魔法，也应该如此，编程是一种超能力。

这是一个需要仔细思考和考虑的超级能力，特别是对于那些以后不得不阅读你的代码的程序员。

> 编写代码的时候，要把最终维护你的代码的人想象成一个知道你住在哪里的暴力精神病患者。

我希望在我之前的每个程序员在写 3000 行函数之前都读过的一本书是[干净的代码——罗伯特·c·马丁](https://www.amazon.co.uk/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882)。

# [](#why-clean-code)为什么要清理代码

这本书给了我一个全新的视角，它重新构建了我对每行代码的思考方式。这让我意识到成为一名软件工程师是一门艺术。

## [](#it-teaches)它教导:

*   如何识别不良代码
*   如何有意义地命名变量/类/函数
*   如何编写干净、小巧、单一用途的函数
*   什么时候发表评论，或者更确切地说，什么时候**不**发表评论
*   正确格式化代码的准则
*   使用对象和数据结构进行适当的数据抽象
*   如何正确地遵循 TDD 的法则并编写干净的测试
*   处理错误的优雅方式
*   无尽的重构技巧
*   67 种气味和启发式(是的，我数过了。🤣)

还有很多很多。

每当我编程时，我都有干净的代码作为参考。这本书有大量将坏代码转化为干净代码的例子，以及如何实现的一步一步的过程。

例如，就在前几天，我正在重构一个函数，它在许多方面违反了单一责任原则，我正绞尽脑汁想如何正确地分解它。我查阅了这本书来唤起我的记忆，寻找一种有效的方法来完成这个过程，有一个例子**和我的问题**完全一样，唯一的细微差别就是上下文。

# [](#finally)终于

在我看来，每个程序员都应该至少读三遍这本书😄。它会给你一个全新的编程爱好。当你在写糟糕的代码时，你会真正开始明白。你会看着你的旧代码，对它的结构、垂直间距、复杂性、不同步的抽象、无用的注释和意大利面条的性质感到畏缩。嗯，我有，每一天都有。在书中，代码片段是用 Java 编写的，但是同样的规则也适用于大多数语言！

这个博客只是简单介绍了阅读这本书的好处，如果你还没有读过，我强烈建议你读一读。我很想听听你对干净代码的想法，以及它可能如何改变你的观点。我也很想听听其他一些彻底改变了你看待代码的方式的书。
