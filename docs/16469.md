# 清除或保留 GitHub 中旧的、未完成的项目？

> 原文:[https://dev . to/bradley Collins/purge-or-keep-old-unfinished-projects-in-github-1 lbb](https://dev.to/bradleycollins/purge-or-keep-old-unfinished-projects-in-github-1lbb)

背景:几年前，我开始断断续续地学习编程，直到现在，我才真正开始钻研并尝试转换职业。

有了它，我有了一年多的 GitHub 库，还有我开始时不知道自己在做什么的未完成项目。

我应该如何处理这些存储库？我应该删除它们，假装它们从未存在过吗？我应该把它们作为战利品/提醒我来自哪里吗？

如果招聘人员/面试官看到这些仓库，他们会给我白眼吗？

我在想什么:

*   更新自述文件以解释项目(它们现在是空的 D:)
*   弄清楚我是否想完成它们中的任何一个，然后给它们安排一个时间表来确保我能完成
*   任何我计划不完成的项目，确保我在自述文件中解释了原因

或者，

我只是删除它们，从不回头。
