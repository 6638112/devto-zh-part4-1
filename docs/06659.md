## [](#digitalocean)数字海洋

我喜欢 DigitalOcean，因为他们在强大的云托管平台和易于使用的工具之间做了出色的工作。作为一个多年来也使用过亚马逊网络服务、谷歌云和微软 Azure 的人，我真的开始欣赏云产品的简单性。

像 AWS 和其他服务可以做的比数字海洋多得多，但实际上，你现在不需要它！见鬼，如果我们愿意，Next Tech 甚至可以在 DigitalOcean 上运行它的整个网站(现在我们在 Google Cloud 上)。所以它非常强大。

[![Alt Text](../Images/042933e2429076d3a493fb31a4d46ef1.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XnqGtprh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/sotxrd1nv4pyu5jaus8f.png)

## [](#heroku)Heroku

我第一次使用 Heroku 是在几年前，当时我开始编写 web 应用程序。它甚至比 DigitalOcean 更容易获得您的第一个应用程序，所以当您刚开始使用它，并且只想将您的想法放在那里(到云中)时，它是一个很好的资源。

在我看来，最终过渡到像 DigitalOcean 这样的东西是好的，因为如果你这样做，你会获得更多运行自己的服务器的经验。但是对于仅仅是为一个班级或作为第一个网络应用程序建立一个项目来说，Heroku 是很棒的！

[![Alt Text](../Images/12b1d722b6f96c496772742d8d25c24e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ODElbSY_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7ghrev5tywgj3fa1m4gh.png)

## [](#sendgrid)发送网格

这是我们在 Next Tech 用来发送电子邮件的工具。它允许你发送大量的电子邮件，这在给新用户发送确认邮件时非常方便。如果你只是使用个人电子邮件，你最终会受到费率限制，需要像 SendGrid 这样的工具来解决这个问题。

你也可以在他们的网站上使用 SendGrid 的电子邮件模板设计器，这可以让你的电子邮件看起来更专业，而不需要你付出太多的努力。一旦你完成了，你就可以使用 SendGrid 的 API 来发送这些邮件，这使得它非常灵活，也让你获得了一些很好的 API 体验！

[![Alt Text](../Images/4c3a783fdc71dfca05e9d0410dd75c9e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vQNhFL6T--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/awz6w4n1o9ku5br7kk5b.png)

## [](#sentry)哨兵

Sentry 会在应用程序的用户出错时提醒您，然后为您提供有用的调试信息来修复错误。它们还对错误进行了有用的分组，这可以帮助您了解哪些是您需要关注的最重要的错误。

我以前没有使用过 Sentry，但是我们使用了一个非常类似的工具(Bugsnag)来监控应用程序中的错误。当我第一次开始构建 web 应用程序时，我不太了解错误监控，但是在开始使用它之后，你将永远不会停止。

正如你在上面看到的，这个包每月为你提供 500，000 个事件，其中每个“事件”都是一个被触发的错误。对于你在学校的时候所做的，这已经足够了！

## [](#termius)白蚁

Termius 合并了您在 Linux、OS X、Windows、iOS 和 Android 上的 SSH 会话。如果您还不知道，SSH 代表“安全外壳”，允许您使用密码或私钥远程访问您的电脑(推荐)。

我还没有使用过 Termius(我昨天才知道，当他们被添加到包中时)，但是我计划尽快使用它。我用另一个工具从我的手机上做类似的事情，但这个看起来可能会更好！
