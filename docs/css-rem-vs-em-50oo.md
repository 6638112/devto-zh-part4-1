# CSS: rem vs. em

CSS 中的尺寸单位在它们工作的时候可能会令人困惑，而当它们不工作的时候就是一场噩梦。在快速眼动、快速眼动、像素、点数、百分比、vw & vh，甚至是看似无用的 ch 之间，让你的头脑思考 CSS 单位最好用于什么是很棘手的。在这篇文章中，我将比较 rem 和 em 单位，以帮助你清楚地知道应该如何使用它们。

### **EM 如何工作**

EM 是一个代表“元素”的相对单位，它的值严格依赖于最近的定义了字体大小的父容器。这种行为类似于百分比的工作方式，但在较大的值时，em 往往会大大超过百分比，而在较小的值时，em 会明显小于其对应的百分比。

EM 听起来很棒，但它们最大的问题之一是它们带来的级联效应。例如，一个字体大小为. 5em 或 2em 的外部 div 会将它的子元素分别更改为 em 的一半或两倍。这意味着字体大小为. 5em 的 3 个 div 相互嵌套，其值将分别为祖先元素大小的. 5、. 25 和. 125。在大多数情况下，仅仅这一点就让与 EM 一起工作变得异常痛苦。

### **多雷姆**

就像过去 20 年里的每一项网络技术一样，当前技术中最微小的变化都会带来新的东西来解决它。这就是 REM 的用武之地！REM 代表“根元素”，它的值基于最高级的父元素 font-size。最常见的 html 或 body 元素的字体大小。

这里最大的优点是，所有使用 rem 的尺寸都是基于 body 字体大小的，这使得在未来通过简单地改变一个值来自动缩放一切变得非常容易。

浏览器给出的字体大小最常见的值是 16px，并且很容易忘记不是浏览器给出的每种样式都应该被覆盖，特别是正文字体大小。将所有你想要的字体大小除以 16px 是很困难也很烦人的，所以我们使用了一个技巧。

```
body {

font-size: 62.5%;

} 
```

62.5%是个神奇的数字。为什么？因为 16px * 62.5% = 10。通过有效地将根字体大小设置为 10，我们想要的 24px 的 h1 就变成了 2.4rem。这个技巧特别棒，因为它使你的站点对于那些增加浏览器字体大小的用户来说非常容易访问。例如，将 Chrome 中的字体大小从标准的 16px 增加到 32px，会相应地改变所有站点元素的大小！

### **EM 有用的时候**

在阅读了快速眼动的神奇之处后，你可能想知道你什么时候还会用到它们。虽然 REM 通常比 EM 有用得多，而且副作用也少得多，但在某些调整大小的情况下，例如当您需要文本值彼此之间有特定的关系时，EM 仍然有其一席之地。
