# WTF 是服务还是守护进程？

守护进程是一种在后台运行的非交互式进程。它们基本上脱离了键盘和显示器。

服务也不需要终端。它们在后台运行，提供一些类似 apache 的服务。

## 一切都很好，但我们能做什么呢？

我想给你们介绍一下 *`systemctl`。*那是什么？这是一个超级酷的程序，用来检查和控制你的机器上运行的服务。说到服务 *`systemctl`* 就是你的朋友。

你可以用 *`systemctl`做很多事情。*我们将讨论最著名的用例。如果你想知道更多，你一定要访问*人*页面。如果你不知道什么是 *man* pages。快速看一下我的第二个条目。

这里的服务是命令的目标。基本上就是你想用 *`systemctl`管理的服务。*

命令呢？有一个您可以执行的命令列表。

| 命令 | 它有什么作用？ |
| --- | --- |
| 状态 | 返回服务的状态 |
| 开始 | 尝试启动服务🙏 |
| 停止 | 停止服务 |
| 重新开始 | 重新启动服务 |
| 再装 | 重新加载服务的配置。这将应用新配置。 |
| 使能够 | 启动时启用自动启动 |
| 使残废 | 禁用启动时自动启动 |

# 服务的状态

现在，让我们看看每个服务可能具有的不同状态。

| 状态名称 | 状态描述 |
| --- | --- |
| 装载的；子弹上膛的 | 服务的配置文件已成功加载 |
| 活跃的 | 服务正在运行或等待某个事件运行。 |
| 不活动的 | 该服务当前没有运行 |
| 使能够 | 该服务将在启动时启动 |
| 有缺陷的 | 该服务不会在启动时启动 |
