# 如何超越默认帮助我了解酷的东西

> 原文：<https://dev.to/myterminal/how-looking-beyond-defaults-helped-me-learn-about-cool-stuff-4lg0>

*【这个故事最初发布在[我的个人博客](http://ismail.teamfluxion.com/diary/20190712/How_looking_beyond_defaults_helped_me_learn_about_cool_stuff)*

在当前这个技术饱和的时代，几乎每项任务都可以用接近无限多种方式来执行，有数百种工具可供选择。在过去的几年里，我培养了一个新的爱好，那就是比较工具，了解它们，并尝试将它们引入我的工具集，以取代现有的工具。诚然，我认识的许多人宁愿把宝贵的时间花在社交媒体上辩论政治事务，在社交信息群上分享随机视频或疯狂观看无用的电视连续剧上，但嘿，每个人都可以自由地把时间花在他们想做的事情上，我也是！

违约是有原因的。有时他们会在那里推广某个特定的产品，比如将 [Internet Explorer](https://en.wikipedia.org/wiki/Internet_Explorer) 作为[微软 Windows](https://en.wikipedia.org/wiki/Microsoft_Windows) 的默认浏览器，或者是由于赞助商，或者是更好的选择，比如将 [Ubuntu](https://ubuntu.com) 从 [bash](https://en.wikipedia.org/wiki/Bash_(Unix_shell)) 切换到 [dash](https://en.wikipedia.org/wiki/Almquist_shell#dash:_Ubuntu,_Debian_and_POSIX_compliance_of_Linux_distributions) 作为默认外壳。如果你是那种宁愿使用简单有效的东西，也不愿花费几个小时(几天、几周甚至几个月)的宝贵时间从越来越多的选项中选择一个的人，特别是如果你相信在一天结束时你选择哪个并不那么重要，那么也可以使用默认值。

这篇文章提到了一些默认设置，我用自认为更好的替代方法替换了它们，并在这个过程中学到了很多，文章几乎是按时间顺序列出的。

## Windows Mobile 掌上电脑超过诺基亚智能手机

在我出生的那个时代，手机不仅仅是运行 WhatsApp、T2、抖音和 T4 PUBG 的平台。它们曾经是人们可以用来打电话或发短信的小玩意，在 21 世纪初，人们可以运行终端、编写代码和运行网络服务器。

[诺基亚](https://www.nokia.com)曾经是“手机品牌”,尽管与当时手机市场上的其他竞争品牌相比，它的产品线有点贵。从诺基亚的 50 系列、[系列 30](https://en.wikipedia.org/wiki/Series_30) 和[系列 40](https://en.wikipedia.org/wiki/Series_40) 到[系列 60](https://en.wikipedia.org/wiki/S60_(software_platform)) 、[系列 90](https://en.wikipedia.org/wiki/Series_90_(software_platform)) 和[系列 70](https://en.wikipedia.org/wiki/Series_70_(software_platform)) ，到 [Windows Mobile 5](https://en.wikipedia.org/wiki/Windows_Mobile_5.0) ，对我们来说是一次充满学习的‘旅程’。 [Windows Mobile](https://en.wikipedia.org/wiki/Windows_Mobile) (不是 [Windows Phone](https://en.wikipedia.org/wiki/Windows_Phone) )并不面向主流人群，除了来自 [HTC](https://www.htc.com) 的少数机型面向普通观众。想象一下，你需要进入“控制面板”来更改铃声，而不是从一个简单的菜单中切换智能手机上的“通用”、“振动”和“静音”等配置文件。

拥有一个超级强大(但还不成熟)的移动平台作为日常驱动不仅有趣，在某种意义上也是一种授权。与今天的趋势相比，硬件听起来太弱了，处理器运行在几百 MHz，大约 32 到 64 MBs 的 RAM 和类似的内部存储容量，使用 [QVGA](https://en.wikipedia.org/wiki/Graphics_display_resolution#320_%C3%97_240_(QVGA)) 到 [VGA](https://en.wikipedia.org/wiki/Graphics_display_resolution#640_%C3%97_480_(VGA)) 分辨率[电阻触摸屏](https://en.wikipedia.org/wiki/Resistive_touchscreen)显示器。在移动互联网(和 WiFi)普及之前，能够在 Outlook Mobile 上同步电子邮件，使用 [Microsoft ActiveSync](https://en.wikipedia.org/wiki/ActiveSync) 在计算机和[Pocket PC](https://en.wikipedia.org/wiki/Pocket_PC)之间传输数据，安装新版 Windows Mobile(包括从 [XDA Developers](https://www.xda-developers.com) 等开发人员论坛下载的定制 rom)，创建并加入 WiFi 网络，以便在我们的设备之间进行 FTP 和 HTTP 传输，所有这些都太棒了。

## 谷歌浏览器上的谷歌浏览器

回到 2012 年初，我发现很难理解为什么一个人会在自己的电脑上安装第三方网络浏览器，而这个星球上几乎每台个人电脑(假设运行 Windows)都预装了微软的 Internet Explorer。当然，这是一件很酷的事情，并且在 IE 之前就提供了标签式浏览方式，但是为什么要这么麻烦呢？

在获得我的第一个前端中心软件项目任务(因为使用 IE 而被我的团队取笑)后的年中，在快速意识到它对用户和开发人员的影响后，我不得不切换到 [Google Chrome](https://www.google.com/chrome) 。Chrome 中的开发工具要好得多(我甚至不知道 web 浏览器中有这种东西，只需按 F12 就能在屏幕上显示出来)，Chrome 27(如果我没记错的话)当时比 IE9 领先好几光年。我还在 Linux 上尝试了一段时间 Chromium，因为它在包存储库上的可用性更好。

今天，Chrome(在撰写本文时版本为 75)不仅仅是一个网络浏览器，它还有很多不同的用途，比如作为即时 JavaScript [REPL](https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop) 、网络审计工具、屏幕截图应用，甚至在一定程度上是文本编辑器的竞争对手。

## Windows 上的 Linux(和 Mac)

考虑到我们日常工作所依赖的应用程序的数量，我来自一个几乎所有电脑都运行 Windows 的 [Windows](https://en.wikipedia.org/wiki/Microsoft_Windows) 背景，切换到 [Linux](https://en.wikipedia.org/wiki/Linux) 非常可怕。最大的障碍包括 [Visual Studio](https://en.wikipedia.org/wiki/Microsoft_Visual_Studio) 、 [GoldWave](https://en.wikipedia.org/wiki/GoldWave) 、 [Nero](https://www.nero.com) ，以及一堆只能在 Windows 上运行的游戏。

除了最初的犹豫之外，我终于在 2013 年初做出了改变，Linux 很快成为我新的魅力，也是我计算的重要部分。我开始探索各种发行版，比如 Ubuntu、 [Kubuntu](https://kubuntu.org) 、 [Xubuntu](https://xubuntu.org) 、 [Lubuntu](https://lubuntu.net) 、 [Linux Mint](https://linuxmint.com) 、 [Debian](https://www.debian.org) 、 [NetRunner](https://www.netrunner.com) 、 [Elementary OS](http://www.elementarynow.com) 、 [Korora](https://kororaproject.org) 、 [Zorin OS](https://zorinos.com) 、 [openSUSE](https://www.opensuse.org) ，还有一些比我 [XFCE](https://www.xfce.org) ， [LXDE](https://lxde.org) ， [LXQT](https://lxqt.org) ， [Gnome 2](https://en.wikipedia.org/wiki/GNOME#GNOME_2) ， [Gnome Shell](https://www.gnome.org/gnome-3) ，[万神殿](https://wiki.archlinux.org/index.php/Pantheon)，[肉桂](https://en.wikipedia.org/wiki/Cinnamon_(Desktop_Environment))， [Mate](https://mate-desktop.org) 等等，关于[窗口管理器](https://en.wikipedia.org/wiki/Window_manager)就像[打开盒子](http://openbox.org/wiki/Main_Page)，[启蒙](https://www.enlightenment.org)， [相比之下，在](https://i3wm.org)[Windows 3.1](https://en.wikipedia.org/wiki/Windows_3.1x)[Windows 95](https://en.wikipedia.org/wiki/Windows_95)、 [Windows 98](https://en.wikipedia.org/wiki/Windows_98) 、 [Windows XP](https://en.wikipedia.org/wiki/Windows_XP) 或 [Windows 7](https://en.wikipedia.org/wiki/Windows_7) 中工作，显然根本没有桌面环境或窗口管理器的透明概念(或灵活性)。别忘了，每隔几周尝试不同的 Linux 发行版而不丢失所有数据是在 Linux 世界的另一个好处。

我目前最喜欢的工作站配置是运行 i3 的 [Fedora LXQT](https://spins.fedoraproject.org/en/lxqt) ，它主要涉及在运行 [Mac OS Majove](https://www.apple.com/macos/mojave) 的 [MacBook Pro](https://en.wikipedia.org/wiki/MacBook_Pro) 上的[术语](https://www.enlightenment.org/about-terminology.md)、 [Emacs](https://www.gnu.org/s/emacs) 、 [PCManFM](https://wiki.lxde.org/en/PCManFM) 和谷歌 Chrome[虚拟机](https://en.wikipedia.org/wiki/Virtual_machine)中进行切换。

## Emacs over Visual Studio

在我还是一名[网络](https://dotnet.microsoft.com)开发者的时候，Visual Studio 曾经是我电脑上最常用的应用，也是我继续使用 Windows 的一个重要原因。即使在我从 Windows 切换到 Linux 之后，我也不能放弃 Visual Studio，并且习惯于使用运行 Windows 7 Professional 的虚拟机来进行开发。那是在我从工作中的一次随机谈话中被介绍到 Emacs 之前，那次谈话改变了我的开发(和非开发)工作流程的一个重要部分。

Emacs 接管了我工作时的工作站和家里的个人电脑，除了谷歌 Chrome 和一个终端，它仍然是我电脑上使用最多的软件之一。我最终为 Emacs 创建了[几个](https://stable.melpa.org/#/?q=myterminal) [包/插件](https://marmalade-repo.org/profile/myTerminal)，创建了[几个](https://github.com/myTerminal/super-emacs) [启动包](https://github.com/myTerminal/ample-emacs)，写了几篇博客，在 GitHub 上创建了[一个学习报告，在 YouTube](https://github.com/myTerminal/world-of-emacs) 上创建了[几个视频，在我几乎所有的工作场所进行了](https://www.youtube.com/user/tfIsmail)[几次演讲](https://github.com/myTerminal/presentations/tree/master/why-use-emacs-in-2017)，我不知道有多少同事对此感到厌烦。

## ConEmu/Hyper over 命令行提示

众所周知， [Windows 命令提示符](https://en.wikipedia.org/wiki/Cmd.exe)可能是最无聊的毫无特色的终端。在 [Windows 10](https://en.wikipedia.org/wiki/Windows_10) 中，它确实有了很大的改进，但像复制和粘贴这样简单的事情直到最近才直接可用，更不用说更好的命令历史、自动完成等功能了。

在使用过各种 Linux 终端之后，我发现命令提示符根本不可用，甚至连运行一个命令都不行。从搜索替换 [Guake](//guake-project.org) 开始， [ConEmu](https://conemu.github.io) 是一个相当不错的选择，它不仅有下拉终端，还有多标签等功能。我记得有人在工作中在 ConEmu 上有一个带有终端的平铺布局，但我从未探索过这个特性。

Hyper 是一款全新的终端，完全基于 web stack，感觉新鲜，反应灵敏，可以通过配置文件和许多插件进行定制，这些插件类似于 T2 NPM T3 的软件包。Hyper 是我的 Mac 电脑上的主要终端，但我在工作中无法让它在我的 Windows 工作站上工作得这么好，于是又回到了 ConEmu。我使用 Fedora 上的术语，它是另一个很棒的 Linux 终端，具有极快的响应速度、很酷的定制等。

## 主流桌面环境上的 i3wm

在我电脑上的 Linux 发行版曾经附带的所有桌面环境中，KDE 是我用得最多的一个。在其他选项中，它可能是功能最丰富的，它一直被称为最慢的环境，直到最近，他们通过[版本 5](https://kde.org/announcements/plasma5.0) 对其速度和效率进行了重大改进。我对快速桌面环境的追求使我经历了各种选择，如 XFCE，它很快，但不太漂亮，LXDE 太简单，不能用作我的主要环境，Pantheon 附带了基本的操作系统，但我的系统有问题，特别是在足够稳定以供使用方面，Enlightenment 很酷，很快，但不是没有错误。

在尝试了所有我能尝试的堆叠窗口管理器之后，我了解到 T2 的平铺窗口管理器是我一直在寻找的，经过一些研究和试验，我选择了 i3。在通常的初始摩擦之后，i3 对我来说感觉如此自然，它几乎就像是 Emacs 之外的 Emacs。它不仅帮助我实现了快速设置，还使我能够完全用键盘快捷键操作我的计算机，就像我在 Emacs 中做的一样。这也帮我治疗了右手的[腕管综合症](https://en.wikipedia.org/wiki/Carpal_tunnel_syndrome)，这种症状(在我写这篇文章的时候)已经持续了七年多。

## WhatsApp 上的电报/IRC

自从 WhatsApp 流行起来，取代了所有的沟通方式，包括通常的语音通话和短信，在一定程度上取代了电子邮件，它就一直困扰着我。围绕这款应用的想法很棒，但它的用户将它视为一种宗教，它成功地吸引了非常广泛的受众，比我见过的任何其他数字平台/服务都要广泛。在我的家乡孟买，WhatsApp 是负责让人们通过手机获得互联网服务的平台，在那里，互联网被认为是一种奢侈品，不是每个人都可以使用互联网，至少在他们的手机上。人们失去了理智，我听说过类似“不使用 WhatsApp 的人首先就不需要智能手机”这样的观点。有人甚至告诉我，我不需要手机，因为我不使用 WhatsApp，我希望他们会问我用手机做什么，我会如实回答:“使用[待办应用](https://www.rememberthemilk.com)管理我的任务，提高工作效率”。

抛开对 WhatsApp 的憎恨不谈，我在日常工作流程中确实有一个消息平台，那就是 [Telegram](https://telegram.org) ，我用它来进行一些简单的交流和更新，但很高兴，我没有沉迷于它。我是为非常具体的主题创建的极少数群组的一部分，不会收到不相关的垃圾邮件。我确实尝试过 [IRC](https://en.wikipedia.org/wiki/Internet_Relay_Chat) 和一些类似 [Riot.im](http://riot.im) 的版本，但是效果不太好，尽管我会继续尝试。

## 鱼战胜鱼

从只涉及鼠标点击的工作流开始，到主要涉及键盘快捷键的工作流结束， [shell](https://en.wikipedia.org/wiki/Shell_(computing)) 无疑是我的设置中的一个重要部分。以前，我花了很多时间围绕快速切换到终端、在选项卡之间移动、隐藏选项卡等功能来寻找和比较界面和功能。我最近重新阅读了一些不同的 shell，很少阅读，就可以很容易地找到人们转而使用不同 shell 的文章。

了解到 bash 至少有两个主要选项: [zsh](https://en.wikipedia.org/wiki/Z_shell) 和 [fish](https://fishshell.com) ，它围绕两者之间的差异以及我可以将哪一个引入到我的工作流中展开了另一项深入的研究。考虑到特性和可定制性之间的平衡，我决定改用 fish-shell，至少目前是这样，但很快我发现我的 bash 配置不能直接用于 fish。这在我的[点文件](http://github.com/myTerminal/dotfiles)中开始了另一个长期运行的学习/开发练习，其中一部分涉及到将我的可重用 shell 配置从 bash 配置中分离出来，这样我至少可以将我已经可用的 shell 别名用于 fish。撰写本文时，大规模更新仍在进行中。很快，我将在我所有的工作站上完全切换到 fish，并且能够定制它，我可能最终会学到相当多的 bash！

在这一点上，我不会介意我永远不切换到 fish(甚至不使用它),因为我已经了解了很多关于 bash、其他 shells 以及它们的存在。
