# 这不是你要找的开发商吗？

> 原文：<https://dev.to/jjtowle/this-is-not-the-developer-you-are-looking-for-4kdm>

我希望你不介意我一吐为快。为这看似微不足道的咆哮道歉。

这些年来，我的收件箱里塞满了不少招聘广告，昨天又来了一条，其中有一条**应该有**，我觉得这让我有些厌烦？

这种 **should** 似乎表明，如果你参加聚会、有博客或为 stackoverflow、github 等人做贡献，你只能将自己归类为一个有能力、有价值的开发人员(编码员、程序员、软件工程师，在此插入你最喜欢的工作描述)。基本上，我觉得这表明你需要为编码而活、呼吸并准备为编码而死。因此，生命没有别的东西可以提供。

不可否认，这是一个相当夸张的说法，让我澄清一下，我 100%支持上面列出的活动。这更像是招聘广告上的建议，你只能做这份工作，如果你也经常做这种活动的话。我在 Github 上只有几个存储库，在 Stackoverflow 上没有任何活动，公平地说，我对“社区”的贡献非常有限，在 Dev.to 上(2 个帖子包括这个帖子和对其他帖子的奇怪评论)。

我也有过一次工作面试，感觉面试官几乎是想让我对工作之外的“男人”代码印象深刻。再次重申，我完全支持这一点。我当然会努力找一些空闲时间来保持我的技能与时俱进，学习新的东西，这样我就可以在工作中用到它们。我只是发现这有点令人倒胃口，我想这是因为如果我面试一个开发人员的角色，我也会对他们感兴趣，当他们不在计算机前开发下一个开创性的框架/库时。我需要知道他们能拔掉插头。

我不确定这是否值得再写一篇，但这是不是业内的一种精英主义问题？如果你不是全天候被劫持，你就不酷？

我只想以一个小小的免责声明来结束我的演讲，以确保我自始至终都很清楚，我并没有质疑任何人在业余时间选择做什么。我们每个人都做对自己有益的事情，做让我们最快乐的事情，我希望你们都渴望这样。我只是质疑我们是否适合某个角色，是否应该以我们在工作环境之外的所作所为来衡量。

#### 咆哮结束
