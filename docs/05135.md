# C#项目创意 100+项目创意

> 原文:[https://dev.to/samborick/100-project-ideas-oda](https://dev.to/samborick/100-project-ideas-oda)

**编辑**:大家好！在这篇文章引起了惊人的反响后，我创建了一个叫做每周项目俱乐部的东西。每周你的收件箱里都会有一个需要解决的问题。你可以努力解决这个问题，你会得到整个俱乐部的帮助，让你保持在正轨上。了解更多并报名[这里](https://weeklyproject.club)！

前几天我注意到一个模式。我注意到很多人试图学习编程，但是他们头脑中没有一个明确的目标。我已经谈到了理解你为什么想学习编程可以帮助你选择学习什么语言[就在这里！](https://pickaframework.com/articles/why/)，以及如何实际做出那个决定([过来！](https://pickaframework.com/feature_fishing/))但是一个项目有什么帮助呢？

当我指导程序员时，我发现有一个项目可以帮助排除一些其他干扰，比如想知道你是否使用了正确的语言。通过专注于一个特定的目标，你就没有那么多精力去担心*这个*是不是你应该使用的语言。结果是你构建了一些简洁的东西，并且你在这个过程中学到了一些东西！一石二鸟。

这就是为什么我为初级程序员策划了这个项目列表。许多人列出了大量学习编程的项目，但很少按照难度进行组织。我已经浏览了几个流行的编程项目想法列表。如果你想看完整的列表，你可以在页面底部找到来源。

我已经把它分成了教程和想法。教程包括一个资源链接，而创意只是对项目的一般描述。我还列出了我最喜欢的初学者列表。

看看吧，看看有没有什么启发你的！

## [](#easy)容易

### [](#99-bottles)99 瓶

*   创建一个程序，打印出歌曲“墙上的 99 瓶啤酒”的每一行
*   不要对所有号码使用列表，也不要手动键入所有号码。请改用内置函数。
*   除了“拿下一个”这句话，你不可以直接在你的歌词中输入任何数字/数字名称。
*   记住，当你还剩 1 个瓶子时,“瓶子”这个词就变成了单数。

### [](#magic-8-ball)魔法 8 球

*   模拟一个神奇的 8 号球。
*   允许用户输入他们的问题。
*   显示正在进行的消息(即“思考”)。
*   创建 20 个回答，并展示一个随机回答。
*   允许用户问另一个问题或退出。
*   奖金:

    *   添加一个 gui。
    *   它必须有一个用户输入问题的框。
    *   它必须至少有 4 个按钮:

        *   要求
        *   清除(文本框)
        *   再玩一次
        *   退出(这必须关闭窗口)

### [](#rock-paper-scissors-game)石头剪刀布游戏

*   创建一个石头剪子布游戏。
*   请玩家选择石头、布或剪刀。
*   让计算机选择它的移动。
*   比较选择，决定谁赢。
*   打印结果。
*   子目标:
    *   给玩家重新游戏的选项。
    *   记录分数(如玩家:3 /电脑:6)。

### [](#countdown-clock)倒计时钟

*   创建一个程序，允许用户选择时间和日期，然后以给定的时间间隔(例如每秒)打印出一条消息，告诉用户离所选时间还有多长时间。
*   子目标:
    *   如果选定的时间已经过去，让程序告诉用户重新开始。
    *   如果你的程序需要年、月、日、小时等等。另外，允许用户能够键入月份名称或其编号。
    *   提示:利用内置模块，如时间和日期时间，可以将这个项目从一个噩梦变成一个简单得多的任务。

## [](#medium)中等

### [](#pomodoro-timer)番茄定时器

创建一个番茄定时器。

番茄定时器是一种时间管理方法。这种技术使用一个计时器将工作分成间隔，传统上长度为 25 分钟，由短暂的休息分开。这些间隔被命名为 pomodoros，意大利语单词 pomodoro(番茄)的英文复数，以西里洛在大学时使用的番茄形状的厨房定时器命名。
原始技术有六个步骤:

决定要完成的任务。设置番茄定时器(传统上为 25 分钟)。
处理任务。
计时器响起时结束工作，在一张纸上打上勾号。
如果您的复选标记少于四个，请稍事休息(3-5 分钟)，然后转到第 2 步。
在吃了四个番茄之后，休息一段时间(15-30 分钟)，将你的对勾计数归零，然后进行第一步。
想了解更多关于番茄定时器的信息[点击这里](https://en.wikipedia.org/wiki/Pomodoro_Technique)

    *   句子可以输入任意数量的空格。

### [](#dice-rolling-simulator)骰子滚动模拟器

*   允许用户输入骰子的边数和掷骰子的次数。
*   你的程序应该模拟掷骰子，并记录每个数字出现的次数(这不必显示)。
*   最后，打印出每个数字出现的次数。
*   子目标:
    *   调整你的程序，如果用户在需要的时候没有输入一个数字，程序会一直提示他们输入一个实数，直到他们输入为止。
    *   将程序放入一个循环中，这样用户就可以继续模拟掷骰子，而不必重新启动整个程序。
    *   除了打印出每一面出现的次数，还要打印出它出现的百分比。如果可以，将百分比四舍五入到总共 4 位数或两位小数。
*   奖金:
    *   你将要玩一个棋盘游戏，但是你意识到你没有骰子。幸运的是你有这个程序。
    *   1.创建一个程序，打开一个新窗口，并绘制 2 个六面骰子
    *   2.允许用户退出，或再次滚动
    *   允许用户在屏幕(1-4) 2 上选择要抽取的骰子数量。将骰子的总数加起来并显示出来

### [](#count-and-fix-green-eggs-and-ham)点清绿鸡蛋和火腿

你们中的一些人可能还记得苏博士的故事“绿鸡蛋和火腿”。对于那些不记得或从未听说过它的人来说，[这里的](http://pastebin.com/XMY48CnN)就是故事。但是，我给你讲的这个故事有一个问题——每次用到 I 这个字，都是小写的。
因为这个问题，你的工作是做好以下几点:

*   把我给你的故事复制到一个普通的文本文件中。
*   创建一个程序，阅读整个故事，并使字母 I 在任何时候都大写。(确保在 sam-I-am 的名称中使用时也进行更改。)
*   让你的程序创建一个新文件，并让它正确地写出故事。
*   打印出纠正了多少错误。
*   当你完成时，你应该已经纠正了[这么多](https://i.imgur.com/GRkj3yz.jpg)的错误。

## [](#hard)努力

### [](#random-wikipedia-article)随机维基百科文章

如果你去过维基百科，你可能会注意到在屏幕的左侧有一个随机文章的链接。虽然看到你看到的文章很有趣，但有时看到文章的名字会很好，如果它听起来很无聊，你可以跳过它。幸运的是，维基百科有一个 API 允许我们这样做[点击这里](https://en.wikipedia.org/w/api.php?action=query&list=random&rnnamespace=0&rnlimit=10&format=json)。
然而，这里有一个难题。由于维基百科有关于世界各地话题的文章，其中一些文章的标题中有特殊字符。例如，关于西班牙画家[伊拉斯托·科尔特斯·华雷斯](https://en.wikipedia.org/wiki/Erasto_Cort%C3%A9s_Ju%C3%A1rez)的文章中有é和á。如果您查看这篇特定文章的 [API](https://en.wikipedia.org/w/api.php?action=query&prop=info&pageids=39608394&inprop=url&format=json) ，您会看到标题是“Erasto Cort\u00e9s Ju\u00e1rez”并且\u00e9 和\u00e1 替换了前面提到的两个字母。(有关这是什么的信息，首先查看文档中的第[页](https://docs.python.org/2/howto/unicode.html)的前半部分)。为了让你的程序工作，你必须以某种方式处理这个问题。

*   创建一个程序，从官方维基百科 API 中提取标题，然后一个接一个地询问用户是否愿意阅读那篇文章。
*   示例:
    *   如果第一个标题是 Reddit，那么程序应该问一些类似“你想了解 Reddit 吗？”如果用户说是，那么程序应该打开文章让用户阅读。
    *   提示:点击[这里](https://en.wikipedia.org/wiki?curid=39608394)查看如何使用文章的 ID 来访问实际的文章。
*   子目标:
    *   如前所述，对标题中出现 unicode 的可能性做点什么。
    *   你是想让你的程序简单地过滤掉这些文章，还是想把代码变成可读的字符，这取决于你。
    *   一旦用户选择了要阅读的文章，让程序暂停，并允许他或她在完成阅读后继续浏览不同的文章标题。
    *   允许用户只需按下回车键就可以询问一篇新文章。

### [](#whats-the-weather)天气如何？

如果你想知道什么是 API 的基础知识，看看 iamapizza 的这篇文章。

*   创建一个程序，从 OpenWeatherMap.org 提取数据并打印出当前天气的信息，如最高、最低和你所居住地区的降雨量。
*   子目标:
    *   打印出未来 5-7 天的数据，这样你就有了一个 5 天/周的预测。
    *   将数据打印到另一个可以打开并查看的文件中，而不是在命令行中查看信息。
    *   如果你懂 html，写一个可以打印信息的文件，这样你的项目会更有趣。
*   小贴士:
    *   Json 中的 API 本质上是列表和字典。请记住，要引用列表中的某个内容，您必须通过它在列表中的数字元素来引用它，而要引用字典中的某个键，您必须通过它的名称来引用它。
    *   不喜欢摄氏？将&units=imperial 添加到 API 的 URL 末尾，以接收您的华氏数据。
