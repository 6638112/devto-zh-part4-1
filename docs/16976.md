# 如何在 Elm 中设计程序

> 原文：<https://dev.to/dwayne/how-to-design-programs-in-elm-1ecf>

我希望我知道。不好意思！再见！

玩笑归玩笑，如果你愿意花时间建立一个坚实的基础，我想我也许能引导你走向正确的方向。

在阅读了[计算机科学课程的结构和解释](https://cs.brown.edu/~sk/Publications/Papers/Published/fffk-htdp-vs-sicp-journal/)和[如何设计程序](https://htdp.org/)之后，我认识到**我们把领域知识和程序设计混为一谈，编程教得很差，我们从来没有明确地向学生展示如何设计程序**。

好酷，但是 [HTDP](https://htdp.org/) 是给初学者用的，它使用[球拍](https://racket-lang.org/) [教学语言](https://docs.racket-lang.org/htdp-langs/index.html)。那对我的榆树有什么帮助？

这是令人惊讶的事情。它之所以相关，是因为它教授的所有概念都可以 100%移植到任何编程语言，尤其是 Elm。

阅读“[一个图形编辑器](https://htdp.org/2019-02-24/part_one.html#%28part._sec~3aedit1%29)”来明白我的意思。

或者，阅读[如何设计一个对](https://htdp.org/2019-02-24/part_two.html#%28part._sec~3asort.I%29)进行排序的函数，看看设计方案是如何导致实现失败的。

事实上，阅读一下“[设计世界程序](https://htdp.org/2019-02-24/part_one.html#%28part._.D.K._sec~3adesign-world%29)，看看它和[浏览器](https://package.elm-lang.org/packages/elm/browser/latest/)有多相似。

我不喜欢这本书的唯一一点是它不在埃尔姆。

但是我的主张是**如果你想学习如何在 Elm 中设计程序，那么你应该阅读 [HTDP](https://htdp.org/)** ( *直到类似的 Elm 特有的东西出现*)。

最后，对于那些对编程教学感兴趣的人，我留给你们《马蒂亚斯·费勒森:第一年》。
