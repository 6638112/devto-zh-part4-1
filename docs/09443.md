# ZingSoft 团队如何使用 CloudApp 简化他们的媒体捕获工作流程

> 原文:[https://dev . to/zing grid/how-the-zing soft-team-uses-cloud app-streamlined-them-media-capture-workflow-5 aid](https://dev.to/zinggrid/how-the-zingsoft-team-uses-cloudapp-to-streamline-their-media-capture-workflow-5aid)

在 ZingSoft，我们一直在寻找可以使我们的工作更轻松的新工具，这也意味着我们已经尝试了许多许多不同的产品和服务。如果我不得不选择一个在过去一年中对我的团队的整体生产力最具变革性的工具， [CloudApp](https://www.getcloudapp.com/) 将是我最有可能获得该头衔的两个竞争者之一。也许听说媒体捕获应用程序可以对软件开发团队产生如此大的影响令人惊讶，但事实是它填补了我们没有意识到的空白——我们没有意识到它对我们的工作流程有如此大的影响。

事实证明，从软件开发人员到图形设计人员，再到 QA 测试人员，每个人都可能需要捕捉屏幕截图、GIF 或视频记录，来记录他们计算机屏幕上发生的事情，原因有很多。更重要的是，期望团队中的任何一个角色都想要注释和分享这些记录是完全合理的。进入 CloudApp，这是一个基于云的媒体录制应用程序，它不仅包括上述所有功能，而且还在云中存储所有录制的媒体(您注意到这里的一个主题了吗？)以便以后从任何桌面访问和检索。以下是 CloudApp 捕获的样子:

从表面上看，CloudApp 提供的功能似乎很容易被忽视。毕竟，截图本身很简单，现在许多基本的图像查看应用程序都有某种形式的基本注释性功能。然而，CloudApp 通过让捕捉、注释和共享运动图形和视频的过程像静态截图一样简单，使自己与我的团队脱颖而出。此外，它的非侵入性 UX 意味着该应用程序在我们需要时就在那里，但在我们不需要时几乎看不见。这种深思熟虑的功能和最小化的存在的结合最终使我的队友能够如此轻松地将其集成到他们的各种工作流中，而不管他们的角色是什么。

我们发现自己使用 CloudApp 的用例很多。他们从捕获我们构建的 JavaScript 库中的错误，到记录我们各种网站和应用程序中的怪异行为，到可视化地标记我们在网络上遇到的鼓舞人心的事情，到为我们的营销材料创建内容(就像这篇博客帖子中的截图)。然而，除了它的许多用例之外，CloudApp 的共享选项也是它对我们的无价之宝。我们可以通过拖放文件直接共享单个媒体，或者通过各种基于云的链接共享选项(例如，直接链接、下载链接、嵌入代码等)。我们还可以将录制的媒体直接分享到各种平台，如 Slack、Trello、Twitter 等。从 CloudApp 的界面。

在更高的层面上，CloudApp 使我的团队能够更容易地向现有和潜在客户直观传达我们的 [ZingGrid](https://www.zinggrid.com/) 和 [ZingChart](https://www.zingchart.com/) 数据可视化产品的价值。我们的两个 JavaScript 库主要是表示性的，拥有一个工具来直观地表达我们的产品能做什么非常容易，这是一种变革和节省时间的方式。由于 CloudApp 提供的帮助，制作产品教程、电子邮件活动、错误报告/故事和内部状态演示等内容变得比以往任何时候都更容易。
