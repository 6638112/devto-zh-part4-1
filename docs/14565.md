# 没有完美的系统，所有的都是权衡

> 原文:[https://dev . to/mooreds/不存在完美的权衡系统-17pb](https://dev.to/mooreds/there-is-no-perfect-system-it-s-all-about-the-tradeoffs-17pb)

亲爱的新开发人员:

我想建立完美的系统。一段完美的代码优雅而简洁地解决了一个问题，这真是太美妙了。尤其是如果它附带了一组单元测试和很好的文档。

但是，每当我开始解决任何现实世界的问题时，权衡就开始起作用了。让我们以一个简单的网站为例，该网站将显示一些关于企业的信息，并会偶尔更新。在考虑构建什么时，需要考虑一些技术权衡:

*   业务有多突出？
*   企业的品牌是什么？
*   多久更新一次？
*   网站需要多大？
*   企业想要达到什么目的(简单的在线展示？潜在客户/未来客户？降低客户服务请求频率？其他商业目标？)
*   谁将进行更新？他们有多精明？
*   预算是多少？
*   时间线是什么？
*   谁可以做这项工作？

每一个都会对你所建立的系统产生影响。一个当地珠宝商的网站，只是想有一个网络存在，显示商店营业时间和突出偶尔销售将是一种方式。一个报纸网站，需要定期更新，经得起偶尔的大流量高峰。然而，它们可能都是非常简单的网站。你需要找出目标是什么，这将告知你必须做出的权衡。

重点不是一个网站或技术比另一个更好，而是承认并确保每个人都在相同的页面上做出选择。(这些选择会随着时间的推移而演变。我最近在几个网站上工作，这些网站在当时是用非常好的技术实现的，但是现在已经过时了。)
