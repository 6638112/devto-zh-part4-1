# 软件开发人员的十诫。

> 原文:[https://dev . to/bensassialadin/the-ten-commands-for-software-developers-4j1m](https://dev.to/bensassialadin/the-ten-commandments-for-software-developers-4j1m)

有几条规则是每个开发人员都应该遵守的，并且永远不要违反。医生、律师、法官有他们的誓言，但作为开发者，他们没有这些东西，我们至少应该有一个约束我们的道德准则。所以我决定采取主动，为软件开发人员写下我认为应该遵守的十诫:

你不应该撒谎:无论你是过度承诺、谎报自己不了解的技术知识，还是直截了当地谎报自己完成任务的能力，撒谎不仅会伤害你的客户，还会损害你的声誉。我不知道你怎么想，但比起一次性付款，我更看重我的名声。

**你不应该偷窃:**偷窃不仅仅是以偷钱的形式出现，它可以是任何事情，从在你的客户不知情的情况下在你编写的项目中保留后门，到从你的机构或同事那里挖走客户。

**你不应该漠不关心:**漠不关心是一种诅咒，它摧毁了一个人的创造能力，其结果是，工作会变慢，产量充其量也只是平庸。

**你不应故意伤害客户:**客户是指那些指望我们每天都全力以赴，并相信我们会把他们的最佳利益放在心上的人。因此，我们应该努力赢得这种信任，并将其发展为相互尊重。

不应伤害同事:同事也是我们不应以任何理由故意伤害的人。作为开发人员，我们建立了一个乐于助人、友好的社区，就像一个大家庭。一个家庭*会照顾好自己。*

尊敬你的导师和导师:教我们手艺的人，是我们应该永远感激的人。他们不仅花时间向我们提供他们学到的小信息，以便我们年轻的大脑能够消化它们，而且他们还在我们身上看到了一些他们认为值得他们投入时间的东西。

不要要价过高:和其他职业一样，开发人员有好有坏，有时这些糟糕的开发人员会向客户要价过高，只是因为他们不知道更好的东西，这比偷窃更糟糕，这是利用缺乏知识的人，愚弄他们。想象一下，如果一个医生多收了你奶奶的钱，你会有什么感觉？

你应该找到平衡点:我们都知道工作对我们来说有多重要，但我们都需要给自己留点时间，远离每天的压力。工作与生活的平衡是我们都应该努力的目标。

**你不应该交付未完成的工作:**作为开发人员，我们知道当我们构建的产品几乎没有功能时，虽然从外部看起来足够好，但内部却是一个不同的故事。永远不要交付那种项目，直到你确定它在你眼中是完美的。

**你应该传播知识:**尽你所能在任何地方以任何方式传播知识，向每一个年轻的开发人员和有需要的同事开放你的思想、你的源代码和你的臂膀。

十诫到此为止，现在去征服我的孩子吧。
