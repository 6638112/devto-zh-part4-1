# 在 GDevelop 中制作游戏

> 原文：<https://dev.to/triptych/making-a-game-in-gdevelop-8cp>

我刚刚发现了 GDevelop，虽然我认为我在以前的版本中见过这个引擎，并认为“它还没有完全成熟，我稍后会检查它”，但直到几周前我想起它时，我才忘记了它。这就像 Construct 2 和其他“动作表”一样，你不需要编写代码，更像是为某些事件设置监听器，然后操纵预先构建的对象来制作游戏。

GDevelop 本质上是一个用 C++编写的混合应用，但逐渐移植到 ReactJS 和 Webassembly 上。看看作者的这个[非常有趣的幻灯片](https://slides.com/florianrival/native-web-apps#/)，它讲述了应用程序的转变。

对我来说，一年多来我一直在尝试开发一个游戏制作工具，我也学到了很多关于如何制作一个应用程序的知识。Godot 引擎是制作游戏的优秀工具，无论是 2D 还是 3D 游戏，它甚至有足够的功能来创建其他类型的应用程序。

我甚至正在写一本关于在 Godot 中制作应用程序的书，稍后会有更多的介绍。今天的主题是 GDevelop。

我决定尝试其中一个教程，看看在 GDevelop 中创建一个简单的游戏会有多复杂和可行，我不得不说这非常好。

如果你想跳到游戏部分——在 itch.io 查看我的[演示游戏](https://godotengine.org/)。

本质上，我实现了这个演示。这很简单，只有一些小故障。事实上，我在应用程序中发现了一个错误，提出了一个问题，并在 30 分钟内得到了回应。

显然 GDevelop 是由一个人在业余时间维护的。对于一个副业项目来说，这看起来相当不错，但老实说，我认为他们需要允许更多的人来贡献。

它似乎正在积极开发，几个月前刚刚添加了一些令人兴奋的功能。我喜欢这款应用的发展方向，我计划今年尝试在其中开发一款真正的游戏。

我在终端/校园电影节期间的目标是在#ittybittyrpg 上取得进展，并在 GDevelop 中获得地下城下午工作的基本演示。我甚至可能在我的主页上写一些教程。

因此，如果你是一个“非编码者”或者正在寻找一个有很好的框架来构建游戏的应用——这是开源和免费的——试试 GDevelop。
