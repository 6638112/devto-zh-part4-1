# 豪奢之态

[![HaxePunk August 2019](img/b1bb3d3ca56cf17ceac1866c3533efc9.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--N17MNVUc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/dsxq7cfxq94wwc2wypdy.jpg) 
说到更新，HaxePunk 一直缺乏可能已经不是什么秘密了。最初的维护者的时代已经过去，但是 HaxePunk 还活着；也许他们将来会回来。别担心，哈克斯庞克没死；正如你将看到的，它最近受到了很多关注。

如果你最近想尝试 HaxePunk，你可能会有几个问题。坦率地说，哈克斯庞克现在处于遗留状态。上一个发布的版本是在 Haxelib 上发布的，依赖于旧版本的 Lime 和 OpenFl。该版本是在以前的项目维护人员从 HaxePunk 退休之前发布的，因此距离发布任何修复已经有一段时间了。另一方面，我简单地推荐使用 HaxePunk 的 dev-branch，dev 分支在 5 月份复活了。虽然 HaxePunk 的 dev 分支已经更新，现在可以与最新的 Lime 和 OpenFl 一起工作，但 Lime 中有一个错误，它阻止了我们使用 WebGL 进行渲染。记住这一点。如果你需要 HTML5，使用兼容 Lime/OpenFL 的 Haxelib 版本。

所以，该不该接 HaxePunk？好吧。。。网站上需要澄清一些事情，但是 HaxePunk 一直都有可靠的文档，简单易用的工作流程，并且仍然是一个经过良好测试的引擎。你可以使用 dev 分支或者过去的版本来开发和发布游戏，这样就可以了。现在，我们正处于一个关键的过渡时期，试图给项目注入新的活力，这将引导我进入下一部分。。。

# 过渡的状态

如果你还没听说，最近有一个让 HaxePunk 再次相关的倡议。我们的任务是利用 Kha，而不是石灰，同时保持你所知道和喜爱的 API。我们这样做是为了简化 HXP 的工作流程，并获得 Kha 必须提供的几个好处:支持桌面、web、移动、控制台和本地图形抽象。。。经过编译的跨平台着色器语言、平台一致的输入等等。

我们在 6 月份就开始了这一雄心勃勃的转变，尽管过程中有一些坎坷，但我们仍在取得进展。我们目前有标准的输入工作和一个编译项目。这可能看起来不多，但它是我们继续下一步的完美起点。

尚未完全投入使用的功能有:渲染、音频、资产和配置管道，以及一些我们尚未看到的东西。从技术上来说，使用 HaxePunk 中一半或更多的功能是可能的，但是一个视频游戏如果没有视频就没有意义了。简而言之，HaxePunk 的核心没有被改动，我们重新连接了隐藏在核心 API 之外的底层内容。

在我们开始对这个即将到来的版本进行 beta 测试之前，要完成的任务数量少得惊人。这并不是说它还需要几个星期或几个月的时间才能进入 beta 状态，但是有一条足够直的路让我们去追寻(**非常感谢 Matrefeytontias** )。这可能不是一个完全准确的图片，因为剩下的大部分工作是渲染。Matrefeytontias 是我们的渲染员，所以要由他来挖掘。只是有很多细节需要考虑，这让 Lime 版本变得非常令人头疼。不过没必要担心！Matrefeytontias 对此很有信心，并且已经显示出有希望的结果，所以几周或几个月仍是可能的。也许我们会在新年前给每个人一份礼物。

我很想以一种注重体验的方式来介绍这个博客，而不是对我们工作的总结，但是当你忙起来的时候就会这样。我的下一篇文章将会有更多的试错/经验内容；这仅仅是我最大的努力来告诉你们发生了什么和正在进行的事情。

# 哈克的未来

我们在前进的道路上有很多雄心壮志！是的，我们可以继续做酸橙版本的。是的，我们也可以同时支持酸橙和 Kha。但是我们没有，为什么呢？我们为未来选择了 Kha，我们放弃了 Lime，因为 Kha 之后就没有必要了。它就像那样生硬，有一个很好的理由，但最重要的是，Kha 是便携的。由于 Kha 是一个“超便携多媒体框架”, HaxePunk 将拥有它以前所拥有的所有功能，并且覆盖范围更广。想象一下，哈克斯庞克是一名火车售票员，但后来决定成为一名飞机驾驶员——同样有能力，但目的地不限。因此，我们对 HaxePunk 的主要目标是让它经得起未来的考验，为它提供一个新的基础，让它按照我们的方式扩展，在几个现有平台和那些尚未为人所知的平台上变得强大和本地化。

我们现在正致力于使#KhaHaxePunk 版本变得可用，所以你可能想知道 HaxePunk 是否会有其他的东西出现。我应该提请注意这样一个事实，就目标而言，Matrefeytontias 和我之间有一点点不同。我们都专注于确保 HaxePunk 是可用的、稳定的和强大的。我们都同意，在过渡期间所做的任何改变都应该是最小的。我开始关注的是 KHA 港之后的特征和未来。在我们旅程的开始，涌现了许多想法，因为所有的大计划都制造了某种炒作，但这些已经从主画面中移除，我们将回来考虑它们的适当性。

尽管如此，我还是要让你体验一下我的想法，所以这里有一个我想带给 HaxePunk 的东西的列表(这些都没有被期望得到官方的支持，甚至没有被实现——它们只是分享起来很有趣的“酷”的想法)
