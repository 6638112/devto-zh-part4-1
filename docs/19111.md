# 重塑品牌的老人指南

> 原文:[https://dev . to/slawinski/an-old-man-s-guide-to-rebranding-3d6l](https://dev.to/slawinski/an-old-man-s-guide-to-rebranding-3d6l)

看来我的文章引起了你的注意。如果你想了解最新情况，请考虑订阅我的[时事通讯](https://slawinski.dev/newsletter/)。

你可能认为，为了在没有经验的情况下在开发领域找到一份工作，你需要知识和技能。不仅如此。你还需要勇气和运气。伙计，改变你的职业需要很大的勇气。尤其是当你处在一个可能已经建立了一个团队的年龄，而资深开发人员平均比你年轻五岁。请允许我向你展示并非所有的希望都落空了，以及如何增加你被聘用的机会。

## [](#getting-ready)做好准备

好吧！让我们假设你确信你想要学习编码，并有一个不远的目标，那就是成为一名专业的开发人员。下一步是什么？

一如既往，最困难的部分是开始。幸运的是，有了网络技术，你甚至不用离开你的沙发。你所需要的一切就在你的指尖——在被称为互联网的广阔空间里。但是信息的丰富是压倒性的。大量的文章、数小时的视频和数十门在线课程应该会让这个过程变得更容易，但通常情况恰恰相反。更不用说你会发现的东西的质量可能会有所不同。我应该参加 Udemy 上那个打折的课程吗？我应该从书上学习吗？教程，播客，或者新兵训练营呢？它包含好的实践还是只教授反模式？让我们进入实质吧！(哦，不要犯什么都想学的错误。选择一种语言并坚持下去！)

## [](#online-courses)在线课程

如果你不知道从哪里开始，或者你正在寻找一个特定领域的综合课程，或者你喜欢被人牵着手，这是一个很好的选择。通常不贵-你要么按课程付费，要么按月付费。哦，在 Udemy 上支付全价之前，请记住，你有时可以获得高达 90%的折扣。作为一个缺点，你可以考虑作为一个初学者，你可能会在从一门课程过渡到现实生活中遇到问题，如[移动过去教程](https://dev.to/aspittel/moving-past-tutorials-a-course-on-problem-solving-for-programmers-3oa4)中所述。

### [](#books)书籍

关于书籍的事情是网络开发世界变化非常快(与其他技术相比)。所以当一本书被写作、编辑、印刷，或者被翻译时，它突然变得不再相关了。但是当一本书是关于一个基本的东西:算法、设计模式等等。，你可以很容易地找到一本几十年前的书，仍然可以学到有用的东西(尽管对初学者来说不是)。我发现印刷书籍是一个很好的知识来源(我也喜欢新书的味道)，但我知道其他人可能更喜欢从屏幕上阅读 pdf。这取决于你。哦，我建议选择最新版本。

### [](#howtos)操作指南

互联网上到处都是，这意味着你需要知道要寻找什么。视频、文章、博客文章——应有尽有！似乎每一个主题，每一种可能的工具配置，都已经有人涉及并发布到网上了。如果你知道你想要创造什么，但是你被卡住了，这很有用。你可能会翻遍成吨的书，直到找到合适的那本。即使这样，也不能保证我们无耻地复制到项目中的代码能够工作。实话实说吧。我们都有过这样的经历——50%的代码来自 StackOverflow，50%来自其他人的项目。

### [](#podcasts)播客

如果你每天花几个小时做一些平凡的事情，那会很神奇。比如日常琐事或者朝九晚五的通勤。戴上耳机沉浸其中。背诵整块代码并不容易，所以播客大多关注新闻、工具、个人经历等。

### [](#boot-camps)新兵训练营

我对他们了解不多，但从我读到的来看，有好的，也有不好的。它们要花很多钱，所以如果你想走这条路，一定要小心。和几乎所有的事情一样，最好的方法是联系参加过新兵训练营的人，征求他们的建议。参加一个新兵训练营不是必须的，但是一个好的训练营肯定会促进你成为一名程序员。

### [](#live-coders)现场编码员

当我得知 Twitch 上有很多人在流媒体上工作时，我感到很惊讶。为什么有人会在意？实际上，看别人写代码可以教会你一些精致的教程所不能教会的东西——给定的程序员解决问题的方法。在照本宣科的教程或课程中，所有东西，甚至是 bug，都必须被有意地实现。培训师希望向我们展示如何解决错误，这很好，但这往往表现得(很差)好像是一个惊喜，我们不能确定他们涵盖了所有的错误。但是当直播出现错误时，相信我，这种挣扎是真实的。

最终，没有魔豆，你仍然需要学习——很多。要成为一名摇滚明星开发者，你必须具备一种学生心态，因为你必须学习，直到基本退休。不要立即跳入每个人都在谈论的新的趋势框架。等一段时间，等到技术成熟。你不想把时间浪费在那些不会持续很久的东西上，是吗？记住-前沿不是你想去的地方。

刚开始学编码的时候，我用了四个星期才达到牵手蜜月的巅峰。我确信我所学的 HTML 和 CSS 知识足以让我找到一份工作。就在那时，我第一次申请了。对我来说，这是一个巨大的失败，对那家公司来说，这完全是浪费时间。不久之后，信心崩溃了。很明显，我不能正确评估自己的能力。一年多一点过去了，我可以说我正处于上升期。我最终被录用了，但是艰苦的工作并没有结束，事实上，艰苦的工作才刚刚开始。让我给你看另一个图表——邓宁-克鲁格-斯拉温斯基效应，它描述了当你最终找到一份工作时会发生什么

## [](#survivor-bias)幸存者偏差

你还记得我说过你需要尽快开始申请吗？因为我在乎你，我希望你失败。没错，你没看错——**不及格**。在招聘过程中，失败是不可避免的，你越早意识到这一点越好。失败也是免费的，所以失败得快，就像他们喜欢说的那样。

你可能会问他们是谁？他们肯定不是你可能在网上读到的成功故事的作者。你知道我说的是哪几个。关于改变某人的生活，从事编码工作，在没有计算机科学学位的情况下在几个月内找到一份开发工作。我读了几十本，一本比一本印象深刻。我不想嘲笑任何人，我也没有任何理由不相信他们，但他们都不是关于某人失败和放弃的。

这种现象有一个名字幸存者偏差。这意味着只有成功的故事才能得到适当的关注，而你很少听说有人失败了(除非你把 facebook 上面向初学者的编程书籍的大量销售计算在内)。所以，如果你正在寻找一个现实生活中的例子，一个和你现在处境相同的人，那么你只会看到一堆成功。你不觉得如果你也能读到那些没有成功的人会很公平吗？我相信每个人都会编码，但不是每个人都适合这份工作。我知道这听起来很刺耳，但是我希望新兵训练营不要忘记在他们的广告中提到这一点。

## [](#hiring-process)招聘流程

根据[技术面试坏了，但我们可以修复它](https://dev.to/ambroselittle/technical-interviewing-is-broken-but-we-can-fix-it-4964)和 [F***你，我退出——招聘坏了](https://medium.com/@evnowandforever/f-you-i-quit-hiring-is-broken-bb8f3a48d324)，整个事情总体上是有缺陷的。我不知道-我才刚开始。但当我阅读这些文章时，我对作者提出的每一点都不断点头。从我自己多次面试失败后的经历来看，我感觉作为一名候选人没有得到公平的评估。

我只能想象，对于有经验的候选人来说，这将更加令人愤怒。应该预料到的是，他们在申请以前的工作时已经接受了审查，并且可能已经创建了一个成功的代码。但是，不…如果你熟记课本，并能在白板上解决不相关的算法，这还需要再检查一遍。一个成功的候选人必须具备的其他技能呢，比如融入团队、认同公司价值观或按时交付成果？

但是对于没有什么经验的人来说，这并没有太大的区别——一次失败的面试就是一次失败的面试。不管怎样，在这个疯狂的世界里，你得到工作的机会随着每一份申请的发出而增加。你还记得我在开始写的时候，运气是成为一个开发者的关键吗？从发出第一份简历到最终接受初级 javascript 开发人员的邀请，我花了 7 个月的时间。整个事情是一个匹配和适合的问题，所以继续坚持下去，你下次可能会走运。下面你会发现我求职努力的[流程图](http://sankeymatic.com/)，我推荐你自己做一个。

[![](../Images/cad291bd101d75252560cd50568262d0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--JBFAi6kg--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/wlfi8otqnfxox2y495tx.png)

所以最终，如果你愿意尝试足够长的时间，你会得到一份工作。但是在你签任何东西之前，想想你在做什么。在漫长而疲惫的求职过程中，诱惑可能会让你接受第一份工作，但你仍有可能最终得到一份没有前途的工作，薪水低于你应得的。我不是说*永远不要接受第一份工作*，我只是说*不要让最初的成功扭曲了你的观点*。冷酷地评估这个提议。这是你想要的吗？你能从中榨取更多吗？最终，对事情说*不*的人就是掌控自己东西的人。

## [](#imposter-syndrome)冒名顶替综合症

但是你得到了这份工作！恭喜你！从现在开始只有甜蜜的工作。你觉得舒服吗？不是真的？更害怕在同事面前出丑？你可能有冒名顶替综合症。这是一种感觉，当每个人似乎都比你聪明，在你看来，你是装出来的，而你即将被揭穿是个骗子。这种感觉并不罕见——我几乎每天都有这种感觉。

你可能想通过表现出你了解自己的东西来对抗冒名顶替综合症。假装直到你成功，好吗？对不起公司里的人过几天就会看穿你。要诚实。不要害怕说*我不知道*。在你承认你不知道一些事情之后，没有人会马上解雇你。几乎没有新员工会在头几个月带来收入，即使是在高级职位上。

我通过把我的编码成就列成一个清单，每当我再次质疑我的技能时就看看它，来克服冒名顶替综合症。它们甚至可以是很小的事情，比如*我自己解决了一个合并冲突*或者*我为一个很长的命令行做了一个别名*。当我再次开始体验冒名顶替综合征的症状时，这对我帮助很大。下面推特上的兔子也给你一点建议。

## [](#my-way)我的路

四个字:读、解、网、创。我能有今天是因为我做到了——我完全沉浸在编程中，在一年多一点的时间里什么也没做。我说的当然是我的空闲时间，因为我还得去工作！

**阅读**:能读多少读多少。在家阅读，在公共汽车上阅读，在杂货店排队时阅读。只要你在读关于编程的书，它就会起作用。你不喜欢读书？那就看视频吧，但是记得要有针对性，看一些关于编程的东西。

**解决**:这意味着白板挑战。目前，这是几乎每个工作面试的必经步骤，所以如果你想通过，你需要知道如何计算第 n 个斐波那契数。

**网络**:尽快开始联网。尽量和写代码的人在一个房间。去聚会。结交和你处境相同的朋友。找到一起编码的人。与更有经验的程序员交流。找个导师。附近找不到见面的地方吗？数字化网络。

**创建**:尽快学习 Git，登录 GitHub，从创建账户的那一刻起就开始投入到自己的项目中。你的 GitHub 时间轴看起来越不像一个恶心的俄罗斯方块游戏越好。找工作时，如果你没有经验，你的作品集就足够了，所以最好有几个完成的项目。他们不一定要很大或给人留下深刻印象，但定期贡献和展示进步是非常重要的。

## [](#blinding-finding-because-sick-pick-was-taken)致盲发现(因为生病摘被摘走)

这里有一堆有用的(而且免费！)我希望在我的编码之旅开始时就知道的链接。我真的希望你会像我一样发现它们很有帮助。

*   [freeCodeCamp](https://www.freecodecamp.org/)——我欠他们最多，尤其是在我的编码冒险之初。我真的真的推荐它给任何想编码但不知道从哪里开始的人。这是一个学习基础知识和更多网络技术的好地方。
*   面试 30 秒钟——他们对招聘人员可能提出的几乎所有棘手问题都有答案。可惜我是在第一次技术面试失败后才知道这个网站的。尴尬的是，当我决定把它加入书签时，我发现它已经在我的书签文件夹里了。
*   [你不知道 JS](https://github.com/getify/You-Dont-Know-JS)——我不知道哪本关于 JavaScript 的书应该是你的第一本，但我确定你不知道 JS 应该是你的第二本。凯尔·辛普森涵盖了很多材料，并且很快变得非常专业；这就是为什么我不建议初学者使用它。它还回答了采访中所有的棘手问题，包括一个彻底的技术证明。
*   韦斯·博斯和斯科特·托林斯基的播客。他们谈论了 100 多集与网络开发有关的一切，甚至更多(通常是烧烤和跳舞)，而且一秒钟都不无聊。最好的。播客。永远不会。
*   你对白板挑战和武术感兴趣吗？那么这个网站就是给你的。对于每一个解决的算法(kata)，你得到点数(kyu ),最终得到一条腰带。就像《空手道小子》里一样！
*   这是一个完整的课程，它的设计将教会你计算机科学的基础。哈佛有教，但网上也有。

## [](#conclusions)结论

嘿，你还在看书！我要说的是你将要表演你职业生涯中最伟大的特技。一路上你可能会后悔，可能会哭一会儿(或者只有我会哭)，但那种获得世界上最好工作的满足感是无价的。希望看完这个，你的成功之路会顺畅很多。
