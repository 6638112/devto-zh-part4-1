# 成功开发人员的 7 个坏习惯

> 原文：<https://dev.to/leewarrickjr/7-bad-habits-of-a-successful-developer-35l1>

欢迎来到我的博客！

我是一名前端开发者、meetup、T2 组织者、T4 播客主持人，我为其中一些事情获得报酬。

在我们开始之前，我不是说你必须成为或做所有这些事情才能成功。作为一名成功的开发人员，开发软件是我唯一的衡量标准。如果你编码和制作东西，那才是最重要的。在我看来，其他一切都是意外收获。

也就是说，我想谈谈我可以做得更好的一些事情。

这些都是我的坏习惯。

## 1。在我的桌子上吃饭

在我作为开发人员的第一份工作中，我是一个新的年轻团队的成员，每天午餐时间都会进行集体散步。我们会从办公桌前跳起来，放下工作，出去活动半个小时。我们会讨论一些工作上的事情，一些非工作上的事情，但最重要的是我们会休息一下，做点运动。

现在，我在一个不集体散步的团队工作。他们有一起出去吃饭的习惯，但我带午餐，不参与。我也经常在这里或那里利用空闲时间处理我的播客、meetup 或其他项目。午餐时间是完成一些额外事情的主要目标。所以最终发生的是，大多数日子我都在办公桌前吃饭。

不要这样。休息一下，离开屏幕，出去走几分钟。

## 2。下兔子洞

 [![Photo of a peaceful Rabbit](img/40e91dab966a33e8ffb6733d17d075cc.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--deSDs8te--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://leewarrick.com/blog/static/a321c6d327761b0f5e89e8ca5b4a5da7/c739e/rabbit.jpg) 

“掉进兔子洞”是指当你在编写代码时遇到一个问题，并开始在研究解决方案之间跳跃，而没有实际完成任务。

例如，您想要构建一个允许用户上传和裁剪图像的功能。你进入 stack overflow 并阅读关于上传文件的内容，然后在 Vue 中查看文件上传组件，然后在 React 中，即使你正在使用 Vue，然后 google 告诉你 HTML5 canvas 可以帮助你，所以你开始查看 codepens…

四个小时后，你有了 8 个潜在的解决方案，10 个新的副业项目，并且了解了 3 个你想尝试的新 JavaScript 框架。你掉进兔子洞了。

探索新技术是好事，但不要忽视手头的任务。有时候你只需要把事情做好，目前任何可行的解决方案都足够了。

## 3。太频繁地说“是”

有时，这是朋友或同事问我是否正在从事一个潜在的自由职业项目，尝试一些新技术(稍后会有更多介绍)，或者其他机会。大多数时候，它是在对我推销给自己的想法说“是”。

在野外有太多的开发工作需要完成。你可能会想，*伙计，我甚至找不到工作*，但我告诉你这是真的。每个企业，非营利组织，俱乐部等等。需要一个网站，还有一大堆问题等着科技来解决。如果你坚持下去，你会发现有大量编写代码的机会在等着你。

关键是要知道该答应什么事情。当机会来敲门的时候说不是错误的，但是有时候你必须在承诺之前认真考虑一下你当前的工作量和可用性。有时候你需要说不。

## 4。有太多的副业

这与前一点有点关系。俗话说得好吗？

*“追两只兔子的人一只也没抓到”*？

我有个人项目，我想做的演讲，要计划的聚会，要录制的播客，现在还要写博客。这只是在发展领域。很难同时关注 10 件事情并真正完成一些事情。

我们生活在一个赞美多任务处理的世界，一个崇拜工作的行业。不要觉得你必须做所有的事情。一次专注于一件事是可以的。

## 5。把我所有的空闲时间都花在编码上

我认为一种趋势正在形成。

我是一个拖延者，天生有点懒，但不知何故，我发现自己在办公桌前不停地编码、研究和计划。这是我不知不觉养成的一种阴险的习惯。

“找一份你热爱的工作，你这辈子就一天都不用工作了。”

相反，如果你放任自流，你将永远无法停止工作。我有时不得不强迫自己离开办公桌，到外面去，读一本书，或者只是暂时做一些没有效率的事情。

## 6。不使用生产力工具

有很多应用程序可以帮助一个人记笔记、安排提醒和计划项目。

使用那些工具！

我无法告诉你，如果我把它写下来，计划好，记录下来，设置一个提醒，或者只是做一些除了记笔记之外的事情，我能记住多少被遗忘的事情。

心理笔记是敌人。不要记笔记。我们在一个需要关注和专注的领域工作。我保证当你开始编码的时候，你的大脑对这个重要的记忆所做的第一件事就是抛弃它。

##### 侧注:

不要陷入制作自己的生产力工具的陷阱。我的意思是，编码和学习是很好的，但有时只使用现有的应用程序并就此收工会更好。我对此感到[内疚](https://leewarrick.com/goaler)。

## 7。学习太多东西

在软件中大约有 40 千[件事](https://github.com/kamranahmedse/developer-roadmap)要学，我想学完它们**所有的**。我很好奇，我很上进，我需要确切地知道我接触的所有东西在引擎盖下是如何工作的。

问题是死亡率。我们在地球上的时间是有限的，即使我们把所有的时间都用来学习编码，我们也不会知道所有的事情。

我叫它:**你不可能全部学会。**

我希望有一天我有智慧知道我应该关注的事情，而忽略其他的事情，或者在需要的时候查找它们。

## 临别赠言

记住你不能做所有的事情，你只是一个凡人。善待自己，谨防倦怠。最后，时不时地花点时间回忆一下你已经走了多远，并为自己的成就感到骄傲。

感谢阅读。
