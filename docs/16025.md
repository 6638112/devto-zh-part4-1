# 如何中断你喜欢的事情的 5 个技巧

> 原文：<https://dev.to/spukas/5-tips-for-how-to-breakpoint-something-you-love-2ap4>

这篇文章不是关于爱，也不是关于破坏东西，而是调试你的代码更有效的方法。我将列出几个最常用的 Chrome DevTool 断点引用，它们将停止正在运行的代码，并让您更深入地了解评估和执行。

另外，考虑到你知道如何打开 DevTools，切换到 Source 标签页。

### 事件监听器断点

使用事件侦听器断点，您可以在特定事件被触发后立即停止代码。您可以在右侧的“源”选项卡的“事件侦听器断点”窗格中找到事件列表。该列表包含所有 javascript 事件，可以通过展开特定类别来选择这些事件。

例如，要暂停一个按钮的注册点击事件的代码，需要勾选面板事件监听器断点>鼠标>中的复选框，点击并与按钮交互。

缺点是文档的所有点击事件都会暂停，直到它到达你的函数，所以要小心，继续执行脚本，直到你看到你要找的东西。

### 行代码断点

这是最流行和最常用的断点。它将在断点行之前暂停代码。要使用它，你必须知道你想在文件中暂停的确切位置，并手动设置该点。

在 DevTools 中通过按 Command+P (Mac)或 Control+P (Windows)打开您想要检查的文件，输入文件名并选择它。在文件左侧，通过勾选行号，您将设置断点，下次执行该文件时，代码将暂停。

此外，您将可以访问已经评估过的范围变量，只需转到控制台选项卡并摆弄它们。

### 条件断点

非常类似于代码行，除了您可以输入暂停的条件。我发现它在遍历文档列表时非常有用，但只需要暂停并检查特定的一个。

要设置条件打开文件，右键单击代码行。从出现的菜单中选择“添加条件断点”，并设置条件，例如“row.order === 2”。下次执行循环时，当条件评估为 true 时，代码将停止。

### 登录点

我最喜欢也是最简单的方法之一是将信息记录到控制台中。忘记在源代码中放置 console.log()行并重新加载页面。无论您是否需要记录变量或查看代码行是否正在执行，请使用断点菜单，右键单击文件行号并选择“添加日志点...”选项。在显示的输入中，输入一个要在浏览器控制台中打印的值(可能是一个字符串或评估的范围变量)。当触发日志点时，与网页交互并在浏览器控制台中查看打印信息。

### DOM 断点

当 DOM 元素改变时暂停代码。可以设置 3 种暂停代码的条件:修改元素子元素、改变属性和删除节点。

要设置断点，从 DevTool 的 Elements 选项卡中选择元素，右键单击它，悬停在“Break on”上，并选择三个断点条件之一。

下一次 DOM 元素改变时，代码会暂停，您可以确定是什么动作引起的。

这几个断点选项只是大局的一小部分。Chrome DevTools 为调试代码提供了更多的功能，断点是一个很好的开始。
