# 对开发商的迷恋

> 原文:[https://dev.to/gypsydave5/on-developer-fetishes-20i4](https://dev.to/gypsydave5/on-developer-fetishes-20i4)

## [](#caveat-lector)告诫选择者

接下来的更多的是几天前的晚上流出来的意识流，非常未经编辑和非常粗糙。请不要把它看得太严重，或者个人(那太讽刺了)——它针对我和任何人一样多。我怀着遗憾将它*出版。*

### [](#programming-languages-do-not-care-about-you)编程语言不在乎你

你热爱你的语言。你已经花了几个小时，几天- *几个月* -从里到外了解它的复杂性。你是 JavaScript 的大巫师。你是 Rubyists 的闪亮红宝石。你是 PHP 中的 P，是 Haskell 中的 Hoss，是最老土的 Rustacean，是最傻的 Gopher，是 JVM 商店中最大、最热、最热的一杯 Java。

那又怎样？

你的语言实际上只是围绕你能做什么的一套有色眼镜。你最终会被你的语言所限制；相信我，它不能做所有你想让它做的事。

对于一门语言来说，至关重要的不是这门语言——它的特性，它的缺点，它令人惊叹的一行匿名函数调用的语法，或者数组析构，或者模式匹配...

或者它漂亮的登陆页面，鳕鱼哲学，可笑可爱的啮齿动物吉祥物。

这甚至不是它教给你的东西，关于这个世界，关于计算机。这很重要，也很有用，但不是最重要的。

一种语言最重要的是说这种语言的人和他们用这种语言解决的问题。尽量保持这种状态。

### [](#you-are-not-the-framework-you-use)你不是你使用的框架

如果对于语言来说是这样，那么对于框架来说更是如此。但通常是这些人工制品导致了最大和最喧嚣的开发者盲目崇拜——是的，我在和你说话，2019 年的前端 JavaScript 开发者。

没人在乎你的网站是不是 React 做的。或者有棱角。或者 Vue。或者预先行动。或者 Bumfle。你没听说过邦弗尔吗？我刚编的。我的网站是用 Bumfle 写的。这是最好的。我是 Bumfle 开发者。在过去的 20 秒内一直是。我最擅长 Bumfle。请报名参加我的免费 Bumfle 培训课程。我们 Bumfle 社区欢迎所有 Bumfle 贡献者。此外，我刚刚更改了 Bumfle API 以使用 Bumfle 钩子，我称之为 Books。因为我读过 DDD 的书。你所有的 Bumfle 代码都因此而被破坏了。不客气。此外，Bumfle 现在被称为摸索。我现在也在构建下一个失球。它叫布莱恩。

没人关心你是否比 React 的上一个版本更喜欢一个框架，或者它是否比 Vue 的下一个版本更简单或更容易使用。尤其不是框架。

因为在两分钟的时间内，你所有的热情、努力、爱和时间的主题将通过维护者的奇思妙想、时尚的令人兴奋的变化、3.6.6.6 的更新、引入 DDD 书中每隔一个术语的新 API 而蒸发...

(Bumfle 对象现在称为 Baggregates。不客气。)

不要依附于一个框架。不要爱上这些昙花一现的人。框架能做的最重要的事情是解决特定的问题。尝试解决更多的问题，不要再用你最喜欢的魔术盒来寻找问题。停止发明问题来解决你最喜欢的魔术方块。

### 你的文本编辑器不能定义你

Vim 很好玩。每次正确记住一个命令，你都会觉得自己通过了一次小考试。你会得到少量的多巴胺。你是一个成功的人，也是一个强大的软件开发者。至少是 5X。你一定是因为你知道如何删除`a` `w` ord。然后你想告诉你的朋友你在编写一个在每行末尾添加一个逗号的宏时取得的巨大成功。

试着压抑这种欲望。也许吸点含糖的东西五分钟。会过去的。

谁在乎你能多快删除一个单词？还是一条线？还是剪贴一段？你用引号*围住一个单词的速度真的是你工作的限制因素吗？你不能写你的算法，因为你花了太长时间做复制和粘贴？你的创业失败是因为你打字不够快？*真的吗？**

所有这些插件——这也适用于 VS 代码人员——你使用它们吗？当你安装它们的时候，它们看起来是个好主意。免费的东西。但是你曾经使用过它们吗？他们只是用更多的噪音填充你的屏幕吗？花这三个小时选择合适的配色方案值得吗？

当你写代码的时候，好好的，长时间的，努力的看看你实际上做了什么。真的有那么多事情是 Windows 记事本无法实现的吗？

你的文本编辑器*只是另一个工具*。这是达到目的的手段。它不是身份的象征。如果你花了超过 1%的时间来设置编辑器...停下来。

### 你的终端不能表达你的个性

完美的字体。完美的配色方案。完美的外壳。

当然是狂欢。不，不是巴什。没有人再使用 bash 了。你是谁？离开这里。不，我们用 zsh。不，我们也不知道怎么发音。但是我们用它。或者鱼。或者别的什么。太神奇了。

我们使用 tmux。主要是为了让我们能看到一座大钟。我们有一个自定义提示。它告诉我们 JavaScript 框架还有多少分钟就会过时。我们有一个终端模拟器，当我们输入时它会发出焰火。背景是黑色的。因为我们是 10 倍的开发者，就像机器人先生一样。

停下来。求你了。这是一个终端。它是一个贝壳。你可以花一个月的时间定制它，但最终你不会成为一个更好的开发者。你只会讨厌你所创造的任何令人厌恶的东西，然后再花一个月的时间去改善它。另一个。另一个...

不要担心它看起来怎么样，只要使用它。做一些开发。你知道的。你应该做的事。

## [](#solutions)方案

有人告诉我，只是抱怨所有这些东西而不提供任何解决方案是不太好的。所以我会试着变得更好。

*   语言。多学一个。学两个以上。旅行能开阔心胸。你将学会更加热爱你的第一语言。你将被迫去思考解决老问题的新方法。你会更好地理解计算机。当你谈论语言时，谈论它们之间的不同，以及用不同的方式做事有多好。警告:不要只是学习如何用二十种语言写`hello world`；很好地学习一些不同的语言。这不是浪费时间。

*   框架。如果你的框架没有给你可以转移到现实世界的技能和知识，我会担心。确保你知道它是如何工作的。不要被困住。每个框架都是一艘正在下沉的船。每一个都是。是的，我们都是老鼠。不要一次学一个以上。当它最终抛弃你的时候，忘掉一切。

*   编辑。这与其说是建议，不如说是一些想法。删除你的插件，关闭语法高亮，给它一周时间。找到使用你编程技能的方法(你是程序员，对吗？)通过使用它与 shell 的交互来扩展您的编辑器。换个编辑一个月试试别的。尝试使用`ed`一周。

*   终端。对你正在使用的任何电脑使用现成的设置，等到你*不能*做你*需要*做的事情时，再开始摆弄它。例外:如果你在一个漆黑的房间里工作，你可能会有一个黑暗的背景。但是也许问问你自己，为什么你要在一个漆黑的房间里做所有的编码工作。你还好吗？你想要与某人交谈吗？
