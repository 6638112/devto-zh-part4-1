# 现代发展的热潮

> 原文:[https://dev . to/elastic crash/fanfare-of-modern-development-58 H5](https://dev.to/elasticrash/fanfare-of-modern-development-58h5)

几个月前，我 40 岁了。我提到这一点只是为了表明我在这个行业和总体发展方面有相当多的经验和经验。老实说，我不觉得自己 40 岁了。

我喜欢成为一名软件工程师。我喜欢解决复杂问题和设计系统。我喜欢尝试新的语言、框架和新概念，并且通常保持与时俱进。我喜欢探索和使用新的工作流程方法(比如敏捷),因为它们有助于提高工作质量。

但我就此止步。那是我总是停下来的地方(我稍后会解释它背后隐藏的意义)。

最近(过去 5 年有一些持续的变化)，在我参与的大多数团队中，已经引入了与 em 有关的各种变化，无论是我们的工作方式，还是让我们参与、满意的方式，或者是让我们在职业生涯中进步的方式(不确定这些如何帮助我们)。老实说，我可以把它们归类并描述为。

A.改进 b .毫无意义的
C .令人恼火的
D .破坏性的

敏捷，敏捷当然是一种进步。但是，只有当项目或业务工作流适合它时，它才是一种改进。同样的事情也适用于伴随敏捷而来的一切(站立，

然后还有其他一些我不介意的变化，它们被归类为无意义的，这些变化通常包括额外津贴，比如乒乓球桌、游戏机、周五的啤酒等等。我理解他们，我理解为什么有些人喜欢他们，只是他们不是我对快乐的定义...更不用说我已经喜欢我的工作，我很高兴设计和编码。

我对乐趣有不同的定义，它主要包括那些不与工作混在一起的事情，比如和乐队一起演奏音乐，或者骑自行车，或者和朋友出去，或者和我的女儿和妻子一起玩棋盘游戏。

还有一些我不理解的“东西”,它们对我来说主要是背景噪音。比如用时髦的名字给你的团队命名为“stargazers”“rocket”“ducks”，或者用随机的流行语给你的 sprint 命名(我会在任何名字前加一个日期，这样我就可以很容易地知道 X ticket 是什么时候第一次被添加到 sprint 中的),但是很多开发人员似乎很喜欢它们。

在这个列表中，我还可以加上，频繁的项目追赶，当没有什么要追赶的时候，因为在最后两次冲刺中你在做一个不同的项目。

最后，有些事情实际上非常烦人和具有破坏性。像你站起来后需要欢呼的事情，像“加油，加油，火箭队”(我现在的工作单位没有这种东西👍如果我们这样做，我肯定会退出)。或者喜欢没有特定的桌子，在任何有空间的地方坐下(可能是模仿咖啡店)(呕吐)。
