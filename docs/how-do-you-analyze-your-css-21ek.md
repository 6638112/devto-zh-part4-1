# 你如何分析你的 CSS？

> 原文：<https://dev.to/starbist/how-do-you-analyze-your-css-21ek>

最近我在 Twitter 上发布了一个问题:

> 嗨，开发者们！我想问你一个关于 [#CSS](https://twitter.com/hashtag/CSS?src=hash&ref_src=twsrc%5Etfw) 的问题。
> 
> 1)你考你的 CSS 吗？你是怎么做到的？
> 2)如果不是，为什么不是？我经常使用特异性可视化工具来比较版本。[https://t.co/ncNIvBWk5L](https://t.co/ncNIvBWk5L)
> 
> 和最近 CSS 统计工具:[https://t.co/mmOLY7gyyW](https://t.co/mmOLY7gyyW)。
> 
> 有什么建议吗？[pic.twitter.com/nC0RGvaKfX](https://t.co/nC0RGvaKfX)
> 
> — Silvestar Bistrović (@malimirkeccita) [July 11, 2019](https://twitter.com/malimirkeccita/status/1149382859775496194?ref_src=twsrc%5Etfw)

我只收到了一个回复，但它意义重大。Bart Veneman 发布了[他的 CSS 分析工具](https://www.projectwallace.com/oss)。对我来说这是一个伟大的发现。从那以后，我就一直在使用这些工具，并试图学习新的方法来分析我的 CSS 文件。

在此之前，我经常使用 [CSS Stats](https://cssstats.com/) 来查看我的 CSS 的性能，使用[特异性可视化器](https://isellsoap.github.io/specificity-visualizer/)来查看特异性图。

CSS 仍然是 web 技术堆栈中被低估的一部分。像这样的工具是让人们注意到高效和高性能 CSS 代码的重要性的正确步骤。

你知道其他工具吗？你如何使用它？你在这些分析中特别寻找什么？
