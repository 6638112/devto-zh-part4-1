# 我的咖啡壶是如何教会我喜欢类型的

> 原文:[https://dev . to/mariowhowrites/how-my-coffee-maker-teached-me-to-love-types-1l5l](https://dev.to/mariowhowrites/how-my-coffee-maker-taught-me-to-love-types-1l5l)

打字稿。

几个月来，这个词一直萦绕在我的脑海中。VS 代码会为我创建的每个新文件建议`.ts`扩展名，每次我的回答都会是，“今天不行。”查找 JavaScript 函数的实现会把我带到 TypeScript 类型定义文件，它看起来总是有点太吓人而无法解析。我读过关于类型化 JavaScript 日益增长的必然性的引述，并认为这种必然性并不适用于我。我什么时候遇到过打字可以解决的问题？

我一点也不知道，将我转化为打字方式的福音传道者会是我自己的咖啡机。

和我认识的许多喜欢工作的人一样，咖啡占据了我工作日的大部分时间。和我认识的许多开发人员一样，我追随我的创造性灵感，不管它们会把我带到哪里——经常到深夜。在工作时间结束时，我通常会在 autobrew 上设置一壶咖啡供第二天早上喝，这样我就可以开始新的一天，不会错过任何一个节拍。

几周前的一天，我醒来时发现了一场灾难:到处都是咖啡渣。咖啡渣在我的咖啡里，咖啡渣在柜台上，咖啡渣顺着我的柜门流下来，积在下面的厨房地板上。

事实证明，在我一大早被代码弄糊涂的昏昏沉沉中，我忘记了在清洗后把过滤器放回咖啡里。就在清理咖啡渣的那一刻，我意识到为什么我必须立即开始输入我的 JavaScript。

实际上，我意识到我的咖啡机就像香草 JavaScript。让我神志不清、漫不经心的自己随心所欲地设置我的咖啡是一件快乐的事情。没有检查来确保我放入了新的过滤器，没有检查我放入了新的咖啡渣，没有检查我放入了水，没有任何检查来确保我在早上真的会有咖啡，而不是浪费的咖啡渣瀑布。任何由我的咖啡设置引起的问题都必须在冲泡的时候处理，而不是在设置的时候，并且实际上有时间和(一些)注意力来解决问题。

如果我的咖啡机是严格打字的话？当然，当我唯一想做的事情就是睡觉的时候，我的机器会哔哔叫我修复我不正确的设置，也许我会很恼火。但我还是会解决问题，我的努力会得到早餐咖啡的回报。

对我来说，没有什么比咖啡更重要的了，正常工作的代码就是其中之一。因此，虽然我们何时能看到强大的打字技术进入家用电器还没有定论，但我当时就决定开始我的打字之旅。
