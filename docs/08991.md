# 初级软件开发人员入门

> 原文:[https://dev . to/marvinkweyu/getting-started-as-a-junior-software-developer-j3f](https://dev.to/marvinkweyu/getting-started-as-a-junior-software-developer-j3f)

在我们开始之前，我想祝贺你选择了这条美丽的道路；软件开发。这确实是一个漫长的旅程，有它的回报，但也有它的陷阱，就像你周围的任何事情一样。
那么，什么是软件开发呢？

软件开发是构思、说明、设计、编程、编制文档、测试和修复错误的过程，涉及创建和维护应用程序、框架或其他软件组件。

很长，不是吗？简而言之，软件开发可以被描述为以解决现实世界问题为目的的编写计算机程序的过程。选择踏入这个被一些人称为“古怪王国”的地方后，这里有几条建议会对你有所帮助。

**1。开始时选择一种语言，并坚持下去。**

我不能坚持开发者需要得到的这个关键指针的重要性。我知道你可能已经看到了某一种编程语言所能做的惊人的事情，但是当你回头看时，却看到了这种语言的另一个很酷的特性。相信我，他们是相当多的。

相反，除了四处看看其他人的项目，用多种语言做一个`"Hello world"`之外，选择一个，就一个。当你专注于一件事的时候，你开始挑战你的极限和你的工具的极限:一个框架，一种语言，一个项目，等等。而你会在这个过程中获得更深层次的知识。抵制成为万事通而不精通的冲动。

**2。把基本的记下来。**

在流沙上建房子之前，先把基础的东西做好。这可能需要浏览您选择的语言的文档。触手可及的编织细节。了解基本语法，如何创建基本函数，如何声明变量，使用 OOP 等。学习底层的概念和算法。请记住，语言和框架会改变，但算法将永远保持不变。在您开发项目时，它一定会为您节省大量时间，这就引出了下一个问题。

**3。创建一个真实的项目。**

它不一定是“下一个谷歌”或下一件大事。这是马拉松，不是短跑。从小处着手。创造一些直接影响你日常活动的东西。例如，作为第一个项目，我用 python 创建了一个简单的家务时间表。这是一个 excel 表格，每周都会改变，以适应我们当时的需要。在我的本地计算机上有了这个，下一步将会是给桌子的参与者邮寄一周的杂务。就这么简单。有了这个，我学会了一些 OOP 概念，学会了与 excel 交互，最后学会了用 python 发送邮件！

本质上，学习从来没有停止过。从来没有一个软件开发人员可以说“我刚刚学完 C！”骄傲地把它作为成品藏在壁橱里。从来没有发生过。为了说明这个事实有多真实，当 C 语言的创始人丹尼斯·里奇被问及作为一名 C 语言程序员，他给自己打了 7 分。我想你能明白这意味着什么。

所以出去吧，用 Javascript 创建一个 TODO 列表(如果这是你选择的语言。)或者制作一个简单的网站来展示你的俱乐部在这一周、一个月等等的活动。让它个人化，在你所处的实际环境中你会想要做的事情。

**4。和身边的开发团队取得联系。**

至此，你应该已经准备好了一些东西。那么接下来呢？协作。从 [github](https://github.com/) 到 [bitbucket](https://bitbucket.org/) ，获得可以合作的团队，点对点编码。一起做东西，给彼此挑战，检查彼此的代码。在帮助朋友写出更好的代码的同时，了解你还可以做些什么来提高。

**5。学习。**

作为岗位的结账，*学*。学习学习学习。作为一个持续的提醒:

> 软件开发中唯一不变的是变化。

总有一些被否决的函数，新的库使你的工作变得更加容易，新的声明函数、变量的方式，等等。就像他们说的，“作为一名网络开发人员，自愿离线一个月左右将是一个错误。”为什么？因为 web 应用程序中的需求和用户偏好总是在变化的。今天的这个特性很可能明天就会被抛弃。所以学吧。永远保持警惕，不要害怕变化。

PS:对于还在用 Python 2.x 的，2020 年安全更新停止。
