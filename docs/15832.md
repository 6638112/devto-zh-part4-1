# 学习编码时避免失望的 5 种方法

> 原文:[https://dev . to/lavie _ encode/5-ways-to-avoid-to-code-1ec 3](https://dev.to/lavie_encode/5-ways-to-avoid-disappointment-while-learning-to-code-1ec3)

没有什么比学习编码时对自己失望更糟糕的了。

甚至不仅仅是编码经验本身——在你的过程中，有太多的地方你会错过一些东西或犯一个错误，这真的会影响你快速找到工作的能力。

我想强调“失望”与“气馁”相对，因为失望导致气馁。沮丧会导致你完全放弃编码。

但是当我们有不切实际的期望时，失望就会发生。我有很多这样的照片。😅

所以，这里有三种方法可以让你避免失望:

# [](#1-allow-yourself-to-be-new-and-remember-that-everyone-was-new-once)1。允许自己是新的，记住每个人都曾经是新的

当我第一次开始编码时，我经历了两个阶段:

1.  我学习了所有这些新事物，并用它们来创造，这是一种完全的欣快、幸福和自豪
2.  害怕我知道的不够多，害怕我应该知道更多

在游戏进行了大约三个月后，第一名让位于第二名。更深入地浏览 Dev Twitter，看到人们正在做的所有大事情，这给它提供了动力。

于是，我开始为自己是新人而感到羞耻。但是，我经常忘记提醒自己，实际上每个人在某一点上都是新的。

不断地期望自己知道得比自己多是适得其反的，因为这会让你紧张，让你很难学习。我们开始恐慌。我非常恐慌，结果有点孤立无援。

我对自己没有更进一步，或者还没有工作感到失望，这让位于沮丧。

老实说，是社区的支持让我走上了正轨。大约 10 个月后，我得到了那份工作。

无论如何，不要这样对自己。你是新来的，接受现实吧。

庆祝你的新生，享受你可以犯错误而没有任何后果。一切都是学习的经历！

# [](#2-build-and-maintain-at-least-project-to-keep-up-your-confidence-and-attract-employers)2。建立并保持至少一个项目来保持你的信心(并吸引雇主！)

我经常听到学生说他们对开始项目缺乏信心，或者在他们创造了一个 MVP 后倾向于放弃他们。

或者，他们从来没有完成任何事情，因为他们不知道如何创造一个真正的 MVP，并期望它是完美的，然后放弃，因为我自己也很失望。😂

你们不需要创建一大堆项目。可能需要几个不同的项目才能找到你真正想坚持的东西，但你会找到你热爱的东西。

令我惊讶的是，有些人没有我的创造力——没有幸灾乐祸，这只是我的大脑工作的方式。我真的无法停止想点子。我跟不上他们所有人！

开始一个你感兴趣的项目，并创建一个 MVP。但是不要放弃它——总有新的特性可以添加到网站中。

通过在每一步分享你的项目来获得反馈，如果你不确定下一步要添加什么，问问其他人的想法！你不必听他们的，完全按照你的想法去做。但是，它至少会给你一些想法。

我在这一点的第一段提到了“信心”这个词，因为如果你不能维持至少一个项目，这就是你所缺乏的。

此外，如果你没有得到面试机会，维持一个项目只会帮助你吸引雇主的注意。

额外提示:利用您的 readme.txt 文件来讲述项目过程的故事。解释你为什么决定添加一个特定的特性，以及你在实现它时所面临的挑战！

# [](#3-adjust-your-expectations-on-how-long-it-will-take-you-to-learn-to-code-andor-get-your-first-dev-job)3。调整你对学习编码(和/或获得第一份开发工作)所需时间的预期

尤其是你在自学的时候！

不要把你的进步和新兵训练营的毕业生相比。他们做了一个超级严格的，有指导的项目，对他们的学生有很高的期望。(如果你想像一个自学成才的学生那样学习，那就和别人一起为自己制定一个计划，然后投入进去！)

每个人都以自己的速度学习。有时候我学得很快，有时候我学得很慢——尤其是当事情具有挑战性或者非常抽象的时候。

如果你没有学习编码的具体计划，你也不能走得那么快。考虑到这一点，因为你可能像我一样在错误的地方花费时间。

如果你因为外部义务而缺少时间，你也会走得更慢。

善待自己！无论你有多少时间，你都可以学习编码。只要创造自己的个性化配速，并坚持下去！

# [](#4-practice-your-practice)4。练习你的练习

当你学习新东西时，你应该做的第一件事就是使用它。

现在，我不是在说只在给你的那个小代码挑战中使用它。如果这是你学到的一件小事，那就足够了。

但这只是一个例子。[聪明、深思熟虑的练习](https://www.lavieencode.net/podcast/020-improve-your-skills-with-deliberate-practice/)需要把它们混合起来，给自己各种不同的例子来解决。

这对于对你来说具有挑战性的事情尤为重要。如果你很难理解它，不要在解决了一次之后就继续前进。

每当你记不起你需要的东西时，你就会对自己感到失望(尤其是当你知道你曾经学过的时候)。

如果你看不到明显的进步，你会对自己失望的。

建立强大的实践技能将增加你的信心，因为你看到你已经走了多远！

# [](#5-pay-attention-to-your-own-education-and-progress)5。关注自己的教育和进步

这与第三点有关。如果你关注别人的教育进步多于自己的，那你会让自己失望的。

一个 Twitter 连接发布了一个非常棒的项目，使用了一个你不知道的框架或库。

*那又怎样？*

你完全有可能不需要了解你想要的工作类型。或者即使你知道...如果他们学会了，可能要花一点时间。

你和他们的区别在于他们起步更早。他们并没有天生比你更擅长这个。

但我可以告诉你一件事:你们都将从完全相同的地方开始:开始。

所以，把你的眼睛放在你自己的领域上，关注你技能上的显著进步。总会有人知道的比你多。习惯就好！:)
