这些年来，我为许多公司工作过，既有代理形式的业务，也有大型组织的内部开发人员。

一路走来，我注意到企业主在调整业务实践以满足市场需求时经历的一些成长的烦恼和学习的经历。

首先，要明白一个企业主最大的支出之一是员工成本，不管这个企业是大是小。

这就是为什么它如此诱人...

#### 将劳动力外包给非美国的劳动力资源

我无法告诉你我见过多少次公司犯这样的错误。要么是被雇来清理遗留代码的人，要么是解决像这样的事情造成的巨大混乱。从长远来看，它最终总是让企业付出的代价大于节省的成本。一个很好的经验法则是，你总是得到你所付出的。

第二个错误，如果他们已经没有犯第一个是...

#### 降低成本，在骨干员工的基础上运营

这往往是工人们最痛苦的感受。在这种情况下，没有人真正赢。如果你的开发人员过于分散，他们将会产生一些没有经过深思熟虑的不合格的工作，并且从长远来看会产生更多的问题。虽然一个应用程序在前端可能看起来功能性很强，但是你可以打赌在后端的壁橱里有一些骨架正在被创建。通常，当开发人员需要抄近路以满足截止日期时，诸如安全性、文档和 QA 之类的东西将被牺牲以按时完成某些事情。不管是现在还是以后，这些问题最终都会浮出水面。

会让您的组织遭受损失的第三个问题...

#### 没有明确的流程，跳过文档

许多生产技术的公司已经转变为敏捷的或者至少是类似敏捷的结构和过程。虽然我最初觉得这种结构有点吓人，但它最终是非常有益的。敏捷方法真的可以帮助开发人员创建他们感兴趣的有意义的工作。它们还可以帮助他们创建更准确的评估，而不是将它们限制在一个小时的期限内，同时考虑到开发项目时的一定数量的未知或发现，特别是如果它是一个您没有使用过的新 API 或技术。

文档通常是最后优先处理的东西，这是无限重要的。有了好的文档，你可以很容易地避免地狱般的场景:每个人都去找一个已经在那里工作了很多年的开发人员来寻找最基本的信息，而每个人都在休假时哭泣。

#### 为之工作的好公司是成功的公司

为一家好公司工作就像一台加好油的机器，能够满足市场的需求，同时生产出高质量的产品。偷工减料，虽然极具诱惑力，但必然导致品质和文化两方面的腐朽。

当心企业主，当心！:)
