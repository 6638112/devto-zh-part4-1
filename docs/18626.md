# 创建 shy.js 来构建有状态 DOM。

> 原文:[https://dev.to/rayeddev/jquery-stateful-dom-3pj1](https://dev.to/rayeddev/jquery-stateful-dom-3pj1)

多年来，我一直在编写不同大小和规模的 web 应用程序。
我一开始是从 jquery 开始的，写了很多生产级别的 app。我喜欢用选择器保存 DOM 元素的 JQuery 概念，并且可以随心所欲地使用它。
在我使用(React.js)之前，它的工作方式对我来说非常神奇和有意义，它确实让我大吃一惊。我的意思是，组件设计范例的概念和通过组件树传递状态是最先进的，节能和高效的渲染。

但它也有代价，那就是学习曲线，我的许多朋友是网页设计师，发现很难采用(React.js)，当他们看到用奇怪的东西(ES6，Typescript，JSX)写的(React.js)例子时，他们总是很疯狂。他们几乎只参与他们设计的登录页面中的简单应用程序(表单、简单的滚动互动、价格表、产品偏好检查、购物车等)。).
然而，Jquery 对他们来说看起来非常容易，容易包含和启动。

所以我认为将(Reactjs)概念与 jquery 风格相结合可能会有所帮助:)

这就是为什么我为存在于登陆页面中的应用程序创建了(shy.js)微型 javascript 库。

[https://github.com/rayeddev/shy.js](https://github.com/rayeddev/shy.js)

尽管如此，在早期开发中，我希望它对你有意义，并请反馈祝福
