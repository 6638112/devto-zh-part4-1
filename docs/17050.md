### [](#be-sure-about-it)确信无疑🤔

### [](#take-the-dribbble-challenge)接受滴水挑战🏅💪🏼

[![](../Images/6653a687dbe1e94714b7b48a1306b535.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--GvLnYllt--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ehxckdrn59mx135nkbo4.png)

我喜欢 Dribbble，当你陷入困境时，它是灵感的巨大源泉。但是在我的团队中，它之所以流行是因为一个原因**“dribble 挑战”**。Dribbble 的挑战非常简单:找到擅长 CSS 或设计的人(如果你不知道，随时发微博给我 [@sarthology](https://twitter.com/sarthology) )。他们会从运球中随机选择一个设计给你练习。你的任务是尝试和复制它，并尽可能接近它，只使用 HTML 和 CSS。然后截图发给那个朋友。然后你的朋友会根据你与原始设计的接近程度给你打 1-5 分。在接下来的 30 天里每天都这样做。我把这个挑战给了我的一个同事，这对她帮助很大。这是她挑战第七天的作品。

[![](../Images/ddd4ea047d8588ef7cc0295d2732acf9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--fTfccRKZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/x9nxge06st8exwdi5btq.png)

疯狂。对吗？但是我给了她 3.5/5，因为我注意到一些东西仍然不像最初的设计。你将从这个挑战中学到的一件事是成为一名优秀的 CSS 开发人员最重要的部分，那就是*尊重设计师的工作*。作为开发人员，你的工作是公正地对待 UX/UI 设计师给你的设计。应该是完全一样的。

这在技术上可以帮助你两件事:

*   这将促进你良好设计意识的发展。
*   这将开始您探索更多 CSS 属性的旅程，以便完成工作。

### [](#dont-leave-devtools)不要留下债务工具🏃🏻‍♂️

[![](../Images/aeb564851e718c248155cf384ec452d3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XsTIHDfU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ce3g3x15a7cli56f74be.png)

### [](#contribute-to-open-source)贡献开源

尽管听起来很奇怪，但这很重要。一旦你足够自信你的 CSS 技能已经有了很大的提高，并且你已经获得了超级坏蛋忍者的地位，你需要冲向开源项目，因为**伴随着巨大的能力而来的是巨大的责任**。那些项目需要你！
这不仅会帮助你进一步提升你的技能，也会帮助整个操作系统社区。许多优秀的开源团队总是需要一个能很好处理 CSS 的开发人员。这也将帮助你提高你的设计感，发展你自己的艺术风格，正如你所知，任何伟大的产品都需要一点你自己的创造力。许多伟大的开源工具缺乏良好的设计。去 Github 上找几个这样的好产品，叉上，添加你的 CSS，提交你的拉取请求。

*愿源头与你同在。*
