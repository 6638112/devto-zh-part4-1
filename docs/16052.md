## [](#linux-101)Linux 101

从技术上来说，linux 是指 Linux 内核，但现在它也指运行在这个内核上的操作系统(内核= >操作系统的核心)。它是由 [Linus Torvalds](https://en.wikipedia.org/wiki/Linus_Torvalds) 在 1991 年创建的，他是一个美籍华人，也创建了 git 版本控制系统。它类似于 AT & T 做的 OS Unix，这是一个重要的区别。MacOS 和 BSD 风格是基于 Unix 的，所以它们在技术上不是 Linux，但它们是相似的。

今天，Linux 运行着大多数的服务器和智能手机，所以可以说互联网(和整个世界)都运行在它上面。原因是 Linux 是开源的。首先，服务器要求稳定和安全。经过 8，000 多名贡献者几十年的努力，Linux 的开源特性确保了这一点。除此之外，它还是免费的。

## [](#the-king)国王？

但是当涉及到消费者的台式机和笔记本电脑时，Linux 就没有那么普及了。Windows 是第一个出现的，并成为标准。它易于使用，兼容一切。

但是如果你愿意牺牲 Windows 的这两个品质，你会很快发现 Linux 非常优越。它的可定制性和速度无与伦比。有这么多发行版(发行版= >基于 linux 内核的操作系统)可供选择，您可以找到一个真正符合您需求的。

但是还有更多。作为开发人员，您可以更多地利用 Linux。包管理器防止了$PATH 地狱，并使安装库和框架变得轻而易举。bash shell 在合适的人手中是一个无限灵活的工具，Linux 需要更少的系统资源，允许你用更多的资源来构建你的 Android 应用。

## [](#distro-talk)区谈

经过大量的研究，我决定在我的笔记本电脑上安装 Arch Linux。Arch 提供了一个很棒的 wiki 和一个简单的 DIY 体验。这允许用户只安装他们需要的东西，让他们即使在劣质系统上也能有流畅的体验。我的系统在 200MB 内存的情况下处于空闲状态，可以在不到 2GB 的内存下支持 18 个 chrome 标签。相比之下，Windows 空闲时需要 1.5 到 2GB 的内存。Arch 也有一个大的包存储库+巨大的社区包存储库，所以我可以得到我需要的所有包！得到一个 GUI 并使它符合我的需要花了一些时间，这是真的，但是我的桌面看起来是这样的，所以这些时间是值得的。
