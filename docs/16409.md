# 让你落后的 7 个错误

> 原文：<https://dev.to/jsmanifest/7-mistakes-that-keep-you-behind-4481>

在[媒体](https://medium.com/@jsmanifest)上找到我

学习 JavaScript 并不是对每个人都有意思，尤其是对那些没有耐心的人。有时候，人们会开始质疑这是否是他们正确的职业道路。

但是 JavaScript 从来就不是在公园散步。

好消息是，你的学习过程可以通过做出更好的决定和学习方法来改进。*的坏*消息？它不会自己发生的。唯一能改变这一切的人就是*你*。

本文将讨论开发人员在学习 JavaScript 时容易犯的 7 个错误。

## 1。快速浏览概念

[![rushing through concepts](img/e0d7195fc37b3ab5d31618e4dfb210e9.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AIopP90t--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/rushing-through-concepts.jpg)

一旦你开始理解像[闭包](http://javascriptissexy.com/understand-javascript-closures-with-ease/)或[高阶函数](https://codeburst.io/here-are-7-ways-higher-order-functions-can-improve-your-life-a392aa6b29d2)这样的概念，没有一些实践经验就跳到下一课几乎从来都不是好主意。

但我知道，这可能非常吸引人，让我们进入下一个话题。每个人都会这样。但是请相信我，如果你在中间花一些时间亲自动手编写代码，即使只有五分钟，它也会帮助你在将来更容易地构建项目。通常情况下，对一个概念进行编码的感觉会一直留在你的脑海中，并帮助你长期记忆。

如果你最终匆忙地浏览概念，你将会忘记并在旧资料中搜索以试图刷新你的记忆。当这种情况发生时，你会非常沮丧，因为你已经“学会”了这一点，却在浪费宝贵的时间。然后那些消极的想法开始堆积在你的脑海里。

这很重要，因为人们经常说他们[因为*时间*不足而](https://javascriptissexy.com/a-significant-number-of-students-quit-their-programming-education-find-out-why-before-you-sign-up-for-a-program-and-suffer-the-same-fate/)退出了他们的编程生涯。

当你觉得一个概念对你来说是小菜一碟的时候，你应该准备好继续前进！

## 2。不拥有你的代码

[![not owning your code](img/55cb8ae57a3932008db2c1aff82f6c91.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--u9rdSDXS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/not%2Bowning%2Byour%2Bcode.jpg)

我今天仍然看到的一个常见现象是人们变得过于依赖他人的工作——无论是从教程中复制另一个人的文件，还是使用 lodash 库做任何事情。虽然在很多任务中使用 lodash 并没有什么错，但是如果你自己还没有应用这些概念，从长远来看，你也可能会让自己落后。概念是*将会坚持其他工具/框架/语言*的东西，因为它们是编程中的*思维方式*。

这就是为什么很多面试仍然给出编码挑战的原因。他们想看看你是如何思考的，或者你是如何将概念应用于开发中的常见问题的。您可以单独使用组件库来创建最好看的用户界面，同时让 lodash 为您处理应用算法，但是当问题出现时，它们需要您准备好应用概念并解决它们。他们不期待 npm 安装另一个库来解决他们的问题。他们需要你。

你不想在应用你已经学过的概念时遇到困难。所以花一点时间自己练习是个不错的主意。*拥有自己的代码*！

## 3。过度思考一切

[![Overthinking everything when learning javascript](img/4131b37ce295e3d3480bee01b80e0ead.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--otxiKsfk--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/overthinking%2Beverything%2Bwhen%2Blearning%2Bjavascript.jpg)

有了网上可以找到的所有信息，在学习 JavaScript 的下一个主题之前，坐下来收集所有你需要的资源是非常诱人的。

但我不认为我见过任何人说这是学习和变得更好的方法。然而，我们似乎有这样做的冲动，因为我们希望所有的信息都摆在我们面前，这样我们就能为今年剩下的时间做好准备。

*都在你脑子里*。

不要太沉迷于收集 80 个教程、6 本电子书、超过 50 个资源书签，并试图开始一起解决它们，只要你觉得自己是 web 开发之王。

如果你正在学习 JavaScript，动手学习吧。刚开始写东西。利用你在过去一个小时里学到的东西，为自己做一个小例子。这些活动最终会在你的脑海中停留更长时间，而不是马拉松式地阅读你收集的所有学习材料。

## 4。用太多的信息淹没自己

[![overwhelming yourself with too much information in javascript](img/13047da22a38c5460139d457a0f11fe5.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--k9AwF_8i--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/overwhelming%2Byourself%2Bwith%2Btoo%2Bmuch%2Binformation%2Bin%2Bjavascript.jpg)

当你花大量的时间来获取信息而不是在其间编码时，你实际上是在让自己忘记并在以后回到材料上。

一个让你害怕的地方是花所有的时间去学习，因为你觉得你在学习。但是几个月后当你最终忘记的时候会发生什么呢？你会发现，由于某种原因，你无法应用几个月前学过的概念和材料。

如果你需要建议，每消耗一小时信息，至少花十五分钟编码。

## 5。将自己与更高级的开发人员进行比较

[![comparing yourself to more advanced developers](img/73f8c548a2e9a6947c22eb68336bfa81.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--gk06N2a4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/comparing%2Byourself%2Bto%2Bmore%2Badvanced%2Bdevelopers.jpg)

当你把自己和别人比较时，你实际上不会去想他们是如何到达那里的，而只会去想他们现在在 T2 的什么位置。

当人们看到像丹·阿布拉莫夫这样成功的程序员时，他们会自然而然地认为他们的学习过程是从编写干净的代码开始的。

只要把注意力集中在你现在所学的东西上，你就会越来越好。很快，你就会处于这样一个位置，人们开始想知道你是如何到达那里的。

## 6。学习基于 JavaScript 构建的工具和框架

[![learning the tools and frameworks built on top of javascript](img/5ae6eea6a25e9113aa58ad711f65391e.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--u4oz6zaT--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://jsmanifest.s3-us-west-1.amazonaws.com/posts/7-mistakes-that-are-keeping-you-behind/learning%2Bthe%2Btools%2Band%2Bframeworks%2Bin%2Bjavascript.jpg)

所以你应该学习 JavaScript，但是你却马上学习了 React/jQuery/Angular/Vue。当一种新的革命性工具问世，而你做出了改变，会发生什么？你最终会依赖别人来找出这些巧妙的技巧，并希望有人分享它们，因为他们学会了 JavaScript，而你没有！

如果你还没有牺牲大量的精力来学习普通的 JavaScript，我强烈建议你真正地学习这门语言——因为如果你学习这门语言，那么你应该理解在它之上构建的工具，*为什么要构建它们，*它们解决了什么问题。通过学习 JavaScript 中的 *why* ，它也将帮助你避免做出糟糕的代码设计决策。**

我职业生涯中最后悔的错误之一是在开始学习 JavaScript 之前学习 jQuery。当 React 出现的时候，我也加入了这股潮流并开始学习 React。然而，在我意识到我不知道*为什么我必须使用 react 或者为什么人们需要它*之后，这变成了一个*非常尴尬的*学习过程。它突然变成了一个记忆 react 代码的游戏。在 JavaScript 工具或框架中，很多重要的部分都被隐藏起来了，而*你需要看到*却看不到。
