# 标签提示选择器

> 原文:[https://dev.to/suntong/tag-prompt-selector-127o](https://dev.to/suntong/tag-prompt-selector-127o)

当涉及到给我的博客添加合适的标签时，我总是希望 dev.to 有一个标签提示选择器，就像 stackoverflow 一样，这样我们就可以选择最常用的标签，而不用重新发明轮子。

像前几天一样，我在问一个关于*“系统区域设置”*的问题。当我选择标签时，网站提示我应该使用*“地区”*，而不是*“地区”*，因为它涵盖了两种情况。

今天，我写关于*“ssh 代理”*的博客。那么我是应该使用“代理”，还是已经有一个更具体的标签*“ssh-agent”*，还是以*“ssh _ agent”*的形式存在，或者...？

所以，请像 stackoverflow 一样提供一个标签提示选择器。它不需要在 post composer 中，一个独立的地方就足够了。

或者在某个地方已经有这样的东西存在？
