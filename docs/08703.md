# [](#users-categories-)用户类别:

Linux 中的每个文件都有 3 类用户，权限适用于这 3 类用户。

1.  **用户**:文件的所有者用户，通常是创建文件的用户。2.**组**:属于文件所有者组的所有用户，通常是所有者用户的主要组。3.**其他**:系统中所有其他用户，不是所有者用户，也不是所有者组的成员。

# [](#types-of-permissions-)权限类型:

每个用户类别可以有 3 种类型的权限。根据应用于文件还是目录，这些权限可能有不同的含义。

1.  **Read** (r):在一个文件上，意味着文件的内容可以被读取。在目录中，这意味着目录的内容可以被列出。
2.  **写** (w):在一个文件上，意味着文件的内容可以被改变。在目录中，这意味着您可以创建或删除该目录中的任何文件。
3.  **执行** (x):在一个文件上，表示该文件可以被执行。在目录上，这意味着目录的内容可以被访问。

# [](#lets-see-it-in-action-)让我们看看它的作用:

运行 ls -l 命令，它是一个带有-l 选项的列表目录内容命令，用于长列表格式，提供了关于每个文件的更多信息。

[![Long listing command](../Images/54e249c8602c0804595544ce2ffb996d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jabupmZR--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/g8q9i80uy04dj2vcv7a9.png)

每行都是一个文件或一个目录。你可以在最后一栏看到它的名字。但是前 4 列是什么呢？

*   第一列“-rwr—r—”是该文件的实际权限。我们一会儿将讨论它。
*   第二列是每个文件拥有的硬链接的数量。这与今天的话题无关。
*   第三列是文件所有者用户的名称。
*   第四列是文件所有者组的名称。

从上图可以看出。index.php 文件归用户 www-data 所有，其所有者组是 www-data。www-data 用户不是真正的人类用户，而是为 web 服务器进程创建并使用的用户。它用于限制服务器的访问和权限。在 Linux 中，每个服务都必须有一个用户来运行。

现在让我们详细讨论第一列。这里可以看到“-rwxrwxrwx”。这个字符串实际上由 4 个子字符串组成。

*   第一位确定文件类型“-”表示硬链接(文件)，“d”表示目录，“l”表示软链接(快捷方式)
*   接下来的 3 位决定了所有者用户的权限，“rwx”意味着他拥有所有权限。
*   接下来的 3 位决定了所有者组的权限，同样“r-x”表示该组中的每个人都有读、执行权限，但没有写权限。
*   接下来的 3 位决定了不是所有者用户或不在所有者组中的每个其他用户的权限，“r—”表示他有读权限，但没有写或执行权限。
