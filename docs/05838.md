# 开发人员避免 CSS 的原因

#### [](#break-the-css-stigma-real-developers-dont-design-and-dont-use-css)打破 CSS 污名:真正的开发者不会设计也不会使用 CSS

[![The reason developers avoid CSS](../Images/757f34174c9d865b35ea33ed3193e834.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--OJWDpz9_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://blog.sethcorker.com/content/images/2019/08/Artboard-Copy-3.png)

并不是所有的 web 开发人员都很了解 CSS，我与之交谈过的一些人认为他们不需要了解。除了我在 *[中提到的掌握 CSS 的挑战，新开发人员在学习 CSS 时犯的最大错误是什么？](https://dev.to/darthknoppix/what-is-the-biggest-mistake-new-developers-make-when-learning-css-5095)* 我想解决一个潜在的系统性问题。从开发者的角度来看，设计并不总是被尊重的。这被视为事后的想法。每个开发人员都害怕的最后一块拼图。它被视为妨碍实际工作的一种不可避免的邪恶。这种可悲的现实影响了开发人员与 CSS 和设计的整体关系。它在开发者和设计者之间制造了一个楔子，强化了负面的刻板印象和偏见。为什么 CSS 被降级到最后一分钟黑客或简单地传递给驻地 CSS 专家？

### [](#why-is-css-seen-as-inferior-by-developers)为什么 CSS 会被开发者视为低人一等？

通过我作为开发人员的经验和对设计的研究，我对许多开发人员眼中 CSS 的看法很感兴趣。语言和技术之间的竞争是不可避免的。它来自充满房间的笑话和强烈的观点，但 CSS 似乎比任何其他语言都受到更多的鄙视。

### [](#its-not-a-real-language)这不是真正的语言

从表面上看，反对 CSS 的观点是有道理的。它不是最直观的语言，也不是最容易掌握的语言。CSS 没有 JavaScript 这样的脚本语言的特性。它需要一种不同的思维方式，这与现有的假设相冲突。所有这些都是因为开发人员告诉自己，CSS 不是真正的语言，所以我为什么要学习它？

如果把 CSS 看作不是一种真正的语言，那么用它做任何有用的事情都会被认为是低劣的和具有挑战性的。一旦这种心态形成，它通常最不受关注。它被留到最后一分钟，并且由于急于完成某事或缺乏对最佳实践的理解，它是错误的常见原因。大型和/或长期存在的代码库加剧了问题，并给开发人员带来了恐惧。心态变成了一个骑士去与龙战斗；开发商面临着一个不可逾越的敌人。当他们努力让 CSS 屈服并在不破坏整个网站的情况下更新样式时，他们受到了祝贺。对“浪费时间做 CSS”的恐惧强化了另一个论点，即设计不那么重要。

### [](#it-doesnt-matter-its-only-design)没关系，只是设计

很难准确定位为什么一些开发者认为设计不重要。也许是缺乏对设计的理解意味着开发者缺乏欣赏力。也许这是一种防御机制，以避免穿上盔甲与龙战斗。当设计改变时，我听到过抱怨。我甚至对设计的特殊性表示了失望。如果你认为设计在网络上无关紧要，那么考虑一下这些事实:

仅靠 CSS 是不可能打造一个成功的网站的。

没有 CSS，你不太可能打造一个成功的网站。

设计被视为纯粹的美学，其核心焦点是向目标受众传递价值。这导致 CSS 被视为次要问题。这是拼图的最后一块，受到的关注有限。这又是一个我听过很多次的谎言。直接说“我们以后再考虑设计，让它工作就好了”，间接说“功能是最重要的，所以先关注它”。设计很重要，很难否认它不重要。即使是未经训练的眼睛也很容易发现糟糕的设计。有大量精心设计的网站和产品可供选择，设计是一个差异化因素。这是用户对你的网站的第一次体验，帮助用户确定他们是否信任你的产品。这导致了对设计的有趣的理解；这是一个女性化的话题。

### [](#why-is-design-perceived-as-feminine)为什么设计会被感知为女性化？

高中时，我除了艺术和设计课，还上了编程课。在大学里，我学习图形和媒体设计以及计算机科学。计算机科学和设计似乎是性别严重失衡的领域。在我的设计课上，女生比男生多一点点。然而，在计算机科学课堂上，男性至少占了 80%，这是一个显著的差异。我只认识几个和我一起学习计算机科学和媒体设计的人。

我之所以强调这种不平衡，是因为图形和媒体设计被认为是女性的主题。而计算机科学被认为是一门男性学科[【2】](https://news.gallup.com/poll/15202/public-believes-men-women-equal-abilities-math-science.aspx)。不幸的是这种看法存在，我相信两个领域应该是平等的，但这是一个很难打破的看法。STEM 学科和艺术学科之间的紧张关系使这种动态变得更加复杂。根据我的经验，一些开发人员持有一种潜在的信念，认为他们比设计师更重要。这种错误的无意识偏见是为什么设计被认为是劣等的核心，而事实并非如此。我们都必须努力克服这些挑战，并进行反思，以避免落入认为设计不重要的陷阱。

### [](#how-can-we-overcome-the-css-stigma)怎样才能克服 CSS 污名？

CSS 是整个网络的一部分。你不能无视它，希望它消失。不会的。最好的做法就是做开发者最擅长的事情，学习。一旦你认为 CSS 是你的 web 开发者工具箱中不可或缺的一部分，就拥抱它吧。让理解成为你克服对 CSS 的恐惧或怨恨的首要任务。恐惧是未知的，所以消除恐惧的唯一方法是深入了解它，并成为他人的火炬手。

如果你的设计经验有限，那么你可能不会欣赏设计中的重要工作和深入思考。UI 设计不仅仅是在这里或那里多留一些空白。这不仅仅是使用哪种颜色或者标题应该有多大的问题。许多因素影响着设计，就像编程一样。设计就是不用说一句话就能传达你的意图。它是黑夜中引导人们注意你的产品的探照灯。是看不见的手给用户指明了正确的方向。是胶水把这些碎片粘在一起。花些时间去学习设计，你会慢慢欣赏它。

在开发人员和设计人员或者你团队中的任何人之间竖起围墙是没有好处的。同理心是打破障碍和抛弃认知偏见的有力工具。平等对待团队成员的意见很重要。设计和 CSS 应该被整合到规划和开发中，作为构建网络产品的核心支柱。把它留到最后一分钟，表明你比你的设计师更重视你的时间。
