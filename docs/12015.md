# 我过了糟糕的一天

> 原文:[https://dev.to/mattbee_25/i-ve-had-a-bad-day-4odn](https://dev.to/mattbee_25/i-ve-had-a-bad-day-4odn)

我今天过得很糟糕。我正在转换一个遗留的 haml 模板来作出反应。我告诉自己，这很简单。在花了几天时间尝试与 Ruby、Drone(AWS 持续集成平台)和 Docker 一起工作之后，我认为这将会给我一点激励。那个 Ruby 的任务已经被搁置了，因为有更有见识的人把我们的 CI 整理成一些我在我们的应用上升级 Ruby 时需要做的事情。抱歉，我在瞎扯。

我以为我今天会收获颇丰。把这个简单的模板转换过来，我就可以在工作中展示一些进步，感觉有效率，也许还能在过程中学到一些东西。

我所学到的是，你给一个项目增加的复杂性、依赖性或技术(或任何东西)越多，你陷入的陷阱就越多。

我遇到的具体问题是该组件根本无法呈现。什么都没有，除了一个红色的大错误告诉我我做错了什么。

```
You might have mixed up default and named imports. 
```

不，我已经检查过 10 次了。

我正在处理的页面有什么不同吗？不，标准页，我可以渲染另一个组件罚款。

我给事物命名时有什么古怪吗？不，我没看到。

我开始怀疑我是否能成功。我在这上面花了 3 个小时。3 个小时只是为了得到一个简单的 React 组件渲染。没有逻辑，只是一些道具。

这应该很容易。

公平地说，我的信心受到了打击。

是我误读了错误信息吗？我做错了什么？是我在 Ruby、Rails、Hypernova、React、Webpack 或我们使用的许多其他工具中不知道或误解的东西吗？

但是后来灵光一现！我发现了它。我正在导入一个组件，它从一个命名组件列表中导入一个组件！React 是正确的(因为我想使用的图标在旧模板世界和新 React 世界中的命名不同，并且我传递的 prop 没有在组件中的组件中命名)。我的意思是我没有混淆默认和命名导入，但是我向一个奇怪的图标导入逻辑传递了一个错误的值。

我希望能爽一下。我做到了，3 个小时的调试，凝视，休息，思考，更多的凝视，最终我发现了问题并解决了它。

但是后来我有了一个想法。我很沮丧。接近愤怒。我回想起我做的关于学习 web 开发的演讲。我做这份工作已经 10 多年了。是的，我仍然缺乏一些技术知识，但是这不应该是很难的。但是由于我们使用的巨大技术堆栈，我迷失了方向，不知所措。我觉得 React 是受害者在责怪我。

更确切地说，他不知道 Rails world 和 React world 之间的图标名称已经改变。

在一天结束的时候，我确实犯了一个错误，但我觉得我有这么多地方可以找到问题，我有点迷失了，觉得我不知道下一步该去哪里找，如果我找到了，我甚至会理解问题吗。

我一直在这里漫谈，这比我预期的要长得多。但我学到的两件事是:

1)你拥有的技术越多，你就会陷入越多的小陷阱，而且不一定知道在哪里寻找问题。如果你是一个没有太多经验的沮丧的开发人员，不要因为不能解决问题而沮丧——我已经做了 13 年了，但仍然被卡住了(但是如果你认为我能帮上忙，请问我，因为有时我确实觉得我知道我在做什么！)

不幸的是，今天不是其中之一。
