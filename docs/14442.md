# 打破记录太容易了

> 原文：<https://dev.to/tuxhedoh/too-easy-to-break-a-streak-5cf1>

我有整整一周的时间。我发布更新，并确保完成我的任务...然后假期来了，我的时间表全变了，我错过了一个...二...现在已经 5 天了，我没有完成任何任务。我是唯一让我负责的人。

我发现当我在工作的间歇时，很容易跳到我的 FCC 或打开 VS 代码并开始工作，但当我在家时，我有很多时间，这还不符合“我在家做的事情”的类别。

我目前的问题，也是我在做前端挑战时经常面临的一个问题，是我不是一个真正的创意设计师类型。所以现在我在 FCC 的 HTML/CSS 挑战中构建一个致敬页面，但我并不关心如何让它看起来更好。我能很好地遵循这些要点来满足要求....

因此，两个障碍出现了，缺乏预定的时间，缺乏从创意的角度实际构建页面的承诺。

你做了什么来确保你在不同的时间表中保持在你的目标之上，在我的例子中是 100 天的代码？

当你不在乎这些任务时，你有什么技巧来完成它们吗？
