# 2019 年，开始一个 web 开发项目，哪个栈最好？

> 原文：<https://dev.to/prahladyeri/in-2019-which-is-the-best-stack-to-start-a-web-development-project-1kad>

我认为这些是目前主要的(在开源或非企业领域):

### 后端堆栈

1.  **PHP**
    *   symphony
    *   拉勒韦尔
    *   代码点火器
    *   CakePHP
2.  **Python**
    *   姜戈
    *   瓶
    *   桥塔
3.  **节点**
    *   表达
    *   哈皮

* * *

### 前端堆栈

1.  **温泉**
    *   反应
    *   角度(1/2)
    *   view . js-检视. js
    *   骨干网. js
2.  **非水疗**
    *   jquery
    *   d3.js
3.  **用户界面**
    *   引导程序
    *   尾翼 CSS
    *   材料-用户界面
    *   角形材料
    *   primeng

* * *

你认为这些(后端+前端)中哪一个最有机会适应未来？

我没有包括 Java 和 ASP.NET，因为它们只被大型企业使用，而且它们在 web 开发中的市场份额很小。

此外，还有一些小众后端堆栈，如 golang、ruby 和 erlang，但它们的份额要么很小，要么在下降。
