# 不要在面试中说负面的话

> 原文:[https://dev . to/mortoray/don ' t-say-negative-things-in-a-interview-1998](https://dev.to/mortoray/don-t-say-negative-things-in-an-interview-1998)

面试不是表达消极想法的地方。你是来给人留下好印象的，而这最好通过积极的观点来实现。即使你可能有最好的意图，也要小心一些陷阱。

说错话是[在编码面试](https://interview.codes/success/how-to-fail-a-programming-interview/)中失败的一种方式，所以让我们从三个角度来避免这种情况:

*   与面试官建立融洽的关系
*   避免判断性的观点
*   尊重面试官这个人

## [](#keep-the-good-rapport-to-the-end)将良好的关系保持到底

随着面试的进行，你会变得更加冷静。你和面试官之间建立了联系。这很好，因为它能让你清晰地思考。但是你可能会放松警惕。不管你认为自己相处得有多好，你仍然对面试官一无所知。

你正在发展的良好关系是因为你很随和，并且和面试官有共同的思维模式。你们俩都在同一页上。理想情况下，你会一直保持这种状态直到最后。为什么要冒消极的风险呢？

也许你正在用 Python 编码，你看到了与 C++相比的机会。这样做是好的，但前提是你能保持比较中立。不要泄露 Python 更优越的观点。如果面试官喜欢这种语言，那就太糟糕了。我们中的许多人都在用我们不喜欢的语言编程。面试官的热情可能在于 C++，他们甚至可能参与 C++讨论板。

以我的经验来看，脱离关系并不需要太多。我的面试进行得很顺利，但是感觉过于自信，说了一些我不应该说的话。每个人都保持微笑，但我可以看到紧张。当时我甚至没有意识到我的话可能会被误解。除了消极，我学会了完全避免挖苦和讽刺。

在桌子的另一边，我也遇到过一些优秀的候选人脱口而出让我感到惊讶的事情。这可能是一句小小的俏皮话，但它大大削弱了正在建立的善意。

面试和工作是不一样的。还没有人信任你，而观点需要信任的基础。你在努力赢得面试官的尊重。

## [](#are-you-qualified-to-be-critical)你有资格挑剔吗

许多批评需要对材料有深刻的理解。在 Twitter 上发表一个固定的观点可能没问题，但在面试中会显得肤浅。当然，有可能面试官同意，但更大的可能是他们不同意。

例如，假设您正在查看一些使用依赖注入的代码。面试会问你一个替代方法，这是一个合理的问题。您正确地回答了全球配置器也可以工作。然后，就像很多面试中发生的那样，你会被问到，“这是一个好方法吗？”

这是一个像陷阱一样的引导性问题——不管这是否是面试官的意图。确保你已经[仔细倾听](https://interview.codes/success/listen-carefully-to-your-interviewer/)了面试官，了解他们想要什么。关键的措辞“这样好吗”听起来像是征求意见。试着只关注你知道的细节。指出这些方法的优点和缺点，但避免说总的来说一个比另一个更好或更差。

如果你感到有压力要竖起大拇指或竖起大拇指，那就不要。可以说你的经验更多的是在一个领域，你对它感觉更舒服。你甚至可以说你甚至不太了解一种方法，不知道它最适合用在哪里。

但这并不意味着一种方法比另一种更好。很可能你知道的还不足以做出坚定的判断。很可能面试官也不太了解。不要冒意见冲突的风险。

问问面试官的看法。这种转移可以让你摆脱困境。此外，遵从面试官有助于建立我之前提到的那种融洽关系。

## [](#reallife-is-strictly-awesome)真人真事[严格牛逼](https://geni.us/XQCltZR)

你应该对编程以外的任何话题保持完全积极的态度。人们的个人习惯甚至比他们的技术偏好变化更大。如果面试官说他们喜欢早间卡通，你说:“酷，哪些？”然后把它们记下来，就好像你以后会看一样。如果面试官的桌子上摆满了木偶，你会说，“这些很棒，是谁做的？”如果事实证明面试官在业余时间是一名通灵侦探，你可以说“太棒了，我很想知道更多。”

如果这看起来很肤浅，那是因为它确实如此。但是你故意让你的大脑保持积极。你想阻止它随意发表评论的倾向。你想抑制任何笑声或任何可能对这个人的生活不屑一顾的东西。

没有人想被嘲笑，一点也不想。你认为面试官会如何评价你，那个贬低他们的陌生人？

## [](#be-attentive)殷勤

鉴于我参加过的一些采访，这个建议显然没有被很多人放在心上。当任何一种势头建立起来的时候，许多人关闭他们的大脑，让他们的嘴接管。

不要让这种事发生在你身上。

想好每一个答案，再给出。用赞成和反对来表达所有的观点。承认面试官有缺点，避免与他们发生冲突。

积极点。
