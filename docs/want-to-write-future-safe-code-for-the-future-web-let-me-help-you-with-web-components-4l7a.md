# 想为未来的 web 编写未来的安全代码吗？让我来帮你做 Web 组件吧

> 原文：<https://dev.to/umerkk164/want-to-write-future-safe-code-for-the-future-web-let-me-help-you-with-web-components-4l7a>

不管你是否闻到这种味道，事情正在向好的方向转变。
*Web Components 是一套不同的技术，允许你创建可重用的定制元素——将它们的功能封装在代码的其余部分之外——并在你的 Web 应用中使用它们*
这些是 W3 联盟的规范，是所有浏览器的标准(或者至少是承诺如此)。这项技术意味着可能很快就不再需要*任何*依赖第三方框架(你现在只需要依赖相关工具*)，安装膨胀，或者连续学习几个月，来制作一个看似基本的现代 web 应用程序。如果你想现在就为未来的网络编写代码，也就是说，在 2 到 3 年后，一群追随潮流的网络开发人员会出现在哪里，这是适合你的技术。这篇文章将帮助你开始。*

 *你可能会问，为什么你应该放弃冷静的反应，和充满棱角的，和 Vue😎这篇文章的其余部分旨在解释这一点。

很久以来，框架一直是开发 web 应用程序，甚至某种程度上是静态站点的垫脚石。当你被像“反应”、“有角度”或“Vue”这样的词困扰时，你可能会花两天时间去学习 HTML。这意味着学习 HTML(&CSS)和 JavaScript 不足以*有效地*编写甚至初级水平的应用程序。

你*需要*安装一些像 npm 这样的包管理器，经历一些命令行地狱，(假设你正在学习 react)安装 create-react-app，运行 create-react-app，这将需要半个小时或更长时间，即使你的机器高于平均水平(硬盘，唉)。
这是最简单的方法，现在，你还没有做任何事情。这只是你代码的脚手架。现在你需要更多的模板 react 代码，唉，你还需要 Redux，还有 20 个 react 库用于那个动画(这在现代网络中是理所当然的)，等等...您需要在挂钩和基于类的反应之间做出选择(这对您来说没有任何意义，因为您刚刚了解了 setTimeout last weak)。嗯，它并没有停止在这里，这是地狱，但你得到的想法。

这不是编写代码的方式。如果所有的框架**最终**都旨在实现相同的**基本**功能，并且你的客户使用几乎相同的浏览器和完全相同的互联网，为什么没有一个通用的标准方法，我们教开发者构建他们的应用程序，除非他们的项目*要求*一个专用的、高级的、固执己见的框架，适合他们的特定需求。如果 JavaScript Web 框架现在能为你我做同样的事情:正常的、理智的开发人员，正在寻找构建数百万已经存在的同类 Web 应用程序，为什么不呢？为什么不是未来的安全、非隔离、即插即用？

(请不要写这些 buzz 框架有什么不同，它们不适合我。它们不是为我想进入 webdev 的朋友准备的，也不是为 comp sci 毕业生准备的，它们对你这个 25 岁的 JS 开发人员来说是不同的，因为 Angular 使用双向数据流过程，它直接更新真实的 DOM，而 React 只更新虚拟 DOM 和 abracadabra alakazam hocus pocus)

我的观点是，如果你是一名 web 开发人员，并且对 JavaScript 和一些工具非常熟练，你应该将精力转向新的标准:Web 组件。也许不是直接在这个底层，但也许可以尝试 lit-html 和 Lit Element，它们都是非常优雅的解决方案，可以达到框架- **像**一样的功能，而不会让你一头撞在墙上。我上面链接的文章应该足够了。*
