# 选择云提供商的 6 个技巧

> 原文：<https://dev.to/eddiesegal/6-tips-for-choosing-between-cloud-providers-3igd>

公共云计算服务为开发人员和 IT 团队减少了麻烦，提高了灵活性。它们让您可以将许多 IT 问题(如文件存储、网站维护和数据备份)外包给第三方云提供商。通过减少硬件和内部 IT 人员的前期投资，云服务还可以大幅削减小型企业或新项目的前期成本。

然而，云服务之间的选择范围是巨大的，为您的项目选择正确的服务可能是一个真正的挑战。在本文中，我将深入探讨真正重要的东西，并为您提供根据您的需求选择最佳云提供商的七个技巧。

## 1。阐明你的要求

一个常见的错误是在没有明确定义自己需要什么的情况下就开始研究云计算。云服务的存在满足了许多不同的需求，在你寻找之前明确你在寻找什么是很重要的！

比如有基于云的[团队协作工具](https://www.skyhighnetworks.com/cloud-security-blog/top-10-cloud-collaboration-tools/)如 G Suite，云文件存储选项如 Dropbox 或(亚马逊网络服务)AWS，云 CRM 软件如 Salesforce。这些都是非常基本的用例，我建议您从这些用例开始，除非您具备考虑其他用例的技术知识。

随着业务的增长，您可能需要考虑更高级的云计算服务，以满足日益复杂的 IT 需求，例如可以在云中托管应用程序的虚拟计算资源，包括 AWS 和微软 Azure 的应用服务。

## 2。选择现收现付的价格

云的主要优势之一是它降低了前期 it 成本，这在开始一个项目时会成为您资源的真正负担。不同的云服务有不同的定价结构，但是，我建议您只选择那些提供现收现付定价计划的服务。

当然，成本通常取决于你使用不同云服务的多少，但云的一个巨大优势是你可以只为你需要的东西付费，仅此而已。与要求大量预付款的云提供商打交道时要小心。

## 3。想想云备份

云备份是所有云用户都需要思考的问题。如果您的本地服务器和计算机系统中有大量关于您业务的重要数据，那么在云中有一个备份选项是必不可少的，以防灾难袭击并擦除您所有的业务信息。

然而，为本地数据提供辅助位置并不是使用云进行备份的唯一方式。我已经提到过使用云来存储文件和在 AWS 等服务上运行应用程序。然而，AWS 服务也不能幸免于灾难，如果你想在使用云时格外谨慎，还有第三方的 [AWS 云备份服务](https://n2ws.com/product/aws-backup)值得考虑。

## 4。优先考虑信息安全

如果您决定使用云来存储敏感的公司文档和数据，在选择提供商时优先考虑信息安全是至关重要的。将公司文件委托给第三方会带来巨大的风险，因此与任何服务提供商建立基于信任的关系非常重要。

将安全认证、数据加密和严密的用户认证策略作为最低要求。此外，根据您所在的行业，您可能需要遵守某些规定您如何处理客户数据的法规要求。确保您的云提供商保证遵守您所在行业的任何相关法律。

## 5。密切关注 SLA

[服务水平协议(SLA)](https://www.wired.com/insights/2011/12/service-level-agreements-in-the-cloud-who-cares/) 是您、客户和云提供商之间的协议，规定了您可以期望的最低服务水平。在评估任何云提供商的适用性时，通读其 SLA 并留意重要方面，如提供商的安全实践、在数据丢失或中断时提供商可以做什么以及潜在的停机时间。

## 6。寻找好的文档

如果一家公司在其网站上提供了关于使用该服务的清晰文档，这是一个易于使用的服务的良好标志。例如，Amazon Web Services 有大量的指南来解释其服务的架构和用例，以及清晰的技术说明。

您希望最大限度地减少麻烦，并尽可能平稳地在云中启动和运行，好的文档对您的起步大有帮助。

## 总结起来

对于第一次建立 IT 运营的小型企业和大型企业的新项目，目标是最大限度地减少硬件资本支出，并找到可为您提供所需功能的随用随付服务。使用本文中的七个技巧来更好地决定在您的下一个项目中利用哪些服务和哪个云提供商。
