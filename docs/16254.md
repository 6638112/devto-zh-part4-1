网页可访问性是一个非常广泛的主题。为了让它看起来不那么令人生畏，我们可以把它分解成万维网联盟所说的可访问性的四个主要原则:可感知的、可操作的、可理解的和健壮的，或者称为 P.O.U.R .这四个原则是世界赖以存在的支柱*。我们一起来恢复七大王国*...抱歉，柱子不对...

# [](#perceivable)可感知的

定义:“信息和用户界面组件必须能够以用户能够理解的方式呈现给用户。”

可感知是指用户通过视觉、听觉或触觉收集你在软件或网站上展示的所有信息的能力。这意味着您的用户需要能够以某种方式感知页面的所有信息和 UI 组件。换句话说，它不可能对你的任何用户都是不可见的。

这可能是人们在想到可访问性时想到的最常见的原则，因为这与那些视力受损的人有关联。

重要的是要考虑如何以不容易被每个人理解的方式传达重要信息，例如，仅使用颜色传达某个字段是必需的，或者在网站上仅给出听觉指示。一定要确保你的内容以至少两种不同的感知模式传达。例如，在图像上使用描述性的`alt`文本，或者在红色必填字段旁边显示图标。

浏览您的应用程序，确保信息在多个渠道中呈现，并问自己以下问题:

*   所有图片都有相同的标题或文本吗？
*   您的`alt`文字是否描述了图像并传达了意图？
*   所有音频剪辑都有相关联的抄本吗？
*   所有的视频剪辑都有相关的文字记录吗？
*   你的文字颜色和背景颜色之间有足够的对比吗？
*   如果你有一个文本图像，比如一个扫描的文档，基于文本的替代方案是一个选择吗？

# [](#operable)可操作

**定义:**“用户必须能够通过可用的组件和导航来操作界面。”

“可操作”意味着用户必须能够使用软件的所有功能。对于一些用户来说，这可能意味着只使用 tab 和 enter 键导航网站。对于有学习障碍的用户来说，这可能意味着开发人员需要确保用户有足够的时间来响应定时的 UI 元素，比如警告。这甚至可能意味着该软件或网站对癫痫患者是安全的。

不是每个人都有使用鼠标的能力。有些人使用语音控制软件来浏览在线内容。其他人使用由手、脚或头控制的开关来导航。您的内容必须可以通过这些不同的输入模式进行导航和操作。甚至还有一些鼠标用户可以执行而触摸屏用户不能执行的操作，反之亦然。

考虑使用不同输入设备的工作流程非常重要。确保所有可以用鼠标完成的操作也可以用键盘完成。总是考虑用户在导航时如何知道他们在软件中的位置。始终考虑定时动作*实际上*是否需要定时，必要时考虑延长时间。

问自己几个问题:

*   你能只用键盘导航到页面上的每一个交互元素吗？
*   如果你能通过键盘导航到一个元素，你能离开它吗？(确保你的用户不会陷入“键盘陷阱”)
*   页面上所有元素的焦点顺序有逻辑意义吗？
*   您的表单元素是否与它们对应的标签相关联？
*   在你的应用程序中，所有可以交互的元素都很容易交互吗？

# [](#understandable)可以理解

**定义:**“用户必须能够理解信息以及用户界面的操作。”

内容或操作不能超出用户的理解范围。什么是“可理解的”很大程度上取决于应用程序及其上下文。在内部企业应用程序中，可理解性对于在公司内部工作的人和不在公司内部工作的人来说有着不同的含义。

它还包括在整个应用程序中使用可预测的导航和一致的标签，使用户能够构建思维地图。

规划可理解内容的最佳方式是尽可能将所有交互和信息保持在最低限度。确保你的用户没有信息过载的情况。

确保你的内容用通俗易懂的语言写，并解释任何行话或首字母缩略词。

问自己的问题:

*   代码中是否标识了站点的语言(例如，`<html lang=”en”>`)？
*   是否解释了不常用的词、首字母缩写词和行话？
*   图像和图形是用来帮助理解的吗(记住它们需要是可感知的)？
*   您的表单是否有适当的验证和错误消息，让用户了解在表单无法提交的情况下发生了什么？
*   对于辅助技术，除了主要内容的语言之外的其他语言的单词是否被正确标记？
*   用户是否容易理解他们在应用程序中的位置？

# [](#robust)健壮

**定义:**“内容必须足够健壮，能够被各种各样的用户代理可靠地解释，包括辅助技术。”

这意味着随着技术变得越来越好，用户应该仍然能够访问你的内容。健壮的网站使用标准化的标记，可以跨不同的浏览器、操作系统和硬件类型进行可靠的解释。您的用户应该能够在他们选择的任何设备上访问您的内容(平板电脑、智能手机、不同的浏览器等)。例如，如果你的网站只能在 Chrome 上运行，那么它就不够健壮。

创建健壮的产品允许用户定制他们的技术来满足他们的需求。例如，需要 Flash 的网页内容会将那些设备不支持该技术的用户排除在外。您的用户拥有的控制权越多，他们就越有可能轻松有效地访问您的内容。

**在每个浏览器中测试。**

问自己一个问题:

*   您的应用程序在各种浏览器、操作系统和平台(手机、PC、平板电脑等)中是如何出现和运行的？)?
*   如果用户通过较旧的浏览器或尚未开始支持某项功能的浏览器使用您的应用程序，您是否正在使用 [polyfill](https://developer.mozilla.org/en-US/docs/Glossary/Polyfill) ？
*   此应用程序是使用当前最佳实践开发的吗？

# [](#closing-thoughts)关闭思绪

实施 P.O.U.R .原则将您的客户和用户放在首位。一般来说，网页可访问性是指以一种包容的方式进行设计和开发，这样每个人都有机会从你的网上商店购物，阅读你的博客文章，或者以一种更有效的方式完成一项任务。
