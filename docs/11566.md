# 我犯了一个大错

> 原文:[https://dev.to/theryanfurrer/i-ve-made-a-huge-mistake-cek](https://dev.to/theryanfurrer/i-ve-made-a-huge-mistake-cek)

## [](#being-overzealous)过分热心

我想这个星期我做了力所不及的事情。我并不是说我不能做我想做的事情，而是我严重高估了我对 JavaScript 的适应程度——甚至像创建待办事项列表这样的基本事情。

昨晚我坐下来，没有做我原本计划要做的事情，也就是完成 [doMore](https://do-more.netlify.com/) 的设计，而是决定搞乱 JS。事情是这样的:

我慌了。

我不知道为什么，但我做了。我一直很兴奋终于可以离开树屋课程独自和 JS 一起工作了，我愣住了。所以我回到了我最初的最终设计计划，尽管这比预期的要长。我觉得自己有点失败。

现在我知道这是荒谬的，我只做了几个月，我就一头扎进了 JS，然而，我仍然觉得自己很失败。然后我提醒自己失败也没关系。感觉不舒服是正常的-天哪，在 JS 上工作我感觉不舒服。

## [](#feeling-uncomfortable)感觉不舒服

[![](../Images/c2a642b736ec0566026a7c6eaa9353d1.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_3Z99MnE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5hleppkphbdep9w1rmkv.jpg)

糟透了-不可否认。这让我感觉更糟，因为我为自己擅长 CSS 而自豪，甚至感觉昨晚花的时间比应该花的时间还长。

也许这是一个糟糕的夜晚，或者我不能真正集中注意力的事实，或者也许我只是不知道这些东西，就像我认为我知道的那样。不过没关系。每个人都有糟糕的日子。每个人都有感觉事情比想象中更困难的时候。你有没有一天觉得起床就是一个巨大的挑战？

> “走出你的舒适区。只有当你愿意尝试新事物时感到尴尬和不舒服，你才能成长。”——布莱恩·特雷西

随着我想尽我所能成长为最好的开发人员，我将按照这些话继续前进。我有这样做的动力和(我认为)职业道德。我可能不得不调整下周的目标，以便给自己留出更多时间来完成本周的目标，这没关系——我正在调整我的最后期限。

当你觉得自己失败了的时候，你通常会做什么？你有什么克服它的建议？

我知道这个博客和我以前的有点不同，但是每个人都会经历这些时刻。我没什么不同。
