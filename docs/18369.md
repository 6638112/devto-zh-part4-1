在一次会议演讲后，我经常被问到的一个问题，奇怪的不是关于我演讲的内容，而是关于我的 Linux 桌面环境。人们更感兴趣的是那个漂亮的发行版，而不是我刚刚做的精彩演示😂

我不是在抱怨，我喜欢我的桌面设置。我太喜欢它了，以至于当我该买一台新电脑的时候，我害怕再买一台。我担心我会把事情搞砸(我过去已经这样做过很多次了，我想 Linux 用户会认同我)

所以我决定为那些有兴趣使用 Linux 作为主要开发操作系统的人捕捉我的发行版中最重要的方面。

[![](../Images/ad6a18bd673c1ca3ef4ffcb0ad560894.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--OwCE4hGr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3zajm5va5xplo9mg7804.png)

这不仅仅是我的工作笔记本电脑；这是我的主要机器，我用它来做下面所有的事情。

*   Java、JS、TS、Go、Python 和 web 开发
*   捷普斯特开发公司
*   在本地运行多个 web 应用程序
*   运行码头集装箱
*   用于 Windows 测试和其他虚拟机的 VirtualBox
*   Kubernetes、Terraform、CloudFormation 开发和部署
*   使用必需的 CLI 工具进行 Azure、AWS 和 GCP 部署
*   大量使用浏览器
*   电子邮件、聊天和视频会议
*   Plex 媒体服务器
*   博客
*   Youtube 和社交媒体

## [](#machine-configuration)机器配置

## [](#operating-system-and-desktop-environment)操作系统和桌面环境

最重要的，当然是操作系统。我目前正在以 [GNOME 3.32.2](https://www.gnome.org/) 为桌面运行 [Fedora 30](https://getfedora.org/) ，我对它非常满意。我发现 Fedora 比其他发行版更适合开发机器，因为它发布周期短，而且相当稳定，所以你可以一直得到最新的&稳定软件。

## [](#development-tools)开发工具

现在，这些大多是客观的选择，只要你对自己选择的工具感到满意，这些都没关系。以下是我对一些重要发展类别的选择。我没有包括 Vim、Git、NodeJS、Docker、Kubernetes 等显而易见的东西。

壳牌:这对于开发者来说是最重要的。我使用 [ZSH](https://www.zsh.org/) 和令人敬畏的[哦我的 ZSH](https://ohmyz.sh/) 作为我的外壳。现在，如果没有一些漂亮的插件和主题，这将是不完整的。我使用 [powerlevel9k](https://github.com/bhilburn/powerlevel9k) 主题进行一些定制。我还为我的 zsh 使用了[zsh-自动建议](https://github.com/zsh-users/zsh-autosuggestions)、git、docker、docker-compose、autojump、[zsh-语法-高亮显示](https://github.com/zsh-users/zsh-syntax-highlighting)、dnf 和 npm 插件。这是我的 [`.zshrc`](https://gist.github.com/deepu105/9723cb70032eca98ff09d110369af67a) 和所有的定制。**更新**:这个帖子上的一个[评论](https://dev.to/java4africa/comment/c319)建议将 [powerlevel10k](https://github.com/romkatv/powerlevel10k) 作为替代主题，我试了一下，结果它确实比`powerlevel9k`快得多。所以我想我会用`powerlevel10k`作为我的外壳主题。

**终端**:没有好的终端，一个好看的外壳有什么用。幸运的是，我们有 [Tilix](https://gnunn1.github.io/tilix-web/) ，最好的终端应用之一。它有工作区、标签、分割窗口、地震模式等等。

[![](../Images/979ad74bbde471f7c21ac1db028b062e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---IaB97Rh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/flq8pew1hr6l1hwj6egw.png)

**集成开发环境**:[IntelliJ IDEA Ultimate](https://www.jetbrains.com/idea/)——我用这个进行 Java &其他 JVM 语言的开发

**代码编辑器** : [Visual Studio 代码](https://code.visualstudio.com/)——我的得力助手。我喜欢它。我用 VSCode 做 web 开发，Go，Python，JS 开发，DevOps，还有 JVM 语言以外的一切。没有一些好的插件，VSCode 的设置是不完整的。[这里的](https://gist.github.com/deepu105/4599d3b381218b9d1a63119fbf9ca537)是我正在使用的插件。您可以运行脚本来安装它们。

我使用的其他值得注意的开发工具有用于 Git 回购管理的 [GitKraken](https://www.gitkraken.com/) 、用于代码比较的 [Beyond Compare](https://www.scootersoftware.com) 、用于 NodeJS 版本管理的 [VirtualBox](https://www.virtualbox.org/) 、 [NVM](https://github.com/nvm-sh/nvm) 以及用于 JDK 版本管理的 [SDKMan](https://sdkman.io) 。

## [](#productivity-tools)生产力工具

生产力工具也相当重要，以下是我的选择。

浏览器:谷歌浏览器是我的主要浏览器。我有时也用火狐& Opera。就 UX 而言，我确实喜欢 Opera，我想把它作为我的主要浏览器，但我想念我在 Chrome 中与谷歌账户同步的一切。

电子邮件:我用 T2 的 Mailspring 作为我的电子邮件客户端。这是一个相当不错的邮件客户端，有很好的主题和简单的用户界面。

办公套件:我通常使用谷歌文档(Google Docs)&和微软在线办公软件，但当我需要在桌面上处理一些东西时，我会使用 [LibreOffice](https://www.libreoffice.org/) ，这是一个很好的办公套件，甚至可以处理微软 Office & Keynote 格式。

**通讯**:当然我用 [Slack](https://slack.com) ，视频会议我用[蓝战士](https://www.bluejeans.com/)。

**截屏**:我用这个叫做 [Peek](https://github.com/phw/peek) 的漂亮工具进行截屏，用[快门](https://launchpad.net/shutter)进行截屏。

## [](#conclusion)结论

我还使用了许多其他小巧的实用工具；大多数是命令行实用程序。有一些值得注意的提及，如 [Timeshift](https://github.com/teejee2008/timeshift) ，这是很好的备份你的机器。

当然，在 Linux 世界中，并不是一切都是完美的，但对于每个操作系统来说都是一样的。在切换到 Linux 之前，我是一个长期的 Windows 用户。所以像每个 Linux 用户一样，我时不时会把事情搞砸(能力越大，责任越大，Peter)。Linux 世界有很多怪癖，但是没有什么让我很困扰的。下面是我过去遇到的一些最烦人的问题，现在，我没有任何明显的问题。

*   切换应用程序时滚动位置跳跃-升级到 Fedora 30 后修复
*   休眠被打破-升级到 Fedora 30 后修复
*   插入耳机时音频输出选择被破坏-为我修复了 Fedora 28 之后的问题
