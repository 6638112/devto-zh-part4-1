# 请不要忘记写变更日志

> 原文:[https://dev . to/some dood/please-don-t-forget-to-write-the-changelog-38 M6](https://dev.to/somedood/please-don-t-forget-to-write-the-changelog-38m6)

开发周期中一个经常被忽视、不被重视、看似平凡的阶段是编写变更日志的过程。对于那些没有意识到的人来说，变更日志基本上是一些软件自上次更新(或版本)以来的新功能、变化、调整和错误修复的简单列表。变更日志可以根据需要而变得冗长，但是为了简洁起见，通常不包括 bug 修复列表。

变更日志很棒，尤其是从用户的角度来看！作为一名用户，我真诚地期待我所使用的软件的新版本。我对新功能非常感兴趣。就电子游戏而言，没有什么比期待新的游戏机制、角色、物品和任务(以及其他功能)更令人兴奋的了。优化的承诺和过多的错误修复只是满足了我内心的完美主义者。

但是，唉，没有什么比一个懒洋洋地写的变更日志更能破坏我的兴奋了。我相信我们在生活中的某个时候都至少见过一个通用变更日志(如下所示)的许多变体中的一个:

> “错误修复和改进。”“修复了一些性能问题。”
> “让 app 变得更好。”
> “确保打开自动更新以获得最新功能。”
> “开发商未提供的信息。”

这种适度的变更日志有很多理由。有人可能会指出，时间和资源对于开发周期中的变更日志来说是一种奢侈。其他人，尤其是开发早期和实验性软件的开源维护者，可能会认为源代码控制的提交日志(如 Git)作为变更日志已经足够好了。

无论哪种方式，都不能提供良好的用户体验。它要求用户为了获得更新信息而费尽周折。在最糟糕的情况下，一些用户甚至会因为害怕引入意外和不受欢迎的应用程序行为而对安装新的更新感到犹豫。提供一个合适的变更日志减轻了这种“更新偏执”的感觉。

在这篇文章中，通过比较各种例子，我将讨论许多伟大的方式来编写一个更好的用户体验的 changelog。作为其中一个例子的额外收获，我还将讨论如何使用变更日志来利用“炒作”来建立品牌身份。

> **免责声明:**我与我在下面举例提到的任何品牌都没有关系。这不是某种#赞助的推广贴或任何类似的东西。我只是觉得，作为开发人员，我们可以从下面的例子中学到很多关于编写伟大的变更日志的东西，不仅仅是为了变更日志，而是为了良好的用户体验。

# [](#exhibit-a-discord)展品一:不和谐

[![Alt Text](../Images/53f177c10bf6ced4b733c1204cab04ae.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--O1RveiiR--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ineeucasw2zk75eehfm5.png) 
[【不和】](https://discordapp.com)是一款主要面向游戏玩家的实时通讯应用。基本上就是 Skype...但是对于游戏玩家来说。它支持在线消息、语音通话、视频通话和屏幕共享。作为游戏社区中的一股力量，它的主要优势在于它能够为(某个游戏的)粉丝提供一个集中的平台来相互交流和分享兴趣。

然而，我发现 Discord 最令人印象深刻的是它如何通过其变更日志的随意和轻松的语气吸引其游戏用户群。就我记忆所及，他们的变更日志中戏谑的行文——偶尔有意引用流行文化——对他们品牌的个性来说是标志性的和重要的。changelog 不仅成为新功能的信息参考，也是品牌与游戏玩家社区关系的核心方面。

# [](#exhibit-b-visual-studio-code-vs-code)附表 B: Visual Studio 代码(VS 代码)

[![VS Code Logo](../Images/7e951b44b1d98083b4038ebc06a3c680.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--P5gP5R8M--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9kjrhvnkwd0hmuoxfvgy.png) 
既然我们都是这里的开发者，我想我没必要解释什么是 [VS 代码](https://code.visualstudio.com)。2019 年栈溢出调查证明了一个事实，即它确实是开发者中最受欢迎和最受青睐的文本编辑器。

与 Discord 类似，VS Code 也通过更专业的语气吸引其开发者用户群。[它的变更日志](https://code.visualstudio.com/updates)充分详细地解释了前端和后端的新功能和调整，并附有 gif 图以增加清晰度。发行说明中额外的严格措辞对于开发者的细致观众来说是合适的。通过这样做，VS Code 已经确立了自己作为一个严肃的行业级软件的地位，无论是业余的还是专业的开发人员都可以使用。

# [](#exhibit-c-paintnet)展品 C:Paint.NET

[![Paint.NET Logo](../Images/110c1835d6b9481cdee493a49d65d4a6.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AdoFfWBG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/w4c9gph2maknjj6gder3.png)
[Paint.NET](https://www.getpaint.net)是一个简单且可扩展的图像处理程序，由 [Rick Brewster](https://twitter.com/rickbrewpdn) 维护。它原本是作为微软画图的替代品，但是这个项目发展到不仅仅是。除了常见的图像处理工具，如裁剪、调整大小、颜色调整、混合和形状插入，还有一个画图工具。NET 最显著的特点是它的插件系统，用户可以通过分布式 dll 扩展标准功能。就我个人而言，我自己使用 Paint.NET 来创建我的文章的封面图片(以及其他应用程序)。

关于 changelogs 的话题，[画图。NET 的修改日志](https://blog.getpaint.net)似乎比之前提到的那些*更驯服*。它有组织的信息呈现是显而易见的。使它区别于其他变更日志的是它的直接性和对可读性的考虑。它清晰的格式表明一个好的变更日志不需要太奢侈。只要它实现了正确传达*变化*的主要目标，就能确保良好的用户体验。

# [](#exhibit-d-minecraft)展品 D:《我的世界》

[![Minecraft Logo](../Images/ef9d2e8ddd94b60bac01b535a8b95ea8.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--hL0rQ6E2--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ac3tneikxpv5mjsbciaz.png) 
不言而喻，是最近一段时间最受欢迎的电子游戏之一。它曾被堡垒之夜短暂推翻，直到它再次流行起来，这很大程度上要归功于瑞典 YouTuber 的支持。

《我的世界》的变更日志在它的长期成功中扮演了一个真正重要的角色。正如本文开头所提到的,(全部或部分)变更日志会产生“炒作”,尤其是对视频游戏而言。“炒作”源于对新游戏特性的期待。这是流行文化中游戏相关性的生命线。

通过提供变更日志，《我的世界》已经能够通过“炒作周期”在游戏文化中牢固地巩固自己。YouTube 上关于游戏最新功能的视频已经使用官方发布的变更日志作为他们传播即将到来的重大更新的主要信息来源。

对《我的世界》来说，变更日志已经成为不可或缺的传播工具。由于公布的新功能产生的“宣传”,玩游戏时的用户体验显著改善。玩家可以更容易地浏览游戏的最新内容。到下一个主要更新的过渡变得无缝和期待已久，而不是意想不到和不受欢迎的。一个积极的结果是，玩家会更兴奋地玩游戏。

因此，“炒作周期”在每次重大更新后都会重复...

# [](#conclusion)结论

编写好的变更日志是一门值得掌握的困难艺术。变更日志不仅仅是交流变更的一种方式(从以前的版本开始)；它们也是品牌身份和个性的一种有意义的表达，吸引目标受众。此外，特别是对于视频游戏，变更日志会产生“炒作”，巩固游戏的整体影响力和相关性。

一个好的变更日志(或多或少)包括以下品质:

*   它必须**信息丰富**并且对用户有价值。
*   肯定是**组织的**。
*   根据需要，它可以是简单明了的。
*   如果有必要，可以通过 gif 和图像获得更大的清晰度。
*   它可以表达一个**品牌的身份和个性**。
*   可以产生**“炒作”**。

将这一理念应用于变更日志最终会带来更好的用户体验。用户再也不用在更新应用时经历“更新偏执”了。此外，用户参与度的提高是“炒作”的结果。简单地说，更好的变更日志会带来更好的用户体验，从某种意义上说，用户可以更好地了解他们使用的软件。

请不要忘记写变更日志。你不会想成为漏洞修复和改进先生。
