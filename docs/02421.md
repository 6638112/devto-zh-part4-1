## [](#captains-log)船长日志:🤦‍♂️

你知道当你开始一个新的兼职项目时，你充满乐观，一切都很顺利的感觉吗？那是一种很棒的感觉，你可以永远保持这种感觉，永远不要开始那个兼职项目。

对我来说，事情很快就从好变得很好，又变得很糟。我不知道从哪里开始。是的，我知道:xcode。

[![family guy peter griffin grinds my gears](img/21398f15331f3539657fb4394c8030fe.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_uu2Hfj5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fwvarxmss4osf422q35r.jpeg)

啊 xcode，每个人都喜欢讨厌的编辑器，但是有很好的理由。太恐怖了。随着 swiftUI 的引入，今年的测试版尤其糟糕。一切都很顺利，直到你把任何`ObservableObject`添加到你的应用中。然后预览开始失败，永远不会恢复。最终我了解到预览只对快速的视觉调整有好处，现在就把它关掉。

还有一件事真的磨我的齿轮，这是一个流氓斯威夫特进程旋转我的 CPU 风扇就像它是一个该死的阿帕奇直升机。关闭 xcode 甚至不能停止它，不，这个过程需要 [Rey](https://en.wikipedia.org/wiki/Rey_(Star_Wars)) 因为我不得不强制退出它。

## [](#progress-still-happened)进步还是发生了

学习任何新事物都会引起一些暴躁的猫的时刻，但 swiftUI 真的很棒。也许我应该关注积极的一面。我确实得到了一个工作的选择器，并将应用程序状态序列化到一个 json 文件中，我将在明天讨论具体的技术内容
