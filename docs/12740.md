# [](#introduction)简介

这是一系列的帖子，在这些帖子中，我分享了我为大学毕业设计构建一个非常基本的面向消息的中间件的经验。介绍了系统的设计并给出了源代码。

通过这个项目，我想提供一个基础设施，使企业能够为他们的系统实现集成解决方案。这里，我所说的集成是指企业不同应用程序之间的通信。更具体地说，这种通信遵循消息传递集成风格。

警告:这是我第一次尝试构建这种项目，因此，你不应该把这里的任何东西作为最佳实践或正确的做事方法，除非它有一个参考。最好把这个当做原型。此外，对于大学来说，许多设计决策都考虑到了这一点。

*   在第一部分中，我们将展示系统的架构并介绍其主要组件(消息模型、协议、模块...)以及我们将使用的框架/工具。这更像是对我们将要做的工作的一个概述。

*   在第二部分中，我们简单介绍了 Seastar，然后开始编写我们的“服务器”代码，这是我们应用程序的主要入口点。最后，我们展示了如何编译这个项目。

*   本系列的第三部分是关于为 MOM 设计的消息传递模型。诸如它是什么，我们为什么需要它...将被探索。此外，在这一部分，我们展示了 Mess 的实现。模型。

*   第四部分将讨论 Moza 协议(我知道的奇怪的名字)。在这里，我们将介绍该协议，解释它，并显示其 PDU 的结构。此外，我们将实现一个 cli 来与我们的 MOM 交互。

注意:我不确定这是不是我想写的全部，所以我可能会添加其他部分。

我希望这一系列文章能对你们中的许多人有所帮助和启发。特别是礼宾部分和 Seastar 的使用。你应该注意到在第一版中我没有充分发挥 Seastar 的潜力，但是我计划做一些改进来提高性能。
