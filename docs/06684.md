# Kotlin vs Python 为什么 Kotlin 作为第一编程语言可能比 Java 和 Python 更好？

> 原文:[https://dev . to/dzh/why-kot Lin-may-be-better-than-Java-and-python-as-first-programming-language-37 D5](https://dev.to/dzh/why-kotlin-may-be-better-than-java-and-python-as-the-first-programming-language-37d5)

在第三届特里瓦格夏令营的第三天下午 3 点左右，我和我们的一位导师马蒂亚斯·恩德勒就编程语言进行了一次小小的讨论。问题是，在当今世界上，每年都有数百种编程语言似乎坚持使用一种语言，成为这种语言的“铁杆粉丝”，向它祈祷，随时随地尽可能地使用它，在这样的世界上，这种语言还有意义吗？

现在我每天使用越来越多的 Kotlin，毫不奇怪我来自 Java 背景。相反，Matthias 作为 Python 的后端开发人员有很好的经验。

根据我个人的学习经验，我认为如果你学习了 Java，那么你在理解其他语言和编程本身方面不会有太大的问题。所以我向马提亚斯提出“作为第一编程语言，学习 Java 比学习 Python 更好”，但是马提亚斯不同意这个论点。不仅如此，大约 10 年前，他写了一篇文章，提出了完全相反的建议[矫枉过正——将 Java 作为第一编程语言](https://matthias-endler.de/2010/overkill-java-as-a-first-programming-language/)。
我的意思是…来吧，伙计们…这是 Java！谁不喜欢 Java？！:D

所以我读了他的文章，我同意他的一些观点。例如，我同意关于简单的“Hello world”示例的观点，当您作为一个新手可能会对一次在这么小的代码片段中出现这么多未知的东西感到震惊。我还记得我上学最后一年那段令人气馁和失眠的时光，因为我们是在 [Doctor Java IDE](http://www.drjava.org) 开始学习 Java 的，博士前缀本身当时就已经吓到我了。有时候，仅仅因为我漏掉了某些行中的分号或者输入了错误的变量名，我就浪费了数小时的“调试”时间。

然而，回顾过去，我很感激 Java 博士，因为他从第一天起就向我展示了“编程不是用奶昔和巧克力饼做的花哨东西。99.99%什么都不是。程序要么有效，要么无效”。

因此，我认为现在声称“Java 比 Python 更好”是不太正确的，因为我没有足够的 Python 经验，所以我提出了另一个建议“为什么 Kotlin 作为第一编程语言可能比 Java 和 Python 更好？”。

Kotlin 的伟大之处在于，您仍然拥有或多或少与 Java 相同的概念，但是最终您可能会在提供几乎相同信息量的情况下拥有不太陡峭的学习曲线。你不需要了解任何关于 OOP 的知识就可以开始写一些代码。另一方面，如果您开始学习编程时没有使用像 Eclipse 或 Intellij IDEA 这样的高级 ide，那么用 Kotlin 编写代码可能不会那么令人沮丧，因为您不必寻找遗漏的分号。

同时，当我们比较 Kotlin 和 Python 时，Kotlin 的一大优势可能是它被大量用于移动开发。作为一名学生，你可能会问自己“我为什么需要它？这些知识在哪里可以实现？”对于许多学生来说,“移动开发”可能比神奇的“Python 机器学习”更可行、更有动力，在“Python 机器学习”中，你基本上有成千上万的数学知识。

另一个有趣的方面可能是，大多数计算机科学“编程”课程要么基于 Java，要么基于 C++(下降趋势)。我从来没有听到有人说“在大学我们开始用 Python 学习编程概念”。由于 Java 和 Kotlin 之间的兼容性和相似性，使大学课程适应 Kotlin 比适应 Python 要容易得多。因此，如果大学不想停留在技术的过去，那么 Kotlin 可能是他们的最佳和非激进的解决方案。

在结论中，我希望所有的学生在开始学习编程后都要勇敢，不要放弃，因为有时它可能看起来令人沮丧和沮丧，但我相信我们都经历过这一点。给科特林一个机会，我相信你不会后悔的。
