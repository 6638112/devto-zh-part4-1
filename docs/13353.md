# 书评:托里·波德马耶斯基的 UX 战略写作

> 原文:[https://dev . to/wired ferret/book-review-strategic-writing-for-UX-by-torrey-podmajersky-4 hcf](https://dev.to/wiredferret/book-review-strategic-writing-for-ux-by-torrey-podmajersky-4hcf)

披露:我得到了这本书的评论副本。

进一步披露:托里是粉发科技人的一员。我们是军团。

* * *

这本书是对那些在用户界面上创造奇迹的词汇的一次敏捷而集中的探索。波德马耶斯基准确地识别出需要多说的地方，然后避开，因为这不是那本书。老实说，作为一名作家，这给了我灵感，也让这本书看得很紧。

在很多方面，这让我想起了凯西·塞拉的惊人之作《坏蛋:让用户变得棒极了》——这两本书都不懈地关注用户、体验和同理心。对我来说，最有用的两个部分《UX 的战略写作》是关于反复编辑以达到特定目标的一章，以及关于 UX 文本模式的一章，这一章列出了许多我知道但没有很好词汇的东西。

这本书非常简洁和有用，我打算把它添加到工作中的图书馆，因为我认为围绕这一点拥有一个共享的词汇表对每个人都有好处，即使我们不直接做设计或开发。里面全是我想钉在墙上的精辟有力的名言，比如:

> 人类没有反馈就无法提高。我们可以做出改变，但是如果我们不注意，我们就不能确定这些改变是好是坏。

和

> 最基本的可用性是可访问性。

阅读如果:

*   你没有 UX 人或团队，但你很确定 UX 需要发生在你的产品上
*   你是 UX 人，你正在寻找一种方法来组织你的 30/60/90 天计划，并向其他人解释你的工作的商业价值
*   你是开发商，你认为 UX 不是你的问题。(是)

跳过如果:

*   你整天只和机器说话
*   (您仍然应该阅读关于错误消息的部分)
