# 有效改进

> 原文:[https://dev.to/stickwarslit/improving-efficiently-d34](https://dev.to/stickwarslit/improving-efficiently-d34)

在某个时候，你想成为一名更好的程序员。你这样做的理由可能非常多。开发商赚了很多钱。软件可以改善用户的生活。不管是什么原因，提高代码质量会让你达到目的。现在意识到这一点，你会怎么做？

首先，你可以专注于你正在写的代码。你通过 linters 运行你的代码。你写更多的文档。删除重复的代码。你制造更多的描述函数。你应用你浏览互联网时偶然发现的每一个好主意和最佳实践。

但是，你会在一天中的某个时刻。也许是午餐时间到了，现在你因为在餐馆吃了一顿大餐而感到昏昏沉沉。或者也许两小时睡眠的影响最终影响到了你。慢慢地但肯定地，你的注意力减弱了，留给你一个专注自我的空壳。

尽管很累，你还是继续前进。毕竟，你是一名编码员(或者试图成为一名编码员)。除了编码，编码员还能做什么？前进吧。

没有意识到，你的输出质量下降。你从 stack overflow 复制代码片段，却不知道它是如何工作的。你跳过失败的测试，这样你的更新就通过了。您将 random booleans 设置为 true，并希望代码能够工作。无论如何都要完成这个功能或门票。只是编码编码编码。

你的目标是什么？成为更优秀的程序员？你看着你的代码，一团乱七八糟的意大利面，对未来的维护者来说将是一场噩梦。与你开始工作之前相比，你已经成为一个更好的开发人员了吗？一开始你在实施好的想法。然而，这些想法并没有反映在最终产品中。

这个结论乍一看似乎有悖常理。编码是一门技能，技能需要实践来磨练。掌握一项技能需要 10000 小时的练习。有几十家初创公司每天工作 10 个小时，很少休息。你只是在做你应该做的事情，增加你的时间来增加你的能力。

实际上，你的工作量是有限的。有可能工作太少；每天写代码会比一周写一次让你走得更远。但是人类每天只能有几个小时的积极专注。在这段时间里，工作会磨砺你的技能。问题是，一天中剩下的时间该做什么？

有许多低强度的活动可以提高你的编码能力。可以看看相关技术的文章。如果你觉得与人交谈很舒服，你可以在讨论板上交谈或者参加 IRL 会议。也许，你甚至可以开始写博客。虽然做这些不会直接提高你的技能，但它们有望让你的头脑充满好的想法，当你实际编码时，你可以实现这些想法。

我并不主张总是做一些编程或技术相关的事情。你需要休息一下。去散步。锻炼身体。花时间和心爱的人在一起。睡吧(拜托)。还要记住，虽然被动地从阅读和交谈中获取信息是很好的，但你仍然需要经常编写代码来取得进展。请记住，当您停止向文本编辑器中写入内容时，编码并不一定要结束。
