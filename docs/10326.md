# 与游戏发行商合作的陷阱

> 原文:[https://dev . to/nprimak/陷阱-与游戏发行商合作-poe](https://dev.to/nprimak/pitfalls-in-working-with-a-game-publisher-poe)

全世界有成千上万的独立游戏开发者制作游戏。但是，只有一小部分开发者有与游戏发行商合作的经验。当我发现有机会为一家发布教育游戏的初创公司工作时，我就是其中的一员。我向公司提交了我的游戏作品集，不久就被接受了。这是一个超级激动人心的时刻。我迫不及待地开始与一个合法平台上的游戏发行商合作。

不幸的是，我在这家公司的工作经历并不理想。也许早期有一些警告，但我不知道该寻找什么迹象。此外，该公司似乎急于与我分享如何成功完成项目的信息。他们与我进行了视频通话，并通过电子邮件向我发送了演示所需需求的幻灯片。他们立刻给了我一个平台，其他开发者可以在那里提交他们的游戏，这样我就可以得到一些灵感，知道我应该制作什么样的游戏。

我立即注意到，通过该平台提交的许多游戏都非常简单。这是有道理的，因为要求在三个月内完成游戏。尽管如此，我还是很有信心能做出一个更有趣的游戏。让游戏变得更吸引人显然是有动机的。开发者按玩他们游戏的用户百分比获得报酬。

我没有意识到，也没有让我清楚的是，我制作的游戏必须在旧款 iPads 的互联网浏览器上无缝运行。这就是我在教育平台上看到的游戏如此基础的原因。不幸的是，在我入职期间，我的游戏需要有多简单的技术问题并没有被讨论。我不知道我不能有一个三维游戏，你可以用箭头键在一个环境中移动一个角色，因为运行 Chrome 的 iPads 不能以那种帧速率处理渲染。

当测试人员审查我的游戏时，这对我的游戏开发动机和兴奋度是一个巨大的打击。他们说在要求的平台上无法播放。这也是一个震惊，因为我已经多次使用该公司提供的测试平台。在我提交游戏之前，我虔诚地在测试平台上玩了一遍。如果“通过”测试实际上并不意味着它将在最终系统上运行，那么公司提供一个测试框架似乎是非常违背直觉的。

我试图通过降低复杂性来重新调整我的游戏。首先，压缩图形并简化环境中的 3D 模型。经过另一轮测试后，我意识到如果不做巨大的改动，我的游戏不可能在 iPad 的网络浏览器上运行。此时我只剩下几周时间了。

<figure>

你们中的一些人可能会奇怪，为什么我不自己制作一个游戏并在 iPad 上运行。问题是游戏必须连接到公司专有的 API，而这些 API 只能在测试平台所在的公司域上运行。更糟糕的是，他们的 API 也不能在 iPad 上运行——只能在电脑上运行。

<figcaption>A brief video preview of my game, Grand Canyon Adventure.</figcaption>

</figure>

回想起来，我不禁想知道该公司是否试图利用热切的独立开发者。他们太年轻，不知道如何区分粗略的安排和合法的安排。我是那些相信我成功所需的信息会被提供给我的开发人员之一。也许是公司的组织混乱和管理不善使得技术需求不明确。不管怎样，结果是一样的。

我想分享这个故事，因为我相信还有其他独立开发者在寻找合同来证明他们的能力，并让更多人看到他们的作品。这是一个令人钦佩的目标，我绝不会阻止任何印度人这样做。然而，重要的是要意识到许多公司利用了印度人渴望获得专业经验的心理。我希望在开始制作游戏之前，我做了更多的研究，问了更多的问题。希望这篇文章能帮助你的读者意识到一些陷阱。尤其是当你与一家对他们接受的游戏类型有非常具体的规定的发行商合作时。

对我来说，制作这款游戏并不是缺乏动力或兴奋。我通读了规则，访问了论坛，并花时间探索了公司使用的平台。可悲的是，在我得知这个游戏不能在平台上发布之前，我已经完成了这个游戏..

花三个月的时间在一款游戏上，最终却没有任何回报，这已经够糟糕的了。更糟糕的是，一整年过去了，这种不好的味道依然存在。我只能想象如果我签一年的合同会有多糟糕。我知道其他开发者也遇到过这种情况。我真诚地相信这个行业需要做得更好。尤其是在使技术限制透明化方面，没有利用独立开发者的热情。

尽管我的游戏在与游戏发行商合作时没有被接受，[我还是决定在 itch.io 上公开发布我的游戏，而不是](https://nadya-primak.itch.io/grand-canyon-adventure)。毕竟，做一些东西却在我的硬盘上看不见，这是一种耻辱。这是一个针对中学生的教育游戏，你可以驾驶一艘船通过大峡谷的急流。你通过收集宝石和回答关于侵蚀的问题来获得积分。

你是独立开发者吗？你有过与游戏发行商合作的糟糕经历吗，在教育游戏或其他方面？我很乐意在评论中听到你的意见。
