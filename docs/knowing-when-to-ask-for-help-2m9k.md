# 知道何时寻求帮助

> 原文：<https://dev.to/silvestricodes/knowing-when-to-ask-for-help-2m9k>

## 一段时间后...

你坐在电脑前，被一个功能卡住了。你觉得你已经花了很多时间试图找到解决你面前问题的方法。你在网上寻找指导。你不确定你可以做什么不同的事情。

是时候寻求帮助了。

## 但也许...

你已经坚持做了几天的一个功能。你尝试了几种不同的方法。他们都没有按照你想要的方式工作。你不确定要在谷歌上搜索什么才能继续。

是时候寻求帮助了。

## 或...

给了你一个你从未遇到过的问题去解决。您不确定可以在互联网上哪里寻找指导来开始您的方法，并且在您正在处理的代码库中没有明确的示例可以用作指导。

是时候寻求帮助了。

## 你考虑过吗...

你有一个问题要解决。你想和同事谈谈你的想法。

是时候寻求帮助了。

## 好了，不举例了。

这些例子并没有完全涵盖你需要寻求帮助时的所有情况。有更多的分支路径可以引导你去寻求指引。

寻求帮助不一定是你认为的最后手段。在你发展过程中的任何时候，你都有机会寻求帮助。

## 什么时候应该求助？

只要你想。
