## [](#why-elm)为什么是榆树？

越来越多的前端开发人员选择 Elm 来构建应用程序，其优势如下:

*   *实际上没有运行时异常:* Elm 的编译器可以及早发现问题，防止用户在运行时出现异常。
*   *没有`null`或`undefined`错误:* Elm 为表示`null`提供了更多样的类型。编译器还确保您处理应用程序中所有可能的空值。
*   没有 JavaScript 疲劳:你不必选择和连接不同的框架和库来构建一个应用。Elm 有一个用于创建应用程序的内置框架，即 Elm 架构。
*   *可预测的代码:*所有的 Elm 代码都没有副作用，所以你可以相信你的函数总是根据它们的参数产生相同的结果。
*   *不可变数据类型:*不用担心自己的代码或者第三方代码意外改变数据，导致 bug。您的数据将是一致和安全的。
*   *强静态类型:* Elm 的编译器使用静态类型来确保你用正确的参数类型调用函数。您不会遇到微妙的类型强制错误。
*   *定制类型:* Elm 的定制类型让您可以创建全新的类型来清晰地建模您的业务领域。强大的模式匹配通过确保一致地处理自定义类型来防止未定义的情况。
*   *高级工具:* Elm 的`Debug`模块使得检查数据以捕捉 bug 变得容易，并且在你准备好实现它之前向你的代码添加占位符。像 create-elm-app 这样的第三方工具可以让您快速启动 elm 应用程序，并为即时开发反馈提供强大的开发服务器。

## [](#whats-in-the-book)书里有什么？

这本书的前五章主要讲述如何构建应用程序。您将创建一个名为 Picshare 的照片共享应用程序，并在每一章中添加新功能。

第 1 章向您介绍了 Elm，解释了函数式编程的一些基础知识，并让您创建一个基本的 Picshare 应用程序。

第 2 章解释了 Elm 构建应用程序的框架，即 Elm 架构。您将使用 Elm 架构来管理 Picshare 应用程序中的状态和事件。

第 3 章详细介绍了 Picshare 应用程序。您将学习重构代码的模式，以及如何向 Picshare 应用程序添加新特性。

第 4 章让你创建一个更真实的 Picshare 应用程序。前端应用程序通常需要与服务器通信才能发挥作用。您将学习如何调用 API 并将 JSON 安全地解码成静态类型。

第 5 章进一步发展了 Picshare 的交互性。您将使用 Elm 订阅和 WebSockets 来实时接收更新。

接下来的六章关注于扩展、调试、
集成和维护 Elm 应用程序的高级模式。

第 6 章解决了包含大量代码的复杂应用程序的伸缩问题。您将使用诸如可重用助手函数、可扩展记录和消息包装器等模式来将应用程序重构为更易维护的状态。

第 7 章介绍了 Elm 的工具。尽管 Elm 的编译器通过静态类型防止了大量的错误，但逻辑错误仍然会导致错误。您将使用 Elm 的`Debug`模块在运行时调试值。您还将使用强大的第三方工具捆绑和部署应用程序。

第 8 章讲述了与 JavaScript 代码的交互，这对于访问不纯的 API 或者将现有的 JavaScript 应用程序迁移到 Elm 非常重要。您将学习如何使用 Elm 向现有的 JavaScript 应用程序添加新特性。

第 9 章介绍了确保代码正确的测试。您将使用 elm-test 通过测试驱动开发来创建一个模块，通过模糊测试来测试代码的属性，并通过 elm-html-test 来测试一个 elm 应用程序。

第 10 章教你如何用 Elm 构建现代的单页应用程序。您将学习如何处理路由和协调不同的页面组件。

第 11 章以加速你的代码结束。您将了解常见的性能问题，如何测量性能，以及如何使用高效算法、惰性设计模式和`Html.Lazy`模块来优化应用程序。
