# Changelog(newdev.struggles)

> 原文：<https://dev.to/brettmchatton/changelog-newdev-struggles-38ei>

即使你是那种拥抱变化的人，做出改变也是困难的。但是，在 30 多岁时转行，同时维持一个家庭似乎是不可能的。此外，软件世界对新开发人员来说也是势不可挡的。后端、前端、php、node.js、Ruby、CI、git、框架、服务器端、客户端、deploy、codecamps 等等。因此，决定你的最佳方向可能是一项令人畏惧和沮丧的任务。如果我说我自己没有气馁，那我就是个骗子。

我觉得我犯的最大的错误是，学习语言而没有开发一个作品集。然而，我也有许多其他的陷阱。追逐潮流，跟随教程而不是学习基础知识，在完全理解当前语言之前跳过语言，等等。

追逐潮流——我注意到，想要成为全职开发者，每天都有新的潮流。很难权衡哪些是你应该学的，哪些是你可以学的。如果你追逐这些趋势，你可能会陷入一个学习的死循环，永远不会真正理解你在编码什么。

教程-教程可以是一个伟大的工具，但他们也可以是拐杖。对于我的技能水平来说，我已经记不清有多少次遵循了过于高级的教程。所以最终你可以复制代码，但不能自己使用它。我明白了，基础比试图做一些“又酷又有趣”的事情更重要。打开教程时，请记住您的技能水平，随着您技能的提高，这些教程将成为参考指南。

跳跃语言-从 html 开始，现在我需要 css，现在我需要功能 JavaScript，等等，我需要数据库 SQL，现在我需要使用 php 数据...兔子洞走得更远。很容易掉进兔子洞，除了沮丧之外，永远不会真正到达任何地方。当你没有经验时，在不理解当前语言的情况下从一种语言转换到另一种语言是很容易做到的。

我将在这一点上结束，建议是伟大的，但每个人都有自己的堆栈，可能会告诉你这是最好的。这会让你掉进另一个兔子洞。小心这些兔子洞，它们会绕过你，让你沮丧和气馁。
