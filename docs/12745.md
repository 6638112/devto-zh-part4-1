# 我在 Refactr 的经历。Tech 2019

> 原文:[https://dev . to/kaleigh/my-experience-at-refactr-tech-2019-2 bjk](https://dev.to/kaleigh/my-experience-at-refactr-tech-2019-2bjk)

上个月，我参加了 REFACTR。亚特兰大的科技公司。REFACTR 完全是关于科技的多样性&包容。令人惊讶的是，所有参加会议的公司都有巨大的 D & I 计划，希望雇佣与会者。

我申请并获得了一张免费的门票，其中包括一个工作室，来自[名编码](https://www.womenwhocode.com/)的女性。我非常感谢他们提供活动和会议的免费门票，在编码女性的帮助下，我已经能够参加一些活动和会议了。👩‍💻

## [](#day-1-gatsby-workshop)第一天——盖茨比工作坊

[![gatsby workshop](../Images/59518bddb4c34928332e61c3775e2623.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--3_Td7RuQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/347bb148feb114005b34eaf86ec40141/c2b56/gatsbyworkshop.jpg)

REFACTR 有 6 个不同的研讨会选项，我非常兴奋地看到在 [Gatsby](https://www.gatsbyjs.org/) 上会有一个，因为我一直在寻找一个新的 CMS 来管理我的网站项目。几个月前，我偶然发现了盖茨比，但我没有时间去了解它，所以这是完美的。来自 Gatsby 网站的解释:*“Gatsby 是一个基于 React 的免费开源框架，帮助开发者构建极快的网站和应用”*。听起来很不错！

这是一整天的研讨会，布兰登·卡恩斯从头开始教我们关于盖茨比的一切。我们在盖茨比的网站上做了[教程](https://www.gatsbyjs.org/tutorial/)，还有一个[心满意足](https://www.contentful.com/)作为我们的无头 CMS。一旦我们这样做了，我完全被说服了。用 Gatsby/React 管理站点，并且仍然有一个易于其他人更新的 CMS？拿我的钱(jk 都是开源的)。

## [](#day-2)第二天

这是第一个全天的会议，上午 9 点多一点，由[李美玲](https://twitter.com/ladyleet)所做的关于包容和创造更好的科技文化的开场主题演讲拉开了序幕。

我参加的第一个会议是与克洛伊·康登的“内向者的物联网”。她向我们展示了她是如何设计一个 [Flic](https://flic.io/) 装置让自己脱离社交场合的。我肯定要这么做！

[![iot for the introvert](../Images/cc8ac08473eec701d6043ba566d51385.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--CwqkqUgp--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/721dd7dffb510cfeea8b4db0f3b57c60/df8a4/iotfortheintrovert.jpg)

接下来，我参加了“用 React 构建可访问的 UI！”和泰鲁尔·亚历克西斯一起。她讲述了自己的无障碍之旅和最佳实践。

午饭前，我了解了艾丽西娅·巴雷特是如何重写《重构》的。《潜入盖茨比》(Diving into GatsbyJS)中有盖茨比的科技网站。好像我还没有被《盖茨比》迷住，我甚至了解到它制作网站的速度和移动友好性。

[![gatsby rewrite](../Images/9a93153aaa8487028c30308f485ce30d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--b_pPzWVZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/ec7d953b398b1860233a9251443e727c/243ce/gatsbyrewrite.jpg)

当所有人都坐下来吃午餐后，有一个“打造包容性创业之路”的小组，成员有[杰米·哈勒尔](https://twitter.com/jamieharrellmba)、
T3、、[林内特·麦克基西](https://twitter.com/TheGoodrCo)、[科特尼·齐格勒](https://twitter.com/fakerapper)、[斯蒂芬妮·朱厄特](https://twitter.com/StefanieJewett)和[内尔·泽泻尔](https://twitter.com/naelyall)。

[![lunch panel](../Images/5833a0cfe57e53b93cfd2ad281ad51ca.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--U8DW6DYY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/bb18ff56a09e1b3a5b843da47bd31f9d/243ce/panel.jpg)

午饭后，我和瓦拉里·雷加斯一起参加了一个名为“DevOps:一个爱情故事”的会议。我还没怎么接触过 DevOps，所以我对这个很感兴趣。Valarie 不仅很有趣，而且她解释了所有的步骤以及为什么 DevOps 的每个部分都很重要。

[![stack trace session](../Images/ca85f3c2f8ed8b3d348bc6ede2acbbc5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WjbiNtve--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/08514798dad5cc7216202233e82a8c9d/216dc/stacktrace.jpg)

詹妮弗·布兰德的《不要害怕 JavaScript 堆栈跟踪》棒极了。她教了我们很多不同的方法来用堆栈跟踪解决你的项目问题，包括一个非常酷的表格。

## [](#day-3)第三天

REFACTR 的最后一天以 Kortney Ziegler 博士的主题演讲开始。他谈到了他的科技之旅，以及他为什么要做他正在做的事情。

[![kourtney ziegler](../Images/5206153069859aef1dae3db317a7a0f5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uMEUvnLm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/b2c3a2c5671e967d78766d3196170936/243ce/kourtney.jpg)

今天的第一堂课，我和 [DevLifts](https://devlifts.io/) 的创始人 [J.C. Hiatt](https://twitter.com/jchiatt) 一起参加了“重塑你的身体以获得更好的表现”。他高度鼓励开发人员保持活跃，因为在我们的工作中，如果你忙得不可开交，可能要等上*个小时*才能离开办公桌。

接下来是克雷格·罗德里格斯的《Visual Studio Code Sparks Joy》。我对这次会议感到兴奋，因为我最近(在过去的一年里)进入了 [VSCode](https://code.visualstudio.com/) ，并且没有回头。他分享了一堆技巧和最喜欢的扩展。

接下来，我和 Jeremy Likness 一起参加了“WebAssembly:你的浏览器是新的操作系统”,因为我对 WebAssembly 了解不多，但我听说过很多。他展示了一堆高性能 web 应用程序的例子。

午饭后只剩 2 节了(好难过！).我从
*和[雷·杰苏阿尔多](https://twitter.com/RayGesualdo)那里了解到*将我们的心态从“自学”重新配置为“社区教育”。我真的很喜欢这个，因为他真的强调了我们都来自不同的背景，我们是如何走到今天这一步的，我们需要用我们所学到的东西互相支持，互相帮助。

[![community taught](../Images/851308596e01fa6df5f7d6114daf0ccc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--AINpU24Y--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/1fa93f20977e39fc343a0b5bcc914d87/243ce/community.jpg)

“构建社交应用的挑战”是与令人惊叹的艾丽西娅·卡尔(Alicia Carr)一起进行的，她创建了帮助家庭暴力受害者的 [Pevo 应用](https://www.pevo.us/)。她讲述了自己的科技之旅和开发这款应用的经历。这是我最喜欢的会议之一！

[![alicia carr](../Images/343c19b92fe219118ae58e1d99c6f1f9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ySk2RcR0--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://kaleigh.dev/static/44fa7401e728c04c20dcac124a1543c5/243ce/aliciacarr.jpg)

最后，阿历克斯·卡斯蒂洛和他的公司,[“神经疾病”](https://www.neurosity.co/)正在做的事情结束了。这真是一些很酷的东西！
