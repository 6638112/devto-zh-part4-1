# 你时间的价值

> 原文：<https://dev.to/deciduously/the-value-of-your-time-3amg>

上周，我发表了一篇关于我计划重构一个大约一年没碰过的老项目的文章。我计划的重构范围很广，但是不着急——这个工具现在工作得很好。

当然，我们生活在一个神秘的世界，几天后，管理员给我打电话，问我是否愿意和他们一起开发一个全新的功能。为什么现在，过了这么久，我不知道。我只能假设神的干预。

简而言之，让教室全天保持适当的人员配备是一个不小的问题，让大楼周围的工作人员了解最新的调整甚至更复杂。他们希望有一个用户界面，允许他们建立这些时间表，了解员工与孩子的比例，并生成当天的个人员工卡。然后，工作人员只需导航到一个网页就可以看到当前状态，而不是依赖于对讲机上的电话游戏。任何能消除混乱的东西。

我出乎意料地被撕裂了。

优点:

*   反正我已经打算深入研究这个代码库了。
*   我相信他们已经正确地认识到了他们的问题，他们提出的解决方案将会有很大的帮助。
*   自从辞去那份工作后，我和这些人一直是朋友——我们帮助朋友。
*   我想做。这是一个巧妙而有趣的问题，自从他们问了我这个问题，我的大脑就一直在为这个问题超负荷运转。我已经有了一个好主意。
*   这是一个项目——你通常在做这些事情的时候学到东西，然后有一个更大、更令人印象深刻的投资组合。
*   其中一名管理员正在学习如何编写自己的代码，并愿意在这个过程中做出贡献和学习。

缺点:

*   这需要投入大量的时间。
*   这次我不会得到补偿。
*   前面提到的学习者*真的真的很新*，可能不会对总构建时间产生太大影响。这比骗局更“中性”，我认为像这样的项目无论如何都是培养技能的好方法。

真的，就是这样，但是我担心时间投入超过了好处。我只是不确定如何量化它。

在我看来，我花在构建软件上的时间确实有一些内在价值，但很难准确指出到底有多少。首先，我从来没有为我写的任何代码得到过一分钱的报酬。单就历史先例来说，我的时间不值什么，但我知道没那么简单。我有能力通过软件创造价值，因为我写的软件解放了带薪员工，让他们把更多的时间花在电脑无法完成的任务上。这里面有内在价值。

不过，谁在乎呢？知道我在帮一个忙并不意味着这个忙是不值得的，如果我现在写任何代码都没有得到报酬，我为什么要关心我写这个代码也没有得到报酬呢？我深深地喜欢编码，我选择它作为一种消遣，所以为什么要担心价值。

然而，我的时间也不是无限的，我还用它做其他事情。我知道我自己，我会想为他们完成这件事，这必然会占用其他努力的时间。

我并不觉得工作过度，事实上，恰恰相反，但是当我开始分解我是如何度过一周的时候，我开始感觉有点紧张:

小时/周: **168** (7 天* 24 小时/天)

按小时划分的活动:

*   睡眠:56 小时(我的确努力争取每晚 8 小时，但有时达不到)
*   全职非技术工作:44 人(包括通勤)
*   大学课堂作业:15-20(每周不同)
*   技术求职:10-12 天(简历调整、列表搜索、冷邮件、面试练习)
*   跑腿:约 3.5 英镑
*   体育锻炼:~3.5

这些是不可协商的项目，使用更高的界限使我每周有 28 小时的时间做我想做的其他事情，或者每天 4 小时。这实际上听起来很轻松，但很快就消失了——我有一些正在进行的兼职项目，我有一个女朋友，她会对她放在“兼职项目”下感到恼火，我喜欢玩长板游戏和进行大型徒步旅行等——我确实觉得不让我生活的这些其他方面下滑非常重要。上面的清单上写满了“本正坐在电脑前”，我也不喜欢在一周的剩余时间里坐在电脑前。

我可能会同意，但警告说这不会很快，但我很想知道你们是如何处理这种请求的。当你还是业余选手的时候和你成为职业选手的时候有什么不同吗？你会同意这个项目吗？
