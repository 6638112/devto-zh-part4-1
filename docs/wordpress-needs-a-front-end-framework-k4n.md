# WordPress 需要一个前端框架

> 原文：<https://dev.to/wking/wordpress-needs-a-front-end-framework-k4n>

*跨帖来自:[https://www . wking . dev/content/articles/WordPress-needs-a-frontend-framework/](https://www.wking.dev/content/articles/wordpress-needs-a-frontend-framework/)*

网站的复杂性呈指数增长，Wordpress 也不能幸免。虽然通常被认为是一个博客平台，但我工作的公司和其他成千上万的公司用它做更多的事情。

### 缺失的一环

框架、工具甚至新的语言已经被创造出来来帮助管理复杂性。然而，WordPress 并没有一个关于这些新工具如何适合他们平台开发的故事。这就是我想帮助解决的问题。

也就是说，我想以正确的方式将 Elm 引入 WordPress 社区。当与一个不仅后端编写方式如此根深蒂固，前端也是如此的系统一起工作时，你如何证明 Elm 是正确的？有没有用 Elm 构建 WordPress 站点和应用程序的案例？显然，我认为答案是肯定的，但我想了解并分享更多关于该系统的信息。

### 那么，有什么计划？

这个问题问得好。我现在唯一的计划是利用我在 WordPress 和 Elm 的经验，开始探索这两种技术相互支持的交叉点。又名，我只是要开始建设，看看它带我去哪里。

### 多多益善

有很多更有经验的 WordPress 或 Elm 开发人员，希望在公开场合这样做能够鼓励那些像我一样看到这一点并热衷于将这两个世界融合在一起的人的帮助！

现在这个计划是非常不稳定的，并且会随着时间的推移而改变。希望给予更新，提供突破。此外，可能会有研究/反馈需要来自更广泛社区的输入。
