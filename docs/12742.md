优化您的 PHP 代码以获得更高的性能
一般来说，优化您的 PHP 代码以获得更高的性能分为两类:

*   运行更少的代码
*   减少对外部资源的调用。

运行更少的代码

运行更少的代码有两种显而易见的方式；关闭那些没有增加多少价值的插件，这是一种不太明显的方式；瞬态缓存和代码优化。

瞬态缓存非常简单:

*   找一段花很多时间生成一些 HTML 的代码。
*   弄清楚它是个性化的，还是可以全局缓存，或者作为一组变体。
*   使用 WordPress Transients API 来缓存 HTML。

像 New Relic 这样的工具可以帮助您识别缓慢的代码块，因此您可以高效地进行修复。

瞬态缓存也可以用于数据集，而不仅仅是 HTML，但是一般来说，缓存越接近最终输出，影响就越大。

代码优化更难，这也是你可能需要开发人员钻研的地方。这可能就像切换到更有效的算法一样简单，但通常需要对事情的处理方式进行更大的反思和返工。

拨打更少的外部电话

当您向 Twitter 这样的 API 发出 MySQL 查询或 http 请求时，PHP 会等待响应。

你应该少做点这个。

Transients API 可以解决这个问题，在不需要更新的地方缓存 API 调用的数据。在数据库方面，您可以从智能页面设计中获取更少的数据。

一个页面真的需要一次显示 100 个帖子的内容吗？大概不会。

TL；巨大的胜利是可能的，但是先从临时缓存和设计中寻找容易的胜利。

当谈到你的网站的性能，以及 PHP 如何影响它时，这都是关于聪明的。了解在 PHP 工作线程数和 CPU 内核数方面需要什么，以及如何有效地使用资源，而不需要额外的代码来减慢速度。

如果你觉得力不从心，那就和一个托管的 WordPress 主机谈谈，它可以帮助你减轻压力，让事情顺利进行。
