# 2019 年你需要的三个软件测试小技巧

> 原文：<https://dev.to/clientserver/three-software-testing-tips-you-need-in-2019-26j1>

玛丽·克鲁兹和萨梅尔·纳克维是 ECS Digital 公司的质量保证&连续交付顾问，他们有将近 20 年的质量保证经验。他们在六月伦敦的“硅的女人”会议上谈论了让软件测试工程师的生活更容易的技巧。以下是他们会议的三大收获:

**提示 1:软件测试必须又快又好**

*萨梅尔纳克维:*

随着软件成为在所有市场创造竞争优势的关键，公司现在在开发软件时没有选择速度或质量的奢侈。敏捷过程已经成熟了很多，DevOps 现在也进入了这个环境。持续部署、持续集成和持续测试实际上是保证质量和速度的关键。在所有这些中，持续测试是最具挑战性的。它们主要是基于工具的活动，而持续部署是基于团队和工具的活动。持续测试不仅需要工具和团队，还需要个人和服务。如果连续测试做得正确，它实际上是敏捷开发过程中的杰作。

**提示 2:考虑左移还是右移**

*玛丽克鲁兹:*

左移是关于尽早测试。有很多调查表明，越晚发现错误，修复它们就越昂贵和耗时。想象一下如果有一只活着的虫子。你会花多少时间来修复这个 bug？左移测试的想法是尽可能早地防止错误。我们希望缩短测试周期，以便更快地交付产品。这显然不是什么新鲜事，但我认为这种趋势将在今年和未来许多年继续发生，尤其是因为业务预期在不断变化。

《向右转》正在进行生产测试，这听起来可能有点吓人。为什么要在生产中进行光测试？这个测试环境的目的是什么？右移测试的目的是因为我们知道在生活中可能会发生前所未有的情况。现在有很多公司正在使用的技术，比如 A/B 测试或金丝雀释放。因此，对于 A/B 测试，我们实际上是基于客户所看到的来交付应用程序的两个版本。客户 A 会考虑变型 A，客户 B 会考虑变型 B。我们根据设计表现更好的性能来收集指标。然后，分析团队将收集这些数据，并将其反馈给开发团队。这样做的好处是，我们正在开发客户真正想要的东西。

**提示 3:合同测试越来越好**

*萨梅尔纳克维:*

很久以前，在一个遥远的星系中，我们曾经测试过单一应用程序上的 web 应用程序。只处理一个数据库和一个存储库真是太棒了——这使得监控变得非常容易。但是，当流量和依赖性增加时，单一的应用程序有点失败了。然后，应用程序从大量的微服务开始大量的依赖，测试它们变得更加麻烦。我们尝试使用模拟测试，但它通常不可靠。然后我们转而使用 fly 依赖，但这非常昂贵且难以维护。现在，我们使用合同测试。

在契约测试中，我们有消费者和提供者。消费者是需要来自其他服务的数据的服务。提供者是提供信息的服务。这些信息保存在一份由经纪人保管的合同中，这可以是一种归档系统。

如果你在同一个文件存储库中既是消费者又是提供者，你可以把它们保存在文件上或者云上，比如谷歌云。但另一个选择是把它放在 Docker 容器上，这现在真的很流行。
