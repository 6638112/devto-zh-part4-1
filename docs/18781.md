# 个人项目-我的观点

> 原文:[https://dev.to/justsharkie/personal-projects-my-take-5gh0](https://dev.to/justsharkie/personal-projects-my-take-5gh0)

Web 开发领域给个人项目带来了巨大的价值。是的，疯了。

实际上，我从两个“耶，个人项目！”和“但我有其他爱好……”我认为这两种观点都有价值。个人项目很棒，我自己也有几个。但是我认为没有必要。

我们来列个利弊清单吧！因为谁不喜欢列表呢？

# [](#pros)优点

1.  学习——个人项目是学习一门新语言的绝佳工具，温习一门你已经很久没有接触过的语言...我坚信边做边学，个人项目是尝试新事物的好时机！他们给你时间去犯错误，改正错误，并且通常在没有人监督你或即将到来的最后期限的情况下解决所有问题。

2.  创造力——个人项目就是——个人的！你控制一切，从设计到你使用什么语言到你想如何实现它。这是一个你可以疯狂发挥创造力的地方，让它变成你想要的样子！

3.  乐趣——希望你喜欢这样做。至少一般来说是这样。如果你不喜欢做这件事，我看不出你有什么理由要在这件事上浪费你的个人时间。但是个人项目可以很有趣，如果你真的喜欢，就去做吧。

4.  填补空白——拥有一个完全符合你生活中想要和需要的应用程序，岂不是很棒？猜猜看，你能做到！为什么不呢？然后，最终，你得到了你想要的东西，带着自己亲手制作的满足感。

5.  展示你的作品——公司喜欢看东西。确实如此。他们喜欢漂亮、大的投资组合，以及有很多最近提交的 Github。所以，这很重要。老实说，我不会在这一点上施加太大的压力，因为我认为所有其他的点都更重要。这只是一个不错的小附加物。

# [](#cons)弊

1.  时间-上帝，我希望一天中有更多的时间。为了一切。有时，我们没有足够的时间做个人项目。这完全没问题。有时候我们想把时间花在做其他事情上，这也没关系！这也让我...

2.  爱好——我们都有与发展无关的爱好，这些爱好也应该在我们的生活中占有一席之地！我们不应该忽视它们，把我们所有的职业和个人时间都花在编码/设计/你做的任何事情上。因为如果我们这样做，我们会得到...

3.  精疲力竭-是的。倦怠。不好玩，你不想要，有时候个人项目也可能是问题的一部分。如果你从一天 8 小时的工作中回到家，只是为了坐下来写更多的代码，你可能最终会精疲力尽。这很糟糕。

4.  憎恨——有点类似于精疲力尽，但是有可能你最终会憎恨你的项目。如果你做了这么长时间，而它就是不想做好，你最终会开始讨厌它，害怕它。谁愿意花时间做自己讨厌的事情呢？不是我。

# [](#conclusion)结论

所以，我们都清楚，这些只是我对个人项目的想法。它们非常有益，但我从不认为它们是成为一名优秀开发人员的必要条件。
