# 组件库的好处

> 原文：<https://dev.to/domysee/benefits-of-a-component-library-2baa>

在我的工作中，我们现在开始创建一个公司内部的组件库。

这将是一点工作，但显著提高我们的 UI 开发。

最明显的好处是:

*   减少代码重复
*   速度加快
*   改进的用户界面一致性

一个不太明显，但支撑所有这些的好处是**减少决策努力**。对于设计师，开发人员和这些群体的互动。

在我看来，组件库是用来限制可能的设计的，而不仅仅是一些基础组件。

而且在可能性有限的情况下，思考过程会更快。设计师没有无穷无尽的选择，只有很少的几个。同样的事情也适用于开发者。他们看到设计，知道使用哪些组件。只有当所有涉众明确提出并批准时，才可以不考虑覆盖样式。

如果你不把组件库看作是限制设计可能性的一种方式，你只能得到一半的好处。是的，逻辑将被抽象，但是样式仍然经常被覆盖，在某些时候甚至可能需要定制逻辑。

这就是我目前认为我们可以充分利用组件库的方法。

你有什么看法？你觉得有意义吗？也许这对设计师来说太局限了？会阻碍他们的创造力？或者甚至为它提供便利？你知道组件库还有其他好处吗？
