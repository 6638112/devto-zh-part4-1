# 一个程序员的不幸遭遇

> 原文:[https://dev . to/guru guy 00/the-misadventures-of-a-a-programmer-000-35lo](https://dev.to/guruguy00/the-misadventures-of-a-programmer-000-35lo)

# [](#the-start)开始

关于我，我已经做了大约 15 年的开发人员，前 10 年左右都在 VBA *(对于那些不知道这是微软应用程序脚本语言的人)*。在 Excel 和 Access 中做各种各样的项目，在这 10 年左右的时间里，请注意所有这些“脱离网格”的项目*(意思是它们不被 IT 认可或支持)*。

在我踏进这个门之后，虽然我还在 CIS 完成我的学士学位，但我似乎总是被精英团队的角色选中，或者也许我只是错觉，我不知道...

# [](#getting-laid-off)下岗

我在这家公司的最后一个角色是财务，负责简化和自动化各种任务和演示。大约一年后，公司决定将整个财务部转移到杰克逊维尔。我必须在 12 月前找到新的东西。

我很幸运，我的经理当时正试图做一些事情，以便他的团队可以继续做其他职能。他很成功地做到了这一点，但在 10 月份左右，和我一起工作的招聘人员给了我一个可能的机会，在一个合适的 IT 部门*(所以我想)*做一名 VBA 的程序员。经过第二次面试，我得到了这份工作。

# [](#a-new-place)新的地方

从一个大公司到一个 50 人左右的小公司是一个很大的变化。我发现他们使用的唯一数据库是一个错综复杂的 access 数据库网络，由运营总监编写和维护，并且锁定在 windows XP *(我们谈论的是 2014 年 12 月)*上，因为由第三方编写的应用程序无法在 Windows 7 上运行。

在新的地方，拥有 10 年以上任期的员工在圣诞节和新年之间有一周的假期。在这一周，我第一次从 VB.net 跳到了 VBA，在那一周，我重新开发了那个应用程序，这样我们就可以开始部署 7 台机器了。

从那以后，我开始越来越多地在 VB.Net 写作。首席执行官希望有一个程序来管理我们向供应商发出的采购订单，在将订单发送给供应商之前有一个审批系统。简单地说一下，mu 经理一直在提议获得一个合适的服务器来安装文件服务器和其他一些东西，目前的服务器都是老旧的超功率台式机。因此，在构建这个审批应用程序时，它使用 access 作为数据库，速度很慢。所以有人问我怎样才能更快，这就是我们如何得到 SQL Server 的。

在我加入团队之前，mt 公司已经将一个用 C#/MVC 编写的 Web 项目外包给了第三方，到目前为止，我对此毫无经验，并希望将它转交给我以保持更新。我说服了他们给我两种课程。从 VB.NET C#的转换很容易，所有新的应用程序都是用 C #编写的，并且使用了面向对象和更多可重用的类。出于某种原因，MVC 就是不坚持。

# [](#a-clear-path)一条清晰的道路

今年早些时候，我向我的经理提交了一些基于职位、工作地点和经验的平均工资数据，以表明我远低于平均水平，希望加薪幅度接近平均水平。来回等待的时间长得可笑，但最终还是解决了。伴随着加薪的是一条通往高级和/或领导职位的道路，我认为可以晋升到经理职位。高层管理人员说，他们想继承一名初级开发人员，我将帮助继承和指导他*(这是有道理的，因为我目前是公司唯一的开发人员)*

通过对一个三年级学生的讨论，我开始思考如何用工具、责任和一大堆我从未想过的事情来最好地发展团队。说了这么多，我正在考虑记录我的前进方向，尝试记录我目前工作的当前 IT 流程的几个级别。

# [](#tldr)TL:DR

我是一名患有冒名顶替综合症的程序员，用自己的方式编程，指导一个团队，并在此过程中实现新的工具和流程。系好安全带，享受旅程。
