# 第一次 VC 的自白

> 原文:[https://dev . to/codestream/decentations-of-a-first-time-VC-103i](https://dev.to/codestream/confessions-of-a-first-time-vc-1o3i)

安德鲁·米克拉斯先生于 2009 年联合创立了 PagerDuty 公司。2017 年 12 月加入 S28，担任合伙人。米克拉斯先生于 2001-2006 年获得滑铁卢大学软件工程学士学位，并于 2006-2008 年获得多伦多大学计算机科学硕士学位。

Claudio Pinkus:感谢您今天抽出时间与我交谈。第一个问题是，你认为是什么造就了成功的公司？最重要的成分是什么？请把范围缩小到本质上。

安德鲁·米克拉斯:我认为传统的答案——“创始人、市场和产品”——非常接近正确的答案。更具体地说，我和我的合作伙伴有一个理论，对于任何成功的创业公司，可能只有七个左右的独特因素推动其成功。

对于不同的企业来说，这些事情会有所不同，尽管它们会回到我上面提到的“创始人、市场和产品”三位一体。举个例子，在 PagerDuty，我相信其中之一是我们成为 DevOps 旗手之一的方式，devo PS 是一项始于 2010 年左右席卷工程界的运动。另一个原因是，在我们的营销信息和产品设计中，我们把从业者放在了第一位，而在这个行业，情况往往并非如此。

CP:你说这些东西推动了企业的成功，这是什么意思？

AM:我们的信念是，企业之所以做得好，是基于那些在创业之初就融入其中的一些特殊观察或想法的力量，因此对这些东西之外的问题具有惊人的弹性。换句话说，只要关于一个企业的七件左右“正确”的事情仍然是正确的，那么很多事情都可能出错，很多错误都可能犯。

CP:你的意思是，即使你有一个伟大的团队，如果你逆流而上，你会面临一些障碍，而如果你选择了一个稳定地朝着正确方向发展的市场，你就不需要面对这些障碍。

艾米:没错。当我看到一家公司瞄准一个我认为正在转向其他领域的市场时，我会问这个市场十年后会是什么样子。人们会说“还有很多像 X 这样的企业，所以瞄准它是有意义的。”这家初创企业可能会在这个行业赚到数百万美元。我非常不愿意在那上面投资。我更愿意投资于我觉得未来会更大的领域，即使该行业的收入和指标现在要小得多。

**CP:你做风投多久了？**

艾米:我个人投资已经有几年了，去年开始专业投资。

CP:你做了很长时间的企业家，现在你作为投资者有了很大的不同？这是怎样的转变？

艾米:还是有点奇怪。当我与人交谈时，他们会说“哦，你从事金融工作？”我仍然认为自己是一名技术专家和工程师，只是碰巧在投资。我现在还在适应的一个非常不同的事情是深度和广度。作为一名工程师，你所做的就是考虑一个非常特殊的问题。作为投资者，这完全是颠倒的。你应该有足够的广度去理解许多行业。作为一名投资者，这个想法是你能够拼凑出经济中许多领域正在发生的事情，但当你与某人交谈时，你很快就会结束对他们特定领域的理解。

我过去的工作方式是，如果有什么东西我不懂，我就把自己锁在房间里 3 个小时，学习或玩它。通过实践弄清楚这个新库是如何工作的。作为投资者，这是行不通的。一天中没有足够的时间做那件事。

另一个转变是投资的反馈周期，我已经开始有点习惯了。当你在一家公司工作时，你会很快发现你做的事情是否正确。你有这个敏捷的过程——你尝试一些东西，把它放到市场上，如果它不起作用，你就尝试不同的东西。投资需要很多年才能明白你到底有没有能力。对我来说，这是一种令人不安的感觉。

CP:我完全明白。我每天晚上做饭。有时候有人问我为什么在忙的时候还要这样做。我可以用我的时间做些别的事情。答案是，我花了这么多年投资于长期的东西，我需要一些更直接的东西，给我短期的结果，让我感到一定程度的满足。(笑)你可以把一些东西切碎，扔进烤箱，一两个小时后，你就可以吃了。这种感觉就像投资不能给你的即时满足感。

艾米:是的。肯定不是。另一个不同是，作为一名工程师，我有这种系统化和建设的动力。我不打算只解决这个问题一次，我要弄清楚如何重复做，并建立一个系统，这样我就不必再这样做了。我最初也是这样对待投资的。当你去筹集你的下一笔资金时，你希望能够说“这是我所做的，这是我如何知道我可以再做一次的。”但我也意识到这个领域对此有多抵触。这么多真的只是取决于偶然事件。每天都如此不同。你看到的这些公司是如此的不同。它不适合工程思维所创造的自动化或系统化。

CP:这和你开始说的有点关系。也就是说，如果你决定投资一家公司的最重要标准是它在一个良好的市场轨迹上，那么随之而来的是，所有其他事情都取决于你是否发现了一些有可能因其市场而变得重要的东西。即使是好市场中的一般公司也比坏市场中的伟大公司要好。

艾米:是的。是啊，当然。甚至当你问这个投资机会具体是什么的时候，这个投资机会也会让你被吸引。我试着与其他投资者交谈，并问:你在这里看到了什么？还是没看到这里？我试着预测他们会说什么。这很难，因为我甚至不知道在他们的头脑中是否有一个清晰的框架来看待事物。同时，我不认为这全是运气。我坐在这里并不是说伟大的投资者一次又一次地走运。但是无论他们在做什么，在某种程度上，都违背了纯粹的系统化。

**CP:作为 PagerDuty 的前联合创始人，你会建议公司早期的创始人关注什么？对他们的成功最重要的是什么？** *

艾米:第一，这是建立一个团队。你需要一个好的联合创始人，问题是你在联合创始人团队中寻找什么？比如互补的技能组合，在一起工作得很好。互相尊重，不要小心翼翼地让人觉得你们不能互相质疑。有一个非常小心的平衡。你需要真正理解为什么你要解决你试图解决的问题，为什么这个团队是唯一适合这样做的。一个由两个工程师组成的团队，他们一生只从事编写代码的工作，可能有一个商业计划，需要向一家银行进行百万美元以上的销售。以他们的技能，这可能很难做到。此外，他们是否有毅力和灵活性的正确结合。你需要那些不会在第一次遇到阻力时就放弃的人，但你也不需要这样的人，“这个问题需要用这种方式解决，我正在用这种方式解决，别人告诉我什么都不重要。”

CP:你所指的是，如果你成功了，你真的在创造一种文化，这种文化会渗透到你的组织中。也许你知道那是什么。也许你不知道。但它还是发生了。

艾米:是的。成为文化的是创始人之间的共同点，他们甚至不一定意识到他们有，因为他们是你不谈论的事情。当你听到早期公司的人开始谈论文化时，这通常是令人向往的事情。很久以后，在一个组织中，也许甚至在创始人离开之后，你试着问自己这个地方有什么不同？你可以追溯到创始团队。这是创始人们从未想过要大声说出来的东西。因为他们的运作方式已经根深蒂固，几乎不值得讨论。

CP:让我们换个话题。宏观趋势。你认为今天推动增长的宏观趋势是什么？

AM:从本质上来说，如何通过软件实现工业的自动化或最优化还有很长很长的路要走。还远没有完成。我们的投资组合公司之一 Social Construct 正在研究如何更有效地建造公寓建筑的内部。他们几乎是在将软件工程工作流原则应用到构建过程中。这是一个了不起的想法，也是他们采取的一个了不起的方法，从本质上把软件带入历史上没有软件的行业。

CP:看起来软件开发仍然是一个工匠式的工作。我认为在可以自动化的事情上浪费了很多时间，我们最终会发现软件开发人员使用的工具仍然不如他们为客户制作的工具有效。比如“理发师的发型是镇上最糟糕的”什么是正确的平衡，你如何划分创造力和自动化之间的界限？

AM:基本上，软件开发有一个方面几乎是偶然的。这只是工具之间交互的结果:工具和您将要使用的语言选择，数据库如何与其他数据库交互等等。对我来说，这些问题实际上是需要更好的工具来减少的。

过去，软件工程师和系统分析师之间是有区别的。这些领域越来越紧密。软件工程师不再坐在那里手工组装流程图。现在软件做到了。有软件可以处理很多操作方面的问题。以前，您需要有人协助将计算机连接在一起，而现在我们有了 AWS，我们实际上是在 AWS 的基础上再移动一层提取层，您实际上感觉不到它下面有一个实际的系统。回到原来的问题，我看开发者工具或者运营工具公司，是关于理解“那些东西对其他人来说是什么？”。具体来说，我们的软件工程师的思想必须集中在他们正在处理的问题的内在复杂性上，而不是处理如何实际使事情工作。

CP:与十年前相比，今天的情况有什么不同？

AM:十年前，AWS 才运行了几年。你有能力为主机服务按小时付费，但是你没有这些更高层次的抽象。创业所需的努力肯定少于十年前，当然也比十年前更便宜，但仍然没有我认为的今天这么容易。

另一个大的变化是现在资本的筹集方式，尤其是在早期阶段。这与我过去所习惯的完全不同。你得到了一群天使，也许你得到了种子投资，然后就差不多了。12 到 18 个月后，你必须进行首轮融资，否则你就不会这么做，仅此而已。现在，融资前阶段的时间和内容变得更加灵活。你有小种子基金，大种子基金，公司做一个种子，然后做另一个种子，然后可能是另一个种子，这只是一个更流动的过程，这里面发生了什么。一旦你点击“A ”,事情就变得更像传统的工作方式了。与七年前的另一个很大的不同是 A 轮大了多少！

另一个变化是，技术之外的人对技术有了看法，我不知道这是在 10 年前发生的。也许这发生在第一次网络泡沫时期；我不是为此而来的。但现在似乎有更多——我几乎想说科技是一种不可忽视的力量。谷歌和脸书以及所有这些家伙都被认为是声誉良好的。这种转变几乎正在发生——人们谈论谷歌和脸书就像——我会说“大数据”，或者可能是“大技术”，对吗？同样，他们谈论“大石油”或类似的东西。看到这些事情很奇怪，在某种意义上，看到你自己，意识到其他人把你看作是一个恶棍。十年前不是这样。

CP:肯定不是，但他们正在考虑，因为谷歌不会选择“不作恶”作为他们的座右铭，否则。

艾米:的确如此。它没有那么普遍。你不会每天打开《纽约时报》看三篇关于‘科技将如何摧毁民主’的文章吧？作为技术专家，我正在思考这些事情。我们真的建造了对世界没有好处的东西吗？

CP:我想我们已经做到了。唯一的问题是，这是不可挽回的吗？我们也造成了一些损害，至少对社会中被认为有价值的部分造成了损害。

艾米:一个例子是几乎发生的行为修正，特别是在智能手机这样的消费产品中，你以牺牲其他一切为代价来推动参与，这几乎重新训练了人们的思想。我过去的注意力比现在更集中。

CP:我们将会看到我们如何为改善这些情况做出贡献。

艾米:我认为那将是一个投资工作或其他工作的好地方。我在寻找思考这个问题空间的公司或团体。

**CP:最后一个问题。你投资了 CodeStream。你认为这个机会最吸引你的是什么？**

AM:这里有多种答案，但最强有力的一个是——当软件工程师构建软件时，有他们产生的实际代码，然后有他们头脑中进行的心理过程，以及那个框架，关于他们做了什么和为什么做，以及如何扩展到所有这些。困难在于，软件工程师通常在公司之间来来去去，基本上每 18 个月换一次工作，当他们离开时，你可以保留代码，但你不能保留他们脑子里在想什么。不管是谁来接手代码，你现在都必须弄清楚，在一个人真正开始自己编写代码之前，如何把这个人脑子里想的东西放到这个新人的脑子里。我们显然知道如何捕获实际的源代码。我希望 CodeStream 能够捕捉和传输更多工程师头脑中发生的事情，如果这有意义的话。

CP:是的。正如我们倾向于说的，我们是关于“为什么”，在某种程度上，我们可以抓住“为什么”，我们将会回答很多 wtf。

非常感谢您抽出时间。

艾米:你也是。
