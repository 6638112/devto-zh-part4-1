# 为提高自己做贡献

这个标题可能看起来有点模糊，因为它可能意味着很多事情。你可以捐助慈善机构，你的当地社区，...这样也有助于提升自己。但是我将在这里与您分享的具体主题是，作为一名开发人员，为开源项目做贡献如何有助于提高自己。这是我试图尽可能多地应用的东西，我可以确认它帮助我学习了新的编程语言、环境和工具。

## 亲身经历

我参与的第一个项目是我在个人项目中使用的 Android 库。我在库中添加了荷兰语翻译，它只是一个 XML 文件中的几个字符串，所以非常小，但我仍然从这个贡献中学到了很多。我刚刚获得学士学位，开始作为一名职业程序员工作，所以当时我对 Git 和 Github 没有太多的经验。因此，有了这个贡献，我学会了如何在 Github 上创建一个项目的分叉，以及如何在我的笔记本电脑上把它拉到本地，这样我就可以在上面工作了。在 Github 上提交了对我的远程项目的更改后，我还学会了如何对项目进行拉请求。对我来说，这个小小的贡献是我为许多项目做出许多或大或小贡献的开始。

## 寻找好项目

### 趣味项目

在 Github(目前最大的开源项目平台)上四处看看，浏览人们分享他们项目的文章，或者看看你使用的应用程序或工具，很有可能其中至少有一个是你也可以贡献的开源项目。找到一个好项目最重要的部分是它也能让你感兴趣。它是你使用的或者你认为做重要工作的东西。如果你决定做一些你可能觉得没用或没意思的事情，很有可能你永远也完成不了你的贡献。

### 要寻找的东西

如果你已经找到了一个你想参与的项目，这里有一些在你投入之前可以参考的建议。

#### 激活？

我认为这是一个非常重要的问题。检查该项目是否在过去 3-4 个月内一直处于活动状态。这将是一个耻辱，你可能已经投入了大量的时间在你的贡献上，它将仍然是一个开放的拉式请求，从来没有得到被合并的荣耀。

#### 未决问题

查看未解决的问题也是项目活跃程度的一个很好的指标。然而，有很多开放的问题并不意味着它是一个“死”的项目，它也可能表明它有一个活跃的社区，所以查看问题的日期也很重要。另外，关注未解决的问题也是找到你可以贡献的东西的好方法。许多项目也使用标签系统，这可能有助于找到一个好的初学者友好的问题。

#### 好的指导方针

这不是一个要求，但确实有助于你的贡献得到批准。项目有好的指导方针吗？他们是否描述了如何投稿，他们是否有你需要遵循的编码标准，他们使用的测试工具等等。他们提供的信息越多，你的贡献被拒绝的可能性就越小(如果你遵循他们的指导方针)。

#### 安装文档

这可能并不适用于所有类型的项目，但在许多情况下这仍然是必要的。是否有完善的最新安装文件？如果你正在建立一个你以前从未工作过的新环境，这是非常有用的。

## 贡献什么

特别是对于新程序员来说，为一个项目做贡献可能是令人害怕的，即使是一个较小的项目。相信我，我经历过。你可能会认为它会被拒绝(不管出于什么原因)，或者你可能会写“糟糕的代码”，他们只会粉碎你的信心。对我来说，这不是我的经历，我参与的每个社区都非常欢迎贡献，如果有什么不对的地方，他们会指出来，我会调整我的代码。所以不要犹豫，开始吧！

做贡献并不总是一个新的特性，你可以为一个项目做很多事情:

*   文档，很多项目喜欢这些贡献
*   错误修复
*   现有功能的改进
*   更新包/库
*   重构旧代码

## 舒适区

在你感到舒适的环境中做一个项目很容易，但这是一个做一些新东西的机会，一些可能需要你花几个小时在网上搜索或寻求帮助的东西。这是一件好事，在这里你会学到很多。

如果你用 PHP 工作，试着做一个 Node.js 项目，如果你用 Java 工作，为什么不试试 Python 或 Ruby 项目，等等。学习新环境可能会给你解决问题的不同视角。你将会学到不同语言相比的好处。为什么有些语言在特定的用例中工作得更好。

除了学习一门新的语言，你还可以使用项目正在使用的新工具。建筑工具，测试工具，林挺工具，...也许甚至是像 Github bot 这样的小东西，你可以在你的下一个项目中使用它。有很多工具为开源项目提供了免费层。

### 提示

从小事开始，比如文档的改进。这给了你机会去习惯像 Git、Github、环境(如果你的贡献需要的话)以及审查和合并过程。

## 反馈

在准备好你的贡献并对项目提出拉动请求后，总有可能它不会马上被接受和合并，但也不会被拒绝。可能会有来自维护者或社区成员的评论，要求改变一些东西。你不应该把这看作是负面的评论，而应该把它看作是正面的反馈，从中学习。这可能是你犯的一个简单的编码风格错误，一个缺少的空格或者一个他们想要不同命名的变量。

## 好处

当然，主要的好处是学习一个新的环境，获得新的知识，作为一名开发人员，你应该不断学习，努力打破你的舒适区。

其他的好处是学习新的工具、编码风格和/或代码测试。根据项目的类型，你可以从小小的贡献中学到很多东西。你获得了知识，项目得到了改进，这总是一个双赢的局面。

除了上面提到的知识收益，还有网络收益。在从事不同项目和与不同社区互动的过程中，你会认识新的人，他们会给你建议，甚至成为朋友。如果你需要帮助，或者如果你正在寻找一个新项目或工作，你可以在社交媒体平台上联系，并在未来联系你的网络。

除了这些好处，还有一个长期的好处，你可以展示你的工作成果。许多招聘人员或公司会查看公开的 Github 个人资料，这向他们展示了你曾在什么样的环境中工作过，并可能表明你有什么兴趣，以及你是否适合他们的团队/项目。

## 结论

在支持你喜欢的项目的同时，为开源做贡献是一种学习新事物的有趣方式。对于作为开发人员的你个人来说，这将总是有短期和长期的好处。

作为一名开发人员，犯错是正常的，即使是高级开发人员也会犯错。通过贡献，你的变更在合并之前至少会被一个人审查，如果不是，那是项目维护者的错，但那是另一个话题。
