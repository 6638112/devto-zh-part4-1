Tumblr 正在切换到一个[无头 CMS](https://www.gatsbyjs.org/docs/headless-cms) 架构。😲

当其他人都在关注 Automattic 为 Tumblr 支付的很低很低的价格时，我被一个细节打动了——首席执行官马特·莫楞威格承诺 Tumblr 将很快由 WordPress 提供支持，但保留其前端和内容管理用户界面。

这意味着 Tumblr 将成为现存最大的无头 CMS 项目！

对于任何依赖 headless WordPress 作为其堆栈关键部分的开发人员来说，这是一个好消息。我相信 Automattic 会投资改进 WordPress Rest API，使它像处理 Tumblr 迁移所需的那样健壮和稳定。

众所周知，马特·莫楞威格从不热衷于将 REST API 项目添加到核心 WordPress 中，一些人指责他在添加后放任其发展。现在 Matt 和 Automattic 有了一个商业理由，要么增强 REST API，要么投资一个不同的项目，让 [headless WordPress 成为可能，比如 WPGraphQL](https://www.wpgraphql.com) 。

## 通往 50%及以上网络的路线图

所以我的第一个预测是 WordPress 将会朝着成为一个无头 CMS 的方向发展。之后会发生什么？

我认为我们将会看到 WordPress 社区的创新浪潮，它不太关注主题化和前端，而更关注内容管理问题。

我认为我们将会看到 WordPress 开发者向需要的大公司出售像“内容工作流工具”和“数字资产管理器”这样的软件，而不是向企业主出售 30 美元的主题或者向营销人员出售 SEO 插件。

是的，这些插件已经存在于 WordPress 生态系统中。但是说实话——它们远没有专注于 Wordpress 表示层的产品那么完美。

我的第三个预测是，网站开发者将首先选择 WordPress 作为内容管理的一个选项，不管他们用什么栈来构建他们网站的前端。WordPress 将成为 Gatsby 或任何其他静态站点生成器的默认工具。

这就是为什么我认为 WordPress 将管理大众网站一半的内容，但你无法检测到它，因为大多数网站将使用 WordPress 作为一个 CMS，与你的浏览器无关。

整个 WordPress 社区激动人心的时刻即将到来！
