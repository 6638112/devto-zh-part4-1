# 使用原型聪明地失败

> 原文：<https://dev.to/dochan/startup-use-prototyping-to-fail-intelligently-19mo>

什么是原型制作，为什么我需要它？当我们很多人听到 prototype 时，脑海中闪现的第一件事是机器人，它们身上有一些电线，或者为我们下一个重大的技术想法制作工作参考/蓝图。原型制作不仅仅是测试一个创业想法，并让想法适应市场。
**注意:原型设计就是要尽早失败，这样你就不会晚失败。**

这不是第一次或第二次就做对，而是通过试错快速改进你的想法，这比在测试你的想法之前进行全面部署的风险要小。

**举例**
[![A chef](img/9b2687776e3a4a3a82ecdbdb7cc65363.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--lPWQN288--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images.pexels.com/photos/887827/pexels-photo-887827.jpeg%3Fauto%3Dcompress%26cs%3Dtinysrgb%26dpr%3D2%26h%3D650%26w%3D940) 
假设你是一个有新想法的厨师。你所做的是为你的顾客制作不同版本的产品，当他们消费你的产品时，你要求反馈，这将有助于你完善你的食谱以适应他们。这不仅仅是向人们展示你的想法，也是对它的改进。
