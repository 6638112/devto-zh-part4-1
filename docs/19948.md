# 科技(供应链管理/VCS)巨头的互利共生

> 原文:[https://dev . to/rinaldorex/on-mutualism-of-tech-SCM-VCS-giants-32dk](https://dev.to/rinaldorex/on-mutualism-of-tech-scm-vcs-giants-32dk)

闭上你的眼睛，深呼吸，因为，我们将不得不从一个整体的角度来考虑一些从很多角度来看都有争议的事情。

相信你们都看过**权力的游戏**(对于没看过的，我在下一段加了个要诀。跟着读，你就明白了)。上个赛季…嗯，简而言之，我承认没有达到很多人的期望。不过话又说回来，我个人不喜欢只为了炒作的东西去看东西。我的意思是，如果你不能从中获得某种洞察力，那什么是好的呢？尤其是你可以利用它从不同的角度来看待生活？是的——***视角*** ，那是让我一直着迷的东西。这是我认为每个人都应该坚持的，尽管他们有不同的观点。

这是一系列的事件，导致不同的王国走到一起对抗一个共同的敌人，我想让你们注意到这一点。尽管看到一个就想割破对方的喉咙，但他们确信，考虑到手头问题的重要性，他们需要把那些放在一边——如果不处理好，将会毁灭所有的王国。(几乎所有的王国都聚集在一起与一群冰冻的僵尸战斗)我认为这是我们需要密切关注的事情。意见分歧将会一直存在。无论你在谈论或考虑什么——游戏、生活、你最喜欢的演员、一个人、一个榜样、政治、宗教等等。

我试图理解如何更好地处理这种意见分歧——在不断变化的科技行业。尤其是在开发者的平台上。

我对我们的生活方式有很多批评——尤其是对我们这种依赖科技的群体。当然，我知道有多种方式来看待这个问题。它从来都不是黑白的——几乎总是天生的灰色。

作为一名软件开发人员，我最近偶然注意到一件让我微笑的事情，尽管我认为这是科技巨头们持续不断的竞争。当然，我理解竞争是健康的(*也就是说，如果以健康的方式进行*)。从进化的角度来看，无论我们是否希望它们存在，它们总是会存在的。

在我继续之前，这里列出了一些你可能需要知道的事情，以便理解我正在漫无边际地谈论的要点(尽管你不需要知道它们来理解我想说的)。

*   SCM/VCS: [源代码管理](https://en.wikipedia.org/wiki/Source_control_management)，是开发人员维护他们代码/工作的版本库的一种方式。他们现在更容易被称为 VCS(版本控制系统)
*   Github/Gitlab/Bitbucket:这些就是我说的科技巨头。简单地说，他们是 VCS 生态系统中的大人物。

既然已经解决了，我们开始吧。

Github 有着非常悠久的历史。这是第一批。在很多方面，他们是 git 存储库在线中心的先驱。然后是 Atlassian 的 Bitbucket。他们在市场上有自己的份额。然后… Gitlab 是最近的巨头，但是如果你是开发者，它的特性集会让你大吃一惊。(我个人很爱 Github 和 Gitlab，原因多种多样！)

但可以肯定的是，他们无疑是竞争对手。而且是凶猛的动物。考虑到最近开源项目的激增，人们(甚至是初学者)随时可以获得的资源，以及许多其他原因，包括市场营销的易用性，这太激烈了。让我们从进化的角度来看这个问题，好吗？

那些在技术行业、软件公司工作的人，永远无法否认我们领域内固有的竞争。这是众所周知的适者生存的处理事情的方式。你要么(不)故意吃掉别人，要么被吃掉。但是，很少有人——我是真心实意地说——真正准备好面对事实的人很少——尽管同属一个家庭，但他们的竞争和尊重他们自己的差异。

既然已经确定竞争是存在的，不管它是不是有意的，那么让我们来看看发生了什么，以及为什么我要用《权力的游戏》来类比。

5 月 2 日星期四，发生了一个跨越上述所有服务的事件——Github、Bitbucket、Gitlab。这是某个未知团体对用户存储库的攻击。有趣的是，他们都受到了类似的攻击。现代的软件行业毫不掩饰地依靠上述服务生存(不管是好是坏)

当这种事情发生时你会怎么做？具体来说，如果你受到影响，而你的竞争对手也受到影响，你会怎么做？显而易见的解决方案是解决你自己的问题，让你的竞争对手自己解决。但是…

一个人在这里应该做什么，这是一种道德义务。所有的服务都是以用户为基础的(任何成功的企业都应该如此)，更快地找到解决方案*比单独找到解决方案*更重要，但*要慢。归根结底，什么才是重要的？**人(*用户，*如果你是开发者)。***

 *所以这三家科技巨头都做了一些非常值得称赞的事情，让我(和数百万开发者)对整个软件行业微笑。

当你和你的竞争对手面对一个共同的敌人时，最好是把你们的分歧拉开，先除掉这个敌人。

就像那句名言，

> 朋友是和你有相同敌人的人。
> 
> *—亚伯拉罕·林肯*

这就是他们所做的。这不是很棒吗？当他们注意到他们的竞争对手也受到了攻击时，他们没有选择将自己的业务置于客户之上。他们分享他们的情报和发现，以便尽早解决问题。这是我希望我们所有人(尤其是开发人员)从这件事中学到的一课(除了技术方面)。

即使你们在工具、语言、政治、宗教方面有不同意见，如果你放眼全局，这些都无关紧要。这才是一天结束时最重要的。

从进化的角度来看，竞争的存在有各种各样的原因。如果你对我所说的有一丝一毫的了解，你就会知道健康的竞争是改善每一个物种的必要手段。但是，如果你深入探究健康竞争的含义，你会发现所谓的[](https://en.wikipedia.org/wiki/Mutualism_(biology))*【2】。无论科技巨头们是否意识到这一点，这都不是问题的关键。而是尽管他们知道，它是否正在发生。*
