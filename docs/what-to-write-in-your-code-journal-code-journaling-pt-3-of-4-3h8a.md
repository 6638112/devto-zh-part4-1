# 在你的代码日志中写些什么

在我的代码日志系列的第三部分中，我将介绍一些在你的代码日志中实际记录什么的想法。这些只是想法，你应该挑选适合你的。

对于在代码日志中写些什么，你有其他想法吗？请把它们写在下面的评论里。

## 写什么

如果你不知道写什么，那就写吧。每天写几个句子，收集你的想法或记录你做了什么是一个很好的开始。也就是说，这里有一些你可能想加入的跨页(一页或多页)的想法。

> 开始写吧，不管怎样。直到水龙头打开，水才流出来。
> 
> 路易·爱

## 索引和/或章节索引

这是你应该在笔记本上创建的第一个东西。要在笔记本上创建索引，只需在代码日志的开头留出 2-4 页，并将它们命名为索引。有些笔记本有页码，如果没有，你只需要在用的时候给一页标上页码，然后把内容写在索引里。

章节索引是一个有用的工具，可以更全面地分解你的笔记，并且是一种笔记类型的分组(我将在后面讨论一些章节的想法)。基本的想法是，您将区段索引添加到您的索引中(例如，p.8 项目索引)。然后在你的部分索引页上，你为那种类型的笔记创建一个索引(例如 p.9 Project Parlour，p.23 Minki Inc .等)。这种分组意味着您不需要搜索整个主索引，试图找出您需要的，您只需找到适当的部分索引，并在这个较小的索引中找到它。

对于数字代码期刊，根据您的设置和使用的平台，索引将采用不同的形式。有了 Evernote，你可能会使用不同的数字笔记本和标签，可能会有一个笔记作为索引。如果你正在使用在线文档，比如 Google Docs/Drive，那么你可能会想要一个文档作为索引，并使用文件夹来组织你的文档。数字期刊中的搜索功能会有所帮助，但最好能一目了然地看到你的笔记内容。

## 定期/每日笔记

即使你不每天做笔记，在编码的时候养成写点什么的习惯也是很好的。它可能只是一个关于你下一步需要做什么的笔记，但是关于你为什么做出决定的笔记，或者关于你或其他人编码的观察，在你将来回顾它们的时候会很有用。记下你下一步需要做的事情，当你再次回到那个项目时，会节省你很多时间。

## 目标跟踪

许多人设定了年/季度/月的目标，但是当他们到达时间表的末尾时，他们怎么知道他们是否已经达到了目标呢？他们还记得为此而努力吗？

保持你的目标，将它们分解成步骤，并在代码日志中记录你的进展，这意味着你可以建立一个回顾你的目标和进展的仪式，使你更有可能在预定的时间范围内达到目标。

你的目标是在明年学习一门新的编程语言吗？把它写下来，连同你需要采取的步骤(或者至少是你需要采取的下一步)，记录下你为实现目标所做的一切。当你看一个课程视频，去一个聚会或者写一些代码，记下你的进步。

养成定期回顾你的目标的习惯，至少每月一次(对于短期目标可以更频繁)。这会让它们在你的脑海中保持新鲜，并让你朝着每个目标的下一步前进。

## 训练&技能

除了正式的培训，你可能还会学到很多非正式的东西。在设定未来目标、更新简历或要求加薪或升职时，记录你目前的技能和学习情况将会有很大帮助。

你可以有一个页面来记录你在某个特定主题上的任何学习，或者有一个页面来记录所有的学习，或者你可以有一个页面来记录课程，一个页面来记录视频课程，等等。代码日志应该对你有用，所以你可以尝试一下。

## 项目跟踪

记录你的项目是一个好主意，不管它们是你日常工作中的还是个人的。

这是你经历的一部分，在准备面试、更新简历、争取晋升或加薪时，它会像记录你的培训和技能一样有用。它还可以帮助你找出你喜欢什么和不喜欢什么，帮助你规划你在编码方面的未来。

## 技术

关于你使用的技术的笔记是另一个对未来规划、你的简历、加薪等非常有用的东西。当你在寻找一个问题的解决方案，或者你是一个架构师的时候，这也是一个非常有用的记录。

记录你使用过的技术，如何使用的，以及你研究过的技术是一种宝贵的资源。下次你在寻找解决方案时，你可以很快看到你过去忽略的任何技术，以及原因。你也有关于某项技术利弊的信息，所以你可以很快判断出它是否能解决你的新问题。

记录有用的技术资源，或者你是如何用一种特定的语言解决问题的，当你想再次使用该技术时，这将是一个有用的资源。

## 成功，失败&赞美

当你经历成功或因某事受到表扬时，你应该把它记录下来，不仅仅是为了像要求加薪这样的事情，也是为了在情况不太乐观时提醒自己积极的事情。回顾过去的成功和表扬会帮助你走出骗子综合症的阴影，如果你患有这种病的话。口头表扬尤其重要，因为你没有其他记录。

记录你的失败也很重要，这不是贬低你自己，而是让你从过去吸取教训，以便将来不再重复这些失败。为什么会出问题？事后你会怎么做？这是可以预见的吗？你将来如何避免这个问题？是怎么修好的？它能以更好的方式修复吗？

## 决策

在你的项目、技术研究、目标和未来规划的背景下，记录你、你的团队和其他人的决定是一件好事。它们会在将来为你节省时间，你不需要再做同样的决定，因为你会有一个记录，为什么你会以这样或那样的方式做决定。

如果你需要向某人报告一个项目的进展，或者为什么事情进展顺利或不顺利，这个记录也会很有用。如果你有一个记录，为什么这个项目会朝着一个特定的方向发展，那么这个过程就会容易得多。

## 开始做|继续做|停止做

这是一个很好的定期回顾；每月，每季度，每年，甚至每周都做一些新的事情。它让你有机会分析什么对你有用(坚持做)，什么对你没用(停止做)，以及你如何改进(开始做)。

这个过程是在你自己的工作和过程中实施敏捷的好方法。你每天、每周或每月都在不断地尝试和改变来提升自己。

> 你采取的每一个行动都是对你想成为的人投下的一票。没有一个单独的例子会改变你的信仰，但是随着投票的增加，你新身份的证据也会增加。
> 
> 詹姆斯·克利尔

## 未来&职业规划

类似于目标，但时间范围更长。你对未来和职业的目标可能会随着时间的推移而改变，也可能不会。记录下你对未来的想法和想法将使你能够设定短期和中期目标，引领你朝着目标前进。

随着你获得更多的经验，你可以更新这些计划，甚至转向完全不同的东西。以这种方式使用你的代码日志将有助于你计划并记住你的计划，以便让你走上实现它们的道路。定期思考你的未来意味着你可以避免在一条你不想走的路上走得太远。
