# 戈多野生果酱 12

> 原文:[https://dev.to/ncot_tech/godot-wild-jam-12-1922](https://dev.to/ncot_tech/godot-wild-jam-12-1922)

2019 年 8 月 9 日至 16 日，我参加了 [Godot Wild Jam #12](https://itch.io/jam/godot-wild-jam-12-one-year) 。让我们回顾一下当时的情况，以及这次我学到的东西。

[![MatchUp, Created in a week using Godot](../Images/d2358a13a748e67649b0e432c19a2ddf.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--f6hiTZMA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/j263i7ld896o3jtqapff.png)

我的游戏， [MatchUp](https://ncotsoftware.itch.io/matchup) 是一个相当简单的匹配彩色积木的游戏。不是 match-3，因为游戏不在乎你怎么搭配瓷砖，每个关卡都有不同数量的瓷砖需要搭配。把它想象成 DS 上那些“相同游戏”方块游戏和动物园管理员的混合体。

对于这个游戏，我试图创造一些有趣的东西，有很多视觉上的兴趣。我以前制作的游戏没有太多的新奇和设计。我实际上也测试了这个游戏，并花了相当一段时间来调整游戏，使它玩起来更有趣。

## [](#so-what-did-i-learn)那么我学到了什么？

### [](#a-week-is-a-long-time)一周是一段很长的时间

有足够的时间来设计和测试一个游戏，并使它看起来和位置都很好。没必要急！

### [](#using-tools-saves-time)使用工具节省时间。

我用的是 Godot，它有很大的用户群，人们为它制作了很多教程。当我陷入困境时，我可以去阅读或跟随一些，而不是花几个小时自己的时间从基本原则上解决问题。

### 因为东西坏了，需要很长时间才能修好，所以把它们砍掉是一个好策略

这是一件大事。当玩游戏时，鼠标拖动是有缺陷的，而且这些缺陷很烦人，所以我完全去掉了鼠标控制，用键盘控制代替了它们。修正了所有的控制问题，也让游戏更有趣了！

### [](#iterate-quickly)快速迭代

我最初的两三个想法都是垃圾，但通过努力去实现它们，我明白了它们为什么是垃圾。

### [](#not-everything-can-be-planned-from-the-start)不是每件事都可以一开始就计划好的

我不太清楚自己在做什么，但“滑块益智游戏”是大方向。一旦我有了基本的工作机制，我就可以自由地去试验并找到一些有用的东西。

### [](#dont-clone-games-but-do-be-inspired-by-mechanics)不要克隆游戏，但一定要从力学中得到启发

没有人想要另一个比赛 3 益智游戏，真的不需要再有一个“同样的游戏”了。但是一般的机制很有趣，而且把东西融合成一个新的想法并不需要太多。
