## 余烬 3.6 和余烬 3.7

Ember 最近还发布了两个新版本:Ember 3.6 和 3.7。

本地类构造函数的更新现在已经完成，新的部分已经在 3.6 中发布。Ember 中本地类的行为现在是稳定的。

Ember 目前不支持或推荐使用类字段或装饰符，因为它们仍然在 TC39 进程下。

3.6 还推出了路由器服务 RC 的最后阶段。如果您想在应用程序中更多地使用路由器，3.6 版提供了更多的 API 来使它变得更容易。

Ember 3.7 专注于小的变化，主要是错误修复(包括 Ember 数据)。3.7 也放弃了对节点 4 的支持。

3.7 还对 Ember CLI 做了一些改动，包括插件自述文件中的兼容性部分。这个对生成器文档的补充会自动生成一个部分，告诉你的插件的用户推荐使用哪个 Ember 和 Ember CLI。

Ember 3.7 还删除了巴别塔 6 的最后一个用法。

Ember 3.8 也是最近发布的。在[发布的帖子中了解更多信息。](https://blog.emberjs.com/2019/02/27/ember-3-8-released.html)

## 余烬辛烷

Ember Octane 是 Ember 社区的第一个“版本”。版本是一种新的发布类型，是一种内聚点。不是余烬 4，5，等等。相反，它是一个时间点，在这个时间点上，一组特定的特性被发布、稳定并可以使用。

Ember Octane 中的一些功能包括西兰花 2.0、“微光组件”、原生 es 类等等。

Ember Octane 的主要关注点是生产力和性能。
