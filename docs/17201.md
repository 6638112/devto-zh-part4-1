嗨
我是斯蒂芬，[全栈开发者工作和简历板](https://fullstackjob.com)的制造者。2 月 19 日，我开始了我的项目。你可以在这里阅读更多关于[的信息](https://dev.to/golangch/omg-i-love-to-develop-software-how-i-built-fullstackjobcom-with-go-vue-a2i)

一切都始于一份简单的全栈开发人员工作清单。第二步是订阅一份简单的工作简讯。在我做了一些改进后，我从用户反馈、使用分析等方面看到/得到了一些改进，比如更好的过滤选项、改进的工作汇总等。

然后大约一个月前，我发布了候选人在线简历生成器，也有以下改进，如隐私设置，通过招聘人员的表格联系的可能性等。你可以在这个视频中看到当时的状态

因此，你可以看到，我不断为用户提供新的功能，并改善后端的东西，我认为自动化是有意义的。

上周末，我发布了一个高级工作提醒和简讯功能，全栈开发人员工作的候选人可以为每日/每周工作简讯和工作提醒定义多个基于过滤器的订阅，一有新工作添加到公告板就发送:
[![](../Images/4274e278106ae60eb7468c83500543db.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ktsOsbBZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kwavq1vm3123e4v1fiqa.png)

现在...我只是在这个周末做了一个“密码重置”功能。
好的，有这样的框架，你可以将它和 Auth 等一起开箱即用。我使用 Go，没有框架，完全靠自己完成，主要是因为灵活性的原因，这是很好的学习。但是为什么我做得这么晚？好吧。当制造商为产品和使用产品的用户带来一些价值时，他们就会做一些事情。只要我没有太多的注册用户，就没有必要这样做。如果用户忘记了他的密码，我会手动为他设置一个新的密码(或者我也可以，只是从来没有发生过)。

但是现在，有了新的功能，我有 200 多个用户，而且还在增加，所以我想把这个功能给用户，也让我的生活变得更容易。:-)

编辑:
重点是不要让安全或其他重要的东西留下来，而是对用户来说不那么重要或不会给产品带来任何其他好处的功能。因此，在提到的“密码重置流”的情况下，这并不意味着密码没有被安全地存储，而是没有 UI 和 API 端点来这样做。
