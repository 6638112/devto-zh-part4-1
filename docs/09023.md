#### CSS&SCS

##### [](#can-you-nest-media-and-support-queries)[可以嵌套@media 和@support 查询吗？](https://css-tricks.com/can-you-nest-media-and-support-queries/)

是的。问一个简单的问题，得到简单的答案。但是要了解更多细节，请阅读这篇文章。:)

##### [](#bringing-css-grid-to-wordpress-layouts)[将 CSS 网格引入 WordPress 布局](https://css-tricks.com/bringing-css-grid-to-wordpress-layouts/)

对于那些在 wordpress 领域的人来说，这是一个非常好的工具，可以很容易地利用 CSS 网格，特别是在一个面向 Gutenberg 块的站点中。看看吧。

##### [](#new-css-features-in-firefox-68)[火狐 68 中新的 CSS 特性](https://hacks.mozilla.org/2019/07/new-css-features-in-firefox-68/)

虽然我通常不会强调单个浏览器版本，但我对`::marker`伪元素的到来感到非常兴奋。能够用 CSS 直接定位列表项目符号在过去会节省我很多时间，我希望它在未来能节省我很多时间。

#### [](#javascript)JavaScript

##### [](#why-react-hooks)[为什么对钩子有反应？](https://tylermcginnis.com/why-react-hooks/)

通过 React 随时间的推移而演变的方式，以及大多数背后的一些原因和思考，我们可以看到一个很好的视角

##### [](#how-to-sync-props-to-state-properly-in-react-hooks)[如何在 React 钩子中正确同步道具到状态](https://dev.to/paramharrison/how-to-sync-props-to-state-properly-in-react-hooks-53fn)

一个很小的 React hooks 教程，这个教程解决了如何将道具复制到本地状态，并保持本地状态随着变化而更新的常见问题。

##### [](#vuejs-3-futureoriented-programming)[Vue.js 3:面向未来的编程](https://blog.bitsrc.io/vue-js-3-future-oriented-programming-54dee797988b)

深入了解 Vue 新的面向函数的 API 如何让编写面向未来的代码变得更加容易。易于适应新地点的代码。当然，虽然这个 API 是为 Vue 3 设计的，但你不必等待，因为有人已经写了一个插件，所以你今天就可以使用它。因为这就是我们在 JS 世界里的行事方式。:)

##### [6 分钟学会 JavaScript 闭包](https://www.freecodecamp.org/news/learn-javascript-closures-in-n-minutes/)

闭包如何工作的可靠解释者。闭包是提高 JavaScript 技能时需要理解的关键概念之一，我喜欢作者用来描述它们的短语:“闭包是有状态函数”。从这里开始，他一步一步地帮助你理解。干得好！

##### [](#how-to-group-an-array-of-objects-through-a-key-using-array-reduce-in-javascript)[如何在 javascript 中使用 Array reduce 通过一个键对一个对象数组进行分组](https://dev.to/paramharrison/how-to-group-an-array-of-objects-through-a-key-using-array-reduce-in-javascript-35bi-temp-slug-3573713)

另一个很好的基础教程，这是关于使用 Array reduce 实现一个非常常见的模式(按键分组)。

#### [](#other-awesomeness)其他牛气

##### [](#truths-about-digital-accessibility)[关于数字无障碍的真相](https://ericwbailey.design/writing/truths-about-digital-accessibility.html)

这是一篇很好的文章，可以帮助你了解构建可访问的网站和应用程序所需的思维模式。在开发人员的头脑中，可访问性经常被捆绑到一个单一的类别中，但这是一个多方面的挑战，辅助设备在许多方面都不同于开发人员的标准环境。

##### [](#unexpected-accessibility-tips)[意想不到的无障碍提示](https://www.cjcid.com/articles/unexpected-a11y-tips/)

这真是太棒了。学习开发无障碍网站的最大挑战之一是，无论你多么努力，都很难计划一次你很难感同身受的经历。上面的文章试图通过让你进入心态来解决这个问题...但是这个给了你一套测试和活动，你可以直接模拟人们面临的挑战。我喜欢在火车上使用移动设备让你体验高度干扰的想法，用伸直的手臂拿着移动设备模拟移动挑战，在投影仪上查看网站模拟颜色挑战。精彩！

##### [](#branching-out-from-the-great-divide)[从大分水岭上分叉出来的](https://css-tricks.com/branching-out-from-the-great-divide/)

我认为这是当前前端世界中最重要的对话之一。随着前端中期望的职责和工具的范围继续增长，我们如何组织和理解具有不同技能集的人，他们都被认为是前端开发人员？这并没有给出什么大的答案，但是给出了一个很好的思考方式。

##### [](#the-real-dark-web)[真正的暗网](https://www.sonniesedge.net/posts/real-dark-web/?ref=heydesigner)

这是一个很好的重要提醒，尽管我们都喜欢关注前沿，但业内绝大多数人都不在前沿。所以不要让你自己被它吓到，只要继续做好工作。最终目标是一个工作产品，而不是最新的&最伟大的 JS 框架+ CSS Grid + CSS-in-JS +任何其他我们可能都想尝试的魔法。
