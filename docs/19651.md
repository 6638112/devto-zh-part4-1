# 哪一个工具/栈/框架尽管流行，但你却不感兴趣？

> 原文:[https://dev . to/briwa/what-that-one-tool-stack-framework-the-what-one-tool-stack-framework-that-you-different-in-dangerous-3j3e](https://dev.to/briwa/what-is-that-one-tool-stack-framework-that-you-have-no-interest-with-despite-the-popularity-3j3e)

在技术世界中，总是有太多流行的工具已经存在了一段时间，但是包括我在内的一些人就是没有时间/容量/资源/插入-原因-来尝试它。

对我来说，是 Docker。大多数回购现在都有一种与 Docker 和基础设施服务(如 AWS 或 Digital Ocean)进行设置的方式，这些服务也提供与 Docker 的集成。到目前为止，宣传仍在继续，人们也开始在开发中使用 Docker。然而，在开发中运行它的努力与收益并不相称。也许它在制作上很棒，但我最近一直在放静态网站。我想我没有时间在这上面投资，也许有一天当我需要它们的时候。

你们都怎么样？我很好奇 dev.to 的人是不是更像那种总是尝试(或掌握)流行的人，或者更像我一样，坚持常规，直到有必要尝试流行的。
