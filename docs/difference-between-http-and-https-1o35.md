# Http 和 Https 的区别

> 原文：<https://dev.to/notifyvisitors/difference-between-http-and-https-1o35>

在日常购物(包括使用电脑)的整个培训过程中，大多数人都认为发送和接收的信息是安全的。能够安全传输数据的创新可以归功于称为 SSL(安全套接字层)的过程。

【Http 和 Https 的区别。

用非专业术语讨论的 SSL 证书定义了用来屏蔽在线通信的安全程序。它最常见的用途是保护个人信息，如个人信息。

当个人通过点击提交按钮或在网站上输入数据开始交易时，该过程开始建立安全链接。然后，互联网浏览器检查 SSL 认证，确保它是有效的以及网站是真实的。信息的安全性以及 web 浏览器和网站审查的代码随之而来。人类有能力保护数据，但是使用 SSL 的计算机无疑更快更有效。SSL 证书具有计算机所有者的公开秘密，因此 PKI(公开秘密框架)中的技术使得 SSL、TLS 以及 HTTPS 能够共享加密数据；因此 SSL 证书得到了澄清。

HTTPS(超文本传输协议安全),显示在电脑的地址栏上，提供美学提示，以确认正在访问的网站具有安全连接；然而，HTTPS 认证并不是一个独立的程序，而是依靠 SSL 证书来加密利用这一现代技术的网站上的信息。HTTPS 证书是一种利用 SSL 程序的数字安全工具，最常被网上银行、信用卡支付网站以及那些非常依赖安全链接的消费者和企业使用。

随着世界范围内越来越多的电脑网络朋克能够访问敏感的文件和信息，对于金融机构、信用卡公司以及当地大大小小的企业主来说，保护他们的利益实际上变得非常重要。许多其他公司拥有或租赁计算机系统工具来帮助他们进行库存、订购和开发票、工资单以及无数其他应用程序。通过使用 SSL 证书提供的现代技术，基于 HTTPS 和加密的交互允许商家以及那些进行在线组织的商家保证他们的个人信息受到保护。SSL、TLS 和 HTTPS 在同步倡议中合作，以确保网络邻居的无风险、受保护的环境。
