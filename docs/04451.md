# 成为 2019 年的全栈工程师

> 原文:[https://dev . to/freako monk/to-be-a-a-full-stack-engineer-in-2019-97m](https://dev.to/freakomonk/to-be-a-full-stack-engineer-in-2019-97m)

希尔

我将列出一些资源，以帮助在 2019 年成为一名全栈工程师。

我在这里包括最流行的技术栈 React，Redux，Node，GraphQL 以及一些系统设计资源。

为了详细理解 React、Redux 或 Node，建议稍微了解一下 Javascript。

Javascript :学习 JS 取决于你的经验，但是一个很好的起点是从基础开始。

*   https://github.com/getify/You-Dont-Know-JS 是开始学习 Javascript 的好地方。它从最基础的开始，深入挖掘高级概念。此外，它还涵盖了 JS 编译器的内部。

React 官方文档对这个库的解释给人留下了非常深刻的印象。https://reactjs.org/

*   官方网站([https://reactjs.org/tutorial/tutorial.html](https://reactjs.org/tutorial/tutorial.html))提供的教程足以让你掌握所有基本知识。最重要的是，Kent C. Dodds 的 egghead 课程是一门涵盖 React 广度的令人惊叹的课程。https://egghead.io/courses/the-beginner-s-guide-to-react

Redux :在开始使用 Redux 或任何库之前，理解它背后的需求是有用的。本文很好地解释了使用 Redux 的原因:[https://blog . log rocket . com/why-use-Redux-reasons-with-clear-examples-d 21 bffd 5835/](https://blog.logrocket.com/why-use-redux-reasons-with-clear-examples-d21bffd5835/)。

*   想要了解更多，官方文件是一个很好的来源[https://redux.js.org/introduction/getting-started](https://redux.js.org/introduction/getting-started)。

Node 是构建服务器端应用最流行的 JS 框架，它有一个非常不直观的单线程架构。节点架构可以从这个文档来研究:[https://nodejs . org/en/docs/guides/event-loop-timers-and-next tick/](https://nodejs.org/en/docs/guides/event-loop-timers-and-nexttick/)

*   构建可伸缩的异步应用程序需要正确理解节点架构。

GraphQL 是正在兴起的最新技术，学习它是有意义的。GraphQL 有助于为 web 应用程序构建优化的数据通信 API。

*   [https://graphql.org/learn/](https://graphql.org/learn/)
*   [https://www.howtographql.com/](https://www.howtographql.com/)

**设计**:

*   这是一个令人惊叹的系列，其中谷歌解释了浏览器架构:[https://developers . Google . com/web/updates/2018/09/inside-browser-part 1](https://developers.google.com/web/updates/2018/09/inside-browser-part1)
*   [https://www.interviewbit.com/courses/system-design/](https://www.interviewbit.com/courses/system-design/)
*   https://leetcode.com/也是练习 JS 编码的好地方。
