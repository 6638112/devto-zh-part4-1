# 老极客有了新工作

> 原文:[https://dev.to/lbonanomi/old-geek-gets-a-new-job-cf0](https://dev.to/lbonanomi/old-geek-gets-a-new-job-cf0)

上周在 sprint 规划期间，我通过视频会议被重新介绍给了吉拉团队。我记得大约 4 年前最初扩建时唯一的工程师和业务/支持主管。当其他业务/支持人员在后台工作时，我被介绍给他们，他们现在看起来开心多了。监督这场闹剧的技术负责人似乎很热心，要求澄清，而不是做出执行(即完全没有根据的)决定，总体上看起来是合理的。

sprint 规划者扔给我一个垒球:对 Confluence 实例进行快速/禁止监控，因为*现在没有应用程序监控*。

年初，我们开通了一个吉拉票证，用于支持分析师可以遵循的故障排除说明。

在我去 GHE 团队之前，有一场关于吉拉范围的争论，该系统是否应该对业务人员以及委托它的工程团队开放？似乎没有人考虑成本就选择了“完全开放”模式，我们需要[审计闲置席位](https://dev.to/lbonanomi/reclaiming-jira-license-seats-24c8)。

没人要求我去审计 KTLO 的工作队列，但他们雇了一头猪，所以我要去嗅一嗅松露。
