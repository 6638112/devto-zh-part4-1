# 今天介绍 CodeThat.today🙌🍾

> 原文：<https://dev.to/theodesp/introducing-codethat-today-48io>

我想介绍 CodeThat.today，这是一个托管各种编码和开发教程、文章和系列文章的站点，包含多个相关内容。我已经在这个项目上工作了一段时间，因为我一直想有自己的技术空间，在那里我可以分享我的文章。我对此非常兴奋。请允许我解释一下我为什么做这件事以及怎么做的。

# 我为什么建这个网站

大约两年前，我开始在 Medium 上撰写文章，主要是用 Codeburst.io。开始的时候还可以，因为我用 [SOLID Principles 系列](https://codeburst.io/understanding-solid-principles-dependency-injection-d570c15560ab)取得了一些成功。然而，我不喜欢 Medium 提供的模型。你实际上什么都控制不了。这是一大锅混在一起的东西。

此外，我开始用 Fixate 和 PhraseApp 写文章，并把它们列在网络的各个地方。在某个时候，我想把它们合并到一个地方，这样我可以更好地引用它们。顺便说一句，我忘了说我还在 Go 上创作了一个[视频课程。](https://www.packtpub.com/application-development/advanced-go-programming-7-days-video)

所以把它们迁移到一个平台上，这样我就可以推广我的内容，我认为这是最好的选择。

# 爽，你打算写什么？

我有一大堆我想写的主题，但我想先创建这个网站，然后开始写内容。此外，我将在 dev.to 上交叉发布它以供参考。

# 关于站台

地点位于 [codethat.today](https://codethat.today) 。我在 [Github](https://github.com/theodesp/codethat) 上托管了整个网站，但是现在有点粗糙。我需要写一个自述文件，并对其进行一些润色。但如果你想用它为自己的手段，请随意。

# 接受新作者吗？

是的。不用说了。我需要为新加入的作者添加一个新的页面，但与此同时，如果你想作出贡献，请在这里给我发消息。

我也想尽可能用金钱或赃物补偿作者。因为我相信制作高质量的内容需要时间，而时间是有限的——我们只有一次生命，不是吗？该网站可以接受来自赞助商的广告，以便他们可以帮助资助新作者的内容。
