# 超越//待办事项

> 原文：<https://dev.to/ssustrich/moving-beyond-todo-400e>

我们都见过//TODO 标签。像//TODO 和//FIXME 这样的标签非常有用，ide 通常会突出显示它们，在某些情况下会将它们添加到生成的代码中，就像 Eclipse 中的“用 Try Catch 包围”一样，它是帮助您做您需要做的事情的其中之一。

为什么我们不扩大标签的使用范围，不仅仅是帮助我们编写一段代码，还有帮助我们成为更好的开发人员的标签呢？这是我作为初级开发人员时的想法，我认为他们给了我很大的帮助。

我想到的两个标签是//LEARN 和//ASK。

我第一次看到“？”像 max = (a > b)这样的代码中的运算符？甲:乙；我记下//了解这是如何工作的。这是你不需要的东西之一(如果是？也许会让事情稍微复杂一点)。//学习帮助我不断成长。毕竟你不需要？就像你不需要知道 For Each 循环一样。你可以坚持使用 for 循环，得到相同的结果，就像你不需要知道收缩，但是一旦你知道了，它们会让你的生活更容易。

//问的也差不多。当你看到代码并想知道“为什么要这样做？”或者“这个部分是如何工作的？”。
