# 12 点虚拟主机安全清单 2022

> 原文:[https://dev . to/blog/a-12-point-web-hosting-security-check list-2019-26bh](https://dev.to/blog/a-12-point-web-hosting-security-checklist-2019-26bh)

在选择网站主机时，有些事情是最需要注意的。其中之一就是“安全”虚拟主机安全会变得非常复杂。事实上，通过虚拟专用服务器(VPS)托管或共享托管服务器托管的 WordPress 网页可能容易受到黑客攻击。这些黑客通过向服务器上传恶意软件或其他方式的恶意代码来完成他们的工作。

事实上，事情越来越糟。所以，正如我们所说，**安全性是一个大问题**，你选择的托管服务将取决于此。

### [](#why-is-web-hosting-security-essential-for-any-site-in-2020)为什么在 2020 年，虚拟主机安全对任何网站都至关重要？

互联网是无国界的，任何人都可以访问任何东西。这是互联网最大的优点，也是它最大的弱点。这种自由开放的结构使网站容易受到各种安全威胁，其中数据泄露是最重要和最常见的问题。

互联网已经变成了一个比开始时更加危险的地方。它对所有人都开放，不管是好是坏，更重要的是，它已经成为许多公司的重要工具。

对于网络罪犯来说，网上交易是数十亿美元的共同资源。不幸的是，由于各种原因，这间接地转化为对小企业网站所有者的威胁，尤其是 WordPress 网站，甚至个人博客。

即使你的网站无关，网络犯罪分子也可以利用你网站的资源对其他网站发动攻击。

此外，我们不要忘记数据是新的石油，例如，如果你在你的网站上收集用户信息，这也是值得的。

网络上也有涂鸦黑客——攻击、贬低网站只是出于好玩的人。

每天有数十亿美元的交易在互联网上进行，这意味着在最脆弱的 WordPress 网站上有一个目标。一旦一个 WordPress 站点被识别为易受攻击，你可以确信黑客们将能够立刻使它瘫痪。

因此，对网站经营者和房主来说，提供尽可能好的安全性成为一种特权；这意味着，选择一个可靠的虚拟主机服务是必要的。

考虑到这一点，寻找安全的虚拟主机解决方案是值得的。嗯，阻止一个意志坚定的攻击者几乎是不可能的，但一点一滴都很重要。

为了避免成为上述情况的受害者，我们编制了一个列表来帮助你选择虚拟主机提供商，以确保 WordPress 站点的安全虚拟主机。

那么，在安全方面，你对虚拟主机服务有什么要求呢？

## [](#web-hosting-security-checklist-2022)网络托管安全检查表 2022

以下是一些需要考虑的事情...

### [](#1-backups-and-restore)1。备份(和恢复)

备份不仅适用于您的电脑，也适用于您的网站。然而，你可以控制数据备份的许多方面，但对于网站来说，这通常取决于你的主机提供商。

大多数主机提供免费备份，但也有不同的主题。例如，有些人可能会要求您手动执行备份过程，而其他人可能会自动执行，并要求您在需要数据恢复服务时联系他们的支持团队。

理想的情况是，找一个能定期自动备份并允许你随时恢复的虚拟主机提供商。如果您的站点出现问题，这可以最大限度地减少潜在的停机时间。

托管和保护是一个优秀的托管 WordPress 托管的好例子。他们有每日备份功能-他们确保每天备份你的数据，这样就不会丢失你网站的信息。

### [](#2-network-monitoring)2。网络监视

WordPress 网站通常托管在安装在大型数据中心的服务器上。现场工作人员这么少。控制在很大程度上是自动化的。因此，了解您的 web 主机是否正在监控其服务器的网络流量是非常重要的。

为此，我们通常使用控制和监控工具来检测可疑的流量或事件。这样，任何想要渗透恶意软件或进行攻击的人都可以被快速检测到。

不幸的是，这不是很多虚拟主机提供商出售的东西，所以你可能需要问他们更多的细节。至少你可以安心地知道他们对服务器的保护有多好。

### [](#3-firewalls-and-ddos-prevention)3。防火墙和 DDoS 防御

WordPress DDoS 攻击是一场噩梦。它看起来就像 300 磅重的大猩猩一样冲向你的网站，决心把它粉碎。通过 DDoS 攻击，黑客试图通过向网站注入大量流量来删除网站，以至于网站服务器超载并关闭。

你最好的选择是使用托管的 WordPress web，在信誉良好的数据中心托管主机，这些数据中心配备高端网络硬件，并提供针对 WordPress DDoS 攻击的初级保护。

这些通常通过使用良好的内容交付网络(CDN)来缓解，如 Cloudflare 或网站防火墙。一些网络主机如 Host And Protect，Site ground 在其主机包中包含 Live Firewall，而其他如 InMotion Hosting 则没有，而是让初学者使用。

防火墙也很重要，因为它们是抵御网络入侵的第一道防线。

### [](#4-antivirus-and-malicious-analysis)4。防病毒和恶意分析

在您的个人计算机上，您必须运行防病毒软件。在 web 服务器上，您完全依赖您的 web 托管服务提供商来为您安装、运行和监控它们。重要的是至少要知道他们是怎么做的，以及他们能为你提供多少关于潜在问题的信息。

一些网站主机允许你查看他们的分析报告，而另一些则作为软件包的一部分运行。有些主机提供了比其他主机更广泛的选择，但至少你可以从没有被感染的先前版本恢复你的站点。

一些虚拟主机提供商提供独特的恶意软件防御系统，这补充了他们的主机计划。它不仅寻找恶意软件，而且有一个集成的警报和删除工具来保护网站。

### [](#5-secure-ftp)5。安全 FTP

如果你是虚拟主机的新手，将大量文件传输到你的虚拟主机上有时会很有帮助。使用 FTP 或文件传输协议可以最有效地实现这一点。SFTP 是 FTP 的安全版本，可以在传输过程中保护您的数据。

虽然几乎所有的虚拟主机服务提供商都提供 FTP 访问，但并不是所有的都支持 SFTP。如果你看看我们的第一选择，你会注意到他们中的许多人提供 SFTP 访问。

### [](#6-spam-filtering)6。垃圾邮件过滤

这是一个灰色地带，垃圾邮件在技术上不会影响你网站的安全性。然而，如果您突然被大量垃圾邮件淹没，它可能看起来像 DDoS。如果您的主机提供反垃圾邮件过滤，攻击首先通过其反垃圾邮件过滤器。

此外，通过清除垃圾邮件，这些反垃圾邮件过滤器可以为您的电子邮件文件夹节省一些空间。几乎所有的主机都有垃圾邮件过滤器，但有些需要少量的手动配置。

理想情况下，寻找一个提供各种反垃圾邮件选项，提供不同类型的垃圾邮件保护。

### [](#7-internal-security)7。国内治安

同样，这个项目不是你的主机包的一部分，但是许多主要的主机提供商确保他们的服务器免受攻击。这意味着他们将不断更新最新的补丁和安全工具。

有这么多可用的虚拟主机包，其中有几个安全措施，如 KernelCare，自动修复主机保护和服务器加固。用户将知道这些安全措施可以保护他们自己和网站，让他们更加安心。

### [](#8-ssh-or-ssl-secure-socket-layer)8。SSH 或 SSL 安全套接字层

系统管理员每天都在使用安全套接字层(SSL)协议，开发人员也经常使用它。它有潜力帮助我们在服务器上执行各种任务。

建立 SSH 连接很简单，我们需要的唯一工具是一种终端或控制台类型，比如经典的 Linux 和 Mac 控制台，或者像 Windows 中的 Putty 这样的程序。

SSH 连接使用三项:用户、端口和服务器。只有这三个要素，我们才能在两台服务器之间建立安全的连接。

这种安全性是通过使用密钥和加密技术来实现的。每个服务器都有自己的加密密钥，当第一次与服务器建立连接时，我们必须将该服务器添加到可以安全连接的服务器列表中。SSL 的加入有助于提高用户满意度、SEO 和品牌效应。

### [](#9-sqli-or-sql-injection)9。SQLi 或 SQL 注入

原则上，每个网站和网络应用程序都可能受到 SQL 注入的攻击。数据库的语言是 SQL 就够了。很多时候，程序的制造商没有提供足够的安全级别。

被发现的漏洞在网络世界里不会长期保密。举例来说，有些网页上的信息存在安全漏洞，可以立即向犯罪分子揭示如何通过谷歌搜索找到网络项目。

使用标准错误报告，可以快速验证列出的引用是否代表潜在的攻击目标。然而，一些虚拟主机提供商[用他们的安全措施来防止 wordpress SQL 注入](https://secure.wphackedhelp.com/blog/wordpress-sql-injection-hack/)。

除此之外，他们还会借助在线的恶意软件扫描软件(T1)，该软件会扫描网站，找出一些常见的 T2 安全漏洞(T3)，并通过电子邮件通知你，然后让他们的专家人员根除这些漏洞。

### [](#10-access-restrictions)10。访问限制

主机可以通过禁用登录权限和限制对站点根级别的访问来防止恶意个人侵入您的资源。IP 地址限制不应作为保护站点的唯一手段，也不应作为用户身份验证的替代手段。

客户端 IP 地址是在客户端发送的网络数据包中指定的，该信息很容易被欺骗。此外，黑客经常使用隐藏其真实 IP 地址的攻击技术。IP 地址限制不能保护服务器免受此类攻击。

### [](#11-change-the-password-regularly)11。定期更改密码。

保护 web 服务器最简单的方法之一是定期更改密码。如果密码更改如此频繁，黑客将更难找到它。选择一个别人很难猜到的强密码。在用 WordPress 建立你的网站安全时，密码是我们必须注意的主要问题之一。

很多时候，是我们自己没有采取必要的预防措施为我们的用户设置安全密码。然而，也有其他时候，尤其是当我们允许用户在我们的网站上注册时，他们可能会产生一个容易猜到密码的安全漏洞。

### [](#12-turn-on-cphulk-in-cpanel)12。在 cPanel 中打开 cPHulk。

综合搜索关键字或暴力攻击是系统地检查所有可能的字符组合以获得正确的密码。即使数据被加密，并且搜索密钥成功，仍可能进行未经授权的访问。为了防止 [WordPress 暴力攻击](https://secure.wphackedhelp.com/blog/wordpress-brute-force-attacks/)的发生，建议在 cPanel 中启用 cPHulk。
