# 个人和组织的 IT 安全

> 原文：<https://dev.to/brettclawson75/it-security-for-individuals-and-organizations-3c3p>

**确定威胁的风险评估**

防止计算机安全漏洞的最佳方法是首先了解存在的潜在风险。风险评估调查是识别个人或组织可能遇到的风险的好方法。这将允许根据发生的可能性和严重性(如果发生的话)对特定风险设定威胁级别。有许多方法和实践有助于降低安全漏洞的影响。从业务连续性规划(发生安全威胁时的一套应急计划)到帮助加快恢复操作的风险管理实践，无所不包。这种对待信息技术的态度非常好，也就是说，如果我们能够从一开始就防止这些违规行为的发生，并保持积极主动，那么其他一切都只是次要的预防措施。

**访问控制**

第一道防线是外围安全，包括[门禁](https://searchsecurity.techtarget.com/definition/access-control)措施。信息安全与物理安全有很大不同，但同样的原则也适用。根据 Tech Target access control 的 Rouse M 的说法，当提到它时，它被定义为“一种管理谁或什么可以在计算环境中查看或使用资源的安全技术”(Rouse M，第 1 页第 1 段，2018 年 9 月)。这当然说起来容易做起来难，真正的问题是如何让一个程序足够开放，同时又有足够的安全措施。当您将安全性和业务相结合时，问题是允许一个开放但安全的工作环境。例如，这意味着对访问控制的安全限制越严格，开展业务就越困难，但会导致更好的[事件响应](http://blog.gigamon.com/2018/12/17/breached-whale-optimizing-incident-response-to-get-your-business-swimming-again/)。另一方面，你的访问控制措施越开放，你的信息就越不安全，从理论上讲，做生意就越容易。这通常涵盖了个人或组织的需求基础。

**访问控制措施的类型**

幸运的是，有各种各样的选择。根据共享和存储的信息，它通常决定安全级别。在物理安全方面，出入控制旨在将交通集中到有限数量的区域，这些区域配备有栅栏、警卫、照明系统、闭路电视设备等。关键是，一旦确认了正确的凭证，就必须由安全人员授予访问权限。回到 IT 世界中的访问控制，这是由两个主要因素组成的相同概念身份验证和授权。对于我们许多人来说，单点登录访问控制措施通常是我们所需要的。SSO 将允许我们使用一个密码和用户名组合来访问多条信息。相对于 IT 安全的安全性方面，SSO 更倾向于开放性，因此不建议用于保护敏感信息。

**强制访问控制**

对于那些更倾向于安全性而不是开放性的个人和组织来说，强制访问控制是一个很好的选择。对于处于中间位置的人来说，自主访问控制可能适合您，它将控制权交给了系统管理员。这意味着系统可以由管理员定制，以更好地满足他们的需求。重要的是首先通过风险评估实践了解存在的风险识别最有可能发生的潜在风险将有助于建立预防性安全措施的基础。通过使用访问控制；谁、何时、什么、在哪里以及如何传输信息都可以根据个人或组织的需要进行管理，从而使访问控制成为第一道防线。
