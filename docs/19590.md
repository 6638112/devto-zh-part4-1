# 事半功倍

> 原文：<https://dev.to/dbanty/achieve-more-by-doing-less-1c5a>

长期以来，一心多用被誉为快速完成更多工作的一种方式。我相信(我并不孤单)多任务实际上是生产力的敌人。人脑🧠和计算机之间的类比💻已经被做死了，所以我现在想要为我将要使用它的事实道歉。你集中注意力的能力——也许你很特别，我不想这样假设...我的专注能力就像单核处理器。虽然从技术上来说，我的大脑同时在做许多事情，但我用来完成有意义工作的那部分(即思考)一次只能做一件事。我们似乎同时完成多件事情的方式是通过上下文切换。

上下文切换是将你的注意力从一个任务转移到另一个任务的过程。它涉及到许多移动的部分，但很简单，你的大脑需要存储它正在做的任何事情的状态，加载它将要做的任何事情的状态，然后开始做那件事。我只是无意中给自己树立了一个榜样:

1.  我对前面一句话的关注被一种口渴的感觉打断了。
2.  我的大脑储存了我剩余的想法。
3.  我的大脑把我喝酒的地点放在了脑海的最前面
4.  我亲自执行了任务(即喝了一杯)
5.  我的大脑将饮料的位置储存在记忆中，以备后用
6.  我的注意力回到了写这篇文章的任务上，我完成了这个句子

哇，为“我喝了一杯”写了这么多！我们的大脑是不可思议的，它让大多数的干扰看起来很容易，但是在转换焦点上仍然有一些浪费的时间，这些时间累积起来。现在我并不是建议任何人不应该在写博客的时候喝酒(请保持水分💧)，只是最小化上下文切换很重要。我试图同时做的事情越多，我的注意力就会越分散，我就会做更多的上下文切换。因此，最小化浪费时间的方法就是最小化正在进行的工作。

正在进行的工作，或 WIP，描述了我已经开始做但还没有完成的所有事情。随着它的积累，WIP 会导致一些严重的问题。首先，我正在进行的工作越多，完成工作的速度就越慢。浪费在上下文切换上的时间是我可以用来完成一项任务的时间。另一个严重的后果是，我的大脑的工作内存(相当于计算机的 RAM)是有限的。随着记忆的填满，我要么会失去处理新信息的能力(曾经感到不知所措吗？🤯)或者开始忘记细节🤔。这可能会导致错误😨，挫败感🤬，生产力损失🤤，也是我们最喜欢的恶魔，压力😖。

那么一个极限是如何工作的呢？好的开始需要对要做的工作有很好的理解，✅.理想情况下，这意味着把它写下来。它可能在笔记本里📓，在便笺条上📝(对看板爱好者来说)，或者其他数字化的地方📲(我个人用谷歌 Keep，工作用吉拉)。关键是把需要完成的事情储存在大脑以外的地方，这样它就有最大的燃烧潜力🔥放下任务。不是所有的事情都应该写下来，记录信息本身就是一种开销。这里的目标是防止浪费，所以如果只是做任务比记录下来供以后使用更少打扰，那么遵循耐克的建议，“只是做它。”

对于一些任务来说，一个简单的注释就足够了，但是更大的任务需要分解成更小的子任务📝。这很重要，因为试图一次完成一项大任务和处理一堆小事一样糟糕。如果我的大脑不能容纳完成任务所需的所有信息，🥵，我将会浪费时间去记忆工作的不同部分。除了足够小之外，每个任务都应该有明确的开始和结束状态。如果结束状态🏁因为任务不明确，我还不会开始。这样做是一个很好的方法，迅速结束了兔子洞🕳.

在一个完美的世界里，我会在开始之前把所有的大任务分解成完整的子任务。唉，这个世界和我都不完美，所以我经常发现🕵🏾‍♂️:我工作时需要做的新事情。当这种情况发生时，我在上下文切换⏳。重要的是，在它完全打乱手头的任务之前，快速识别这种情况何时发生。鉴于我已经被打断了，我花一点时间快速写下这个新发现的工作，然后马上回到正轨。开始新的工作，但没有完成当前任务的最终状态，这意味着我在添加 WIP，这意味着我在添加麻烦。

原来如此。通过限制我正在做的工作量，我能够更快地完成事情，获得更多，并在做的同时保持相对较低的压力水平。我试图保持这篇文章相对抽象，因为这些技术可以应用于任何类型的工作。我限制我一次做多少来提高编程输出(质量和数量都要！)，管理家装任务，甚至磨练我的电子游戏技能。这篇文章的灵感来自于一本叫做《凤凰计划》的书，如果你还没有读过，我强烈推荐这本书。这个故事是虚构的，但它很容易阅读，并且充满了好的建议。

如果你一路走到这里，感谢你的阅读！这是我迄今为止最长的帖子，所以希望它是有帮助的，不会太无聊。
