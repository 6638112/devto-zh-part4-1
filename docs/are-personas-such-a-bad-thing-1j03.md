# 人物角色是一件坏事吗？

> 原文：<https://dev.to/damdigital/are-personas-such-a-bad-thing-1j03>

我们对人、对地方、对活动、对事物都是又爱又恨。

我绝对喜欢伦敦！文化、艺术、时尚和夜生活。这就是让我一次又一次爱上这座城市的原因。同时，我深深的讨厌伦敦！早上 8 点我被挤在中央线上时的通勤，从城市的一个地方到另一个地方的两个小时的旅程，以及第二个地方咖啡变得越来越小——尽管我确实喜欢好的扁平白咖啡！

人物角色似乎是行业中企业和从业者既爱又恨的主题之一。我曾与初创公司和国际公司合作过，我注意到我们没有充分讨论人物角色的目的和目标，有时企业会错误地使用它们，而不知道如何处理它们。

我听过很多 UX 成熟度低的公司谈论他们有多了解他们的用户。当我与利益相关者讨论用户体验时，他们总是确保让我知道他们已经在这个行业工作了几年，这应该会自动转化为他们对用户的了解程度。我注意到，通常没有研究来验证假设，也缺乏文档来汇总这些知识。有时候，企业觉得仅仅了解他们的用户就足够了，但是当你想让每个团队都参与进来时，这可能还不够。

如果我们不谈论用户，也不记录这些概念，我们怎么能确定我们谈论的是同一批用户呢？当你雇佣团队中的某个人时，你不能只相信那些自称擅长使用 Photoshop 的候选人。你要求先看他们的作品集和简历。如果他们看起来有能力胜任这份工作，你就邀请他们参加面试，试着更多地了解他们，看看他们是否适合这个团队，以及他们是否具备完成这份工作所需的技能。大多数公司比这走得更远，但你明白我的意思。为什么我们不问用户同样的问题？

我们来问一下用户是谁，他们的目标是什么，为什么要做某些决定。为什么我们害怕问这些问题？

数字机构 Clearleft 的创始人安迪·巴德(Andy Budd)将人物角色描述为一种沟通工具和“[……]一种将通过观察、采访和调查产生的大量丰富而复杂的数据合成为可以在组织中传播的东西的方式，并由不一定参与对话或甚至可以访问客户的人消费。”

我发现人物角色是一个重要的工具，可以整合利益相关者对用户的了解。当我们重新设计 Rethink 精神疾病网站时，我们创建了主要和次要人物角色来总结从分析、调查和用户访谈中收集的数据。在与利益相关者交谈时，它们变得至关重要，以清晰地调整每个人的思维。每个人都知道“经历抑郁的简”或“照顾她儿子的贝丝”以及我们创造的其他角色。人物角色的使用在机构内部也很有价值，可以有效地与团队的其他成员沟通，比如视觉设计师和开发人员。

人物角色之所以有效，是因为利益相关者发现他们易于联系，易于记忆，易于在多个团队中共享。不需要什么专业知识。如 NN/Group 所述,“一个角色代表一组具有相似行为模式、目标、动机和期望的用户；人物角色使产品设计决策聚焦于用户需求”，这就是为什么我发现他们是谈论用户的有力工具。

当我为“反思精神疾病”组织利益相关者研讨会时，我把人物角色打印在 A3 纸上，并把它们贴在墙上，让房间里的每个人都有一个参考点。看到利益相关者环视房间并参考特定人物角色的需求来验证他们的观点是很棒的。

我并不是说人物角色在每个设计过程中都是必须的，但是我肯定会推荐他们，尤其是当客户不采用以用户为中心的方法时。如果人物角色不是吸引利益相关者的正确方式，那也没关系。你可以尝试其他工具来帮助你和你的企业与你的用户产生共鸣。

上个月，我参加了由 Tech Circus 在伦敦举办的“领导力设计”活动。很有意思的是，我们听到小组成员谈到了作为领导者以及在我们使用的流程中采取灵活方法的重要性。我发现对于研究和设计没有一个放之四海而皆准的解决方案。尤其是当你在一家机构工作，你要和许多不同的客户打交道，你需要准备好调整你的流程，倾听业务需求，找到理解用户的最佳方式。

因此，以我的经验来看，人物角色是一个很好的交流工具，它们帮助我推动以用户为中心的方法。当客户参与到人物角色的创建中时，人物角色工作得最好。当明确了项目中使用了哪些人物角色时，业务中的每个人，以及设计师和开发人员，都可以将它们作为参考。
