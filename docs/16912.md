# 作为软件工程师如何造假

> 原文:[https://dev . to/jeffshillitto/how-to-fake-it-as-a-software-engineer-jbo](https://dev.to/jeffshilllitto/how-to-fake-it-as-a-software-engineer-jbo)

我要坦白一件事:我不知道我在做什么。直到最近，我还是一名软件工程师，尽管我连使用计算机的资格都没有，更不用说拥有计算机科学学位了。

在过去的 5 年里，我从一家小型创意机构工作到一家大型跨国公司，并最终在澳大利亚领先的科技公司之一[标志性的](https://www.theiconic.com.au/)获得了一个职位，在该国最聪明的软件工程师、数据科学家和产品所有者组成的大型团队中工作。

像许多开发者、制造者、黑客和业余爱好者一样，我自学了编码。

建立一个网站，整合一个 CMS 或电子商务系统，管理一个数据库和管理一个服务器，过一段时间就成了习惯。

在你职业生涯的某个时刻，像我一样，你可能会发现这些简单的项目或独自工作已经不够了，你想成为一名成熟的软件工程师，与大型企业、科技公司或快速发展的初创公司的志同道合的工程师团队一起工作。

随着苹果、谷歌和其他大型科技公司现在表示他们将接受没有计算机科学学位的申请人，造假变得前所未有的容易，所以记住这一点，以下是我关于如何获得理想工作的三大建议。

## 申请你不胜任的工作

这是我的最高提示，也是对我的造假之路影响最大的一条。即使成功的可能性很小，也要申请工作。我建议你瞄准那些要求比你已经有的工作经验多 1-2 年的工作。

除非你非常有才华，否则你很有可能会被拒绝。不要因此而气馁，在其他地方再试一次，但是要准备好面对更多的拒绝。

为什么？因为你得到的反馈将是无价的。雇主会告诉你他们到底在寻找什么样的技能、技术和技巧。经过几次申请和面试后，你会开始注意到一个趋势，你可以忙着学习你没有经验的最受欢迎的技能。

这相当于市场调查，把你未来的雇主当成顾客，弄清楚他们想要什么，然后离开，适应并交付给他们。

几年前，一些问题引导了我的学习方法:

*   解释你对多态性的理解
*   你对代码进行单元测试的方法是什么？
*   说出三种设计模式，以及何时使用它们
*   您将如何使用缓存来优化性能？

## [](#create-your-own-side-project)创建自己的副项目

在我看来，这是最好的学习方式，也是最有收获的。你可以从书籍和在线教程中学习，但它们往往会引导你走上一条特定的道路，完成一个教程不会像创建自己的副业项目那样带来同样的兴奋或成就感。

通过致力于一个副业项目，特别是如果你打算向公众发布源代码或推出软件产品，你会 a)发现自己更致力于学习，b)你将被迫发展你的解决问题的技能——没有指南告诉你该做什么，你需要自己解决问题。

兼职项目允许你实验、实践和展示最新技术、语言、框架、技巧和最佳实践的知识，这些知识你可能没有机会在其他地方学习，比如你的日常工作。

一个副业项目也会给潜在的雇主留下深刻的印象，你生活在软件中，呼吸着软件，你热爱编码，你站在新兴潮流的顶端。

## [](#copy-others-work)抄袭他人作品

他们说，如果你找不到堆栈溢出的答案，那就做不到。

如果你想快速学习和构建东西，那么从 Stack Overflow 复制解决方案，谷歌答案或在 GitHub 上找到类似的东西。

然而，伪装的诀窍是批判性地评估你找到的解决方案，对照其他建议交叉引用，然后寻找如何改进和完善它，使它看起来像你自己的工作。

但是不要就此打住，如果你已经和其他软件工程师一起工作或者认识他们，就复制他们正在做的事情，从他们的辛勤工作和错误中学习。一个好的软件工程师会非常乐意指导你，传授他们的知识。

## [](#you-can-do-it)你可以做到

不出五年，你就能伪装成一名成功的软件工程师，希望你能在一家备受追捧的科技公司获得梦想中的角色。真正需要的是耐心、努力和奉献。

## [](#one-last-thing)最后一件事

如果你成功地伪装了，那么你的工作就是把你获得的知识和智慧传递下去。

回答关于堆栈溢出的问题，写一个博客，或者为一个开源项目做贡献，并鼓励那些和你走在同一条路上的人拥抱他们的好奇心，发展他们解决问题的技能，并帮助他们成为伟大的软件工程师。
