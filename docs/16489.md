# 投票:为什么点击一个开发者评论链接会孤立地显示评论？

> 原文:[https://dev . to/John Carroll/poll-why-clicking-a-dev-comment-link-display-the-comment-in-isolation-2k5n](https://dev.to/johncarroll/poll-why-does-clicking-a-dev-comment-link-display-the-comment-in-isolation-2k5n)

对于那些不熟悉的人来说，当你点击 DEV.to 中的评论链接时，你会被带到一个单独显示该评论的页面(这里的例子是)。如果你想要更多的上下文，你可以(必须)点击`VIEW POST`或`VIEW FULL DISCUSSION`链接。中等的工作方式类似。

## [](#question-do-you-like-this-ux)问题:你喜欢这个 UX 吗？

对我来说，评论本质上是与上下文相关的。

*   有没有一个时候，在没有上下文的情况下查看评论是值得的？什么时候/为什么？
*   当前的 UX 有技术原因吗？
    *   Github 评论链接由 anchor 标签驱动，因此浏览器只需显示完整页面并滚动到相关部分(非常直观)。
