# 自制与购买:你不应该建立自己的 CMS 的 5 个理由

> 原文：<https://dev.to/alexandernaydenov/make-vs-buy-5-reasons-why-you-should-not-build-your-own-cms-2enl>

如何在没有相关成本和风险的情况下获得内部 CMS 的可定制性？考虑一下[无头内容管理](https://graphcms.com/academy/headless-cms)。建立自己的 CMS 是一个漫长的旅程，你不需要再做了。没有必要重新发明轮子或冒不必要的风险。开发人员资源被释放出来，现在可以完全投入到与您的核心业务直接相关的功能上。

### 1。原生的 headless CMS 是高度可定制的

从理论上讲，构建自己的 CMS 可以确保对特定用例的完全可定制性。在实践中，定制系统更可能是一个折衷的拼凑物。这将是内部政治争夺优先特性的结果，最终让每个人都不满意。它将在特定的时刻适合你的过程，但是任何后来的改变将需要重写你的代码的大部分。

一个没有主见的、无头的解决方案(例如 [GraphCMS](https://graphcms.com) )适合最广泛的内容建模和 API 设计需求。前端和后端是分离的，因此您能够定义和构建**所有面向客户的功能**——您的整个前端。具有精细访问权限的自定义角色等功能使您能够调整 CMS 以适应最复杂的编辑流程。

### 2。无头 CMS 的设置时间要短得多

使用一些商业无头解决方案，包括内容架构、API 设计和连接前端，您可以在不到一个小时的时间内建立一个完整的网站。

构建您的内部系统将花费您几个月，甚至几年的时间来设计、构建和调试。

### 3。构建、维护和转换成本更低

内部 CMS 的总成本包括*大笔前期*和*持续维护*成本。对于最基本的 CMS 来说，以开发时间衡量的前期成本将使您花费 18-48 个月的开发时间。持续维护包括托管、CDN、身份验证层、开发人员工具的成本，以及开发人员用于监控和修复 bug 的时间。

一个无头 CMS 的月费用从几十美元开始。即使是要求最苛刻的企业公司也不会为每个项目花费几千美元来获得最高级别的服务。

由于无头系统的模块化性质，转换到新供应商的成本也显著降低。一个本地的 headless CMS 是无个人化的，允许你像乐高积木一样构建你的内容基础设施。如果你决定搬家，你不需要从头开始构建你的前端和后端。

### 4。保持持续创新的方法

最好的情况是，内部解决方案只能在一段固定的时间内满足您的需求。如果你从不改变你的流程，从不采用新技术(例如，智能手机或接下来出现的任何东西)，这是可以的。或许，你需要适应继续做生意。因此，您需要在 1-2 年内扩展您的系统。再一次，一个新的项目团队来集思广益，创新，构建，测试，部署和修复错误。

一个无头的 CMS 公司的面包和黄油是在创新。升级将会定期进行，而且费用会低得多，如果不计入现有价格的话。选择一家具有成长型思维的年轻公司，意味着至少在未来 5-7 年内，令人兴奋的新功能会不断涌现。

### 5。一种避免处理不想自己构建的系统的方法

像无头 CMS 这样的复杂解决方案由多个子系统组成，这些子系统必须完美协调地工作。对于其中的许多组件，您肯定不想亲自维护它们:服务器设置、安全层、认证层、数字资产管理和 CDN 设置等等。其他子系统需要由具有难以找到的专业知识的工程师来构建:内容模型 API、GraphQL 服务器设置、工作流设计、用户角色设计等等。
