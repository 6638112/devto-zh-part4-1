# 帮助想出一个术语来描述针对开发人员和测试人员的产品？

> 原文:[https://dev . to/GD Cohen/help-come-up-with-a term-description-a product-targeted-at-developers-and-testers-43ic](https://dev.to/gdcohen/help-come-up-with-a-term-to-describe-a-product-targeted-at-developers-and-testers-43ic)

我们(zebrium.com)希望有人能帮助我们想出一个术语来描述我们的工作。

我们正在开发一个产品，帮助开发人员和测试人员发现、分类和修复 CI/CD 管道中的问题。我们的技术通过使用机器学习来完美地构建日志数据。从那里，我们可以对日志数据执行有效的异常检测(这是发现问题的一个很好的代理)，并建议允许您“指纹化”一系列事件(包括事件中的参数值)的签名，以便在问题再次发生时识别问题。因为底层数据是结构化的，所以我们也使得导航和构建任意复杂的查询变得非常容易。

我们一直在试验“CI/CD 取证”、“开发/测试取证”、“CI/CD 验证”等术语。这些术语中的任何一个能引起共鸣吗(例如，作为开发人员或测试人员，你会对这个名称感兴趣吗？这个名称能抓住上面描述的东西吗)？

任何建议将不胜感激！
