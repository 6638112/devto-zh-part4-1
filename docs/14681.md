# ELI5:为什么自托管编译器？

> 原文:[https://dev . to/decudio ly/Eli 5-why-self-host-a-compiler-hll](https://dev.to/deciduously/eli5-why-self-host-a-compiler-hll)

让你的新语言的编译器自托管有什么好处？对我来说，这似乎只会增加你的构建的复杂性。如果它能为你的语言生成代码，那么用什么语言写又有什么关系呢？

我知道编译器是一个很好的概念验证程序。它们通常是无依赖性的，会让你的语言通过测试。用你的新语言编写一个编译器的练习是一个好主意，我不是在争论这一点。我不太清楚为什么让自托管编译器成为你的语言的规范版本是有益的。用 Haskell 或 OCaml 这样的语言编写编译器，难道不会使它更具可扩展性、更灵活、构建更快吗？
