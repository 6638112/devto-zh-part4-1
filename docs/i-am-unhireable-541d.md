# 我不能被解雇

> 原文：<https://dev.to/vorahsa/i-am-unhireable-541d>

在围绕程序员招聘的讨论中，我经常遇到一些人谈论他们对谁能获得面试机会的简单标准。当考虑这些标准时，我通常会发现我不符合这些标准。我猜这一切让我无法被解雇？

(引号中的短语是我看到的常见态度的释义，没有引用任何特定的人。)

## 我现在 45 岁左右

“编程是年轻人的事。”直到 35 岁左右，我才成为一名专业开发人员，尽管在那之前我的工作确实涉及到编码。当然，我不认为我的思维像以前那样敏捷了，但是这也帮助我避免让我的代码变得太复杂。而且我确实有相当多的经验，甚至在现已过时的平台上的经验也能证明是有用的。

## 我在同一个地方工作了 10 年

"我怀疑任何在同一个地方呆得太久的人。""一份工作的最佳时间长度是 2-3 年."我实在看不出仅仅为了改变而改变有什么意义。我喜欢我的工作，我喜欢这家公司，我有机会在有趣和富有挑战性的方向上发展自己。

## 我在大学里学会了编程

"所有优秀的开发人员都是从年轻时开始，玩他们童年时代的电脑."我小时候我们确实有电脑，但我主要是用它来玩游戏。我知道一些基本的，但无论我用它做了什么，后来都没有多大帮助。我甚至不明白子程序的意义。当然，总体来说，它可能帮助我适应了计算机，但是我并没有做任何人们期望的事情。

## 我的 Github 很小，无关紧要

“我总是浏览候选人的 Github，期望找到专业的作品集。”以下是我的 Github 来稿

此外，我的 Github 中几乎所有的东西都是小项目，没有展示我实际的专业工作。任何浏览我的 Github 的人都会对我的技能有非常扭曲的看法。

## 我对编码没有“热情”

"优秀的开发人员把空闲时间花在编码上，他们不能停下来."我从来不觉得只是为了编码而编码。我偶尔会有兼职项目，但这些都是我想解决的真正问题，其中的解决方案涉及编码的东西。我确实喜欢编码，这很好，因为这是我的工作，但是当我结束工作时，我会在空闲时间做些别的事情。

## 我有一个博士学位

"我想雇用务实的人，而不是象牙塔里的学者。"这也许在任何地方都不是一件坏事。但是在我的祖国芬兰，它绝对是。我的一些在该行业工作的同学在工作中隐藏了他们的博士学位，以避免被束之高阁。最近，我有一种感觉，在软件开发中，一般来说，越来越多的人开始反对更多的学术背景。

## 尽管如此...

尽管我在所有这些方面都是“不可解雇的”，但我仍然被雇用，仍然在工作中受到尊重，并且在我的编程和社交技能方面一直得到很好的反馈。所以也许对一个特征做出快速判断不是最好的方法？

当我的同事谈到使用这些标准或类似的标准时，我确实很难过。我已经习惯说“那样做意味着你不会雇用我。这就是你想要的吗？”但是，正如人们所料，这并没有真正让他们重新评估自己的想法。

最后，当然所有这些都无关紧要，因为我拥有开发人员最抢手的技能:我擅长在白板上编写算法。
