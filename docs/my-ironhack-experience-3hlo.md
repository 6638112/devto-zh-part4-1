# 我的 Ironhack 经历

> 原文：<https://dev.to/abijsummers/my-ironhack-experience-3hlo>

从互惠生，到学生，再到助教。

### 在 Ironhack 之前

我是 2018 年 8 月 [Ironhack](https://www.ironhack.com/en) 的学生。在此之前，我几乎没有做过编码——我试图自己学习，但总是停滞不前/没有动力。2018 年 3 月，我参加了一个 WeCode 研讨会，在那里我学到了 HTML、CSS 和 JavaScript 的基础知识。活动结束后，在与 Ironhack 的员工/校友交谈后，我申请了这门课程，并获得了预习的机会。

我花了很多时间在 [codewars](https://www.codewars.com/) 上做前期工作和在线代码挑战，因为我想做好准备。在课程开始之前，我还参加了两次 hackshow(刚完成训练营的学生展示他们的最终项目),看看有什么可能。在两次 hackshows 之后，我非常兴奋地开始了——看到学生们在九周的编码之后能够构建出什么，这是令人难以置信的。

### 铁黑学生

我热爱学生时代。真的很艰难，有些日子比其他日子更艰难。知道班上的每个人都在经历同样的事情，真的让我更容易坚持下去，拥有一个令人惊叹的教学团队帮助了我很多。

在课堂上，会有课程和练习的混合。有些练习是单独的，但大多数练习是成对的。每一组练习，每个人都会被分配给不同的人。Ironhack 的每个人都有不同的背景，所以每个人都有自己的优势。在训练营期间，有三个项目。项目一是使用 HTML、CSS 和 JavaScript 创建的基于浏览器的游戏，项目二是使用 Node.js 的全栈应用程序，项目三是使用 Node.js 和 React 的全栈应用程序。

作为一名学生，项目二是我最喜欢的项目。我真的很喜欢学习后端以及一切是如何联系在一起的。讲述安全性的基础知识，学习如何存储数据以及如何检索数据，这很有趣。

### Ironhack 的助教

完成课程一周后，我开始做助教，而冒名顶替系统是真实存在的。我不确定从哪里开始帮助，也不确定我是否能够解释概念。但是经过三批之后，我学到了很多，并且接受了不知道所有事情也没关系。

#### 以下是我作为助教学到的一些东西:

*   每个人都能编码。你的背景如何，年龄多大，都不重要。如果你专心致志，愿意花时间和精力去编码，这是有可能的。你不需要一台昂贵的电脑，如果你不能参加训练营或课程，在家学习也是可能的。
*   建造东西是最好的学习方法。建造你想建造的东西是很重要的，因为你更有可能回到过去。你会不断地学习，这意味着你可以不断地改进你正在构建的东西。
*   不要害怕打破你的代码。打破你的代码是学习和理解正在发生的事情的最好方法之一。试错是知道如何解决问题的重要部分
*   用简单的方式解释一个复杂的概念确实有助于你理解它
*   学习永无止境——你总是需要寻找解决问题的方法。总会有你不明白的地方，我们鼓励你寻求帮助
*   你对你所学的东西有完全的自由——如果你对某个特定的东西感兴趣，那么就去找一些文章，做一些东西来练习或者参加一些活动。编程是一个非常广阔的领域，你永远不会知道所有的事情，所以花时间挑选你感兴趣的
*   休息一下很重要。如果你被卡住了，无法解决问题，离开电脑休息一下会很有帮助。10 分钟后带着清晰的心态回来真的很有帮助
*   使用 github 是追踪你的进度的好方法。你将能够回顾你以前做过的代码，看看你提高了多少，如果你在找工作，这将向雇主展示你有能力做什么。你甚至可以在 github 上免费托管一个静态站点
*   参加活动是结识业内人士的一种很酷的方式。你会遇到各种水平的人，并能从他们身上学到很多东西

我见过很多人参加过 Ironhack 训练营。这并不总是容易的，但这是值得的。你会遇到一些非常酷的人，他们有着不同的背景，也有你通常不会遇到的人。你可以建立一个支持网络，不仅是和你一起学习的人，还有以前的学生和未来的学生。学习编码没有对错之分，只是需要时间、努力和奉献。
