# 如何策划一个项目？

> 原文：<https://dev.to/rlxdprogrammer/how-to-plan-a-project-3h6h>

介绍

当我与其他软件开发人员交谈时，我真的经常听到他们这样的观点:我们只是因为这个项目的糟糕规划而受苦，当然公司从我们的工作中赢得了很多钱，我不知道项目经理们一直在做什么等等。

为了更好地了解这个主题，我想写一篇关于项目计划和管理的简介，澄清一些不清楚的地方。

有几种不同的工作模式，项目规划模式和情况。由于我是一名开发人员，我也没有很深的知识，所以我真的希望只在高层次上展示所有这些事情是如何工作的。

# 规划

在开始一个项目之前，该项目应该是工厂。需要确定项目的确切目标是什么，所谓的包含活动是什么，例如:硬件集成是项目的一部分吗？需要哪一级别的测试？接下来，所使用的技术至少需要在高层次上决定:应该使用哪种编程语言，在开发过程中还需要哪些软件和硬件？

基于其结果，应该进行技能分析:从事项目工作的团队需要哪些技能，需要什么级别的技能？例如，如果它是一个使用 QT framework for GUI 用 C++开发的项目，用 Google Test framework 测试，它需要在 Windows 和 Linux 上运行，那么团队将需要 GUI 应用程序的经验，基于 GUI 的应用程序的软件架构的知识，强大的 C++和 QT 知识以及 GTest 和 GMock 的经验。

一旦完成，是时候深入项目了。整个项目需要分成更小的步骤，因此需要定义所谓的里程碑，例如:

1.  里程碑:GUI 原型，没有后端功能

2.  里程碑:用户处理相关功能:登录、注销、新用户注册、更改密码

3.  里程碑:使用 X、Y 和 Z 功能

…

名词（noun 的缩写）里程碑:具有完整功能的应用程序，经过全面测试。

规划 N+1 也是一个好的实践。错误修复的里程碑。不同的里程碑应该具有大致相等的大小。这种实践对于传统管理的项目(V-model 等)也是一样的。)和敏捷项目。

# 估计

下一点是对里程碑大小的估计。这仍然是所谓的高水平估计。为了能够做到这一点，它需要在一个高层次上列出所有应该为一个里程碑做的事情，并需要添加一个粗略的时间估计。这种评估可以在一个月、一个星期、一个工作日，甚至是冲刺或故事点中完成。这种估计是计算里程碑的到期日以及项目不同阶段所需的团队规模和设置的基础。而这一切都将是成本估算的一大依据。

# 如何计算价格

让我们看看价格里都需要包括什么？通常最大的成本是开发人员的工资。通常不是根据实际工资计算，而是根据给定知识水平的平均工资。举个例子:根据估计，该项目将需要 10 个月，整个长度，我们将需要 2 名高级开发人员和 4 名初级人员。高级开发人员的平均月薪是 6000 美元，初级开发人员是 4500 美元。整个开发团队的工资成本是 30 万美元。还需要考虑的是，还有管理人员，他们也会从事这个项目。通常公司里还有额外的人员:人力资源部、高级管理层、销售部等。他们不直接参与项目，但工资需要由项目收入支付。这些成本通常被称为管理费用，并设置在一定比例的额外成本中。

此外，还需要为项目制定培训预算。您还需要考虑硬件和软件许可成本。如果客户与贵公司不在同一城市，可能还需要考虑额外的差旅费用。

你还需要考虑团队的办公室租赁费用。

一旦收集和评估了所有的东西(基于项目的预计长度和团队的规模)，你需要考虑公司想从项目中赚一些额外的钱，以及如果事情进展不顺利和额外的成本进入画面(例如。:项目需要的时间更长)项目应该还是有收益的。所以你应该为这个项目设定一个明显高于成本的价格。

# 项目的跟踪

基于所有这些评估，项目经理将为项目制定一个时间和成本计划，并跟踪当前的项目状态。因此，如果团队开始落后于预计的时间，项目经理需要做出反应，以便能够保持项目对公司有益。

尽管如此，仍然有许多项目赚的钱比成本少。在这种情况下，公司需要从其他项目转移预算。

有时公司开始的项目从一开始就很清楚不会有什么好处。它通常有一个战略原因:他们想赢得一个新客户或建立一个特定领域的知识库。

如果你需要提前计划整个项目，那是相当危险的。幸运的是，还有其他方法:基于时间的方法，当客户在项目完成的实际工作时间后付款。在这种情况下，几乎没有风险，但当然潜在的好处也较低。作为替代方案，当每个较小的任务都有价格时，您可以使用敏捷方法，并且您可以定期与客户一起决定应该完成哪些任务以及价格。
