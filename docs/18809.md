# 准备你的第一次演讲

> 原文：<https://dev.to/3sanket3/preparing-your-first-talk-5emg>

在这篇文章中，我想分享我准备演讲的经验和演讲时需要考虑的事情。

这篇文章的目标读者是计划第一次发言或者刚刚开始在会议上发言的开发人员。

经常发言的人可能从这篇文章中得不到什么东西。但是，我们也想学习你的经验。请一定要分享和指导。

## 你不必成为专家😎

是的，你应该做好充分的准备，努力证明你的听众在你身上花费的时间是合理的。但是，你不必成为专家。

有时候，你可能没有在你的常规项目中做你要交付的主题。在这种情况下，至少在分娩的前几周，试着了解你的话题的周围环境。尽量多做例子。查看实用的解决方案和案例研究。

你可能无法回答观众的一些问题，但这没关系。你可以扔给观众。或者，你可以得到问的人的联系方式，回家，研究，回复。

## 可以有一个公开的讨论话题，而不是对单一话题的深入探讨👐

我们刚刚在印度苏拉特成立了一个聚会小组。在我们的第一次聚会中，我们不确定本地社区希望从庞大的 JavaScript 生态系统中得到什么主题。所以，我们把它作为一个公开的讨论。我们中的两个人简要介绍了 JavaScript 的各个领域，并试图向观众开放来讨论它们。

我们发现这真的很有帮助，因为第一次接触的人能够专注于*出来讲话*，而不是担心深入的技术知识。

## 与某人配对👭

同时有助于获得信心，中间的休息帮助你安顿下来，重新开始下一个话题。

## 通过给你的朋友做演示演讲来练习👨‍🏫

这对于练习你的演讲和复习是非常必要的。如果可能的话，您可以尝试向拥有不同技术和经验水平的朋友进行演示。它将使你的内容对你实际上将要拥有的各种各样的观众来说是完美的。

## 大量练习你的代码👨‍💻

如果你打算进行现场编码(你应该这样做)，你最好至少练习 3-4 次。这样你可以训练你的潜意识，即使在紧张的时候也能少犯错误。

## 如果你在现场编码时犯了错误，观众会学到更多🐛

从您在现场编码时所犯的错误中，观众将了解到复杂的情况。所以不要担心错误。除此之外，你还在和所有观众进行结对编程。如果你陷入困境，他们可能会帮助你。

只要确保你清楚这些概念并有足够的实践，这样错误就不会完全阻碍你并花费大量时间。

或者，你也可以预先录制你的代码并随身携带，这可能有助于在最糟糕的情况下进行救援。

## 保持你的社交账户的最新状态💄

你将成为 meetup/conf 的英雄之一。人们很可能会试图在互联网上找到你。你最好给他们看看最新的信息。

## 你有了营销的机会📢

如果你正在开发一个产品，为开源做贡献，开始一份时事通讯，设计课程或者做任何令人兴奋的事情，人们会在那里听到它。

## 待人和善，表示感激🙏

感谢组织团队给了你一个演讲的机会。

此外，您还可以免费感谢您在演示中使用的服务。可以是免费的 PPT 模板，图片，gif 或者其他任何东西。这将有助于这些服务接触到更多的受众，并最终使其变得更好。

希望这对您有所帮助。还有，在评论里加入你的建议，让我和其他人学习你的经验。
