# 开发人员犯的头号错误

> 原文:[https://dev . to/kylel Johnson/the-1-mission-developers-make-5ff 5](https://dev.to/kyleljohnson/the-1-mistake-developers-make-5ff5)

我从事软件开发已经二十多年了，我一直看到开发人员犯同样的错误。然后，他们想知道为什么他们没有赚到他们认为应该赚的钱，没有晋升的机会，并且在遇到编程/应用程序问题时经常感到沮丧。

## 1.开发人员不了解围绕他们代码的整个环境

根据我的经验，今天大多数开发人员都是专家。这没什么不对，但随之而来的肯定会有挫败感。如果问题发生在您的代码中，而不是由您的代码引发的，您将很难找到问题，因为您对代码周围的环境没有任何线索。

例如，假设您是一名 web 开发人员，您将代码部署到服务器的那一刻，它就爆炸了(开发语言)。我一次又一次地看到开发人员花费数小时在代码中寻找问题，或者更好地搜索堆栈溢出。他们这样做的一个原因是因为那是他们唯一熟悉的地方。他们不知道 web 应用程序如何与底层的 web 服务器、DBMS 或操作系统交互。我见过一些 web 开发人员，他们只知道一种服务器端语言和一些 HTML。

我不知道有多少次一个开发人员向我寻求关于应用程序问题的帮助，而这与代码无关。

我承认有些地方有 100 人的团队，如果你有服务器或数据库问题，你可以通过即时消息向某人寻求帮助，但请记住，你必须等待某人的帮助，这会降低你的工作效率。为什么不花些时间了解一下代码周围的环境呢？不会有伤害。知识就是力量。
