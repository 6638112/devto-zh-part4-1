# 我的 IndieWeb 时间轴

> 原文：<https://dev.to/johanbove/my-indieweb-timeline-50on>

我要列出我是如何加入[# indie web](https://social.johanbove.info/tag/indieweb)-pecha kucha style *-ish 的...-随后进行演示🤞

*   2017 -受够了广告收入支持的社交媒体筒仓被商业利用和数字跟踪和关注。
*   2017 / 2018 -测试了乳齿象，但被大量无趣的帖子淹没了，我无法决定加入哪个乳齿象实例。我也对开源项目的管理方式有些担忧。建立一个自己的实例需要一个更昂贵的网络主机服务，这是我在我的优秀的网络主机公司对手上做不到的。([https://www . antonist . nl](https://www.antagonist.nl))
*   2018/2019——蒂姆·伯纳(thé Tim Berner)的李(Lee)发现了 SOLID——通过查看 SOLID(测试版)用户界面的当前源代码，并试图修复一个 bug，从而(有点)参与进来。与 Melvin 一起参与测试和编写一些文档。这是一个难以下咽的饼干，因为有大量的旧代码和一个时髦的用户界面，有一点学习曲线。尽我最大努力去理解 RDF 和关联数据。关于 Solid 如何向前发展的学术讨论让我有点受挫。给所有很酷的 JavaScript 演示留下了深刻的印象，并认识了一些非常聪明的人。(梅尔文，鲁本，列文，...)再加上这个项目在我的出生地比利时根特有一个重要来源。但是我现在已经不再跟进这个项目了...主要是因为我不能运行节点。我的 web 主机上的 JS 服务器。
*   2019 年——在 Mozilla 的播客中发现了 IndieWeb 运动，决定开始使用 Known。对坦特克和亚伦的作品印象深刻。从 Greg 和其他 Indieweb“大师”那里得到了很多受欢迎的帮助。我很高兴看到 Known 在 Apache2 上用 PHP 和 MySQL 运行得很好，但看到 Ben 和 Idno 项目经历了非常艰难的时期有点令人担忧。幸运的是，Marcus 在保持开源项目的活力方面做得非常出色。

了解 Web reference、microsub 和 micropub，并尝试所有 Aaron 和其他有才华的开发人员的应用程序，给了我希望和动力，这可能是个人网络应该继续发展的方式。

Indieweb 原则摇滚！[https://indieweb.org/principles](https://indieweb.org/principles)
