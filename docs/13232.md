# PHP 死了吗？PHP 未来五年的就业前景和未来如何？

> 原文:[https://dev . to/yedukondalu 319/is-PHP-dead-what-the-job-outlook-and-future-of-the-next-five-yedukondalu 319-i3m](https://dev.to/yedukondalu319/is-php-dead-what-is-the-job-outlook-and-future-of-php-in-the-next-five-years-i3m)

PHP 远未死亡。由于我们总是收到类似的问题，我将重复另一个讨论的答案。

截止到 2019 年，PHP 还相关吗？

COBOL 是一种 1959 年的恐龙语言，部署在数万台设备上，在 2019 年仍然适用。

PHP 在 2019 年得到积极开发，并部署在数亿个安装上。

因此，仅仅是“在许多网络应用中仍然广泛使用”就足以证明学习它是正确的。

此外，PHP 非常容易学习，在任何地方都可以使用，不知不觉中:

```
a bad reference on your curriculum,
like not knowing any shell or bash command. Sure, you can ignore those but it shows the moment you are assigned some simple tasks. Your boss will know. Your colleagues will know. 
```

PHP 有什么好的吗？

PHP，尤其是 7.0 版本之后，快速、简单、语法丰富。它带有嵌入式安全库，同时也是每个主机提供商提供的最便宜的语言。PHP 运行在多个 CPU 内核上(Node.js 需要额外的工作),通过一个额外的脚本，可以同时用作同步和异步(就像 Node.js 一样)。

PHP 实现了类型提示、接口、外观、特征、单例、闭包、结构化异常、通用容器、迭代器、名称空间、包(Composer，类似于 npm)、单元测试、持续集成…
它附带了数千个打包的(和独立的)库、MVC / REST 框架。有些是行业标准，例如 Laravel。所有这些都可以一点一点地学习，因为基础 PHP 就像写“print("Hello world ")”一样简单。即使是小孩也可以学习 PHP。
因此，你可以在世界各地找到便宜的 PHP 开发人员，而不仅仅是在加州。
