# 类比命名-是还是不是？

> 原文:[https://dev.to/idanarye/analogical-naming-yay-or-nay-3jh5](https://dev.to/idanarye/analogical-naming-yay-or-nay-3jh5)

我正在做一个项目，正在为两件困难的事情中的一件而奋斗——命名。我有名字——但我并不真的喜欢它们(我能为两个主要角色找到的最具描述性的名字只有一个字母不同)

当我给这个项目取名字的时候，我用了一个类似于奇幻冒险的比喻。软件本身与幻想类型无关，但我开始考虑使用相同的类比来命名我的架构中的一些主要组件和过程。

但是这个我还是不确定。命名项目本身是很好的——项目需要独特且容易记忆的名称——但是项目内部的名称应该尽可能具有描述性，以帮助需要理解什么做什么的读者。我并不是建议任意命名——我有一个清晰简洁的类比，我不会随便为每个需要命名的软件术语选择一个幻想的术语——但是我仍然担心将类命名为`Adventurer`或`Wizard`或`GuildHall`会显得不专业...

我应该去吗？
