# [](#what-is-codetalks)🤓什么是密语？

作为程序员，我们知道在编程语言、框架、方法和工具方面保持最新是多么重要。我们必须不断学习。保持与时俱进的最佳方式之一是参加技术会议和活动，但许多这些活动不在我们的城市，或者我们根本无法参加。积极的一面是，这些活动中的大多数都被录制下来，并可以通过不同的视频平台(主要是 Youtube 或 Vimeo)免费获得，这很好，但也有一些问题:

❌:每周世界各地都有很多有趣的事件和谈话，很难意识到它们的存在。

❌每个事件发生在不同的日期，发布在不同的账号和视频平台上，这使得追踪我们感兴趣的事件变得非常复杂。

这就是为什么我们决定创建 [Codetalks.tv](https://codetalks.tv) ，这是一个为开发者提供的**平台
，它将全世界最好的编程讲座汇集在一个地方。**每个视频都按标签和事件分组，因此发现会议、观看讲座和继续学习非常容易。

# [](#what-should-our-mvp-have)📋我们的 MVP 应该有什么？

考虑到这是一个 MVP(最小可行产品),我们不希望过多地延长开发时间，因此，我们试图确定赋予项目意义的关键功能，这样我们可以在继续投入更多时间和精力之前验证我们的想法是否有意义。我们认为最重要的特征是:

*   会谈搜索器
*   按标签查看会谈
*   按事件查看会谈
*   查看可用标签
*   查看现有事件
*   发送演讲(Youtube 和 Vimeo)
*   订阅特定标签
*   接收反馈和想法

至于设计，它必须简单直观。创造任何新的和奢侈的东西是没有意义的，这就是为什么我们选择了一个类似于 Youtube 的设计，这样任何人都可以立即感到熟悉。

# [](#%F0%9F%9B%A0-lets-get-to-work)🛠我们开始工作吧

是时候编码了！

关于副业的一个有趣的事情是，你可以负担得起实验和尝试新事物，所以，我们不想错过这个机会。

这里是我们使用的技术栈的一个简要总结，包括后端和前端。如果你觉得有趣，我们会发表更多的文章，更详细地解释开发过程的一部分。

### [](#frontend)前端:

*   [创建 React 应用](https://github.com/facebook/create-react-app):创建没有构建配置的 React 应用
*   [Apollo](https://github.com/apollographql/apollo-client):graph QL 的完美合作伙伴。该客户端旨在帮助您快速构建一个使用 GraphQL 获取数据的 UI。
*   [Styled-Components](https://www.styled-components.com/) :一个非常流行的 CSS-in-JS 样式库，它使用 JavaScript 中的标记模板文字和 CSS 的强大功能来提供一个平台，允许您编写实际的 CSS 代码来样式化组件。

#### [](#other-libraries-or-tools-that-have-helped-us-a-lot)其他对我们帮助很大的库或工具:

*   [Storybook](https://github.com/storybooks/storybook) :开发和组织 UI 组件的开源工具
*   [Plyr](https://github.com/sampotts/plyr) :这个库是金的。它允许我们嵌入来自 YouTube 或 Vimeo 的视频，并完全定制它们:放上我们的标志，从原始平台上移除按钮…等等..
*   Formik :用简单的方式创建表单(每个人都讨厌表单，对吧？).与输入验证库 [Yup](https://github.com/jquense/yup) 一起使用非常有用。
*   这是另一个简单的库，用颜色执行小操作。

### [](#backend)后端:

*   [GraphQL-Yoga](https://github.com/prisma/graphql-yoga) :基于 Express 的全功能 GraphQL 服务器
*   Prisma 是一个数据库抽象层，它将你的数据库转换成具有 CRUD 操作和实时功能的 GraphQL APIs

### [](#third-party-tools)第三方工具:

*   Sentry :这是我们用来接收发生在前端和后端的错误的工具。这样我们就可以消除出现的小错误。
*   Mailchimp :这是用户订阅标签接收最新视频的方式，当他们这样做时，他们的电子邮件帐户会被添加到 Mailchimp 列表中，然后我们可以从那里发送最佳视频的每周摘要。
*   Cyfe:我们使用 Cyfe 将来自平台不同部分的信息放在一个地方。例如，我们有一个用于分析的面板，另一个用于查看平台上的用户和视频数量。我们可以继续从该平台添加数据和内部信息，以便将它们都放在一个地方。
*   Typeform:我们使用 Typeform 来接收用户的反馈，因为这是一种简单快捷的方法来实现它，并将其与 Slack 集成以即时接收信息。

然后是真相大白的时刻…

# [](#the-launch)🚀发布会

事实是，如果我们离开了技术或产品部分，我们会感觉像离开水的鱼，有人会说营销不是我们的强项。这就是为什么，为了定义我们的发布策略，我们简单地问了自己一个问题:我们的潜在用户在哪里？，根据我们找到的答案，我们列出了可能的获取渠道:

### [](#%E2%98%A0-big-portals-disaster)☠大门户(灾难):

*   **寻找产品:**我们试图遵循我们在不同博客上读到的关于在 PH 成功发布的所有建议，我们知道这非常重要，也是发布的关键之一，然而我们只得到 [9 张赞成票](https://www.producthunt.com/posts/codetalks-tv)(非常难过)。
*   **黑客新闻:**我们只拿到了 [4 分](https://news.ycombinator.com/item?id=19349321)。
*   Reddit: 我们把它发给了 subreddit [r/programming](https://www.reddit.com/r/programming/) 。也没有发表。

在这一点上，在一个真正可怕的初始版本之后，我们想知道[代码对话](https://codetalks.tv)是否真的有意义，我们真的在解决一个问题并给社区带来一些东西吗？我们的动力崩溃了。

然而，经过几天的下降，我们决定继续尝试其他方式，幸运的是，这一次更好:

### [](#%EF%B8%8Fniche-media-not-bad)✌ ️Niche 媒体(还不错):

*   【devs 博客:我们已经出现在该领域的几个重要博客中，这为我们带来了非常合格的流量。
*   **Devs 简讯:**我们在一些简讯中被引用，其中许多都有成千上万的订阅者。
*   Twitter: 从一开始，它就是一个伟大的盟友，我们通过这个社交网络获得了大量的流量，部分原因是我们开发了一个非常酷的机器人，它整天发布演讲并提到演讲者(这是另一个帖子)。

尽管没有一个壮观的发布，我们已经看到在过去的一个月里，流量一直在稳步增加。此外，我们收到了许多积极的反馈，因此总体感觉是人们喜欢它，尽管我们认为我们还没有达到产品适合市场的程度，但我们确实相信我们正在朝着正确的方向前进，值得继续投入更多的时间。

此外，我们对 2.0 版本有一些非常酷的想法，并且…为什么不呢，我们会在以后发布另一个有经验的版本，也许这次我们会更幸运呵呵呵。

# [](#what-we-learnt)🎓我们学到了什么

如果我们必须吸取一些重要的教训，那就是:

*   尝试**解决一个真正的问题**，一个你在自己的肉体中发现的痛苦。这样，你就成了自己的目标客户，你就知道自己需要什么。
*   保持事情简单。不要对 MVP 应该拥有的技术栈或特性着迷。尝试快速启动并尽快验证你的想法。
*   既然你在做副业，试着在这个过程中学习新的东西。如果这个项目最终失败了，你至少会带着新知识和一些有趣的东西加入到你的文件夹中。
*   如果你的项目一开始没有得到你期望的所有关注，不要气馁，这可能有不同的原因。耐心点。
*   倾听你的听众的意见，并试着一点一点地改进你的产品。
*   **进行多次发射**。把你添加到产品中的每一个新特性都看作是一个新的机会来发布新的产品并引起关注。
