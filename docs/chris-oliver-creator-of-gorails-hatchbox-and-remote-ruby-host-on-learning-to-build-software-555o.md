# Chris Oliver:GoRails、Hatchbox 和学习构建软件的远程 Ruby 主机的创建者

> 原文：<https://dev.to/jacobherrington/chris-oliver-creator-of-gorails-hatchbox-and-remote-ruby-host-on-learning-to-build-software-555o>

Chris Oliver(在 Twitter 上被称为[exc D3](https://twitter.com/excid3))是 GoRails.com 的创始人，这是一个为有兴趣学习 Ruby on Rails 的开发者提供的截屏和学习服务。他还创建了 Hatchbox.io，这是一个网络应用托管服务。

除了是一名成功的开发企业家，Chris 还主持远程 Ruby 播客。

周三，克里斯和我一起为我的播客 [devpath.fm](https://www.devpath.fm/subscribe) 录制采访。我们谈到了他在软件行业的起步，他的一些成功，以及他的一些挣扎。

Chris 的生活经历中的一个共同趋势(以及我有幸记录的大多数其他[开发者经历](https://www.devpath.fm/episodes))是对学习新事物的热情。

因此，他花时间向开发人员讲授 Rails 的复杂性也就不足为奇了，因为他的大部分时间都花在了掌握他在视频中分享的技能上。

在我们的谈话中，Chris 传达了三四条关于他自己的旅程的非常有价值的见解，这些见解促成了他迄今为止在职业生涯中所经历的成功。

### 1。重复作品

Chris 讲述了他构建的一个早期应用程序的故事。一个基于 GUI 的工具，用于在不稳定或不存在互联网的情况下下载 Linux 包。

这一过程冗长乏味，历时五年之久。他重写了应用程序 6 或 7 次，但最终，这是他学习编码过程中最重要的一步。

应用程序的每一次迭代都比前一次要好。到最后重写的时候，Chris 已经牢牢地掌握了 Python，并且启动了一个非常成功的兼职项目。

[![Bart Simpson writing "I will not write fat controllers" on a chalkboard](img/e278e1180a6ea86607b6e2b083e24f83.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--vrllkYgE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/l9t3f34nphr1vqxxl66v.png)

这种重复的实践一直伴随着克里斯，从早期的兼职项目，到他作为顾问和教育家的职业生涯。

在采访中，他说，“如果我在教什么东西，我会花上一周的时间来学习。”

此外，克里斯把这个想法追溯到学校的数学。在学校，你被介绍一个概念，然后你的老师布置作业。没有什么比 40 个相同问题的例子更能帮助你记住课堂上讨论的过程和概念了。

当你学习编码时，重复是有用的。很难，但很管用。

### 2。在公开场合学习

Chris 构建的这个项目，用于在没有太多互联网接入的情况下下载 Linux 包，对于刚刚学习编码的人来说，是一个相当大的成功。以至于他出现在了一本 Linux 杂志上。

Chris 在那个小应用上花费如此多时间的原因可以追溯到他所参与的社区。

Chris 谈到在论坛和聊天室中从 Linux 用户那里获得帮助，并希望回馈给那些乐于助人的人。这个项目是他实现这一目标的方式。

一旦他分享了它，它开始得到牵引。其他上网有困难的人下载了这个应用程序，并用它来下载他们想要的软件。

通过公开这个项目，Chris 找到了一个巨大的动力来源来不断改进它并继续工作。他也从建造人们真正使用的东西中获益。后来，克里斯得到了他的第一个工作机会，因为他能够谈论他在学习时写的代码(真实的人依赖于这些代码)。

### 3。打破困难的概念

在每次采访中最难的部分，我会问我的客人关于[冒名顶替综合症](https://www.jh.codes/ryan-bigg/)的问题，我会让他们分享一个他们一直在纠结的领域。克里斯谈到了他对前端和算法思维的失望。

[![Man Working Using A Laptop](img/fc18b4e64c4c40b637002a2c63a76a88.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--dwLfhRam--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images.pexels.com/photos/2696299/pexels-photo-2696299.jpeg%3Fauto%3Dcompress%26cs%3Dtinysrgb%26dpr%3D2%26h%3D750%26w%3D1260)

当我问克里斯他是如何处理这个问题的，他告诉我，这只是一个认识到当它被分解成更小的概念时，每件事情都是可以学习和实现的问题。对他来说，把问题分解成小块意味着它可以被解决，一步一步来。

克里斯用同样的方式准备他的截屏，将一个问题分解成一个易于理解的视频，观众可以跟着一起看。因为它有效。

将 Chris 的一些技术整合到你自己的学习中可能是一种真正有效的方式，可以帮助你作为一名开发人员建立自己的技能集，或者像 Chris 一样建立一家企业。
