阅读时间:5 分钟

**先生。候选人，您已被批准进入招聘流程的下一阶段。现在，我们将发送一个实际测试，□**

一句同时给候选人带来欢乐和扣押的话。没错，不是所有的东西都是花。即使对于最有经验的人来说，这些话也给甄选过程带来了荒谬的动机。

但为什么呢？什么原因？是什么让我们害怕这样的考验？

## **个人看法**

在 15 年的专业市场经验中，我遇到了各种各样的选择性测试。当然，我经历过很多(rsss)，因为谁来判断我的简历，很快就会问:你为什么变化这么大？-但这是另一篇文章的主题。

回到选择性的过程。我经历过很多这样的事情:技术面试，考试，仅仅相信简历上的内容，在酒吧里的谈话……没错，和公司建筑师一起喝啤酒的谈话。总之，到目前为止，我无法想象的过程，在某些情况下，我无法理解到底哪一个是有效的。

## **该公司的广告说:**

必备条件:

*   Spring Boot 中的高级知识；
*   某些前端 JS 中的高级知识；
*   曾与 dock-kubricks-Jenkins-AWS 合作；
*   等等。

好吧，来吧！

提交此空缺简历的候选人至少是具备空缺广告中介绍的“*框架”经验的开发人员。大概是和*后端*和*前端*一起演出，另外还有*义务*的事务。也就是说，今天的背景是*【full stack】*。*

我要暂停一下，引用一些我已经通过的测试。我想冷静地分析一下对于一个候选人来说什么才是真正适合的情况。因此，我们将评估以下主题:

*   技术对话；
*   具有和不具有研究访问权限的技术测试；
*   Hackerrank(黑客等级)：
*   测试家庭办公室(功能在规格范围内提供)。

## **funciona？**

基于候选人简历的技术对话是了解其知识水平的绝佳选择。在简历中介绍自己技术的考生可以通过应用情景和案例，在开发日处理的*框架内，对重要问题进行交谈、分析和回答。我认为这是非常有效的，根据采访者的不同，这可能比书面证据复杂得多。*

 *意见:我觉得相当方便！！

## **技术测试，没有访问研究的权限，工作正常吗？**

谁还记得日期转换 API，我和其他人在记事本++中测试，他们是否需要执行一些需要 API 知识的功能，而不需要访问搜索？

我不明白这种证据。它实际上没有评估潜力，而且排除了一个很好的潜在候选人。当前，不搜索不是一个选项，因为的演变是恒定的。

意见:我认为是个错误！在我看来，这似乎是市场上存在的技术领导者的过剩。

请原谅我的强烈意见，读者们。但是有很多这样的专业人士。更重要的是，对于一个竞争如此激烈的行业来说，专业人士的遗产带给我们的理想不是刺激竞争，而是协作。

## **黑客排名**

我真的很惊讶。对于后端、前端等的空缺说明，我实在不懂，有这种程度的技术证明。如果这是一个数据科学家的职位，没关系……但一个 DEV 的职位，在框架内有要求，大部分都是业务活动？

这个应试者，不得不通过重逻辑测试，包括大多数高级算法，我不确定这是否真的很重要。

一个德夫要冷静，不要绝望，保持冷静。我相信这是受访者希望通过这些测试来分析的，否则，在软件开发市场中我看不到任何适用性。

意见:没必要！！

## **普罗瓦家庭办公室**

我最喜欢的。不是因为你的舒适和时间，而是因为你只是评估了几个重要的场景。

通常是功能、cru、搜索、访问等。这很有趣，假设您要交货:代码质量、运行时间、交货内部版本(是否易于执行？)，完成功能。最后，在执行此类方法时，有 1001 个有趣的分析

## 但毕竟**技术测试真的很重要**？

这太重要了。是面试者和应聘者之间的知识评估和经验交流。但是，有一些变量在技术面试中是无法衡量的。当被问及他们的素质和缺点时，所有候选人都说:∞主动，我有帮助我公司实现目标的知识。还有我的缺点:我很期待！！什么时候开始

没错，但戴伍的日子会告诉你他到底是谁。我见过很多优秀的人--你知道--无法适应和融入开发团队。有些团队是封闭的，没有沟通，他们更关心自己的个人工作，而不是团队。

很难找到一个 DEV，尽管他不知道所有的事情，但他会寻求知识，帮助你的团队，无论是，他会去学习新的东西，锁定并知道哪里错误地告诉你的团队:我经历了这个错误，如果有人需要类似的东西，这里有记录！!”

我的观点:技术方面太重要了。

但是长寿与技术方面无关，因为开发人员会学习公司是如何逐渐发展业务的。肯定会有自己的特色。因此，其寿命取决于候选人的行为和主动性。

我们不需要进行高级技术测试只要确定如果你有问题 DEV 会知道怎么修理房子。

我忘了！
