# 为什么我决定选择鹈鹕

> 原文:[https://dev.to/trevdev/why-i-decided-to-go-with-pelican-5i2](https://dev.to/trevdev/why-i-decided-to-go-with-pelican-5i2)

当我探索新事物时，我往往会变得兴奋，我已经养成了让这个网站成为新的痴迷焦点的习惯。

当我第一次写这个网站时，我学到了全新的 HTML/CSS/JavaScript 技能。然后我学了 Python，用 [Flask](http://flask.pocoo.org/) 重新写了这个网站，让它更有活力。几个月后，我开始学习 reactive JavaScript 前端，并最终将我的作品集改编成一个单页 React 应用程序。在那一点上，我很高兴，但我觉得我仍然缺少一些东西。

我想要一个博客，不想在 JSX 写，也不想做一些无聊的标记渲染。搜索引擎优化不是很符合标准，我想要的东西，将在旧的/不太容易访问的浏览器渲染得很好。我也想要一些低维护性的东西，觉得 React 对于作品集或博客来说可能有些矫枉过正。不用说，很容易被新事物冲昏头脑。

> 静态站点生成器快速、简单且安全

我在推特和乳齿象上读到静态站点生成器时听说过它们。我认为静态站点通常都很棒，使用静态站点生成器有很多原因:

*   他们提供内容管理(像 WordPress 一样)，但是没有任何膨胀。
*   它们使得以简单的文档格式(如 Markdown)编写页面和帖子变得容易。
*   他们生成*非常安全的*网站。只是静态内容。除了 web 服务器本身之外，没有数据库或后端可以利用。

在阅读了一些关于各种静态站点生成器的文章后，我决定使用 *Pelican* ，原因如下。Pelican 非常成熟，用 Python 编写，高度可定制/可配置，包含许多特性。例如，它根据类别、日期、作者和标签生成页面。你也可以利用定制的 post/page 元数据，并将其传递给 jinja powered 模板，这些模板非常清晰，易于推理。要部署它们，您只需要通过 FTP 或 shell 访问一个共享的主机提供者。你也可以使用类似于 *Netlify* 或 *Github Pages* 的东西，为*免费*部署一个静态网站。

为了启动，Pelican 有几十个插件。因为我喜欢 Python，所以对我来说，为我的 VueJS 驱动的[评论系统](https://github.com/trev-dev/pcs-serializer)这样的小众环境编写自己的插件并不那么具有挑战性。

鹈鹕也真的[有据可查](https://docs.getpelican.com/en/stable/)。这在弄清楚如何正确编写新主题和创建特殊页面模板(如我的主页和我的投资组合类别页面)时非常方便。

我能想到的唯一缺点是 Python 比一些较新的静态站点生成器如 Hugo 要慢。也就是说，我仍然能够在半秒钟之内渲染这个网站，对我来说已经足够快了。

我对我和 Pelican 一起创造的东西非常满意，并且很高兴分享一些我在这个过程中学到的技巧和诀窍。我不认为我会很快更新我的网站。
