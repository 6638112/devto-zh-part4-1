# 如果它运行在服务器上，为什么他们称之为“无服务器”呢？

> 原文:[https://dev . to/gsto/why-do-they-call-it-server less-if-it-runs-on-a-server-5gca](https://dev.to/gsto/why-do-they-call-it-serverless-if-it-runs-on-a-server-5gca)

无服务器计算仍然依赖于运行在服务器上的代码，那么为什么每个人都决定用这个毫无意义的名字呢？理解了内涵就有意义了。让我们仔细看看这个术语，这样我们就能理解它了。

> 如果它运行在服务器上，为什么他们称之为“无服务器”呢？最早出现在[的格伦·斯托瓦尔——成长工程师](https://glennstovall.com)。

## [](#what-the-heck-is-serverless)到底什么是无服务器？

如果你对这个术语还不太熟悉，不会被它弄糊涂，现在让我来解决这个问题:

无服务器计算是一种解决方案，允许您根据需要在云中运行代码。AWS 的 Lambda 函数最流行的形式之一。你写一个函数，当它被调用时，它在某个地方运行，然后你得到一个结果。不需要资源调配。但是代码“实际上”在哪里运行呢？

## 无服务器就像拼车

[![Rideshare cars](img/94ac0ad65c1d3f4a9b1c9565ad01184d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UtlcQnWY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/onk372iedeiofqawwypo.jpg)

如果你住在一个足够大的都会区，你可能不需要开车到处跑。随着更多地方的步行距离和公共交通，你可以得到你想去的任何地方。这是一种奢侈，我希望我有，但相反，我在离体面的地方两英里远的地方工作。然而，有时你需要在特定的时间到达某个地方，或者在公共交通所能提供的时间和地点范围之外。

所以你打电话给优步。

你可以说你过着“无车”的生活。你不需要拥有甚至租用自己的车。拼车应用创造了汽车即服务。即使你没有车，你的旅行仍然需要使用汽车。你根据需求交易打车服务。它为你节省了汽油、保险和汽车贷款。

## [](#serverless-is-like-uber-for-the-cloud)无服务器就像优步对于云

Lambda 函数可以在服务器上运行，但是您不必担心自己租用、拥有或维护这些服务器。在这方面，它与 EC2 等其他亚马逊网络服务不同。像 rideshare 一样，它可以削减你的开销，为你省钱。
