# 为什么开发人员喜欢讨厌其他编程语言

> 原文:[https://dev . to/flipped coding/why-developers-love-to-hate-other-programming-languages-CCI](https://dev.to/flippedcoding/why-developers-love-to-hate-other-programming-languages-cci)

不管你知道什么样的技术栈，总有某个地方的开发者认为他们使用的栈更好。Java vs Python。角度 vs 反应。你可以找到关于为什么一个是垃圾而另一个不是的激烈争论。有时候，有些开发人员如此强烈地讨厌特定的语言是很奇怪的。

每种语言和框架都有其优点和缺点，但是它们真的比其他语言和框架优越吗？开发者讨厌其他编程语言有几个原因，而且都很有趣。

## 他们只是更喜欢自己的语言

当你的大部分经历都围绕着一两个技术栈的时候，很容易就会偏向其他人。新技术栈的第一个困难迹象意味着它很糟糕。如果它的工作方式与您通常的技术堆栈完全不同，并且您不太了解语法，也没有关系。这是堆栈的错，它不像你想的那样工作。我想我们都曾经这样做过。

习惯 Python 中的缩进与 C#中的语法有很大不同。或者说，在 Angular 中制作组件与在 React 中制作组件略有不同。当你比其他人更了解一种语言时，你会更喜欢你的母语。当你不得不使用一个你不喜欢的框架时，这让你有点讨厌它。这只是开发人员相对容易被放大的烦恼之一。

## 这是科技文化的一部分

web 开发社区有自己的小怪癖。讨厌其他编程语言恰好是其中之一。我们喜欢我们关于科技事物及其背后所有原因的演讲台。因为你以前发现的一些奇怪的东西，尝试记下整个语言是很有趣的。你听说过有人因为用 JVM 做事情而感到紧张吗？

大多数开发人员并不真的讨厌其他语言。这几乎就像我们玩的游戏。试图让某人相信我们的语言比他们的更适合某个应用程序是很有趣的。令人难以置信的是，开发人员对技术栈的历史以及它们如何工作的细节了解得如此之多。没有什么比争论哪种语言更好的了。

## 它帮助我们保持学习

试图证明你的筹码的优势意味着你真的必须知道你在说什么。这促使您跟上技术领域的最新发展。然后，到了痛击其他语言的时候，你就有足够的弹药了。确保你知道你在抨击什么将有助于你保持当前的技能。

另一方面，听开发者谈论他们技术栈的细节会让你接触到不同的做事方式。您可能不喜欢另一种语言，但是您看到了一种可以在您的堆栈中实现更好的东西的方法。听人们谈论他们的筹码有多多好，会让你学到一些你只能从经验中学到的细节。

开发人员并不真的讨厌其他编程语言、框架或技术栈，但他们确实喜欢这样争论。还有，有些人不得不使用 Objective C，所以他们可能真的讨厌那种语言。不管我们使用哪种不同的堆栈，它们都各有利弊，具体取决于您的应用程序。

还是会？你认为真的有一种语言可以用来做任何事情并且做得很好吗？我真的很好奇，因为似乎所有最常用的语言都以非常相似的方式做事。请在评论中告诉我你的想法。
