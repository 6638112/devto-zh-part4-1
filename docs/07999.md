# 如何成为一名优秀的软件工程师导师

> 原文：<https://dev.to/flippedcoding/how-to-be-a-good-software-engineer-mentor-4b>

指导是帮助初级开发人员提高技能的最佳方式之一。作为一个中级或高级开发人员，你想成为什么样的导师取决于你自己。作为导师，最重要的是要记住，这不是关于你的。这不是关于你的经验或者你能多么雄辩地写代码。

作为导师，你的目标应该是帮助初级开发人员理解代码以外的东西。他们可以读到这些。你需要向他们展示你只能从经验中获得的其他东西。这里有几件事你可以做，成为一个好的网站开发人员导师。

## 解释业务方及其决策

编写代码通常不是最难的部分。困难的部分可能是理解业务方想要什么。当你与客户或产品经理开会时，让你的初级开发人员列席或花几分钟时间解释发生了什么。他们需要知道他们的任务来自哪里，以及它们是如何被创建的。帮助他们理解业务方面，让他们在完成任务时有更多的背景知识。

还要确保当他们深入到他们的任务中时，您涵盖了业务逻辑。通过一个快速演示来解释该应用程序目前是如何工作的。这样他们就知道如何进入特定的屏幕，或者为什么我们要以特定的方式处理用户输入。对应用程序功能决策的简要概述将有助于初级开发人员学习如何从整体上理解应用程序，而不仅仅是代码。

## 在代码评审期间，谈论我们为什么以某种方式编写代码

无论开发人员的经验水平如何，一个好的代码评审都涵盖了最佳实践。特别是对于初级开发人员，您永远不会想跳过这一步。如果你看到他们做的一些工作，但写得很差，向他们介绍最佳实践，这样他们知道下次如何写更好的代码。还要向他们解释团队做出的一些代码决策。这有助于让他们了解应用程序是如何增长并达到当前状态的。

这是解码我们使用的行话的好时机。当你谈论最佳实践和代码时，定义你使用的术语。理解所有的术语需要一段时间，许多初级开发人员试图从上下文线索中理解意思。告诉他们什么是接口或者存储库是如何工作的，跳过所有困惑的表情和三心二意的“ok”。这会让他们对核心概念有更深刻的理解，而不是语言的具体内容。

## 抓住机会突出职业发展

多年来，作为一名开发人员，您可能担任过不同的角色。你知道做 DevOps 或者前端开发这样的事情需要什么。当你和你的初级开发人员一起工作时，突出这些事情。如果你发现你的初级开发人员学习 SQL 查询比其他任何东西都快，那么向他们介绍数据库管理，并告诉他们他们需要学习什么。向他们展示尽可能多的机会，因为这将帮助他们在职业生涯中导航。

当你向他们展示他们拥有的所有选择，并告诉他们如何获得这些机会时，从长远来看，你为他们的职业生涯建立了更成功的基础。他们可以开始考虑他们想专攻什么领域，以及如何着手去做。这给了他们不断尝试新事物的动力。随着他们尝试新事物，他们成为一个全面发展的开发者，这提高了他们所做的一切的质量。

## 在你告诉他们“正确的”方法之前，让他们完成他们的想法

当你接近一个 sprint 的末尾，并且你知道你需要尽快完成时，抵制住排斥初级开发人员的冲动。这是最好的教学时刻之一，因为它迫使他们快速和批判性地思考，并教会他们如何做出决定。即使这是一个你没有把握的难题，也让他们尝试一下。这有助于他们超越代码的表层，深入到应用程序的核心。

给他们一个机会，让他们告诉你他们在想什么，然后你再跳出来拯救世界。当你切断他们的联系时，会让他们觉得你不在乎他们要说什么或者他们一直在想什么。虽然这可能是真的，但如果每次事情变得困难时你都接手，这无助于他们的成长。让他们知道你在倾听他们，如果他们没有完全理解，向他们解释为什么以及你打算做什么。

## 当你和他们一起写代码时，谈谈你的过程

在你职业生涯的某个时刻，你曾坐在某人的办公桌前，看着他们编写代码。为了让这对于你的初级开发人员有用(而不是完全无聊),谈谈你的思考过程。看着你默默地写代码可能会有一点帮助，但是当你让他们看到你写的代码背后的思维过程时，你真的教会了他们一些东西。他们开始看到你是如何把这些片段连接起来的，以及你为什么要这样写代码。

一旦你这样做了几次，交换角色，让他们也这样做。如果你注意到他们没有正确使用继承这样的概念，让他们知道。通过倾听他们的思考过程，你可以帮助他们朝着正确的方向前进。让初级开发人员大声说话也能让他们更清楚自己在想什么，这样他们就不会盲目地写代码。

做一个好的导师也不一定要花很多时间。如果你能每天花 30 分钟做这些事情中的一件，你就已经在一周内以五种不同的方式帮助了你的孩子。当你这样做的时候，你会发现他们的技能在提高，信心在增强。我敢肯定，你记得发现多少你不知道作为一个初级开发。成为你希望了解的资源。
