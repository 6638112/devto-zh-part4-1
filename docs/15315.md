# 雇佣能容忍失败的人

> 原文:[https://dev . to/veebuv/hiring-failure-tolerant-personalities-a64](https://dev.to/veebuv/hiring-failure-tolerant-personalities-a64)

我很幸运成为几个强大的组织和创业公司的一员，更幸运的是，现在我可以经营自己的公司了。

像许多人一样，我也曾做出过非常糟糕的招聘决定，当我追溯这些决定时，会出现一些模式，将我的糟糕招聘与好招聘区分开来。当然，在任何情况下，糟糕的招聘的根本原因是招聘经理，因为他们选错了人。

这是我想谈的话题。

## [](#hiring-failure-tolerant-personalities)雇佣宽容失败的性格

这是我在公司、中小企业和创业生涯中雇佣的许多人身上发现的最明显的特点。在我的经历中，那些容忍失败并愿意在编码或方法上冒险的人是最成功的。

一位正在将公司规模从 60 人扩大到 80 人的朋友说了同样的话，“我们不需要只听命令的人，我们需要敢于尝试的人，他们很快就会失败，掸掉身上的灰尘，然后继续做更大的事情。”

这可以通过多种方式实现:
1)采用新的文件夹结构
2)尝试新的机器学习算法来更好地保护您的数据
3)引入在线读取的编码模式
4)尝试可能更快的新数据库，该数据库更新
5)实施新的 git 流，这可能会消除拉请求中的瓶颈，等等

我注意到容错型人格的另一个特点是，当上层管理允许时，他们愿意承认自己的错误并成长。我将自己归类为一个失败容忍型人格，并记得在我的第一家公司中挣扎，因为管理团队想要创新，但不理解创新的过程(即尝试-失败-重复)，直到事情发生。

这让我想到了下一个部分，如果你是一名招聘经理或者你正在寻找招聘对象，请留意那些能够容忍失败的人，因为他们将会创新和发展你的公司和工程实践。更重要的是，确保你给他们尝试和失败的舞台。

我们最近想在前端改善我们的服务器交互，并决定尝试 Apollo 的前端商店，但它对我们不起作用。因此，我的两个团队成员花了一周的时间来尝试和测试不同的方法，以确保新的模式能够满足我们的需求，这当然意味着一路上会有很多抱怨和失败，但我们最终提出了一个超级灵活的模式和一种机制，它确实为我们做了很好的缓存，同时保持了开发 API 的简单

这对作为开发者的你意味着什么(如果你是作为开发者阅读的话)？推动自己进入一种心态，在这种心态下，不断尝试、失败和不气馁都没关系。我的意思是，这实际上是数据科学的核心——尝试不同的算法、偏差因素和方差，直到某些东西最终匹配我们正在寻找的理想模式。

你可以采取可行的步骤来创造这种心态:

## [](#analyse)分析

评估当前的工作环境，找出你个人希望改进的地方。对可能的解决方案做一些研究，并努力实现它们。

## [](#iterate)迭代

在实现的所有阶段，不要指望实现第一步中的最终结果，而是在迭代的路径中发展您的解决方案。当实现一个新的 GQL 和 Apollo 服务时，我们不仅仅是拿出 REST 并把它放进去，我们还分阶段引入了实验，所以各方都很满意

## [](#abstract)摘要

从你的解决方案中抽象出你的情感。创业领域的一句俗话是“热爱问题，而不是解决方案”，所以用心去尝试你的方法，因为你会让你的大脑接触到新的方法来试验想法，而不仅仅是反复进入兔子洞

## [](#communicate)沟通

我的团队，至少是负责这项任务的两个开发人员，会更经常地和我聊天，让我了解他们在做什么。这确实锻炼了他们的学习，帮助他们进一步掌握了他们的解决方案的概念，在下一篇博客中会有更多关于这一点的内容——费曼技术
这也是你让你的高级管理团队知道没有代码是永久的，所以让我们不要太依赖于一个解决方案，因此让我们尝试新的方法来取得成功。

人们尊重能更好地解决问题的新方案，而不是因为它们只是为了更好而更好。

所以简而言之？招聘经理，寻找这种个性和工程师，实验-人们会真正欣赏你！相信我😀
