## HTML 标签告诉搜索引擎什么？

某些 HTML 标签告诉搜索引擎如何正确地阅读你的文档，并根据特定的关键字对其进行索引。换句话说，当他们抓取网页时，他们会用一些关键词对你的网页进行分类，当用户搜索这些关键词时，这些关键词就会出现。

其中一些标签也改善了访问者在这些搜索引擎中查看您的内容的方式。除此之外，社交媒体还使用内容标签来展示你的文章，例如👇🏽。

## [](#1-title-tag)1。标题标签

也许最重要的标签之一是标题。这被用作你的页面的标签，并定义了像 Google 和 Bing 这样的搜索引擎如何看到你的页面并在他们的搜索结果中显示它。

在大多数情况下，这个标签反映了文章/页面本身的标题。这并不是一个坏习惯，但是你会错过一些在像 Google 这样的搜索引擎上大赚一笔的机会。

接下来的问题是，它必须和关键字一样吗？答案是绝对不会。然而，最好确保你没有让读者感到困惑😉。但是这给了机会去扩展你的页面与搜索引擎的相关性。无论如何，标题应该是简短的，所以有不同但相关的标签使得它可以被更广泛的搜索到。

一个普通的标题标签如下所示:

```
<title>Your Title Text for Search Engines</title> 
```

听起来很简单，对吗？虽然看起来很简单，但以下是你写标题时必须考虑的几点:

*   仅使用一个标题标签
*   尝试创建一个独特的标题
*   不要使用你的标题关键字作为你的元标签
*   使其与内容相关
*   最好控制在 60 个字符以内
*   尽量不要使用停用词
*   最后显示品牌
*   让它具有可读性
*   如果相关，使用日期、数字和问题

## [](#3-header-tags)3。标题标签

你不应该低估标题标签。它们通常用于分解内容，以便于阅读。想象一下，在一个巨大的段落中阅读我的文章，而不是这些精心分割的片段。

据发现，大约%55 的用户最多会花 [15 秒浏览你的文本](https://blog.bufferapp.com/55-visitors-read-articles-15-seconds-less-focus-attention-not-clicks)。就 SEO 而言，大多数搜索引擎使用标题标签来帮助确定内容片段，并创建[特色丰富的片段](https://www.greengeeks.com/blog/2017/02/14/why-should-you-want-your-site-to-be-in-the-google-featured-snippet/)。

这些标签如下:

*   `<h1></h1>`–通常用于网页标题。
*   `<h2></h2>`–突出显示后面内容的主题。
*   `<h3></h3>`–反映与主题相关的要点。
*   `<h4></h4>`–支持来自`<h3>`的点。
*   `<h5></h5>`–不常使用，但非常适合`<h4>`的支撑点

这通常是图像标签最重要的部分，不仅仅是为了 SEO，也是为了可访问性。不幸的是，搜索引擎无法确定显示的是什么图像。这时`alt`标签就派上用场了。

例如:

```

## [](#7-robots-tag)7。机器人标签

有些文章对排名的影响可能不是最大的。有很多原因可以解释你为什么想要这样做，但是假设你有一篇质量差的文章，你想在它出现在任何搜索引擎结果之前得到一些用户的意见。

```

## [](#8-canonical-tags)8。规范标签

一个[规范标签](https://moz.com/learn/seo/canonicalization)对于组织你的内容和区分一个网页相对于一个重复网页的优先级是非常有用的。给你举个例子，我总是喜欢先在[我的网站](https://yashints.dev)上发布帖子，然后再发布到其他地方。

我这样做有几个原因。最重要的一点是，在像 [Dev.to](//dev.to) 这样的平台上，覆盖范围可能会更大。然而，我想让搜索引擎知道这个帖子不是一个单独的帖子，事实上它是首先在我的网站上发布的内容的副本。换句话说，这是一种告诉搜索引擎一个特定的 URL 代表了一个页面的主副本的方式。

您应该使用此标签的另一个原因是，搜索引擎可能会使用以下任何 URL 形式对您的网站进行爬网:

*   [http://www.yashints.dev](http://www.yashints.dev)
*   [https://www.yashints.dev](https://www.yashints.dev)
*   [http://yashints.dev](http://yashints.dev)
*   [http://yashints.dev/index.html](http://yashints.dev/index.html)
*   [http://yashints.dev/index.html?param=value](http://yashints.dev/index.html?param=value)

对一个人来说，所有这些 URL 都代表一个页面。然而，对于爬虫来说，这些 URL 中的每一个都是独一无二的`page`。即使在这个有限的例子中，我们也可以看到主页有五个副本。然而，实际上，这只是您可能遇到的变化的一小部分。

```
<link rel="canonical" href="http://yashints.dev/index.html?param=value"/> 
```

因此，通过在我的主页上放置上述标签，我可以告诉搜索引擎，这个 URL 代表的是同一个页面，而不是一个单独的页面。

## [](#9-responsive-site-meta-tags)9。响应网站元标签

我们先来看一个事实，中国超过 80%的流量来自移动设备。在澳大利亚，这个数字是%56。这些数字向我们展示了关注响应式设计的重要性。

出于同样的原因，让搜索引擎知道这个网站有响应式设计也很重要。如果用户通过移动设备进行搜索，这将确保页面是表面的。

```
<meta name="viewport" content="width=device-width, initial-scale=1"/> 
```

## [](#so-no-keyword-tags)所以没有关键词标签？

你可能想知道为什么我没有提到关键字标签。这是因为谷歌等大多数搜索引擎使用内容对页面进行分类。因此，拥有它们并不会增加多少价值，事实上，它们只会给你的页面增加额外的重量。
