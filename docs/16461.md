需要通过多种不同的设备访问相同的信息，这种需求在未来几年中只会不断增长。

物理存储限制，再加上越来越便宜的云计算，现在使我们能够将精力集中在虚拟帐户上，而虚拟帐户可用于任何能够访问互联网的设备。

每个设备的独特性是供应商的一个障碍，供应商需要为每个方案开发单独的接口。这一直在普及术语**。**

 **随着分层应用软件体系结构的普及，跨平台在[技术产业](////eusoudev.com.br/carreira-de-desenvolvedor/)变得可行。最受欢迎的体系结构是**MVC(model-view-controller)**，虽然自 1979 年以来就存在，但随着 **AJAX** 的出现而广泛使用，这是一种无需重新加载即可呈现网页的数据处理方法

MVC 基本上将应用程序分为三个不同的层，可以分别开发和开发:

*   ***模式*** 或模式，是数据和主要业务规则集中的地方。此层的发展通常对新的设计功能至关重要。它是应用的核心；
*   ***【view】***或远景，是最终用户查看应用程序并与特定目的交互的方式。每个设备都有不同的视图是非常常见的，并且不一定可以访问模型中的所有业务规则；
*   ***控制器*** 或控制器，是负责模型与视图之间数据传递的层。控制器接收来自视图的用户交互并发送数据以执行预期的操作。

## **例**

举一个跨平台应用的实际例子，让我们考虑一下您银行的应用程序和网站，以及一个假设的用途:支付过期帐户。

*   您可以从两个可用视图中选择一个视图:“应用程序”或“网站”。
*   无论选择哪一种，您都需要登录；
*   登录时，无论接口如何，访问都是完全相同的系统；
*   每个视图都是独立开发的，但控制器接收数据的方式相同。
*   按钮会在那边。以一种方式在应用中，以另一种方式在网站上；
*   登录时，您可以在手机上读取条形码。就在电脑上，“t0”copy-paste 是车轮上的一只手。在很多情况下，你最喜欢的*观点*往往是最能提供安慰的观点。第一种是数码票，第二种是印刷品。太好了！
*   无论*视图*和格式如何，机票号码都是一样的。条形码在界面内，点击/触摸□
*   此时， *view* 将其条形码发送给控制器，控制器将值标准化，然后发送给 model，model 将在数据库中检索该卡号；
*   model 回答控制器:我找到票了，但是已经过期了。以下是利息和罚款的计算；
*   新更新的控制器已接收信息，计算利息和罚款，并向视图报告；
*   (较旧的)网站还不知道这些计算是存在的。总结为给出错误消息，提示用户手动计算并填写罚款和利息。需要查询票，而手里拿着计算器，用户在计算中有风险；
*   应用程式(最新的)已准备好接收此额外资讯，并会自动计算利息，让使用者只需按一下即可完成程序。
*   密码验证后，该过程结束。

此示例用于显示每个层的生命周期。随着视图随着移动技术的发展而快速变化，业务规则将保持不变，并在较新版本的界面中稍作改进以改善用户体验。因此，系统不再需要进行大规模的重新设计，而是部署了越来越复杂的控制器。

随着[【虚拟现实】](////eusoudev.com.br/realidade-virtual-na-saude/)的普及和现实的增强，视图将越来越沉浸其中，而琐碎的功能可以在控制器中永久存在。买受人在客厅里几乎可以看到的选择沙发的体验会非常舒适和自信，但是模特会继续以同样的方式处理销售。

系统核心的发展将侧重于生产、物流和后期行动准备等其他业务关注点。每一项计划，都要求控制器的发展和各自的观点的发展。

虽然跨平台应用程序的最显着发展在视图中已显着，但一个日益增长的趋势是，在未来十年中，用户界面的最大发展是向控制器引入了“人工智能”。
