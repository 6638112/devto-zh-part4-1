# “无聊软件”宣言

> 原文:[https://dev.to/mevdschee/the-boring-software-manifesto-1bba](https://dev.to/mevdschee/the-boring-software-manifesto-1bba)

作为软件开发人员，我们已经厌倦了最新最伟大技术的传播者所做的虚假声明。我们将不再面对他们缺乏对计算机科学基础知识的理解，也不会为我们缺乏对他们大肆宣传和多变的技术的了解而辩护。行业现状迫使我们重视:

*   不仅仅是工作软件，
    *   还有精心制作的软件，
    *   专为流行和成熟的工具构建。
*   不仅仅是应对变化，
    *   而且不断增加价值，
    *   同时减少依赖性和复杂性。
*   不仅仅是个人和互动，
    *   也是一群专业人士，
    *   分享可验证声明的最佳实践。
*   不仅仅是客户协作，
    *   还有富有成效的伙伴关系，
    *   缩小我们构建的软件的范围。

也就是说，在追求“敏捷和技术”的过程中，我们发现“无聊的软件”是不可或缺的。

### [](#some-examples)一些例子

左边的东西是很好理解和证明的，右边的东西是炒作和波动的。

*   三层应用程序与微服务
*   关系数据库与 NoSQL
*   页面重载与单页应用程序

还有许多其他的例子，但这应该能让人理解这个想法。
