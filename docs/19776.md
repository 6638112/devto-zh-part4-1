# 你如何衡量代码质量/工程团队的表现？

> 原文:[https://dev . to/ms hel/how-do-you-measure-code-quality-engineering-team-performance-CPL](https://dev.to/mshel/how-do-you-measure-code-quality-engineering-team-performance-cpl)

我已经研究这个问题有一段时间了，下面是我想到的一些事情(不过南觉得他们都很对):

1.  测试代码覆盖率
2.  缺陷数量/特性数量(%)
3.  AVG 页面加载时间(假设我们谈论的是某种网站)
4.  圈复杂度(代码的散列表)应该很小
5.  基本复杂性(不必要的循环和 ifs 应该很小

所以问题是你们是怎么做到的？在什么时候(每两周/发布时/每半年/每天/每次提交/千载难逢的一次)
