# 开始使用 Bit 构建像乐高这样的 React 应用

> 原文:[https://dev . to/the wizardjs/start-using-bit-to-build-react-apps-like-Lego-15lg](https://dev.to/thewizardjs/start-using-bit-to-build-react-apps-like-lego-15lg)

您是否经常在多个项目中编写类似的组件？有时你会觉得你可以使用一个私有的 NPM 注册中心来维护所有的组件，以便在项目间共享。但是在敏捷开发团队中；打包、发布和维护多个组件是一项挑战。此外，一旦发布，你不能轻易直接从你当前的项目中做出微小的改变。

React 组件不像库，它们以更快的速度发展，开发人员为组件构建新的功能和扩展来满足应用程序的需求。

你不能像其他库和工具一样打包并在 NPM 上发布。要做一个小的改变，你不能设置一个孤立的工作空间&引入改变，而是你想在当前项目本身中摆弄组件。一旦你对自己的改变感到满意，你就会希望推动这些改变。Bit 正是让这个工作流程成为可能。将您的组件和它的源代码一起导入到您的项目中，*调整它，导出，弹出*，它的更新版本就可供团队的其他成员使用了。简单？

Bit 很强大&让 React 开发者的生活变得容易多了。它就像是 git 和 NPM 的融合，只是针对组件。Bit 允许真正的组件驱动开发，并在软件开发生命周期中像对待一等公民一样对待组件。

Bit 是 React 的一个很棒的**组件工作台**。React 组件的功能特性使 Bit 成为 React 的理想选择。这有助于节省将所有组件保存在单独的存储库中的开销。您可以轻松启动自己的组件库。

### [](#make-changes-to-your-component-on-the-spot)当场修改你的组件

Bit 使您能够在当前项目中对组件进行修改。您不需要退出您的项目，您可以直接导入源代码并当场进行更改，然后试验您的组件。您甚至可以在发布之前测试您的更改。Bit 让这个强大的工作流成为可能。

### [](#search-for-components-on-bit)在 Bit 上搜索组件

Bit 是为组件而不是库构建的，因此可以更好地发现组件；您可以浏览您的团队开发的组件集合。
搜索&预览组件&在你的项目中重用合适的组件。Bit 成为你的迷你组件市场，供你的团队挑选。

### [](#truly-collaborative)真正的协同

这个分布式工作流模型允许开发人员将组件导入到他们的项目中。Bit 允许你和你的团队真正毫不费力地共享代码。因此，它允许您的团队协作并构建有意义的共享组件，而不是在多项目环境中为组件编写类似的逻辑。

### [](#isolate-your-component)隔离你的组件

Bit 让你能够分配你的组件编译器，本质上它们是环境。这是一种特殊的扩展，用来构建组件。您也可以编写定制的编译器来满足自己的需求。此外，Bit 使用测试人员针对您的组件运行测试用例，以确保您的更改不会中断。

这种体验将开发工作流的灵活性和速度提高到了一个新的水平。Bit 使得在应用程序、团队成员之间共享组件变得更加简单，并使用它们更快地构建新的东西。
