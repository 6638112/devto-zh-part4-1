# 如何发现你的后端是否正在触发一个异常？

> 原文：<https://dev.to/inspector/do-you-know-if-your-application-is-firing-an-exception-now-37b2>

大家好，我是 Valerio，来自意大利的 Laravel 认证开发人员。

我几乎每天都交付新的代码变更，但是等待用户报告错误或瓶颈，我实际上是将 bug 发现外包给用户。用户不会报告错误，他们最终会停止使用我们的应用。

这就是为什么对我们的业务健康至关重要的应用程序无法进行手动监控的原因。如今，大多数开发人员使用的手动监控过程成本高、耗时，并且存在巨大的误差。

## 手动应用监控有哪些风险？

说到期望值，客户主要是在你的应用中寻找两个关键属性:可用性和性能。错过这两者中的任何一个都会在几个方面对你的经济利益产生负面影响。

### 生产力减缓

弄清楚某件事情不工作是容易的，但是理解它为什么不工作…嗯…

如果不实时监控您的应用，当客户向您报告出现问题时，您将被迫放弃正在进行的工作，开始尝试重现该场景，然后在您自己的工具集中重新捕获和分析日志。

获得正在发生的事情的真实情况可能需要几个小时，或者根据我的经验，甚至几天。

与此同时，您的工作效率降低，您的客户在等待消息。

## 降低客户的长期期望

从长远来看，糟糕的数字体验会影响客户对你的服务是否适合他们的看法。

经历过应用程序冻结、崩溃、太慢或涉及太复杂功能的用户不会考虑在未来使用该应用程序，如果被问到，这些用户中的大多数会与其他同行分享他们的负面经历，从而损害你的声誉。

我的很多客户其实都是其他科技公司。我努力让自己被视为值得信赖的合作伙伴。但是如果工作变得缓慢，我不断地制造延迟而不是解决方案，他们很容易转向其他人。

## 自主监控工具给你什么好处？

应用程序的自动监控可以极大地改善您的工作流程，并最终改善您的业务。

通过控制您的应用程序，从第一天起您就可以获得三个直接的好处。

### 领先一步

实时提醒意味着您可以在用户打电话之前几个小时就知道出了问题。

一个真正的开发者工具应该直观地表现你的代码执行流程。这使您能够在用户意识到之前一眼就识别出错误。

这一功能可以大大减少客户流失。

### 更高的开发效率

解决生产问题可能非常困难和耗时。

将性能测量和监控委托给自治工具有助于开发人员将更多的时间花在代码改进上，而不是花在应用程序调试上。

此外，如果出现问题，他们会立即收到警报，这样他们晚上就能睡得更好。

时间就是金钱，找到有技能的人一点都不容易。重要的是让他们继续致力于创新新产品，从而发展你的业务。

让您的开发人员更加高效的工具是非常有价值的。

### 更好的协作

统一的见解有助于团队对应用程序错误和异常进行故障排除，使后续决策清晰并基于真实的测量。

在非统一环境中，每个开发人员都需要重现错误，重新捕获日志，并在他们自己的工具集中重新分析它们。

### 减少对昂贵工具的依赖

应用程序监控非常有助于发现改进的机会。这里的 SQL 查询调整，那里的一些代码重构，你也许可以通过一些优化来降低你的托管成本。

自动化监控工具可以帮助您了解您的应用程序如何使用外部服务、数据库等等。围绕这些依赖关系进行一点性能调优可以让您缩减应用程序资源。

## 结论

在当今竞争激烈的市场中，对基本业务应用程序的手动监控不再有效。

通过实施自动化监控解决方案，您企业的应用程序将持续关注它们，一旦出现任何问题，您就会收到警报，甚至在您的用户意识到有问题之前。

您将节省时间和金钱，让开发人员更有效率地使用他们的时间，并有机会通过缩减资源来削减成本。
