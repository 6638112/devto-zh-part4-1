# 在决定下一步学什么时，你面临的最大挑战是什么？

> 原文:[https://dev . to/kball/what-s-your ' s-maximum-challenge-decision-what-the-study-next-5168](https://dev.to/kball/what-s-your-biggest-challenge-deciding-what-to-learn-next-5168)

在最近的一期 JSParty 节目中，出现了“作为 web 开发人员，人们面临的最大挑战是什么”的话题。

特邀嘉宾 [Chris Ferdinandi](https://twitter.com/ChrisFerdinandi) 强调了他得到的第一个问题:

> 有一种回应比其他任何回应都来得多。这至少是我得到的一半回答…而且是“我不知道从哪里开始”的某种变体。我被跟上所有事情的需要淹没了”或者“我不知道首先选择哪个框架。”

最近，我向 [Friday Frontend](https://zendev.com/friday-frontend.html) 订阅者发送了一份关于同一话题的调查，得到了类似的结果:

> 要学的东西太多，时间不够，资源太多。很难集中精力，保持领先，甚至不知道所有的部分是如何组合在一起的。

我认为这是我们行业面临的一个巨大挑战，我想知道我们是否可以一起集思广益，探讨这个问题背后的原因。下面是提示:

当你决定下一步该学什么时，你遇到的最大挑战、挫折或问题是什么？

我先说:我真的很难知道一个新的大肆宣传的框架或工具什么时候“足够成熟”,以至于学习它不会是一项因改变 API 和糟糕的文档而沮丧的练习。

你呢？

当你决定下一步该学什么时，你遇到的最大挑战、挫折或问题是什么？
