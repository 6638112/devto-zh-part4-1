# 面向外行的 Linux

> 原文:[https://dev.to/kohloth/linux-for-the-layman-194e](https://dev.to/kohloth/linux-for-the-layman-194e)

*最初发表于 2019 年 4 月 30 日[socket-two.com](http://www.socket-two.com/main/resource/windows-vs-linux)。*

在过去的十年里，描绘笔记本电脑和台式电脑操作系统市场份额的图表看起来大同小异。无论你看哪里，你都会看到 windows 一直以较大的优势占据主导地位，占据了大约 90%的机器。MacOS 一直是远远落后的第二名，占 7% - 8%，馅饼的剩余部分——非常薄的一块——一直是胖得出奇的小企鹅 Linux 的天下。

Windows 为什么这么受欢迎？是因为更好吗？还是仅仅因为这是人们习惯的？

也许这个问题的答案可以用另一个问题的形式来解释:在信息技术的背景下，全球人口中有百分之多少的人害怕陌生的前景，并会不遗余力地避免它？

几年前的夏天，黄昏过后，我和几个朋友坐在一个花园里。chiminea 燃烧着温暖的光芒，鸟儿们刚刚安顿下来。这样的场景可能会获得哲学思想的倾向，而我热爱 Linux 的那部分无法抑制自己试图改变信仰的小小尝试。(对一些人来说不幸的是，Linux 转换者经常怀有这种冲动)。

我解释了操作系统的可定制性如何让我在工作时更加高效和舒适。特别是，我描述了命令行作为接口的表现力和直接性带来的便利，以及命令行在可重复和自动化 bash 脚本中的适用性带来的强大功能。我还谈到了如何设置操作系统级的可配置键盘快捷键，以便非常容易地打开程序并在程序之间切换:当我将 Vim、Terminator 和 Chrome 作为一个分散的开发套件一起使用时，这大大加快了我的工作流程。

“这一切都很好……”我的一个好朋友开始说道，显然已经领会了我描述的要点。“但是如果你不是电脑程序员，如果你只是用电脑做日常工作...文字处理、网上冲浪、播放音乐、看电影...Windows 没问题。”

我皱着眉头点点头，承认这是真的。他关于 Windows 非常适合此类任务的观点在技术上是正确的。但是我突然想到了一件非常有趣的事情。虽然 Windows 对于大多数人的休闲计算需求来说很好，甚至在这个水平上，Linux 可以说是更好的——尽管个人偏好比比皆是。一个可访问的 Linux 发行版，如 Ubuntu，将实际上提供基本的操作系统功能，所有用户都依赖于它的可靠性、效率，最重要的是，它的用户友好性使 Windows 看起来不寻常、顽固。

在讨论之后，我也意识到，相对来说，习惯现代 Linux 是非常容易的。从任何版本的 Windows 过渡到 Ubuntu Linux 或 Linux Mint 所涉及的学习曲线，并不比从 Windows XP 过渡到 windows 7 或从 Windows 7 过渡到 Windows 10 更痛苦，因为操作系统只用于基本任务。也就是说，一点也不痛苦。在这种情况下，Linux 的不受欢迎，以及它对所有用户的出色适用性和它的免费价格标签可以被认为是自相矛盾的。

在继续之前，我应该澄清一下，我并不是说 Linux 从根本上，或者客观地说，就其本身而言，一定是比 Windows 更好的工具。然而，我认为大量的计算机用户很可能会感到惊喜，如果他们努力进行这种转换的话，而且这种努力是微不足道的。

同样，客观地说，我也不会说橡皮锤比钢羊角锤更好。即使考虑到工人必须在不损坏大型软木部件的情况下将它们锤入到位的特定场景，我也不会声称橡胶锤可以说是客观上更好的——工人可能拥有羊角锤的出色技术，他们可能会觉得这是幸运的。各种各样的边缘事物可以使相反的工具更有利于生产力。然而，考虑到橡胶槌将力更均匀地分布在更大的表面积上这一简单事实，我将提请注意一个明显、显著且广泛适用的优点的存在。

我把这个理论告诉了我的朋友，引用了用户界面、对更新的放松态度、增加的稳定性和软件安装的简易性，得到了和我自己相似的回应。他点点头，微微皱起了眉头。没有口头回应，但信息是明确的。

“那可能是真的...可能是吧...但我并不真的倾向于采取众所周知的信仰的飞跃。”

因此，作为一名 Linux 爱好者，我将尝试在这里为采用 Linux 提供一个更有说服力的理由。然而，在此之前，我想重申，这不是一个学术的、基于证据的讨论，也不是声称 Linux 在根本上是优越的。然而，这是我希望读者考虑的一些观点的集合，以支持这样的理论，即在执行常见的、偶然的任务时，Linux 在许多方面比 Windows 更方便、更友好。正是出于这个原因，在这些参数范围内，我提出我的论点。

# [](#linux-for-the-layman)Linux 对于外行来说

## [](#the-graphical-interface)图形界面

由于所考虑的用户不一定精通技术，并且可能来自 Windows 环境，因此假设图形用户界面将是他们与机器交互的主要方法是合理的。因此，良好的图形用户界面对他们畅通无阻地使用系统是很重要的。它是将所有计算机窗口应用程序粘合在一起的粘合剂，也是用户最能敏锐地意识到的操作系统功能。它应该是直观的，理想的吸引力，它应该很好地实现最有用的功能。在这里，可定制性是一个值得关注的问题，但是只有当定制很重要并且对临时任务非常有利的时候。当考虑用户界面的组件时，对于一个普通用户来说，最重要的部分可能是程序启动器和停靠栏(也称为“开始栏”)、窗口管理系统和默认的文件浏览器程序。

Windows 开始菜单和 dock 可能是它最标志性的功能。它最早的化身背后的概念很简单。用户可以点击最底部最左边的按钮来查看已安装程序的列表，并点击其中一个程序来启动它。当一个具有图形用户界面的程序打开时，一个代表它的小框会被添加到 dock 中，当它关闭时，这个小框会消失。

Windows XP 发布后,“开始”菜单弹出窗口展示了比其 Windows 95 / 98 前身更多的一些功能，现在还包括一个常用程序、实用程序和文件系统位置的列表。不难想象，对于大多数用户来说，这些都是方便的附加功能。特别是，到控制面板的清晰链接可能有助于在临时用户需要时将他们引导到系统配置。

然而，Windows 8 对开始菜单进行了戏剧性的改造，小弹出窗口被一个全屏的快捷方式、实用程序和文件系统位置列表所取代。此外，还添加了引人注目的小工具，如天气预报、新闻报道，甚至一些几乎不加掩饰的产品促销。Windows 8.1 和 Windows 10 注意到了用户群表达的愤怒反对，并恢复了用户熟悉的弹出窗口。然而，引人注目的小部件仍然是菜单的一部分。Windows 10 仍然展示了一些不寻常的图形界面行为，这些行为是 Windows 8 中出现的“触摸屏设备优先”设计理念的结果。按钮仍然是大而方的，在小空间里显示相对大量信息的倾向依然存在。图形组件仍然从屏幕的侧面和底部滑入和滑出，以响应(通常是无意的)用户手势。

在我看来，Windows XP 的图形用户界面在功能和简单性之间达到了完美的平衡，此后变得有点失败。

有趣的是，与 Windows 10 开始栏相比，Linux Mint 启动栏与 Windows XP 开始栏有更多的共同点。因此，更喜欢 Windows XP 开始栏或很难适应 Windows 10 开始栏的 Windows 用户应该可以使用 Linux Mint 开始栏。吸引普通用户和有经验用户的是它简单而整洁。显而易见的是，有一个可以滚动或搜索的分类节目的详尽列表。关机/待机/锁屏控制很容易找到。还有一个到控制面板的链接，虽然没有标注，但这是一个相关的附加内容。打开的程序在 dock 中以块的形式出现，这也是 Windows 用户喜欢的。

Windows 文件管理器充其量是可生存的。几乎所有的基本功能，临时用户将从中受益，包括在内。支持列表视图、缩略图视图、排序和重命名。用户可以在地址栏中粘贴文件系统的位置。还有一个侧边栏，可以用来显示一个文件树(对于重新组织文件非常重要)，或者一个快捷方式列表。

尽管存在这些基本特性，但缺少一个与文件重组相关的关键特性。在程序的一个窗口实例中，不可能通过选项卡或拆分窗格的方式打开多个文件系统位置。这相当于重组文件时的不便。例如，一个同时浏览音乐和整理照片的用户需要打开两个(或更多)独立的窗口。如果他们想最小化、最大化、移动或关闭这些窗口，他们将不得不一次一个窗口地这样做。当一天要做好几次的时候，摆弄文件浏览器窗口会令人沮丧。

Windows 文件管理器的美学大部分是无害的，会取悦那些喜欢白色和蓝色临床阴影的人。然而，在文件浏览器的顶部，按钮、控件和图标的列表创建了一个稍微有点奇怪的集合，非常混乱，尽管被贴上了标签，但感觉不直观。

Windows 用户会很高兴地得知，绝大多数 Linux 发行版都有一个图形文件管理器，并且默认的 Linux Mint 文件浏览器 Nemo 与 Windows 文件管理器非常相似。

不同的是，它的美学外观可以完全随着各种皮肤而改变，并且也更少杂乱。左上角清晰的图标按钮将用户带到目录树的后面、前面或更上面，左下角的按钮在书签位置列表和文件树之间切换。它还可以让用户创建标签和窗格，如上面的截图所示，对于那些有此倾向的人来说，可以轻松地重组照片、音乐、文档或 javascript 文件。

无论用户喜欢哪种文件管理器，每个系统都不会反对安装第三方文件管理器。然而，Windows 中的文件管理器更加根深蒂固，这将导致它在调用各种异常对话框和快捷方式时更容易窃取优先权。

最后要考虑的是桌面环境本身。在操作系统中，桌面环境定义了系统的整体外观——工具栏、按钮、下拉菜单、图标、对话框，在某些情况下，还定义了整个程序的整体外观(尤其是本地程序，如文件管理器)。桌面环境还定义了组织程序窗口和用户将拥有的一般工作空间的方式。

windows 桌面环境本身绝对不差。像文件管理器一样，它提供了一套相当全面的功能。然而，由于它比文件管理器更根深蒂固地植入到系统中，而且它几乎没有定制选项，用户被它卡住了——如果你甚至有一些小小的疑虑，这可能是一个问题。一周使用几次，一些小小的不安会造成很大的不便。在 2019 年，对于普通上班族、摄影师或电影观众来说，这不是一个特别密集的使用频率。

相比之下，Linux 系统中的桌面环境是完全可定制的，而且是完全可替换的。

如果愿意，Linux 用户可以安装一个由深色背景和浅色文本组成的深色主题。这有助于减少眼睛疲劳和电池消耗——这并不是程序员独有的意图。深色主题也可以更时尚，如果用户有兴趣用漂亮的绿色、红色、深色、浅色、丝质或金属色皮肤来惊艳他们的朋友，他们可以很容易地做到这一点。安装 Compiz fusion effects 甚至可以赋予他们像法师一样的能力，将火吸引到屏幕上，或将火变成水，在视频会议显示器或桌面背景上产生涟漪。另一种利用桌面环境模块化的更实用的方法是安装一个平铺窗口管理器，它可以根据各种可配置的标准自动组织窗口，甚至在你打开它们的时候。诚然，这是只有技术无畏的用户才会想要追求的东西。

许多 Linux 桌面环境支持多工作区已经有一段时间了，这为用户提供了在多个虚拟桌面阵列上分布窗口的能力。这是多任务处理时手动分组相关程序窗口的一种方式。Windows 确实有这样的功能，但只是最近才推出。

## [](#software)软件

一般来说，Windows 只有一种公认的常用软件安装方法。用户将下载某种形式的可执行文件(通常是 exe 或 msi)，然后执行它以提示操作系统开始解释和吸收它。这是一个不错的系统，尤其是当用户在执行这些步骤之前就知道他们想要安装什么时，这是可以原谅的。然而，打开一个浏览器，浏览一系列搜索结果，然后是一系列可用的下载是一件很麻烦的事情。同样重要的是，对于临时用户，甚至对于不知情的用户来说，总是存在无意中下载实际上会安装病毒的可执行文件的风险。

Linux 以其手动安装程序而闻名，包括运行一个以关键字`make`和`install`为特征的文本命令。然而，大多数版本的 Linux 也支持“浏览、下载和安装”类型的系统，类似于上述的 Windows 等价物。对于 Debian 和 Ubuntu 系统,“exe”文件的扩展名是“deb ”,对于一个普通用户来说，它的工作方式是一样的:安装只是双击的事情。

然而，Linux 确实有更好的安装软件的方法。一个经过审查、推荐的免费软件目录是 Linux 的王牌。Windows 没有一个相对成熟、流行和普及的系统。当用户安装特定程序时，以及当用户寻找将实现特定目的的任何程序时，这是非常有利的。

例如，如果用户想看硬盘上的视频，但格式模糊，他们可以选择安装 VLC 视频播放器。为此，他们可以按一个按钮打开终端，然后键入`sudo apt-get install vlc`。几秒钟后，VLC 播放器将被安装。如果系统需要其他软件让 VLC 工作，这些软件会被同时安装——一切都会自动处理。毫无疑问，这是一个普通用户会喜欢的东西。寻找一个丢失的运行时或 DLL 文件并不有趣。

另一方面，如果用户正在寻找一个图形编辑程序，但他们头脑中没有一个特定的程序，他们可以简单地通过发出命令`sudo apt-cache search paint`看到一个可用于自动安装的程序列表，然后选择随后用`apt-get install [x]`命令安装一个程序，其中`[x]`是程序的名称。

这些信息有点不恰当，因为这些例子描述的是“随意的”用户通过控制台与操作系统进行交互。但是，请考虑这些命令很简单。还要考虑到，用户可能会使用各种优秀的图形程序来代表他们利用这些命令。这些程序同样会搜索在线软件目录并从中安装东西，而不需要键入控制台命令。新立得软件包管理器(如下所示)就是这样一个工具，它作为标准安装在一些系统上。

同样值得承认的是，Windows 有一种打包无用程序的趋势——膨胀软件。这是一个小小的打击，因为经常是电脑制造商用这种暴行来装饰操作系统。然而，对许多用户来说，这是一种权宜之计。这对于没有经验的人来说可能很麻烦，因为这样的人群很少能够区分占用资源的麻疹和重要的操作系统实用程序。出于这个原因，许多用户甚至不会尝试清理它们，并且会被弹出的闪屏、注册提醒和不需要的辅助程序启动器永久纠缠。

## [](#heckling)起哄

一个刚从 Windows 转换到 Linux 的用户可能会因为明显没有诘问而变得眉开眼笑。

在接触 Linux 之前，windows 用户通常会认为重启计算机以响应软件安装、卸载或更新是所有计算机的惯例。然而，Linux 的体系结构是这样的，需要重启的情况非常少，这对所有人都有好处。

另一种所有 Windows 用户都遭受的更普遍、更不方便的诘问是强制更新。

有时，Windows 用户会结束他们在电脑上的工作，并告诉电脑关机。在准备继续他们的业务时，他们会注意到(如果他们幸运的话)计算机实际上并没有关机，而是会在接下来的五到十五分钟内安装更新。这种情况的影响可能从微小的不便到重大的个人干扰不等。

例如，在辛苦工作一天后关闭办公室的电脑，员工可能会突然意识到他们忘记发送一封重要的电子邮件。他们的火车将在 15 分钟后从办公室附近出发，但是他们的电脑现在暂时无法运行，这意味着他们要么发送电子邮件，要么错过火车。过热的笔记本电脑也可能在用户关闭机器并将其塞进包里的情况下遭受损失，而没有意识到它没有遵守他们的关机指令。第三种棘手的情况包括需要重启，这是完成关键程序安装所必需的，这使得软件更新的安装成为一个捆绑动作，非常耗时。

相比之下，Linux Mint 从不强迫用户安装更新。它也不会纠缠用户安装更新。在用户认为合适的时候安装更新。让 Linux 软件保持最新并不重要。

## [](#stability-security-and-efficiency)稳定、安全、高效

最普通的用户可能不会对稳定性、安全性和效率的概念特别感兴趣，但是没有人想经历一次崩溃，也没有人想发现他们的计算机已经感染了某种形式的键盘记录、网络摄像头监视个人文件加密病毒。

以我个人的经验来看，(我看不出有什么理由会对其他用户有很大的不同)，当你实际上在期待一个与你的计算机的情绪状态无关的输出时，Windows 更有可能挂起、崩溃或向你显示一个错误对话框。

如果 Windows 系统出现错误，您通常只有两种选择——阅读故障排除指南，希望其中有适合您情况的内容，或者等待微软、软件开发人员或微软与软件开发人员一起发布修复程序。

因为 Linux 是一个具有透明实现的开源系统，所以有经验的开发人员可以选择在系统中挖掘，试图自己修复它。一个没有经验的用户可能没有这个选项，但是他们有一个额外的选项，那就是联系一群热情的人，(或者在谷歌上搜索其他做过这件事的人的记录)，希望有人已经采取主动来补救这个问题。由于代码是开源的，并且有资格由许多有资格的人来进行改进，所以经常会出现这种情况。

关于病毒，大多数病毒是针对 Windows 编写的。这当然是因为它更受欢迎，也可能是因为它更不安全。Linux 系统可能会受到病毒的攻击，但相对于 Windows 电脑，这种可能性要小几十倍。

Linux 在几个方面也比 Windows 轻得多，这带来了很多好处，每个人都可以欣赏。它的安装负载更小，因此下载速度更快。安装后，它会占用更少的硬盘空间，为照片、视频和其他应用程序留出更多空间。它还具有 USB 可引导的新优势，这意味着任何人都可以在自己的电脑上运行一次 Linux，而无需对其进行任何永久性的修改。这对于那些想在购买前尝试的人来说是有价值的，对于那些想使用他们已经习惯的操作系统的人来说也是有价值的，不管他们面前的机器是什么配置。最后，Linux 安装速度更快。我上一次同时安装 Windows 和 Linux 是几年前的事了，但是注意安装持续时间的差异仍然很有意义。Windows 花了大约三个小时。Linux 花了大约三十分钟。

还应该提到的是，Linux 的一些版本(Crunchbang、Lubuntu、Xubuntu、Puppy Linux)是专门为在非常低规格的机器上运行良好而设计的。当试图给旧的或小的、专用的、低功率的计算机注入活力时，这创造了一个比安装过时的 Windows 版本更可取的选择。

# [](#room-for-growth)成长空间

普通用户通过采用 Linux 将获得的最后一个优势肯定是潜在的。如前所述，高水平的可配置性和潜在的高水平自动化是 Linux 最大的优点之一。与上述益处相比，这种益处不会很快或很容易获得，因此，临时用户最初不会体验到这些益处。然而，Linux 的开放性和庞大的爱好者支持网络确实意味着，与 Windows 的同类产品相比，这样的效果来得非常容易。因此，一个潜在的 Linux 转换者可以自信地进入操作系统，因为他们知道，如果他们决定要获得对系统的更多控制，它不会在他们试图这样做时与他们对抗。

# [](#caveats)告诫

如上所述，有许多因素使 Linux 成为普通用户的理想解决方案。然而，使用 Linux 有一个明显但相对较小的缺点，如果本文要完整，就必须提到这一点。

一些软件供应商并不为他们的程序创建 Linux 兼容版本。

承认这一点是对 Linux 的一个小小的打击，因为这个问题通常只能归咎于第三方软件供应商的行为。(当然，不能责怪资本主义的第三方软件供应商非常谨慎地节约资源。这种策略与令人愉快的无差别的全球软件仓库不相称，但对业务本身来说通常是明智的选择)。

这有一个明显的影响。严重依赖少数专业的第三方程序的用户通常会发现，在切换到 Linux 后，他们不再能够轻松或方便地使用构成他们以前基于 Windows 的集合的大约 10% - 40%的专业的第三方程序。

然而，这并不是一个巨大的让步，通常只会影响到*不太随意的*用户，因为:

*   大多数人可能使用的大多数程序都可以本地安装。例如:谷歌浏览器、Mozilla Firefox、Spotify、Skype、《我的世界》、VLC 媒体播放器和雷鸟邮件。Chrome 和 Firefox 的可用性值得注意，因为 web 浏览器能够作为各种其他应用程序的基础。
*   事实上，一些更专业的程序有本地安装程序。例子:MySQL Workbench，Audacity，Blender，Godot。
*   一些更专业的程序提供了辅助机制来让他们的软件在 Linux 上工作，而不是完全成熟的本地安装候选。例如:Adobe Creative Suite、Unity game engine、Visual Studio、Reaper audio desk。
*   一些没有本地安装程序的程序可以被高度相似的 Linux 替代。例子:微软桌面计算器-> Linux 桌面计算器。记事本-> Gedit。Microsoft Office ->打开 Office。

此外，如果上述考虑因素之一不适用于任何给定的计划，可以采用多种方法来规避或减轻损失。用户可以:

*   在 Windows 旁边安装 Linux，当他们需要在纯 Windows 程序上执行工作时，引导进入 Windows。这可能是一个麻烦，但是对于不经常使用的程序，特别是长时间使用的程序，这是一个非常合理的解决方案。
*   在 WINE 上安装 Windows 专用程序，WINE 是一个 Windows 仿真系统，在 Linux 中运行 Windows 程序。这避免了使用替代软件的需要，以及引导进入 Windows 的需要。然而，在 WINE 中运行的 Windows 程序有时会出现错误。
*   在 Linux 中安装一个完整的 Windows 虚拟机，以便在中运行 Windows 程序。完整虚拟机的完全主权和完整性将确保 Windows 程序完全像在本机 Windows 安装中一样运行(可能的例外是性能速度略有下降)。在 Linux 内部的虚拟机程序中运行 Windows 提供了同时性的优势。涉及同时使用 Windows 和 Linux 程序的工作流变得可行。

这些软件损失缓解策略在便利性、效果和参与难度方面有很大的不同。因此，在某些情况下，对 Linux 的缺乏经验和对专业程序的渴望会在某种程度上使减轻软件损失变得更加困难。然而，尽管如此，对于大多数临时用户来说，明智地应用每一个可行的策略通常可以充分抵消“程序损失”的一个(典型的，小的)度量的影响。

# [](#afterthoughts)事后诸葛亮

考虑到上述情况，可以说 Linux 在普通用户看来是:

*   简单易学。
*   快速和安全地尝试或安装。
*   非常用户友好，对于某些发行版来说，与 Windows 没有什么不同。
*   易于安装各种安全、免费的软件。
*   灵活，易于建立基本的定制。
*   方便。
*   免费。

我希望任何以前不熟悉 Linux 的读者现在可以把它看作是新的或现有的、家庭或办公室计算机的一个非常容易接近的选择，并且在没有任何必要的、半专业软件的潜在瓶颈的情况下，将会热衷于采用它。
