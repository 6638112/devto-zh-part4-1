# 前端与后端

> 原文:[https://dev.to/zellwk/frontend-vs-backend-30jg](https://dev.to/zellwk/frontend-vs-backend-30jg)

以前觉得懂 JavaScript 就写后端很容易。我认为这很简单，因为 Node 是 JavaScript。我不需要学习一门新的语言。

但我错了。

但是后端很难学。我花了很长时间才学会它。(而且我还在努力掌握)。

我意识到我在学习后端方面有问题，因为我认为前端和后端是一样的——它们只是代码。

这是一个大错误。

**前端和后端完全是不同的野兽**。在我能正确地学习后端之前，我必须尊重他们的差异。

如果我要解释前端和后端之间的区别(就实现而言)，我会说:

*   前端是关于感知
*   后端是关于沟通的

## [](#frontend-is-about-perception)前端是关于感知的

当我们构建前端时，我们非常关心用户如何看待我们所构建的东西。我们花了大量的时间来问这些问题(如此之多，以至于为每个问题创造了无数的就业机会)。

1.  好看吗？
2.  有意义吗？
3.  它对用户友好吗？
4.  可以访问吗？
5.  快吗？

我们在乎。

有时，我们希望提升用户的感知体验(带来愉悦；耶！)，我们会添加以下内容:

1.  视频/图像/gif
2.  动画片
3.  颜色主题

对于每个附加功能，我们还必须考虑它们可能的含义:

1.  视频/图片/gif 是否过大？
2.  有人受不了我们的动画怎么办？
3.  用户喜欢浅色主题还是深色主题？

我们从用户的角度考虑了很多事情。这就是为什么我说前端是关于感知。

我们为用户着想。我们为他们打造。

(不幸的是，一些开发者自欺欺人地认为。他们为自己而不是应该为之服务的人建立网站)。

## [](#backend-is-about-communication)后端是关于沟通

我来具体说一下。当我说后端时，我指的是前端和数据库之间的层。它也是前端和您需要与之通信的任何 API 之间的层(如果它通过您自己的服务器)。

为了让一些人的事情变得简单，我在谈论你应该在哪里创建一个 Express 应用程序。

[![Image of a frontend, a server, and a database.](../Images/f5063bd730f3c34ae586e005a8b9e938.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--aJJT83aN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://zellwk.com/images/2019/frontend-vs-backend/server.png)

当你在后台工作时，你会注意到我们不再创建接口了。你不填表格；你不能点击按钮。那是因为不需要它们。

后端与感知无关。没人需要看到任何东西。它必须这样...工作。

但是“工作”是什么意思呢？

**后端允许前端和数据库**(或任何外部 API)之间的通信。想象一下早期的电话接线员。如果你打电话给某人，这些接线员必须手动将你的电话接通给你要通话的人。(如果他们弄错了，你就打错人了)。

当您构建后端时，您就像电话接线员。你把事物联系起来，这样它们才能工作。

当我们构建后端时，我们会问这样的问题:

1.  我们需要来自前端的什么信息？
2.  我们应该向数据库/API 发送什么信息
3.  操作是否正确保存在数据库/API 上？
4.  数据库/API 返回了什么信息？
5.  我们应该向前端发回什么信息？
6.  我们在渲染一个网站吗？如果是，我们应该渲染哪个页面？
7.  当信息错误或丢失时，我们该怎么办？

我们也考虑速度和可靠性。(想象一下，如果你发了东西，却没有收到任何回复，会发生什么...).但是这个更像是一个高级的话题。

## [](#different-mental-models)不同的心智模式

这是我在学习后端的早期观察到的一个常见模式:

1.  我会假设事情以特定的方式运行
2.  我会在谷歌上搜索如何以我认为的特定方式实现事情
3.  (谷歌搜索结果会是空的)
4.  我会绕过它，建立自己的基础设施
5.  结果发现(很久以后)我的方式很糟糕

我几乎总是错的，因为当我学习后端时，我不自觉地依赖于我的前端经验。我不得不通过艰难的方式来重新构建我的思维。

这是我最大的收获:

不要假设你对你正在学习的东西有所了解。你几乎总是错的。慢下来，这样你就有时间和空间去学习。当你学习的时候，总是要验证，这样你就记住了，并且重新连接了你的大脑。

(希望下次学新东西的时候还记得这个)。
