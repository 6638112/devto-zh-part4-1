# 让我们编码吉尼

> 原文:[https://dev.to/kurzgedanke/let-us-be-coding-djinnis-2hih](https://dev.to/kurzgedanke/let-us-be-coding-djinnis-2hih)

想象你正在翻一个旧抽屉。在某个地方你会发现一个旧咖啡杯☕️.看起来不错，你也喜欢。你自己想:“Mh...辛苦工作之后，喝杯咖啡，发几篇博客，放松一下是不错的选择！”你拿起杯子，清除几十年的灰尘，把它放在咖啡机下，咖啡开始流动。第一滴酒刚落到杯底，一只小精灵就出现在🧞‍♂️🧞‍♀️.面前起初它看起来很困惑，但很快就清醒了。“我是你的私人编码 djinni！我会为你编写一个应用程序。应该是什么？!"你开始思考...“把你的答案写在评论里吧！”

* * *

每个开发人员都知道这种感觉:“我有很多很酷的技术，我想测试并提高我的技能”，但不知何故，你想做一些有成效的事情，而不仅仅是“Hello World”、计算器和滴答脚趾。你有想法，但有时有完全超出范围的事要做。简单又高效的东西？不...没什么想法。

我现在也有同样的问题。前几天开始学习 react-native，发布了第一个 app。[喜欢就看看吧！(HexToDec - iOS AppStore)](https://apps.apple.com/us/app/hextodec/id1471429873) 但是我有些没主意了，接下来该怎么办。

所以我问你:你还有哪些尚未解决的问题？你缺什么 App，网站(甚至博客)？把它贴在评论里，也许会有人来处理它，来实践这个时髦的新框架，或者只是为了扩展投资组合。

让我们众包我们的需求、想法和技能，这样每个人都能受益！为了方便浏览注释，我建议一些简单的标记，比如:

平台:例如 iOS
问题:问题的简短描述
解释:对问题/app 的深入讨论。

分享你的想法，享受建筑的乐趣，学习一些东西，最重要的是:不要忘记自我护理！
