# 我如何为 Linux 设置硬盘分区。

> 原文：<https://dev.to/johnfound/how-i-am-partitioning-the-hdd-for-linux-setup-o7m>

以下描述的工作流程不被普遍接受。更重要的是，我不确定这能被定义为“好的实践”或“unix 方式”。

然而，它非常方便，有助于在灾难期间保存我的工作文件。

请注意，这种方法仅适用于由单个用户使用的计算机。但是，我们毕竟是在“个人电脑”上工作。

所以，我通常同时安装两个 Linux 发行版。首先是为了简单的灾难恢复，也是为了当我想测试一些发行版或者在不同的发行版上测试我的软件时的实验。

首先，如果需要的话，我会创建引导和交换分区。

然后，我创建两个相对较小的分区(20..50GB)用于安装两个 Linux 发行版。

在硬盘剩余的空间里，我创建了一个巨大的分区，名为“工作”。

然后，当我安装 Linux 时，我把这个分区挂载为一个 **`/work`** 目录(是的，在文件系统的根目录上)，然后我把我所有的工作文件放在这个分区中，把 **`/home/USERNAME`** 目录留给操作系统，让它在里面做它想做的任何事情。

这种方法有几大优势:

1.  您可以在同一路径上的每个已安装的操作系统中使用您的文件。即使您从 USB 拇指驱动器或 CD 启动，您仍然可以挂载工作分区并访问所有重要的文件。

2.  操作系统对这个路径一无所知，所以它永远不会把它的文件放进去。(与 Linux 拥有的 **`/home/USERNAME`** 形成对比)。

3.  工作文件的路径总是很短并且相同。

4.  您总是可以格式化一个或两个 OS 分区，并安装不同的 Linux 发行版，而不需要备份您的工作文件。

有人可能会说你可以把这个工作分区挂载在 **`/home`** 或者 **`/home/USERNAME`** 上，但这确实是个馊主意。

主要是因为从不同的 Linux 实例启动会立刻毁掉你放在主目录下的个人配置文件。原因很简单，不同的 Linux 发行版在创建和使用配置文件的方式上经常不同。
