# 采访编码的简单指南

> 原文：<https://dev.to/bensassialadin/a-simple-guide-to-coding-interviews-3oni>

大多数人没有意识到的是，面试本身就是一项技能，你可以通过学习和实践来学习并变得更好。不仅要测试你的算法和堆栈知识，还要测试你的性格、整体知识和在压力下工作的能力。

作为面试官和被面试者，我参加过很多次面试，最终我积累了足够的经验，知道如何获得这份工作或合同，并通过谈判获得我能得到的最高工资。而这也是我希望通过这份指南传递给你的技巧。

大多数年轻的毕业生，认为编码面试是一个巨大的怪物，站在他们和他们梦想的工作之间，但知道只要有足够的准备，你可以让这个怪物成为你的婊子。

现在让我们言归正传，我将把这个指南分成 5 个部分，技术问题，设计问题，行为问题，心理技巧，最后是一些谈判技巧，以确保你的银行账户中有足够的钱时不时地犒劳自己，如果我帮你找到工作，也许可以请我喝杯啤酒。

技术问题:

这方面取决于你申请加入的公司的规模，公司越大，流程就越复杂，但我会为你准备好每一种可能的情况，虽然流程可能不同，但也有很多共同点。

大多数小公司会让你通过一些编码面试，通常是两次面试，最多三次，超过三次，你就只能靠自己了，开玩笑的，我会支持你的，别担心。

现在回到正题。这些面试几乎总是让你在白板上写代码或解决问题，有时你还会被要求独自完成一个小项目，或者回答算法问题。

现在大公司的面试方法略有不同，通常从电话筛选开始，然后在家进行几次编码面试，最后如果你通过了所有的考试，你将被邀请参加现场面试。

对于技术部分，你可能是一个伟大的开发人员，但你将会得到的技术问题将集中在数据结构和算法上，当然，你会被问到你面试的公司使用的语言和框架，但这些与你将被问到的深入的算法问题相比是容易的，因为大多数开发人员只记得他们在大学时的知识，而这些知识早已被遗忘。但是你没有什么好害怕的，因为有一些工具你可以用来温习它们。Hackerrank、Learneroo 和 Leetcode 非常适合学习数据结构和算法。确保你每天都练习。

当涉及到特定的技术、语言和框架时，你应该实践你选择的堆栈，也许用它构建几个项目，以更好地掌握它，并添加到你的投资组合中，更不用说这样，你会在开发时发现你的弱点，并有机会改进它们，而不是让面试官在面试中为你发现它们。值得注意的是，有很多网站和书籍提供关于大多数语言的深入课程，我建议你看一看涵盖你的堆栈的网站和书籍，并更好地了解它。

设计问题:

设计问题，就是面试官给你一个产品，不管是搜索引擎、社交网络还是求职公告板，让你设计这个产品的系统架构。

大多数开发人员都熟悉一些设计元素，如 DNS 服务器、负载平衡器、数据库等。然而，即使是最疲惫的工程师，也很难回答这类问题，所以如果你不知道如何通过面试的这一部分，不要担心，面试官已经知道你是一名初级开发人员，以前从未——可能——设计过可扩展的生产级系统。

虽然面试官不指望你完美地回答这类问题，但他们会根据你处理大问题的方式来判断你，并记住你应该问很多问题来更多地了解你正在设计的系统，这样你就可以验证你的假设，并在面试官的帮助下以非常迭代的方式回答问题。

为了准备设计问题，您需要阅读每个软件系统的不同组件，尽管您不需要了解所有细节，但您应该了解不同的组件和概念。

在大多数系统中，您会发现一些组件是域名系统(DNS)、内容交付网络(CDN)、负载平衡器、反向代理、应用层、数据库、缓存、异步…

行为问题:

一家公司雇用你不仅仅是因为你的技术技能，还因为你的性格，以及你是否适合他们已经建立的公司文化。

为了确定这一点，面试官会问你一些问题，比如“你为什么想在这里工作？”“你在与团队合作时遇到过什么困难，你是如何帮助解决的”，或者“你是否曾经扼杀过某人的生命，深深地看着他们的眼睛，看着他们的生命之火熄灭”也许不是最后一个，但你会明白。

你应该尽可能诚实地回答这些问题，除非你的答案会将你牵连进一起谋杀案，因为如果你撒谎并得到了这份工作，你可能会发现自己在一个并不真正属于你的地方工作，所以带着自信和信念回答，一切都会好的。

心理诡计:

有很多“技巧”可以用来与面试你的人建立联系，增加你获得工作的机会:

永远要根据面试官的年龄来回答问题。

找到和面试官的共同点，可以是任何事情，从去同一所大学到工作之外的共同兴趣。

模仿面试官的肢体语言，但不要太明显或让人毛骨悚然。

赞美公司，对他们取得的成就表示尊重。

富有表情地说话。

也谈谈你的弱点，向他们展示你只是一个普通人，正在努力改善自己。

确保你同时表现出自信和尊重。

展示你的潜力。

保持眼神交流，但不要咄咄逼人。

穿干净、简单、无害的衣服。

当他们说话的时候要注意听，并确保问很多问题。

面试前洗个澡，你们中的一些人会认为这是显而易见的事情，但从我作为面试官的经验来看，我向你保证这不是。

谈判技巧:

恭喜你！你听从我的建议给自己找了份工作。现在是时候学习如何谈判，尽可能地为自己争取最好的交易了。

如果你想在 70 岁生日前还清那些挥之不去的学生贷款，下面是一些你绝对应该遵循的谈判技巧:

永远敞开大门，永远不要放弃你的谈判能力，直到你 100%准备好做出最终决定。在不放弃继续谈判的权力的情况下，尽可能多的讨论一些要点。

信息就是力量，我的朋友，所以永远不要暴露你的底牌。大多数公司都想知道你目前挣多少钱，或者其他公司给你提供什么，所以确保你一直口头回避这些问题，永远不要透露或评论任何细节，除非你要求澄清。

保持积极的态度，即使这笔交易对你来说不够好，也要保持积极的态度，让他们知道，尽管报酬/时机/数字不尽如人意，但你仍然对公司及其使命非常感兴趣，并且愿意一起制定一个双方都满意的方案。

要能赢，不仅仅是给公司留下你喜欢他们或者对他们的使命充满热情的印象，而是要对他们开诚布公，不玩任何愚蠢的游戏。明确你的喜好和时间表。

有备选方案，因为如果不止一家公司在找你，这会让你在谈判时有优势，给他们一种紧迫感，所以要利用这一点。

别傻了，把一切都写下来。

更多提示:

你可以做很多其他的事情来确保你在申请职位时有优势，你准备得越充分，你最终搬出父母家的地下室的机会就越大，或者找到一份更好的工作，这样你就终于可以每周吃两次以上的饭了:

在开始真正的面试之前进行模拟面试是建立自信的好方法。下载一般面试问题的电子表格，让你的朋友带你浏览一下。

面试的时候问问题，这会让你看起来更聪明，也让你有机会更好地了解公司和员工。

在编码阶段，专注于编写干净、有风格和整体可理解的代码，并确保向面试官解释你正在生成的代码。

前进并征服:

我已经把我知道的一切都教给你了，我的孩子，现在去检验一下你新发现的面试技巧吧。申请你喜欢的公司，一定要让我骄傲。用你新发现的知识造福人类，或者你可以赚很多钱，我想这两种方式都不错。祝你好运！
