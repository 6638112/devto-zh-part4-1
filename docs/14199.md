# 为初学者讲解 Web 开发 vs 移动开发 vs 软件开发

> 原文:[https://dev . to/peteratticusberg/web-development-vs-mobile-development-vs-software-development-explained-for-初学者-23ed](https://dev.to/peteratticusberg/web-development-vs-mobile-development-vs-software-development-explained-for-beginners-23ed)

### [](#software-development)软件开发

软件开发是这些术语中最宽泛的一个，它包括任何涉及编写代码的东西。软件就是运行在硬件上的任何一组指令。我们可以把硬件看作是用于任何类型计算的任何设备。

软件的例子:微软 word，一个手机游戏，一个浏览器，一个操作系统，你从一个外壳执行的“cd”或“dir”命令，一个 unix 机器的内核，一个你写的把“hello world”打印到终端的脚本

硬件的例子:台式电脑、电话、树莓派、高速公路上的电子交通标志、计算器、路由器、调制解调器、电子咖啡机、洗碗机、智能电视

这意味着软件开发可能需要任何东西，从编写运行在咖啡机上的代码到建立一个网站到操作系统。

### [](#web-development)网页开发

Web 开发是软件开发的一个很小的子集。它包括构建软件，该软件由可通过互联网访问的服务器提供服务或在该服务器上运行。服务器只是一台监听其他计算机请求的计算机(即服务于请求的计算机)。

由服务器提供服务的软件的例子:html、css、javascript、作品集网站、react 应用程序、在你的浏览器中运行的游戏、在你的浏览器中运行的文本编辑器、在你的浏览器中运行的 facebook 的前端、任何网站。

运行在服务器上的软件的例子:ruby、python、java、node、ruby on rails 应用程序、HTTP api、facebook、dropbox、nginx

前端开发包括编写由服务器提供服务的软件(例如 html、css、javascript)。

后端开发包括编写在服务器上运行的软件。这可以在任何语言中完成，但最常用的语言是 node、java、python、php、ruby 和 C#。

因为从头开始做后端开发会涉及到解决一堆已经解决的问题，所以几乎所有的后端开发都是在框架或库的帮助下完成的。例如 Ruby on Rails (ruby)、ASP.net(c#)、Django (python)、flask (python)、Laravel (php)、Spring (java)、express (node)等。

### [](#mobile-development)移动开发

移动开发包括为手机(通常是 Android 或 iOS 设备)编写软件。我们使用的大多数移动应用程序都可以与 web 服务器通信，尽管一些简单的应用程序不能。

与服务器通信的移动应用程序:twitter、facebook、纽约时报应用程序、spotify、candy crush(用于支付)、谷歌地图、Venmo

不与服务器通信的移动应用程序:iOS 上默认的“相机”应用程序、默认的计时器应用程序、计算器应用程序、词典应用程序、无论飞行模式是开还是关都工作正常的任何应用程序、没有社交组件的简单移动游戏、应用程序购买或广告。

因为大多数移动应用程序与服务器通信，所以构建一个移动应用程序通常需要同时进行后端开发(web 开发)和移动开发。在一个团队中，同一个应用程序的后端开发和移动开发可以由不同的人或组来处理。

移动开发通常用 java 或 kotlin 在 Android 上完成，用 objective-c 或 swift 在 iOS 上完成。

也有一些框架可以用来开发一个运行在 android 和 iOS 上的移动应用程序，如 react native(使用 javascript)和 flutter(使用 dart)。
