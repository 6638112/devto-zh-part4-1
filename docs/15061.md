# 追求生产力

我们的世界已经对生产力疯狂了。应用商店里有**成千上万**的待办应用，每个人都想尽可能多产。我打赌你知道我在说什么。这个问题的根源可以在[新教工作伦理](https://en.wikipedia.org/wiki/Protestant_work_ethic)中找到。

我们曲解并简化了[生产率](https://www.theguardian.com/lifeandstyle/2010/sep/11/pain-gain-work-ethic-burkeman):我们认为它是通过**你生产了多少**或者**你有多忙**几个并行的项目来衡量的。

此外，当人们的工作缺乏清晰度和目标时，他们就会变得忙碌(你可以在一本名为[“本质主义”](https://www.goodreads.com/book/show/18077875-essentialism)的伟大著作中读到这种影响)。

从历史上看，对生产率的研究仅限于生产实物的工厂。这个类比很有诱惑力。但是我们是知识工作者，而不是工厂工人。我们的生产方式与工厂不同，尽管在高层次上工作流程可以用类似的方式管理。我们的动机是[不同于其他工种](https://www.youtube.com/watch?v=u6XAPnuFjJc)。

在 Metadevelopment，我们更喜欢在 [Trello 生产力指南](https://info.trello.com/hubfs/Personal_Productivity_Ebook_v2.pdf):*中给出的生产力定义，确保你在**正确的时间框架**内，以一种成功的有效的方式*完成**正确的事情**。

这个定义包含了项目的优先级、效率和及时性。

这就是症结所在:**为生产力而优化可能是有害的**！这是一个[过度生产系统](https://medium.com/@flpvsk/work-hard-enough-and-you-wont-finish-anything-d631d65e7478)的好例子。这里有一个很好的解释，说明**无聊** [如何能让你更有创造力](https://www.wired.com/2017/01/clive-thompson-7/)。

### [](#productivity-and-software-engineering)生产力与软件工程

当你把自己当成一个生产单元或者[曲解了“高绩效”](https://dev.to/mtdvio/work-misperception-1mg1)的时候，就非常容易陷入优化生产力的陷阱。
此外，如果你的唯一工作是**不问问题地写代码**，那么你很容易成为一个生产率狂人。

除了追求生产力的直觉，我们还忽略了作为工程师需要注意的其他影响和因素:

*   网络效应(越多人使用你的产品，它就越有价值)
*   [可持续性](https://en.wikipedia.org/wiki/Sustainability)(在不损害未来形势的情况下改善当前形势)
*   [人为因素](https://en.wikipedia.org/wiki/Human_factors_and_ergonomics)(考虑安全、舒适、人为错误)
*   [社会影响力](https://en.wikipedia.org/wiki/Social_influence)(影响一个人的情绪、观点和行为)
*   [认知偏差](https://en.wikipedia.org/wiki/Cognitive_bias)(判断中偏离规范或理性的系统模式)
*   [应用伦理学](https://en.wikipedia.org/wiki/Applied_ethics)(对私人和公共生活中特定道德问题的分析)

软件工程是对软件开发、操作和维护的系统化、规范化、可量化方法的应用(IEEE90 定义)。

作为一名工程师，你可以在产品开发周期的每个领域产生影响。如果你可以成为**高效的工程师**，为什么你只关心成为**高效的程序员**？
