# 公司黑客马拉松的快乐时光

> 原文:[https://dev . to/grepliz/fun-times-at-the-company-hackathon-3i9g](https://dev.to/grepliz/fun-times-at-the-company-hackathon-3i9g)

上个月，我参加了公司的首次官方黑客马拉松。规则很简单:

1.  挑选一个团队(没有单人黑客)。
2.  选择一个想法(产品想法或内部工具)。
3.  工作两天。
4.  在 3 分钟或更短时间内演示项目。

包括我自己，我最后成了三人组。我们探索了不同的想法，最终决定开发一个简单的 React 应用程序，在我们试图“提升”技术技能时记录“学习项目”。老实说，这实际上只是一个美化了的 todo 应用程序，上面还点缀了一些游戏化元素。但是我发现，在如此短的时间内，与一个我从未合作过的团队一起，从白板设计到工作应用程序(以及其间的所有迭代)的过程是非常宝贵的。

我们最初在 CodeSandBox 中使用了 live 功能，但这很快就不起作用了，因为我们开始互相攻击对方的工作。尽管我们最终使用了自己的本地开发环境，并最终将它们合并在一起，但这是尝试其他协作工具的好借口。我推荐 CodeSandBox 中的 live 功能用于短期协作，但不用于长期持续使用。

我也有机会第一次尝试使用 React Native ,尝试将该应用移植到移动平台上。剧透警告:这不仅仅是把你的 React 代码渲染成 React Native 那么简单。但我会说我超级容易起床，并开始使用 https://snack.expo.io/的 T2。

两天很快就过去了，在我们意识到之前，是时候展示了。最后 3 分钟的演示都是通过 [Zoom](https://zoom.us/) 完成的，这样远程团队也可以参与进来。这真的很好，因为我们能够记录一切，如果他们不在办公室，没有人会感到被遗忘。在演示结束时，整个公司都可以投票选出他们最喜欢的项目。

黑客马拉松完成了几件事:

1.  给员工一个从日常工作中休息一下的机会。
2.  提供与不同的人一起工作的机会，创造独特的团队建设体验。
3.  给员工一个探索其他技术和语言的机会。
4.  给了产品团队许多原型想法。
5.  将公司团结在工程创新的周围。

总的来说，这是一次很棒的经历，我坚信这对公司和员工都有好处。显然，公司的领导也认为这是成功的，我听到传言说希望每 6 个月做一次。

我等不及了。
