# 寻找我的开发者社区

> 原文:[https://dev . to/sarahscode/finding-my-developer-community-54mi](https://dev.to/sarahscode/finding-my-developer-community-54mi)

关于我的编码训练营经历的一件有趣的事情是，我们不仅仅学习编码——我们学习成为开发人员。这包括学习编码，但也包括像结对编程和 git 工作流之类的东西。我们被鼓励写博客，参加开发者活动，发展我们的 LinkedIn 网络。训练营的目标不仅是给我们开始开发生涯所需的技术知识，而且帮助我们成为全球开发人员社区的一部分，并以超越编写代码的方式做出贡献。

当我从训练营毕业时，我想成为那个社区的一员。我总是喜欢成为某件事情的一部分，为社区做贡献，但是我不完全确定从哪里开始。我参加了一些聚会，但我真正与人交谈的只有我的朋友参加的聚会(大多数与我交谈的人都是我的朋友)。我创建了一个 twitter 帐户，用于与开发相关的推文，但我大多只是阅读其他人的推文，从不参与对话。我查看了一些开源软件库，看看是否有我想解决的公开问题，但对任何事情都不太满意。

我试着成为社区的一份子，但我真的不觉得自己是。

当我开始我的第一份训练营后的工作时，我并没有真正关注成为开发者社区的一员。这对我的公司来说并不重要，因为我还没有在开发者社区中找到一个让我感到舒服的地方，所以我让自己把它放在了一边。我时常想到去参加聚会，但我很少真的去。有时我会打开我最喜欢的开源回购，看看公开发行的债券...但是我从来没有采取任何行动去做贡献。我甚至一度停止写博客，因为我很忙，没有把它作为优先事项(尽管写博客对我来说一直是了解我的情况的一种个人方式，我从来没有明确地写博客成为社区的一部分)。

我越是感觉不到自己是开发者社区的一部分，就越难摆脱缺乏参与的状态。2018 年底，我决定 2019 年将是我更多参与开发者社区的一年。我又开始写博客了。我试着发更多的微博。我看了看开源的 repos，还是没有找到我想投稿的地方。然后在二月中旬，我大吃一惊——我的公司没有支持我承担新责任的愿望(基于与我老板的一些谈话，我曾希望他们会这样做),而是决定让我离开。这促使我进一步尝试加入开发者社区。我希望积极参与社区活动能帮助我找到下一份工作。我看到社区成员转发求职者的推文，许多人说他们试图将候选人与招聘人员联系起来，所以我决定尝试参与进来，看看这是否能帮助我找到下一份工作。

我从小事做起。我在 Twitter 和 LinkedIn 上发布了我正在寻找一个新的职位。此时，我又开始写博客了，所以在我分享后，我会贴上每篇博文的链接。这有助于分享我的信息，但它并没有真正给我一种成为社区一员的感觉。

感觉自己是社区的一部分意味着与我不认识的人互动，这有时很难做到。我有时会听一个名为 [CodeNewbie](https://www.codenewbie.org/) 的播客，我知道他们有一个 twitter 聊天，所以我开始尝试参与他们的聊天。它给了我一个机会来分享我对某个主题的想法，看看其他优秀的代码爱好者对该主题有什么看法，并与一些真正聪明有趣的人进行互动(尽管老实说，我倾向于喜欢别人的推文而不是回复，因为我并不总是有话要说)。我不是每周都参与(有时我会忘记，有时我会很忙，有时我只是感觉不到)，但在我可以的时候参与聊天让我感觉像是社区的一部分。

我一直在尝试做的另一件事就是在 twitter 上与其他人互动。有人发我觉得有意思的东西，我就点赞或者回复。我不太擅长参与 twitter 对话，但当我觉得有什么要补充的时候，我会努力找到最好的方式与社区分享。

科技界的许多人都熟悉网站 [DEV.to](//DEV.to) ，我最近加入了这个网站，并开始参与其中。过去我偶尔会读到 DEV.to 上的文章，但从未冒险注册，直到有人鼓励我开始在那里交叉发布我的一些博客帖子(现在这种情况会自动发生，这要感谢 DEV.to 上的一个漂亮功能，它可以从我的 RSS 提要中提取)。我注册并发现了一个新的平台来分享我的想法——我发现了有趣和信息丰富的文章，发人深省的讨论，以及一个伟大的人分享支持和庆祝的社区(我喜欢分享本周胜利的想法！)，还有知识。

上周，我有幸参加了有史以来第一次 IRL DEV 社区会议(有开发团队的成员参加并提供帮助，但几乎完全是由社区成员组织的)，我见到了一些很棒的人，听到了一些非常有趣的演讲(包括一次让我重新思考我的财务战略的演讲)，所有这些都是在我感到完全受欢迎和包容的环境中进行的。事实上，我感到如此受欢迎，以至于当没有任何食物可供我食用时，我第二天给其中一位组织者发了封电子邮件，以提供评论/建议(这是我从来没有做过的事情，因为在大多数情况下，我不喜欢要求特殊住宿)，他的回复大致是这样的:“对不起，我以为你可以吃到 ** food x ** ，我没有意识到这将是一个问题。 我们会在未来考虑到这一点”——这是一个非常周到和包容的回答，当时他本可以说一些类似于“我们尽了最大努力，但我们不能适应每个人。” 我所做的最重要的事情是“参与进来”，加入 DEV.to 社区真的感觉就像是成为社区的一部分。

通过更多地参与开发者社区，我可能还没有找到下一份工作。但我确实发现了一个拥有共同经历的社区(即使我们所做的一切都不一样)。一个所有经验都有效的社区，鼓励和分享知识是目标，而不是异常。在这个社区中，我可以分享我所知道的东西(包括技术相关的和开发人员生活相关的)并向他人学习。

是的，开发者社区有负面的部分。在某些领域，仍然存在把关和缺乏对代表不足群体的包容。有些人太专注于一个观点，以至于忽视或轻视那些有其他观点或关注点的人。还有人利用自己的平台去伤害别人。但是，我在社区中参与得越多，我就越发现自己能够关注好的方面，把不好的方面放在一边。重要的是要承认社区中有负面的部分，要确保每个人都感到被包容和受欢迎，还有很长的路要走。但我知道有很多人在为此努力，我们越努力让这个社区成为一个温暖和欢迎每个人的地方，我们就越能赢得更多。

我仍然不觉得自己是一个超级活跃的开发者社区成员。我见过几个人，但肯定不出名。我的贡献是有的，但它们是次要的。但我确实感觉自己是这个社区的一部分，这在六个月前并不是真的。我仍然希望更多地参与进来，通过参与更多的 twitter 对话和聊天，通过继续分享我的想法和经验，以及可能通过为开源软件做出贡献(如果我找到适合我的方法)。但是现在，我对自己的现状很满意，我很高兴成为这个了不起的社区的一小部分。
