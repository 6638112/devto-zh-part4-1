# 武士公爵和 OpenJDK 的传说

> 原文：<https://dev.to/heroku/samurai-duke-and-the-legend-of-openjdk-418>

杜克是什么？没有人知道他的物种或属。人们说他是一个 Java Bean 或软件代理，但我们能确定的是，他让我们想起了 Java 语言及其社区 20 多年的遗产。Java 社区对杜克有着如此密切的关系，以至于设计师们创造了冲浪杜克、宇航员杜克、摇滚明星杜克、麦克拉梅杜克，当然还有 Heroku 的武士杜克。

但是所有这些杜克变种是如何存在而不违反版权法或商标法的呢？毕竟，Duke 代表了软件历史上最激烈的版权之争中的一种语言。事实证明，这个答案可以教会我们很多关于如何培养一个开源社区的知识。

Java 的历史充满了起起落落、对与错、原告与被告以及一大堆 XML。正如那些不读历史的人注定要重复历史一样，学习 Java 的过去可能会帮助你发展你所贡献的社区和产品——即使你只是在读一个吉祥物。

## 起源故事

杜克是由乔·帕朗在爪哇早期创造的，他是一位帮助制作电影《怪物史莱克》和《越过树篱》的艺术家。当时，Duke 是 Sun Microsystems 的财产，和 Java 本身一样是专有的。但是杜克大学已经准备好带领 Java 进入一个勇敢的新世界。

2006 年，Duke 在 BSD 许可下开源，这与 GNU GPL 许可下 Java HotSpot 虚拟机和编译器的发布相吻合。这是真正开源 Java 的第一个子集，Sun Microsystems 承诺 JDK 的其余部分将在明年以 GPL 的形式发布。已经是 Java 吉祥物的 Duke，成为了 Java 可以自由发布的新未来的先驱——就像 Duke 可以由设计师自由定制一样。

今天，一个完全自由和开源的 Java 似乎是不可避免的，但是在 15 年前，这是不确定的。尽管 Sun 生产了一些业内最具创新性的技术，但它仍在努力赚钱，而且免费赠送他们最成功的产品似乎不现实。

在这十年即将结束的时候，Sun 被 Oracle 收购，新东家的第一个 Java 版本 JDK 7 也是第一个参考实现在 GNU GPL 许可下免费开放源代码的版本。今天，有多个 OpenJDK 发行版，开放 Java 生态系统正在蓬勃发展——与 Duke 一起。

这就是为什么 Heroku 一直支持 OpenJDK 的免费发布版本，当它需要为它的 Java 产品设计一个 logo 时，它知道该去哪里找。

## 武士公爵之道

实际上，杜克并不是 Heroku 的第一选择(对不起杜克)。杜克如何成为一名武士的故事笼罩在神秘、阴谋和律师之中。

著名的爪哇咖啡杯冒出蒸汽(我们不允许在这里展示)是 Heroku 尝试的第一个标志。但我们的律师告诉我们，我们侵犯了甲骨文公司对该图像的许可。

就这一次，律师帮了我们一个忙。通过取消咖啡杯选项，他们迫使我们更加努力地思考 Heroku Java 应该是什么样子。

我们想到了公爵，但是一个普通的裸体公爵对于 Heroku 的标准来说是不够令人兴奋的。我们决定把他打扮成经典的 Heroku 风格，所以我们给他一套武士服。最初，武士公爵的腰带里有一把武士刀，但我们的公司团队通知我们，它太暴力了，必须移除。最终的结果是一个可爱的，非暴力的武士公爵，我们爱他。

Heroku 长期以来一直是自由开放的 Java 和 OpenJDK 项目的盟友。武士公爵是我们隐喻的自由保护者。下次你在会议上见到 Heroku，一定要打个招呼，要一张我们的武士公爵贴纸。或者拿一份我们的[武士公爵壁纸](https://www.heroku.com/art/duke)。你将帮助我们庆祝 Java 和开源。
