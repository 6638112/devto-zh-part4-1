# fuchsia OS:OS 世界的游戏改变者？

> 原文：<https://dev.to/jay_tillu/fuchsia-os-the-game-changer-in-os-world-3o9>

大家都知道 OS 碎片化。今天世界上有五种主要的操作系统。在电脑领域，Windows、macOS 和 Linux 占据主导地位，而在智能手机领域，Android 和 iOS 占据主导地位。但是这种分散给开发者和消费者带来了很多问题。

以下是操作系统碎片化导致的一些问题:

1.  **更多的操作系统意味着更多的平台和更多的应用—** 由于操作系统碎片化，开发人员需要为每个平台开发更多的应用，以增加他们对客户的接触。
2.  **更多的 OS 意味着更多的资源和更多的公司管理需求—** 更多的 OS 需要更多的资源和更多的公司管理工作。例如，谷歌需要为每个安卓版本提供安全补丁。此外，这也让开发者感到头疼，因为在创建应用程序时，他们还需要考虑旧的 android 版本。所以有时他们需要牺牲一些功能。

3.  **用户发现自己有不同的用户界面体验—** 由于操作系统种类不同，用户面对的用户界面也不同。每个操作系统的工作和功能都不同。当用户从一个操作系统切换到另一个操作系统时，他们需要学习那个操作系统，忘记使用旧操作系统的习惯。

这是我们面临操作系统碎片化的三个主要原因。现在你可能在想为什么你要告诉我们这些原因。伙计们，我陈述了这些原因，以便你们能够理解 Fuchsia 在 OS 世界中的需求和影响。

## 什么是紫红色项目？

* * *

Fuchsia 首次在互联网上为人所知，而该项目于 2016 年 8 月首次出现在 GitHub 上。当时谷歌没有公开宣布这样的项目。Google 在包括 BSD 3 条款、麻省理工学院和 Apache 2.0 在内的各种许可下免费开放源代码。

它的主要部分是用 Dart 编写的，但 Google 也使用其他语言，如 C、C++、Swift、Go、Python、Rust 和 Shell。与 Android 和 Chrome OS 不同，Fuchsia 不是基于 Linux 内核，而是基于一个名为 Zircon 的新微内核。

另一个令人惊讶的事情是 Fuchsia 的用户界面和应用程序完全是用 Flutter(移动开发 SDK)编写的。这使得 Fuchsia 的 UI 更快(120 fps)，更流畅，跨平台。Flutter 的小工具使用材料设计界面指南，这意味着 Fuchsia 的 UI 和应用程序将比其他的漂亮得多。Flutter 还提供了基于 Vulkan 的图形渲染引擎“Escher ”,可以提供更快的图形渲染。

## Fuchsia 操作系统的预期实现

* * *

从 GitHub 的代码检查来看，Fuchsia 似乎能够在从嵌入式系统到智能手机、平板电脑和个人电脑的通用设备上运行。

这意味着一个操作系统适用于所有平台。如果 Fuchsia 实现了它的全部潜力，那么我们可以期待它无处不在。无论是你的智能手表、智能眼镜、智能冰箱、智能音箱、智能助手、个人电脑、安全设备还是你能想象到的任何东西。如果充分发挥潜力，倒挂金钟将无处不在。

Fuchsia 仍在开发中，我们可以期待未来更多的功能和设备支持…
