# 创意街区

让我们从一篇文章开始，曾经开始一篇文章，并停留在第一段...甚至第一句话；我被第一个词卡住了。我花了很长时间才学会不去尝试和寻找“正确或最好的表达方式”更糟糕的是为这个标题取了一个名字...这看起来令人畏惧，你什么也没想出来，你更加努力地思考...没什么。我对这些问题的解决方法是，当你遇到障碍时使用占位符，并且不要为此感到压力。灵感随后就来了，开始写吧！*不管看起来多么漫无目的或者语无伦次，坚持向前。*

# 灵感？

我经常听到的另一个大问题是，“我应该做什么？”或者“接下来我该学什么？”更多类似的问题。逃避的答案是，“你想要的任何东西”或我最喜欢的，“找到你热爱的东西，”好吧，最后一个还不错，但这并没有给一个在雾蒙蒙的森林中迷路的人一张地图。所以我告诉人们的快速答案是做一个计算器和一个投资组合网站。现在，一个计算器听起来似乎很简单，建议是不要使用原生的数学 API，并尽可能使它成为复杂的科学计算器，添加越来越多的技术和工具层，因为你想学习它们。你可以用它做什么没有限制，我认为上限将是你的计算器开始看起来像 [Wolfram Alpha 计算器...](https://www.wolframalpha.com/)我几乎可以保证，如果这成为你的目标，那将是一个漫长的过程。
作品集总是一个很好的建议，你可以用它来帮助找工作是的，但也可以是你持续时间最长的项目，总会有改进和补充。如果你愿意，你甚至可以改变正在使用的技术栈和工具，从一个创建-反应-应用程序到一个 Gatsby 网站，任何你想学习和尝试的东西。

*   我经常从播客、阅读 dev.to 上的文章、技术社区的 Twitter 帖子、浏览开源项目等事物中获得灵感。

# 绿地项目

根据维基百科的定义，“绿地项目是一个缺乏先前工作强加的约束的项目。”这可以带来很多好处，你可以使用前沿技术，实现最新最好的方法，基础设施等...

那么坏处是什么呢？就像第一部分一样，创造者在另一个完全不同的层面上阻止。你缺乏参考，因此，你可能会感到更加没有方向，可能会开始过度思考和寻找开始的地方。

幸运的是，我们有 create-react-app，Gatsby，Vue 也有类似的东西，有很多技术可以帮助你开始，只是让球滚动起来。更好的是，像 dev.to 这样的非常有帮助和友好的社区会支持你，尽你所能帮助你。

# 开源

如果你想开始，但又想用已有的东西工作，开源项目太棒了！他们看起来很吓人，但我仍然会被吓到。当你学习并努力为项目做贡献时，有开源项目、精彩的社区和人们会与你一起工作。这并不完全是无私的，这也是他们的最佳利益所在，那些试图做出贡献的人会有所改善，这样会有更多的贡献发生并得到改善。
