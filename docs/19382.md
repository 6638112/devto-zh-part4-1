> 原文:[https://dev . to/Ben/is-there-a-future-for-the-atom-editor-5d no](https://dev.to/ben/is-there-a-future-for-the-atom-editor-5dno)

当 Atom 第一次出现时，它是新颖的，因为它是基于 web 技术构建的，因此对于从事这些技术的人来说非常灵活和令人兴奋。它被称为“21 世纪的可破解文本编辑器”。

但是后来，VSCode 出现了，它采纳了许多类似的想法，并很好地执行了这些想法。它起飞了。尽管这些类型的编辑器偶尔会有性能上的缺陷，但我始终是一个信徒。

大约 18 个月前，我写了我从 Atom 到 VSCode 的转变。该帖子已经收到了近 15 万次阅读，所以看起来很多人都对做出同样的改变感到好奇。

小小的竞争不足以终结一个软件生产力工具，但是 Atom 是 GitHub 创造的，GitHub 后来被 VSCode 的创造者微软收购了。我只是想知道这是否会导致他们的“另一个”代码编辑器最终变得晦涩难懂，这个编辑器正试图适应同样的环境。

我没怎么关注过 Atom，所以不知道这个问题的答案。我很好奇那些仍在使用 Atom 或者关注过围绕这个项目的讨论的人在想些什么。
