这篇文章不是“如何做”，因为我绝不是专家。(虽然，我会提供资源。)这篇文章的目的是鼓励你(和我自己)优先考虑响应式设计，不管 CSS 有多令人沮丧。

### [](#avoiding-responsivility-responsibility-responsiveness)逃避责任👈🏾(责任+响应)

我知道这很俗气。😂

当我第一次开始构建项目时，响应性甚至不是我的一个想法。我只会创建一个项目，然后把它命名为 day，而不会考虑它在除了桌面之外的设备上是什么样子。我的想法是“它可能很丑，但很有效。🤷🏾‍♀️"

2018 年， [58%的网站访问量](https://www.stonetemple.com/mobile-vs-desktop-usage-study/)来自移动设备。🤯考虑到这一点，我意识到无论在什么设备上观看，我的项目都必须看起来不错。唯一的问题是，我真的不知道如何应对。我受到了媒体的询问，等等。在教程中，但不知道如何在我的项目中实现它们。

几个月前，我偶然发现了 [Bootstrap](https://getbootstrap.com/) ，最初只是想用它来避免学习纯 CSS 的响应能力。经过一番思考，我认为在转向 Bootstrap 之前，更好地学习纯 CSS 对我最有利。

不要误解我的意思，Bootstrap 是一个非常棒的工具，但是我相信，如果你打算使用它，你应该知道为什么代码以这种方式呈现，而不是一味地复制和粘贴。

### [](#taking-responsivility)承担责任

在过去的一周里，我花了无数个小时让我的[投资组合网站](https://dayrentolliver.com/)上的项目变得有反应。有没有更好/更简单的方法来解决我遇到的一些问题？大概吧。我的项目看起来比一周前好多了吗？绝对的！

如前所述，我不是 CSS 专家。即使当我在我的投资组合站点的当前迭代中进行最后的项目时，我仍然花费了永远和一天的时间来使它响应。但关键是我付出了真诚的努力，最终让它发挥了作用。我知道随着时间的推移和更多的练习，它会变得更容易。

我想这里的主要收获是花时间学习响应式设计，并在你当前的项目中使用它。
