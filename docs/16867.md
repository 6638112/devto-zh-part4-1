# 目标很重要

> 原文：<https://dev.to/mcarterj/goals-matter-2aic>

没有成功的明确定义，就很难知道该朝哪个方向走，或者你是否取得了任何进展。目标为你的团队提供了方向和清晰度。

一个目标是你们集体同意要一起完成的事情。这同时也是你努力的结果，也是指引你努力实现目标的指南针。

目标可以从一个人扩展到整个组织，但只有在团队规模足够大、能够有效实现目标的情况下，目标才能发挥最大作用。在 Leaf，我们经常使用目标来帮助在具有挑战性的开发环境中获得牵引力。

## 目标应该是智能的

*   具体 -他们陈述将要做的事情。
*   **可测量的**——可以客观地证实它们已经实现。
*   可实现的 -他们的团队可以合理地实现这些目标。
*   **相关** -他们对自己所在的组织有一定的好处。
*   基于时间的(Time-based)-他们设定了某种到期日。

使用 S.M.A.R.T 标准有助于确保您的开发团队同意他们都理解的、他们可以完成的、对业务重要的、并且可以在商定的时间框架内完成的目标。考虑下面的例子:

> 到 2019 年圣诞节，改进我们的 web 应用程序，将需要更正的客户数据百分比从 25%降至 5%。-乔·施莫，项目负责人

这是一个好目标。它是具体的、可测量的、基于时间的，在乔·施莫的世界里，它是可实现的、相关的。但是请记住，可实现性和相关性是由手头的情况和组织决定的。

## 目标需要聚焦

在大多数组织中，开发团队同时致力于多个目标。然而，为一个团队设定多个目标并不奏效。当多个目标优先时，你的目标没有一个是优先的。

使用有独立目标的小团队要好得多，允许并行性，同时保留单个团队的焦点。用这种方法，没有人有一个以上的目标或优先事项，但许多事情仍然可以完成。

很可能你最终会有太多的目标可供选择，但是因为你只能专注于一个，所以通过问自己以下问题来做出一个好的选择:

你的团队会对哪个目标产生最大的影响？
您团队的优势和劣势在多大程度上符合每个目标？
你的团队对每个目标有多大热情？
当你的团队能够在一个非常适合他们优势的目标上产生重大影响，并且他们对开始工作充满热情时，你就会知道你已经取得了很好的成绩。关注这些。

## 目标促进成功

**关注** -目标让团队致力于实现单一的结果。这对软件开发来说是非常宝贵的，在软件开发中，复杂的变更会花费大量的时间。有一个合适的目标可以防止时间浪费在繁忙的工作上，帮助你的事业快速进步。

构思 -人是构思机器。在没有指导的情况下，他们会想办法改善组织的各个方面。然而，这些想法并不总是你生意的重中之重。目标有助于引导这种想法去解决有用的、高价值的问题。

自主性(Autonomy)——当人们无法继续工作时，一个目标可以帮助他们做出下一步该做什么的明智决定。不再有“我受阻了，我现在在做什么？”。

**一致** -没有目标，团队经常会意外地完成相同的任务，或者是互相矛盾的任务。明确分配的目标有助于防止这种车轮旋转，并让每个人都在同一页面上。
