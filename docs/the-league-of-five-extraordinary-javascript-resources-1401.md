# (五)非凡 Javascript 资源联盟

> 原文：<https://dev.to/briwa/the-league-of-five-extraordinary-javascript-resources-1401>

无视标题就好。我真的找不到一个标题来代表我要分享这篇文章的所有链接，但无论如何。我看过关于 Javascript 的文章、视频、教程，但真正站出来的只是少数。通常是那些视觉上吸引人，引人注目和娱乐性的。

当然，你把那些拿走，依然是值得分享的东西。但我真的很钦佩那些付出额外努力来形象化他们的想法，以他们和他们的观众喜欢的方式传达他们的信息的人。这真的是一件令人钦佩的事情，也是我正在学习的一项技能。

无论如何，只是一个小小的免责声明，我没有以任何方式支持或赞助这些链接，我也没有从分享它中获得任何好处。反正大部分都是开源的。我只是分享它，也许它也能启发你。开始了。

* * *

# 1。不可信(【https://alexnisnevich.github.io/untrusted/】T2)

我不记得在哪里找到这个的，但是第一次看到的时候，让我想起了小时候玩过的 DOS 游戏。在那个时候，图形界面是有限的，但不知何故我甚至不记得它有那么糟糕，主要是因为故事情节和当时它有多酷。

就像这个游戏一样，在我的记忆中并不是有限的界面，而是它结合了我热爱的两个世界:冒险和学习 Javascript。这个游戏开始时很简单，就像任何其他冒险一样，用 Javascript 解决简单的问题以进入下一个级别。你将不得不使用你所知道的关于 Javascript 的所有东西(有一定的规则)，有时你需要跳出框框思考。我不能再多说了，否则会破坏比赛的！

真的很有趣。造物主的疯狂道具。你也应该去看看。

附注:说到简单却令人上瘾的游戏，我想再一次为[https://github.com/doublespeakgames/adarkroom](https://github.com/doublespeakgames/adarkroom)欢呼，这是一款基于文本的游戏，可以让几分钟变成几小时。

# 2。https://regexr.com/([https://github.com/gskinner/regexr](https://github.com/gskinner/regexr))

如果你和我一样，还在为什么符号在正则表达式中起什么作用而绞尽脑汁，并且一直需要视觉提示，`regexr`就是为你准备的。它分解了正则表达式语法中的每一个标记，并解释了它的作用，同时将语法保留在您可以定制的示例字符串中。不仅如此，在侧边栏上还有一些你会用到的普通正则表达式标记的参考，也有例子。

这个网站真的很棒，因为它的呈现方式。从沙盒到社区驱动的例子，它应有尽有。而且是开源的！如果你想投稿或者对代码感兴趣，你可以随时查看 Github repo。非常推荐。此外，如果你有其他关于正则表达式的网站，请在评论中告诉我，因为我真的需要加强我的正则表达式游戏。

注意，这主要适用于 Javascript 和 PHP RegEx 语法。

# 3。https://dwitter.net/([https://github.com/lionleaf/dwitter](https://github.com/lionleaf/dwitter))

我最近看到了几个来自[#挑战](https://dev.to/t/challenge)标签的帖子，我立刻想到了 Dwitter。它将代码挑战和代码高尔夫带到了一个全新的水平！给你一个`canvas`,它有一些预定义的代码，足够让它启动并运行。然后，您可以用 140 个字符编写自己的代码，尽情发挥您的想象力！人们大多是在创作动画，但我想我以前见过有人创作一个游戏。

它不只是展示你能创造什么，它也类似于 dev.to，在那里你可以浏览，“喜欢”，分享和评论他人的创作。你也可以重新混合它们，就像你在 Glitch 中做的那样。它还有一个健康的社区。看看这个。

# 4。[https://observable HQ . com/](https://observablehq.com/)([https://github . com/observable HQ](https://github.com/observablehq)

让我动笔的文章之一是[这篇关于费希尔-耶茨洗牌的文章](https://bost.ocks.org/mike/shuffle/)。一个对我来说可能很复杂的概念以这样一种方式呈现出来，就好像我在读一个故事。节奏，视觉效果，写作的方式。从那一刻起，我一直在思考这篇文章，我的梦想是写一篇这样的文章。

后来，我发现作者实际上是[迈克·博斯托克](https://en.wikipedia.org/wiki/Mike_Bostock)，一个 Javascript 可视化库 [d3.js](https://d3js.org/) 的幕后人员。我一直在为一个不同的项目使用这个库，从那以后我就成了它的粉丝。稍微挖掘了一下，然后我还发现他参与创立了另一个名为 ObservableHQ 的项目。

我以前看过朱庇特笔记本，ObservableHQ 让我想起了它。一个可以写“会说话”的文章的地方，也是一个分享的地方。里面都是和我之前提到的类似的文章。关键其实是兼具可视化和交互性；它真的重新定义了你如何提出一个想法。我真的很喜欢它，如果你没有看过，你应该看看。

# 5。杰克·阿奇博尔德——消息灵通人士(【https://www.youtube.com/watch?v=cCOL7MC4Pl0】T2

这最后一个有点不同；取而代之的是视频(因为我的网站快用完了，也因为它超级酷)。我很幸运有幸在一次会议上观看了现场直播。[杰克·阿奇博尔德](https://github.com/jakearchibald/)提出了事件循环的概念，这是我经常混淆的东西，让我意识到滥用`setTimeout`从一开始就不是一个好主意。他这样解释，我相信刚进入前端开发的人都能理解。

除此之外，他是舞台上一个有趣的演讲者；他知道如何让观众活跃起来，不断地讲笑话。我可以从他那里学到很多关于事件循环本身和公共演讲的知识。不确定这个人是否在 dev.to 上，但是大声喊出来。

附:我第二喜欢的视频来自大卫·尼尔([@ reverentgeek](https://dev.to/reverentgeek)):【https://www.youtube.com/watch?v=-I8QHkZreyo】T2。他在陈述中的画很中肯。你也应该看看这个。
