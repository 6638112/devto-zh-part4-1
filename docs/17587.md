# 围绕科技多样性和包容性的讨论包括年龄吗？

> 原文:[https://dev . to/tedstark/does-the-discussion-around-diversity-inclusion-in-tech-include-age-523 o](https://dev.to/tedstark/does-the-discussion-around-diversity-inclusion-in-tech-include-age-523o)

我在 Twitter 上关注了相当多的开发人员和技术人员，他们中的许多人在技术领域的性别、性、种族和残疾问题上倡导包容、多样性和代表性。但我很少听到或看到科技领域围绕年龄的讨论，尤其是当涉及到婴儿潮一代和 X 世代的“年长”员工时。我开始好奇，为什么工人年龄似乎退居其他更突出的类别之后。

本月早些时候,《纽约时报》发表了帕特里卡·科恩的一篇文章，内容是关于美国职场中的年龄偏见，以及年长员工在找工作或从失业中恢复过来时面临的挑战:[https://nyti.ms/31eZlQa](https://nyti.ms/31eZlQa)。对于失业的老年工人来说，挑战和偏见很多，令人沮丧。在阅读了这篇文章和由此产生的评论后，人们可能会同意，正如一位评论者所说，“年龄歧视是一种歧视。”...处于或接近我们当前社会问题的顶端，以及接近我们补救努力的底端。”

作为一个肯定符合“年长工作者”定义的人，开发和技术中的年龄歧视问题是我特别感兴趣的问题之一，截至上个月，我的 web 开发生涯刚刚过去一年。虽然这对我来说似乎不太可能，但我现在离退休比 90 年代初大学毕业更近了。尽管我不打算很快退休。我比大多数人晚来开发，但比一些人年轻，我绝对喜欢它。它同时滋养了我的分析能力和创造力。见鬼，这其实可能是我长大后想做的事！我希望我有这个机会，因为我不再年轻了。

综上所述，我很想知道在讨论科技领域/行业的多元化和包容性时，人们对年龄的看法和反馈。年龄是讨论的范畴吗？或者它是其他类别的修饰符？或者仅仅是因为其他类别的高度敏感性而被完全压倒了？“年长”的声音有机会在围绕多元化和包容性的讨论中发言吗？如果不考虑其他特征，婴儿潮一代和 X 一代被认为“太老”而对技术没有价值，他们可以被列入“代表性不足”的绰号吗？

我知道这些问题没有简单的答案。我当然没有。我知道我相信多元化和包容各行各业的事业，但我不确定经验丰富、经验丰富或年龄较大的员工是否属于这一事业的一部分，只有当员工属于性别、性、种族或残疾等其他代表性不足的类别时，才能参与讨论。还是我好奇心作祟，错过了部分讨论。你怎么想呢?
