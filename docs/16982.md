# DevOps 外包:入门所需的一切

> 原文：<https://dev.to/maxchurilov/devops-outsourcing-everything-you-need-to-know-to-get-started-3j8a>

如果一家公司需要持续获得一流的 DevOps 技能，它可以保留一个内部团队，或者将项目外包给一个值得信赖的合作伙伴。我们将解释 DevOps 外包何时以及为什么更好，以及您的企业如何从中获益。

WhatsApp、Slack、Github、Groove、Opera、Google、Skype 这些公司有什么共同点？

除了他们都是成功的数百万美元公司这一事实之外，他们还在某个时间点使用了来自远程员工或离岸合作伙伴的 IT 外包服务(或者继续依赖第三方 IT 提供商来完成某些任务)。

一般来说，外包，尤其是 DevOps 外包，在正确使用时是一种强大的方法。这篇文章将帮助你弄清楚:

1.  为什么你需要 DevOps。
2.  DevOps 外包与内包:需要考虑的差异。
3.  从开发运维即服务中可以期待什么。
4.  合作如何开始。
