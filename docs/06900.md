# 为什么左移测试在连续测试中很重要？

> 原文:[https://dev . to/katalon/why-shift-left-testing-matters-in-continuous-testing-13og](https://dev.to/katalon/why-shift-left-testing-matters-in-continuous-testing-13og)

在当今竞争激烈的时代，上市时间变得比以往任何时候都更加重要。这产生了更多进化的方法，如左移测试和连续测试，其中产品在开发生命周期的早期进行测试。为了满足市场需求以及获得更好的投资回报，越来越多的公司正在将左移测试与连续测试相结合。为了帮助您更好地理解这两个突破性的解决方案，在本文中，我们将深入研究每个概念的定义，回答为什么您应该在持续测试中应用左移测试的问题，并解开这些概念，看看它们如何适应这个大思想。

## [](#what-is-shiftleft-approach-in-software-testing)**什么是软件测试中的左移法？**

简单地说，左移测试是一种将测试提前到开发人员生命周期中，同时提高质量度量的方法。与传统的测试方法不同，在传统的测试方法中，测试是在开发过程的末尾进行的，左移测试的思想是从设计阶段的开始就进行测试，以建立适当的测试策略。尽快发现问题减少了解决问题所花费的时间，并减少了成为快速发布瓶颈的测试变更。

此外，随着左移测试实践的到位，产品团队现在在他们的管道中早期合并测试，从而更快地前进，无缝地改进并赢得市场竞争。

## [](#continuous-testing-what-is-it)**连续测试是什么？**

连续测试是一种软件测试类型，在这种测试中，产品在早期、经常性地被评估，并且贯穿整个连续交付过程。基本上，它使用了一个在开发生命周期早期经常测试的过程。

将持续测试整合到您的组织的测试策略中，不仅可以加速您的上市时间，还可以保持您的客户所期望的质量。此外，持续测试还能让开发人员在发布到产品之前不断得到反馈以修复错误，因此，它确实有助于整个过程运行得更快。

## [](#why-shiftleft-testing-matters-in-continuous-testing)**为什么左移测试在连续测试中很重要？**

随着测试过程越来越严格、广泛和彻底，越来越多的组织正在过渡到持续测试。然而，仅有技术是不够的，将左移测试与连续测试相结合是至关重要的。左移测试应该作为你连续测试循环的基础。

左移测试应该被合并到连续测试中，这样测试人员就可以通过实现真实的功能数据来生成更频繁、更全面、更实用的测试。组合“ [shift-left 连续测试](https://www.katalon.com/resources-center/blog/shift-left-testing-approach/)”增加了应用程序测试的自动化，确保在整个产品开发过程中尽可能多、尽可能早、尽可能连续地使用它。

换句话说，通过合并左移测试和连续测试，可以在早期阶段更有效地进行 bug 检测，从而以更少的努力获得更高质量的反馈和更快的问题解决。

## [](#how-to-apply-shiftleft-testing-in-continuous-testing)**如何在连续测试中应用左移测试？**

由于左移测试和连续测试在每个阶段都尽可能经常地采用相同的测试过程，所以可以说左移测试是连续测试的一部分。虽然左移测试侧重于定义的目标，但持续测试是在 DevOps 管道中执行测试的过程的一部分，以尽快获得关于 bug 的反馈。因此，如果持续测试包含了 DevOps 管道的全部(即从规划、设计、开发到部署)，shift-left 会占用每个周期的一部分，并为周期中的每个任务设置测试策略。左移测试增强了连续测试，使其成为对抗 bug 的更强武器。

此外，您可以通过 BBD(行为驱动开发)应用左移测试。对于那些还不知道的人来说，BDD 是一种通用的敏捷软件开发方法，它增强了开发人员和非技术团队成员(例如经理)之间的协作。

## [](#conclusion)**结论**

毕竟，时间就是金钱。请记住，当您更快地将软件推向市场时，您就获得了竞争优势。将左移测试和连续测试结合起来，一定会帮助你击败你的竞争对手。
