# 3/4 堆栈(web)开发人员

> 原文：<https://dev.to/ldelelis/the-3-4ths-stack-web-developer-2l1m>

*或者如何避免跟不上 JavaScript 潮流*

* * *

几年前，我们都熟悉前端和后端开发的概念；也就是说，完全控制解耦的“一半”(或者不！)web 应用。这意味着独立于另一半团队的进展/阻碍而工作。听起来很棒，对吧？

嗯，*是*很好，**如果**你们在团队中工作。

但是，对于那些对产品/新奇事物有一个可爱想法，但不知道 Nodejs 如何工作，或者不知道如何连接到数据库引擎来存储他们的东西，或者只处理终端来运行`npm run serve`并让 GNU/Linux 的神奇仙女来处理变化的独狼来说，该怎么办呢？

经验丰富的开发人员设计出了*完美的*，100%可靠，为面向未来的电子商务进行了尝试和测试，但不能制作出符合今天(在我看来是高度夸张的)风格标准的登陆页面，或者弄清楚 CORS 请求如何与 JavaScript 的神秘 XMLHttpRequest 对象一起工作，这又如何呢？

你现在可能会想:*但是老兄，你喜欢，完全跳过全栈开发者老兄，拜托！*

你是对的，我没有提到它们，*因为学习完整的堆栈需要时间、努力和妥协。当然，如果你能负担得起，那就去学习你缺少的那一半吧。*

**但是**在你掏出信用卡，在*介绍<当前流行框架>* 课程中清空你最近的薪水之前，想想你是否真的需要学习一种全新的思维方式、工具集、语言等等。完成你的项目。

* * *

让我们回顾一下。你有一个伟大的想法，这需要在硬币的两面都努力，而你只能在其中一面尽你的一份力。你不能停留在一个“半栈”开发人员的位置，完全学习另一个栈会花费太多的时间去考虑它。

进入“3/4 堆栈开发人员”概念。也就是说:您可能无法创建企业级的 99.9999(...)9% [SLA](http://www.businessdictionary.com/definition/service-level-agreement.html) 系统，具有迷人的、移动优先的、进步的、完全响应的视图，但至少你可以提供一个不会让潜在客户在第一次接触时就想关闭屏幕的产品。

这就很好了:我们的主要想法是不依赖于一个团队，做好我们所缺少的*。*

 *但是老兄，我怎么才能成为你提到的那个奇怪的堆栈开发者呢？。好吧，幸运的是，尽管我们双方都有经过战斗考验的、包罗万象的开发框架，如 Django 和 React，但我们*和*也有 tiny，“写 10 行就完事了”的工具，可以省去我们繁琐的工作。

这些工具对 3/4 堆栈开发人员特别有用的地方在于，它们是按照“微框架”方法设计的。它们允许开发人员以最少的工作启动并运行，并从那里开始构建，如果需要的话，能够扩展到实际的框架。

一些例子如下:

*   前端
    *   顺风
    *   ZURB 基金会
    *   预测(JS)
    *   模板引擎(Django，Rails 等。)
*   后端
    *   辛纳特拉(红宝石)
    *   烧瓶(蟒蛇皮)
    *   Koa (Javascript)

* * *

总结:书库不再是你床下巨大可怕的怪物了。用最少的努力在任何一方获得简单的运行是极其容易的。有了现代的最佳实践，如果你的想法成功了，扩展这些选项(我祈祷它成功了，读者！)是一个拥有足够知识的无缝过程。

所以，带上你的简单工具、一些教程和*让我们开始忙碌吧！**
