# [](#disclaimer)免责声明

从一个刚开始涉足该领域的初级开发人员的角度来看，这是一篇非常主观的帖子。

# [](#context)上下文

为了进入主题，我必须谈谈我的旅程。3 年前，我开始做开发人员；当时，我对我后来进入的 JavaScript 世界一无所知。我在攻读计算机科学学位时是一名全职实习生。在法国，这是一个叫做学徒制的系统，我不会在这篇文章中详细介绍，但是你可以在这里找到相关信息。

在我的公司，我整合了两个人指导下的一个产品开发团队。第一个人在六个月后辞职了，我在剩下的学徒时间里一直受第二个人的监护。他后来成了我的导师，直到今天，他还是我的“师父”(老师，但用中文)和朋友。

一切都进行得相对顺利，我遇到了“这”的困难(JavaScript devs 会理解)，组合与继承对我来说意味着什么，我迷失在 Vim 中(我最终摆脱了)并学会了使用 git(有点，git 对我来说仍然很神奇)。其中一个亮点是，我得到了一只皮卡丘，而不是一只橡皮鸭来进行“鸭鸭”调试，我猜你可以说这是“皮卡鼠兔”调试。[插入鼠兔鼠兔迷因]

我爱上了 JavaScript，剩下的就是历史了。

# [](#all-good-things-have-an-end)天下没有不散的宴席

“师傅”在我加入公司开始第一份全职工作之前辞职了。“这是一个熟悉的场景”，他们说，“这是周期的正常部分”，更有经验的同事告诉我，“你最终会习惯的，因为你才刚刚开始”也是一个短语。呃，剧透一下，我还没习惯。

这是一个打击，我预料到了，因为我们已经讨论过了，但感觉还是不好。之后的很长一段时间里，我都感到不舒服，但我说服自己，这是冒名顶替综合症；我害怕了，因为我必须在没有指路明灯或安全网来接住我的情况下解决问题。

两年过去了，另一位同事辞职了。我们相处得很好，更重要的是，我从他身上学到了很多。不舒服的感觉再次抬头，但这一次我知道这不是临阵退缩。“为什么它让我如此烦恼？”我问自己。我和他相处得真的很好，好吧，但是既然我们还保持着联系，当然不可能仅仅是这样吧？经过一段时间的自省，我终于找到了答案:失去了我认为是导师的人，我感到沮丧。认识来得很晚，但我最终明白了潜在的真相。

我从这些人身上学到了很多东西，我可以问问题而不怕被嘲笑。当它只是一个随机开发的主题时，我们在一个公平的竞技场上交换。我感到可以安全地提问、质疑、辩论，从而学习和成长。我可以对他们的想法做出反应，我确信一个不太好的想法会被识别出来，并且会有一个解释。人们可能会说“这是一个普通的、更有经验的开发人员应该做的”，这是真的，但事实是这对初学者来说太难了。对包括我在内的一些人来说，信心是一种易变的东西，我怎么强调都不为过的是，知道你与之互动的人，关于你不掌握的工作(这是否可能本身就是一个主题)，愿意倾听而不瞧不起你是多么重要。

好吧，当然，我们明白了，他们是可怕的 blablablabla，但他们醒来，闻到玫瑰；人们有自己的生活，事情会改变，事情就是这样，你对此无能为力。我明白这一点，我真诚地希望，并且仍然希望，在他们的职业生涯中做到最好。我想谈的不是他们离开的原因或方式，而是他们的指导对我以及他们教给我的东西产生了并将继续产生的影响。

# [](#what-i-was-taught-and-what-i-learned)我被教的和我学到的

在我看来，人们天生就想要成长，展示这种成长，并最终为自己的成就感到自豪。我得到的指导帮助我做到了这一点。我得到了工具，并被允许在日常生活中自由使用它们。这不是突然发生的。这是我首先要强调的，耐心。我被教导要原谅自己，并意识到你不只是成为编程之神，这需要时间，这很正常。我再说一遍，正常，对于那些初级开发人员来说，让他们明白。你不应该试图比音乐走得更快，这会对你的自信非常不利。

当我们谈到这个话题时，信心，这种易变的东西，是一把双刃剑。我会因为在一项任务中实现了我想要的东西而兴奋不已，然后因为花了两天时间在一个 bug 上而一蹶不振。我们作为人类，有时(经历可能不同)，真的很擅长发现伤害我们的事情，而忽略可以帮助我们微笑的事情。意识到每一个错误，每一个拼错的变量，每一次搞砸都是成长的机会，这也许是我最大的收获。要相信你在成长，积累经验，并确信有一天你会转向更大更难的问题，但被遗忘的逗号或拼写错误仍会回来咬你，只是你会更好地处理它。

组织很重要。记下你在做什么，你在与什么做斗争，以及任何看起来重要的事情。人类的记忆不是绝对可靠的，你不可能随时记住所有的事情。当需要时，你可以回头参考你的笔记以获得准确的上下文信息。它对其他事情也很有用:当你觉得你什么都没做时，回头看看你写的所有东西，确保你已经把工作放进去了。

压力是真实存在的！！！！注意你的心理健康。休息是好的，根据你的工作环境尽可能多地休息。"因为休息而感到内疚，不是吗，长官！"玩笑归玩笑，后退一步实际上会帮助你向前迈进一步。当你感到沮丧，似乎无处可去时，深呼吸，散散步，去喝杯咖啡或茶，放松一分钟。

大声说出来，说出你的观点，因为大多数时候别人无法读懂你的想法。他们可以帮助你感到交谈舒适，但如果可能的话不要回避(社交焦虑是一个婊子)。大声说出来并不适合每个人，还有其他的解决方案，你应该找到适合自己的。关键是与你周围的人交流，交流你理解的东西，更重要的是交流你不理解的东西。

不要让任何人告诉你“软”技能不重要。首先，它们不是软技能，其次，在我看来，它们和“硬”技能一样重要。

同事不一定要成为你的朋友才能对你产生积极的影响，任何心态包括分享知识和提升他人的人都可以产生这种影响。在你的同事中寻找那些能帮助你达到新高度的人，这是值得的。

“你不知道你拥有什么，直到它消失了”可能有点夸张，但我必须承认，很难衡量一个人的缺席会产生什么影响，所以如果你身边有好的导师，就尽可能多地受益吧，因为它不会永远持续下去(有人说永远不会，但我敢相信)。

# [](#impact)冲击

这对我的未来有什么影响？我想到了几件事。

我现在很注意把我尊敬的人和公司联系起来。我这么说是什么意思？不可否认，我的导师对我产生了良好的影响，但自动认为“这家公司对我很好”是一种谬误。可能是这样，也可能不是，但这肯定取决于更多的因素，而不仅仅是我和一个人的关系，因为当那个人换工作时，你会很挣扎。

我认为现在“为什么”比“如何”更重要。也许是因为我已经经历了太多的代码，我希望我可以问以前的开发人员“你对此的推理是什么？是什么让你选择这条路线而不是另一条？”。也可能是因为每次我想做一些我向我的导师提出的事情时，总是会被问到“为什么？”；如果你不知道为什么，很可能你真的不知道你在做什么，而且不会有好结果。

# [](#conclusion)结论

还有什么好说的？在我看来，伟大的人会提升他们周围的人，并帮助他们改善自己。我很幸运能和帮助过我的好人一起工作，我现在比以往任何时候都更加确信，没有人能独自完成这项工作。

我希望有一天能成为一名导师，继续努力，帮助那里的其他学生。那会很有趣，也很有压力，但最终还是值得一试。
