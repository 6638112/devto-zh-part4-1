# 什么是并行测试？

并行测试被定义为一种软件测试，其中同时测试多个应用程序或功能，从而减少时间消耗。大多数情况下，这是在一款应用发布新版本时完成的。需要检查不同设备、浏览器和操作系统组合的最新版本的兼容性。

例如，如果有一个新版本的操作系统正在启动，那么就需要测试应用程序，以检查是否所有的功能都可以在新的操作系统版本中正常工作。

因此，当要在不同版本的操作系统上测试应用程序，或者要启动新版本的应用程序时，可以使用并行测试。

要执行并行测试，您可以简单地为应用程序的不同功能创建从项目，并创建一个主项目来运行这些项目。

**为什么以及何时执行并行测试**

并行测试是为了确保新版本的应用程序运行良好，或者检查新旧版本之间的一致性。还可以检查新应用程序的完整性以及版本之间的数据格式。当一家公司从旧系统迁移到新系统并执行同步时，有必要检查遗留数据是否从一个系统导入到另一个系统，以及是否更准确地定义了所有结果，尤其是在财务领域。

**并行测试的好处**

并行测试已经成为 QA 团队的一种需求，因为它的好处不容忽视。让我们来看看执行并行测试的一些好处。

更好的开发流程:更好的报告和快速反馈[并行测试](https://www.pcloudy.com/parallel-testing/?utm_source=onetechstop&utm_medium=gpost&utm_term=p&utm_campaign=pcloudy)使测试人员能够与开发团队共享结果。这种实践也促进了持续的集成工作，这导致了产品的及时交付。并行测试还有助于在 QA 团队和开发人员之间建立更好的沟通，因为测试和报告是及时完成的。

更少的努力:不用说手工测试是非常重要的，但是在不同的设备组合上一次又一次地做回归测试将会非常耗时。执行并行测试将减少您必须投入的工作量，因为您将只需要处理一个脚本。该脚本将在多个设备上复制。

节省时间:并行测试节省了大量时间，这些时间可以投入到探索性测试中。即使你在两台设备上测试你的应用，你的测试速度也会提高两倍。人为错误可能会给你不正确的数据，所以最好使用并行测试来进行这种重复测试，这也可能会很无聊。

更好的测试覆盖率:当你在不同的设备浏览器-操作系统组合上测试你的应用时，测试覆盖率会增加。测试应用的设备越多越好。这将确保最终用户无论使用何种设备都能获得最佳体验。这样，并行测试确保了更好的用户体验，这意味着更好的 ROI。

更少的脚本:您可以只编写一次脚本，然后在多个设备上重新运行[测试。旧版本和新版本的应用程序可以安装在不同的设备上，以检查兼容性和一致性。目标是确保新版本的应用程序在最新版本和旧版本的设备上运行良好，与旧版本相似，但有一些更新。](https://www.pcloudy.com/mobile-app-testing-on-multiple-devices/?utm_source=onetechstop&utm_medium=gpost&utm_term=p&utm_campaign=pcloudy)

**并行测试的挑战**

并行测试有它自己的一系列挑战，在做决定之前熟悉这些问题总是更好的。尽管下面也提到了这些挑战的解决方案。

基础设施的局限性:建立测试环境的成本会对公司的财务产生影响，因为基础设施和维护成本很高。移动和网络设备会增加成本。此外，该公司将不得不雇用高技能的专业人员来维护设置。相反，公司可以选择基于云的服务，如 pCloudy，他们将获得所有想要的设备进行测试，可以随时随地访问这些设备。

数据依赖性:如果测试用例依赖于某些特定的数据，那么制定并行测试的策略可能会很困难。如果我们以注册页面为例。可以有针对用户登录运行的测试脚本，但是如果不是以连续的方式完成，那么任何其他脚本可能会首先运行，这将从数据库中删除用户凭证。可能还会运行另一个脚本，要求用户输入数据。在这种情况下，由于数据依赖性，很难将脚本集成到并行测试中。这就是为什么需要编写真正独立于数据的测试脚本。因此，测试脚本应该具有测试运行所需的配置数据，然后可以修改脚本以并行运行。

**结论**

综上所述，并行测试成倍地减少了测试 app 所花费的精力和时间。它还增加了您的测试覆盖率，并有助于提供更好的用户体验。进行并行测试的最佳方式是使用云平台，因为您不必投资基础设施，并且可以随时随地访问设备。因此，如果您的测试团队分布在全球各地，这将是您的最佳选择，并且在维护中节省的所有额外资金可以很好地用于其他一些功能开发过程。
