> 原文:[https://dev . to/nerve wax/disabiling-the-native-mobile-keyboard-d59](https://dev.to/nervewax/disabling-the-native-mobile-keyboard-d59)

有时候，与原生移动用户界面背道而驰，选择更一致的定制体验是一个好主意。你可以在[expedia.co.uk](https://www.expedia.com)的实践中找到一个很好的例子，他们使用自定义的日期选择器进行日期输入。

然而，在手机上，当专注于输入时，本机键盘通常会碍事，所以这里有一个快速提示来防止这种情况发生。

## [](#mobile-detection)移动侦测

## [](#native-keyboard-prevention)原生键盘预防

一旦我们确定设备是可移动的，下一步就是防止本机键盘打开。

我发现了一个很好的方法，通过*使用`readonly`输入属性欺骗*浏览器。然后，我们将把我们的输入连接到`isMobile`变量，就这样！

```
document.querySelector('input').readOnly = isMobile; 
```

这也意味着输入仍然可以用 js 填充，但是点击它不会触发键盘。
