# python vs . PHP——拔河比赛！2019 年选择加入哪个比较好？

> 原文：<https://dev.to/avantikashergil/python-vs-php-the-tug-of-the-war-which-is-better-to-opt-in-2019-3kma>

如果你曾经和开发人员挤在一个房间里，并想成为争论的焦点，只要声明一种语言比另一种语言更适合后端开发。横幅将被举起，很快讨论室将变成一个战场，不同的开发人员将为支持前端和形成网站主干的不同语言而哭泣。

为完美的后端语言寻找答案对每个企业来说都是一件麻烦事，即使他们将讨论限制在流行和广泛使用的语言上，如。Net、PHP、Python、Ruby 或 Java 从头开始构建网站。

当我们回顾过去两年的 web 发展趋势时，可以看到 PHP 和 Python 有一个急剧上升的趋势。在竞争激烈的网络空间中，两者都被认为是后端编程语言王冠的最有力竞争者。PHP 和 Python 之间的大战似乎不会提前结束，这使得很难找到一个明显的赢家并欣然接受这种语言。

## 这里，两者的一对一比较——Python 和 PHP 会有所帮助。让我们开始吧！

### 学习曲线

学习曲线意味着开发人员需要时间来掌握语法、工具和资源，以将 web 理念塑造成一个功能齐全的网站。与语言相关的学习曲线越低，开发速度越快，成本越低。

尽管 PHP 是一门古老的语言，但 Python 在学习方面胜过它。Python 以其可读性、易于编写的短代码以及编码过程中不会中断程序的小错误而闻名，因为它对格式错误有很高的容忍度。

另一方面，PHP 对初学者并不友好，其丑陋的语法并不出名，而且是为复杂的 web 应用程序设计的，这就是为什么开发人员在支持利用 PHP 进行后端开发方面还有很长的路要走。

**结果:** Python 以更大的优势赢了 PHP。

### 文档

详细的文档对于新手开发者开始构建网站的后端至关重要。无止境的论坛、讨论板、网站和教程为开发在激烈的竞争中脱颖而出的优秀网站提供了更大的帮助。

当我们看到 PHP 和 Python 的文档时，这两种语言都有丰富的文档，可以在各个方面帮助开发人员。

**结果:**在文档问题上有一个明显的平局。

### 社区支持

PHP 是技术块上的一个老孩子，开发者社区是巨大的，专家 PHP 程序员像专家一样不断地贡献和提供帮助。PHP 开发人员获得了更广泛的社区支持。

Python 是一种不太古老的编程语言，但是由于其出色的特性和资源，它的社区正在快速发展。两种语言提供的社区支持几乎是相等的。

**结果:**中性

### 表现

这是一个通用的协议——速度是一个杀手，如果它的加载时间超过三秒，就可以杀死网站。当我们考虑 PHP 编程语言时，它动摇了 7.0 版本后的 web 空间，使它的执行速度比 Python 快三倍。

尽管 Python 有[即时(JIT)编译器](https://en.wikipedia.org/wiki/Just-in-time_compilation)来减少代码编译时间，但在性能方面，这种语言还是被 PHP 打败了。

**结果:** PHP 得分超过 Python

### 调试

大量调试工具的存在提高了开发人员的生产力，因为开发人员不必浪费时间在繁琐和重复的调试任务上。

有了一个强大的开发者生态系统，Python 拥有强大的调试器——Python Debugger(PDB ),它有很好的文档记录，包含许多调试工具，而且易于使用，这反过来又使新手能够即时掌握和使用它们。

PHP 提供了一个名为 XDebug 的调试器包，但与 Python 的调试器相比，调试工具的数量较少，这增加了开发人员在 web 开发中的工作量和时间投入。

**结果:** Python 占了 PHP 的上风。

### 库支持

web 应用程序开发人员更喜欢使用现有的库，而不是重新发明它们，因为这样可以节省大量的时间和精力。Python 附带了一系列预构建的库和第三方库，使得 web 开发变得轻而易举。例如，Python 提供了对机器学习库的支持，如 TensorFlow、SciKit Learn、Theano 或 Keras，它们可以毫不费力地与 web 框架集成，并使机器学习支持的 web 开发变得简单、快速和容易。

PHP 还有一个名为 Packagist 的包存储库，它作为 PHP web 开发的强大支柱而闻名，但它的功能不如 Python 的库丰富。

**结果:** Python 成为构建高级 web 解决方案的首选。

### 人气

Python 被认为是后端开发的一个很好的选择，但是与 PHP 相比，它还是一个新手，还没有占据很大的市场份额。虽然 Python 在被 Google 用于一些项目后已经开始普及。现在，它被 Instagram、Spotify、YouTube 和 Pinterest 等领先平台用于后端目的。

PHP 以 80%的市场份额统治着网络世界，并为包括雅虎、Tumblr、Viber、脸书等热门网站提供支持，抢尽了风头。

**结果:**人气大赛出现平局。

### 框架

框架通过简单的功能集成来帮助开发人员，而不需要从头开始构建。当我们看 Python 提供的框架时，诸如 Django、Pylons、Pyramid 和 Flask 等 Python 框架是稳定、安全、通用和强大的，甚至已经成为全面应用程序开发的事实上的标准。但是，Python 提供的框架选项是有限的。

说到 PHP，PHP 提供了大量成熟的、功能丰富的、非常快速且易于使用的框架。PHP 的数量和 Python 的质量因子在比较中产生了平局。然而，框架带来了前期成本，但它在缩短开发时间和简化设置方面得到了回报。

**结果:**无疑，不存在优于另一个。

**也可阅读:**[2019 年可供选择的顶级 Web 开发框架](https://www.topdevelopers.co/blog/top-web-development-frameworks-to-opt-for-in-2019/)

### 包装

这些模块和包支持应用程序中的系统模块化和代码重用。无缝的包管理通过以一种任何开发人员都可以轻松集成到应用程序中的格式促进包的编写、构建和共享，将各种项目绑定在一起。

Python 支持使用 PIP 工具进行包管理，以便于 Python 包的安装和管理。包管理存在于 PHP 中，但是代码库不能被重用到这个级别。

**结果:**在这个环节中，Python 可以被宣布为胜利者。

**总而言之**

那么哪个最好呢？好问题，这当然是技术比较博客的结尾。然而，很难将选择范围缩小到一个，尤其是当两个竞争者在不同的场景中超越对方时。几个因素支持 PHP，其他因素支持 Python。

正如人们普遍认为的那样，PHP 已经走过了漫长的道路，很容易学习并成为专家。Python 有助于创建健壮的代码，支持代码重用，与调试工具紧密结合，并支持可伸缩性。从中立的角度来看，两者都有巨大的社区支持、详细的文档和框架作为后盾。

前面提到的几点是决定性的，将它与业务用例的多样性相匹配有助于做出最终决策。考虑用正确的技术驱动 web 后端的关键要素。
