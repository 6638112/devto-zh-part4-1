# 我正在寻找一个静态网站生成器的网络用户界面

> 原文：<https://dev.to/oscherler/i-m-looking-for-a-web-ui-for-a-static-site-generator-49a5>

## TL；速度三角形定位法(dead reckoning)

我正在寻找一个 web 界面来管理网站(而不是博客)的静态站点生成器的内容，我很喜欢你的帮助找到一个。

## 故事

当我开始写博客时，WordPress 还不存在。我使用了 Movable Type，这是一个用 Perl 编写的博客引擎，它有一个有趣的特性，即当管理后端是动态的(CGI)时，公共前端都是静态文件，当你发帖或有人评论时会根据需要生成或更新，从而产生一个可以处理大量流量的快速网站(没错，它是一个静态站点生成器，在这个术语开始流行之前的五年多时间里。)

今天，我想同类型的软件，但简单的网站，而不仅仅是一个博客。一个后端，让人们用一个漂亮的网络界面编辑内容，和一个引擎，用它生成静态页面，模板，导航等。

## 我的愿望清单

*   编程语言无所谓，我打算在 Docker 里磨合；
*   后端和站点生成器不需要是同一个产品，只要我能从前者获取内容到后者即可；
*   我想在我自己的服务器上托管它；
*   我想使网站，层次页面，而不仅仅是一个博客(层次页面控制如何显示导航)；
*   首选所见即所得编辑器；
*   基于网络，所以不同的人可以编辑网站，而不必互相发送源文件；
*   既然我要用它来为朋友们制作小网站，我宁愿这个解决方案是免费的和/或开源的。至少不是每个站点的许可证。绝对不是 SaaS。肯定不是每年 499 美元。

## 我试过什么

*   Ghost:非常好的界面，但是只用于博客。常规页面太基础；
*   Jekyll Admin:只适用于 Jekyll debug web 服务器，并不真正适合生产；
*   Grav:与我所寻找的正好相反:你必须编辑静态文件，它们是动态提供的；
*   Netlify CMS:似乎有一个非常复杂的方法来做简单的事情。

如果有人有建议让我尝试，我会非常感激。提前感谢。
