## 我有史以来的第一篇文章和第一个 npm/yarn 包！

我不熟悉 mongodb，尤其是 nodejs。在我用 nodejs 学习 mongodb 的过程中，我遇到了一个叫做 mongosejs(mongoose)的工具，它可以帮助开发人员轻松地使用 mongodb。我的 [repl.it 页面](https://repl.it/@waptik/mongoose-test)是我第一次开始练习 mongodb 和 mongoose 的地方。它包括我在 Mozilla 的 [MDN 上关注的教程。](https://developer.mozilla.org/en-US/docs/Learn/Server-side/Express_Nodejs/mongoose)

我以前玩 mysql 和 nodejs 的时候，一般都是用 [Sequelize](https://github.com/sequelize/sequelize) 和它的 [cli](https://github.com/sequelize/cli) ，所以我在给 mongoose 找一个类似的 cli。我在网上搜索了一下，找到了这些[https://gist . github . com/waptik/c 7557d 7 AC 1 f 7 ba 2 c 3a 6b 589 ee 30 b 2 EC 5](https://gist.github.com/waptik/c7557d7ac1f7ba2c3a6b589ee30b2ec5)。他们中的大多数要么没有配置文件，要么没有维护，要么只做一件事(迁移/播种)。

我对自己说，为什么不自己造些东西来解决我的问题呢？我回到 sequelize-cli github 页面，克隆了回购协议，开始摆弄东西，这就是 mongosejs-CLI 的诞生。我还在[产品搜索](https://www.producthunt.com/posts/mongoosejs-cli)上发布了它。

如果你使用 mongodb 和 nodejs 或者 mongoosejs，请试一试，让我知道你的想法。
