# DEV 在一年前的今天开始开源，我们有更多的计划！

> 原文:[https://dev . to/dev team/dev-went-open-source-一年前的今天，我们有更多的计划-530f](https://dev.to/devteam/dev-went-open-source-one-year-ago-today-and-we-have-so-much-more-planned-530f)

今天标志着 DEV 开源一周年。开放操作已经成为我们如何构建软件的核心价值和根深蒂固的原则，很难想象这仅仅是一年。

该网站最初的理想是提供高质量的发布工具和包容性的社区功能。老实说，早期我并没有想到将这个平台作为一个开源项目来构建。但是，在观察了我们每次给社区更多合作机会所获得的收益后，我们发现这是一个显而易见的选择——我们分享了这艘船的钥匙，开放了源代码，并且没有回头。

选择开源我们的代码库让我们成为应用程序代码、数据和开发实践透明度的领导者。我们计划在未来几年越来越多地扮演这种领导角色。

## [](#a-vision-towards-phase-2)展望第二阶段

您可以将第一阶段视为构建 DEV。支持这个了不起的软件开发人员社区平台将永远是我们团队的核心重点。但是我们也为代码和更广泛的社区制定了未来的计划。

我们已经[公开地](https://dev.to/ben/how-devto-could-topple-facebook-2nj3)谈论了我们关于在微观世界开发者社区方面重复使用我们的社区创建平台的想法(像 *stripe.dev* 这样的社区是为了平台开发者的成功，或者 *node.dev* 是为了与 fullstack JS 的构建者进行更深入的集成)。这些独立的软件实例将拥有自己的数据和对操作的完全控制。我们也可以根据更通用的兴趣来设想用例(比如 *design.to* ，甚至*community.nba.com*)。

我们在社区内的企业家中经历了很多建立个人实例的需求，但是因为实际原因要求人们推迟。随着第二年的开始，我们将开始支持这些使用案例。期待未来几个月的官方路线图。

## [](#a-more-organized-core-team)更有组织的核心团队

随着我们一起向前发展，我们希望与全球越来越多的开发人员合作。第一年是一个大众传播和实践混沌共情的实验，因为我们一起建设。在接下来的一年里，我们将寻求制定更好的指导方针来参与这个项目。我们将更快地响应问题，更清晰地划分责任，并更快地解决拉动式请求。

我们在这方面已经相当不错了，但我们希望在这方面成为真正的领导者，就像我们做的其他事情一样。

## [](#built-on-github)建立在 GitHub 上

很难想象在一个没有 GitHub 的世界里，我们在开源的第一年取得了什么成就。我只想承认这种事实上的伙伴关系到目前为止是多么的共生。在很大程度上，我们只是另一个项目——但我们在很多方面也是独一无二的。

首先，作为一个包容性的开发者社区，我们在这里互相教授如何在这个职业中生存和发展，我们正在建立 GitHub 开创的社区概念。

此外，作为一个面向消费者的开源项目，我们不太常见(相对于框架之类的东西)。我们能够从其他项目的范围内的高层次和低层次获得很多线索。随着我们继续将自己树立为软件行业的一股强大力量，我希望我们能与合作的组织建立更牢固的关系。

## [](#built-on-rails)建在铁轨上

Ruby on Rails 无论如何都不是新的热点，但是对于我们所做的选择来说，它是如此自然的合适。与目前的其他框架相比，Rails 是一个相当“捆绑”的软件，为我们想要做的大多数事情提供了一个内置的解决方案。Ruby on Rails 是由 Basecamp 开发的基于 Ruby 的 web 框架，它在后端使用 Ruby，在前端集成现代 JavaScript。

作为与我们有许多共同特征的组织和项目的首选技术(想到了 GitLab、Shopify 和 Discourse)，很明显，在未来的许多年里，我们将以这样或那样的方式与成千上万优秀的开发人员合作。当然，我们对项目和社区的健康和福利感到非常满意。Ruby on Rails 网站将 Basecamp、GitHub、Airbnb、Twitch、SoundCloud、Hulu、Zendesk 和 Square 列为一些值得注意的用户。

## 让第二年开始

在过去的一年里，我们能够从项目中雇佣贡献者来扩充我们的核心团队。我们计划在第二年做更多这样的事情。在社区增长方面，我们的流量增加了两倍多，eclipse 20 万注册会员增加了一倍，真正加速了我们的开源业务。我们计划在第二年实现更大的飞跃。

我们能够与今年成功合并拉取请求的数百名开发人员合作。我们计划在第二年改进我们的贡献模型，这将使我们能够支持数百或数千名新的贡献者来帮助完善和发展这个项目。

我们希望将开源推向主流。我们相信，我们可以帮助将透明度和安全性的新标准提升到社区平台领域。帮助我们到达那里。

## 什么是 Forem？

Forem 是用于构建社区的开源软件。为你的同行、客户、粉丝、家人、朋友以及任何其他需要聚集在一起成为集体一员的时间和空间的社区[参见我们的公告帖子](https://dev.to/devteam/for-empowering-community-2k6h)以获得 Forem 是什么的高级概述。

[dev.to](https://dev.to) (或者只是 dev)由 Forem 托管。这是一个软件开发人员的社区，他们撰写文章，参与讨论，并建立自己的专业档案。我们重视支持性和建设性的对话，为所有成员追求伟大的准则和职业发展。该生态系统涵盖从初学者到高级开发人员，欢迎所有人找到自己的位置…
