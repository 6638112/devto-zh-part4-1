## [](#javascript)JavaScript

这是一种异步的客户端编程语言，它在浏览器中做着令人惊叹的事情！使用 [Node.js](https://nodejs.org/en/) 它也可以在服务器端运行。它变得很受欢迎(尤其是在 [codebar](https://codebar.io/) 学生中！)因为它在整个堆栈中的使用，它闪亮的吸引力，也许，它是古怪的。

我挣扎于在我的代码中写满`function()`的需求，但是学习如何使用它们来执行代码块帮助我更好地理解 Ruby。
JS 具有被称为构造函数的特殊函数，这些函数定义了将由它们的实例继承的属性和方法。把`Donkey`想象成一个函数。驴子的每个实例都具有相同的驴子特征。比方说`jenny`和`jack`(它们是`Donkey`的实例)在沙滩上骑行时有点慢，你可以在`Donkey`函数上使用`prototype`来添加特性，比如给每头驴添加一个`jetPack`的能力！

## [](#configuration-files)配置文件

我在对一个 bug 做了一些调查后提出了一个 pull 请求(关于一个被[制造商](https://makers.tech/)学生用来 lint Ruby 代码的`.yml`文件),这个请求在本周被接受了。我没有真正理解为什么 YAML 文件在文件目录中，但是当我链接我的代码时它们抛出错误，我想知道为什么！

配置文件用于配置某些程序中的设置。这些文件应该是人类可读的(像所有代码文件一样)，并且它们可以包含键/值对来指定配置设置。

> 💡你可能在`.json` `.xml`或`.yml`文件中看到过通用配置文件(特别是对于开源软件)，比如:
> 
> *   [【ruboop】](https://github.com/rubocop-hq/rubocop/blob/master/.rubocop.yml)s YAML 文件
> *   [ESLint](https://eslint.org/docs/user-guide/configuring) 的 JSON 文件

编辑配置文件使您可以灵活地只使用您想要的库的一部分，并使用您自己的代码定制其他人的文件如何运行，这样您就不会受限于默认设置。
