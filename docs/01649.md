# 兰尼·加西亚·克雷斯波:React 推动了网络的边界

> 原文：<https://dev.to/gitnation/lenny-garcia-crespo-react-pushes-the-boundaries-of-the-web-140o>

## 我们采访了莱昂纳多·安德烈斯·加西亚·克雷斯波，他是 Twitter 的前端工程师，在网络团队工作，之前是 TweetDeck 的成员

在 React Advanced 的这次采访中，Twitter 的软件工程师 Lenny Garcia Crespo 分享了他对 React 的热情，并解释了是什么使该框架及其背后的社区如此成功。Lenny 还谈到了他为 Match.com 所做的工作和他目前在 Twitter 上的工作，Twitter 与他以前工作过的任何其他地方有何不同，以及为什么这个社交媒体(相对于脸书)在 web 开发人员中如此受欢迎。10 月 25 日，Lenny 在伦敦的 [React Advanced](https://reactadvanced.com/?utm_source=blogpost&utm_medium=devto&utm_campaign=interview) 发表演讲。

#### 你好 Lenny，欢迎来到 React Advanced 的访谈！请介绍一下你自己。

嗨！我叫 Lenny，是 Twitter 的一名软件工程师，目前居住在英国伦敦，来自阿根廷的布宜诺斯艾利斯。

我开始用 Visual Basic 编程，但不久之后，我就进入了。NET，在那里我开始做 web 开发。

我想直到我开始用 jQuery 和后来的 Backbone 做有趣的事情，我才开始喜欢 web 开发(更确切地说，是前端)。我渴望学习所有的技巧，并弄清楚在实现非常交互式的页面时发生了什么，所有的事情都在变化，事情似乎是实时更新的。在这个过程中，我了解了 Javascript 的古怪之处，也了解了它创建交互式应用程序的能力，以及它与其他语言相比有多简单。

在与 Backbone 一起工作了几年后，我开始了解 React。我没有立即爱上它，实际上我第一次看到它时就抛弃了它，认为它是“又一个新框架”，但是在阅读了 David Nolen 关于 MVC 的未来的博客文章后，我又看了它一眼，就在那时，我再也没有回头。

在那段时间里，我也做了一些有角度的工作，这让我对 React 如何是我更好的选择以及它所解决的问题有了一些看法。

[![Lenny](img/5b0272c72a8589b2ab9dba9fd9637f64.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ws1SgmJK--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ru7tht1v86qmehawaimd.jpg)

#### 能否请你简要描述一下你在 Twitter 目前职位上的工作经历？

如前所述，我开始做一些 Visual Basic，但后来转移到。NET 使用 ASP.NET web Forms 开发 Web 应用程序，然后是 ASP.NET MVC，这是一种更类似于 Ruby on Rails 的架构。

在完成一些大项目后。NET，我想搬到 Ruby，因为我真的很喜欢这个社区和正在发生的创新(我在。NET，那里基本上是来自 Ruby 的思想的港口)。我这样做了，当我在几个项目中使用 Rails 的时候，我开始用 Backbone 探索一些高级的 UI 操作。我参与了一个有很多交互的实时仪表盘的项目，我们用 Backbone 做了一个单页应用程序，回头看看当时考虑的替代方案，这是一个伟大的决定。

然后我做了一些其他的项目，也是带图表的交互式仪表盘，但是这次是 Angular，与主干相比，这又是一个进步，因为它给出了更多的结构。与此同时，我正在研究 React 的辅助项目，实际上，不久之后我就开始在工作中使用它了。

在加入 Twitter 之前，我在 Match.com 从事一个非常大的项目，致力于应用程序的重新设计，该应用程序是作为一个单页应用程序与 React、Flux 以及后来的 GraphQL 一起构建的。

最后，当我加入 Twitter 时，我开始在 TweetDeck 工作，直到最近。TweetDeck 是一个令人难以置信的网络应用程序，可以让你以超级用户的身份查看 Twitter，同时查看多个时间线，所有时间线都是实时更新的，还具有过滤和安排推文等高级功能。不久前，我加入了 Twitter.com 团队，仍然来自伦敦，做平台类的工作。

#### 你在推特上做什么？

我目前在网络团队工作，主要负责 twitter.com 和 mobile.twitter.com。我最近加入了团队，从事基础和平台类的工作。之前我在 TweetDeck 团队工作，也是一名网站开发人员，这也是我将在我的演讲中谈到的一部分。

[https://www.youtube.com/embed/N5v8Ul6ph90](https://www.youtube.com/embed/N5v8Ul6ph90)

#### 你认为在 Twitter 工作和在其他社交媒体公司工作有什么不同？

谈到社交媒体，我以前只在 Match.com 工作过，但我不确定我能比较经验，因为情况和公司类型与 Twitter 完全不同。我不太了解在其他社交公司工作，所以不太清楚这种情况下的区别。我所看到的与我以前的经历不同的是公司内部发生的跨团队合作的数量，所做的实验的数量，这是我以前从未见过的，以及一切是如何由数据驱动的。此外，你能够用你的代码对数百万人产生影响，这真的很酷。

#### 你为什么认为 Twitter(相对于脸书)已经成为网络开发者的首选社交媒体？

我认为 Twitter 被选为网络开发者的媒介是因为所有的事情都发生在公共场合，而且我们的行业非常擅长分享(代码、学习、思想、技能)。作为一名 Twitter 用户，我主要使用它来了解行业的最新动态，当我开始使用 Twitter 时，我真正喜欢它的是它是多么容易开始关注人们，并根据你喜欢的东西获得你的时间表中充满有趣的信息。这是一个永无止境的信息流，包含不相关但相关的信息、博客帖子、分享的想法、对话，以及你感兴趣的特定事物。这也是拓宽你对其他事情的视角的一种方式，这些事情可能不完全是你的兴趣所在，但是重要且相关，主要是作为一个与其他人一起工作的人。以我的经验来看，它帮助我在我真正喜欢做的事情上做得更好，但也让我意识到并在我不知道我应该做得更好的事情上做得更好。

#### 你最喜欢的技术栈是什么？你为什么更喜欢对 Angular 或 Vue 作出反应？

我真的很喜欢使用 React 和 GraphQL，以及 Relay 或 Apollo 这样的客户端，因为我认为它消除了产品开发人员的许多问题，使他们可以专注于构建产品的创造性部分，而不是重复(但重要)的任务，如数据获取，同时在这些情况下做正确的事情。

我喜欢 React，因为它(对我来说)在灵活性和性能之间进行了恰当的权衡。它允许我非常快速地开发用户界面，并允许我混合和匹配尽可能多的自定义逻辑，而无需求助于框架的高级功能或变通方法。与其他 API 相比，它的表面很小。

此外，我真的很喜欢 React 核心团队和社区如何思考要解决的下一个问题，并推动 web 的边界，以使体验感觉更像本地。

[https://www.youtube.com/embed/mbbZ3LktluE](https://www.youtube.com/embed/mbbZ3LktluE)

#### 在 React 中，你认为最有争议的是什么？你认为 React 作为一项技术会取代 Vue 和 Angular 吗？

最近有一些关于 React 如何更喜欢不可变类型的架构的讨论，在这种架构中，您不直接修改东西，而是用修改创建副本并传递那些新值。这本身听起来没有争议，但是由于这个决定和它为 ui 带来的好处，一些问题的解决方案可能对人们来说有点争议，比如钩子在函数体内并一直被调用，使得副作用似乎与渲染的纯粹性混为一谈。

事实上，我认为语法可能是 React 中一直有争议的东西。JSX 在一开始是有争议的(也许现在仍然是？)，和 hooks now ( [这是概念语法](https://overreacted.io/react-as-a-ui-runtime/#static-use-order))。

我认为在某种程度上，React 已经是 Vue、Angular 和其他库的一部分，所以即使它不会取代它们，在某种程度上也会，因为 React 的思想在所有其他库中都是活跃的。

话虽如此，但由于核心团队和社区对未来的思考超出了网络目前所能做的，我认为 React 至少会存在很长一段时间。

#### 你对开源有什么参与？你参与了哪些项目？哪些项目是你最喜欢的？他们中你最引以为豪的是谁？

我偶尔做一些开源，最近没怎么做。我有一些像 [React Sound](https://github.com/leoasis/react-sound) 这样的小项目，以及一个用于 graphql 持久化文档的[加载器](https://github.com/leoasis/graphql-persisted-document-loader)。我通过改进 Typescript 支持为 Apollo 做出了贡献，并参与了查询、订阅和变异组件的介绍(现在已经有了这些组件的钩子)。我真的很喜欢关注 React、[接力](https://github.com/facebook/relay)和[阿波罗](https://github.com/apollographql)中发生的事情，并且我密切关注时事通讯，例如 Philipp Spiess 的 React 中的[。](https://this-week-in-react.org)

#### 你会如何完成这句话:成功的开发人员应该…？

有好奇心，愿意把事情做得更好，有团队精神，对他人好。

#### 你有什么爱好吗？你在推特上提到你喜欢开糟糕的玩笑，你能给我们讲一个吗？

我有一个女儿，所以我在家的时候真的很喜欢和她一起玩。我喜欢演奏和听音乐，我弹低音吉他(也弹吉他)，我喜欢爸爸的笑话和双关语，比如这个:“一个组件从哪里订购比萨饼？DOM 节点的"

#### 你对即将在伦敦召开的会议感到兴奋吗？你将谈论什么，你对这次活动有什么期望？

我很激动！期待与人们见面，谈论技术和非技术的事情，并向使用 React 做非常先进甚至疯狂的事情的人学习。

在我的演讲中，我将谈论一些我们用来有效地迁移部分 TweetDeck 代码库到 React 的技术，以便处于相同情况的人可以从中学习，甚至可以应用它们来更新他们的堆栈，并获得使用 React 的好处。
