# 关于 UX 的一个观察(来自 noob) - Twitter 喜欢 vs 其他

> 原文：<https://dev.to/alexantra/an-observation-about-ux-from-a-noob-twitter-likes-vs-the-rest-2f60>

最近，我和未婚夫就我们如何在社交媒体上使用赞进行了一次有趣的对话。

这是因为我发现我不“喜欢”twitter 上的东西，相反，他喜欢他看到的许多推文。

我解释说，我只在 twitter 上“关注”那些我打算以后再看的东西，如果我喜欢，我会转发它。实际上，我将 like 函数用作书签功能。

(是的，我知道 Twitter 现在有了书签功能)

“但这是一个喜欢。”他回答道。“你喜欢脸书和 Instagram 上的东西，但你不打算以后再去浏览。”他接着说。

他是对的，那么 Twitter 为什么与众不同呢？

**呈现特征的上下文不同于其余的**

当你打开你的 twitter 账户时，前面和中间是四个关键的标签。推文、推文和回复、媒体和喜欢。

这款应用首先向你展示的是如何立即访问你喜欢的东西的列表，对我来说，这立即改变了喜欢的设置。

在脸书和 Instagram 上，我喜欢发布帖子的人，这是我说“干得好”的方式，我不知道如何访问我在脸书喜欢的帖子列表，虽然这个功能现在存在于 Instagram 上，但它隐藏在设置中。

所以提前宣传你有能力回到你喜欢的一切改变了我对待它的方式。

当我可以喜欢 facebook 上的任何东西，并且知道这些喜欢消失在空气中时，就没有责任了。知道喜欢的人会进入一个列表，让我觉得我应该聪明地对待他们，或者稀缺。

我不想三周前不得不依靠那个列表来找到我喜欢的特定推文，却不得不费力地通过数百条推文才找到它。

或者你可以公开你的喜欢，我不想再考虑是否喜欢一条推文，可能是克里斯·海姆斯沃斯的赤裸上身，我可能不想让人们看到我喜欢过。

我知道我是这里的少数派，并且显然过度考虑了这个特定的功能，但是它回避了为什么是这样的问题？

难道这些功能不是被包装在多层的 UX 和客户旅程中，目的是让我更多地使用它们吗？然而它对我产生了相反的效果。

只是一些奇怪的事情我的大脑在思考。我在 IT 行业工作，所以我接触到这些概念，但当然知道我在做什么。我只知道足够危险。

你的想法？
