你在考虑使用亚马逊 EBS 吗？简单的可伸缩性和按使用付费的选项使它成为一项有吸引力的服务，但如果您想避免 AWS 账单中的任何意外，还有一些事情需要考虑。在这篇文章中，我们提供了五个技巧来帮助你有效地降低亚马逊 EBS 的使用成本。

## 什么是亚马逊 EBS？

亚马逊弹性块商店(EBS)是向亚马逊网络服务(AWS)用户提供的服务。该服务提供块存储卷，可以与 Amazon 弹性计算云(EC2)结合使用，也可以在 AWS 云中使用。

Amazon EC2 提供可调整大小的基于云的计算能力，这可以促进网络规模的云计算。EBS 通过块级存储卷为 EC2 实例提供存储。
EBS 卷用途广泛，可以用作数据库的主存储设备，也可以作为根分区附加到 EC2 实例。使用快照系统，该卷可以作为备份，在 EC2 实例被删除后仍然存在。它的主要特点是:

*   这是一个即插即用的系统
*   它允许自动复制每个卷
*   低延迟性能
*   易于扩展

## 多少钱

亚马逊 EBS 存储是一种按使用付费的服务。公司根据每月需要多少千兆字节的存储空间付费。这与 [EC2 实例](https://aws.amazon.com/ec2/instance-types/)不同，EC2 实例只在运行时产生费用，EBS 卷只要附加到实例上就会产生费用。即使实例已经停止，也会发生这种情况，但删除实例时不会发生这种情况。

因此，存在 EBS 卷在后台产生费用的风险，但由于其连接的实例未运行而未被注意到。这相当于 AWS 法案中累积的一大笔钱。

如何在最大化 EBS 功能的同时降低成本？下面我们解释降低 EBS 成本的五个诀窍。

## 降低 EBS 成本的 5 个技巧

### 1。根据您的性能要求调节音量

如果您购买的存储空间超出您的需求，您最终将为未使用的存储空间支付更多费用。因此，选择正确的 EBS 卷大小至关重要。应该根据容量、流量(IOPS、每秒输入/输出操作数)和应用程序吞吐量等因素来调整数据块。您应该监视数据块的读写访问量，定期进行配置以检测未使用的数据块。如果请求/响应量很低，您应该缩小 EBS 块的规模并降低成本。

[EBS 卷](https://aws.amazon.com/ebs/)有三种大小:通用(SSD)、配置 IOPS (SSD)和磁性。一个好的技巧是将 EC2 连接到通用 SSD 卷，并将调配的 IOPS 预留给任务关键型和需要高单位卷吞吐量的应用程序。

### 2。删除未连接的卷

即使在 EC2 实例停止后，该卷仍然存在，并且只要 EBS 连接到 EC2 实例，就会产生费用。这种持久性有利于保留数据，但最终您可能会为未使用的存储付费。

降低成本的一个简单方法是删除未连接的 EBS 卷。当它们被标记为“可用”时，它们不能接受任何流量，因此您不能使用它们。只有在确认不需要孤立卷中的数据后，才应该删除该卷。一个好的选择是拍摄 EBS 卷的快照，然后删除它。

您可以使用 [AWS 快照进行灾难恢复](https://n2ws.com/blog/aws-cloud/how-to-create-a-disaster-recovery-plan-for-aws)。它们是压缩数据的好工具，而且由于托管在亚马逊 S3，价格更低。这允许您存储不需要经常访问的冷数据，但是如果需要，您可以使用这些数据来恢复 EBS 卷。

### 3。识别空闲卷

消除了未连接的卷后，您需要注意那些仍然连接但没有做任何事情的卷，它们会产生不必要的费用。要发现这些空闲卷，一个好的技巧是查看卷吞吐量和 IOPS。如果该卷在一段时间内没有任何流量或磁盘操作，则该卷不在使用中，可以被删除。

### 4。标记所有内容

标记是有用的工具，它允许您在数据库中定位高成本区域。
通过将标签应用到 EBS 卷，您可以使用元数据搜索、管理和过滤资源。

此外，您可以使用标记来组织和编辑资源，在 AWS 控制台中形成组。

### 5。管理快照

AWS 快照是 EBS 卷中数据的副本。由于快照比活动 EBS 卷便宜，因此在终止未连接的卷之前，快照是一种简单的备份方法。

然而，快照有时会过时。一个很好的经验法则是设置一段时间，过了这段时间快照就不再相关了。之后，您可以设置系统定期删除旧快照，同时决定每个卷保留多少快照。

## 底线

虽然一开始管理 AWS 平台的成本看起来有些棘手，但归根结底还是要定期监控您的使用情况。关键是识别未使用的空间并重新组织它。有了这些技巧，您可以最大限度地降低成本，优化平台性能。
