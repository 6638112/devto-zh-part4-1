# 你的代码有多“环保”？

> 原文:[https://dev.to/protium/discuss-how-green-is-your-code-3fle](https://dev.to/protium/discuss-how-green-is-your-code-3fle)

大家好。

我经常思考用电的问题。运行的服务和服务器从不停止(抛开按需实例，但它们也在服务器中)。

从用户导航的那一刻起，很多事情就发生了:服务器、互联网提供商、海底电缆、卫星、GSM、电力和更多的电力。

我们如何在日常工作中帮助我们的星球？或许:

*   **高效算法:**如果用户拥有 16g 内存和四核 CPU，这并不意味着我们可以使用所有可用的资源
*   **轻量资源:**让用户下载大量资源会消耗更多数据，从而消耗更多电池
*   **可配置的后台服务:**为什么不给用户提供设置来调整他/她需要更新数据的频率？
*   **颜色:**不要使用让用户增加屏幕亮度的颜色
*   ...

也许我说得太过了，但是你们中有谁在写代码的时候会想到这一点吗？这能激励你写出更好的代码吗？有可能写出节能代码吗？
