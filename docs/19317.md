## LWC 开源

我更感兴趣的是 Lightning Web 组件的开源。你可以在 LWC.dev 上查看它的文档。所以我们可以使用 lightning Web 组件路径平台。因此，我们的技能将转移到，如果你已经开发了闪电网页组件。，你的技能将转移到制作网站，而不是在 Salesforce，这是一个奖金。

## LWC 本地开发

他们还宣布了当地的发展。它将使用本地 Express 服务器提供 Lightning Web 组件，并使用代理服务器与 Salesforce 通信，从您的组织获取实时数据。这将是一个很大的奖励，当你进行开发变更，需要快速迭代设计，获得快速反馈。这比保存组件，等待页面刷新，然后才能真正看到它要好。因为在本地开发中，你也可以热重装。这样你就不必在保存文件时按刷新键了。它会自动更新，你也会变得更漂亮，不是像在格式中那样漂亮，而是在网络浏览器中更好看的错误代码。

它会自动解决你的依赖。所以静态资源，你不需要弄清楚静态资源是什么，它会自己做出决定。

测试版将于今年夏天推出。

路线图上还将有，他们正在考虑为离线或本地提供 Flexipage 渲染支持，如果他们在那里，他们将考虑模拟数据，比如记录来自您组织的数据或提供样本数据，类似于您组织中数据的树导出。很明显，他们想要你的反馈。所以，如果你去，我会看看我是否能找到一个链接，包括下面你可以提供反馈的地方。这就是我感兴趣的两件主要事情，本地开发和开源以及 Lightning Web 组件。
