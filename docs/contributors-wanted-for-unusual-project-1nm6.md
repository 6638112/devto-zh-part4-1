# 不寻常的项目需要贡献者。

> 原文：<https://dev.to/johnfound/contributors-wanted-for-unusual-project-1nm6>

我正在为我的项目 [AsmBB](https://board.asm32.info) 寻找贡献者。

## 这个项目是关于什么的？

**AsmBB** 是一个论坛引擎，用汇编语言编写。它使用 FastCGI 作为服务器协议，使用 SQLite 作为数据库引擎。

此外，该项目的 FastCGI 框架可以用于其他汇编语言的 web 项目。

该项目的主要目标是:

1.  **性能** -后端最终是快速和轻量级的。它可以处理大量的游客同时没有减速或故障，即使在非常小和便宜的 VPS 或共享主机帐户。

2.  **几乎没有外部依赖性** -该引擎只需要一个工作正常的 web 服务器和 Linux 操作系统。它不需要安装任何其他库或特殊的服务器配置。

3.  国际化-引擎有翻译成不同语言的用户界面，每个用户都可以设置自己喜欢的语言。

4.  **易于定制** -引擎同时支持多种皮肤，用户可以根据自己的喜好更改皮肤。此外，自动切换到移动皮肤是可用的。

    皮肤完全基于文本模板和 css/less 文件，可以很容易地编辑和创建，而不需要弄乱后端代码。

5.  **安全性** -与普遍接受的观点相反，这款发动机非常安全。抽象层的数量少和缺少依赖关系大大减少了可能的攻击媒介。

## 怎么了？

**前端和设计！**

尽管引擎支持多种皮肤，我也创建了其中的几种，但我的前端和设计师技能非常低，大多数访问者都抱怨前端不是 2019 年需要的样子。

## 贡献者出来了吗？

因此，我正在寻找贡献者，把论坛的前端应该是在 2019 年。

创建新皮肤和修复现有皮肤都是可能的，也是可取的。

但与此同时，以一种最轻量级的方式来制作它是很好的，因为，嗯，一个自称是世界上最快的论坛的网站，也应该在浏览器端保持最快。

保持论坛在没有 JS 的情况下工作也是很好的，至少在可能的地方。(比如实时聊天，没有 JS 是不可能实现的)。

当然，如果有人想为后端做出贡献，他也是受欢迎的。有许多功能需要在那里实现。

顺便说一下，这是一个在学校问题之外完善你的汇编语言技能的好方法:“在寄存器 ax 和 bx 中给出两个数字，在 cx 中计算总和”。

还有，您的 SQLite 查询优化技能。

## 什么技术？

在后端项目使用**[【FASM】](https://flatassembler.net)T3】汇编语言。还有一些 **C 代码**以未修改的形式使用:SQLite 和 MUSL C 库。**

前端有一个**模板引擎**、 **CSS** 和**香草 JS** 。

使用的版本控制是 **[Fossil scm](https://fossil-scm.org/)** 这是小项目最好的 DVCS。

因此，贡献者将有机会使用并不广泛的技术堆栈。

对于一个好奇的开发人员来说，这可能是一次**有趣的旅行**,他想尝试这些技术并寻找可能的最佳解决方案。
