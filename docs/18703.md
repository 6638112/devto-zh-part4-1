但是在深入“视情况而定”的兔子洞之前，我认为有必要检查一下我们是否问对了问题。

比较 Kotlin 与 Flutter 在许多方面都是将苹果与橙子进行比较。

[***Flutter 是一个完整的跨平台 UI 工具包。***](https://flutter.dev/)

因为 Kotlin vs Flutter 的分歧如此之大，所以原问题的范围相当大；这使得在没有首先探究各种其他问题的情况下很难回答。

在 Kotlin 和 Flutter 之间的选择将归结为一些更具体的问题。

## [](#why-do-you-want-to-learn-either-kotlin-or-flutter)为什么要学 Kotlin 或者 Flutter？

首先要问自己的一个问题可能是“为什么你想开始学习 Kotlin 或 Flutter？”

*   想学移动开发吗？
*   你想找一份移动开发者的工作吗？
*   你想作为一个单独的开发者或小团队来开发自己的产品吗？

### [](#do-you-want-to-learn-mobile-development)想学移动开发吗？

如果你的主要目标是简单地涉足移动开发，或者建立一个小的业余爱好项目，那么在 Kotlin 和 Flutter 之间的选择可能没有太大关系。

然而，重要的是要记住，你不只是在 Flutter 和 Kotlin 之间进行选择。你在选择:

*   颤动/飞镖
*   iOS/Swift/Objective-C
*   Android/Kotlin/Java

Flutter 是一个完整的应用开发工具包。这包括使用 Dart 作为编程语言，一组新的 UI 组件，一组独特的编程模式和最佳实践，以及它自己的调试工具和工作流。

为了公平地将 Flutter 与其他平台工具包进行比较，您需要考虑您正在投资的整个生态系统。

### [](#kotlin-vs-flutter-to-get-a-job-as-a-mobile-developer)Kotlin vs Flutter 找一份移动开发者的工作？

如果你的主要目标是找一份移动开发人员的工作，那么考虑对不同技能的需求可能是个好主意。

在 LinkedIn 上快速搜索与 Kotlin 和 Flutter 相关的工作，会出现以下结果(在美国)

*   颤振:315 个结果
*   科特林:3342 个结果

Flutter 仍然是一项新技术，与在 Android 项目中引入 Kotlin 相比，它代表了移动开发过程中更大的变化。因为这些事情，很少有提到 Flutter 的工作。

这种工作机会的差异可能意味着两件事

1.  在你所在的地区找到一份令人激动的工作可能会更难
2.  如果你很了解 Flutter 的话，在某个地方迈出第一步可能会更容易

这些有点矛盾，最适合你的可能取决于你目前的生活状况，你需要多快找到一份工作，以及你有多愿意为这份工作重新安置。

另一方面，对原生 Android 开发人员的需求仍然很大，许多工作都在寻找知道或愿意学习 Kotlin 的人。更多的工作意味着你有更多的选择，但也可能带来更大的竞争。

### [](#kotlin-vs-flutter-to-develop-your-own-product-as-a-solo-dev-or-small-team)Kotlin vs Flutter 以 solo dev 或小团队的身份开发自己的产品？

如果你正在 Kotlin 和 Flutter 之间做决定，因为你想建立自己的应用程序，并将其转化为业务，那么后退一步并考虑整个画面是一个好主意。

如果你想开发一个产品，那么你很可能会想为 iOS 和 Android 开发。

你可以用 Flutter 做到这一点，但不能单独用 Kotlin 做到这一点。

如果您想使用 Kotlin 进行跨平台开发，您需要熟悉原生 iOS 和 Android 框架。你可以构建一个 Kotlin 多平台项目并通过 Kotlin 共享代码，或者你可以构建两个完全独立的应用程序。不管怎样，你都需要了解 iOS 和 Android 的原生开发。

所以，如果你想同时为多个平台开发一些东西，Flutter 可能是一个不错的选择。我们将在下一节更详细地讨论这个问题。

## [](#what-do-you-want-to-build)你想造什么？

可能与*为什么*你想学 Kotlin 或 Flutter 有关的是你想造什么？

*   你想建立一个很棒的安卓应用吗？
*   你想建立一个很棒的 iOS 应用吗？
*   要不要同时打造一个 iOS 和 Android 的手机 app？
*   你想为移动之外的平台构建吗？
*   你想写你自己的后端服务吗？

### [](#do-you-want-to-build-an-awesome-android-app)要不要打造一个牛逼的安卓 app？

如果你的主要目标是构建一个独立的 Android 应用程序，拥有 Android 提供的最新和最强大的功能，那么选择原生 Android。

这不一定意味着原生 Android + Kotlin。Android + Java 非常适合构建高质量的应用。然而，我确实相信学习 Kotlin 会使开发体验更加愉快。

### [](#do-you-want-to-build-an-awesome-ios-app)要不要打造一个牛逼的 iOS app？

同样，如果你想开发一个拥有所有最新功能的独立 iOS 应用程序，那么我建议使用 Swift 进行原生 iOS 开发。

### [](#do-you-want-to-build-for-both-ios-and-android-at-the-same-time)要不要同时为 iOS 和 Android 打造？

如果你想同时为两个移动平台构建，这个问题就有点有趣了。你有三个主要的选择，包括科特林和颤振。

1.  构建原生 Android 应用和原生 iOS 应用
2.  构建两个本地应用，但是使用 Kotlin 多平台来共享代码
3.  打造一款 Flutter 应用

#### [](#building-2-separate-native-apps)构建 2 个独立的原生应用

这个选项需要你或你的团队在没有任何共享代码的情况下从头开始构建这两个应用程序。如果你是一个单独的开发者，这可能是最慢的选择，除非你已经精通 iOS 和 Android 开发。

这也可能是最安全的选择。两个平台的原生开发都非常成熟。苹果或谷歌不太可能突然放弃任何一个平台，你会尽快获得新功能。

#### [](#build-2-separate-native-apps-with-kotlin-multiplatform)使用 Kotlin 多平台构建 2 个独立的原生应用

如果您想在本地构建应用程序，但又想尽可能多地共享代码，您可以选择学习 Kotlin 并构建一个 Kotlin 多平台项目。

对于缺乏经验的开发人员来说，这可能不是最好的选择。Kotlin 多平台生态系统非常新，您可能会发现自己现有的工具/库很少。

#### [](#build-a-flutter-app)打造一款颤振 app

构建一个 Flutter 应用程序可能是让一个应用程序同时在 iOS 和 Android 上运行的最快方法；尤其是如果你是一个单独的开发者。

Flutter 的开发时间非常快，这在很大程度上要归功于它强大的热重载功能。您还可以为两个平台利用一个单一的代码库。只要您不需要对系统 API 进行真正的低级访问，或者对每个平台的外观/感觉区别对待，您就不需要为特定于平台的需求编写大量定制代码。

如果你想快速地为两个平台建立一个应用程序，尤其是一次性的应用程序，Flutter 可能是个不错的选择。

### [](#do-you-want-to-build-for-platforms-beyond-mobile)你想为移动之外的平台打造吗？

Flutter 团队继续致力于使 Flutter 对桌面和 web 开发都可行。虽然这项功能仍在开发中，但有可能将您的单一 Flutter 代码库用于其他平台。

您也可以考虑将 Kotlin 多平台项目扩展到移动领域之外，但是价值主张不同。使用 KMP，你可以共享代码，但是仍然需要编写本地平台代码来创建你的用户界面和利用共享的代码。随着您添加更多的目标平台，这将增加所需的工作/技能组合。

### [](#will-you-want-to-write-your-own-backend-services)你会想自己写后端服务吗？

要考虑的一个边缘情况是，您是否想要为您的项目编写自己的后端服务。

Kotlin 可以用来编写后端代码，而 [Ktor](https://ktor.io/) 可以让这变得相当容易。因此，如果除了某种移动应用程序之外，你还打算开发后端服务，Kotlin 可能是一个不错的投资，因为你可以跨多个领域重用该技能。

## [](#do-you-already-have-development-experience)你已经有开发经验了吗？

另一件要考虑的事情是你当前的开发经验水平，因为它适用于你学习 Kotlin 或 Flutter 的主要动机。

*   你已经是安卓开发者了吗？
*   你已经是 iOS 开发者了吗？
*   你是编程新手吗？

### [](#already-an-android-developer)已经是安卓开发者了？

如果你已经习惯了原生 Android 开发，那么学习 Kotlin 将会比学习 Flutter 快得多。了解 Kotlin 只会增加你现有的知识和技能，也许会让你在构建原生 Android 应用时更有效率。

学习 Kotlin 还可以让您用 Java 之外的另一种语言编写后端服务，并使您能够使用 Ktor 来简化这个过程。

学习 Flutter 将需要学习一门新的语言(Dart)以及一个新的开发生态系统。所需的时间很可能比学习科特林语要多几倍。

如果你不想开始利用 Flutter 的跨平台优势，那么只为 Android 学习 Flutter 没有太大意义。

### [](#already-an-ios-developer)已经是 iOS 开发者了？

如果你是一名 iOS 开发者，想要学习 Flutter 或 Kotlin，情况会有所不同。

如果你想继续构建原生 iOS 应用，那么 Kotlin 和 Flutter 都没有太大的好处。Kotlin 至少会给你另一种语言来编写后端代码。

如果你是一个想要开始编写跨平台应用程序的 iOS 开发人员，那么你可以走 Kotlin 多平台项目或 Flutter 项目的道路。

就像我们之前讨论过的，Flutter 将成为两个移动平台的独立开发体验，而 KMP 项目将需要 Kotlin 和原生开发栈的知识。

所以在这种情况下，无论哪种方式，你都在学习一门新语言(Dart 或 Kotlin)和一个新框架(Flutter 或 Android)。

### [](#are-you-new-to-programming)你是编程新手吗？

如果你对编程完全陌生，那么你可能需要后退一步，考虑一下 Flutter 与原生 Android 开发的比较。

如果不将 Kotlin 应用于移动或后端开发，学习 kot Lin 本身并没有多大用处。

因此，如果你想开发移动应用程序，那么你会想要在 Flutter 和原生 Android 开发之间进行比较选择。

## [](#how-do-you-learn)你是怎么学习的？

怎么学最好？对于 Kotlin 或 Flutter 来说，在开始新的学习之路之前，这可能是一个重要的问题。

*   寻找深度指导课程？
*   想把学习作为大学课程的一部分吗？
*   你会向经验丰富的兽医寻求建议和最佳实践吗？
*   你经常看博客文章吗？
*   你喜欢边做边学吗？

### [](#looking-for-indepth-guided-courses)寻找深度指导课程？

如果你喜欢学习详细的、有指导的课程，你可能会在本地开发世界中有更多的选择。

因为 Flutter 是一项新技术，目前还没有太多的资源。

公平地说，也没有太多专门针对 Kotlin 的课程，但是 Kotlin 是一个很小的概念。如果学习 Kotlin，很可能是在 Android 开发的背景下，有太多的课程可供选择。

### [](#want-to-learn-as-part-of-your-university-curriculum)想把学习作为你大学课程的一部分？

我和一些对在大学学习原生 Android 开发感兴趣的人谈过。如果你有这个机会，我建议你抓住它。当你从学校出来时，你很可能在找工作。拥有像 Android 开发这样的适销对路的技能是非常好的，你不应该担心 native 是否过时，你应该学习 Flutter。

先找份工作，然后在有人付钱给你的时候，根据需要提高技能。

### 你会向经验丰富的兽医寻求建议和最佳实践吗？

围绕 Android、Kotlin、iOS 和 Flutter 的开发者社区是学习、指导和解决问题的绝佳资源。

当你想找人追随或寻求指导时，你可能会在原生的 Android/iOS 世界中过得更轻松。再说一次，这是因为 Flutter 是一项较新的技术，只是还没有多少人使用它。

然而，这并不是说你没有选择。有许多优秀的开发人员在 Flutter 中工作，谷歌的 devrel 团队正在通过 T2 GDE 项目(T3)努力认可这些开发人员。

无论是哪种情况，这里都有一些关于 Kotlin 和 Flutter 信息的开发人员列表:

### [](#do-you-regularly-turn-to-blog-posts)你会定期翻博文吗？

学习任何新技术时，博客文章都是一个很好的资源。当想到 Kotlin vs Flutter 时，有几件事需要记住。

Flutter 相对较新，所以现有的资源较少。此外，随着 it 的快速发展，其中一些资源可能已经过时。

Kotlin 可能有更多的可用资源，但也在快速发展，关于高级主题的帖子可能在几个月后就过时了。

如果研究 Kotlin 在 Android 和/或 iOS 开发中的应用，你会有好几年的帖子需要筛选。其中许多将仍然相关，但其他的不会。因此，当您找到这些资源时，您必须确定它们的准确性和更新性。

值得庆幸的是，我认为无论你选择 Kotlin 还是 Flutter，开发者社区和 devrel 团队都将继续通过撰写博客文章、构建代码实验室和录制视频教程来帮助我们。

### [](#do-you-prefer-to-learn-by-doing)你更喜欢边做边学？

如果你是那种喜欢一头扎进去做点东西来学习的人，那么你很幸运。Kotlin 和 Flutter 都非常容易尝试。

## [](#have-you-considered-the-risks-of-choosing-kotlin-vs-flutter)你考虑过选择科特林 vs 颤振的风险吗？

采用任何新技术都有风险。在选择 Kotlin 和 Flutter 之前，考虑一下这些风险可能是值得的。

*   采用 Kotlin 或 Flutter 的组织风险有多大？
*   在 Kotlin 和 Flutter 之间的个人选择有什么风险吗？
*   是否存在失去长期支持的风险？

如果考虑在您的组织中采用 Kotlin 或 Flutter，值得停下来考虑相关的风险。

Kotlin 已经稳定，并完全支持 Android 年了。它可以一次集成一个文件，并将与现有的代码很好地工作。将它添加到您现有的项目中风险很低，影响也很小。

Flutter 的风险要大得多，因为它背离了原生开发体验。它没有成功项目的长期记录。熟悉该技术的开发人员越来越少，这使得招聘变得更加困难，并要求您现有的团队学习新的框架/工具/语言。Flutter 在绿地应用中工作得最好，这使得它很难逐步采用。

这些都是需要考虑的重要问题。

此外，谷歌有一个很少通知就取消项目的记录，所以可能会有一些小的担心，谷歌会放弃 Flutter 或 Kotlin。

实际上，我认为谷歌至少在未来 2 年多的时间里会继续支持这两种技术，所以如果你现在考虑 Kotlin 和 Flutter，我不会太担心它们中的任何一个会消失。
