# 网络开发者能从阿波罗 11 号中学到什么

> 原文:[https://dev . to/Steve blue/what-web-developers-can-learn-from-Apollo-11-h6k](https://dev.to/steveblue/what-web-developers-can-learn-from-apollo-11-h6k)

通过我父亲的视角看阿波罗 11 号是一次令人惊奇的经历，我父亲在过去的周末参与了这项任务，所以我想我应该与 dev 社区的其他人分享一些收获。

对于一些人来说，50 年前发生的如此重大的事件与我们作为网络工程师的工作之间的关系似乎并不明显。那是一个非常不同的时代，那时美国的工程师不在敏捷团队中工作。那时没有互联网。据我们所知，那时甚至没有电脑。然而，我们可以从阿波罗 11 号的成功中学到很多东西。

# [](#mission)使命

如果你正在做的项目没有明确的目标，它可能注定会失败。需要一个所有人都支持的使命。设定明确的目标意味着每个人都在同一页上。在设计应用程序的各个部分时，拥有项目路线图也非常有帮助。你可以预见未来的挑战，避免技术债务。建立一个人们能够信任并及时执行的使命是至关重要的。

# [](#teamwork)团队合作

即使在我父亲工作的 NASA 周围的承包公司里，也有一种感觉，你是任务的一部分，你是“整体团队”的一部分。对大多数人来说，感觉自己是某件事情的一部分，为项目做出有意义的贡献是很重要的。管理者需要授权给单个贡献者来做决定。单个贡献者应该尽可能地为团队增加价值。在大多数情况下，巨大的自我会成为阻碍。团队合作对成功至关重要。

# [](#visibility)能见度

许多在阿波罗 11 号上工作的工程师觉得他们正在执行“一项显而易见的、符合国家利益的艰巨任务”。根据我的经验，当项目有数百万用户时，即使是最平凡的任务也会变得非常有价值。工程学主要是解决问题，对一些人来说，从事一项艰巨的工作是一种回报。当两者结合时，它可能是一个成功的组合。移动指针是极其困难的，但也是极其有益的。不要满足于平凡的事物，作为一名工程师，你要不断挑战自己。

# [](#can-do-it-attitude)能做到的态度

老一辈中有许多人悲叹美国丧失了“我能行”的态度。我强调，在与许多工程师一起工作后，他们没有模型就不能开始一项任务，当后端没有准备好时，他们就停止在前端工作，或者责怪流程阻碍了他们的进展。为了一个成功的网站开发，你需要团队成员去做。没有借口。如果这意味着为前端引导一个节点中间件来发出模拟请求，直到后端准备好。去做吧。如果这意味着与设计师密切合作，为设计系统提供稳定的 CSS 样式。去做吧。尽一切努力完成任务。这是一个有“能行”态度的人的标志。

# [](#learning-from-failure)从失败中学习

1967 年 1 月 27 日，三名宇航员(爱德华·h·怀特二世、维吉尔·I·“格斯”·格里森和罗杰·b·查菲)在参加阿波罗 1 号发射模拟训练时死于一场可怕的火灾事故。引起火灾的问题本来是可以避免的。这场灾难随后促使阿波罗指令舱的重大设计变更，这对阿波罗 11 号的成功起到了巨大的作用。

有时我们无法预见我们编写的代码会出现的问题。我们可以尽我们所能防止同样的错误发生两次。从错误中学习是我们作为人和工程师的发展方式。我们只有迈出下一步才会变得更好。如果你幸运的话，下一步将是人类的又一次飞跃。
