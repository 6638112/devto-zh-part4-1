# 我在学习编码时犯的 5 个错误

> 原文：<https://dev.to/jameesy/5-mistakes-i-made-whilst-learning-to-code-3l75>

事后回想总是一件美好的事情。这里是我认为我个人在学习编码时犯的五个最大的错误，这样你就不会落入同样的陷阱！

🌱在开始之前没有进行适当的研究——首先，不清楚我对哪种技术感兴趣，是一个错误。我不明白这些语言都做什么，我不知道什么是“全栈”或“前端”开发人员，因此无法选择我想学的东西。这意味着我花了几个月的时间才意识到 web 开发和 Javascript 对我来说是浪费了宝贵的时间和精力。

🚀没有一个明确的目标或计划——当我开始时，我只是想“擅长编码”。我最初没有一个努力的目标，现在我意识到，确定这些努力的目标极大地提高了我的生产力。

👨‍🎓课程交换——在任何看起来有趣的课程之间进行交换在当时看起来很有趣，但这意味着我在重新学习我已经学到的东西，而从来没有将它们应用到现实世界的应用程序中并加以实践。我总是说你不能通过看乐谱来学习演奏一种乐器，你需要实践经验和一遍又一遍的练习。

😧跳过难懂的概念——这不需要任何解释，但我会跳过或掩饰难懂的概念。花三个月的时间学习一个概念从长远来看对你更好，即使看起来你的进度已经慢下来了！我还发现，如果不理解至少能够解释一些 web 开发的核心概念，那会让你吃不了兜着走。最初，我很乐意说“我只是一个实干家”。我可以实现事情，但不能解释它是如何或为什么工作的。”我至今还在为此纠结！

📈拿自己和别人比较——我学得相当慢，所以拿自己和别人比较是一个错误，因为我的信心经常受到打击！我们都以不同的速度学习，所以试着不要和其他人一起追踪你的进步，因为这可能会以失败告终！
