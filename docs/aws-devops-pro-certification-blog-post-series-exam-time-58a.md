## 我希望我事先知道的事情

我应该做的最主要也是最明显的事情是将我的知识集中在构建基础设施和利用 AWS 服务的最佳实践上。

大多数问题的格式与[样本考试](https://d1.awsstatic.com/training-and-certification/docs-devops-pro/AWS%20Certified%20DevOps%20Engineer%20-%20Professional_Sample%20Questions.pdf)和模拟考试相似。他们希望你挑选出满足客户需求的最佳答案。

有太多的服务我没有实践经验，这意味着我花了很多时间试图对每个服务做什么和它们的用例有一个基本的了解。事实证明，我需要更深入地了解我得分低的领域。

我认为唯一让我感到惊讶的领域是 SDLC automation，因为我每天都做很多 CI/CD，尽管这往往是围绕非 AWS 服务:Circle CI、Travis CI 和 Azure DevOps。

那么，如何才能发现最佳实践是什么呢？大概有两个关键文件:

*   AWS [白皮书](https://aws.amazon.com/whitepapers/)，通常包含结合了各种 AWS 产品和服务的解决方案
*   服务的用户或开发者指南

我在重考前的学习将集中于此。我只是在考试前一周浏览了一下白皮书，但这次我打算做更多的笔记，并试图记住所提供的解决方案的图表。

## 应试技巧

### 检查你的齿轮

考试中心会要求你把所有物品放在一个储物柜里。

有些中心允许你带几杯水来参加考试，如果你能带的话，就带两杯。

还会给你纸和笔，或者类似的东西来做笔记。开始考试前检查笔的工作情况！

考试前排空你的膀胱，有些地方要求你引起监考老师的注意，这会耗费你宝贵的考试时间。

### 计时

开始习惯每隔 2 分钟扫描一次问题。这是你应该花在阅读和回答一个问题上的最大限度。

在一个完美的场景中，你可以完整地回答每个问题，不跳过任何一个，这将很容易知道接下来的 2 分钟间隔。在现实中，你可能会在不到 2 分钟的时间内跳过或回答一个问题，所以尽管这对大多数人来说可能是显而易见的，但我还是使用了以下方法来保持自己的正轨:当我开始下一个问题时，我在考试倒计时定时器上记下了当前时间，并计算出我必须回答这个问题的时间，因此如果定时器是 1 小时 7 分钟，我就需要在 1 小时 5 分钟或之前回答这个问题。

### 跳过且不抖动

除非你对自己的能力特别有信心，否则你可能会陷入盲目的恐慌(就像我一样)。给自己 30 秒(如果你能在更短的时间内完成就更好了)来浏览这个问题，如果你甚至不知道从哪里开始，那就跳过它。这些时间最好花在你对答案有模糊概念的问题上。

跳过问题是可以的，当你做完所有的问题后，你会发现还有时间重温这些问题。

如果你发现你跳过了很多问题，也不要惊慌，这可能是你的紧张。
