# 整理工具

> 原文：<https://dev.to/arleneandrews/sorting-the-tools-35c3>

最近在 Twitter 上，有一个关于你最喜欢的工具是什么的问题。我不仅对回答的数量(以及我获得的关于建议的信息)感到高兴，而且对人们在回答之前所做的思考感到高兴。我怀疑他们是在专门寻找测试工具，但是我的答案几乎适用于我生活的所有领域。

选择工具——无论是您编写的东西、正则表达式、定位器还是商业产品——都非常依赖于您需要它做什么，以至于一般的建议对其他领域的人来说可能是无用的。即使在数据库的世界里——它似乎一直存在，并且所有的选项都是众所周知的——在关系数据库和文档数据库以及如何最好地利用它们来获取所需信息方面仍然存在很大的分歧。

为一个全新的项目选择几乎任何类型的工具都是一件令人沮丧的事情。

值得庆幸的是，几乎没有全新的想法——你可以看看比赛的当前版本，并获得一些你将能够使用工具做什么的想法。从许多来源(包括那些你可能不同意的来源)看一看潜在的不久的将来，将有助于增加要考虑的事情的列表。使用上面的数据库示例:数据库需要存储来自产品的 VR 或 AR 版本的信息吗？如果是——为了允许访问这些版本的新旧数据，可以添加或删除什么？

正如我所选的图片所暗示的，合适的工具可能不适用于所有类型的问题。是的，您选择的工具可能非常好，但是您的项目可能已经超出了它的范围，或者现在一个不同的工具是一个好的选择。工具已经成长和改变——并且有了以前不存在的竞争。因此，需要投入时间来查看当前的工具箱是否需要升级。

这不仅增加了查看选项的时间，还增加了改变系统的潜在时间，这可能不是一个受欢迎的想法。回头去改变目前运行良好的事情从来都不是一件令人愉快的事情——即使你知道做出这些改变是一件好事。增加错误的可能性也增加了，因为它并不总是带来新的信息或新的价值。然后检查这些变化——当你身处其中时，这似乎是一个无止境的、有点吃力不讨好的任务。

我对最喜欢的工具的回答是“笔记本和时间”。如果你能每周计划一点时间，从你使用的、可信的各种来源做笔记，你就能有一个好的开始。当需要考虑一个新的选择时，你应该了解那里有什么，以及每件事会产生或解决什么问题。这将允许您继续向工具箱中添加最佳选择。
