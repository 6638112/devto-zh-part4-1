# 采访你的面试官

> 原文：<https://dev.to/kcarrel/interview-your-interviewer-2jc1>

作为我上一篇关于[技术面试方法](https://dev.to/kcarrel/an-approach-to-technical-interviews-3l66)的文章的延续，我想我也将分享一些我最喜欢的公司/文化问题，以及我想问面试官的潜在后续问题。记住面试是双向的——你也在面试这家公司，看看它是否适合你。这家公司会支持你的目标和成长吗？

## 面试前研究

1.  仔细阅读公司信息——他们有关于我们的页面、文化页面或公司博客吗？不要引入任何通过快速搜索不容易回答的问题。
2.  重新阅读你所申请的职位的描述——能够预见到该职位特有的问题可以帮助你对自己的回答充满信心。
3.  如果你有面试官的名字，在 LinkedIn 上查看一下！这可以为你提供与他们在组织中的角色相关的谈话要点或背景，以及根据任期应该问什么样的问题。
4.  查看 glass door/员工评论——虽然我总是建议你对所有事情都要有所保留，但这也能让你的面试官了解你的问题。

## 定位问题

1.  团队正在使用的技术是什么？
    *   这个堆栈是否曾经被使用过，或者是否有尝试新语言/框架的灵活性？
    *   当前技术堆栈背后的决策过程是怎样的？
2.  这个角色所属的团队有多大？
    *   团队有多多样化？
    *   团队内部的资历分布如何？
    *   你认为团队目前面临的最大挑战是什么？
3.  作为开发人员，公司/团队是否支持正式的成长(例如:证书、培训或参加会议)？
4.  开发人员是否有一个正式的流程来接收反馈(例如:代码评审、角色评审或有经理参与的定期评审)？
    *   这个角色的 3 个月目标是什么？6 个月？
    *   这个角色的成功标志是什么？
5.  这个团队使用什么开发方法？(例如:敏捷/看板/等等)
    *   是否利用了成对编程？
6.  这个职位典型的一天是怎样的？
7.  这个职位的员工入职的方法是什么？
    *   他们会是一个与资深员工结对的机会吗？
8.  在[X 公司]，文化对你来说意味着什么？
9.  如果员工想参与业务的不同方面，他们可以在团队之间调动吗？
10.  面试候选人时，你的目标是什么？
    *   你如何评价自己实现目标的情况？
11.  这个角色的日程安排是怎样的？
    *   这个角色有随叫随到的职责吗？
12.  开发者离开这家公司最常见的原因是什么？
13.  公司/团队如何参与开发者社区？
    *   他们主持本地聚会团体吗？
    *   出席/参加会议？
    *   参加黑客马拉松？

请记住，这些只是我最喜欢问的几个问题，具体到我和我的价值观。我很想听听你在面试中使用的典型问题！
