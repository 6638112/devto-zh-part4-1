# 初级面试的 5 个快速技巧

> 原文:[https://dev . to/nataliedeweerd/5-初级面试快速提示-6j6](https://dev.to/nataliedeweerd/5-quick-tips-for-junior-level-interviews-6j6)

下面是我在面试初级开发人员时注意的一些事情。这显然不是一个全面的列表，但希望能让你对一些面试官在寻找什么有一点了解。

不管怎样，我在英国工作。

## [](#1-be-honest-about-what-you-know)1。对你所知道的要诚实。

不要在面试中撒谎。如果你不知道一些事情，诚实点。如果你有信心能很快学会，就说出来，并解释原因:

> “我还不知道任何 Javascript 框架，但我对普通的 Javascript 代码有很好的理解，所以相信我可以很快学会”。

通过承认你不明白的事情，它增加了你之前所说的一切的可信度。

## [](#2-dont-be-afraid-to-show-your-passion)2。不要害怕展示你的激情。

说说你的副业吧！如果你做了一个让你非常自豪的兼职项目，但是因为它很讨厌而对分享有点担心，无论如何告诉我！我是否理解引用并不重要，它显示了对编码的热情。

如果你充满激情，我不在乎你知道什么。我可以教你编码，但教不了你热情。

## [](#3-ask-questions)3。提问。

面试是双向的。我正在面试*你*看看你是否是这个角色的合适人选，就像你面试*我和公司*一样，以确保我们非常适合你。如果你接受了这个职位，你将会每天在公司里呆上将近 8 个小时——确保你对此感到满意！

也问问你将要加入的团队。它有多大？你会得到多少支持？社交活动频繁吗？

## [](#4-ask-more-questions)4。多提问！

问问题也适用于你可能遇到的任何“带回家”的问题。有不懂的问题，就问；不要问解决的方法。比如不要问:*“我怎么给一个数组排序？”*，而是*“您希望数组按名称排序，还是按总数排序？”*)。

这显示了正确理解问题的渴望，也显示了你的一点知识，因为你知道数组可以以不同的方式排序。

## [](#5-research-the-company)5。研究公司。

公平地说，这对于任何级别的面试都是正确的，但是看在上帝的份上，在你去面试之前做好你的调查。我不会试图让你犯错，或者问你我们最近的产品发布会是什么，但是如果你还没有看我们的网站，我会马上知道。

如果我被面试，我喜欢做的一件事就是找到公司最近做的一个很酷的项目，并在面试中提出来。
