# 区块链的逻辑组件

> 原文：<https://dev.to/ajayatgit/logical-components-in-blockchain-32fe>

本文指出并概述了区块链中的各种逻辑组件。

当人们第一眼看到区块链空间时，它看起来非常混乱和复杂，这是毫无疑问的，因为创建和实现区块链生态系统所涉及的低级编程和数学很困难。

在本文中，我将解释构成任何区块链生态系统的单个逻辑组件的用途。

要理解区块链技术的工作原理，不考虑其风格或应用，有必要理解区块链生态系统的逻辑组件及其角色。

区块链生态系统可以在逻辑上分为这 4 个逻辑组件。

*   节点应用程序
*   共享分类帐
*   一致性算法
*   虚拟计算机

1.  节点应用
    任何想要成为区块链生态系统一部分的机器，都需要运行特定区块链的一个节点。以 bitcion 为例，机器必须运行 Bitcion 钱包应用程序。

2.  共享账本
    把它当做一个数据结构，在节点应用内部。如果你运行的是以太坊客户端，你可以看到以太坊生态系统账本，并根据那个生态系统的规则(智能合约、支付等)进行交互。).如果您运行的是比特币客户端，您可以根据比特币节点应用程序的程序代码中规定的规则，参与比特币生态系统。
    您可以运行任意区块链生态系统的任意多个节点应用程序，但特定区块链生态系统将只有一个共享账本。

3.  共识算法
    该逻辑组件提供了在区块链生态系统中如何实现共享总账单一视图的“规则和规定”。不同的生态系统有不同的方法来达成共识，这取决于生态系统所需的期望特征。各种著名的共识算法是-工作证明、权威证明、利害关系证明、流逝时间证明。

4.  虚拟机
    这是区块链生态系统的最终逻辑组件。它充当一个桶或容器，所有逻辑组件都在其中休息、工作和交互。

虚拟机是由计算机程序创建的机器(真实的或虚拟的)的表示，并且用语言中包含的指令来操作。它是机器内部的机器的抽象。
