我曾经想过写一些关于如何开始 Haskell 开发的东西，但是认为没有人会感兴趣。

然后我看到了这个评论:

[![citizen428 profile image](../Images/ccda4f7ebb3a651cb0c336fed9926d1a.png) ](/citizen428) [ Michael Kohl ](/citizen428) • [<time datetime="2019-07-07T09:01:32Z"> Jul 7 '19 </time>](https://dev.to/citizen428/comment/cl77) 

> 我认为两个社区都有一个明确的位置

这是肯定的，但可惜现实世界中的哈斯克尔的书有点过时了(它是 2008 年的)，而且也很长(700 多页)。我想看到的是大约 300 页的东西，侧重于例如建立一个完整的网络应用程序或类似的东西。无论如何，我也不会写这些(我选择 F#作为我的函数式语言)，所以我没有资格抱怨。

这是普遍的感觉吗？我不认为我会写 300 页，但是建立一个简单的 Haskell 服务器正是我最近所做的，它有许多不同的部分，显示了该语言的不同特性。

Haskell 倾向于不从 web 服务器开始的原因是，为了制造一个 web 服务器，IO 单子是必需的，单子被认为是可怕的。我认为这是可以做到的，但话又说回来，我很擅长低估任何我觉得有趣的事情的复杂性。

如果你真的想读你写的这样一个系列，请用独角兽来回应，这样我可以估计对这个主题有多少兴趣。我会把红心算作“这很好，但不适合我”。
