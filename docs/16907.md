# 人们应该明智地选择他们的项目名称！

> 原文:[https://dev . to/prahladyeri/folks-should-choose-their-github-project-names-wisely-4pe 0](https://dev.to/prahladyeri/folks-should-choose-their-github-project-names-wisely-4pe0)

无论它是一个严肃的还是次要的项目，无论何时你决定开始一个新的开源项目，给项目命名是你需要做的决定之一。

虽然很多现代主义者可能不相信我说的话，但这个名字确实在决定一个项目的成败方面发挥了巨大的作用！尤其是如果成功依赖于大众的欢迎和采用，这在开源软件中是很常见的。

现在，我不是在谈论无神论者总是热衷于一笔勾销的一些“18 世纪的伪科学或迷信”的废话，我是在纯粹的心理学基础上谈论这个。一个事物的名字对你的心理有很深的影响。举例来说，你自己的名字对你来说就是你的身份，你下意识地把它和你自己联系在一起，每次听到它都会产生共鸣。

当你听到别人的名字时，也会发生类似的事情，只是这种联想是针对他们的(或者是他们的项目，如果是项目名的话)。这个名字就像你大脑中的一个记忆地址，它链接到你心灵数据库中那个人或事物的清晰想法。当你听到“苹果”这个名字时，你会根据上下文联想到某个想法(无论是水果还是小玩意)，同样的还有“谷歌”、“微软”等等。

人们同样会开始把你的项目的名字和你的项目联系起来，并通过“罪恶感联想”你自己！现在，你希望这种联系是什么样的:积极的联系还是消极的联系？它应该唤起优雅和幸福的感觉，还是应该提醒你痛苦和悲伤的事情？

考虑这个最近发布的名为 [*熵*](https://github.com/entropic-dev/entropic/) 的雄心勃勃的 github 项目。这应该是一个替代[*【NPM】*](https://www.npmjs.com/)的软件包管理器，这是一个值得称赞的努力，考虑到我们与 NPM 的问题(大量的依赖，不支持软件包签名，追求利润的公司负责，不理会用户，等等。).

但是考虑一下熵这个词的含义，这个项目的名字就是从这个词而来的。除了一个热力学相关的技术术语，除了理论物理专业的毕业生没有人会知道，在英语词典中，这个词还有第二个更流行的意思:

> 缺乏秩序或可预测性；逐渐衰退成无序。

如你所见，这是一个非常负面的意思。信不信由你，它确实会影响人们对你的项目的态度和看法，即使他们没有敏锐地意识到熵意味着什么(因为潜意识通常知道一切！)

尽管在正确的方向上做出了如此巨大的努力，但该项目自形成以来没有获得任何牵引力，尽管考虑到 NPM 的包装问题，它值得很多牵引力。

现在，我不会继续说是他们的项目名称导致了这一点，因为这显然只是其中一个因素，我甚至没有看过源代码。人们的昏睡态度，使这么大的切换到另一个包经理通常是主要原因！

也许再过几年，`entropic`可能真的会成为 JavaScript 世界里一个流行的 NPM 替代品，即使用的是这个名字，但我认为这种可能性很小。依我拙见，他们应该把这个名字改成一个更积极、更优雅的名字。
