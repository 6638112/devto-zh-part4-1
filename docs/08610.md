# 大 O 符号

> 原文:[https://dev.to/marcdwest32/big-o-notation-4ah0](https://dev.to/marcdwest32/big-o-notation-4ah0)

计算机科学中的时间复杂度是机器处理一个算法所花费的时间。具体来说，我们感兴趣的是处理时间与输入大小的关系。因为对于相同大小的不同输入，处理时间可能不同，所以考虑最坏情况时会考虑时间复杂度符号。由于这可能涉及一些非常复杂的数学计算，我们使用大 O 符号作为各种时间复杂度的简写。让我们来看看编程中常见的复杂类型，从最佳到最差的性能。

### [](#constant-time-o1)常数时间 O(1)

恒定时间意味着无论输入大小如何，算法运行所需的时间都将保持不变。常数时间算法的一个常见示例是在数组中的已知索引处查找值。

### [](#logarithmic-time-olog-n)对数时间 O(log n)

对数时间意味着随着输入大小的增加，处理时间也会增加，但速度会越来越慢。二叉查找树就是一个很好的例子。其原因是搜索域在每个搜索周期减少一半。

### [](#linear-time-on)线性时间 O(n)

线性时间意味着随着输入大小的增加，处理时间以一致的速率增加。线性时间算法的常见情况是可能必须在整个输入上循环的任何函数。许多高阶函数(如 map、filter 和 forEach)都是线性时间算法的示例。

### [](#quadratic-time-on2)二次时间 O(n^2)

二次时间是指处理时间以倍增的速度增长。常见的例子是在循环中嵌套循环。简而言之，随着输入大小的增加，处理时间会随着输入大小的增加而增加。

### [](#exponential-time-ocn)指数时间 O(c^n)

指数时间算法是指处理时间随着输入大小的增长而呈指数增长。一个常见的例子是暴力破解密码。这意味着随着输入大小的增加，处理时间会以某个常数的输入大小的幂增加。

还有许多其他类型的时间复杂度我们在这里不讨论，但是理解这些例子对于在你的程序中开发高性能算法是至关重要的。我们都应该努力改善应用程序的时间复杂度。
