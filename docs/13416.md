## [](#introduction)简介

Credits 日复一日地改进其技术，并使 head 对去中心化的未来抱有可信的信念。在现代 IT 市场取得成功的唯一方法是与精通技术的研究人员并肩工作，以弥补任何弱点。正因如此，Credits 团队发起了第一阶段的 Bug Bounty 战役。Credits 邀请所有感兴趣的开发人员和安全专家参与该计划。第一阶段旨在优化源代码，消除漏洞，提高平台的安全性。第一阶段的总奖金为 50 万美元。根据开发商的喜好，所有款项将以美元或 BTC/瑞士法郎/瑞士法郎支付。

## [](#investigating-and-reporting-bugs)调查和报告 bug

如果您发现了 bug，请通过在 Credits Github 上创建新问题来提交报告。请注意，您只能提交有关“范围内软件”中包含的平台组件的报告。

## [](#qualifying-vulnerabilities)合格漏洞

*   对于所有“范围内的软件”来说，都有不同程度的错误，这些错误会有不同的回报。
*   对于具有一个潜在根本原因的多个错误，可以应用一个修复程序进行补救，我们将把它视为一个漏洞，并且只奖励一次。
*   唯一第一个发现漏洞的开发者将获得奖励
*   开发者可以在 Github 上使用“Pull Request”提交已发现 bug 的修复程序。如果开发者的修正被认为是可行的，奖励金额将增加 3 倍。
