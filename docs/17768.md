# [](#introduction)简介

Typescript 是 Javascript 的类型化超集。因为它是一个*超集*，所以任何有效的 Javascript 都是有效的类型脚本。由于类型检查、接口、泛型...

Typescript 最近很受欢迎。在 Javascript 调查的最后一个[状态](https://2018.stateofjs.com/javascript-flavors/typescript)中，声称自己使用过 Typescript 并将再次使用它的用户比例从 2016 年的 20.6%上升到了 2018 年的 46.7%。

就我个人而言，我听说过 Typescript，但从未花时间去学习它。我最终从事一个已经在使用 Typescript 的项目只是时间问题。时候到了...

*注:以下想法仅为个人感受。我对打字稿的经验有限。我将很高兴收到来自更有经验的 Typescript 用户的反馈😉*

# [](#collection-of-thoughts)思想的集合

## [](#a-different-mindset)心态不同？

在花了一些时间使用 Typescript 之后，我觉得我的方法必须与普通的 Javascript 稍有不同。我突然不得不考虑变量的类型。

我承认我以前很少想到某些问题。这个变量会一直是一个字符串吗？ *这个函数要返回什么类型？*...

不得不考虑这些事情需要在我写代码之前做更多的准备，以及不同的心态。

## [](#selfdocumentation)自我文档化

编写的代码是 Typescript，感觉文档化程度更高，一旦我习惯了语法并知道我在读什么，我就觉得与其他人编写的代码更加一致。

显然，这高度依赖于开发者的努力。你可以用 Typescript 写出适当的垃圾，或者用普通的 Javascript 写出优雅可读的诗歌。

# [](#conclusion)结论

老实说，我仍然对 Typescript 的实用性感到矛盾。我很喜欢我目前所看到的，但我需要更多的实验才能有一个明确的观点。

我希望得到 Typescript 用户的反馈。

*   你从普通 Javascript 过渡的经历如何？
*   你注意到代码的质量更好了吗？
*   Typescript 是如何改变开发类型的？
*   你注意到在小团队和大团队中使用 Typescript 的区别了吗？
