你可能会管理内部团队。如果你擅长你所做的，那么你已经知道如何有效地管理一个团队。如果你的组织由几个团队组成，你就有团队领导。您定期与他们沟通，以获得进展的更新，解决问题，并在必要时提供资源。这种通信中的一些可以是数字的。一部分可能是本人。

## 管理海外团队和内部团队的区别

有一件事可能会改变，那就是你将要使用的数字工具。能够走到大厅里和某人谈论一个重要的细节是一回事。例如，当你考虑如何管理乌克兰的离岸团队时，那就是另一回事了。

仅拿会议方面来说。在内部，你和你的团队领导坐下来，或者和整个团队开会。现在突然之间，你需要管理位于半个地球之外的乌克兰的离岸开发团队。会议将如何进行？

这只是你需要团队管理工具的一个例子。以下是管理远程团队的其他工具列表，您可能会发现这些工具非常有用。甚至 [如果你是一个非技术人员](https://www.romexsoft.com/blog/how-to-manage-a-tech-team/) ，他们将会是用户友好的，并且将会使每个人保持在点上。这些绝不是用于所述目的的唯一工具。但是它们很结实，值得一看。

## 顶级项目管理工具

### 大西洋 JIRA

[![Atlassian JIRA dashboard](../Images/1cec84a5225b6c6148d1652b9323c208.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Jf11TsqL--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/Agile-board.png.png)

*来源:亚特兰蒂斯 JIRAT3】*

[【JIRA】](https://www.atlassian.com/software/jira)是专门为软件开发团队和敏捷项目管理设计的项目管理工具。事实上，它可能是市场上最好和最全面的。

你可能不知道或不理解诸如"[](https://www.romexsoft.com/blog/agile-2-0/)"或敏捷框架如" [Scrum](https://www.scrumalliance.org/why-scrum) "或" [看板](https://leankit.com/learn/kanban/what-is-kanban/) "之类的术语，但是你的远程开发团队会的。它可能已经使用 JIRA 无数次了。对你来说，这个工具的美妙之处在于它提供了你的项目的完整画面，谁在做什么，以及他们如何合作来完成它。如果您的远程项目经理正在使用这个团队管理软件，那么您会得到很好的帮助。

想想这个:JIRA 被 Square、易贝、思科和 Airbnb 这样的公司使用。

### Trello - 顶级项目管理工具之一

[![Trello dashboard](../Images/96a92f3bf6ac5065868696a0da815a1e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--s3Chs1-8--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/fab67db10bfbd3b940e94b1b155d5fbd.jpg)

来源:特雷罗

您可能熟悉 [【特雷罗】](https://trello.com/) ，但如果不熟悉，它也是内部团队项目工作中最常用的协作工具。任务被列出和委派，用任务卡设计的板子被形象地描绘出来。每个人都可以看到其他人已经完成了什么，他们目前正在做什么，以及预计的完成时间表。在软件开发中，一切都必须按顺序进行，而特雷罗让每个人都保持同步。

此外，你还可以用它来管理你自己的杂务，并为你需要完成的各种行动创建卡片，以推动项目进展——例如对最近的演示提供反馈，发出设计的品牌指南等等。

### 松弛

[![slack example](../Images/f79abd6398d7a84d2beef38c228c072e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--5aeXKus---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/slack-webdesign.jpg)

*来源:Slack*

这是一个非常棒的基于云的日常交流工具。发送直接消息、拨打电话、放下文件、图像、文档等。直接参与分享，并为他人的交流添加评论。

[](https://slack.com/)之所以成为如此受欢迎的工具，一个原因是可以为每个项目或小组设立单独的渠道。消息可以发送给整个团队、少数人或个人。至于远程团队的工具，这是一个伟大的工作聊天应用程序。

### Skype

[![skype view](../Images/a816b254afbbe9b1a6006fef1e600035.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--OMgdRJoH--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/fa34514_1-1.png)

*来源:Skype*

本地、国内和国际的在线交流变得简单。[【Skype】](https://www.skype.com/en/)可将视频、音频或消息集于一身。无论您是需要单独与远程项目经理进行会议，还是需要在会议环境中与整个团队进行会议，这都是顶级的企业远程协作工具之一。你可以在任何微软支持的设备上使用。

### 谷歌驱动

[![google drive dashboard - top project management tool](../Images/c0dde8dab3009f44b678aa06cf49e444.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--EF9sclLj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/unnamed.png)

*来源:Google Drive*

这就是你对在线文件存储的回答，你可以通过电脑、平板电脑或智能手机访问这些文件，并与你喜欢的人分享。存储法律文件和任何可能添加的修订；存储图像、图纸、设计，甚至录音和视频。

对于您和远程团队来说，这真是太棒了，尤其是当每个人都不在同一个位置并且需要访问文件或文档时。这就是技术人员的特点——他们可能在当地的咖啡店或凌晨 3 点有更高的生产力。从 [谷歌硬盘](https://www.google.com/drive/) 访问工作文件，这不成问题。

### bit bucket——顶级项目管理工具之一

[![bitbucket dashboard - top project management tool](../Images/b502207268e86bee827c15730153ddf4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--b6NU8DFc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/bitbucket-overview-pull-request.png)

*来源:比特桶*

【Atlassian(上面提到的 JIRA)的另一个产品，[](https://bitbucket.org/)是用于存储源文件的解决方案，通过许可共享它们，并允许团队成员在开发过程的每一步进行协作，从编码到部署。

它是基于网络的，因此用户可以从任何地方访问它，并且适用于使用两个版本控制系统的项目——Mercurial 或 Git。虽然您不会理解这个工具的技术方面，但可以说开发人员喜欢它，因为它安全地存储他们的源文件，并允许从任何地方进行协作。

### GitHub - 顶级项目管理工具之一

[![github dashboard](../Images/05cea4d0cbce4e004d176a8a2ddd71a5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--HjRhRCUP--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/repository.png)

*来源:GitHub*
[![](../Images/2970f7e05e5d7db6c5fe6b5f6d598d87.png)](https://www.romexsoft.com/contact-us/)
[GitHub](https://github.com/)是 Bitbucket 的近亲。这是开发人员用来存储、共享和协作源代码文件的另一种流行服务。你可能知道 GitHub 是一个流行的开源平台，但它也允许存储收费的私人项目，不像 Bitbucket 那样按用户收费。

此外，这款应用还能与 Jenkins 等 [流行的自动化测试工具](https://www.romexsoft.com/blog/testing-tools/) 有效同步，这意味着您的开发人员可以同时编写和检查代码。

顺便说一下，在 Romexsoft，我们在不同的项目中使用两个选项，Bitbucket 和 GitHub。如果您不知道根据您的需求选择什么，我们可以很容易地为您提供建议。

### 香脂

[![Balsamiq example - top project management tool](../Images/b98dab0af8dd713c6eb9275c97f7da05.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Ds_a5Eyu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/wiki-sketch.png)

*来源:Balsamiq*

线框图是开发项目早期的一项任务。基本上，它是内容和功能的视觉布局。 [开发人员可以根据用户需求和导航，勾画出他们的想法，删除行不通的，替换，改造，并提出他们的最佳解决方案。无论何时你需要向你的团队说明某些要点，勾勒用户流程或者对设计给出一些反馈，你都可以这样做。Balsamiq 用起来真的很简单。](https://balsamiq.com/products/mockups/)

### Screencast-O-Matic

[![screencast-o-matic view](../Images/cf72d1c10a10871bb97f572668309482.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--NmMrwYvr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.romexsoft.com/wp-content/uploads/2017/03/screencast-o-matic-02-700x484.jpg)

*来源:Softonic*

当您或团队中的某个人想要阐明概念、创建媒体并与其他团队成员协作共享时，在获得许可的基础上，[Screencast-O-Matic](https://screencast-o-matic.com/)是一个很好的选择。在他们自己的屏幕上工作，个人可以捕捉他们拥有的任何数字内容，然后分享给其他人进行分析和回应。这是一个很好的存储解决方案，它还可以归档每一件作品和每一次修改。
