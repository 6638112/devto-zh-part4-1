# 科技“通才”的概念已经消亡了吗？

> 原文：<https://dev.to/shroudedmoon/is-the-idea-of-an-tech-generalist-dead-3k3j>

我在科技行业已经有 20 多年的经验了。其中 18 年是在一家大型财富 500 强公司。我一直认为自己是个多面手，而不是专家，因为我总是有很多事情要做。

例如，有一次，我做了以下事情(还有其他事情):

*   管理遗留应用程序的中间件和服务器
*   遗留应用程序的编码更新
*   管理 3 个开发团队
*   设计和编码遗留应用程序的替换
*   设计和实现 Azure 基础设施
*   为另一个项目设计和实现 AWS 基础设施

看着新的角色，似乎一切都非常专业化，我没有看到我过去担任的这些类型的通才角色。我猜这是敏捷/devops 思维方式的结果，这种思维方式并没有融入我的团队。

很想听听其他人的看法，以及多面手职业中是否有生活。
