# RxJS 学习悬崖与开发者心理健康

> 原文:[https://dev . to/oleks andr/rxjs-learning-cliff-and-developers-mental-health-6gf](https://dev.to/oleksandr/rxjs-learning-cliff-and-developers-mental-health-6gf)

当我决定改变我的职业道路，从高级支持工程师角色转变为初级开发人员时——命令式编程和 OOP 是我非常了解和理解的东西。在开始的时候，一切都很好，直到我进入大量使用 RxJS 的项目——对我来说是全新的东西。当我在攀登那个悬崖时，后来当我已经决定创建一个关于那个图书馆的视频课程时，我发现努力学习会产生这样的效果，你可能在精神上感觉不好。有人称之为精神问题。

这里有一些我在学习 RxJS(或者只是学习一些新东西)时发现有用的预防措施。

# [](#1-start-with-a-limited-number-of-topics)1。从有限的几个话题开始

如果你开始学习新的东西，你会被你必须阅读的各种各样的东西淹没。在 RxJS 的情况下，有许多函数和操作符可以学习——所以有时你可能会想‘我永远也学不会所有这些东西！’并为此感到不安。

但是这里的经验法则是——将范围限制在一些核心主题上(最多 5-7 个),并且只开始学习它们。

在 RxJS 情况下，可以是:

*   什么是可观察的
*   映射、过滤、区分和更改运算符
*   combineLatest 和 withLatestFrom 函数 Thas it。

# [](#2-repeat-and-continue-learning-by-small-parts)2。通过小部分重复和继续学习

不要以为一口气吞下一些视频课程就能学到所有东西。虽然我从这种方法中学到了很多——RxJS 不是一条容易抓到的鱼。所以只要坚持下去，一小步一小步地学习。

# [](#3-one-coffee-cup-a-day)3。一天一杯咖啡

嗯，这可能是非常主观的建议，但是你喝咖啡越多，你的精神活动就越活跃——一天结束时你就越感到疲惫。而且在某个时候，我发现不值得，还会造成情绪不稳定。现在每天喝 1 杯咖啡，感觉还挺好的。如果我累了，我需要休息，但不是一杯新咖啡。

# [](#4-change-the-context)4。改变上下文

通常，如果我完全沉浸在编码中，无法停止思考它，对我有帮助的是——我必须做其他事情:与人交谈，慢跑，与孩子玩耍，社会志愿活动(你转向其他人的问题，这将你的思想从编码中解放出来)，在街上散步。最有帮助的是慢跑和一些手工(我也有家务要做——一些不太重要的事情，如果你有时间可以做——修理，打扫，等等..)

跑至少 3 公里完全消除了任何思维停滞状态，我感觉有能量继续编码。

# [](#5-dont-change-the-context-too-often)5。不要太频繁地改变上下文

这看起来可能有争议，但事实并非如此:-)
除了作为一名开发人员，我还在 codementor.io 上做导师，并阅读和撰写 Angular 和 RxJS 的文章。所以有一天你有工作要做，两个不同的导师调试会议，这意味着你必须切换很多。在第三次这样的转换之后，我的头脑被头痛困扰着。所以现在我计划一天不超过两个这样的大型活动。如果我有工作要做——不超过 1 次额外的辅导。

另一件可能损害你内心稳定性的事情是——试图阅读你智能手机上的所有信息。我删除了其中一些，并经常打开静音模式——整个世界都偏离了它的轴心。

# [](#6-driving-in-silence)6。安静地开车

我现在每周通勤时间约为 7-8 小时，能够安静地开车是我心理健康自我治疗的一部分。只管开车——其他都不重要。

事实上，即使不开车，坐在一个没有人的森林里，静静地盯着我的前方也会有所帮助:-))

# [](#7-allow-yourself-not-to-know-something)7。允许自己不知道一些事情

这是我陷入的一个心理陷阱:一种恐慌，认为技术发展如此之快，我必须学习更多，否则我就会失去资格！

这导致了某种程度的永久焦虑，直到我发现了丹·阿布拉莫夫的一篇好文章:“[截至 2018 年](https://overreacted.io/things-i-dont-know-as-of-2018/)我不知道的事情”。

花一周时间什么也不学。看到了吗？地球还在转，你还有工作。拥有技术，不要让技术拥有你。

# [](#8-community-is-good-but-dont-overestimate-its-meaning)8。社区是好的，但不要高估它的意义

我遇到的另一件事——我发现 Twitter 上充满了有趣的人，他们发布了许多关于前端的有趣的事情。我花了很多时间去读完所有的书，但是后来我意识到读这些书会让我完全不知所措。明智地使用社区——你必须工作，你必须学习，如果你遇到问题——问社区。不要把所有的时间都花在阅读所有的东西上。不读书让你成为专家——编程让你成为专家。

# [](#9-be-thankful)9。心怀感激

即使没有编码，生活也是美好的:-)编码是一个中介目标，但不是一种生活感。
