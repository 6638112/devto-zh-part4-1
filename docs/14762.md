# 对于软件开发来说，什么样的“老方法”会让初学者感到惊讶/有趣？

> 原文:[https://dev . to/willameaseares/what ' s-a-old-way-of-doing-things-it-d-be-surprise-interest-for-beginner-what-it-to-to-willameaseares-15df](https://dev.to/willamesoares/what-is-an-old-way-of-doing-things-that-it-d-be-surprising-interesting-for-beginners-when-it-comes-to-software-development-15df)

和往常一样，正如我所料，我在工作中不断意外地学到一些令我兴奋不已的东西。最后一个例子发生在一次会议中，一个开发人员必须在呼叫中将他们在文件中所做的更改发送给另一个开发人员，我很好奇地看到他们运行了一个`git diff`命令并将输出发送到一个文件中。我无法猜测他们会如何处理这个名为`changes.patch`的文件。

然后他们说:“只要拿着这个文件，在上面运行一个`git apply`就可以了”。

另一个戴夫回答道:“就像以前一样……”

我只是在头脑清醒的时候观察...

问题是我学习了很多 Git，我知道开发人员过去常常通过电子邮件共享文件，只要他们想与团队共享更改，但是看到一个`git diff`的输出可以这样使用，真是令人惊讶。

因此，我想知道人们有什么相似的故事可以分享，以及他们是否希望现在的事情是这样做的。
