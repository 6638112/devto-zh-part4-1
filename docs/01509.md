# 你已经完成了训练营。你能被雇用吗？

> 原文:[https://dev . to/256hz/so-you-ve-finished-boot camp-are-you-employee-1n9l](https://dev.to/256hz/so-you-ve-finished-bootcamp-are-you-employable-1n9l)

[![Brian, from Life of Brian](img/34d01f297cdb8b4d85d963516cf1d1d0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ltxvJ-Gr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/or51eryclkxlpqrld9ns.jpg)

我就在那里——汗流浃背，衣衫褴褛，目瞪口呆，骄傲自大；经过 15 周的强化网络开发培训，我做到了。我向自己，甚至其他人证明了我理解编程的基本原理，并在两种语言上达到中等水平。我已经在两个主要框架中部署了应用程序，并且掌握了它们的 DSL。我在 Github 上提交了一千多个。我是可以被雇用的，而且马上就会有工作！

无数次拒绝之后，我可以告诉你，在一个竞争激烈的市场中，情况根本不是这样。然而，在过去的几周里，我越来越有兴趣了。我已经进入了多家公司的第二轮和第三轮，今天我接受了第一选择的邀请。据我所知，这有几个原因，我想给你们这些刚毕业的学生详细说说。(此外，如果没有训练营，我永远不会走到这一步，所以这绝不是该计划的替代品。把这些看作是把你和其他不愿意付出努力的毕业生区分开来的策略。)

## [](#specialize-)专攻。。。

[![Be a specialist.](img/e5ed3c6b8f51d646acc5dfff45f6618b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--YRa7Sc4k--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/962jwz2j9v7v52dlktrd.png) 
如果有任何语言或框架或工具或你到目前为止接触过的任何东西让你更感兴趣，那就一头扎进*那个*。我选择了 frontend，特别是 React/React Native，因为我对这些最感兴趣。人们喜欢看到你能真正学好东西——你可以教授技能，但没有真正深入的动力。保留你简历和技能清单上的所有内容，但是从你选择的专业开始。我受雇于一个前端角色，但我对后端角色感兴趣，一家公司说这是因为我表现出的反应能力。

## [T1。。。但是，也可以尝试不同的堆栈](#-but-also-try-a-different-stack)

[![Pancakes. Photo by Luke Pennystan on Unsplash](img/bace5cb2179f51f10cede59c38b44e5f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--buK5qPUy--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/lo2771o5bj2ug8varbbh.jpg) 
我在学校学过 Rails & React。后来我试着去了 MERN，以确保我能把我的训练应用到新技术中。尽管我现在对 Angular 有点厌恶，但在这些新技术中融入世界并不太难，这在我的简历上看起来很棒。这也强化了你快速学习新事物的想法，不需要 9 个月的入职培训。

## [](#test-ya-damn-code-and-write-testable-code)测试丫该死的代码(并写出可测试的代码)

[![You're a scientist now.  Follow proper testing procedures.  Photo by Trust "Tru" Katsande on Unsplash](img/7e9e0329b2da56d0f7507817faa8dfa3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--5qIojMv_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6c31hgjb3g9ofrwj4kcn.jpg)

有两次我被要求对我的 Github repos 缺乏测试发表评论，我在两次采访中都没有进步。这也是我没有通过第一次带回家的代码挑战的主要原因。巧合吗？我认为不是。

在学校，我们所有的实验室都是围绕通过 Rspec 或 Jest 中的自动化测试而建立的。接触测试驱动开发？检查！他们没有教我们的是自己写测试。我们中的许多人离开时都觉得这是一个盲点，一个需要独自应对的可怕的盲点。你猜怎么着对于 intro 项目，进行一些单元测试是非常容易的。

测试在现实世界中无处不在，即使公司没有测试他们 50%的代码库，他们也希望你觉得他们应该这样做。幸运的是，当很多开发人员都需要做某件事情时，一些人会做繁重的工作，所以这对我们其余的人来说很容易。现代测试工具被设计成很容易学习。

学习编写容易测试的代码有点难。回到你的一个 bootcamp 项目，为每个特性编写详尽的单元测试。有些事情会很容易-你的链接导航到正确的网址吗？但是你很快就会发现你的部分代码是不可能单独测试的。亲爱的读者，这是不好的。从这个优秀的博客中了解更多信息。

## [](#data-structures-and-algorithms-dsas)数据结构和算法(DSA)

在找工作的过程中，你很可能会被要求解决一个算法问题——一种谜语。各个级别的开发人员都在喋喋不休地说，面试过程的这一部分被打破了。说真的，你不会每天都在解小谜语。当我经历熨斗学校的时候，我们仅仅触及了表面；然而，他们是标准 CS 学位的一部分，知道 DSAs 经常被认为是 CS 学生比新兵训练营的学生更受欢迎的原因。

在我看来，学习 DSA 是非常宝贵的——能够在压力下选择合适的 DSA 确实证明了这一点。每个开发人员的工作不仅仅是制定解决方案，还包括制定*高效的*解决方案，以及*高效地制定高效的解决方案*。如果你能很快地将任何问题分解成最有效的 DSA，你就能专注于更大的设计模式，或者解决其他低级开发人员代码中的低效问题。你会成为一个更有效率的程序员。

也就是说，对于初级职位来说，我得到的问题还不算太糟。我在 Leetcode 上进行[简单面试，直到我能理解每一个问题，并且涵盖了我见过的 90%的问题。其他人的工作可能在初级以上。](https://leetcode.com/explore/interview/card/top-interview-questions-easy/)

看看我聪明的对眼朋友 Rylan Bauermeister 的作品:[二叉树](https://medium.com/swlh/understanding-data-structures-binary-search-trees-a6612daf00dd)，[链表](https://medium.com/@rylanbauermeister/understanding-data-structures-linked-lists-9ca06e2a711a)。[也从零开始学习实现一个 hash 类](https://www.toolsqa.com/data-structures/hash-tables-in-data-structures/)。这出现了很多次。

## [](#get-good-at-vanilla)擅长香草

[![Ice cream.  Photo by sheri silver on Unsplash](img/d64b7356f48bee50b1c03a2e89e18a0b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--nPiVN_Hk--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/wtd4osz5id3x2in6bjtf.jpg) 
我先用 CSS 的例子。在学校的时候，我主要使用 Bootstrap 和 Semantic 之类的框架来设计我的项目，因为它们可以快速部署并且看起来很好。在早期的一次采访中，当我将可堆叠列列为语义这样的重量级框架的一个好处时，我就被这个问题抓住了。这表明我从未涉足 Flexbox，对现代 CSS 功能一无所知。

因此，我从一个电子商务网站随机取出一个基于卡片的页面，并用 React 和 vanilla CSS 重新创建了它。我花了大约三天时间。你知道你可以在普通 CSS 中使用变量吗？在一张有一个属性的卡片的四面都显示一个均匀的边框怎么样？使用`calc`使你的规模动态化(或者任何动态的东西，真的)？现在我知道了。并且知道我可以在没有框架的帮助或加载的情况下做到这一点，这让我作为一名前端开发人员更加自信。

## [](#drink-from-the-source)饮水思源

[![Get your face right in there](img/d061a34d1f60c1aaf7c2b189e744779f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--RW8wFZci--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6azhh8wn4vrdv5xpiext.png) 
拿一个你已经熟悉的框架，想出一个让你难以正常工作的特性。(哪个不是超级重要。)现在打开它的 Github repo，看看它实际在做什么。通过向它传递不同的输入来确认它。修补匠！一个广泛使用的开源框架的源代码很可能是为了可读性而格式化的。也许这个特性没有被很好地记录下来——贡献给 README 吧！我保证这会让你更自信。我在一个谷歌云文档上获得了公关批准，这对我的自尊产生了奇迹。(超好的乡亲，btw！)

同理，学习你的语言的编译器也会让你变得更好。例子:[在 JS 中创建相同的对象模型可以使搜索这些对象的速度提高 7 倍](https://www.freecodecamp.org/news/javascript-essentials-why-you-should-know-how-the-engine-works-c2cc0d321553/)。

在我们这个层次，你不应该被期望从头到尾了解框架源代码或编译器。你应该表现出你知道他们在那里，并且愿意用他们的方式思考。

## [](#the-dreaded-behavioral-questions)令人畏惧的行为问题

[![Where's Your Head At?](img/5ef4c5279dd9c434d07e41668ea9c137.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--a0pqs3LC--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yhwi3llqlmfjnvedoz53.png) 
没有办法绕过这些。你几乎每次都要回答“那么，告诉我你和同事意见不一致的时候”。查阅[明星](https://www.thebalancecareers.com/what-is-the-star-interview-response-technique-2061629)或[卡尔](https://www.ed.ac.uk/reflection/reflectors-toolkit/reflecting-on-experience/carl)回答这些问题的技巧，写下提纲，然后练习。重写并练习，直到你能简明扼要地回答，并表明你(a)是一个人，(b)能以富有成效的方式交流情绪激动的情况，(c)不会是一个可怕的合作伙伴，以及(d)关心做些有益于你周围人的事情。他们只想知道这些。这很合理，对吧？我是说，我想和他一起工作。

## [](#your-cover-letter-amp-resume)你的求职信&简历

当投递一份冷冰冰的申请时，写一封大约三句话长的求职信。提及招聘启事中的一些内容，以及为什么你会被它吸引。正如《洋葱》的天才们曾经宣称的那样，“学生感谢教授教他假装在乎。”不要在网上跟踪每个员工，也不要记住每个补丁笔记。他们只需要知道你愿意让他们成为你身份的一部分。

在简历的一小段中，赞美你自己——不仅仅是你的编码能力(这很重要！)，而是你从其他生活经历中收集到的所有技能。当然是用专业的语言。任何可以被理解为项目管理、团队之间的交流、创造性和技术问题解决的事情——这些都对你的新事业有帮助。在过去的工作中也要强调这几点。

找到你生活中的节点，让你看起来这一直是你的最终目标，你会在这方面表现出色。他们在那里，我保证。从训练营毕业证明你可以快速吸收大量信息！如果你在这份工作的技术层面上缺少经验，这可以让招聘人员度过难关。我认识一个开发人员，他用 Python 完成了整个训练营，之后通过强调这一点，他得到了一份 Ruby 的工作。
