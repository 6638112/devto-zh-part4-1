# 作为一个绝望的没有安全感的程序员

> 原文：<https://dev.to/maxwell_dev/on-being-a-hopelessly-insecure-programmer-31a3>

[2019 码地大会](https://codelandconf.com/)就在明天。兴奋之余，我一直在思考我的职业生涯。

随之而来的想法并不愉快。

自从找到一份真正的编程工作后，我对自己的职业生涯一直没有安全感。无论来自朋友、同事、同事或其他任何人的话语有多少都无法安抚它。总是有那种恐惧爬在我的脑海深处，害怕我会缺乏编程技能、软技能、网络技能、学习技能或其他任何东西，有一天这一切会把我拖垮。

有几次，我感觉到了轻微的恐慌发作。只是看到别人的成功让它爆发。我意识到我一直在孤立自己，只是为了避免重新打开那些感觉。

显然这对我的事业和个人生活都不是最好的事情。

## 看到不安全感的另一种方式

我一直在尽我所能对抗这种感觉，并把它赶走，但运气不佳。因此，我没有试图移除那些决定留下的东西，而是试图从另一个角度来看待它。

我对自己的基础编程知识没有信心。所以我研究并记录了一些基本的 JavaScript 概念，比如 currying 和类型强制。

我对自己的 Ruby on Rails 知识没有信心，因为它在我目前的工作中用得很多。所以我浏览了 Ruby on Rails 教程，并重读了文档，以查看所有的部分以及它们是如何组合在一起的。

我对不理解我的同事所做的代码更改感到不安。因此，我开始阅读他们的拉取请求，并试图了解他们做了哪些更改以及原因。有什么不明白的我以后会查的。

对于收到关于我不太了解的主题或工具的反馈，我没有安全感。所以我在修复的时候对它们做了更多的研究。我最近得到了关于 Rails 中验证的反馈，所以我仔细阅读了 ActiveRecord 和 ActiveModel gems 以及它们的功能。

我对自己的软技能没有信心。因此，我买了一本《烧掉你的作品集》,开始阅读并记下它和它推荐的另一本书。我也已经记下了许多小方法，可以让我更好地与我能评估的人交谈。

我对自己的职业道路没有安全感。因此，我试着阅读其他专业人士的资料，并与他们聊天，看看他们的职业道路是怎样的。我看到他们所有的道路是多么的不同，起起落落，他们处理的斗争，以及他们的不安全感。

## 学会管理不安全感

我没有安全感，也很害怕，但痛苦的是，我把它视为一种力量。这让我不断前进，努力做得更好。不过，我很难控制住这种感觉。有时候我太焦虑了，我想蜷缩起来，停止尝试。

这些感情在可控范围内值得吗？高到足以推动我前进，但又低到不会麻痹我？值得奋斗吗？

我不知道这个问题的答案。但不幸的是，我看不到这种感觉会很快消失。所以不管怎样，我最好学会如何管理它。
