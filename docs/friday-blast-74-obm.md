# 星期五爆炸#74

> 原文：<https://dev.to/horia141/friday-blast-74-obm>

[成为首席工程师(2019)](https://blog.dbsmasher.com/2019/01/28/on-being-a-principal-engineer.html) -许多科技公司在个人贡献者的职业阶梯上有一个“首席”角色，通常很高。作者描述了在他们公司的这个层次上运作意味着什么，但我认为它可以很好地全面翻译。

[日志 vs 结构化事件(2019)](https://charity.wtf/2019/02/05/logs-vs-structured-events/)——我去过一些地方，在那里我们对我们记录的东西非常吝啬，在那里我们记录*一切*。肯定更喜欢后者——作者称之为结构化事件。现代存储系统在处理非结构化日志方面做得足够好，所以没有理由不充分利用它们！

[不要跟着太阳走(2019)](https://lethain.com//dont-follow-the-sun/)——关于建立随叫随到的建议。更准确地说，威尔(我正在读他的新书，不久将会评论)提出，随叫随到应该是一件 24/7 的事情，而不是在地理上分散的团队之间朝九晚五的事情，因为它首先将 it 变成了“必须受到控制”的事情，而不是持续的火灾，其次，它减少了交接错误(这就是为什么，不管是好是坏，这就是为什么你会看到医学上疯狂的转变)。

[让你的工作得到认可——写一份吹牛文档(2019)](https://jvns.ca/blog/brag-documents/)——我已经这样做了一段时间，但我喜欢这个标题，还有一堆额外的东西在那里让它变得更好*。*
