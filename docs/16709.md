# Devart 的数据访问组件现在支持 Lazarus 2.0.2

> 原文:[https://dev . to/devart software/data-access-components-from-devart-now-support-Lazarus-2-0-2-5 fgj](https://dev.to/devartsoftware/data-access-components-from-devart-now-support-lazarus-2-0-2-5fgj)

Devart 的 Delphi 数据访问组件的新版本带来了许多更新。其中之一就是期待已久的对 Lazarus 2.0.2 的支持。

[![Delphi Data Access Components](../Images/caec5b702d700da1d80f87770e7f7f39.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--NtOy6Bbo--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://i.ibb.co/3fGndzj/DAC.png)

数据库管理软件的公认供应商 Devart 宣布了其 DAC 组件的新版本。更新后的工具支持 Lazarus 2.0.2，并提高了 TVirtualTable 的 SaveToFile/LoadFromFile 方法的性能，以及一系列其他重要更新。

**这些是值得特别关注的变化:**
——支持免费的跨平台 IDE Lazarus 2 . 0 . 2；
-实现整个产品线的 DefaultSortType 属性；
-改进了 TVirtualTable 的 SaveToFile/LoadFromFile 方法的性能；
- URL 风格的连接字符串格式和与 IBDAC 6.4 的本地连接；
- PgDAC 5.4。在批处理操作中引入函数调用；
-在 SDAC 8.3 中，多个线程现在可以共享一个连接；
-LiteDAC 3.5 中的直接模式已升级至 SQLite 引擎的 3.27.2 版本。

数据访问组件允许在 Windows、Linux、macOS、iOS 和 Android 上使用 Delphi、C++Builder、Lazarus(和 FreePascal)开发多平台应用程序，适用于 32 位和 64 位平台。这些工具提供了对流行数据库的直接访问，如 Oracle、Microsoft SQL Server、MySQL、InterBase、Firebird、PostgreSQL、SQLite。

如需此版本更新的完整列表，请访问官方 [DAC 页面](https://www.devart.com/dac.html)。
